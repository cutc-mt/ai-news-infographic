OpenAI Codex-5 完全解説

2025年9月20日

Codexとは？

  • 開発でよく利用されるAIエージェント。
  • 競合はClaude Code, Gemini CLI, Cursorなど。
  • 単一のツールではなく、複数の形態で提供されるのが特徴。
Codexは一つのツールじゃない！
CLI、クラウド、VS Code拡張機能など、様々な顔を持っています。

なぜ今Codexなのか？

  • 競合のClaude Codeが利用制限などで勢いを失っている。
  • Codex自体の開発スピードが爆速で、毎週のようにアップデート。
  • ChatGPTサブスクリプションの範囲内で利用可能になり、コストパフォーマンスが向上。
  • ChatGPT本体とは別の利用制限枠があり、実質的にお得。

Codexの3つの顔

  • ターミナル (CLI): ローカルPCで使用する最も一般的な形態。
  • エディタ (VS Code): VS Code拡張機能として、コーディング環境に直接統合。
  • クラウド版: Webサイトから利用。OpenAIのリソースで動作し、GitHub連携も強力。
これら3つを連携させることで、開発体験が劇的に向上します！

モデル性能比較

  • CodexはGPT-5およびGPT-5-Codexモデルを利用。
  • ClaudeはOpus 4.1やSonnet 4、Geminiは2.5 Proと比較される。
  • コーディングにおいては、多くの場面でGPT-5がプライマリーな選択肢になりうる。
モデルの性能が非常に高く、特にコーディングタスクでその力を発揮します。

Codex CLI入門

  • brewまたはnpmで簡単にインストール。
  • ChatGPTアカウントまたはAPIキーでログイン。
  • 重要なコマンド: 
    codex --resume
codex --search
codex --review
  • /newや/compactでコンテキストを管理。

agents.mdでカスタムルール

  • プロジェクトのルールや指示を記述するファイル。
  • グローバル領域: 全プロジェクト共通のルールを設定。
  • プロジェクトルート: プロジェクト固有のルール。
  • サブディレクトリにも配置可能で、モジュールごとに詳細なルールを設定できる。
「日本語で回答して」「このライブラリを使って」など、自分好みにCodexを教育できます。

VS Code拡張機能

  • VS CodeにCodexを直接統合し、シームレスな開発体験を実現。
  • ローカル実行とクラウド実行を切り替え可能。
  • 複数のタスクを並行して実行し、タスクヒストリーで管理。
  • クラウド実行では、最大4つの改善案を並列で生成・比較できる。

Codexクラウド版の威力

  • PCのスペックに依存せず、OpenAIの強力なリソースで開発。
  • GitHub連携が非常に強力。
  • タスク完了後、自動でプルリクエストを作成。
  • コードレビューボットが自動でコードをチェックし、修正案を提示。

Pro Tips

  • 通知音を設定: タスク完了を音で知らせ、待ち時間をなくす。
  • MCPサーバー化: Codex自体をMCPサーバーとして他のツール（Claude Codeなど）から呼び出す。
`config.toml`ファイルを編集して、Codexをさらに便利にカスタマイズ！ 
# config.tomlに追記
[mcp.servers.codex]
command = ["codex", "mcp"]
