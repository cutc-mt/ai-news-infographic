Claude最強モデル「Fable 5」が復活！
利用枠の賢い使い方を解説

おおたにLab で分かりやすく解説された最新情報をまとめました

2026年7月4日

① Fable 5 の衝撃の復活

  • 2026年7月1日からClaude Fable 5が全球で利用可能に復帰
  • 6月12日の米政府輸出規制による停止から、わずか3週間での復活
  • ProMaxTeamプランのユーザーが対象
6月9日に公開されたばかりなのに、3日で止まって3週間で戻った、激動のドロップよ！

② 停止から復活までの流れ

  • 6月9日 Fable 5 / Mythos 5 を一般公開
  • 6月12日 米政府が輸出管理を発動し全ユーザーからアクセス停止
  • 6月30日 米商務省が輸出規制を解除
  • 7月1日 グローバルで順次復活開始
  • 7月7日 プロモーション期間終了（PT 23:59:59）
7月7日までのたった1週間が重要な体験期間になるのよ！

③ 停止の理由を知る

  • Amazonの研究者がセーフガードを回避する手法を米政府に報告したことが引き金
  • 脆弱性の特定と悪用コードの生成が1件のケースで確認された
  • 国家安全保障上の懸念から輸出規制が発動された
  • Anthropicは「Opus 4.8やGPT-5.5でも同様のことができた」と反論
別にFable 5だけが悪かったわけじゃないらしいの。でも規制のターゲットになっちゃった。

④ 週間枠50%の本当の意味

  • Fable 5専用の無料枠が別にあるわけではない
  • 普段の週間利用枠の中でFable 5に使える割合が最大50%までにキャップ
  • 他モデルと合算して週間制限を超えることはできない
  • トークン消費が既存モデルより大幅に多いという報道も
「50%まで使える」は「50%がすぐ溶ける」と紙一重よ。使いどころを選ぼう！

⑤ 消費速度が速いモデル

  • 公式の通知でも「Fable 5はOpus 4.8よりも消費が速い」と明記
  • 同じ量のトークンを扱っても上限が早く到達する
  • コンテキストを大きく積んだセッションでは枠の減りが顕著
  • 日常のライトなタスクを全部Fable 5に振るのはNG
電池の減りが早いスマホみたいに、あっという間に枠がなくなるから気をつけて！

⑥ 対象プランと使える場所

  • ProMaxTeamは週間枠の50%まで無料利用
  • Enterprise（プレミアムシート）は組織が有効化で同様に利用可能
  • Freeと標準シートは対象外
  • claude.ai / Mobile / Desktop / Cowork / Claude Design など全面対応
API・従量課金は最初から$10/$50課金だから、プロモ対象外なのね。

⑦ Claude Codeで使う方法

  • バージョン2.1.170以降が必要なポイント
  • Team / Enterprise管理者はClaude Codeのみオフにできる
claude --version   # バージョン確認
claude update      # 古ければアップデート
/model             # 起動後モデル切替
Claude Codeからの利用は開発者にとって嬉しい！バージョンだけ気をつけてね。

⑧ 7月8日以降どうなるか

  • Fable 5は週間枠の対象から完全に外れる
  • 以降はUsage Creditsを購入して使う形に移行
  • クレジットとドルの換算レートはまだ未発表
  • 十分なキャパシティ確保次第、サブスク標準機能への復活も目標
月額だけでは触れないモデルになっちゃう。お金を払う価値を見極める1週間ね！

⑨ API料金の比較

  • Fable 5 入力$10 / 出力$50（100万トークンあたり）
  • Opus 4.8 入力$5 / 出力$25
  • Sonnet 5 入力$2 / 出力$10
例: 入力200k + 出力40kの重めセッション
Fable 5 → $4.00
Sonnet 5 → $0.80（5倍差！）
Sonnet 5の5倍のコスト！だから「最難関だけに投入する」が正解よ。

⑩ 強化されたセーフガード

  • 停止の引き金になった回避手法を99%以上ブロックする新分類器を導入
  • サイバー系の高リスク要求を検知すると自動でOpus 4.8にフォールバック
  • 米商務省CAISIの研究者が検証し「extraordinarily strong」と評価
  • 24時間365日の監視チーム体制を構築
元通りじゃなくて「強化版」として帰ってきた。セキュリティは一段階上よ！

⑪ コーディングで「戻される」問題

  • 良性のコーディングでもブロックされる可能性があると公式が自認
  • 認証・認可周りのコード修正中にOpus 4.8に切り替わる例が報告
  • CVE番号を含むログ解析や脆弱性説明の文脈で誤検知
  • Fable 5に頼んだのにOpus 4.8の結果で返ってくる体験が発生
悪意なくてもブロックされるの。分類器が保守的に振る舞ってるから注意！

⑫ Fable 5の得意分野

  • 長期的自律性: 数日〜数週間の複雑なプロジェクトで指示を維持
  • 高度な視覚認識能力: 図表・グラフ・PDFの埋め込みテーブルを高精度で理解
  • 深い研究と高度なデータ分析に特化した設計
  • 自律的なコーディング作業で自身のコードを視覚的に評価
「常用するモデル」じゃなくて「一番難しい問題にだけ投入するモデル」ね！

⑬ モデルの賢い使い分け

  • Sonnet 5 → 日常のコーディング・デバッグの現実解
  • Opus 4.8 → 精度が求められる場面・Fable 5のフォールバック先
  • Fable 5 → 複雑な推論・長文解析・戦略的な設計に限定
  • 誤フラグの報告もあるのでルーチン作業は従来モデルで
3つのモデルを使い分けるのが今のベストプラクティス！

⑭ 利用枠を賢く使うコツ

  • 最重要タスクだけにFable 5を投入して残りは他モデルに回す
  • 週の前半に使いすぎると後半が他モデルも使えなくなる
  • コンテキストを小さく保つと消費を抑えられる
  • 途中でOpus 4.8に切り替わったら諦めて同じタスクでリトライ
限られた枠をムダにしない！重要度ABCでタスクを仕分けしてから使うのよ。

⑮ 中小企業が見ておくべき3ポイント

  • 7月7日までを「試用フェーズ」と割り切る
  • コーディング業務はSonnet 5かOpus 4.8を現実解にする
  • AIモデルの「供給リスク」を意識して複数AIを並行運用する設計を
1社の1モデルに全部乗りはリスキー。Claude / ChatGPT / Gemini を使い分けよう！
