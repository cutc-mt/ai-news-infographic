ACPとは？

  • Agent Client Protocolの略。Zedチームが開発。
  • AIエージェントとエディタ（クライアント）間の通信を標準化するプロトコル。
  • ACPをサポートすれば、どんなエージェントもどんなエディタにも統合可能に。
  • MCP (Multi-Agent Communication Protocol) の概念を再利用し、導入を容易に。

連携のメリット

  • Follow Along: エージェントのファイル編集をリアルタイムで追跡。
  • Granular Changes: コード変更をブロック単位で承認・拒否可能。
  • Task List: エージェントの作業リストをサイドバーに常時表示。
  • Slash Commands: 既存のClaude Codeカスタムワークフローをそのまま利用。

ACPの仕組み

  • エディタとエージェントはJSON-RPCで通信。
  • エディタがファイル情報などのコンテキストをエージェントに提供。
  • エージェントはツールの使用許可をエディタ経由でユーザーに求める。
  • エディタはリンター等の内部機能をProxy MCP Serverとしてエージェントに提供。
少し複雑ですが、エージェントがエディタの機能を「ツール」として使えるようにする賢い仕組みです。

使い方 in Zed

  • エージェントパネルの「+」ボタンからClaude Codeを選択。
  • 「@」シンボルでファイルをメンションしてコンテキストに追加。
  • 「/」でClaude Codeのカスタムコマンドを呼び出し。
  • 設定から他のACP対応エージェントを追加することも可能。

AI開発の未来

  • エディタの垣根を越える: VS Code, JetBrains, Zedなど、好きなエディタで同じエージェントを使える未来。
  • チーム開発の効率化: チーム内で異なるエディタを使っていても、同じエージェントを共有可能。
  • タスクに最適なエージェントを自由に選択し、最高の開発環境を構築。
ACPは、AIエージェント開発の「ユニバーサル・スタンダード」になる可能性を秘めています！

オープンソースと拡張性

  • Claude CodeをACPに対応させるためのアダプタはオープンソース。
  • NeovimのCode Companionなど、他のツールも既に対応を開始。
  • 開発者は自身のエージェントをACPに対応させることで、Zedなどのエディタに簡単に統合できる。
