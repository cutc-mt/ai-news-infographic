Chrome DevTools MCP 完全ガイド

2025年9月27日

概要

  • AIエージェントがChromeを操作・検査するためのMCPサーバー。
  • Web開発者がデバッグやパフォーマンス分析を行うことに特化。
  • AIエージェントフレンドリーな設計で、より抽象的な指示で動作。
Playwright MCPとは目的が少し違う、開発者向けのツールだよ！

主な機能

  • コード変更のリアルタイム検証。
  • ネットワークエラーやコンソールエラーの診断。
  • ユーザー行動のシミュレーション。
  • スクリーンショット撮影。
  • スタイリングの問題を分析。
  • パフォーマンスの計測と分析。

Playwright MCPとの違い

  • Chrome DevTools MCPは開発・デバッグ目的。
  • Playwright MCPは汎用的なブラウザ自動化・スクレイピング向け。
  • Chrome DevTools MCPのツールはより抽象度が高く、目的指向（例：「パフォーマンスを分析して」）。
  • Playwright MCPはツール数が多く、より細かい操作が可能（例：マウスの移動）。
開発タスクならChrome DevTools MCPが便利かも！

使い方

  • NPM経由で簡単にインストール。
  • Claude, Cursor, CodexなどのAIアシスタントと連携。
  • チャットで「このページのパフォーマンスを速くして」のように自然言語で指示。
# Claude Code / Desktop で使う場合
@chrome install
# Cursor / OpenDevin / Codex で使う場合
npm i -g @mcp/chrome-devtools

設定のポイント

  • headless: true にすると、ブラウザUIなしで実行可能。
  • バックグラウンドで動作するため、軽量で開発の邪魔にならない。
  • スクレイピング対策されているサイトでは `false` にする必要がある場合も。
通常は `headless: true` がおすすめ！
