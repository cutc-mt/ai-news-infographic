GPT-5 Codex プロンプトガイド

2025年9月27日

GPT-5 Codexとは？

  • OpenAIが開発したコーディング特化モデル。
  • 最近API経由での利用が可能になり、各種IDEやツールに統合。
  • ソフトウェアエンジニアリングの複雑で長時間のタスクに対応。
  • 少ない指示でも自律的にタスクを計画・実行する能力を持つ。

GPT-5との違い

  • 汎用モデルであるGPT-5の代替品ではない。
  • コーディングに関する知識が事前に豊富に組み込まれている。
  • そのため、プロンプトはGPT-5向けのものよりシンプルで良い。
「あなたは〇〇の専門家です」といった前置きは不要だよ！

アンチプロンプティング

  • Less is More: 過剰な指示は不要。
  • 目的を明確に伝えれば、詳細な手順はモデルが自律的に判断。
  • 過剰なプロンプトは、モデルの自律的な判断力を阻害する可能性がある。
  • マイクロマネジメントする時代は終わりつつある。

推奨技術スタック

  • React & TypeScript
  • Tailwind CSS & ShadCN/UI
  • Lucide React (アイコン)
  • Framer Motion (アニメーション)
  • Recharts (チャート)
モダンな技術を使うと、Codexの能力を最大限に引き出せる！

プロンプトの書き方

  • 何を(What)だけを伝え、どのように(How)はモデルに任せる。
  • 一般的なタスクではステップ・バイ・ステップの指示は避ける。
  • ただし、非常に細かい修正や特定の要件がある場合は、具体的な指示が必要。
/* 悪い例 */
あなたはReactの専門家です。以下のコンポーネントをリファクタリングしてください。まず、コンポーネントを3つに分割し、次に...

/* 良い例 */
このReactコンポーネントをリファクタリングして、保守性を向上させてください。
