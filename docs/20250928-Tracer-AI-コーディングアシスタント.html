Tracer AI: 大規模コードベースのためのAIアシスタント

2025年9月28日

AIエージェントが抱える課題

  • コンテキストの欠如: タスクの正確なコンテキストを与えないと、AIは自らギャップを埋めようとします。
  • 目的からの逸脱: 結果として、AIは本来の計画から逸れ、意図しないものを構築してしまいます。
  • 開発者は、意図を伝えるのではなく、謎めいたコードの差分と向き合うことになります。
この「幻覚」や「目的の逸脱」が、AIエージェント活用の大きな壁でした。

解決策: Tracer AI

  • 計画レイヤー: AIエージェントへの入力を明確にし、幻覚の原因となるギャップを排除します。
  • 行動計画の策定: すべてのタスクを、詳細で実行可能な計画から開始します。
  • コードベースへの接地: 計画が常に実際のコードベースと一貫性を保つようにします。
  • 大規模なコードベース向けに構築された、最先端のAIコーディングアシスタントです。

主な機能①：計画性

  • 抜け穴の検出: コーディング開始前に計画の論理的な欠陥を検出し、ユーザーと協力して解決します。
  • 複数プランナー: 複数のプランナーをバックグラウンドで実行し、作業を高速化します。
  • 自動コンテキスト収集: 未知のライブラリやフレームワークに遭遇した際、実装に必要なコンテキストを自動で収集します。

主な機能②：実行と統合

  • IDE統合: Visual Studio Codeの拡張機能として簡単にインストール可能。
  • エージェント互換性: Claude Code, Cursor, Rube Codeなど、様々なAIエージェントと連携できます。
  • 検証プロセス: 生成されたコードを検証し、修正のためのレビューコメントを提供。AIエージェントと共に修正できます。

2つの動作モード

  • フェーズモード (Phases): AIとの対話を通じてタスクの意図を明確にし、管理しやすいステップに分割します。
  • プランモード (Plan): 詳細なファイルレベルの計画を取得し、AIと協力して洗練させた後、エージェントに実行を指示します。
対話的に計画を練るか、詳細な計画を直接得るか、タスクに応じて選択可能です！

デモ：MUIライブラリの統合

  • 課題: 既存のAIが、新しいUIライブラリ(MUI)の導入に失敗。
  • Tracerの分析: Tracerがプロジェクトを分析し、Reactアプリではないことを特定。Reactへの変換を提案。
  • 段階的実行: タスクを「Reactセットアップ」「コンポーネント変換」などのフェーズに分割し、一つずつ計画・実行。
  • 結果: 意図通りMUIを使用した、クリーンでモダンなReactアプリが完成。
