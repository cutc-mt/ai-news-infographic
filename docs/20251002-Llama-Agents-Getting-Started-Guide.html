Llama Agents 入門ガイド

2025年10月2日

1. 準備：Llama Control

エージェント開発の旅を始めるための最初のステップは、CLIツールLlama Controlをインストールすることです。

  • ターミナルを開いてコマンドを実行します。
uv install llama-control
`uv`を使うと高速にインストールできます！すでにインストール済みの場合は不要です。

2. プロジェクトの初期化

次に、llamactl initコマンドで、対話的にプロジェクトを作成します。

  • CLIの質問に答えてプロジェクトの種類を選択します。
  • UI付きのテンプレート（例：Extraction agent with review UI）や、UIなしのワークフローも選択可能です。
  • 自動的にGitリポジトリとして初期化されます。
テンプレートを選ぶだけで、エージェントの雛形が完成します！

3. ローカルでの実行

作成したプロジェクトディレクトリに移動し、ローカルサーバーを起動して動作を確認します。

cd extraction_review
serve
  • 初回実行時にLlama Cloudへのログインが求められます。
  • ブラウザでエージェントのUIが表示され、動作を確認できます。
デプロイ前に、ローカルでしっかりテストしましょう！

4. GitHubリポジトリ設定

Llama Cloudにデプロイするために、GitHub上に新しいリポジトリを作成し、ローカルプロジェクトと連携させます。

  • GitHubで新しい（プライベートでも可）リポジトリを作成します。
  • ローカルリポジトリにリモートURLを追加します。
git remote add origin [Your-Repo-URL]
git push -u origin main
これで、Llama Cloudがあなたのコードにアクセスできるようになります。

5. Llama Cloudへデプロイ

deploy createコマンドで、エージェントをLlama Cloudにデプロイします。

  • デプロイ名やリポジトリ名を確認します。
  • GitHub Appをインストールして認証を行います。
  • ログが流れ、デプロイが完了するとURLが発行されます。
これで、あなたのエージェントが世界中に公開（またはプライベートに共有）されました！

6. 完成！

発行されたURLにアクセスするか、Llama Cloudのダッシュボードからデプロイしたエージェントを確認できます。

  • デプロイメントURLから直接アクセス。
  • Llama Cloudのプロジェクトページに一覧表示されます。
  • 同名のエージェントがある場合、ユニークなIDが末尾に付与されます。
さあ、作成したエージェントで遊び始めましょう！
