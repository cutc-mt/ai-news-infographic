プラグインとは？

  • パッケージ化された開発キット。
  • 複数の機能を一つにまとめて管理・共有できる。
  • 以下の4つの主要機能で構成される:
  • スラッシュコマンド: 特定の操作を即座に実行。
  • サブエージェント: 特定タスクに特化したAIエージェント。
  • MCPサーバー: 外部ツールやAPIとの連携を担う。
  • フックス: 特定イベント（例：コードレビュー）で自動実行。

マーケットプレイス

  • プラグインを発見・管理するための上位概念。
  • プラグインのカタログのようなもの。
  • Gitリポジトリに place.json を置くことで作成。
  • URLを指定して簡単にインストール可能。
# マーケットプレイスのインストール例
/claude plugin marketplace@https://.../place.json
UIコンポーネントライブラリで例えると...
プラグインがボタンやフォームで、
マーケットプレイスがMaterial-UI全体みたいな関係性だね！

導入のメリット

  • チーム開発の効率化: 開発環境やワークフローを簡単に共有。
  • 「このタスクはこのプラグインを使おう！」といった共通認識が生まれる。
  • カスタマイズと再利用: 便利な設定をパッケージ化して、いつでも呼び出せる。
  • プラグインの有効化/無効化が簡単で、必要な機能だけを使える。

使い方フロー

  1. マーケットプレイスを追加: 
    /plugin marketplace@リポジトリURL
  2. プラグインを検索: 
    /plugin browse
    またはスペースキーで一覧表示。
  3. プラグインをインストール:
    リストから選択してインストール。
  4. 機能を利用:
    追加されたスラッシュコマンドやサブエージェントが使えるように！
  5. 管理: 
    /plugin manage
    で有効/無効を切り替え。
