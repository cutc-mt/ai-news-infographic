Playwright Agents完全解説

2025年10月11日

Playwrightとは？

  • E2Eテストとブラウザ自動化ツール
  • フロントエンドでのユーザー操作をテスト
  • スクレイピングにも利用される
  • Playwright MCPなどAIエージェントの基盤
元々はテストツールだけど、AIエージェントの頭脳にもなってるんだ！

Playwright Agentsとは？

  • E2Eテストを強化するエージェント群
  • 3つのサブエージェントで構成
  • Claude Code, VS Code, Open Codeで利用可能
テストを賢く、自動で生成・修正してくれるんだね！

3つのエージェントの役割

  • Planner: ユーザーの操作に基づきテストケースを作成
  • Generator: テストプランをPlaywrightコードに変換
  • Healer: 失敗したテストコードを自動修正
計画、生成、修復のサイクルでテストを自動化！

フォルダ構成

  • specs/: テストの仕様（計画）を格納
  • tests/: 生成されたテストコードを格納
  • playwright.config.ts: Playwrightの設定ファイル
// playwright.config.ts の例
import { defineConfig } from '@playwright/test';

export default defineConfig({
  testDir: './tests',
  /* ... */
});

Seed Specの役割

  • seed.spec.tsがテストの起点
  • ログイン処理などの初期設定を記述
  • グローバルなセットアップを実行
ここからテストの魔法が始まるんだ！

使い方

  • Webアプリのテストシナリオを考案
  • Plannerがシナリオを生成
  • Generatorがコードを生成
  • Healerが必要に応じてコードを修正
開発者はテスト作成の手間を大幅に削減できる！
