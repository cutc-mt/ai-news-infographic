OpenSpec: スペック駆動AI開発のための新しいツールキット

2025年10月12日

OpenSpecとは？

  • スペック駆動開発: コードを書く前に、人間とAIアシスタントが「何を構築するか」を合意するためのツールキット。
  • 軽量: Spec Kitよりも軽量で、APIキーも不要。
  • 確定的: AIの推測ではなく、確定的でレビュー可能な結果を保証。
  • 既存システムの変更に強い: 現在の仕様と提案された更新を明確に分離し、変更の追跡と監査を容易にする。
新規プロジェクトだけでなく、既存プロジェクトの変更・更新にも強いのが大きな特徴です！

サポートツール

  • Claude Code
  • Cursor
  • Open Code
  • Kilo Code
  • Codex
  • Gemini CLI (近日対応予定)
多くの主要なAIコーディングアシスタントが既にサポートしており、エコシステムが拡大中です。

OpenSpecの使い方

  • インストール: `npm install -g openspec-cli`でグローバルにインストール。
  • 初期化: プロジェクトディレクトリで`openspec init`を実行。
  • AIアシスタントの設定: 使用するAIアシスタント（Kilo Codeなど）を選択。
  • コンテキストの入力: プロンプトを使い、AIエージェントにプロジェクトの詳細を`openspec.md`に書き込ませる。
  • 変更提案の作成: 機能要求をAIエージェントに伝え、変更提案を生成させる。
  • レビューと実装: 生成された提案をレビューし、AIエージェントに実装を指示。

Zapierは死なない

  • Agent Kitとの違い: Agent Kitは「回答の最適化」に焦点を当てているが、Zapierは「ワークフローのオーケストレーション」に強い。
  • 8,000以上のアプリ連携: 確実なアクションを多数のアプリを横断して実行できる。
  • MCPとの連携: OpenAIの新しいMCPとも連携し、ChatGPTの回答をトリガーにZapierで自動化が可能。
  • Zapier Cobbot: 自然言語で必要なものを説明するだけで、本番環境で使えるワークフローを構築してくれる。
