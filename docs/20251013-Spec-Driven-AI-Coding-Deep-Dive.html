仕様駆動AIコーディング
Taskmasterによるアプローチ

2025年10月13日

仕様駆動AIコーディングとは？

  • プロジェクト開始前に大量のドキュメントを作成する開発手法。
  • SpeckitやOpenSpecのようなツールを使い、仕様を定義。
  • AIが仕様書を元にタスクリストやマークダウンファイルを生成する。
  • 例：モバイルアプリ開発では、使用するフレームワーク（Expo, Flutter）やAPI（TMDB）などをAIが計画。
理論上は素晴らしい！AIが詳細なプロンプトを自己生成するようなもの。

従来手法の問題点

  • 開発経験者には過剰。自分のコーディング知識を活かせない。
  • AIの提案が最適とは限らない。ExpoとFlutterの違いを知らないと判断できない。
  • 本番環境向けの品質にはなりにくい。「ワンショット」での成功は稀。
  • 「Vibe Coder（雰囲気コーダー）」向けになりがちで、本質的な開発スキルが身につかない。
「AIに全て任せる」アプローチは、特に本番品質を求める場合にはうまくいかないことが多い。

推奨アプローチ: Taskmaster

  • ゼロからのアプリ構築（スキャフォールディング）で効果を発揮するツール。
  • 開発者のパフォーマンス向上を目的とし、開発者を置き換えるものではない。
  • 元々はCLIツールだったが、現在はMCP(Multi-turn Conversation Protocol)として提供。
  • APIキー（Anthropic, Perplexity, OpenRouter）を設定するだけで利用可能。
開発者が主導権を握りつつ、AIの力を借りて効率化するのに最適なツール！

Taskmasterの動作フロー

  • 1. PRD作成: プロジェクト要件定義書（PRD）を自動生成。
  • 2. タスク分解: PRDを解析し、AIコーダー向けの小タスクリストを作成。
  • 3. リサーチ: AIが知らない新しいライブラリ等について調査を実行。
  • 4. 依存関係解決: タスクの複雑性や依存関係を計画する。
わずか2〜3回のMCPコールで、プロジェクトの計画が完了する速さが魅力！ 
# MCP設定のイメージ
- mcp: "taskmaster"
  # APIキーを設定
  api_keys:
    open_router: "YOUR_API_KEY"

Taskmasterの利点

  • プロンプトやタスクに忠実で、脱線しにくい。
  • コンテキストを無駄な情報で溢れさせない。
  • GLMのような計画が苦手なモデルでも、多段階計画により良い結果を出せる。
  • Claude Codeなど、好みのAIコーダーとシームレスに連携。
  • MCPが好ましくない場合、ターミナルCLIとして利用する選択肢もある。
成熟していて、開発者中心。これが推奨できる唯一の仕様駆動開発ツールだ。
