LangChain v0.1.0 アップデート完全ガイド

2025年10月26日

Agentのインポート変更

  • create_react_agentがlangchain.agentsに移動
  • 以前はlanggraph.prebuiltにありました。
  • システムプロンプトのパラメータ名がpromptからsystem_promptに変更。
# Before
from langgraph.prebuilt import create_react_agent

# After
from langchain.agents import create_agent
より直感的になりましたね！

Contextオブジェクト

  • ランタイム引数を渡すためにContextクラスを導入。
  • 辞書形式のconfigを置き換えます。
  • データクラスを使うことで、よりクリーンで便利なコードに。
from dataclasses import dataclass

@dataclass
class MyContext:
    user_id: str

# context引数で渡す
agent.invoke(..., context=MyContext(user_id="123"))

動的プロンプト

  • @dynamic_promptデコレータで動的にプロンプトを切り替え。
  • 実行時の引数（例：ユーザーの役割）に応じてプロンプトを変更できます。
  • Middlewareとして利用します。
from langchain.prompts import dynamic_prompt

@dynamic_prompt
def get_prompt(request):
    if request.context.user_role == 'expert':
        return "You are a deep technical assistant."
    return "Explain concepts in plain language."

Middleware

  • モデル呼び出しの前後でロジックを実行。
  • summarization: 長いメッセージを要約。
  • human in the loop: LLMの出力をユーザーがレビュー。
  • カスタムMiddlewareも作成可能です。
危険な操作の前に確認を挟むなど、安全性が向上します！

AgentState

  • 状態管理を助けるヘルパークラス。
  • messages: メッセージ履歴を自動で収集。
  • jump_to: 特定のノードを決定論的に実行。
  • カスタム状態属性を追加して拡張できます。
エージェントの記憶や状態遷移が管理しやすくなりますね。

構造化出力

  • tool_strategy: ツール呼び出しを利用して構造化出力を生成。多くのモデルで動作。
  • provider_strategy: モデルプロバイダーのネイティブな構造化出力機能を使用（例: OpenAI）。
  • content_blocks: 出力に推論トークンとテキストが含まれるようになり、より柔軟に。

レガシーパッケージ

  • クラスベースの古い高レベルChain（LLMChainなど）は非推奨に。
  • langchain_classicパッケージをインストールすれば引き続き利用可能です。
  • 新しいワークフローはグラフベースのアプローチが推奨されています。
pip install langchain_classic

from langchain_classic.chains import LLMChain
