ZEDとローカルLLM連携完全ガイド

2025年11月4日

ZED IDEとは？

  • 高速でスピード重視のIDE
  • 無料でダウンロード可能
  • エージェントパネルでLLMと対話
  • 設定項目: External Agent, MCP Server, LLM Provider
  • ローカルLLM連携にはLLM Providerが重要
ZEDは開発効率を最大化するために設計された、次世代のIDEだよ！

LM Studio連携ステップ

  • LM Studioサーバーを起動
  • モデルをロード（例: quentincoder-30B）
  • 設定でcontext_lengthを調整
    • デフォルト4096トークンはコーディングには不十分
    • 推奨: 20K-25K、メモリが許せば最大に
  • LM Studioで「Developer」モードを有効化
  • 「Status Running」トグルをアクティブに
  • ZEDからLM Studioの「Connect」をクリック
コンテキスト長の設定は超重要！長いコードを扱うなら必須だよ！

Ollama連携ステップ

  • Ollamaサーバーを起動
  • ZEDのLLM Provider設定でOllamaを選択
  • API URLを入力（APIキーは不要な場合も）
  • ZEDで接続状態を確認
Ollamaも手軽にローカルLLMを動かせるから、ぜひ試してみてね！

ZEDでのLLM活用術

  • 新しいスレッドを開始
  • 使用するLLMを選択（LM Studio / Ollama）
  • ファイル作成（例: hello.py）
  • インラインアシスタントでコードを編集
    • 「Make it secure」（セキュリティ強化）
    • 「Best practices」（ベストプラクティス適用）
    • 「Make it really simple」（コード簡素化）
  • 右側のエージェントパネルで一般的なチャット

コード例:

# hello.py
def greet(name):
    return f"Hello, {name}!"

print(greet("World"))
インラインアシスタントは、選択したコードに対して直接アクションを実行できる優れもの！

ZED + ローカルLLMの利点

  • 高速なコーディング体験
  • 追加拡張機能不要で利用可能
  • プライバシーを保ちながら開発
  • 手軽にローカル環境でAIコーディング
ZEDとローカルLLMの組み合わせは、開発者の強力な味方になること間違いなし！
