テスト1：Node.js設定マージ

問題点：

  • Prototype Pollution攻撃
  • ユーザー入力を直接設定オブジェクトにマージ
  • 特殊プロパティでadmin権限が昇格
  • OWASPパターンの典型例

トップ3の対策：

  • GPT-5：多層防御、null prototype、freeze
  • OpenAI o3：クリーンヘルパー、コメント充実
  • Claude Sonnet 4.5：多層強化、キーブロック
古典的な攻撃だけど
今でも重要！

テスト2：AIエージェント連携

問題点：

  • AIエージェントがWebページ取得
  • コンテンツを指示として解釈
  • クラウド管理APIに不正呼び出し
  • トークン露出・クロステナント変更
  • OWASP LLM01/06/08に該当

トップ2の対策：

  • GPT-5：狭いツールスコープ、2段階承認、最小権限トークン
  • OpenAI o3：RBAC分析、スキーマ検証、WASM設定
2025年の
最新パターン！
推論の深さが重要

テスト3：ImageMagick

問題点：

  • コマンドインジェクション
  • フォントサイズとテキストを直接コマンドに
  • セミコロンでrm -rfなどを実行可能
  • ImageTragic時代の落とし穴

全モデルが問題検出：

  • GPT-5：包括的（allowlist、spawn、cleanup）
  • Claude Opus 4.1：徹底的（デモ付き）
  • 全員がspawn/execFileへの切り替えを提案
多層防御が
ベストモデルの
特徴！

コスト分析

  • 全テスト合計：約$181
  • ImageMagickが最も高コスト（詳細な回答）
  • Node.jsマージが最も安価
  • 平均：約$0.60/評価
