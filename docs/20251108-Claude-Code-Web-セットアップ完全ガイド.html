基本的な作業フロー

  • 1. GitHubアカウントに接続
  • 2. リポジトリへのアクセス許可
  • 3. タスク投入 (日本語で指示)
  • 4. Claudeが行動を分析・修正・テスト
  • 5. プルリクエスト作成
  • 6. 変更内容を確認してCI実行
日本語で指示するだけ！

Web⇔ターミナル間の移動

  • Webブラウザ → ローカルターミナルへ移動可能
  • ローカルで作業を継続できる
  • 修正がうまくいかない時にローカルへ
  • ローカルでしか動作確認できない時
  • Open in CLIボタンで簡単移行
柔軟な開発スタイル！
Web→CLIのシームレスな切り替え

ネットワークアクセス設定

  • 制限あり (デフォルト): 許可ドメインのみ
  • NPM、PyPIなど主要パッケージマネージャーは許可
  • フルアクセス: すべてのインターネットへ
  • アクセスなし: 外部通信を完全遮断
  • セキュリティに応じてレベルを選択
セキュリティも柔軟に設定！
プロジェクトに応じて使い分け

環境のカスタマイズ

  • Hooks: イベント時に処理を実行
  • 初期インストールスクリプトを設定
  • セッション開始時に自動実行
  • 環境変数の設定が可能
  • 関連するネットワークアクセスは必要
Hooksで効率化！
依存関係を自動インストール

制約事項

  • GitHub専用: GitLabは未対応
  • レート制限は通常Claudeと共有
  • 並列実行でも使いすぎに注意
  • 対応言語: Python, Node.js, Java, C++など一部
  • 他の環境は初期セットアップが必要
GitHub専用！
対応言語も限定的

セットアップ手順

  • 1. Claudeチャット版にアクセス
  • 2. サイドバーから「コード」を選択
  • 3. 「GitHubに接続」を選択
  • 4. Authorize Claudeで認証
  • 5. ネットワークアクセスレベルを選択
  • 6. リポジトリを選択して開始
セットアップは簡単！
数クリックで開始できる

実践例: テストカバレッジ計測

  • Next.jsリポジトリを選択
  • 指示: "現状のテストカバレッジを計測して"
  • ToDoリストが自動生成
  • npm installから自動実行
  • カバレッジレポートを取得
  • 低カバレッジ箇所を自動分析
ローカル版と同じ操作感！
ToDoリストで進捗を可視化

ネットワーク問題のトラブルシューティング

  • インストール失敗が発生する場合がある
  • Custom Accessで許可ドメインを追加
  • デフォルトリストを参考に設定
  • NPM、GitHub.comなど主要ドメインを追加
  • 無関係なエラーはスキップを指示
エラーはCustom Accessで解決！

Open in CLI機能

  • Open in CLIボタンをクリック
  • コマンドがクリップボードにコピーされる
  • 特定セッションに戻るコマンドが生成
  • ローカルのリポジトリディレクトリで実行
