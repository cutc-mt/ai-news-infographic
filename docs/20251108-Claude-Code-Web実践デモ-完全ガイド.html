既存リポジトリでの作業専用

環境設定の選択肢

  • Trusted Network Access (推奨)
  • 検証済みソースからパッケージダウンロード可能
  • NPM、apt-getなどが利用可能
  • No Internet Access: ネット接続なし
  • 仮想環境にライブラリがないと厳しい可能性
  • Custom Access: 許可ドメインを個別設定
推奨は「Trusted Network」！
必要なパッケージを自由にダウンロード

Web版の画面構成

  • 左側: リポジトリ選択エリア
  • 中央: プロンプト入力エリア
  • 右下: 可愛い動くキャラクター（タコ?）
  • Defaultボタン: 環境設定変更可能
  • 許可ドメインの設定
  • 環境変数の設定（APIキーなど）

ユニットテスト追加デモ

  • "Reply to Claude"でフォローアップ
  • プロンプト: "ユニットテストを追加して"
  • ローカル版と同様にToDoリストが表示
  • tests/フォルダを作成
  • 1つずつユニットテストファイルを生成
  • pytest実行時にライブラリ未発見エラー
  • 仮想環境を作成してpytestを再実行
  • 最終的にテストが成功
root権限エラーは改善の余地あり！
でも最終的にはちゃんと動く
