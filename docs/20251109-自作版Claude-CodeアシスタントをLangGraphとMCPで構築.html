自作版Claude CodeアシスタントをLangGraphとMCPで構築

2025年11月9日

アーキテクチャ概要

  • オーケストレーション: LangGraph
  • ツール連携: MCPサーバー
  • LLM: Claude Sonnet 4.5
  • インターフェース: CLI (rich)
  • 状態永続化: SQLite
LangGraphでエージェントの複雑な状態を管理し、MCP経由で多彩なツールを活用する構成！

LangGraphの役割

  • 状態管理: ユーザー入力、モデル応答、ツール使用の3つのノードで状態を管理。
  • グラフ構造: ノードとエッジでエージェントのワークフローを定義し、コンパイルして実行。
  • ループ処理: 再帰上限を設定し、エージェントがループ内でタスクを実行。
  • 永続化: SQLite Checkpointingでエージェントの状態をDBに保存。
単なるLLM呼び出しではなく、状態を持つグラフとしてエージェントの思考プロセスを構築できるのが強力！

MCPサーバーとツール群

  • ローカルツール: ファイルの読み書き、一覧、検索、テスト実行など。
  • MCPツール (Web): DuckDuckGoによるWeb検索。
  • MCPツール (GitHub): PATを利用したリポジトリ管理。
  • MCPツール (Sandbox): 安全な環境でのPythonコード実行。
# MCPツールのコマンド例
npx @mcp-tools/duckduckgo-mcp-server
npx @mcp-tools/github-mcp-server
ローカルとリモートのツールをMCPという標準規格で統一的に扱えるのがミソ！
