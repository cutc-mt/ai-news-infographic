効果的なプロンプトの書き方

プロンプト作成の3つの原則

1明確かつ具体的に

  • 対象ユーザー：誰が使うアプリなのか
  • 実現すべき機能：何ができるべきか
  • 必須機能：絶対に必要な機能は何か

2ユーザー体験に焦点を当てる

「どう実装するか」ではなく、「ユーザーが何をしたいか」を説明することが重要です。

3方法ではなく内容を説明する

技術的な実装方法は指定せず、達成したい目的や機能を明確に伝えましょう。

プロンプト例：

「パーソナルトレーナーがクライアントのワークアウト計画を管理し、週ごとの進捗を追跡できるアプリが欲しい」

このシンプルなプロンプトだけで、管理画面とユーザー画面の両方を実装してくれます！

アイデアサジェスト機能

プロンプト入力画面で「More Ideas」ボタンをクリックすると、カテゴリ別のアプリアイデアが表示されます。

使い方：
  1. カテゴリを選択（例：Productivity）
  2. 具体的なアプリ案を選択（例：Markdown to HTML Converter）
  3. 自動的にプロンプトが入力される
  4. そのまま使うか、カスタマイズして送信

学習のコツ

サジェストされたプロンプトを翻訳して読むことで、効果的なプロンプトの書き方を学べます！

スタイリングインストラクション

アプリの見た目を簡単にカスタマイズできる機能です。

使い方

  1. スタイリングインストラクションボタンをクリック
  2. 複数のデザインパターンをプレビュー
  3. 気に入ったスタイルを選択
  4. 詳細なスタイリング指示が自動で追加される
ポイント：

アプリの機能を説明したプロンプトの後に、スタイリング指示を追加することで、機能とデザインの両方を一度に指定できます。

実践：最初のアプリを作ろう

チュートリアル課題

今回は、プロンプトガイドにあった例を使って実際にアプリを作成します。

入力するプロンプト：

「パーソナルトレーナーがクライアントのワークアウト計画を管理し、週ごとの進捗を追跡できるアプリが欲しい」

このプロンプトで実装される機能（予想）

  • 管理画面：トレーナー用のクライアント管理機能
  • ユーザー画面：クライアント用のワークアウト確認画面
  • ワークアウト計画：トレーニングメニューの作成・編集
  • 進捗追跡：週ごとの実施状況の記録と可視化
  • データベース：クライアント情報とワークアウトデータの保存
  • 認証機能：ログイン・ログアウト機能

次のステップ

  1. プロンプトを日本語で入力
  2. 送信ボタンをクリック
  3. Base44が自動的にアプリを生成開始
  4. 次回のチュートリアルで生成されたアプリを確認・カスタマイズ
