Gemini API ファイルサーチ

NotebookLM級の検索機能をAPIで簡単実装

ファイルサーチAPIとは？

NotebookLMの検索技術をAPI化

膨大な情報ソース（最大300ソース）から、いい感じにデータを引き出すRAG技術をAPI一つで実現。

サポートファイル形式

テキストファイル

JavaScript, PHP, Python, TypeScript, TSX, etc.

ドキュメント

PDF, DOCX, TXT, Markdown

データ形式

JSON, XML, ZIP

仕組み

  1. ファイルアップロード: ストレージに保存
  2. 埋め込みモデル: ベクトル化してデータベースに格納
  3. クエリ: ユーザーの質問をベクトル化
  4. 検索: ベクトル近似値で関連データを取得
  5. 生成: Geminiが回答を生成

料金

項目料金
埋め込み（インデックス登録時）$0.15 / 100万トークン
ストレージ無料
クエリ実行時無料
出力（Gemini使用時）通常のコンテキスト料金

実装

3つのAPI

1. ストア作成
2. ファイルアップロード
3. 検索実行

実装は驚くほどシンプル。ストア作成、ファイルアップロード、検索クエリの3ステップで完了。

制約

  • 最大ファイルサイズ: 100MB
  • ストレージ制限: Tier 1で10GB
  • 企業サービスには拡張性でネックになる可能性
  • 個人・社内利用には十分

まとめ

超シンプルで強力なRAG実装

  • 複雑なRAG設計不要
  • チャンキング・ベクトル化を自動処理
  • コストも非常に低い
  • Gemini 2.5 Proとの組み合わせで最強
