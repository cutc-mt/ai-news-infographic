Gemini CLI v0.12.0 完全ガイド

ターミナル内のチャットからスクリプト可能なアシスタントへの進化

大きな変化の概要

もはや単なるターミナルチャットではない

Gemini CLIはスクリプト可能なアシスタントに進化しました：

  • 拡張機能システム - 機能を自由に追加
  • スマートモデルルーティング - タスクに応じて最適モデルを自動選択
  • Jules統合 - バックグラウンドでコーディングタスクを委任可能な自律サイドキック
  • 無料で試用可能 - セットアップも簡単で洗練度が高い

Jules統合 - 自律サイドキック

Julesとは？

Julesは、Gemini CLIから指示できる自律的なサイドキック。管理されたVM内で動作し、バックグラウンドで中規模のコーディングタスクを実行。

リポジトリクローン

指定したリポジトリを自動クローン

依存関係インストール

必要なパッケージを自動インストール

ファイル変更

コードの修正・リファクタリング

ブランチ作成

変更を新しいブランチに送信

Julesセットアップ手順

1
Julesアカウント作成

Jules公式サイトでアカウントを作成

2
GitHubリポジトリ接続

Julesコンソールで作業対象のGitHubリポジトリを接続

3
拡張機能インストール
gemini extensions install jules
4
タスク委任
# CommonJSモジュールをESモジュールに変換 /jules convert commonjs modules to ES modules # ステータス確認 /jules what is the status of my last task

Julesの実践的な使い方

ターミナルでの流れを保ちながら作業

  • バグ修正: 軽度のバグをJulesに委任し、自分は別タスクに集中
  • 機械的リファクタリング: コードスタイル変換、モジュールシステム移行などを自動化
  • フォーマット変換: ファイル形式の一括変換
  • VM作業: クローン、依存関係、編集、プッシュを自動処理

Julesの使い方ベストプラクティス

  • CIスコープのように扱う: 明確にタスクを指定
  • 差分をレビュー: Julesが作成した変更を必ず確認
  • マージをゲート: 自動マージせず、レビュー後にマージ

主要コマンドリファレンス

コマンド 機能 使用例
/model モデル選択 重いコーディングにProを固定、クイックノートにFlash使用
/settings 設定調整 圧縮設定、コードベース調査官の有効化、ターン数制限
gemini extensions install 拡張機能インストール GitHub URLまたはローカルフォルダから拡張をインストール
/extensions list 拡張機能一覧表示 インストール済み拡張を確認
/extensions update 拡張機能更新 最新の変更を取得
/extension explorer 拡張ギャラリー表示 ブラウザで拡張カタログを開く
/chat share 会話エクスポート ツール呼び出しを含む会話をMarkdown/JSONでエクスポート
--output-format stream-json ヘッドレスストリーミング エージェント進捗をリアルタイム監視
/jules バックグラウンドタスク委任 中規模タスクをJulesに委任、ステータス確認、結果レビュー

実践的な使い方

モデル選択を活用

セッション開始後、/modelで重いコーディング作業にProを固定、またはクイックノートにFlashを使用。

圧縮とコードベース調査官を調整

/settingsで圧縮を調整し、コードベース調査官を有効化。ターン数制限でガードレールを設定。

拡張機能をインストール

GitHub URLまたはローカルフォルダからgemini extensions installでインストール。有効化、一覧表示、更新、新規作成が可能。

セッション中に拡張を管理

/extensions listまたは/extensions updateで管理。

拡張ギャラリーを開く

/extension explorerでギャラリーを開き、コミュニティ、パートナー、Google製の統合を発見。

ターミナルタイトルにステータス表示

show_status_in_titleをtrueに設定し、ペインを切り替えながらライブステータスを確認。

会話をエクスポート

/chat share file:markdownまたはfile:jsonでツール呼び出しを含む会話をエクスポート。PR、ポストモーテム用。

ヘッドレスでJSONelをストリーム

--output-format stream-jsonでエージェント進捗をリアルタイム監視。

Julesにバックグラウンドタスクを委任

/julesプロンプトでタスク委任、必要に応じてステータス確認、マージ前にブランチ差分をレビュー。

利点と制限

主な利点

  • プラットフォーム化: スタックをエージェントに持ち込める
  • スマートルーティング: クォータを賢く節約
  • モデル固定: 必要に応じて決定論的な動作
  • ワークスペース認識: コードベース調査官による文脈理解
  • 非同期レイヤー: Julesが中規模タスクをオフロード
  • クリーンな出力: レビュー用のブランチを返す
  • 自動化向け: Stream JSONが強力
  • シークレット管理: セキュアAPIキーダイアログ
  • QoL改善: Markdownトグル、メッセージ編集、非対話的MCP

制限事項

  • 初期セットアップ: 拡張機能は初回設定と調整が必要
  • ヘッドレス承認: 信頼フォルダー、サンドボックス、ポリシー、許可ツールリストが必要
  • IDEプラグイン: エディターによって成熟度が異なる
  • 圧縮調整: 過度な圧縮で文脈忠実度低下、緩すぎると重くなる
  • モデルルーティング: 再現性重視のテストでは/modelで固定が必要
  • Julesプロンプト精度: 不明瞭な指示はノイズの多い差分を生む
