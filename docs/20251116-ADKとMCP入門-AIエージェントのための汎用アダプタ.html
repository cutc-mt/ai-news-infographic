MCPとは？ AIのための「USBCポート」

MCP (Model Context Protocol) は、LLMやAIエージェントが外部の世界（ツール、API、データソース）と対話する方法を標準化するオープンな規格です。ツールごとに専用の接続コードを書く手間をなくし、一つの統一された方法を提供します。

なぜMCPが必要なのか？

  • 能力の拡張: ファイル操作、DBクエリ、Google Maps連携、画像生成など、外部の強力なツールをエージェントが即座に利用可能になります。
  • モジュール性と再利用性: ツール群を独立したMCPサーバーとして構築すれば、どんなMCP準拠クライアントからも再利用でき、開発が効率化します。
  • セキュリティと制御: ツールへのアクセス境界を明確に定義し、安全なエージェントシステムを構築できます。
  • デプロイの簡素化: エージェント本体とツールを分離することで、スケーリングや管理が容易になります。

ADKとMCPの連携方法

ADKはMCPToolSetという組み込み機能で、MCPサーバーとの連携を非常に簡単にします。

# MCPツールセットを初期化
mcp_toolset = adk.mcp.MCPToolSet(
    # サーバーの起動コマンドやURLを指定
    command=["npx", "file-system-mcp-server", "./"]
)

# エージェントにツールセットを渡すだけ
agent = adk.LLMAgent(
    tools=[mcp_toolset]
)

MCPToolSetが裏側でサーバーとの接続、ツールの読み込み、ADK形式への変換、リクエストの送受信をすべて自動で行います。
