ADKによる商品検索AIエージェント開発

Agentic RAGで実現する次世代のEコマース体験

Google AI Agent Stack

  • ADK: 開発者による開発者のためのオープンなマルチエージェントフレームワーク。本番環境への投入の鍵。
  • MCP: アプリと外部システム（ツール等）の対話を標準化するオープンプロトコル。
  • Agent Engine: 開発したエージェントのデプロイと実行を管理するマネージド環境。インフラ管理を簡素化。
  • A2A Protocol: エージェント間の「協調」を目的としたオープンプロトコル。業界標準を目指す。

Agentic RAGとは？

従来のRAG

単純な検索と回答生成。曖昧な質問や複合的なタスクには弱い。

Agentic RAG

エージェントが自律的に思考し、複数のツールを駆使して調査・分析・推薦を行う。

  • まるで人間のコンシェルジュのように、対話を通じてユーザーの意図を深く理解し、最適な提案を行う。

本番利用に耐える検索基盤の3要素

  • マルチモーダル検索: VLM（Vision-Language Model）を活用し、画像とテキストを同じベクトル空間で検索。画像の内容を理解した検索が可能に。
  • ハイブリッド検索: キーワード検索（製品番号など）とベクトル検索（意味の近さ）を組み合わせ、両者の長所を活かす。
  • タスクタイプエンベディング: 単純な「類似検索」を超え、質問と回答の「関連性」を学習したモデル。Vertex AIのEmbedding APIで専門的な推薦モデルを自前で構築することなく利用可能。

デモ: Shopper's Concierge

ADKと高度な検索基盤を組み合わせた、Eコマース向けAIコンシェルジュエージェントの動作フロー。

  1. 対話による意図理解: 「息子の誕生日プレゼント」といった曖昧な依頼に対し、年齢や興味などを質問し、ユーザーのニーズを具体化する。
  2. ディープリサーチ: ユーザーの許可を得て、Google検索ツールを使い「10歳向け 人気ゲーム」などの最新トレンドを調査する。
  3. 複数キーワード生成・検索: 調査結果に基づき、約100件の具体的な検索キーワードを自動生成。
  4. 内部DBでの一括検索: 生成した100件のキーワードを使い、1000万件の商品データベースに対して一括で検索を実行する。
  5. 結果の統合と推薦: 検索結果を統合・分析し、最適な商品をカテゴリー別に整理して、人間のように推薦する。
  6. 音声対話: ADKの双方向ストリーミングAPIにより、自然な音声での対話を実現。
