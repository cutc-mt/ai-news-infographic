GeminiとAgent Development Kitで作るAIエージェント開発

Google Cloud Next '25で発表されたAIエージェント開発の最前線

AIエージェントとは？

  • ユーザーの代わりに目標を追求し、タスクを完了させるソフトウェア。
  • 自律的に考え、ツールを動的に使用・制御する。
  • イメージは「物知りな対話相手」のボットから進化した「自律的に動く執事」。
  • Bot → AI Assistant → AI Agent の順に自律性が高まる。
単なる対話ツールじゃない！
自ら考えてタスクをこなすのがエージェント。

Agent Development Kit (ADK)

  • Google Cloudが提供するAIエージェント構築用フレームワーク。
  • LiteLLMでGeminiやOpenAIなど複数モデルを簡単に切替可能。
  • ループ、シーケンス、並列実行などのワークフローを簡単に構築。
  • Google CloudのAgent Engineへのデプロイが容易。
  • VPC-SCやCMEKなど、エンタープライズ級のセキュリティに対応。
エンタープライズ向けの
本格的なAIエージェント開発を強力に支援！

3つの開発アプローチ

  • ノーコード: Agent Spaceなど。非エンジニアでも簡単だが柔軟性に欠ける。
  • ローコード: Conversational Agentなど。独自ルールが多く学習コストが高い。
  • フルコーディング: ADKなどを使用。柔軟性が最も高く、今後の主流になる可能性。
AIコーディングツールの進化で、
フルコーディングのハードルは下がっていく！

テストと観測性

  • 指標定義: テスト前に「成功」の定義、重要タスク、追跡指標を明確化する。
  • テスト実装: JSONで入出力や期待するツールコールを定義し、`evaluator`で評価。
  • ADK Web: GUI上でエージェントの実行フローを可視化・追跡できる。
  • AgentOps: 簡単な設定で導入できるサードパーティ製の高度な観測性ツール。
エージェントの品質を保つには、
体系的なテストと観測性の確保が不可欠！
