Anthropic長時間稼働AIエージェント

効果的なハーネス設計で精度向上

2025年11月29日

記憶の課題

長時間稼働するAIエージェントの最大の課題は記憶。バトンタッチ時の情報共有不足で、人間のエンジニア引き継ぎと同様の問題が発生。

コンテキストウィンドウの壁

AIエージェントはコンテキストウィンドウ内で動作。20万トークンの制限内でしか記憶を維持できず、使い果たす前の対策が必須。

2つのエージェントに分離

Claude Agent SDKで実装。イニシャライザーエージェント（初期設定・オーケストレーション）とコーディングエージェント（実装）に分割。

イニシャライザーエージェント

INIT.shで初期化スクリプト実行。Git初期化でファイル分を追跡、詳細な機能リスト作成、進捗ログ（.claude_progress）定義を担当。

コーディングエージェント

機能リストから1つずつ選んで実装。完了後は次のステップへ綺麗に引き継ぎ。コミットログとGitコミットで履歴を残す徹底管理。

ハーネスとは

LLMにツール、メモリ、MCPサーバーを提供する仕組み。スキル、サブエージェントなど拡張機能を総合した、問題解決をサポートする制御環境。

Claude Agent SDK

強力で汎用的なエージェントハーネス。Claudeモデルをコントロールしやすくする仕組みを提供。長時間稼働タスクの精度向上を実現。

2つの特徴的な問題

1. 複雑な工程を一度に全て出力しようとしてコンテキスト切れ。2. 早すぎる完了宣言で途中で省略。この2つの癖を理解して対策が重要。

関心事の分離

マルチエージェントで役割を明確に分離。イニシャライザーとコーディングで担当を分け、各エージェントの限界を認めた設計。

200以上の機能要件

早すぎる完了制限を防ぐため、初期化エージェントに200以上の機能要件を書き出させる。マークダウンではなくJSON形式で管理。

デフォルトFalse戦略

全機能のデフォルトをFalseに設定。細かい設定だが、構造を壊さず徹底的に管理することで早すぎる完了を防止。

Gitログの徹底活用

外部記憶としてGitコミットログを活用。進捗ログ、ドットファイル、メタ情報を徹底的に記録。引き継ぎの質を大幅向上。

適切なテスト手法

関心事ベースのタスクに応じた単体テスト。Playwright、Puppeteer、MCPなどで動作確認しながらエージェントが作業進行。

コーディングエージェントの動作

PWDコマンド使用、ログ確認、機能リストから1個選択実行、INIT.shで基本動作確認。体系的なワークフローで精度を維持。

圧縮技術

コンテキスト満杯前に内容を圧縮。重要情報を残して次のステップへ。外部記憶と組み合わせて長時間稼働を実現。

外部記憶の活用

機能リスト、ドットファイル、Gitメタ情報を外部に配置。コンテキストウィンドウの限界を超えた記憶管理を可能に。

ハーネス作りの重要性

今後のAIエージェント開発で最も熱くなる領域。Anthropicが率先してトレンドを共有。実践的な知識が開発者に必須。

徹底の重要性

JSON形式、デフォルトFalse、Gitログなど小さな積み重ね。当たり前に見えることを徹底することで高精度なエージェントを実現。
