Claude Code 5つのバージョン

①CLI版(ターミナル操作・推奨)、②Web版、③GitHubアプリ版、④IDE版(VSCode拡張)、⑤デスクトップアプリ版。CLI版を中心に使い分け。

Claudeモデル3種類

Opus(最高性能・高コスト)、Sonnet(中性能)、Haiku(軽量)。Opus 4.5がSWEベンチ80.9%で全モデル中1位。開発はOpus推奨。

Opusモデルの強み

指示追従に優れる。ハルシネーションレート低い。GPT-4o-1・Gemini 3 Proに並ぶ知能。コスト高いがClaude Codeとの組み合わせで最強。

料金プラン比較

Pro($20):週3000回・Sonnet+Opus。Max($100/$200):Opus 5倍/20倍。API課金ではGPT-4oの2.5~3倍高コスト。プラン内なら割安。

CLI版インストール

npm install -g @anthropic-ai/claude-code。Node.js必要。`claude`コマンドで起動。ターミナルアプリはWarp推奨(サジェスト・履歴・分割機能)。

200Kコンテキストウィンドウ

AIエージェントの短期記憶容量。LLM+メモリ+ツールズで構成。コンテキスト圧迫を避ける仕組み(スキル・サブエージェント)が重要。

claude.mdメモリファイル

ユーザーグローバル領域(全プロジェクト反映)とプロジェクト領域。日本語設定・プロジェクト概要・ディレクトリ構成を記載。できる限りシンプルに抽象化。

段階的開示技術

claude.mdは最小限に。詳細はdocsディレクトリに階層化。必要に応じてAIが参照。注意力150~200個制限に対応。トークン節約・精度向上。

標準ツール・MCP

読み書き・Bash・Grep・TodoList・WebSearch・WebFetch標準搭載。MCP(Model Context Protocol)でツール拡張。Dev Tools MCP推奨(ブラウザ操作)。

スキル機能

特定タスクを高再現性で実行する指示セット。skill.mdに定義(名前・説明・手順・スクリプト)。段階的開示でコンテキスト節約。チーム共有可能。

サブエージェント機能

独立コンテキストで専門タスク実行。関心ごとの分離・並列化・コスト最適化(Haiku/Sonnet使い分け)。結果のみ親エージェントに返却。

サブエージェント作成

/agentコマンドでCreate New Agent。パーソナル/プロジェクト領域選択。デスクリプション・ツール・関心ごと定義。モデル個別指定可能。

ショートカット・コマンド

&(バックグラウンド)、!(Bashモード)、Tab(シンキングON/OFF)、Shift+Tab×2(プランモード)、Shift+Tab×1(自動承認)、Ultra Think(深い思考)。

設定ファイル場所

~/.claude/claude.json(履歴・MCP)、settings.json(パーミッション・Hooks)、claude.md(メモリ)。`open ~/.claude`で開く。

Hooks・プラグイン

Hooks:特定タイミングでコマンド実行(応答終了・圧縮時・ツール前後)。プラグイン:マーケットプレイスから他人作成のエージェント・コマンドをインストール。

GitHub連携・Web/IDE版

GitHubアプリ版:Issue・PR上でClaude操作。Web版:出先から開発依頼。IDE版(VSCode):視覚的整理。デスクトップ版:ローカルプレイグラウンド。

Codex vs Claude Code

Opus環境でClaude Code優位に。SonnetレベルならCodex Max有力。指示追従・コンテキストエンジニアリング・チーム共有ではClaude強い。
