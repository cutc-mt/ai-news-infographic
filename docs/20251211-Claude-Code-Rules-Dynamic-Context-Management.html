CLAUDE.md肥大化問題

Claude CodeではCLAUDE.mdに開発共通ルール設定可能。カスタマイズ継続→どんどん肥大化。700行近く→6000トークン近く常時消費。フロントエンド・バックエンド・APIルール混在。フロントエンド実装時もバックエンドルール読み込む。家でリラックス中も仕事頭から離れない状態。厄介。

.claude/rules/機能登場

数日前追加の最新機能。ルールをファイルごと分割記述可能。pathsというGlobパターン機能あり。ファイル拡張子・パス単位で対象ファイル操作時のみ読み込み。特定フォルダ中操作時のみルール適用。例:.ts拡張子TypeScriptファイルのみルール適用。革命的機能。

自由なフォルダ分け

API開発ルール・フロントエンドルール・バックエンドルール・テストルール。自分好きなようにフォルダ分け可能。領域ごとルール整理。メンテナンス性向上。チーム開発での役割分担明確化。ルール管理の柔軟性。拡張性高い設計。

常時適用ルール・general.md

共通的読み込みルール。paths指定なし→常時適用。例:回答最後に必ず星マーク付加。/memoryコマンド→読み込まれているルール表示。他ルールあるのにgeneral.mdだけ読み込み。CLAUDE.mdとは別に共通ルール設定可能。基本設定レイヤー。

TypeScript専用ルール・typescript.md

.ts拡張子ファイル読み込み時使用。末尾に青ダイヤマーク+「TS」文字付加設定。src/index.ts読み込み→ルール正しく適用。Readツール実行後TypeScript用ルールファイル動的(後から)読み込み。必要時のみメモリ使用。効率的。

API専用ルール・api-rules.md

src/apiフォルダ内TypeScriptファイル用ルール。末尾にコンセントマーク+「API」文字出力。src直下ファイル→API専用ルール適用されず。apiフォルダ内ファイル→Readツール実行時apiフォルダ専用+TypeScript専用ルール両方読み込み。階層的ルール適用。

複数ルール同時適用

apiフォルダ内.tsファイル読み込み→合計3ルール適用。ルール1:apiフォルダ専用。ルール2:TypeScript専用。ルール3:常時適用general.md。それぞれ独立マーカー出力。柔軟な組み合わせ。状況応じた最適ルール自動選択。重複読み込みなし。

/memoryと/contextコマンド

/memoryコマンド→読み込まれているルール表示。/contextコマンド→コンテキスト(記憶容量)確認。現在消費トークン数可視化。ルール読み込み状況監視。デバッグ・最適化に有用。メモリ使用量透明性。開発者フレンドリー。

動的ルール読み込み効率性

既にルールがコンテキスト(頭の中)残存→再読み込みなし。一度読み込んだルール→メモリ保持。無駄な再読み込み回避。同じ拡張子ファイル複数操作→最初だけルール読み込み。後続操作→既存ルール利用。パフォーマンス最適化。トークン節約。

pathsでGlobパターン指定

Globパターンで柔軟なファイル指定。拡張子ベース:.ts、.tsx、.js等。フォルダベース:src/api/**、components/**等。パターンマッチング。ワイルドカード使用可能。複雑プロジェクト構造対応。精密なルール適用範囲制御。

コンテキスト節約効果

フロントエンド開発・バックエンド開発ルール分離。テスト書く時専用ルール(テスト関連拡張子・パス指定)。ドキュメント執筆時ルール。状況応じルール分割→コンテキスト効率使用。不要ルール読み込まない→メモリ空き増加→長い会話可能。AI駆動開発の効率化。

深掘りルール分割戦略

フロントエンド内細分化:コンポーネント開発用コンポーネントルール(pathsでcomponents/**指定)。スタイリング関連拡張子指定スタイリングルール(.css、.scss等)。バックエンド細分化:API関連・データベース関連分離。領域異なる→pathsで分割。細かすぎ→メンテナンス大変。バランス重要。

推奨分割単位

まず大きな分け方推奨。フロントエンド・バックエンド・テスト。3大カテゴリー分割から開始。必要応じてさらに細分化。過度な分割→管理複雑化。シンプルさと柔軟性バランス。チーム規模・プロジェクト規模考慮。段階的洗練。

検証用マーカー実装

動画検証:出力末尾マーカー付加。星マーク(常時適用)・青ダイヤ+TS(TypeScript)・コンセント+API(API専用)。視覚的ルール適用確認。実際開発:コーディング規約・命名規則・アーキテクチャパターン組み込み推奨。品質保証自動化。

実際の開発ルール例

API開発時コーディング規約。TypeScript開発時命名規則。エラーハンドリングパターン。ロギング方針。セキュリティガイドライン。パフォーマンス最適化指針。アクセシビリティ基準。これらをルールファイル化→自動適用→品質一貫性保証。チーム全体で統一基準。

仕事と家の分離比喩

旧CLAUDE.md方式:家でリラックス中も仕事のこと頭にある状態。全ルール常時メモリ占有。新.claude/rules/方式:仕事は職場、家は家で完全分離。必要な時だけ必要なルール読み込み。精神衛生(コンテキスト健全性)向上。効率的思考(処理)。

AI駆動開発の重要課題

コンテキスト管理はAI駆動開発で非常に重要課題。限られたトークン数を最大限活用。長い会話履歴維持。複雑タスク遂行能力向上。.claude/rules/機能でこの課題解決。開発体験劇的改善。生産性向上。未来の開発スタイル。
