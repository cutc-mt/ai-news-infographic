Gemini Conductor Googleスーパーエージェントプランナー

コンテキスト駆動開発・プラン優先・チーム共有AI記憶

2025年12月23日

チャットボットからエージェントへ
カジュアル会話ではなくインテリジェントエージェント管理。プロジェクト制約を理解してから1行もコード記述しない。プラン・ドキュメント・理解を強制する新パラダイムです。
無料Gemini Tier実質コストゼロ
Gemini API無料プランで動作。実験コストが実質ゼロ。重いデスクトップアプリ不要。Gemini CLI実行環境があればどこでも利用可能。手軽に試せます。
コンテキストファイル リポジトリ保存
product.md・tech-stack.md・guidelines.mdをリポジトリに保存。コミットすればチーム全体が同じAI記憶を継承。プロジェクト制約の共有が自動化されます。
トラック管理 Jira チケット方式
フィーチャーブランチ・Jiraチケット単位で作業組織化。プラン→承認→実装→復元ワークフロー。幻覚変数・忘れられたフレームワークを防止します。
Brownfield 既存コード逆解析
レガシーフォルダに投入すれば規約を逆エンジニアリング。あなたが書いたように見えるコード生成。既存プロジェクトにも即適用可能な柔軟性です。
インタビュー形式セットアップ
スタック・ビジョン・制約を一度質問。全ての将来変更がその契約を参照。シニアエンジニアのオンボーディングプロセスを模倣した設計です。
完全ローカル IDEそのまま利用
Node.js環境なら1つのnpmコマンドで完了。重いデスクトップアプリ不要。IDE変更も不要。数秒でインストール完了。既存ワークフローに即統合できます。
プラン優先 実装前承認必須
plan.mdファイルをまず生成。細分化されたステップに分解。Pages RouterではなくApp Routerを使うべき等の誤解を事前キャッチ。時間浪費を回避します。
実装フェーズ tech-stack参照
conductor implementで実装開始。作成プランを読込み。tech-stack.mdでルール遵守確認。ターミナルコマンド実行・ファイル作成を自動化します。
チームコラボ AIブレイン共有
MarkdownファイルをGitHubにコミット。チームメイトがリポジトリプル。全員のエージェントが同じコンテキスト参照。AI記憶をチーム全体で共有可能です。
設計哲学更新 即座に反映
product.mdでモバイルファースト設計に切替。次回navbar構築時に自動反映。制約が共有プロジェクトメモリに組込まれ一貫性を保証します。
繰返し説明不要 AIが記憶
package.jsonをチャットウィンドウに貼付不要。新セッション開始時も記憶保持。TypeScript使用を毎回説明する必要なし。ワークフロー効率化を実現します。
可視性抜群 ステータス即確認
/conductor statusでハイレベル概要表示。エージェント作業内容・実行プランステップ・変更ファイルを即座確認。進捗透明性が向上します。
インテリジェント復元 選択的ロールバック
conductor revertで特定トラックのみ巻戻し。Git履歴全体を破壊せず。望まない方向に進んだ作業を安全に取消。細かい制御が可能です。
長期記憶 短期バッファ超越
従来ツールはコンテキストウィンドウ依存。一時バッファのみで長期理解欠如。Conductorはプロジェクト全体アーキテクチャ・履歴を永続的に理解します。
幻覚防止 制約明確化
10分前削除した変数を幻覚しない。TypeScript使用中にJavaScript提案しない。既存ライブラリのインストール提案回避。制約認識で精度向上します。
実例 映画トラッカーアプリ構築
CineLogで視聴映画記録・評価・ジャンルフィルタ実装。空フォルダから/conductor setupで開始。インタビュー形式で要件収集。product.md自動生成されます。
新トラック作成 粒度細分化
/conductor new trackでプロジェクト構造スキャフォールド。Tailwind設定・メインムービーカードコンポーネント作成。plan.mdで細分化ステップ生成し事前レビュー可能です。
