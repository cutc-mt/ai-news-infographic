Auto Claudeとは

Claude API → 仮想ソフトウェアエージェンシー

標準的なClaude API対話を完全自律型の仮想ソフトウェアエージェンシーに変える新しいオープンソースプロジェクト。Kanbanボードと並列処理を使用。完全無料でオープンソース。自分のAPIキーを持参するだけ。

哲学の転換

標準チャットボットとは完全に異なる

Claude APIの生の知能を構造化されたエンジニアリングワークフローにラップ。「ベビーシッター」から「マネージャー」へ。ループプロンプトではなくスケール可能な管理体制。コマンドライン好奇心から視覚的管理可能な製品へ。

Kanbanボードインターフェース

チャットではなくワークフロー

空白ターミナルやチャットボックスではなくKanbanボードで開始。Planning、In Progress、AI Review、Human Review、Doneの列。AIを従業員のように扱う。タスクカードを作成してチケットとして管理。

タスク実行フロー

計画 → 進行中 → AIレビュー → 人間レビュー → 完了

タスク開始後、Planningに移動してAIがコードベースを分析。ステップバイステップ計画を作成。In Progressに移動して実行。画面上のテキストを見るだけでなく構造化されたワークフロー。

Git Worktrees活用

複数エージェントを並列実行

同じリポジトリの複数ブランチを異なるフォルダで同時チェックアウト。フロントエンドCSSバグ修正とバックエンドDB schema refactorを同時実行可能。それぞれ独自のワークツリーでサンドボックス化。

マージコンフリクト解決AI

専用AIレイヤーで自動解決

通常はマージコンフリクト地獄になる。Auto Claudeはマージコンフリクト解決専用のAIレイヤーを搭載。シニアエンジニアが介入して混乱をクリーンアップ。メインブランチに到達する前に解決。

Agent Terminals View

最大12個のターミナルを同時実行

より実践的な操作をしたい時に使用。12個の異なるジュニア開発者が部屋に座っているイメージ。ターミナル1にテスト作成、ターミナル2にドキュメント更新、ターミナル3に新機能実験を指示。名前変更して追跡可能。

時間の分離

スウォームをトリガーしてコーヒーへ

自分の時間とAIの実行時間を分離。並列化だけでなく「時間のデカップリング」が本質。スウォームをトリガーしたらコーヒーを取りに行ける。ベビーシッターから管理者へ。

Graph Memory + Semantic RAG

コンテキスト健忘症を解決

標準チャットはコンテキストを失う。Auto Claudeはグラフメモリシステムとセマンティック RAG（検索拡張生成）を使用。プロジェクト全体をインデックス化。ファイル間の関係を理解。使うほど賢くなる。

アーキテクチャ理解

Knowledge Graphでコードベース把握

フロントエンドコンポーネント変更がバックエンドルートに影響することを理解。チャット履歴の最後3メッセージではなく実際のアプリアーキテクチャに基づいて決定。特定のコードベースの知識グラフを構築。

Ideation & Roadmap機能

プロジェクト分析して機能提案

コードを分析して「エラーハンドリングが欠けている」「ダークモード実装すべき」と提案。アイデアをチケットとして生成。ボタンをクリックするだけでKanbanボードに追加され実行準備完了。

Change Log Generator

Git履歴から自動作成

Git履歴に接続してエージェントと自分が行った全作業を確認。機能、バグ修正、改善でグループ化。絵文字付きでフォーマット。GitHubリリースのドラフトも作成可能。退屈な作業を処理。

API Rate Limit対策

複数APIキーをローテーション

Claude Opus 4.5を複雑な計画タスクの頭脳として使用するとレート制限に到達。複数のAPIキーを入力してローテーション可能。ヘビーなプロダクショングレード使用のために設計。パワーユーザー向け。

Self-Review機能

AI自身が自分のコードを批評

AIがサンドボックスで計画・実行した後、自分のコードを批評する自己レビューステップ。その後に初めて最終的な人間レビューに提示。差分確認、テスト実行、満足したらマージ。

ワークフローの構造

What（何）を定義、How（方法）はAIが処理

あなたは「何」を定義し結果を検証。AIが「方法」を処理。タスク作成 → AIが計画 → サンドボックスで実行 → 自己レビュー → 人間レビュー提示。最終的に差分確認とテスト実行後にマージ。

Production-Ready Framework

自律型マルチセッションAIコーディング

プロダクション対応フレームワーク。コンテキスト健忘症解決のための永続的メモリ。シングルスレッド性質の解決で複数エージェントが互いに踏まない並列作業。単一API契約を仮想ソフトウェアエージェンシーに変換。
