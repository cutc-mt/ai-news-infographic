Claude Codeの問題点

シングルパスデフォルト性質

Claude Codeは優れた推論能力を持つが、出力が「十分良い」と判断するとすぐに停止。暗黙の思考・実行バジェットに制約。ネイティブな永続ループがあればもっと良い結果が出せる。

Ralph Loop - 強制永続性

Claudeを単純終了させない

Ralphを使えばClaudeは単純終了不可。タスクが本当に完了するまで作業を強制継続。連続的インテリジェントループで自分の作業を自律的に反復。好きなだけ実行可能。

Ralph Wiggum - Simpsonsキャラクター

子供のような永続性を体現

Simpsonsキャラクター「Ralph Wiggum」に基づく。繰り返し失敗、愚かなミスを犯すが、頑固に無限ループで成功まで継続。AIの反復的・エラー多発的だが執拗で自律的なコーディングプロセスを反映。

インストール方法

前提条件: Claude Code 2.0.76+

WSL（Windows Subsystem for Linux）またはターミナルでClaude Codeインストールコマンド実行。/pluginsコマンド使用可能な最新アップデート必須（2.0.76+）。/plugin installコマンドでRalph Loopインストール。

Ralph Loop使用方法

/ralph loop + タスク + 完了条件

/ralph loop [タスク] completion promise as "done"。5ステッププロセス: ①試行終了→②ストップフックがブロック→③同じプロンプトをフィードバック→④繰り返し→⑤完了まで。

最大反復制約

トークン浪費防止

max iteration追加で無限ループ防止。多すぎるトークン使用を回避。セーフティネットとして常に推奨。無意味な無限ループで何も達成しない事態を防ぐ。

プロンプトベストプラクティス

明確な完了基準・段階的目標

「良くして」ではなく厳格なガイドライン必須。実行可能な項目が必要。段階的目標設定（異なるフェーズでAIが達成すべきこと）。自己修正推奨（失敗テスト書く→機能実装→テスト実行）。
