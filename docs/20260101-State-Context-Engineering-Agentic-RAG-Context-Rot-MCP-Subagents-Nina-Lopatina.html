神経科学で報酬学習と意思決定の研究。NeurIPS 2025参加（5回目）。Contextual AIでコンテキストエンジニアリング大規模インフラを構築。新カテゴリーをプロトタイプから本番環境へ。

2024年 - 6ヶ月が1年に感じた年

カテゴリー確立は2024年半ば

コンテキストエンジニアリングが本格化したのは6ヶ月前（2024年半ば）だが、1年のように感じる。カテゴリーがプロトタイプ段階から本番環境スケールへ。2026年は全システムデザインパターンへ移行。

Normal RAGの死 → Agentic RAG

議論は決着済み - Agentic RAGが新基準

クエリ再構成（Query Reformulation）でサブクエリに分解。ドキュメント検索のマッチング精度が劇的に向上。パフォーマンス改善が圧倒的だったため新しいベースラインに。Normal RAGは死んだ。

Sub-agentsとTurn Limits

無制限エージェンシーはパフォーマンス低下

ターン制限と明示的制約が重要。無制限エージェンシーはパフォーマンス低下とハルシネーション発生。Retail Universeハッカソン（10万ドキュメント）で学んだ教訓。検証ループで無限ループ防止。

Retail Universe Hackathon

10万ドキュメント、数十億トークン

Brian BischoffとHamill Husseinが主催。PDF、CSV、ログファイル。Dynamic Agentでチャレンジ。人間ベンチマーク23点、Nina 25点、優勝29点。Mix BreadとClaude、Cursor使用。

Context Rot

100万トークン中70万で検索精度30%に

全ブログで引用されるが、Anthropicが数値化。100万トークンコンテキストウィンドウで70万トークン時点で検索精度が30%に低下。直感的だが数値で示すことで実行可能に。

Contextual AI

エンドツーエンドコンテキストエンジニアリング

コード、法律、小売、Eコマース、サポート全ドメイン対応。マルチモーダル取り込み、ハイブリッド検索、Re-ranker、Dynamic Agent。初のInstruction-Following Re-ranker（2024年3月）。

Instruction-Following Re-ranker

大規模DB推論で精度向上

初期検索で高リコール、コンテキストウィンドウに入れる前に高精度化。レイテンシが最大の苦情だがDynamic Agent（レイテンシ非感受性）ではゲームチェンジャー。大規模データベース推論で重要。

MCP - ドライバーかつ欠陥

巨大JSON定義でコンテキスト圧迫

MCPは巨大JSONツール定義でコンテキストウィンドウを圧迫。10ツールですぐにContext Rot。ただしプロトタイプには優秀。本番環境では直接API呼び出しに最適化すべき。

MCP Server選択

Re-rankerでMCP Server選択

Nina自身がRe-rankerでMCPサーバー選択のプロトタイプ作成。サーバー選択自体がコンテキストチャレンジ。人間が選択すべき（セキュリティチェック必要）。Anthropic公式レジストリはエージェント向け。

ベンチマーク飽和

Princeton HOW - 10月リリース、12月飽和

研究論文再現タスクがClaude Codeで飽和。人間による評価が必要（超人的ソリューションのため）。ゴールドデータセットにエラー発見。100%達成はカナリア（何かがおかしい警告）。

産業ベンチマークの必要性

おもちゃデータセットではなく実規模

ほとんどのベンチマークは小規模データセット。Retail Universeは10万ドキュメント、数十億トークン。コンテキストウィンドウに絶対に収まらない。実規模で評価する産業ベンチマークが必要。

KV Cache決定フレームワーク

変わらないもの→前、変わるもの→後ろ

システムプロンプト・初期ターン→前方。最近のターン・動的コンテキスト→後方。マルチターンエージェントで重要。効率性だけでなく安定環境提供。エージェントは混乱しやすく、パフォーマンス低下とハルシネーション防止。

意図的コンテキスト圧縮

モデル任せにしない、能動的制限

モデルはコンパクション（圧縮）がまだ苦手。Ninaはターン数を能動的に制限。Cursorでも会話途中で新しいウィンドウを開く（コンテキスト損失回避）。Dexの「Intentional Frequent Context Compression」。

Embodied Eval - スノーボードコーチ

25周モーグルレース、3-4ヶ月トレーニング

ChatGPTをスノーボードモーグルレースのトレーニングコーチとして評価。25周（40,000垂直フィート）。長期プロジェクト（3-4ヶ月）。ウィンドウ閉じて再起動したら多くのトレーニング情報喪失。今シーズンは複数モデル評価。

プロンプト最適化 - Jeppa & ACE

遺伝的進化 vs 小さなTweak

Jeppa: DSPyの進化版、目標設定してLLMが自己最適化、遺伝的進化要素。ACE（Dented Context Engineering）: プロンプト全体書き換えではなく小さなTweakで情報損失防止、金融・複雑文書で高性能。

2025年 - プロトタイピング段階

デザインパターンが沸騰中

統一デザインはまだない。デザインパターンが多数出現中。最初は無制限トークン使用、後から制約・最適化（KV Cacheなど）。2026年は本格スケール年になる予測。

2026年予測 - Full System Design

コンポーネント革新 → 全システム設計

「Re-rankerを最適化する方法」から「数十億トークンを本番環境で推論するエンドツーエンドアーキテクチャとは」へ。コンポーネントレベルから全システムデザインパターンへの移行。実規模での複雑タスク解決が増加。
