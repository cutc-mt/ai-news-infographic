iQuest-Coder 40B実機テスト

Code-Flow Training・ループアーキテクチャ・ベンチマーキング検証 - テックナビ

X（Twitter）でバズった理由

Claude 4.5・GPT-5・Kimi K2超えを主張

ベンチマークスコア結果でバズ。青色のiQuest-Coderが全ベンチマークでトップ。Claude 4.5 Sonnet、GPT-5.1、Kimi K2などtrillion（兆）レベルモデルを40Bパラメータで超えたと主張。

ベンチマークスコア詳細

SWE Bench・LiveCodeBench・TerminalBench

撮ったベンチマーク：SWE Benchmark（ソフトウェアエンジニアリング）、LiveCodeBench、TerminalBenchなどコーディング向けベンチマーク。Sonnet 4.5やGPT-5.1級のモデルを40Bで超えたスコア。

Code-Flow Training

プロジェクト20%→80%の進化を学習

完成コードだけでなくGitからコーディングプロジェクトの最初と最後を除外。20%完成→80%完成までのバグフィックス・フィーチャー開発を学習。Git履歴も含めトレーニング。中間段階（新機能追加・大きいバグ修正）が最も有益。

LoopCoderアーキテクチャ

HPM・KVキャッシュオーバーヘッド削減

HPM・KVキャッシュオーバーヘッド削減技術。Dual Post-Training（ThinkingとInstructの2トラック）使用。面白い技術を多数投入。GitHubページで詳細なトレーニング手法解説。

ベンチマーキング疑惑

全ベンチマークでトップは不自然?

全ベンチマークでトップ取得→ベンチマーキング（質問と回答をトレーニングデータ使用、スコア最高化のためトレーニング）の疑い。実際に試してどういう実力か検証必要。確認方法がないので実機テストしかない。

モデルバリエーション

7B・14B・40B・40B Loop

7B、14B、40B、40B Loopバージョンあり。全てHugging Faceでダウンロード可能。iQuestチームのモデルページでローカルテスト可能。40Bは4bitで約30GB VRAM必要。MLXバージョンもあり（Mac 64GBメモリで実行可能）。

LM Studio設定変更

MLX-LMコード修正が必要

LM Studioでエラー発生。MLX-LMのコード変更必要。model.pyでモデルマッピング追加（iQuest-CoderとしてLlamaアーキテクチャ使用）。リモートコード実行でtrust設定。公式サポート待ちを推奨、Ollamaは既にGGUF対応。

リソース使用状況

Mac 30GB→51GB使用

iQuest-Coder V1 40B Instruct、4bit quantization、MLXバージョン使用。ロード前：Mac約30GB使用。ロード後：51GBメモリ使用（約20GB使用）。26 tokens/secの生成速度、First Token: 0.23秒。

生成速度テスト

25-26 tokens/sec

"Hello"即座に回答、26 tokens/sec。500 Wordストーリー生成も25 tokens/sec。First Tokenまで0.23秒。40Bは大きいが4bit量子化で負担軽減。意外と早く使える。

スペースインベーダーゲーム失敗

ファイル作成できず、コード途中終了

VS CodeでClineプラグイン使用、iQuest-Coder選択。スペースインベーダーゲーム作成依頼。テキスト内でコード変更するがファイル作成不可。LM Studioでコピー生成。22 tokens/secでロジック説明も生成。

Pythonコード実行問題

一瞬表示後すぐ閉じる

test.pyコピー、余分部分削除。python test.py実行→何も出ない→一瞬表示後すぐ閉じる。うまくいかなかった。エラーログ出ていない。フォローアップ質問するとPythonコード生成するがリンク・フォーマット不一致。

コード問題点

変数未変更・途中終了

midの変数ちゃんと変えていない。下に「押すところ」だけ1つにイベント書いて、他の音のところ変えないまま残る。自分でテキスト修正。修正後少し遊べるが全然スペースインベーダーゲームになっていない。残念。

ソーラーシステムシミュレーション

HTML・CSS使用、タグ閉じ忘れ

ソーラーシステムシミュレーション作成依頼。HTML使用、ある程度できるはず。17-18 tokens/sec生成。タグ見ると最後の方で閉じずに途中終了。MLX変更が原因か不明。最後の方もコード部分にごちゃごちゃ書いてる。

ソーラーシステム結果

スター3つのみ表示、途中終了

ファイル作成、タグ閉じて実行。3つのスターのみ表示（YouTube画面で見づらいかも）。一応何もないわけではない。途中で止まっている。MLX版はオフィシャルリリース待ちを推奨。プロバイダー付いたら再テスト予定。

ベンチマーキング判定の難しさ

確認方法なし、実機テストのみ

スコア見てベンチマーキング判定は非常に難しい。試してみるしかない。本当に正しければDeepSeek瞬間再来（40Bで数百Bモデル超えはすごい）。ベンチマーキング可能性もある。みんなで試してフィードバック希望。

アーキテクチャ評価

めちゃくちゃ面白い研究

ベンチマーク置いといて、アーキテクチャはめちゃくちゃ面白いことやってる。すごくいい研究。ベンチマークのところだけ引っかかる。Code-Flow Training、LoopCoder、Dual Post-Trainingなど革新的技術多数。

今後の再テスト予定

プロバイダー対応・公式MLXサポート待ち

次回プロバイダー対応、LM Studio公式MLXサポートリリース後に再テスト予定。Open Routerでプロバイダー・エンドポイント提供なし（ベンチマーク走らせない）。7Bモデルでもテスト推奨（40Bテスト難しい）。
