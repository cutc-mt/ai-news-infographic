Claude SkillsでAIライティング完全解説

プロンプトキャッシュ＆コマンドプロンプトで最高の文章生成を自動化

Progressive Disclosure（段階的開示）

必要なタスクが来た時だけ専門の本を取り出す仕組み。トークン消費とアテンション消費を節約。全てを読み込まない効率的な設計。

3レベルアーキテクチャ

レベル1：メタデータ（Name＋Description）、レベル2：指示書（SKILL.md）実行時ロード、レベル3：リソースとコード（必要時のみ）。

プロンプトキャッシュの仕組み

プロンプトの冒頭部分が同一の場合、計算結果を再利用。応答速度を劇的向上、計算コストを削減。完全一致が必要。

メタデータ固定費問題

1スキルあたり100トークン、100個で1万トークン。毎回読み込まれるがプロンプトキャッシュで大幅削減可能。

GPTs作成への応用

プロンプトの最初に抽象的データ（コンセプト、ルール、トリガー）を配置。キャッシュの考え方を応用して効率化。

Skillsの弱点：文章が硬くなる

オーケストレーション層に投げて戻すため事務的で硬い文体に。ChatGPTのAdvanced Data Analysisで書くのと同じ問題。

解決策：読みやすくするスキル

硬い文章を柔らかくする専用スキルを作成。2つのスキルを組み合わせてパーフェクトな文章へ。

コマンドプロンプト（/blog-agent）

カスタム指示に定義。分析→下書き→編集→最終稿まで止まらず駆動。AIエージェント化の鍵。

A/B/Cテスト結果

A（プロジェクト）42点、B（スキル）48点、C（スキル+読みやすく）52点。スコアはCが最高だが文字数の影響も。

読みやすさ：プロジェクトが優位

プロジェクト8点、スキル7点。システムプロンプトで柔らかみのある文章。トークン消費は多いが共感的。

内容の深さ：スキルが優位

論理的展開、構成、具体性、独自性でスキルが高スコア。アテンション予算確保で詳細な説明が可能。

エンゲージメント（共感性）

プロジェクトの方が高い。温かみのある文章、親しみやすさ。スキルは単調で冷たい印象。

スキルの強み：重要ポイント抑える

アテンションがしっかりかかり重要ポイントを確実に抑える。SEO的にも強い。プロジェクトは抜け落ちる部分あり。

コマンドなしの問題

「ブログを書いて」だけだと途中で止まる。AIがインタラクティブ性を求める。最後まで書いてくれない。

自律走行プロンプト

カスタム指示に「/blog-agentでブログスキル＋読みやすくスキルを使用し自律的にゴールまで駆動」と設定。赤文字で認識。

スキル連鎖の実現

3つ、4つ、5つとどんどんスキルを重ね合わせ可能。コマンド1つで複数スキルを順次実行。

前提条件：ノウハウの洗練

コマンドプロンプトが機能する前提はノウハウとスキルの精密さ。大きくずれると修正が大変。

理想的なワークフロー

①ノウハウを練る → ②スキルを練る → ③/コマンドで一気に走らせる → ④微調整でOK。最高のAIライティング自動化。
