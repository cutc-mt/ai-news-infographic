Googleプロンプト繰り返し手法

プロンプトを2回入れるだけで非推論LLMの精度が向上する論文解説

論文概要

Googleが2025年12月17日にarXivで公表。タイトル「Prompt Repetition Non-Reasoning LLMs」（プロンプト繰り返しは非推論LLMを改善する）。

シンプルすぎる手法

プロンプトを2回以上コピペして貼り付けるだけで性能が上がる。「大事なことなので2回言いました」が言語モデルにも有効。

対象モデル：非推論LLM

GPT-4o mini、Haiku、Gemini Flashなど思考過程を生成しないノンリーズニングモデルがメイン。推論モデルでは効果が限定的。

ベンチマーク結果

多くの評価でベースラインを超え、★マークが統計的に有意に精度が良いことを示す。島々の部分が繰り返しプロンプトの結果。

実例：選択問題

選択問題を解かせるプロンプトを1度入力（ベースライン） vs 単純に2回繰り返すだけで精度が大幅向上。

Attention is All You Need

トランスフォーマーの論文で提唱されたアテンション機構。前後の文脈を効率的に参照する仕組み。文脈で単語の意味が変わる。

過去→未来は参照不可

言語モデルは過去のトークンは未来のトークンを参照できない。今ある文章の次に続きそうな単語を予測して生成。

文脈による意味の変化

「The cat called the mouse」（猫がネズミを捕まえた） vs 「I click the mouse」（マウスをクリック）。同じ単語でも文脈で意味が変わる。

順次予測の制約

cat → caught → the → mouseと前から順番に予測。今予測する単語の前の単語情報だけを使用。将来の単語情報は使えない。

順序の影響：選択肢が前 vs 後

①「東京・京都・北海道、日本の首都は」 vs ②「日本の首都は、東京・京都・北海道」。②の方が処理しやすい。質問文を先に見ると文脈がわかる。

Question First > Options First

実験結果から質問文を前に持ってくるクエスチョンファーストの方が、選択肢が前のオプションファーストより精度が高い。

繰り返しで順序影響を緩和

2つ目の文章の選択肢を処理する際に1つ目の文章を参照できる。最初に質問文を入力した時と同じ形で選択肢を処理可能。

推論モデルでの効果は限定的

推論モデルは内部で「ユーザーは何をリクエストしてるのか」を自分で生成してから答えを生成。内部的に繰り返しているため効果小。

翻訳・要約での注意点

「[文章をペタ] → この文章を翻訳してください」だと文脈がわからない状態で処理。先に指示を出す方が原理的に良い。

メリット①：トークン数不変

生成されるトークン数が増加しない。Think Step-by-Stepは精度上がるが出力トークン増で時間とコスト増。

メリット②：フォーマット不変

生成トークンのフォーマットも変わらないため、既存ワークフローに組み込みやすい。現在のプロンプトを2回繰り返すだけ。

特に効果的：Name Index 25 of 50

選択肢が50個与えられる問題でめちゃくちゃ精度向上。複数選択肢があるタスクで試す価値あり。

実務活用：低コストモデル運用

APIでアプリ開発時、コストが安く早いモデル（mini、Flash等）で精度向上。入力コスト増だが出力コスト不変で効率的。
