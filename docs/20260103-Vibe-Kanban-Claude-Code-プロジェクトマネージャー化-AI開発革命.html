Vibe Kanban + Claude Code

Claude Codeをプロジェクトマネージャーに変換・AI開発革命ツール

開発環境の混沌

開発環境が様々なAIツールの混沌とした集まりに。Claude Code用ターミナル、Gemini CLI、Cursor/Windsurf。誰が何をしているか追跡困難。

AIエージェント時代の到来

AIエージェント時代に突入。しかし管理は猫を放牧するような感覚。コンテキスト喪失、どのエージェントがバグ修正中か忘却、半分は待機時間。

Vibe Kanbanとは

この問題を解決する新しいオープンソースツール。単一のビジュアルインターフェースから複数のAIコーディングエージェントをオーケストレート。完全無料、GitHub公開。

驚異的に高速なセットアップ

エンタープライズクラウドアカウント不要。ターミナルでシンプルなnpxコマンド実行だけ。リポジトリクローンしてローカルビルドも可能。

ローカル実行＆自分の鍵使用

自分のマシンで実行し、自分のキーを使用。まさに我々が望む形。コマンド実行後、ローカルWebインターフェースが起動し、魔法が始まる。

チャットボックスではなくボード

Vibe Kanbanを開くとチャットボックスではなくボードが表示。TrelloやJiraのような形式。3日間無視する同僚ではなく、即実行するAIエージェントにタスク割当。

標準カラム構造

To-Do、In Progress、In Review、Doneの標準カラム。「認証ミドルウェアをリファクタリング」というカードを作成し、特定のエグゼキューターに割当。

キラー機能：並列実行

5つの異なるタスクを作成し並列実行可能。Claudeがテスト作成完了を待つ間ブロックされない。キューに入れ、スタートを押し、バックグラウンドでオーケストレート。

非同期ジョブとして扱う

コーディングタスクをブロッキング会話ではなく非同期ジョブとして扱う。これが革命的。フロントエンド作業開始前にテスト完了を待つ必要なし。

専用ビューとコンテキスト分離

タスクカードをクリックするとそのエージェントの作業専用ビュー表示。ターミナル出力、提案変更、差分表示。エージェントAのDB作業がエージェントBのCSS変更と混同しない。

Co-PilotからCo-Workerへ

AIプログラミングの考え方シフト。横に座って入力を助けるCo-Pilotから、別の仕事をする間に作業するCo-Workerへ。Vibe KanbanはそのCo-Workerのマネージャー。

一元化された設定

複数CLIツールで環境変数・無視ファイル・コンテキストルールを毎回設定する苦痛を解消。MCP（Model Context Protocol）サーバーを一箇所で設定。全エージェントが継承。

潜在的デメリット：抽象化レイヤー追加

多くの開発者にとって、Cursorで⌘Kやターミナル入力の速度は優秀。Kanbanボード導入は摩擦を追加。カード作成、説明入力、エージェント割当、ボード管理が必要。

小タスクにはオーバーキル

「このタイポ修正」「この色変更」のような小さく素早いタスクには過剰。額縁を掛けるのに建設クルー呼ぶようなもの。大規模マルチステップ機能向けに設計。

コンテキスト断片化問題

並列実行は素晴らしいが、コードは高度に相互依存。API応答構造リファクタとそのAPIを消費するUI構築を同時実行すると衝突。人間が最終的な調停者として必要。

開発サーバー管理機能

ボードから直接localhostサーバーをスピンアップ＆モニタリング。エージェントの変更が実際にコンパイルされ、クラッシュせず実行されるか確認。完全なミッションコントロールセンター。

視覚的フィードバックループ

数秒でプロジェクト全体のステータス確認。3タスク完了、1失敗、2進行中が一目瞭然。ターミナルでは不可能な視覚的フィードバック。AIの作業を具体的で追跡可能なものに変換。

AI時代のUI/UX実験

自律コーディングの適切なインターフェースを模索中。チャット？IDE？プロジェクト管理ボード？ Vibe Kanbanは複雑なマルチエージェント作業に構造・可視性・点滅カーソル待機からの脱却が必要と主張。
