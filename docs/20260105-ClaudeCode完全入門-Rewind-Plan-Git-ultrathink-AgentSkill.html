AIコーディング3つの進化

第1世代: ChatGPT（AI・開発環境分離）。第2世代: Cursor（AIエディター）。第3世代: Claude Code（自律型エージェント）。環境構築→実行→エラー修正の自律ループ。

Claude Code vs Codex CLI

両者月額20ドル。Claude利用制限厳しい（5時間10～40回）、Codex緩い（45～225回）。Claude日本語説明得意、Codexコーディング特化。

推奨プラン: 月額100ドル

Proプラン週40～80時間。100ドルプランでOpus 4.5使用→上限到達ほぼなし。ベンチマーク高性能、少ない入力トークンで高精度。

3つの始め方

①CLI・IDE（黒画面・Cursor・VS Code）②Claude Desktopサイドバー③GitHub連携クラウド環境。最も一般的はCLI・IDE。音声入力（AquaVoice）推奨。

Planモードで失敗回避

Shift+Tab 2回でPlanモード切替。実装前に計画立案→承認後コーディング。いきなり実装は意図外の結果。2段階ステップがベストプラクティス。

実例: 株バックテストアプリ

FastAPI + HTMX技術スタック指定。Planモード→要件質問→技術選択→10分で実装完了。Yahoo Finance日本株データ取得・分析・可視化。

Playwright自動ブラウザテスト

MCPサーバー設定でPlaywright利用。AIがブラウザ操作→自動テスト→動画録画。/mcpコマンドでenable/disable。テスト時のみenable推奨（コンテキスト消費大）。

許可不要モード（危険）

--dangerously-skip-permissions起動。1回1回許可不要で実行、重要ファイル削除リスクあり。エイリアス「cc」設定推奨。Docker環境推奨。初心者非推奨。

Sandboxモード（安全）

Linux・macOSのみ対応。/sandboxコマンドで起動ディレクトリ以下のみ読書権限。チーム開発・安全重視で使用。Windows未サポート。

/rewind変更巻き戻し

Esc 2回または/rewindで過去会話選択。チェックポイント復元、セッション内のみ有効。手動編集は復元不可。新セッションは過去復元不可。

Git変更履歴管理

Commitでセーブポイント作成。VS Code・CursorでSource Control→Staging→Commit。Discard Changesで変更取消。セッション跨ぎ復元可能。

CLAUDE.mdプロンプト規約統一

プロジェクトルートにCLAUDE.md作成。毎回入力したいプロンプト設定、技術スタック・命名規則指定。/initで自動生成。書きすぎ注意（コンテキスト消費）。

.claude/rules細かい制御

.claude/rulesフォルダで特定パス編集時ルール適用。Markdownでパス指定→コードスタイル決定。CLAUDE.mdより効率的コンテキスト入力。

Thinkingモード・ultrathink

Sonnet 4.5・Opus 4.5でデフォルト有効。ultrathink入力で徹底的思考（虹色表示）。Ctrl+Oで思考プロセス確認。難しい実装時推奨。Max thinking tokens 31999。

Plugin: Agent Skills拡張

/pluginでDiscover→インストール。Frontend Designスキルで非AI的デザイン。.claude/skills/skill.md作成で自作可能。特定タスク知識パッケージ化。

Sub Agents別人格Claude

プロンプト設定済み別人格Claude作成。フロントエンド特化・データ分析特化等でコードレビュー。タスク別にエージェント使い分け。

画像入力・割り込み入力

チャット欄にCtrl+Vで画像貼付け可能。スクショで視覚情報伝達。AI作業中の割り込み入力対応。次ステップに反映される仕様。

学習用: 実装せず説明モード

CLAUDE.mdに「実装せず説明のみ」指示記述。ファイル・変更理由・仕組み解説のみ出力。段階的実装理解、スキル蓄積に有効。初学者推奨設定。
