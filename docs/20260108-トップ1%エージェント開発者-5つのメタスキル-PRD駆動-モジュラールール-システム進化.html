5つのメタスキル・PRD駆動開発・モジュラールール・コマンド化・システム進化

10x生産性の真実: ツールではなくプロセス

AIコーディング潜在能力未活用問題。本当に出荷するエンジニア別ゲーム。道具同一・やり方根本差異。時間複利化メタスキル5選。トップ1%分離要因分析。

スキル1: PRD-First Development

Product Requirement Document最優先。Markdown単一文書=全スコープ定義。Greenfield=POC/MVP全機能記載。Brownfield=既存文書化+次構築明示。北極星文書確立。

PRD分割戦略: 粒度制御

一度に多すぎ→完全失敗原則。API実装・UI構築・認証分離。個別フィーチャー順次開発。イテレーション接続維持。「何を次に」明確化源泉。

/create-prd コマンドワークフロー

会話→合意→PRD出力自動化。テンプレート定義済構造。Target Users・Mission・In Scope・Out Scope・Architecture。スラッシュコマンド一発生成。どのAIアシスタントでも使用可。

/prime 会話開始儀式

新会話必須コンテキスト読込。PRD必読ファイル指定。コードベース自己プライム。「PRDに基づき次は?」日常質問。使い慣れたワークフロー公開。

スキル2: Modular Rules Architecture

グローバルルール肥大化問題。軽量化→コンテキストウィンド保護。agents.md/claude.md短縮焦点。常時適用制約のみ記載。テスト・ログ戦略等限定。

.claude/reference 分離戦略

タスク別ルール外部配置。Frontend・API・Deployment個別MD。グローバルルールから参照。作業種類一致時のみ読込。1000行詳細指示許容。圧倒的柔軟性獲得。

コンテキストウィンド死守思想

過小評価重要性警告。必要文脈全保持・前方積込回避。コマンド内参照パス記載選択肢。見えないコスト可視化。効率的資産管理実現。

スキル3: Commandify Everything

2回以上プロンプト→コマンド化必須。Git commit・Code review・Context load。数千キーストローク節約累積。Markdownワークフロー定義。共有可能再利用性実現。

Habit Tracker全コマンド公開

GitHub完全ワークフロー提供。.claude/commands フォルダ全収録。日常使用コア集合。カスタマイズ自由度100%。Prime・Execute・Plan・Validate全種。

スキル4: The Context Reset

計画→実行間会話再起動原則。計画終了=Markdown文書出力必須。Prime無し実行開始。文書のみコンテキスト。推論・検証余地最大化。軽量保持鉄則。

/structured-plan 詳細設計

実行用完全仕様書作成。Feature・User Story・参照コンポーネント・Task分解。/clear 完全ワイプ実行。/execute [plan] 単独読込。他文脈不要設計完備。

スキル5: System Evolution Mindset

最重要・最後配置意図的。全バグ=エージェント強化機会。手動修正→前進×・システム修正→前進○。ハルシネーションパターン発見時改善。時間経過強化メカニズム。

3層システム改善対象

Global Rules・Reference Context・Commands。誤インポート→ルール追加。テスト忘却→Plan template更新。認証不理解→Reference doc作成。グローバルルール参照パス追加。

バグ後自己反省プロトコル

機能完成・検証後問題発見時。「XYZ動作不良・修正済→プロセス/ルール改善点検討」。実行 vs 計画比較要求。ルール vs プロセス差異分析。再発防止組込思考。

マインドセット中核思想

バグ修正ではなくシステム修正。問題許容システム治癒。音声入力活用推奨。大まかな戦略定義充分。自己反省思考促進。時間複利威力実感。
