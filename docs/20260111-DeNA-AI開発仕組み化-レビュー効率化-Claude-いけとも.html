DeNA流AI開発の仕組み化 - レビュー工数を1/3に削減

Claude Code活用・サブエージェント構築・自動レビュー・ナレッジ循環

①育てるほど楽になるAI開発: DeNAエンジニアの実践事例

DeNAのエンジニアである長田さんが、育てるほど楽になるAI開発の仕組みを公開した。AI使用で開発は速くなるが、低品質コードや矛盾が発生する課題を解決。AIを使えば開発速度は上がるものの、レベルの低いコードができたり、やればやるほど矛盾が生じて難しくなるケースがある。これらの課題をどのような仕組みで解決し、段階的に効率化して精度の高いものを作るのかを詳しく紹介している。

②プロジェクトの課題: 新規サービスの複雑性とタイトスケジュール

新規サービス開発では、設計判断が多くタイトなスケジュールという厳しい条件がある。複雑なドメインロジックと特有の概念により、全体像の把握が困難。特にビジネスロジックが複雑で構造が入り組んでいる場合、開発者が全体を理解するのに時間がかかる。Claude CodeやCursorなどのツールは個人の開発補助として有効だが、チーム全体での活用方法が確立されていない状況があった。

③既存の課題: コンテクスト共有不足とレビュー精度の低さ

10月にチームにジョインした時点で、Claude CodeやCursorが使える開発環境は整っていた。しかしコンテクストが共有されず、ガイドライン外のコードが生成され、レビュー精度が良くないという課題。個人がそれぞれツールを使うだけでは、チーム全体の品質向上につながらない。プロジェクト固有のルールや設計パターンがAIに伝わらず、統一感のないコードが生成されてしまう問題があった。

④解決アプローチ: 2つの軸でワークフローを設計

課題解決のために、闇雲にツールを導入するのではなく、明確な方針を立てた。レビュー作業の量を減らすことと、成果物の品質を標準化するという2つの軸をベースにワークフローを設計。人間はビジネスロジックやアーキテクチャの妥当性といった本質部分を見るべきで、コーディング規約違反や単純なバグ、誤字脱字などはAIによるレビューで解決させる。効率と品質の両立を目指した戦略的なアプローチとなっている。

⑤成果物の品質標準化: 誰でも高品質なコードを高速生成

メンバー全員が品質の高い成果物を高速に出力できる状態を目指した。コーディング規約、設計パターン、ドメイン知識をAIが参照できる形で整備。経験やスキルに依存せず、誰でも一定水準以上のコードを素早く生成できる環境を構築した。新しくチームに加わったメンバーでも、ベテランと同等の品質でコードを書けるようになる。組織全体の開発効率が底上げされる仕組みとなっている。

⑥リポジトリ構造: AIが参照するコンテクスト情報を集約

リポジトリにはAIが参照するコンテクスト情報を一箇所に集約している。CLAUDE.mdファイルがあり、その下にサブエージェント、カスタムコマンド、スキル、ドキュメントが配置。.claude/ディレクトリにサブエージェントの定義ファイル、カスタムコマンドの手順書、エージェントスキル（Git Hub Issue Analyzer、PR Analyzer、Code Generation、Quality Checkなど）を格納。ドキュメントにはAPI開発ガイドライン、コーディングガイドライン、Firestoreガイドライン、プロジェクトナレッジなどを配置している。

⑦3フェーズのワークフロー: 実装→レビュー→ドキュメント更新

AI活用のワークフローを3つのフェーズで構築している。実装フェーズ、レビューフェーズ、ドキュメント更新フェーズで構成。実装フェーズでは開発者がローカル環境でCursorやClaude Codeを使ってコードを生成する。レビューフェーズではGitHubにプルリクエストを出すと、Claude Code Actionsが自動的にレビューを実行する。レビューでの指摘事項がドキュメントに反映されて、AIの出力品質を継続的に向上させる仕組みとなっている。

⑧サブエージェントの構築: タスク特化型で単一責任を実現

全てのコンテクストをメインエージェントに渡すと、コンテクストが肥大化して汚染されてしまう。小さく単一責任のエージェントとして構築すべきという原則に基づき、特定タスクに特化したサブエージェントを定義。例えばAPIスペックレビュー専門のサブエージェントは、OpenAPI仕様の構造や命名規則を厳格にレビューする。コードレビューアは言語のベストプラクティスやプロジェクト固有の設計パターンをチェックする。E2Eテスト実装担当など、役割を明確に分けている。

⑨Claude Code Actionsによる自動レビュー: 変更ファイルに応じて起動

整備したサブエージェントを利用して、レビューを行うワークフローを作成した。PRが作成されると変更ファイルの種類に応じて適切なエージェントが起動して自動レビューを実行。例えばAPIの仕様ファイルが変更された場合はAPIスペックレビューエージェントが、ビジネスロジックのコードが変更された場合はコードレビューアが自動で起動する。人間がレビューする前に、機械的にチェックできる項目を全て確認してくれる。

⑩Selenar MCPの導入: 巨大コードベースで正確なコンテクスト把握

Model Context Protocolを活用したMCPサーバー「Selenar」を導入した。モノレポのような巨大なコードベースでは単純なファイル検索だけでは不十分。AIが必要なコンテクストを見つけるのに時間がかかったり、必要な情報を見つけられないことがある。Selenarはパワフルなコーディングエージェントツールキットとして、コードのシンボルや依存関係などを正確に把握できる。大規模コードベースでの検索性を大幅に向上させる仕組みとなっている。

⑪ドキュメント自動更新フロー: レビュー指摘をナレッジ化

人によるレビューでの指摘事項を確実にナレッジ化する仕組みを構築した。プルリクエストのレビューコメントでClaude Code Actionsをメンションすると起動。レビューコメントや議論を分析して、適切なドキュメントを更新する別のPRを自動作成する。この仕組みにより、レビューで得られた知見が確実にドキュメントに蓄積される。自動更新ではなく、ドキュメント更新の差分提案をAIが行い、人間が最終チェックすることで品質を担保している。

⑫ナレッジ循環の仕組み: どんどんAIが賢くなる

ドキュメント自動更新の最大の価値は、ナレッジが循環する仕組みにある。レビューで指摘された内容がドキュメントに蓄積され、次回からAIがそれを参照して改善されたコードを生成。使えば使うほどAIの出力品質が向上し、同じミスが繰り返されなくなる。この好循環により、時間が経つほど開発が楽になる。変なドキュメントが左されることを防ぐため、人間が最終チェックする仕組みも組み込まれている。

⑬成果: レビュー回数もコメント数も1/3に削減

この仕組みを導入した結果、顕著な効果が現れた。人間のレビュー回数が1/3程度に、コメント数も1/3に削減。コードの品質標準化もできて、ナレッジの蓄積も継続的に行われるようになった。人間はビジネスロジックの妥当性など本質的な部分のレビューに集中でき、機械的なチェックはAIに任せられる。開発速度と品質の両方が向上する理想的な状態を実現している。

⑭コンテクストの一元管理: チーム全体での共有を実現

リポジトリにコンテクスト情報を一元管理することで、チーム全体で共有できる。個人のローカル環境だけでなく、CI/CDパイプラインやレビューフローでも同じコンテクストを参照。新しいメンバーがチームに加わっても、リポジトリをクローンするだけで、すぐにプロジェクトのルールや設計パターンを理解したAI支援を受けられる。属人化を防ぎ、チーム全体の生産性を均一に保つことができる。

⑮GitHubとの統合: PR作成時に自動でレビューが起動

GitHub上でプルリクエストが作成されると、自動的にClaude Code Actionsが起動する。変更されたファイルの種類を判断して、適切なサブエージェントを選択してレビューを実行。この仕組みはGitHub Actionsのワークフロー設定で実現されている。開発者は特別な操作をする必要がなく、通常通りPRを作成するだけで自動レビューが開始される。シームレスな統合により、開発フローを妨げることなくAIの恩恵を受けられる。

⑯エンジニア以外への展開可能性: 業務全般で応用できる概念

この仕組みはエンジニアだけに限らず、様々な業種で応用できる可能性がある。ナレッジ循環の仕組み化は多くの業務で実現したい理想的な状態。現状ではGitHubのような管理ツールや差分管理の仕組みがない業務では導入が簡単ではない。しかし同じような概念でレビューやフィードバックが回る仕組みを構築できれば、非常に有用となる。AI時代の業務改善のモデルケースとして、様々な分野で参考にできる事例となっている。

⑰APIガイドラインなどドキュメントの重要性

AIが高品質なコードを生成するには、参照するドキュメントの質が極めて重要となる。API開発ガイドライン、コーディングガイドライン、プロジェクトナレッジなどを整備。これらのドキュメントはAIだけでなく、人間の開発者にとっても貴重な資料となる。一度整備すれば、新しいメンバーの教育やオンボーディングにも活用できる。ドキュメント作成の初期投資は必要だが、長期的には大きなリターンを生む。

⑱素晴らしい参考事例: AI開発の未来を示す実践例

この事例は、AI時代の開発手法として非常に参考になる実践例となっている。コンテクストの一元管理、サブエージェントによる分業、ナレッジ循環の仕組み化という3つの柱。単にツールを導入するだけでなく、チーム全体で効率的にAIを活用する仕組みを構築している。育てるほど楽になるという概念は、AI開発の理想的な姿を示している。他の企業や開発チームにとっても、大いに参考になる事例として注目されている。
