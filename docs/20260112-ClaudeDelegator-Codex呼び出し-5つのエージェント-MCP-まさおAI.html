Claude Delegator

Claude CodeからCodex(GPT-5.2)を呼び出して5つの専門エージェントに委任 - 設計・レビュー・セキュリティを自動化

①Claude Delegatorとは: CodexをMCP経由で呼び出すプラグイン

Claude Delegatorは、Claude CodeからCodex(GPT-5.2)をMCP経由で呼び出し、5つの専門エージェントに設計レビューやコードレビュー、セキュリティ分析などを委任できるClaude Codeのプラグイン。GitHubで公開され、X(旧Twitter)を中心に世界中で話題となり、導入も3ステップで簡単。プラグインは何でもできるため、コードを確認してから使用することが推奨される。

②仕組みの解説: Claude Code → MCP → Codex

Claude Delegatorの仕組みは非常にシンプル。Claude CodeがMCP(Model Context Protocol)を経由してCodexに接続し、Codex MCPを通じて「Codex Reply」のように返答を得る。CodexはMCPを提供しており、MCPを立てるとCodexに問いかけて返答をもらうことができる。このシンプルな構造を活用して、Claude CodeからCodexの高い推論能力を借りることが可能になった。

③5つの専門家エージェント: 開発フェーズごとに特化

Claude Delegatorは5つの専門家エージェントを提供する。①Architect(システム設計)、②Plan Reviewer(計画レビュー)、③Scope Analyst(要件分析・曖昧性の発見)、④Code Reviewer(バグ発見・品質向上)、⑤Security Analyst(脆弱性チェック)。開発の流れに沿って、序盤の設計から最終的なセキュリティチェックまで、各専門家が担当領域をカバーする。

④セカンドオピニオンとしての活用: GPT-5.2の推論力を借りる

Claudeは既に十分賢いが、答えが無難に収束していく感覚がある場合がある。GPT-5.2の推論ポテンシャルを活用して、違う視点からのセカンドオピニオンを得て、設計や実装を矯正していくことができる。複数の問題を切り分けていく上で、専門ドメインのエージェントがいることで、より注意したいポイントを重点的にレビューできるメリットがある。

⑤Architect(アーキテクト): システム設計の専門家

Architectエージェントはシステム設計を担当する専門家。プロンプトレベルで設計哲学や注意すべきポイントが詳細に定義されており、Claude Code内のMarkdownファイルとして実装されている。例えばカレンダーアプリの設計を依頼すると、Architect経由でCodexに投げられ、詳細な設計ドキュメントが返ってくる。コードレベルでプロンプトが確認できるため、各エージェントが何を重視しているか透明性が高い。

⑥Plan Reviewer(計画レビュアー): 計画の妥当性を検証

Plan Reviewerエージェントは開発計画のレビューを専門とする。計画系のエージェントとして、何が大事か、どういう観点でレビューすべきかがプロンプトに組み込まれている。開発の初期フェーズで計画を投げると、実現可能性、リスク、スコープの妥当性などを多角的に検証してくれる。Claudeだけでは見落としがちな計画の穴を、Codexの視点から指摘してもらえる。

⑦Scope Analyst(スコープアナリスト): 要件の曖昧性を発見

Scope Analystエージェントは要件分析を担当し、曖昧性を発見する役割を持つ。プロジェクトの要件に対して、何が明確で何が曖昧か、どこに認識のズレがありそうかを指摘。開発の中盤で要件の認識違いが発覚すると大きな手戻りになるが、Scope Analystが事前に曖昧な部分を洗い出してくれることで、手戻りを最小限に抑えられる。Codexの推論力を活かした深い分析が可能。

⑧Code Reviewer(コードレビューアー): バグ発見と品質向上

Code Reviewerエージェントはコードレビューを専門とし、バグの発見と品質向上を担う。実装されたコードに対して、ロジックの問題、可読性の低さ、パフォーマンスの懸念、ベストプラクティスからの逸脱などを指摘。ClaudeがコードレビューをMarkdownファイルとして詳細に定義しており、注意すべきポイントが明確。人間のレビュアーを待たずに、Codex経由で高品質なレビューコメントを即座に得られる。

⑨Security Analyst(セキュリティアナリスト): 脆弱性をチェック

Security Analystエージェントは最終フェーズでセキュリティ分析を実施する。SQLインジェクション、XSS、CSRF、認証・認可の不備、機密情報の漏洩リスクなど、OWASP Top 10を含む脆弱性を網羅的にチェック。セキュリティは専門知識が必要で見落としやすい領域だが、Security Analyst経由でCodexに分析させることで、セキュリティの専門家がいなくても高いレベルの脆弱性診断が可能になる。

⑩2つのモード: リードオンリーとワークスペース書き込み

Claude Delegatorは2つのモードを提供する。①Advisory Mode(リードオンリー): ワークスペースを読み取り専用でレビューし、提案のみを行う。②Implementation Mode(ワークスペース書き込み): ワークスペースに実際に書き込みを行い、コードを修正・生成する。用途に応じてモードを使い分けることで、レビューだけ欲しい場合と、実装まで任せたい場合の両方に対応できる。

⑪システムアーキテクチャ: シンプルで透明性の高い設計

Claude Delegatorのシステムアーキテクチャは驚くほどシンプル。Claude Codeのプラグイン機構を利用し、Markdownファイルで各エージェントのプロンプトを定義し、MCP経由でCodexを呼び出すだけ。Claude Codeのルール(claude.md)の細分化バージョンとして、どういう条件の時にどのエージェントを発動するかが定義されている。複雑なスクリプトは存在せず、Markdown内に変なことが書かれていなければプログラム的な危険性は低い。

⑫処理の流れ: 自動ルーティングとリトライ戦略

Claude Delegatorの処理の流れは次の通り。①Claudeがユーザーのリクエストから適切な専門家を特定、②モードを決定してユーザーに通知、③プロンプトを構築、④MCP経由でCodexのエキスパートを呼び出し、⑤結果を取得して返答。自動ルーティングにより、ユーザーは専門家を明示的に指定しなくても、文脈から適切なエージェントが選択される。リトライ戦略も導入されており、エラー時には自動で再試行する。

⑬導入方法: Codexの準備と3ステップインストール

Claude Delegatorの導入は非常に簡単。事前準備: `npm install`でCodexをインストールしてログインしておく。3ステップ: ①マーケットプレイスからプラグインをインストール、②コマンドを実行してセットアップ、③`claude-delegator-setup`コマンドでセットアップ完了。セットアップが完了すると、`setup`と`uninstall`の2つのコマンドが使えるようになり、いつでも有効化・無効化が可能。

⑭実例: カレンダーアプリの設計をArchitectに依頼

カレンダーアプリの設計をArchitectエージェントに依頼する実例。「カレンダーアプリをArchitectに投げて設計してほしい」とClaude Codeにプロンプトを出すと、詳細を確認された後、MCP経由でCodexに投げられ、詳細な設計ドキュメントが返ってくる。サブエージェントに分割するノリで、Codexに業務を委任できる。手作業でCodexに問い合わせる必要がなく、Claude Code内で完結する。

⑮プロンプトの透明性: Markdownファイルで確認可能

Claude Delegatorの大きな利点は、プロンプトの透明性が高いこと。各エージェントのプロンプトはMarkdownファイルとして実装されており、GitHubでコードレベルで確認できる。ArchitectやCode Reviewerがどういう設計哲学を持ち、何を重視してレビューしているかが明確に記述されている。ブラックボックスではなく、ホワイトボックスなので、安心して使える。必要に応じてプロンプトをカスタマイズすることも可能。

⑯Claude vs Codex: 賢さのポテンシャルを使い分ける

ClaudeとCodex(GPT-5.2)はそれぞれ異なる強みを持つ。Claudeは既に十分賢いが、答えが無難に収束する傾向がある。Codexは推論ポテンシャルが高く、より踏み込んだ提案ができる場面がある。Claude Delegatorを使うことで、Claude Codeをメインで使いながら、設計やレビューの部分だけCodexの推論力を借りるハイブリッド開発が可能になる。両者のいいとこ取りができる。

⑰プラグイン利用の注意点: コードを確認してから使用

Claude Codeのプラグインは強力だが、何でもできるため注意が必要。プラグインをインストールする前に、GitHubでコードを確認し、Markdownファイルに変なことが書かれていないか、特別なスクリプトが実行されていないかをチェック。Claude Codeに「このプラグインは大丈夫か?」と尋ねることも有効。信頼できる開発者のプラグインでも、自己責任で確認してから使用することが推奨される。

⑱Claude Delegatorが示す未来: AIエージェントの分業と協調

Claude Delegatorは単なるプラグインではなく、AIエージェントの分業と協調の未来を示している。単一のAIに全てを任せるのではなく、複数の専門エージェントが協力して開発を進める形態が現実的になった。Claude Codeをオーケストレーターとして、Codexの専門家エージェントを必要に応じて呼び出す構造は、今後のAI駆動開発のスタンダードになる可能性がある。作り方も参考になるシンプルな実装で、カスタマイズの余地も大きい。
