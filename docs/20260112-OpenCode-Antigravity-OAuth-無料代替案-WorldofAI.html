OpenCode + Antigravity OAuth

Claude Code Max無料代替案 - Anthropic制限への回答、エコシステムの競争激化

①2026年1月9日: Anthropicがサードパーティツールをブロック

Anthropicは1月9日、より厳格な技術的保護措置を実施し、利用規約を更新した。OpenCodeなどのサードパーティツールがClaude ProやMaxのサブスクリプション認証をOAuthスプーフィングを通じて使用することを効果的にブロック。Claude Opus 4.5やSonnetに補助金付きの定額料金でアクセスしていた開発者にとって大きな打撃。ログインエラー、アカウント一時制限が発生し、開発者コミュニティに衝撃が走った。

②開発者の不満: 月200ドル払ってもツールが選べない

多くの開発者は、月額200ドルのClaude Max「無制限」サブスクリプションをOpenCode経由でルーティングしていた。OpenCodeはオープンソースCLIで、より良いUX、マルチモデルサポート、強力なプライバシー保証を提供していた。Anthropicの視点では、これは不正なクライアントスプーフィング、公式Claude Code用の消費者プランの悪用、ビジネスモデルへの直接的な脅威だった。しかし開発者側は「高額を払っているのにツールを選べない」と不満を表明。

③Anthropicの正当性: 赤字を止める必要性

Anthropicの決定は理解できる。保護措置なしでは明らかに資金を流出させており、Claudeは自社エコシステム内で使用されるべきという明確なメッセージ。特にAnthropicが独自のIDEや深い開発者ツールの立ち上げを計画している場合(近日公開予定)、この動きは戦略的に必要。短期的にはユーザーに痛手だが、長期的には競合に良い製品をより速く出荷させる圧力となる。

④OpenAIの賢い反応: Codexエコシステムのサポート表明

Anthropicの制限に対し、OpenAIは驚くほど素早く反応した。OpenAIの従業員が公にOpenCodeにアプローチし、「エージェント的コーディングツールの繁栄するエコシステムを100%サポートする」と表明。CodexプランでOpenCode内でCodexを使用できるようにし、全てをロックダウンしないという賢明な動き。Anthropicが締め付ける一方、OpenAIはオープンエコシステムを受け入れる姿勢を見せ、開発者の好感度を獲得。

⑤解決策登場: Antigravity無料枠 + OpenCode統合

Claude Opus 4.5をMaxサブスクリプションでOpenCode内で使えなくなったが、完全無料の解決策が登場した。Antigravityの無料枠をOpenCode経由で統合し、ClaudeやGeminiなどの最先端モデルにアクセス。Antigravityは無料枠があり、レート制限はあるが、すべてのクォータはどこかで強制されるため避けられない。レート制限に達したら自動的にGeminiモデルに切り替わり、Gemini CLIのOAuthもOpenCode内で直接統合されている。

⑥仕組み: Antigravity OAuth + Gemini CLI OAuth

OpenCode + Antigravity OAuthの仕組みは非常にシンプル。Antigravityの無料枠とGemini CLIのOAuthをOpenCode内で直接使用。セットアップは簡単で、ターミナルを開き、指定されたコマンドをコピーして実行するだけ。プラグインがAntigravityモデル定義をエージェントに追加し、`/models`スラッシュコマンドでモデルプロバイダーを検索すると、Antigravity内のすべての異なるモデルが表示される。Cisphus(高度な生成エンジン)も併用可能。

⑦マルチアカウントローテーション: レート制限を賢く回避

Antigravity OAuthの強力な機能の1つは、マルチアカウントローテーション。複数のGoogleアカウントを追加すると、レート制限に達した後、自動的に別のアカウントにローテーション。使用状況に応じて異なるアカウントを同時に使用し、レート制限を実質的にバイパス。複数の無料アカウントを組み合わせることで、長時間の開発セッションが可能になる。完全無料で、公式の無料枠を活用する賢い方法。

⑧セットアップ方法: 自動 vs 手動

セットアップ方法は2つ。①自動セットアップ: ターミナルで指定コマンドを実行し、LLMにプラグイン追加を依頼。②手動セットアップ: configファイルに設定を追加し、`opencode oauth login`で認証、モデルを追加。自動セットアップは数秒で完了し、手動セットアップはより細かい制御が可能。新しいターミナルを開いてOpenCodeを起動し、`/models`コマンドでAntigravityモデルが利用可能になる。非常にシンプルで、技術的なハードルは低い。

⑨代替オプション①: OpenAI Pro/Plusプラン

Antigravity以外にも代替オプションがある。①OpenAI Pro/Plusプラン: OpenCodeのプロバイダータブでOpenAIプロバイダーに切り替え、ChatGPT ProまたはPlusプランを提供してOpenCode内で直接使用。OpenAIはOpenCodeサポートを公言しており、Codexプランとの統合が公式にサポートされている。Opus 4.5ほどではないが、GPT-4レベルの品質でコーディングが可能。OpenAIの前向きな姿勢が評価される。

⑩代替オプション②: MiniMaxコーディングプラン

中国発のAI企業MiniMaxもOpenCode内で使用可能。MiniMaxのプランをOpenCode内で直接使用でき、公式ドキュメントにセットアップ方法が記載されている。GPTやOpusと同じ品質ではないが、かなり安価で、世代の品質は驚異的。Gemini 3.0やOpusなどの他のモデルの出力に基づいてトレーニングされているため、実質的にまともな世代を得られる。コストを抑えたい開発者にとって有力な選択肢。

⑪代替オプション③: ZAI GLM 4.7プラン

中国のZAI(智谱AI)が提供するGLM 4.7もOpenCode内で使用可能。GLMシリーズの第5世代(GLM 5)も近日公開予定で、コーディングプランをOpenCode内で直接使用できる。中国のAIモデルは急速に進化しており、英語圏のモデルに匹敵する品質を提供し始めている。選択肢が多いことは開発者にとって良いことで、自分に最適なものを選べる。競争が激化し、品質が向上している。

⑫クローズドプラットフォーム vs オープンエコシステム

今回の一連の動きは、より大きなトレンドを浮き彫りにした。クローズドプラットフォーム(Anthropicのような自社エコシステム重視)vs オープンエコシステム(OpenAIのようなサードパーティツール許容)。開発者はどちらを選んでいるのか? 現時点では、オープンエコシステムが支持を集めているように見える。柔軟性、選択肢、コミュニティ主導の革新を重視する開発者が多い。Anthropicの制限は理解できるが、長期的にはエコシステムの縮小につながる可能性がある。

⑬競争の激化: Anthropic、OpenAI、Googleの三つ巴

今回の事件は、Anthropic、OpenAI、Googleの三つ巴競争が激化していることを示す。Anthropicが制限を強化すると、OpenAIが即座にオープン化を宣言。Googleは無料のAntigravity枠で開発者を引きつける。各社は開発者のロイヤルティを獲得しようと競い合っており、これは開発者にとって良いニュース。競争により、より良いプラン、より良い価格、より良いツールが生まれる。短期的な混乱はあるが、長期的には業界全体が改善される。

⑭教育目的の情報: 利用規約違反のリスク

動画では明確に警告されている。この方法はサードパーティToS(Anthropic/Googleのクォータやルール)に違反する可能性があり、自己責任で使用すること。不正アクセス、制限のバイパス、違反を推奨するものではなく、教育的・情報提供目的のみ。Google Antigravityの公開されている無料ツールとクォータの教育的議論だが、すべてのToSに準拠しているとは限らない。意図された制限を回避することは推奨されていない。

⑮この修正は1週間で変わる可能性: 猶予期間

動画投稿者は率直に述べている。「この修正は1週間以内に変更される可能性があると個人的に考えている。今のうちに最大限活用すべき」。GoogleやAnthropicが別の方法でこの回避策をブロックする可能性が高い。Anthropicの制限は正当であり、資金流出を止め、自社エコシステム内に製品を保つ必要がある。ただし、この猶予期間中に開発者はAntigravity無料枠を活用できる。いつまで続くかは不明。

⑯エージェント的コーディングツールの未来

この事件は、エージェント的コーディングツール(Claude Code、OpenCode、Cursor、Windsurf)の未来について重要な疑問を投げかける。CLIとIDEの境界が曖昧になり、開発者ワークフローがますますAI駆動になる中、誰がエコシステムをコントロールするのか? 公式ツールか、サードパーティエコシステムか? AnthropicとOpenAIの対照的なアプローチは、業界の分岐点を示している。開発者がどちらを選ぶかが、今後の方向性を決定する。

⑰より良いプランの登場への期待

動画投稿者は楽観的な見方を示す。「今後、より多くのプランが登場し、このテーマに対する競争が激化し、ユーザーのためにより良いプランが出力されると個人的に考えている」。Anthropicの動きは短期的には痛いが、長期的には競合他社により良い製品を出させるプレッシャーとなる。OpenAI、Google、MiniMax、ZAIが次々と代替案を提示しており、競争により価格、品質、柔軟性が改善される。開発者は複数の選択肢から選べるようになる。

⑱結論: 議論の余地があるが、競争は歓迎

Anthropicの決定は議論の余地がある。ビジネスの観点からは完全に正当化されるが、開発者の自由を制限する側面もある。しかし、この動きが競争を促進し、OpenAI、Google、中国のAI企業が代替案を提供する契機となったことは間違いない。開発者はOpenCode + Antigravity OAuth、OpenAI Codex、MiniMax、ZAI GLM 4.7など、複数の選択肢を持つようになった。閉鎖 vs 開放、公式 vs コミュニティの議論は続くが、競争による恩恵は明らか。今後数週間で状況がどう展開するか注目。
