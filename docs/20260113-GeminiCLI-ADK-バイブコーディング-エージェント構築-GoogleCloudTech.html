①Agentverseチャンピオンズ: 4人の専門家チーム

Agentverseシリーズでは、専門家チームを結成する。開発者(Shadowblade)、データエンジニア、プラットフォームエンジニア、アーキテクトの4人。各自が独自のクエストに挑み、最終的に協力して最終ボス、つまり強力で安全でインテリジェントなアプリケーションの構築とデプロイを目指す。

②Shadowbladeのミッション: バイブコーディングの習得

今回は開発者向けのShadowbladeのミッション。Gemini CLIを使ってバイブコーディングの技術を習得し、意図を直接機能的なアプリケーションに変換する。プロフィールWebサイトをゼロから作成し、Git管理やImagenでのAI画像生成を行う。

③Gemini CLI: 意志の延長となるツール

Gemini CLIは単なるツールではなく、あなたの意志の延長だ。Cloud Shell Editorで完全なIDE体験を提供し、自然言語プロンプトでGemini AIモデルのパワーをターミナルに直接取り込む。これがバイブコーディングの核心。

④Yellowフラグ: 確認なしで実行

初めてGemini CLIを召喚する時、Yellowフラグが重要になる。「確認なしで私の代わりに実行して」という意味で、Geminiがバイブコーディングの本能を信頼できることを示す。ラボや非本番環境には最適だが、実装時は注意が必要。

⑤Makaker's Mark: デジタルプレゼンスの構築

開発者は武器だけでなく、シグネチャースタイル「Makaker's Mark」で認識される。単一のパワフルなコマンドで、未来的テーマのプロフィールWebサイトをゼロから作成。index.htmlとstyles.cssを自然言語で指示するだけ。Geminiが意図を理解し、アクションシーケンスを計算してコード生成。

⑥Pythonで簡易Webサーバー起動

ファイル作成後、自然言語で簡易Webサーバーを起動する。Pythonを使ってポート8000でアプリをローカル起動し、Web Previewで確認。洗練されたダークテーマのShadowbladeプロフィールが完成。テーマや色を変更するなど、自分のバイブに基づいてカスタマイズ可能。

⑦バージョン管理: Giteaでローカルソースコントロール

AIが誤動作した時に戻せるよう、バージョン管理を実装する。ローカルGiteaサーバーをスピンアップし、ポート3005で動作確認。進捗を失ったり、元に戻せない間違いを避けるために、特定の瞬間にリバートできるようにする。

⑧MCP Server: モデルコンテキストプロトコル

Gemini CLIをGiteaサーバーに接続するために、MCPサーバーという新しいツールを使用する。MCPサーバーは、AIエージェントが外部ツールやサービスと対話できる専用ポータル。ファイル管理、データベースクエリ、画像生成などの具体的なアクションを実行可能。

⑨settings.json: CLIの中央設定

Giteaサーバーの場所をsettings.jsonファイルに記述する。このファイルはCLIの中央設定で、どの外部ツールやアーセナルにアクセスできるかを指示。Gitea Arsenalがアクティブでコマンド準備完了であることを通知し、Gemini CLIを再起動すると利用可能なツールがすべて表示される。

⑩Imagen: プロフィール画像の生成

Webサイトのプロフィール画像が欠けているという問題を発見。Vertex AIのImagen(画像生成モデル)を召喚し、別のMCPサーバー経由でGemini CLIに接続。Giteaで問題をIssueとして記録し、Gemini CLIに「ピクセルアートスタイルでShadowbladeのポートレートを生成」と指示。生成された画像はSacred Bucket(GCS)に保存される。

⑪Context Engineering: ガードレールの設定

バイブコーディングは楽しいが、バイブがずれると危険。LLMが創造的すぎて1つではなく3つのファイルを変更してしまうかもしれない。規律のない刃は危険なので、エージェント構築前にガードレールを設定する。これがContext Engineingで、LLMに異なるレベルで指示を与えて出力を最適化。

⑫3つのコンテキストレベル: システム・短期・長期

Context Engineeringは3つのレベルで行う。①システムレベル(指示やプロンプト)、②短期メモリレベル(会話履歴や設計ドキュメント)、③長期メモリレベル(ポリシー、スタイルガイド、ガードレール)。gemini.mdファイルをプロジェクトルートに作成し、タイプヒント、ドキュメント文字列、命名規則、依存関係管理などPythonベストプラクティスを記載。

⑬ADK: Agent Development Kit

ガードレールを組み込んだ単一コマンドでエージェントを作成する。バイブコーディングなので、正しいコマンドを得るまで数回のプロンプトが必要かもしれない。エージェントのコアロジックが配置され、MCP tool setを使用する設計になる。mcps_server.pyファイルには、ADKのMCP toolデコレーターを使った関数が定義され、7つの新しい武器をエージェントのアーセナルに追加。

⑭ADK Eval: ゴールデンデータセットでテスト

CI(継続的インテグレーション)パイプラインを構築し、コードのテストとビルドを自動化する。ADK Evalは、エージェントをゴールデンデータセットのチャレンジと追加生成された合成データに対して実行し、正しく応答するかテスト。evalset.jsonファイルが全体のガントレットのブループリントで、test_config.jsonがADKにエージェントのパフォーマンスを判断する方法を指示。正しいツールを使用しているか確認。

⑮Pytest: 自動評価の実装

ADK Evalは手動チェックには優れているが、自動CIパイプラインには別のアプローチが必要。Pytestを使って評価を自動化し、エージェントの出力がロジックの正確性を示すかチェック。Gemini CLIに手動で行ったことを自動化させ、変更があるたびにテストを実行。テストファイルは同じエージェント評価ライブラリを使用。

⑯Cloud Build: CIパイプラインの完成

Gemini CLIにCloud Build.yamlファイルを作成させる。このファイルはCloud Buildに、Pytestスクリプトを実行し、テストが通ったらエージェントをDockerコンテナにビルドしてArtifact Registryに保存するよう指示。次のコマンドでキックオフし、Cloud Consoleで進捗を確認。

⑰Cloud Run: サーバーレスデプロイ

エージェントが鍛えられテストされたら、最終ステップはデプロイ。Artifact Registryのコンテナを取り出し、Cloud Run(サーバーレスプラットフォーム)にデプロイして、エージェントをライブの公開サービスに。これで開発者としてのミッションは完了。

⑱ミッション完了: 次のチャンピオンへ

バイブコーディングでアプリを作成し、外部ツールと統合し、ガードレールで品質を強化し、クラウドにデプロイした。Agentverseでの開発者としての最初の一歩を踏み出し、主要武器のGemini CLIを習得。他のチャンピオン(データエンジニア、プラットフォームエンジニア、アーキテクト)と協力すれば無敵になる。Shadowblade salute！
