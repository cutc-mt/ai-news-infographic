①Pythonの限界: サイドブレーキを引いたまま運転

Pythonは素晴らしい言語で学習が容易、汎用性も高い。しかし正直に言おう。重い処理には遅すぎて、まるでサイドブレーキを引いたまま運転しているようなもの。特に数千、数百万のベクトルに対して純粋なPythonループで類似度スコアを計算すると、信じられないほど遅い。

②もし両方の良いとこ取りができたら: Pythonのシンプルさ + Rustの速度

もしPythonのシンプルな構文を保ちながら、システムプログラミング言語の生のパフォーマンスを利用できたらどうだろうか。答えは究極の組み合わせ: Python × Rust。計算負荷の高い部分だけをRustにオフロードし、コードに大規模な筋力アップグレードを与える。

③実例: インメモリベクトルストア

今回のデモでは、多くの人が馴染みのある例を使用する。インメモリベクトルストアを実装し、純粋なPythonループで数千から数百万のベクトルの類似度スコアを計算する処理の違いを示す。世界に新しいベクトルストアが必要だからではなく、処理能力の違いを完璧に示せるからだ。

④前提条件: Rustをインストール

まずシンプルな前提条件として、Rustをインストールする必要がある。まだの場合は、公式のCLIインストーラーを入手すれば良い。Rustをインストールすると、自動的にCargoも手に入る。一言で言えば、CargoはPythonのpipのようなものだが、依存関係だけでなくビルド、テスト、ドキュメントも処理する最高のパッケージマネージャー。

⑤プロジェクト構造: src/lib.rs

まずsrcという名前のフォルダを作成し、その中にlib.rsファイルを作る。ここにRustコードを書く。PythonがRustと対話する方法が必要で、そのためにPyO3というRustのクレート(ライブラリ)を使用する。PyO3を使えば、ネイティブなPythonモジュールに直接コンパイルされるRustコードを書ける。

⑥Rustコード: structとデコレーション

Rustでは、structを使ってデータ構造を定義する。ここではベクトルとIDを保持するベクトルストアを作る。これをPythonから見えるようにするには、#[pyclass]でデコレートするだけ。次に#[pymethods]を使ってメソッドを実装する。newコンストラクタ、ベクトルを取り込むaddメソッド、重い処理を行うsearchメソッドを持つ。

⑦秘密のソース: Rayonで並列処理

ここが秘密のソースだ。Rayonというライブラリを使用している。par_chunks()を見てほしい。これでCPUコア全体に計算を自動的に並列化する。Pythonでこれを試すのは悪夢だった。今は少し簡単になったが、それでもRustでは文字通り1行のコードで済む。

⑧Maturin: RustベースのPythonパッケージのビルドバックエンド

コンパイルしたコードをPython環境に入れるにはどうするか。そのためにMaturinを使う。MaturinはRustベースのPythonパッケージのビルドバックエンドで、UVのような現代的なツールと完璧に連携する。pyproject.tomlでmaturinをビルドバックエンドとして定義する。

⑨ビルドコマンド: maturin develop --uv --release

ビルドコマンドの実行はこれほどシンプル。maturin develop --uv --release。releaseフラグがここで重要。これがないとコンパイラが最適化を適用せず、コードがはるかに遅くなってしまう。最適化なしでは本当のパフォーマンスは発揮できない。

⑩Pro Tip: 型スタブファイル(.pyi)でVS Code対応

ベンチマークを実行する前に、ちょっとしたプロのヒント。PythonでRustモジュールをインポートすると、VS Codeがそれに下線を引くことがある。コンパイルされたバイナリファイルの中身が分からないためだ。これを修正するには、.pyiという拡張子の型スタブファイルを作成。これによりPylanceが理解し、開発時に完全なオートコンプリートが得られる。

⑪ベンチマーク設定: 100万ベクトル × 1,536次元

さて、楽しい部分だ。1,536次元の100万ベクトルを生成するスクリプトを書いた。3つの競合を比較: ①純粋なPython(標準リストとループ)、②NumPy(データサイエンスの絶対的ゴールドスタンダード、最適化されたCコード)、③我々のRust実装(PyO3とRayonを使用)。

⑫ベンチマーク結果①: 純粋なPythonは約11秒

ベンチマークを実行してみよう。引数は2つ。ベクトルの数と検索するtop kだ。これには数秒、具体的には約40秒かかる可能性がある。実行結果を見ると、Pythonでの計算は約11秒かかった。

⑬ベンチマーク結果②: NumPyは0.02秒

NumPyは高度に最適化されたCコードなので、0.02秒という驚異的な速度。これがデータサイエンスのゴールドスタンダードと呼ばれる理由だ。何年もかけて最適化されてきたベアメタルに近い実装。

⑭ベンチマーク結果③: RustはNumPyより3%遅いだけ

そして我々のRust実装は、NumPyよりわずか3%遅いだけ。しかし最も重要な部分はここにある。RustはPure Pythonより377倍速い。ほぼ400倍の高速化だ。Rustは自分で書いた安全で読みやすいコードでありながら、高度に最適化されたCライブラリと同じリーグでプレイしている。

⑮結論: NumPyに匹敵するRustのカスタムロジック

結論は次の通り。NumPyは基本的にベアメタルなので、倒すのは非常に難しい。しかしRustでカスタムロジックを書き、1行のコードで並列化し、そのパフォーマンスに匹敵できるという事実は、かなり狂気的。PyO3との統合はシームレス。

⑯戦略: パフォーマンスクリティカルなボトルネックだけRust化

重要なのは、すべてを書き直す必要はないということ。パフォーマンスクリティカルなボトルネックだけをRustで書き、残りのロジックはPythonに保つ。これが最強の戦略。Pythonの開発速度とRustのパフォーマンスのハイブリッド。
