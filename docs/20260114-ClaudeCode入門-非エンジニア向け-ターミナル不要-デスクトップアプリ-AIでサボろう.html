①デスクトップアプリでClaude Code搭載: ターミナル不要に

最近、ClaudeデスクトップアプリにClaude Codeが搭載された。アプリ内にClaude Codeが埋め込まれているため、ターミナルでインストールする必要がなく、デスクトップアプリから操作できる。この動画では、世界で一番優しいClaude Code入門を提供する。

②前提条件: Claude有料版($20)が必要

始める前の注意点として、Claudeの有料版(月$20)の課金が必要。WindowsのArm64 PCではClaudeデスクトップアプリ内のClaude Codeが未対応なので、従来のCLI版から入門する必要がある。

③なぜClaude Codeを学ぶべきか: コーディング以外にも使える汎用エージェント

コーディングエージェントは名前に反して、ファイル操作、文書作成、データ整理、ブラウザ自動化など、あらゆるデジタル作業を代わりにやってくれる存在。Claude開発者も「コーディングエージェントがあらゆるデジタル活動に使える汎用エージェントだと気づいた」と述べている。

④Claude Cowork登場: コーディング以外のタスク向け

Claudeから「Cowork」という機能が登場。コーディング以外のタスク向けで、現在はリサーチプレビュー。macOSアプリのみで、月$100以上のMaxプラン契約が必要。基本はClaude Codeと同じで、見た目のインターフェースだけを変えたもの。

⑤CoworkとClaude Codeの違い: 機能とインターフェース

デスクトップ版Claude Codeの強みは、クラウドでエージェントを動かせる、スラッシュコマンドなどショートカットが使える。Coworkの強みは、進捗や途中で生まれた成果物を表示してくれる分かりやすいインターフェース。将来的には住み分けされる可能性。

⑥他のコーディングエージェントとの比較: Cursor・Codex・Antigravity

Cursorは優れたUI、Codexは設計力がピカイチ、Antigravityは無料で使えてエディター機能もある。Claude Codeは、Claude Opusモデルの強さに加え、Hooks、Skills、サブエージェントなど独自機能を次々と打ち出している点が魅力。

⑦AIエージェントの現在値を掴む: Claude Codeで制御を学ぶ

Claude CodeはAIエージェントを制御する機能が豊富。Claude Codeに集中することで、AIエージェントの現在値が掴める。どうやってエージェントをうまく使っていくかは、AI活用力を底上げするスキル・考え方になる。

⑧インストール手順: Claudeデスクトップアプリ + VS Code

Claudeの有料版アカウントを作成後、デスクトップアプリをインストール。Googleログインまたはメールでログインし、左上のナビゲーションから「Code」を選択。初回のみランタイム依存関係をインストール。さらにVS Codeもインストール推奨。

⑨VS Codeが必要な理由: 高機能ファイルビューアー

VS Codeは無料のコーディングエディター。Claude Codeが書いた内容を読むため、後から読み返したい時にチャット履歴を読むより生産性が高い。細かい修正を直接編集できる。高機能なファイルビューアーを入れる気持ちでインストール。

⑩UI構成: 左(履歴)・右(チャット)

画面は左右に分かれる。左エリアは過去のセッション履歴を表示(ChatGPTの会話ログと同じ)。右側がメインのチャットエリア。先頭でフォルダーを指定し、Claude Codeはそのフォルダー内だけで働く。

⑪ローカルとクラウド: 実行環境の選択

右側のボックスで「ローカル」と表示されている部分をクリックすると選択肢が表示。Claude Codeを自分のPCで動かすか、クラウド上で動かすかを選択。ローカルが自分のPCという意味。クラウドで動かすにはGitHub上で管理することが必須。

⑫環境変数: APIキーやデータベースURLを安全に管理

歯車アイコンから環境変数を指定できる。データベースのURLやAPIキー(WebでAIに話しかける秘密の鍵)を保存。コードに直接書くのは危険なので、環境変数として管理するのが一般的。

⑬チャット欄の機能: 画像添付・スラッシュコマンド・モデル選択

チャット欄ではテキスト以外に画像を添付可能。デザインや問題点を伝えやすい。3点リーダーからスラッシュコマンドの利用とモデル選択(Opus・Sonnet・Haiku)が可能。Opusが一番賢いが少し時間がかかる、Haikuが一番速い。

⑭スラッシュコマンド: 便利機能のショートカット

チャット欄でスラッシュを入力するとコマンドが表示される。Claudeがデフォルトで用意したコマンドと自分で作れるコマンド(スキル)の2パターン。よく使うのは「/compact」で、会話履歴が長くなった時に圧縮(要約)する。

⑮実行モード: 確認・実行・プラン

実行ボタン左の「確認」をクリックすると3つのオプションが表示。確認モード(都度確認)、実行モード(確認なしでガンガンやる)、プランモード(実装前に計画を立て、仕様の質問をする)。最初は確認モード推奨。プランモードは規模の大きいタスクに有効。

⑯ハンズオン①: 文章作成 - Claude.mdでルール化

「Claude Codeとは何かを優しく解説する記事をMarkdownで書いて」と指示すると記事を作成。Claude.mdファイルを作成すると、Claude Codeが作業前に必ず読む。関西弁で書くなどのルールを記述すれば、毎回守ってくれる。ドキュメントを育てるのが重要。

⑰ハンズオン②: スキル作成 - カスタムコマンドで効率化

自分でカスタムコマンドを作成できる。これを「スキル」と呼ぶ。Skillsというフォルダー内にMarkdownファイルとして作成され、スラッシュコマンドから呼び出せる。よく行う作業をスキル化すると毎回の指示が楽になり、品質も安定。

⑱ハンズオン③: Chrome拡張開発 - SNS見すぎ防止

「SNS見すぎちゃうので自分を止めるChrome拡張を作って欲しい」とプランモードで依頼。AIが計画を立て、仕様を確認し、数分でChrome拡張が完成。ブロックするサイトを追加・削除できるシンプルで可愛いデザイン。言葉を伝えるだけで開発が完了。
