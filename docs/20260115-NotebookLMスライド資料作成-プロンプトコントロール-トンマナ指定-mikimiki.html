NotebookLMスライド資料作成

内容・デザインを自由にコントロールする完全攻略法
mikimiki web スクール 21分 2026年1月15日

①高品質だがコントロール困難: 課題認識

NotebookLMのスライド資料作成はクオリティは高いが具体的な内容やデザインのコントロールが難しい。「この部分に観点を置いて作成」といった指定ができない。情報整理や社内共有には使えるが、外部資料としては細かくコントロールしたいという課題がある。

②Nano Banana統合: 最強画像生成AI

GoogleのNano BananaがNotebookLMに搭載され、スライド資料が最強精度に進化。画像生成機能とGemini 3の文章把握能力を組み合わせることで、ワンクリックで高品質な資料を作成できる。

③ソース選択の自由度: ハルシネーション回避

自分でソースを選択できるためハルシネーションを起こしにくい。ファイル、ウェブサイト、YouTube、Googleドライブ、新規Web検索など多様なソースに対応。膨大なAI情報からではなく、選択したソースからアウトプット。

④Studio機能: 多様なアウトプット

Studioでは音声解説、動画解説、マインドマップ、スライド資料、インフォグラフィックを生成可能。月に2個ペースで新機能が追加される驚異的な進化速度。データテーブルなど最新機能も続々登場。

⑤10分で完成: Google財務報告書の事例

英語の財務報告書（PDF）から約10分で15枚の高品質スライドを日本語で生成。アイコン、色、グラフの使い方まで分かりやすく作成。プロンプトなしのワンプッシュでこのクオリティは革命的。

⑥構成プロンプト: 詳細アジェンダ作成

資料構成を指定するためのプロンプトをNotebookLMに作ってもらうのがポイント。テーマ、ターゲット、想定ボリューム、スライドフォーマット（フローチャート、ロジックツリーなど）を指定して詳細なアジェンダを生成。

⑦フォーマット指定: グラフ種類も自由

プロンプトでウォーターフォールチャート、サマリーボックス、ブレットポイントなど具体的なグラフ形式を指定可能。各スライドの主要データ、引用すべき情報、ビジュアル形式まで細かく指定できる。

⑧ペンマーク編集: プロンプト挿入

スライド資料の右上ペンマークから「作成するスライドについて説明」欄にプロンプトを貼り付け。この方法で資料を細かく指定してアウトプット。長さ（長め・デフォルト・短め）も調整可能。

⑨トンマナプロンプト: デザイン仕様書生成

資料の目的、ターゲット、色の方向性、目指したい雰囲気を指定してトンマナを作成。「誠実なネイビー」「高級感」などを入力すると、デザインコンセプト、カラーパレット、RGBコード、使用イメージまで詳細に出力。

⑩カラーコード自動生成: 配色の専門知識不要

ベース色を決めるだけでアクセントカラー、ゴールド・シルバーの配色、RGBコードまで自動提案。誠実・知性・品格を体現するネイビーを主軸に、プレミア感を演出する配色が完成。レイアウトの専門知識なしでプロレベルのデザインに。

⑪タイポグラフィ指定: フォント規定も詳細

日本語フォント、英語フォント、フォントサイズ、階層ごとのデザイン面まで指定可能。トンマナプロンプトにより、一貫性のある高品質なタイポグラフィが実現。役員報告書にふさわしい重厚感を演出。

⑫二重プロンプト: 内容とデザインの統合

構成プロンプトとトンマナプロンプトを両方貼り付けて生成することで、内容の詳細指定とデザインの統一が同時に実現。デザインだけではなく内容も含めた完全なコントロールが可能に。

⑬メモからソース化: カスタム構成の活用

生成したアジェンダを「メモに保存」→「ソースに変換」することでカスタムソースとして使用可能。元のGoogleレポートのリンクを外し、カスタム構成をソースにすることで、プロンプトなしでも詳細指定が実現。

⑭ソースコントロール: 元データから完全指定

細かい資料構成を作成したドキュメントをソースとしてアップロードする方法。自分で作ると大変だが、NotebookLMに作成してもらえば効率的。前のチャット履歴を削除してから新しいプロンプトで作成。

⑮プロンプト表示機能: 忠実性の確認

生成後、「プロンプト表示」ボタンで入力したプロンプトに沿っているか確認可能。エグゼクティブサマリ、KPIハイライト、ウォーターフォールチャートなど、指定した要素が完璧に再現されているかチェックできる。

⑯Gemini 3との比較: NotebookLMの優位性

同じプロンプトでGemini 3のキャンバスモードと比較すると、NotebookLMの方が数字のレイアウト、全体の一貫性、図表の使い方で圧倒的に優秀。Geminiは文字はみ出し、背景同化などの問題あり。

⑰図解 vs 画像: 資料向きの出力形式

Geminiは画像生成されやすいが、NotebookLMは図・イラスト生成されやすい。資料作成には図解形式の方が適しており、この点でもNotebookLMが有利。同じGemini 3を使用していても出力形式に違いがある。

⑱主流化の予測: 資料作成の未来

NotebookLMの進化が凄まじく、今後の資料作成はNotebookLMが主流になると予測。スタジオ機能の急速な進化、月に2個の新機能追加ペース、Geminiとの品質差から、資料作成のデファクトスタンダードになる可能性が高い。
