Vercel Agent Browser CLI

95%の初回成功率 - Playwright MCPを大幅に凌駕
Cole Medin 15分 2026年1月19日

①ブラウザ自動化の重要性: エージェントの自己検証

コーディングエージェントが自分の作業を検証できることが信頼性の鍵。フロントエンドを実際にユーザーのようにナビゲートし、問題を発見できなければ、手動検証の負担が残る。

②検証データ: 95% vs 75-80%の成功率

厳密なテストの結果、Agent Browser CLI: 95%、Playwright MCP: 80%、Chrome DevTools MCP: 75%の初回タスク完了率。リトライを1回許可すればほぼ100%に。

③従来手法の問題: セレクターとマッチング

Playwright MCPなどはセレクターとサーチによる非決定論的マッチング。アクセシビリティツリーを取得しても、マッチング失敗でリトライが頻発。

④新しいアプローチ: スナップショット＋参照

Agent Browserはサイト全体のスナップショットを取り、要素を@e1, @e2などの参照に凝縮。エージェントはこの簡潔な構造を見て、どの要素を操作するか決定。

⑤トークン効率と速度の向上

凝縮された参照によりトークン使用量が大幅に削減。1回のCLI呼び出しで要素をクリックでき、非常に高速。サーチやマッチングのオーバーヘッドがない。

⑥Vercelの哲学: Less is More

Vercelは「エージェントの邪魔をしない」哲学を採用。17個の専門ツールを2個に減らしたText-to-SQLエージェントで、80%→100%の成功率を達成。

⑦D0ero研究: ツール数削減の効果

VercelのD0ero（Text-to-SQLエージェント）研究。多くのツールで細かく制御するより、少ないツールで柔軟性を与えた方が成功率が高い。ステップ数も減少。

⑧実践的ワークフロー: 機能実装後の検証

Cole Medinの構造化プランでは、コード実装後にWebサイトを起動し、ユーザーのようにナビゲート、スクリーンショット撮影、リグレッションテストまで自動化。

⑨Claude Codeスキル統合

skill.mdをClaude Codeのskillsディレクトリに配置するだけで統合完了。エージェントがAgent Browserの全コマンドを理解し、適切に活用できるようになる。

⑩デモ: WordCloudアプリの検証

ライブデモでWordCloudアプリをコードベース理解→フロントエンド起動→ナビゲーション→スクリーンショット撮影まで自動実行。緑チェックマークが並ぶ。

⑪複雑なサイトでの優位性

シンプルなサイトではPlaywright MCPも同等だが、Amazonなど大規模サイトでAgent Browserが真価を発揮。商品検索、スクロール、複雑な操作でも安定。

⑫サイレントフェイラーの違い

Playwright MCPではエラーメッセージなしに「これは動かない、別の方法を試す」という挙動が多発。Agent Browserではこの「サイレントフェイラー」が大幅に減少。

⑬インストール: 数コマンドで完了

npm install、agent-browser installの2コマンドでセットアップ完了。LinuxではWSL経由での使用を推奨（Windows直接は不安定な場合あり）。

⑭オープンソース: 完全無料

完全無料でオープンソース。Vercelとの提携なしで純粋におすすめ。5-10分試すだけで価値がわかる。GitHubリンクは説明欄に。

⑮Leapter: 決定論的ワークフローの補完

エージェントに柔軟性を与えたくない場合はLeapterで決定論的ワークフローをツール化。価格計算など、ビジネスロジックをハルシネーションなしで実行。

⑯アーキテクチャの違い

従来: 「要素をサーチ→マッチング→操作」。Agent Browser: 「スナップショット→参照取得→直接操作」。エージェントがサイト構造を直接理解。

⑰リトライ耐性: 1回で十分

95%の初回成功率に加え、1回のリトライを許可するとほぼ100%に到達。他のツールでは複数回のリトライが必要で、その間にコンテキストが汚染される。

⑱結論: エージェントコーディングに必須

「初めて"Great"と言えるブラウザ自動化ツール」とCole Medinが評価。AIコーディングワークフローにブラウザ自動化を組み込むなら、これが最良の選択。
