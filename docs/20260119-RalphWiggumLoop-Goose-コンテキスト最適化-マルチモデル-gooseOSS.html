Ralph Wiggum Loop × Goose

コンテキスト最適化とマルチモデルパターンで自律AIを強化
goose OSS 38分 2026年1月19日

①Gooseとは: オープンソースAIエージェント

Gooseはオンマシンで動作するオープンソースAIエージェント。タスクを自動化し、GitHub連携、JetBrains IDE統合、Google Drive接続などの拡張機能をMCPで提供。デスクトップアプリとCLIの両方で利用可能。

②Ralph Wiggum Loopとは: 新しいエージェントパターン

Jeffrey Huntley氏が開発したエージェントオーケストレーションパターン。名前はシンプソンズのキャラクターから。「賢くはないが粘り強い」という特徴を表現。標準エージェントループの問題を解決する。

③標準エージェントループの問題: コンテキスト肥大化

標準のエージェントループはコンテキストウィンドウが肥大化する問題がある。失敗と再試行を繰り返すたびに、ノイズが蓄積されモデルの精度が低下する。

④コンパクションの限界: 品質劣化の連鎖

コンパクション（圧縮）はYouTubeに動画を何度もアップロード・ダウンロードするようなもの。重要なコンテキストが失われ、品質が劣化していく。Jeffrey氏はこれを「非常にロッシー」と表現。

⑤コンテキスト圧縮不安: モデルの挙動変化

コンテキストウィンドウが限界に近づくと、モデルが「チート」を始め、ハルシネーションが増加する傾向がある。これが「コンテキスト圧縮不安」と呼ばれる現象。

⑥Ralph Loopの解決策: ファイルベースの状態管理

各イテレーション後に新しいセッションを開始し、コンテキストをファイルに保存。新セッションはそのファイルを読んで作業を継続。これでコンテキストウィンドウが常にクリーンに保たれる。

⑦設計思想: 記憶ではなく再評価で賢くなる

「モデルを記憶で賢くするのではなく、再評価で賢くする」という哲学。同じセッション内でループするのではなく、毎回フレッシュな視点で問題を見直す。

⑧マルチモデルアプローチ: ワーカーとレビュワー

Gooseのマルチモデル機能を活用し、1つのモデルが作業し別のモデルがレビュー。同じモデルが自分の作業を評価するバイアスを避けることで、品質が向上。

⑨実装構成: レシピとBashスクリプト

Gooseのレシピ機能とBashスクリプトで実装。ralph-loop.sh（オーケストレーター）、ralph-work.recipe（ワーカー）、ralph-review.recipe（レビュワー）の3ファイル構成。

⑩状態ファイル: イテレーション間の連携

task.md（タスク説明）、iteration.txt（現在のイテレーション番号）、summary.md（ワーカーの作業サマリー）、result.txt（レビュー結果）、feedback.md（レビュワーのフィードバック）で状態を管理。

⑪モデル選択: Opus + GPT-5.2の組み合わせ

デモではワーカーにClaude Opus 4.5、レビュワーにGPT-5.2を使用。異なるプロバイダーのモデルを組み合わせることで、多角的な評価が可能に。

⑫デモ: Electronブラウザアプリの構築

デモとしてElectron + React + TypeScript + ViteでカスタムブラウザをRalph Loopで構築。通常は複数プロセス必要な起動がスキル一つで完了。

⑬完了判定: レビュワーの「Ship」宣言

レビュワーが「Ship」または「Revise」を判定。Shipなら完了、Reviseなら次のイテレーションへ。result.txtにステータスが保存される。

⑭コスト効率: 最初の実装との比較

最初のRalph Loop実装（Anthropicスタイル）では$24のトークンコストと51%のコンテキスト使用。正しい実装後は大幅に改善。セッション分離がコスト削減の鍵。

⑮ローカルLLMの可能性: レビュワー向け

ローカルLLMはツールコールが苦手だが、レビュワーとしてなら活用可能。ファイルを読んで「良い・悪い」を判定するだけなら、ローカルモデルでも十分機能する。

⑯Chat Recall MCP: セッション間の記憶

GooseのChat Recall MCPでセッション間の情報を参照可能。「前のセッションで何をした?」と聞けば、過去の作業を思い出してくれる。

⑰ユースケース: 決定論的タスクの自動化

Ralph Loopは探索的・ブレインストーミング用途ではなく、決定論的なタスク自動化向け。Gas Townのような長時間自律実行シナリオに最適。

⑱今後の展望: エージェントパターンの進化

Ralph Loopはエージェントオーケストレーションの新しいパラダイム。Jeffrey Huntley氏の研究は今後も進化し、より信頼性の高い自律AIシステムの構築に貢献する。
