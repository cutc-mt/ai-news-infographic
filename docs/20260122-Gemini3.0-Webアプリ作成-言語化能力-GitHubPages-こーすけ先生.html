Gemini 3.0でWebアプリ作成

再現性あり！言語化能力とオーケストレーションが求められる時代
こーすけ先生のGoogle塾 21分 2026年1月22日

①Gemini 3.0とは: 化け物級のAI

2025年11月に発表されたGeminiの最新モデル。2ヶ月経っても全く飽きないほど、できることの可能性が広がり続けている。

②コーディング能力の飛躍: 誰でも作れる時代

コーディング能力が爆発的に飛躍し、非エンジニアでも作りたいものが作れる時代に。拡張機能もあっという間に開発可能。

③以前との違い: 矛盾やエラーが激減

以前はファイルを分割すると矛盾が発生していたが、今は大規模開発も可能なレベルに進化。AIエージェントに近づいている。

④基本の指示: キャンバスON＋HTML

キャンバスをONにしてHTML構造で作成を指示するだけ。マテリアルデザインや色合いの希望を伝えれば高品質なUIが完成。

⑤実例: PDF分割ツール

「PDFを分割するをHTMLで実現して欲しい」と指示するだけで10分以内に分割・結合機能付きツールが完成。

⑥プレビュー機能: すぐに動作確認

コード生成後すぐにプレビュー画面で動作確認が可能。触ってみて問題があればすぐにフィードバックできる。

⑦Answer Now機能: 思考を強制終了

Proモードで思考が長すぎる時は「Answer Now」で強制的に回答させることも可能。単純な質問には便利。

⑧求められる能力①: 言語化能力（指示）

業務や仕様を的確にGeminiに伝える能力が重要。何を作りたいか、どんなデザインが欲しいかを言葉で表現する力。

⑨求められる能力②: 言語化能力（確認）

作ったものを触りまくって問題点や追加要望を的確に伝える能力。エラーや改善点を正確にフィードバックする力。

⑩反復で改善: 追加要望も簡単

「結合機能も追加して」と伝えればすぐに機能追加。エラーがあっても画面キャプチャで指摘すれば修正してくれる。

⑪GitHub Pagesで公開: 無料で世界へ

完成したHTMLをGitHub Pagesで無料公開可能。index.htmlを作成してPages設定をオンにするだけ。

⑫GitHub手順: 3ステップ

①リポジトリ作成→②index.html作成→③Pages設定で完了。1〜2分でデプロイされて世界中からアクセス可能に。

⑬Googleサイトとの違い: 用途で使い分け

GitHub PagesはHTMLをそのままデプロイでき美しい。Googleサイトは限定公開が簡単という違いがある。

⑭限定公開: Googleサイトが有利

社内向けツールならGoogleサイトが便利。Googleアカウント指定で簡単に限定公開できる。

⑮求められる能力③: オーケストレーション

作られたものをどう配置・配備するか俯瞰して適材適所を見抜く能力。GitHub PagesかGoogleサイトかを判断する力。

⑯新しい役割分担: Geminiが実装者

Gemini 3.0時代はGeminiが実装者、人間は指示者兼確認者。この役割分担がニューノーマル。

⑰セキュリティもAIに: 脆弱性診断

実装後は「ホワイトハッカーとして脆弱性を診断して」と依頼。セキュリティチェックもGeminiに任せられる。

⑱まとめ: 指揮者として全体を束ねる

言語化能力（指示＋確認）とオーケストレーションが求められる時代。Geminiを使いこなして作りたいものを形にしよう。
