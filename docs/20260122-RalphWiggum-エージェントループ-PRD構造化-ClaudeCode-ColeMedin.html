Ralph Wiggum再評価

構造化PRDでProof of Conceptを量産する哲学
Cole Medin 21分 2026年1月22日

①前言撤回: Ralph Wiggumを見直した

以前「バイブコーディングの極致」と批判したが、Geoffrey Huntley（本当の開発者）のコメントを受けて認識を改めた。Anthropicのプラグインは本物ではなかった。

②本質は哲学: フレームワークではない

Ralph Wiggumはツールではなく「哲学」。構造化された計画プロセスを中心に据えることで、長時間実行タスクを管理する考え方。

③真の目的: Proof of Concept用

本番アプリ開発用ではなく、アイデア検証やアーキテクチャ・技術スタックの確認が目的。エンタープライズでも有用なユースケース。

④PRD: 構造化計画の中核

PRD（Product Requirements Document）がループの中核。作りたいもの、機能リスト、検証ステップを明確に定義する。

⑤仕組み: Claude Codeをbashループで実行

Ralph Wiggumの実体はbashスクリプトでClaude Codeを繰り返し実行するシンプルな構造。各ループで1つのタスクを処理する。

⑥Activity Log: 長期記憶の役割

activity.mdファイルがループ間の長期記憶として機能。各ループ終了時に更新され、次のエージェントへの入力となる。

⑦Prompt.md: 毎回送信されるコンテキスト

prompt.mdは各ループで毎回送信されるコンテキスト。PRDを読み込むよう指示し、作業範囲を明確に伝える。

⑧ブラウザ自動化: Vercel Agent Browser CLI

エージェント自身が作業を検証できるようブラウザ自動化を導入。Playwright MCPなども使用可能。

⑨セキュリティ: settings.jsonで制限

サンドボックスモードで実行し、実行可能なコマンドを制限。Anthropicプラグインには含まれていない重要な安全機能。

⑩終了条件: 全機能がpass=true

PRDの各機能に「passes」フラグがあり、全てがtrueになるまでループは継続。これが唯一の終了条件。

⑪/create-prd: 5〜10分で計画完成

Claude Codeのスラッシュコマンドで質問形式でPRDを自動生成。多肢選択または自由入力で技術スタックや機能を決定。

⑫Anthropic Harnessの知見を活用

機能リストの構造はAnthropicの長時間実行エージェント・ハーネスと同形式。カテゴリ、説明、検証ステップ、passフラグで構成。

⑬デモ: AIハビットトラッカーを構築

非トリビアルなアプリを実演。Clerk認証、Neonデータベース、OpenRouter LLMを使用したAIコーチ付き習慣管理アプリ。

⑭一晩で完成: 約5時間の実行

19タスクのPRDを一晩（約5時間）で完全に実装。検証込みで全機能がpasses=trueに。50イテレーションを設定。

⑮低コスト: LLMテストは約7セント

Claude Haiku 4.5を使用し、エージェントコーチとのやり取りは約7セントで完了。API制限（$5）を設定して暴走を防止。

⑯手動検証も推奨: 最終確認は人間

自動検証があっても手動での最終確認を推奨。ただし回帰テストも完了しているため、品質は高い状態で引き渡される。

⑰テンプレート公開: すぐに始められる

GitHubでクイックスタートテンプレートを公開中。必要なファイル構成が揃っており、10分で計画開始可能。

⑱まとめ: アイデア検証ツールとして最適

構造化PRD + ループ + 自動検証の組み合わせでProof of Conceptを量産。本番開発前のアイデア検証に最適なアプローチ。
