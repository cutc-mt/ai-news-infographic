Claude Code TASKS機能

Ralph Loopを置き換える公式マルチエージェント協調機能
AICodeKing 7分 2026年1月23日

①TASKSがTODOを置き換え

Claude CodeのTODO機能がTASKS（タスク）に進化。単なる名前変更ではなく、TODOでは解決できなかった問題を解決。

②背景: Opus 4.5の自律性向上

Opus 4.5が長時間の自律動作に対応し、単純なタスクにはチェックリスト不要に。しかし大規模プロジェクトにはTODOでは不十分だった。

③TODOの問題点

TODOはメモリ上に存在し、セッション終了で消失。サブエージェントは親の作業状況を把握できず、複数セッション間の連携も不可能だった。

④特徴1: ファイルシステムに保存

タスクは~/.claude/tasksに保存。セッション間で永続化され、独自ユーティリティの構築も可能。

⑤特徴2: 依存関係サポート

TODOはフラットなリストだったが、TASKSはタスク間の依存関係を定義可能。「AとBが完了したらCを開始」などが表現できる。

⑥特徴3: セッション・エージェント間協調

複数のサブエージェントや複数のClaude Codeセッションが同じタスクリストを共有。更新は全セッションにブロードキャスト。

⑦並列化が大幅に改善

認証システム、DBスキーマ、テストを別々のエージェントが並列で担当。重複作業なし、衝突なしで効率的に進行。

⑧Ralph Wiggumの公式版

Ralph LoopはClaude Codeを「完了するまで終了させない」ハックだった。TASKSはその哲学を公式にアーキテクチャ統合。

⑨Ralphを超える拡張性

Ralphは1タスク1ループだったが、TASKSは複雑な依存関係を持つ10タスクを複数セッションで分担可能。

⑩環境変数で共有設定

CLAUDE_CODE_TASK_LIST_ID=名前を設定して起動すると、同じIDのセッション間でタスクリストを共有。

⑪CLIモードとSDKにも対応

claude -p（CLIモード）やAgent SDKでも動作。自動化パイプラインでの複数インスタンス協調が可能。

⑫小さなタスクには不要

関数のリファクタやバグ修正程度ならTASKSなしで直接依頼。Claude自身が十分賢くなっている。

⑬推奨ワークフロー

要件定義 → Claudeにタスク分解を依頼 → 依存関係を設定 → 複数サブエージェントを起動して独立ブランチを担当。

⑭Ralphieはまだ有用

タスク協調は公式機能になったが、Git Worktree分離と自動PR作成にはRalphieが引き続き便利。

⑮課題: ドキュメント不足

Xで発表されたが公式ドキュメントがまだ少ない。実験と発表からの推測で使用している状態。

⑯要望: ビジュアルUI

現在はターミナル表示のみ（Ctrl+Tで非表示）。タスク進捗と依存関係のダッシュボードが欲しい。

⑰要望: スラッシュコマンド

環境変数設定はやや面倒。/tasks share groceriesのようなコマンドが欲しい。

⑱AIは「ヘルパー」から「チーム」へ

AIコーディングツールの方向性はチームとしてのAI。チームにはプロジェクト管理が必要。TASKSはその答え。
