GitHub Copilot SDK プレビュー公開

AIツール・エージェント・ワークフローを自分で構築できる新SDK
Shweta Lodha 12分 2026年1月23日

①GitHub Copilot SDKが公開

GitHub Copilot CLIがパブリックプレビューとして公開された。開発者がAIツール、エージェント、ワークフローを自分で構築できるようになる。

②エージェント構築の重荷を軽減

エージェント構築で大変なコンテキスト管理、ツールオーケストレーション、ルーティング、MCP連携をSDKが代わりに処理してくれる。

③必須の前提条件

GitHub Copilotサブスクリプションが必要。Windowsの場合はPowerShell 6以上が必須。CLIのインストールも事前に行う。

④CLIインストール方法

Windows: winget install github.copilot、Mac: brew、npm経由でもインストール可能。3つの方法から選択できる。

⑤5つの言語でSDK提供

.NET、Python、Node.js、TypeScript、Goに対応。SDKクライアントはCopilot CLIとJSON RPCで通信する。

⑥SDKの主要機能

認証、モデル管理、MCPサーバー、カスタムエージェント、ストリーミング、チャットセッションなどの重要機能を提供。

⑦3つのサブスクリプション

個人・ビジネス向けにFree、Pro、Pro Plusの3カテゴリ。無料トライアルから始めてProへアップグレード可能。

⑧使い方のデモ

Python仮想環境を作成し、pip install github-copilot-sdkでインストール。copilotコマンドでCLI起動。

⑨信頼確認の仕組み

CLI起動時に「Do you trust?」と確認される。ディレクトリ内のファイルを信頼するかユーザーが判断して承認する。

⑩GitHub MCPサーバー自動接続

承認後、GitHub MCPサーバーが自動的に接続される。開いているリポジトリに関する質問が可能になる。

⑪ツール実行には承認が必要

ファイル編集などのツール実行時はユーザーの承認を求められる。MCPサーバー呼び出し時も同様に確認がある。

⑫モデル切り替え機能

/model コマンドで利用可能なモデル一覧を表示。GPT-4.1などから選択可能。Pro Plus契約者はさらに多くのモデルが使える。

⑬SDKでプログラム制御

CopilotClientをインポートしてインスタンス作成。CLIサーバーを自動起動して接続を確立する。外部URL指定も可能。

⑭ストリーミング対応

streaming=Trueを設定すると、レスポンスが1文字ずつ順次返ってくる。リアルタイム表示が可能に。

⑮MCPサーバー連携

ローカルまたはリモートのMCPサーバーと連携可能。SDKを通じて様々なシナリオに対応できる。

⑯プレビュー版の注意点

現在はプレビュー段階のため、今後破壊的変更が入る可能性がある。実験的に試すには十分使える状態。

⑰開発者の負担を大幅軽減

認証やモデル管理などの面倒な部分をSDKに任せて、開発者は本来のアプリケーション開発に集中できる。

⑱今後の展開

SDKを使ったAIエージェント作成チュートリアルが今後公開予定。ツール連携やMCPサーバー活用の詳細が学べる。
