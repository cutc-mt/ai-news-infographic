Everything Claude Code 開発実況

Plan→TDD→レビュー→リファクタ→ドキュメント更新を同時並行で回す開発ワークフロー
いけとも ch 25分 2026年1月25日

①Everything Claude Codeとは

CursorにClaude Codeエクステンションを導入し、スラッシュコマンドでプラン、TDD、コードレビュー、ドキュメント更新を実行できる環境を構築。

②導入方法

GitでClaude Codeをクローンし、ユーザーフォルダの.claudeに配置。agent-commands、hooks、rules、skillsを入れると全プロジェクトで使える。

③同時並行の基本

プラン制限が来るまで複数のClaude Codeを同時に立ち上げ可能。Windows+Pで新しいセッションを起動。3〜4体を同時に回せる。

④開発サイクルの流れ

/planでプラン作成→/tddでテスト駆動開発→/code-checkでレビュー→/update-docsでドキュメント更新。このサイクルを繰り返す。

⑤プラン分割の重要性

プラン段階で調査するだけで5〜10万トークン消費。プランと実装を別セッションで分離することでトークンを節約。

⑥プランファイルの活用

プランはplansフォルダに保存。必要なファイルと対象範囲を明記しておくことで、実装時に不要なファイルを読まずにトークンを節約。

⑦テスト駆動開発（TDD）

/tddコマンドでプランファイルを指定して実行。テストが320件（フロント）、750件（バック）と大量に自動生成される。

⑧コードレビュー

TDD完了後、/code-checkで実装内容が正しいかレビュー。問題なければ次のステップへ進む。

⑨リファクタリング

/refactor-codeでコードを綺麗に整理。課題がある場合はリファクタリングを実施してから次へ進む。

⑩ドキュメント更新

/update-docsで開発者向けドキュメントと運用向けランブックを自動更新。ドキュメントが常に最新状態に保たれる。

⑪コンテキスト管理

右下に表示されるパーセンテージでトークン消費を確認。100%になると過去が圧縮されて処理が荒くなる可能性あり。

⑫/clearの活用

セッションをクリアしてトークンを0に戻す。プラン完了後にクリアして、新セッションでTDDを開始するのが効率的。

⑬競合に注意

同じファイルを複数のセッションで編集すると競合する。対象範囲が異なれば同時実行可能。左側のCursorはAskモードで質問専用に。

⑭CLAUDE.mdでの制御

.claudeフォルダのclaude.mdに「プラン後は勝手に開始しないで」などの指示を記載。Claudeの挙動をカスタマイズ。

⑮プロジェクト規模

5日間の開発でファイル数129、行数3.3万行（テスト含む）、ソースコードは4.6万行に成長。

⑯料金プラン

$100のMaxプランでは足りず$200に変更。同時並行を始めてから消費が激増。楽しすぎて深夜までやってしまう。

⑰学びと気づき

最初は1セッションで全部やっていたが、Claudeに使い方を相談して分割と並行実行を知った。わからないことはClaude自身に聞くのが有効。

⑱まとめ

仕様駆動開発→テスト駆動開発→コードレビュー→ドキュメント更新のサイクルを複数並行で回すことで、品質を担保しながら高速開発が可能に。
