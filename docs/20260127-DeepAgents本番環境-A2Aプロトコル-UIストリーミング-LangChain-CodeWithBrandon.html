DeepAgents本番環境構築

A2Aプロトコルを使ったUI統合とストリーミング対応のプロダクショングレードエージェント
Code With Brandon 8分 2026年1月27日

①ターミナルだけでは不十分

多くの人がDeepAgentをターミナルで実行しているが、本番環境にはUIとAPI統合が必要。ポートフォリオプロジェクトにも最適。

②DeepAgentの構造

オーケストレーター＋2つのサブエージェントで構成。例ではアメリカンフットボール専門家と猫専門家を実装。

③LangChainのDeepAgentsパッケージ

LangChainチームのdeep agentsパッケージを使用。シンプルにエージェントを構築できる。

④A2Aプロトコルとは

Agent to Agent プロトコル。エージェントを外部に公開するための標準化されたインターフェースを提供。

⑤ストリーミングエンドポイント

A2Aは複数のエンドポイントを提供。今回はストリーミングエンドポイントを使用してリアルタイム応答を実現。

⑥タスクストアの仕組み

質問がタスクオブジェクトに変換されキューに追加。タスクストアがground truthとして全タスクを管理。

⑦イベントリスニング

A3メソッドでon_tool_start、on_tool_endイベントをリッスン。サブエージェントの中間結果をストリーミング。

⑧サブエージェントはツール扱い

LangChain DeepAgentsはサブエージェントをツールとして統合。そのためtool_startとtool_endイベントで監視。

⑨on_chat_model_endイベント

オーケストレーターが最終回答を生成した時に発火。タスクを完了としてマークしAPI経由でストリーム。

⑩proxy.pyの役割

2つの機能を同時に担当：FastAPIによるプロキシサービスとストリーミングエンドポイント、そしてUIのホスティング。

⑪エージェントカード解決

プロキシは最初にA2A Card Resolverクラスでエージェントカードを解決。その後クライアントをストリーミングモードで接続。

⑫SSE形式で送信

Server-Sent Eventsを使用。WebSocket不要でブラウザが消費可能。type、state、text、request_idを含むJSONペイロード。

⑬UIの構成

index.htmlは3つの機能：入力フィールド、送信ボタン、/api/streamへのストリーミング接続で結果をリアルタイム表示。

⑭リアルタイム表示

タスクの状態遷移をライブで監視。回答がチャンクごとに表示され、サブエージェントの中間出力も確認可能。

⑮プロダクショングレードテンプレート

DeepAgents＝エージェントロジック、A2A＝クリーンなAPIレイヤー、Proxy＝UIとストリーミングの分離された構成。

⑯スワップ可能な設計

エージェント、ホスト、トランスポートをUIを再設計せずに交換可能。柔軟な本番運用を実現。

⑰静的ファイルのマウント

proxy.pyでwebディレクトリを静的ファイルとしてマウント。localhost:8000でUI提供。

⑱まとめ

ターミナルから脱却し、A2Aで公開＋UIで可視化されたDeepAgentが完成。ポートフォリオにも最適な構成。
