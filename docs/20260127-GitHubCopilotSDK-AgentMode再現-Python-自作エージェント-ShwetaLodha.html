GitHub Copilot SDKでAgent Mode再現

自分のコードでCopilotのエージェント機能を実装する方法を8〜10シナリオで解説
Shweta Lodha 12分 2026年1月27日

①Agent Modeとは

GitHub Copilotのチャットウィンドウ下部にあるエージェント機能。ファイル編集、一覧表示、アーキテクチャ変更などを自然言語で指示できる。

②必要なパッケージ

PythonでGitHub Copilot SDKパッケージをインストール。非同期イベント用のパッケージも必要。

③基本的なコード構造

SDKをインポート→クライアント作成→セッション開始→ユーザーとして質問を投げる。終了時はclient.stopを呼ぶ。

④Workspaceツール

SDK内部に定義されたデフォルトツールが自動的に使われる。明示的にツールを指定しなくてもファイル操作が可能。

⑤シナリオ1：ファイル検索

「現在のディレクトリにテキストファイルはある？」と質問すると、requirement.txtがあると回答。ファイル一覧を認識。

⑥シナリオ2：ファイル編集

「100を追記して」と指示するとファイルに100が追加される。指示通りにファイルを編集可能。

⑦シナリオ3：コード実行

「defective.pyを実行して出力を教えて」と指示。Pythonファイルを実行して結果を返してくれる。

⑧シナリオ4：エラー検出と修正

コードにエラーを仕込んで実行→エラー発生→「エラーを修正して」で自動修正。ファイル名を再指定する必要なし。

⑨シナリオ5：テキスト検索

「importという単語を含むファイルを一覧表示して」と指示。複数ファイルを横断検索して結果を返す。

⑩シナリオ6：コメント追加

「defective.pyにコメントを追加して」と指示。適切なコメントが自動生成されてコードに追加される。

⑪シナリオ7：モジュール化

「このファイルをモジュールに変換して」と指示。単純なPythonファイルがモジュール形式に変換される。

⑫シナリオ8：TODO実装

コード内にTODOコメントを書いておくと、「TODOを見つけて修正して差分を表示」で自動実装。ハードコード値をユーザー入力に変更。

⑬シナリオ9：コマンド実行

「ls -leの出力を見せて」と指示するとディレクトリ一覧を表示。Git statusなどのコマンドも実行可能。

⑭コンテキスト維持

会話の中でファイル名を毎回指定する必要がない。現在のコンテキストを理解して適切なファイルを操作。

⑮SDKの再現度

このSDKはAgent Modeの99.9%を再現可能。内部でAgentが使っているのと同じSDKを利用している。

⑯外部ツールの追加

デフォルトツール以外の機能が必要な場合は、独自のツールサポートを追加することで拡張可能。

⑰非同期処理

asyncioを使った非同期イベント処理でエージェントとの対話を実現。レスポンスを待ちながら処理を継続。

⑱まとめと応用

自分のアプリケーションにCopilotのエージェント機能を組み込める。リポジトリに対して自然言語でコード操作が可能に。
