ClawdbotをVPSに安全インストール

自律型AIエージェントをHostinger VPSで動かす手順 - セキュリティを考慮した構築ガイド
トル 18分 2026年1月28日

①Clawdbot（現Motebot）とは

好きなLLMを使って動作する自律型AIエージェント。Discord、WhatsApp、LINE、Telegramなどと連携し、ウェブページのスクリーンショット取得など様々なタスクを実行可能。

②名前変更: Clawdbot → Motebot

「Clawdbot」がClaudeに似ているためAnthropicから連絡があり、名前がMotebotに変更された。機能は同じで名前だけが変わっている。

③セキュリティ上の懸念

自分のPCにインストールするとどのファイルも編集・削除・閲覧可能。情報漏洩やファイル改ざんのリスクがあるため、メインPCへのインストールは非推奨。

④なぜVPSを使うのか

Mac miniを買うよりVPS（バーチャルマシン）で十分。変なことができないよう制限された環境で動かすことでセキュリティリスクを軽減できる。

⑤Hostinger VPSの選択

KVM2プラン: vCPU 2、メモリ8GB、ディスク100GB。GCPやAWS EC2でも可能だが、Hostingerは手頃で安い。足りなければKVM4/8にアップグレード可能。

⑥OSセットアップ: Ubuntu 24.04

OSはUbuntu 24.04 LTSを選択。ルートパスワードを設定し忘れないようにメモ。apt update && apt upgradeで必ず最新状態にアップデート。

⑦専用ユーザーの作成

rootユーザーではなくclawdbot専用ユーザーを作成。adduser clawdでユーザー追加し、usermodで権限を設定。制限付きで運用するのが安全。

⑧SSHキーの設定

ローカルPCでssh-copy-idコマンドを使い、公開鍵/秘密鍵をclawdotユーザーのSSHセッションにコピー。パスワードなしで安全にアクセス可能に。

⑨UFWファイアウォール設定

最初はファイアウォールが無効状態。ufw allow OpenSSHでSSHを許可してからufw enableで有効化。SSHが切れないよう順序に注意。

⑩Clawdbotのインストール

GitHubからcurlでインストールスクリプトを取得して実行。Node.jsなど必要なパッケージも自動インストール。セキュリティ警告が出るがVPSなので続行。

⑪モデル設定: OpenRouter + GLM4.7

LLMプロバイダーとしてOpenRouterを選択。APIキーを入力後、モデルはZAIのGLM4.7を選択。他のモデルも選択可能。

⑫Discordボット連携

チャンネルとしてDiscordを選択しボットトークンを入力。WhatsApp、Telegram、LINEなど他のプラットフォームとも連携可能。

⑬スキルの選択

1Password、Claude Hub、Gemini、Google Places APIなど多数のスキルから選択可能。スペースで選択してエンターで確定。必要に応じてAPIキーを設定。

⑭PATHの設定

インストール後にclawdbotコマンドが見つからない場合、bashrcにバイナリパスを追加してsource bashrcで再読み込み。これでコマンドが使える。

⑮ゲートウェイとポートフォワーディング

clawdbot gateway loopbackでポート18789にバインド。ローカルからSSHポートフォワーディングで接続。127.0.0.1:18789でLLMと対話可能。

⑯systemdサービス化

SSHが切れると使えなくなるため、systemdでサービス登録が必要。/etc/systemd/system/clawdbot.serviceを作成してdaemon-reloadで永続化。

⑰サービス設定の確認

systemctl enable clawdbotで自動起動設定。VPS再起動後もプロセスが自動的に立ち上がるようになり、常時利用可能な状態に。

⑱まとめ: 安全な運用のポイント

メインPCではなくVPSで制限された環境で運用。専用ユーザー作成、ファイアウォール設定、systemdサービス化で安全かつ永続的に利用可能。
