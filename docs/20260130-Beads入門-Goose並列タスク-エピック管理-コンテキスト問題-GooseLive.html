Beads入門 with Goose

Gitベースのタスクトラッカーで並列AIエージェントセッションを制御する実演
Goose Live ライブ配信 2026年1月30日

①Beadsとは何か

Steve Yegge氏が開発したGitベースのタスクトラッカー。エージェントが前回どこまで進んだか、次に何をすべきかを管理する。新しいセッションを開いても「どこまで終わった？」と聞けば進捗が分かる。

②MCPサーバーで簡単導入

BeadsはCLIツールとしてもMCPサーバーとしても利用可能。MCPサーバーとしてインストールすれば、GooseやClaude Codeなどのエージェントが自動的に認識。セットアップは比較的シンプル。

③エピックベースのタスク管理

Beadsはエピック（大きな機能要件）を起点にタスクを分割する。エージェントにエピックを渡すと、ユーザーストーリー・受入基準・技術メモ付きの細かいタスクに分解してくれる。

④依存関係の自動管理

タスク間の親子関係や依存関係（blocked/blocking）を自動で設定。あるタスクが完了するまで次のタスクが待機し、依存がないタスクは並列で実行できる。

⑤4並列セッションの実演

Ghostyターミナルを4分割し、4つのGooseセッションを同時に起動。Cookie Jar機能、パンくずリスト、一括タブ閉じ、レンダリングエンジンの4タスクを並列で実行した。

⑥実装デモ: ブラウザの不安解消機能

自作ブラウザに「Anti-Anxiety」機能を追加。閉じたタブを自動保存するCookie Jar、ブラウジング経路を追跡するBreadcrumbs、セッション保存など、タブ管理の不安を解消する機能群。

⑦実行時間と結果

Cookie Jarタスクは約6分、Breadcrumbsは約13分、レンダリングエンジンは約18分で完了。ブロックされたタスクは60秒スリープして再チェックする仕組みで、依存解決後に自動で開始。

⑧agents.mdの役割

Beadsインストール時に自動生成されるagents.mdがエージェントへの指示書。「作業が完了するのはgit pushが成功した時」「pushせずに止まるな」などのルールが定義されている。

⑨コンテキストウィンドウ問題の本質

「エージェントが大規模コードベースで動かない」問題の本質はコンテキストウィンドウの枯渇。Beadsや類似ツールは複数セッションで分散させることで解決を図る。配信者は「将来的にモデル自体が解決する」と予測。

⑩MCP Agent Mail: エージェント間メッセージング

Beads開発者が推奨する連携ツール。Beadsが共有メモリ、Agent Mailがメッセージングを担当。エージェント同士がリーダーを決めてタスクを分担する「スウォーム」能力を実現。

⑪現時点の課題

Beadsはまだバグが多く、PRが増えて不安定になりつつある。SQLデータベースの使用がオーバーキルとの声も。並列実行時にファイル競合が発生するケースや、タスク完了判定の誤りも見られた。

⑫人間がループにいたい問題

配信者は「自分がhuman-in-the-loopでいたい」と正直に語る。4並列が限界で、10〜30セッション管理は精神的に無理。エージェントに任せきりにするのは難しいという実感。

⑬コンテキスト消費量の実測

4セッションの実測値: タスク1は82%、タスク2は29%、タスク3は56%のコンテキストウィンドウを消費。単一セッションでは確実にコンテキストが溢れるタスク量。

⑭Ralph Loopとの比較

配信者が以前作ったRalph Loopも類似のアプローチ。ファイルでタスク完了状態を管理し、チェックオフしていく仕組み。Beadsはより構造化されているが、自作ソリューションの方が好みとの感想。

⑮Codename Gooseとは

オープンソースのオンマシンAIエージェント。MCPベースの拡張機能でGitHub、JetBrains、Google Driveなどと連携。デスクトップアプリとCLIの両方で動作し、複数のLLMプロバイダーに対応。

⑯サブエージェントとの比較

Beadsでは手動で複数セッションを開く必要があるが、サブエージェントなら自動的にセッション分割してくれる。次回はサブエージェントを試す予定。コンテキストウィンドウ節約の観点ではより効率的。

⑰エージェントパターンの未来

配信者の見解: 将来的にはモデルとエージェントが進化し、これらのパターンは不要になる可能性が高い。ただし、今の段階で学んでおくことは将来のソフトウェアエンジニアリングにとって有益。

⑱まとめ: Beadsの位置づけ

Beadsはバックグラウンドタスクを並列実行させたい場合に有用。ただしhuman-in-the-loopを好む人には物足りない面も。コンテキスト管理問題の現時点での解決策として一定の価値がある。
