Claude Code v2.1

12日間で1096コミット — コーディングアシスタントから開発インフラへの進化
さとるチャンネル 8分 2026年1月30日

①1096コミットの衝撃

v2.1.0からv2.1.22まで、12日間で1096コミット、1日あたり約90コミット。もはやアップデートという言葉で片付けられる規模ではない。ツールの設計思想が根っこから変わりつつある。

②Skillsホットリロード

スキルの定義ファイルを書き換えるたびにセッション再起動が必要だった問題が解消。ファイルを保存した瞬間に次のスキル呼び出しから最新の内容が反映される。2分の再起動サイクルが5秒に短縮され、スキル開発の体験が根本から変わった。

③ToolSearch: トークン節約の革命

MCPサーバーを多数接続すると起動だけで6〜7万トークンが消費されていた問題を解決。必要になったツールだけを必要なタイミングで読み込む遅延読み込み方式に変更。起動時に2〜3万トークンを即座に節約でき、より長い会話やより複雑なタスクが可能に。

④/teleportでセッション転送

ターミナルで作業中に「続きはブラウザでやりたい」という場面で、/teleportと打つだけでセッションがWebに転送される。リポジトリの状態、ブランチ、会話履歴が全て自動で引き継がれる。現時点ではターミナル→Webの一方通行。

⑤タスク管理と依存関係の追跡

複数の作業を同時に走らせる際の依存関係をClaude Codeが自動追跡。VS Codeと統合されてエディタ上でタスクの進捗が視覚的に表示される。PRのレビューステータスもリアルタイムで確認可能に。

⑥Hooks: エージェントの行動を制御

ツール使用前（PreToolUse）、使用後（PostToolUse）、終了時（Stop）の3つのタイミングに自分のロジックを差し込める。ファイル削除前に確認を入れる、外部API呼び出し後にログを残す、本番環境への変更は自動で差し止めるなどの制御が可能。

⑦セットアップフックとオンボーディング

v2.1.10で追加。claude initを1回叩くだけで開発環境のセットアップが全て自動実行。リンターの設定、テストフレームワークの準備、MCPサーバーの接続をsettings.jsonに定義しておける。新メンバーのオンボーディングが劇的に効率化。

⑧ワイルドカード権限とセキュリティ修正

npmコマンドを1つずつ許可する手間が解消。npm*と書くだけでnpmコマンド全般に権限を付与できるパターンマッチ方式に。また、OAuthトークンやAPIキーがデバッグログに出力されるバグも修正された。

⑨プラグインマーケットプレイスと日常改善

/plugと打つだけで公式スキルやエージェントを検索・インストール可能に。ホットリロード＋マーケットプレイスの組み合わせで、作って試してすぐ直せて共有できるエコシステムが完成。日本語全角数字での選択対応やShift+Enter改行など、細かな改善も多数。
