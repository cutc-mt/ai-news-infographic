Agent Swarmと依存グラフで16ステップが1ステップに
AI LABS 10分 2026年1月30日

①Agent Swarm: AIが委任まで自動化

これまで人間がAIにタスクを委任していたが、Claude自体がタスクの分解と委任を自動で行うようになった。プロジェクトマネージャーに話すように広いタスクを渡せば、Claudeが自動的にブレイクダウンする。

②タスクコーディネーターの仕組み

メインのClaudeがタスクグラフ（依存関係グラフ）を作成。各タスクを「順次実行（前のタスク完了が必要）」か「並列実行（依存関係なし）」かに分類し、サブエージェントを生成して割り当てる。

③各エージェントに独立した200Kコンテキスト

以前は単一のコンテキストウィンドウで全てを処理していたが、各エージェントが独自の200Kコンテキストウィンドウを持つ。1つのタスクに集中でき、コンテキストロスの問題が解消された。

④タスク複雑度に応じたモデル選択

タスクの複雑さに応じて異なるモデルを割り当て。フォルダ探索にはHaikuやSonnet、複雑なタスクにはOpus 4.5を使用。不要なトークン消費を抑えてコスト削減にもつながる。

⑤タスクの永続化: JSONファイル構造

タスクは.claude/tasksフォルダ内にセッションIDごとのフォルダで保存。各タスクがJSON形式で名前・説明・ステータス・依存関係を保持。clearコマンドやセッション再起動でもタスクが消えない。

⑥blocksとblockedByの依存グラフ

各タスクJSONには「blocks」（このタスクがブロックしているタスク）と「blockedBy」（先に完了が必要なタスク）のキーがある。これにより正しい実行順序が保証され、タスクのスキップを防止する。

⑦RALPHループの終焉

以前はタスクシステムの再アンカリングのためにRALPHループが必要だったが、タスクのファイル外部化により不要に。compactも以前ほど頻繁に実行する必要がなくなった。

⑧CLI環境変数でセッション名を固定

フォルダ名はデフォルトでランダムIDだが、環境変数にカスタム名を設定するとセッションが名前で識別される。ターミナルを閉じても同じセッションに戻れる。

⑨並列実行の効果: 5波が3サイクルに

タスク1と2に依存関係がなければ同時に実行、タスク3と4はタスク1の完了を待つ、というように依存関係に基づいて最適化。5つの順次ステップが3サイクルで完了する。

⑩実例: 16ステップが1ステップに

PRDの複数セクションが互いに依存していないことをClaudeが分析し、16の順次ステップを並列化して1ステップに削減。複数エージェントが同時に独立して作業した。

⑪Cowork: 非開発者向けのClaude Code

Claude Codeの技術を非開発者向けに展開したCowork。ファイルシステムへのアクセスやフォルダ操作が可能で、より多くのガードレールを備えている。リサーチや企画管理にも活用できる。

⑫Coworkの活用例: フォルダ整理

大量のプロジェクトフォルダの中から特定のスキルを探す際、各プロジェクトの内容をドキュメント化するよう指示。Agent Swarmで複数エージェントがファイルをバッチ読み込みし、各プロジェクトの要約を作成。

⑬Coworkで市場調査

アプリの実現可能性と市場調査をCoworkで実行。質問に答えるとフォルダ内のドキュメントを参照しながら包括的なレポートを生成。専門スキルでフォーマットも整った状態で保存される。

⑭Cowork → Claude Codeの連携フロー

Coworkでリサーチとドキュメント作成を完了させた後、Claude Codeで実装に移行。PRDドキュメントを読み込ませると、各セクションを並列で処理するエージェントを自動生成した。

⑮タスク分解を明示的に指示する方法

Claudeが自動でタスクを分解しない場合は、「break this down into tasks with dependencies」とプロンプトすれば依存グラフを作成してワークフロー管理を開始する。Ctrl+Tでタスク一覧も確認可能。

⑯compactの頻度が激減

以前はコンテキストウィンドウがすぐ埋まってcompactが必要だったが、各エージェントが独立コンテキストを持つためcompactの頻度が大幅に減少。構造化ノートの手動管理も不要に。

⑰コスト最適化の効果

タスクの複雑さに応じてモデルを使い分けるため、小さなタスクに余分なトークンを消費しない。時間短縮だけでなくコスト面でも効率的な仕組みになっている。

⑱まとめ: 抽象化レイヤーの追加

Agent Swarmは人間→AI委任からAI→AI委任への進化。タスクの永続化、依存グラフ、独立コンテキスト、モデル自動選択が組み合わさり、Claude Codeの実用性が大幅に向上した。
