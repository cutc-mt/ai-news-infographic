AGENTS.md vs Skills 精度比較

Vercel公式検証が明らかにした発動率の差 -- 53%から100%への道筋と使い分けの正解
まさおAIじっくり解説ch 14分 2026年2月1日

①AGENTS.md/CLAUDE.mdとは

AIエージェントのコンテキストウィンドウに常にロードされる情報。システムプロンプトのようにタスク実行中ずっと参照され続ける。「このプロジェクトではこう振る舞ってほしい」というルールを書く場所であり、常にエージェントの頭に入っている情報となる。

②CLAUDE.mdはシンプルに保つべき

常にコンテキストに含まれるため、書きすぎるとLLMの注意力が分散してしまう。LLMの注意力は有限であり、大量の指示があると本当に重要なことに集中できなくなる。本当に重要なことだけを書くべきという原則を守ることが重要。

③Skillsの段階的開示の仕組み

YAML frontmatter（nameとdescription）だけが常にロードされ、エージェントが必要と判断した場合のみ詳細を読みに行く段階的開示の仕組み。トークン効率は良いが、エージェントが「このスキルが必要だ」と気づかなければ発動しないというワンクッションがある。

④Vercel検証結果: 驚きの精度差

Next.js 16の新APIでテスト。ベースライン（ドキュメントなし）は53%。スキルありでも53%のまま変化なし。スキル使用を明示的に指示すると79%に上昇。AGENTS.mdに情報を記載した場合は100%を達成。段階的開示の判断ミスが発動率に直結する結果となった。

⑤なぜスキルの発動率が低いのか

段階的開示のワンクッションが必要であることが主な原因。エージェントがスキルのdescriptionを見て「これが必要だ」と判断しなければ詳細を読みに行かない。ただし、descriptionの書き方を工夫することで発動率は改善できる可能性がある。

⑥スキルの2つの型を理解する

リファレンス型（知識・辞書的）とタスク型（ワークフロー・ステップバイステップ）の2種類が存在。タスク型はスクリプトを含む中規模スキルに向いている。今回のVercel検証はリファレンス型の話であり、タスク型スキルは依然として有用。

⑦AIフレンドリーなドキュメント圧縮

AGENTS.mdが常にロードされるならば、人間には読みにくくてもAIに最適化された圧縮ドキュメントにする検討の価値がある。コンテキストウィンドウの効率的な活用として、必要な情報を最小限のトークン数で伝える工夫が重要になってくる。

⑧発動率の計測が改善の鍵

スキルの修正が発動率に貢献しているかを定量的に評価することが重要。descriptionを変更した際にどれだけ発動率が改善したかを計測する仕組みが必要。日常利用で明示的に呼び出すなら計測不要だが、自律的な発動を期待する場合は計測がセットとなる。

⑨まとめ: 使い分けが正解

「必ず読み込んでほしいこと→AGENTS.md」「ワークフロー/タスク→Skills」という使い分けが正解。スキルは不要なのではなく、リファレンス型とタスク型の違いを理解して適切に使い分けることが重要。スキルの型を理解することで、効果的な活用が可能になる。
