LangChain-Toolbox: MCPサーバー連携を簡単に

LangChainエコシステムの新パッケージでMCPサーバーのツールを動的にロードし、AIエージェントに統合
Coding Crash Courses 7分 2026年2月2日

①MCPとは: ツールを一元管理するプロトコル

Model Context Protocol（MCP）は、AIモデルが使うツール（Web検索やDB問い合わせなど）を一箇所で集中管理し、クライアントが動的にロードできる仕組みである。従来はエージェントごとにツールをJSON形式で個別に定義する必要があったが、MCPサーバーを立てれば標準化された方法でツールを提供・共有できる。

②FastMCPでサーバー構築: デコレータでツール定義

FastMCPを使えば、Pythonの関数に@toolデコレータを付けるだけでMCPサーバーのツールを定義できる。関数のシグネチャと型情報が自動的にJSONスキーマに変換され、AIモデルにそのまま渡せる形になる。uv run python server.pyでポート5000上にサーバーが起動する。

③LangChain-Toolboxとは: MCPクライアントの新パッケージ

toolbox_langchainからインポートするToolboxClientが中心となるクラス。コンテキストマネージャ（with文）でMCPサーバーに接続し、ツールを動的にロードできる。LangChainのRunnableインターフェースに完全準拠しており、既存のLangChainワークフローにそのまま組み込める。

④ツールの読み込み方法: 一括と個別の2パターン

load_toolset()で全ツールを一括取得するか、load_tool("ツール名")で個別に取得できる。取得したツールはtool.invoke({})で直接実行可能。同期・非同期の両方に対応しており、非同期の場合はasync withとa_load_toolset()を使用する。

⑤create_agent関数との統合: 最もシンプルな使い方

LangGraphから移植されたcreate_agent関数を使えば、モデルとツールリストを渡すだけでAIエージェントが完成する。レストランBotの例では、ピザの価格一覧と営業時間を同時に問い合わせると、エージェントが自動で2つのツールを呼び出し、人間が読みやすいテキストで結果を返す。

⑥bind_toolsで手動統合: より細かい制御が可能

create_agentを使わず、モデルに直接bind_tools()でツールを紐付ける方法もある。ツール名をキーにした辞書（ルックアップマップ）を事前に構築し、レスポンスのtool_callsを手動でイテレートして各ツールをinvokeする。create_agentの内部で行われている処理を明示的に制御できる。

⑦FastMCPとの互換性バグ: ヘッダー問題の回避策

現時点ではFastMCPとの組み合わせでヘッダーが正しく渡されないバグがある。FastMCPがapplication/jsonのAcceptヘッダーを厳密に要求するが、トランスポート層でヘッダーが内部に伝搬しない問題が発生する。動画ではモンキーパッチによる回避策が紹介されている。

⑧デモ: レストランBotで実践

架空のレストランを題材に、商品一覧・ピザ価格・営業時間の3つのツールをMCPサーバーに定義。GPT-4 Miniをモデルとして使い、「ピザの価格と土曜日の営業時間は？」と聞くとエージェントが自動で2つのツールを呼び出し、JSON結果を人間向けのテキストに変換して回答する。

⑨まとめ: まだ若いが将来有望なパッケージ

LangChain-ToolboxはMCPサーバーとLangChainエコシステムの橋渡しをするパッケージとして非常に有望である。まだ開発初期段階でバグもあるが、ツールの一元管理と動的ロードという強力なコンセプトを実現している。MCPサーバーを活用する開発者にはぜひ試してほしいパッケージである。
