②タスク内容: Webサイト管理画面の作成

③UIは英語のみ: 日本語未対応が盲点

CodexデスクトップアプリのUI表示は英語のみで、日本語には対応していない。これは意外な盲点だった。ただし、プロンプトやAIの応答は日本語でも正常に動作する。インターフェースの言語設定が英語固定なだけで、実際の開発作業における日本語の使用には問題がない。今後のアップデートで多言語対応が期待される。

④プランモードあり: Shift+Tabで切り替え

Shift+Tabキーでプランモードに切り替えられる機能が実装されている。プランモードでは、AIが実行前に作業計画を提示してくれる。今回の検証ではプランが日本語で出力されたため、日本語での作業計画確認が可能だった。実行前に全体像を把握できるため、安心して作業を進められる。Claude Codeのプランモードと似た仕組み。

⑤レートリミットが2倍: アプリ経由の特典

Codexデスクトップアプリを経由すると、レートリミットが通常の2倍に増加する特典がある。これは大規模なコーディング作業を行う際に大きなアドバンテージとなる。Web版と比較してより多くのリクエストを短時間で処理できるため、連続作業やマルチタスク実行時のストレスが大幅に軽減される。デスクトップアプリを使う大きな理由の1つ。

⑥プロジェクト追加機能: 複数プロジェクトを管理可能

アプリ内で複数のプロジェクトを同時に管理できる機能が実装されている。プロジェクト間の切り替えがスムーズで、それぞれのコンテキストを保持したまま作業できる。複数の案件を並行して進めるフリーランスや、マイクロサービスを複数管理する開発チームにとって便利な機能。プロジェクトごとに設定や履歴が分離される。

⑦並列実行: ワークツリーを分けて別タスク実行

Git worktreeを分けて同じリポジトリで別のタスクを並列実行できる機能が実装されている。これにより、メインの作業をしながら別のブランチで別機能の開発を同時進行できる。Gitの分離により、仮にAIが間違った方向に進んでもメインブランチに影響しない。マルチタスク開発の新しいスタイルを実現している。

⑧オートメーション機能: 自動化タスクの設定

スケジュールされたタスクを自動実行するオートメーション機能が搭載されている。定期的なバグチェック、コードフォーマット、テスト実行などを自動化できる。今回の動画では詳しく触れられていないが、CI/CD的な使い方ができそうな機能として紹介された。開発者が不在の時間帯にもAIが働き続けてくれる。

⑨スキル機能: まだ触ってないが面白そう

再利用可能な作業パターンを保存できる「スキル」機能が実装されている。今回の動画ではまだ使用していないが、面白そうな機能として言及された。定型的な作業をスキルとしてパッケージ化しておくことで、同じ作業を繰り返す際の効率が大幅に向上する。次回以降の動画で詳しく検証される予定。

⑩コマンド入力可能: ターミナル呼び出し

アプリ内から直接ターミナルコマンドを呼び出せる機能が実装されている。GUIとCLIのシームレスな連携により、必要に応じてターミナル操作が可能。npm install、git操作、ビルドコマンドなどを即座に実行できるため、開発フローが途切れない。統合開発環境としての完成度が高い。

⑪変更履歴表示: 右側にdiff表示

コードの変更内容が画面右側にdiff形式で表示される機能がある。どの部分が変更されたかを視覚的に確認できるため、AIが加えた修正内容を素早く把握できる。レビューしやすく、間違った変更があればすぐに気づける。Git diffのようなインターフェースで、開発者にとって馴染みのある表示形式。

⑫コミット/プッシュ管理: アプリ上から実行可能
