OpenClaw + Antigravity: AIが自律エンジニアリングチームに進化

無料でOpus 4.5とGeminiを使える最強連携——セルフホスト型エージェントランタイムがローカルマシンで動作

2026年2月3日 WorldofAI 約13分

①OpenClawとは: 自律型AIエージェントランタイム

OpenClawはローカルマシンで動作するセルフホスト型エージェントランタイム＆メッセージルーター。複数のAIエージェントを統括し、タスクのプランニングとオーケストレーションを担当する。従来のAIツールとは異なり、単一のAIに頼るのではなく、専門化されたエージェントチームを編成して複雑なソフトウェア開発タスクを処理する。完全にオープンソースで、GitHubで公開されている。

②Antigravityとは: GoogleのAgentic IDE

AntigravityはGoogleが提供するAgentic IDE（エージェント統合開発環境）。誰でも簡単にAIエージェントをデプロイしてコーディング作業を実行できる。従来のIDEがコーディング支援に留まるのに対し、AntigravityはAIエージェントが自律的にコードを書き、テストし、デバッグまで行う。Googleアカウントでログインするだけで、無料でクラウドAIモデルにアクセスできるのが最大の魅力。

③連携のメリット: 役割分担で最強チーム編成

OpenClawがプランニングとオーケストレーションを担当し、Antigravityが実際の実行環境を提供する完璧な役割分担が実現。OpenClawは複雑なタスクを細分化し、各専門エージェント（UXデザイナー、デバッガー、ビルダーなど）に割り振る。Antigravityはそれぞれのエージェントに必要な計算リソースとモデルアクセスを提供。結果として、まるで人間のエンジニアリングチームのように効率的な協働が可能になる。

④無料でOpus 4.5とGeminiモデルを使える

AntigravityのOAuth認証を通じて、Claude Opus 4.5、Gemini 2.0 Flash、Gemini Experimentalなどの最先端モデルに無料でアクセスできる。通常は月額課金が必要なOpus 4.5も、この連携によって無料で利用可能。ローカルでAPIキーを管理する必要がなく、Google認証だけで複数のモデルにアクセスできるため、セットアップが非常に簡単。開発者にとってはコスト削減と利便性の両方を実現する画期的な仕組み。

⑤インストール方法: 3ステップで完了

インストールは驚くほど簡単。まずnpm install openclawでOpenClawをインストール。次にopenclaw onboardでセットアップウィザードを起動し、Antigravity連携を選択。最後にCLI上でGoogleアカウントにログインすれば完了。所要時間は5分未満。Node.js環境があれば誰でもすぐに始められる。追加の設定ファイルやAPIキーの入力も不要で、初心者でも迷わずセットアップできるように設計されている。

⑥Antigravityプラグイン有効化: コマンド1つ

onboardingプロセスでAntigravityプラグインを有効化すると、OpenClawがAntigravity経由でAIモデルに接続するようになる。従来のようにローカルでAPIキーを管理したり、環境変数を設定する手間が一切不要。プラグインが自動的にOAuth認証フローを処理し、トークンのリフレッシュも自動で行われる。複数のモデルプロバイダーを横断的に使えるため、タスクごとに最適なモデルを選択できる柔軟性がある。

⑦Googleアカウントログイン: CLIから直接認証

ターミナル上でCLIからGoogleアカウントにログインする流れ。ブラウザが自動的に開き、Googleの認証画面が表示される。ログインを完了すると、CLIに戻って認証成功のメッセージが表示される。この認証により、AntigravityがホストするすべてのAIモデルにアクセス権が付与される。一度ログインすれば、トークンは自動的に保存され、次回以降は再ログイン不要。セキュリティも考慮されており、トークンはローカルに暗号化されて保存される。

⑧Agent Kit: 専門エージェントチームを編成

OpenClawにはUXデザイナー、コードレビュアー、デバッガー、ビルダー、テスターなどの専門エージェントがあらかじめ用意されている。各エージェントは特定のドメイン知識とスキルセットを持ち、人間のチームメンバーのように役割を分担する。例えば、新機能開発の際はUXデザイナーが画面設計、ビルダーがコード実装、レビュアーがコード品質チェック、テスターが動作検証を担当。複数エージェントが並列で作業するため、開発スピードが劇的に向上する。

⑨ClawHub: オープンソースのスキルハブ

ClawHubはコミュニティ主導のスキル共有プラットフォーム。開発者が作成したスキル（特定のタスクを自動化するスクリプト）をnpxコマンドで簡単にインストールできる。例えば「Figmaデザインをコードに変換」「APIドキュメントを自動生成」などのスキルが公開されている。GitHub ActionsやDockerfileのような再利用可能なパッケージとして機能し、車輪の再発明を防ぐ。自分が作ったスキルを公開することも可能で、エコシステムが急速に成長している。

⑩デモ結果: CRMダッシュボードを自動生成

動画のデモでは、1回目の実行でシンプルなCRMダッシュボードを生成し、2回目の実行でさらに洗練された高品質なフロントエンドを構築した。AIが自律的にReactコンポーネントを設計し、データフローを実装し、UIのレスポンシブ対応まで完了。開発者が書いたコードはゼロ。従来なら数日かかる作業が、わずか数分で完成。生成されたコードは実用レベルで、そのままプロダクションに投入できる品質だった。

⑪ゲートウェイ対応: WhatsAppやDiscordから操作可能

OpenClawはWhatsApp、Telegram、Discord、Slackなどのメッセージングプラットフォームと統合できる。これにより、モバイルアプリやチャットからDevOps操作を実行可能。例えば、Slackで「本番環境にデプロイして」と指示すると、OpenClawがCI/CDパイプラインをトリガーし、結果を報告する。ノートパソコンを開かなくてもスマホから開発業務を進められる。チーム全体がチャットツールでAIエージェントと協働する未来が現実になりつつある。

⑫セキュリティ注意: 仮想環境必須、グローバルインストールは危険

OpenClawはローカルマシンのファイルシステムにフルアクセスするため、必ず仮想環境やDockerコンテナで実行すること。グローバルインストールやホストマシンでの直接実行は、AIエージェントが予期しない操作（ファイル削除、環境変数の書き換えなど）を行うリスクがある。特に本番環境のシークレットや機密情報には絶対にアクセスさせない。サンドボックス環境でのテストを徹底し、AIの動作を監視しながら段階的に権限を拡大していくのがベストプラクティス。
