OpenAIはConductorやCommanderをコピーしているのか？ macOS専用アプリを徹底レビュー

2026年2月4日 AICodeKing 約9分

①Codexデスクトップアプリの概要: macOS専用ネイティブアプリが登場

OpenAIがCodexのデスクトップアプリをmacOS向けにリリースした。これまでターミナルやVS Code拡張機能でしか使えなかったCodexが、ネイティブのグラフィカルインターフェースとして利用可能に。無料プランのユーザーは1ヶ月間のアクセスが付与され、有料サブスクライバーにはレート制限が2倍に引き上げられる。

②UIデザインとConductorとの類似性: ワークツリーやスレッド構造が酷似

アプリのレイアウトを見ると、Conductorのデザインと非常に似ていることが分かる。スレッド内にワークツリーが配置される構造や、全体的なUIの配置がそっくりだ。Conductor（現Commander）のインスピレーションを受けていることは明らかで、OpenAIのような大企業が小規模プロジェクトのUIを模倣している点は議論を呼んでいる。

③スキルとオートメーション機能: プリセットスキルとスケジュール実行

スキルタブでは、あらかじめ用意されたスキルをインストールして利用できる。ただしカスタムスキルの追加には手動で設定フォルダを操作する必要があり、UIからは作成できない。オートメーションタブでは、名前、プロジェクト、プロンプト、スケジュール（時刻や間隔）を設定してタスクを自動的にスケジュール実行することが可能。

④モデル選択のバグ: GPT-5.2 Codex Mediumしか選べない

モデル選択画面には複数のモデルが表示されるが、実際にクリックしてもGPT-5.2 Codex Medium以外のモデルが選択できないという深刻なバグが存在する。他のモデルをクリックしても反応がなく、事実上GPT-5.2 Codex Mediumしか使えない状態になっている。基本的な機能で発生しているバグであり、リリース前のテスト不足が疑われる。

⑤UIの品質問題: パディングのずれと表示崩れが多発

UIには多くの品質問題が見られる。スキルタブではグレーの背景が消えたり、要素が境界からはみ出したりする不具合がある。複数のページにわたってパディングのずれが確認されており、全体的に「バイブコーディング」で作られたような印象を受ける。1兆ドル企業が出すアプリとしては品質が低いと指摘されている。

⑥コンテキスト認識の問題: プロジェクトのファイルを読み取れない

プロジェクトを選択しても、アプリがプロジェクト内のファイルを正しく認識しないという問題が発生する。設定ファイルを手動で削除しないと修復できないケースがあり、コンテキスト認識の精度に疑問が残る。また、クラウドモードを選択するとプロジェクトの選択が上書きされてしまい、ローカルのファイル構造との整合性が取れなくなる問題もある。

⑦UXデザインの疑問点: 直感的でない操作体系

プランモードがプラスアイコンの裏に隠されており、通常プラスアイコンはファイル添付に使われるため、UXとして混乱を招く設計になっている。ファイル名をクリックするとVS Codeが新しいインスタンスで開き、メモリを大量に消費する。ターミナルは内蔵されているが、ターミナルのフォントを独立して変更できないという制限もある。

⑧Verdantとの比較: エージェンティックUIの最高峰との差

レビュアーはVerdantを最高のエージェンティックUIとして評価している。Verdantはマルチリポジトリ対応、優れたワークスペース管理、低メモリ消費を実現しており、Codexデスクトップアプリよりもはるかに洗練されている。スキル管理もUIから直感的に行え、全体的な完成度で大きな差がついている。

⑨総評: 機能は揃っているが品質が伴わない

コミット、プッシュ、PR作成、サンドボックス実行など必要な機能自体は一通り揃っている。しかしUIのバグ、コンテキスト認識の問題、直感的でないUXデザインなど品質面での課題が多い。OpenAIほどの規模と資金力を持つ企業のプロダクトとしては期待外れであり、既存のオープンソースツールに品質で劣る点が厳しく評価されている。
