AIエージェントがリアルタイムでUIを構築する時代

Generative UI（生成的UI）が変えるソフトウェアの未来——パーソナライズされたダッシュボードとコンポーネント選択の仕組み

2026年2月5日 Cole Medin 約12分

①パーソナライズされたダッシュボード: AIが完全にカスタム生成

デモで紹介されたダッシュボードは、事前設定なしで完全に動的に生成されている。AIエージェントが「セカンドブレイン」のリサーチ結果を分析し、最適なレイアウトとコンポーネントをリアルタイムで選択する。従来のように固定されたUIではなく、入力内容に応じて毎回異なるダッシュボードが生成される。これはウルトラパーソナライゼーションの実現であり、ソフトウェアの未来を示している。

②Generative UIとは: エージェントがフロントエンドを決定する

Generative UI（生成的UI）の本質は、エージェントにレイアウトとコンポーネントを決定させるという発想である。従来はプログラマーがUIを設計していたが、今やエージェントがその役割の一部を担う。ユーザーのニーズや入力内容に応じて、表示する情報だけでなく、それをどのように構成するかまでAIが判断する。これがGenerative UIの核心的なコンセプトである。

③未来の予測: AmazonやGoogleもパーソナライズへ

近い将来、amazon.comやgoogle.comを開いたとき、隣の人とは全く違う画面が表示されるようになる。その人のアプリケーションの使い方や興味関心に基づいて、完全にパーソナライズされたインターフェースが提供される。これは単なるコンテンツの推薦にとどまらず、レイアウト全体がユーザーごとに最適化される世界である。Generative UIがこの方向への大きな一歩となる。

④テックスタック概要: 4つの技術を組み合わせる

このGenerative UIアプリは4つの主要技術で構成される。バックエンドで動くエージェントとしてPydantic AIを使用。コンポーネント仕様としてA2UI（Google製）を採用。エージェントとフロントエンドの接続にはAGUIプロトコルを利用。そしてReactコンポーネントのレンダリングにはCopilot Kitを使う。これらが連携して、シームレスなGenerative UI体験を実現している。

⑤3つのカテゴリ: Static・Declarative・Open-ended

Generative UIにはエージェントに与える制御レベルによって3つのカテゴリがある。Static UI（静的UI）、Declarative UI（宣言的UI）、Open-ended UI（オープンエンドUI）である。それぞれプログラマーとエージェントの役割分担が異なり、トレードオフが存在する。用途に応じて最適なアプローチを選択することが重要であり、技術スタックはどのタイプにも対応できるよう設計されている。

⑥Static UI: 従来型のエージェント活用

Static UI（静的UI）は最も古典的なアプローチである。エージェントは表示する情報を決定するだけで、コンポーネントやレイアウトは選択しない。例えば「ニューヨークの現在の気温」を取得してUIに表示するが、UIの構造自体は事前に固定されている。現在のWebで最も一般的に見られるパターンであり、多くのアプリケーションがこの方式を採用している。

⑦Open-ended UI: エージェントがコードを生成

Open-ended UI（オープンエンドUI）は最も柔軟なアプローチである。エージェントはコンポーネントやレイアウトだけでなく、HTML・CSS・JSXなどのコード自体をその場で生成する。最大限の自由度がある反面、セキュリティとパフォーマンスに大きな懸念がある。任意のコンテンツをレンダリングすることになるため、本番環境での採用には慎重な検討が必要である。

⑧Declarative UI: コンポーネントライブラリから選択

Declarative UI（宣言的UI）は最も興味深いアプローチである。事前に定義されたコンポーネントライブラリからエージェントが選択する。「このコンポーネント、このコンポーネント、このレイアウトで、パラメータはこれ」という形式で指示を出す。自由度がありつつも、プログラマーがコンポーネントライブラリとプロンプトで制御可能な範囲を設定できる。GoogleのA2UI仕様に準拠したアプローチである。

⑨ユースケース1: チャットアプリケーション

Generative UIの有力なユースケースとしてチャットアプリケーションがある。SlackボットやDiscordボットなど、エージェントと会話する場面を考えてみよう。テキスト応答だけでなく、ビジュアルコンポーネントを会話の途中でレンダリングできれば、ユーザーの理解度は大幅に向上する。会話内容にカスタマイズされた視覚的要素を動的に生成することで、より豊かなコミュニケーションが実現する。

⑩ユースケース2: コクリエイターワークスペース

コクリエイターワークスペースはGenerative UIの理想的な活用例である。エージェントと一緒に作業する共有スペースで、キャンバスにはAIが生成したUIが表示され、それが進化していく。今回デモで紹介されたダッシュボードがまさにこのパターンである。ユーザーとエージェントが協調しながら、動的に変化するインターフェースを構築していく体験が可能になる。

⑪ユースケース3: Eコマースのレイアウト最適化

Eコマースストアも優れたユースケースである。現在の推薦アルゴリズムは特定の商品を提案するだけだが、Generative UIではレイアウト全体をユーザーに最適化できる。ユーザーが購入すべきと判断した商品にフォーカスを当てたレイアウトに変更するなど、単なるコンテンツ推薦を超えた体験が可能になる。将来的にはこのようなパーソナライズが一般的になるだろう。

⑫A2UI: Googleのコンポーネント仕様プロトコル

A2UIはGoogleが開発したGenerative UI仕様である。エージェントがフロントエンドでレンダリングするコンポーネントをどのように定義するかを標準化している。各コンポーネントにはID、名前、子要素、そしてprops（表示するデータ）が含まれる。このJSON形式の設定により、バックエンドのエージェントとフロントエンドの間で明確なコントラクト（契約）が成立する。

⑬AGUI: エージェントとフロントエンドの接続プロトコル

AGUIはエージェントとフロントエンドを接続するためのプロトコルである。APIを通じて動作するエージェントをフロントエンドに簡単に統合できる。リアルタイム同期を保つためのイベント標準を定義しており、処理の進捗状況（0%→40%など）をフロントエンドに通知できる。Static・Declarative・Open-endedのいずれのGenerative UIにも対応可能な汎用的な仕組みである。

⑭Copilot Kit: Reactでのレンダリングフレームワーク

Copilot Kitはインタラクティブなフロントエンドアプリを構築するためのフレームワークである。AGUIやPydantic AIと直接統合されており、エージェントからのリクエストを受け取ってReactコンポーネントとしてレンダリングする。Copilot Kitがなければ、同等の機能を実現するのに数百〜数千行のコードが必要になる。これらのツールの組み合わせが開発効率を劇的に向上させている。

⑮バックエンドの仕組み: システムプロンプトでコンポーネントを定義

バックエンドではエージェントにコンポーネントの選択肢を与える。システムプロンプトで利用可能なコンポーネントとその使い方を記述し、A2UI仕様に従った出力方法も説明する。これがエージェントを導くガイドラインとなる。コンポーネントのID、名前、props、使用場面の説明など、詳細な情報をプロンプトに含めることで、エージェントが適切な判断を下せるようになる。

⑯フロントエンドのコントラクト: コンポーネントマッピング

フロントエンドではバックエンドと同じコンポーネントを定義するが、こちらは実際にレンダリングするJSXとpropsの受け取り方を実装する。AGUIフォルダ内に整理されており、テーブルコンポーネントであればheaders、rows、title、subtitleなどのpropsを定義している。エージェントがこれらの値を決定し、フロントエンドはそれに従ってReactコンポーネントを生成する。

⑰Markdown疲労の解決: 壁のようなテキストからの解放

このアプリを作った動機はMarkdown疲労の解消である。セカンドブレインでエージェントにリサーチさせると、いつも壁のようなテキストをパースする必要があった。どれだけ簡潔に生成させようとしても、Markdownを読み続けることに疲れてしまった。Generative UIはこの問題を解決し、ダッシュボード形式でインサイトを素早く抽出できるようにする。プロンプトを調整すれば、表示内容も進化させられる。

⑱ソフトウェアの未来: Generative UIが変える開発体験

Generative UIはソフトウェアの未来そのものである。紹介されたテックスタック（Pydantic AI、A2UI、AGUI、Copilot Kit）は、どのタイプのGenerative UIにも対応可能な設計になっている。GitHubリポジトリが公開されており、これを出発点として自分のアプリケーションを構築できる。OpenClawやNano Clawなどの代替ツールの研究にも活用されており、実用的な成果が出ている。
