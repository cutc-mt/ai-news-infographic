GitHub Agent HQ: Claude・Codexが利用可能に

2026年2月5日

①Agent HQとは

  • GitHubが発表した新プラットフォーム。
  • 複数のAIコーディングエージェントをGitHub上で直接実行可能。
  • GitHub、GitHub Mobile、VS Codeから利用できる。
Copilotだけじゃない！複数のAIエージェントを選んで使える時代が来た！

②Claude（Anthropic）対応

  • ClaudeがGitHub上で公式に利用可能に。
  • パブリックプレビューとして提供開始。
  • ユーザー設定で有効化が必要。
Claude派の人も安心！GitHubから直接Claudeを使えるようになった！

③OpenAI Codex対応

  • OpenAIのCodexもパブリックプレビューで利用可能。
  • コード生成に特化したAIモデル。
  • 設定画面から有効化して利用開始。
OpenAI派もカバー！好みのAIを選んで開発できる！

④料金体系

  • Copilot Pro+またはEnterpriseサブスクリプションが必要。
  • 既存のCopilotサブスク内で追加料金なしで利用可能。
  • 今後、より多くのプランで利用可能になる予定。
Copilot Pro+を契約していれば追加料金なしでClaudeもCodexも使える！

⑤GitHubプラットフォームの強み

  • 開発ワークフローがGitHub内で完結。
  • 別のダッシュボードやAI専用プロセスが不要に。
  • ディスカッションとコード変更がリポジトリ内で一元管理。
コンテキスト切り替えが減って開発効率アップ！

⑥エージェントセッションの開始

  • リポジトリのAgentsタブからセッションを開始。
  • Copilotアイコンをクリックしてエージェントを選択。
  • リクエストを入力してタスクを実行。
数クリックでAIエージェントが作業を開始してくれる！

⑦非同期タスク実行

  • エージェントは非同期で作業を実行。
  • リアルタイムで進捗状況をトラッキング可能。
  • コンテキストを失わずにタスクを実行できる。
AIが作業中に別のことができる！進捗もいつでも確認OK！

⑧サブスク競合に注意

  • Claude MaxやOpenAIの個別サブスクとは別枠。
  • GitHub経由の利用はCopilotのクレジットを消費。
  • 既存サブスクとの重複に注意が必要。
既にClaude Maxを契約している人は二重課金にならないよう注意！

⑨VS Code連携

  • Agent sessionsビューからアクセス (Ctrl+Shift+P)。
  • Local、Cloud、Backgroundの3つのセッションタイプ。
  • 対話型アシスタンスと自律タスクの両方に対応。
いつものVS Codeから離れずにエージェントを使える！

⑩GitHub Mobile対応

  • GitHub Mobileアプリからもエージェントを利用可能。
  • 外出先でもエージェントの進捗を確認。
  • モバイルからタスクの指示も可能。
電車の中でもAIに仕事を振れる時代！

⑪エンタープライズ向け機能

  • 組織管理者がエージェントへのアクセスを一元管理。
  • セキュリティポリシーを中央で定義。
  • 許可するエージェントとモデルを制御可能。
企業のセキュリティ要件にも対応した管理機能付き！

⑫コード品質レビュー

  • Code Quality機能でメンテナンス性・信頼性を評価。
  • Copilotが自動でファーストパスのコードレビューを実行。
  • 完全な監査ログとアクセス管理。
AIがコードレビューの第一段階を自動でやってくれる！

⑬Issue/PRからエージェント呼び出し

  • IssueやPRから直接エージェントに割り当て可能。
  • エージェントがドラフトPRを自動で作成。
  • コメントで@メンションしてフォローアップタスクを指示。
@Copilot、@Claude、@Codexでエージェントに直接お願いできる！

⑭ドラフトPR自動作成

  • エージェントがドラフトPRを自動生成。
  • 人間がレビューして承認・修正可能。
  • すべてのエージェント活動はログに記録される。
AIが作ったPRを人間が最終チェック！安心の二段階体制！

⑮クレジット消費量の比較

  • 各エージェントセッションは1プレミアムリクエストを消費。
  • Claude Opus/Sonnet、Codexで消費量は同じ。
  • メトリクスダッシュボードで組織全体の利用状況を確認可能。
どのモデルを選んでも1リクエスト消費は同じ！

⑯複数エージェントの比較

  • 同じタスクを複数エージェントに割り当てて比較可能。
  • 各モデルの問題へのアプローチの違いを確認。
  • エッジケースやアーキテクチャ上の問題を特定。
ClaudeとCodex両方に同じ問題を解かせて比較できる！

⑰今後の対応予定

  • Googleのエージェントも近日対応予定。
  • Cognition（Devin）も追加予定。
  • xAI（Grok）との連携も計画中。
今後さらに選択肢が広がる！AIエージェント戦国時代の到来！

⑱まとめ：開発の未来

  • GitHub Agent HQでマルチAIエージェント時代が到来。
  • 開発者は好みのAIを選んで使い分けられる。
  • AIコーディング環境の進化に今後も注目。
GitHubがAIエージェントのハブになる！開発の形が大きく変わりそう！
