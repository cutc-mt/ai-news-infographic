Conversational Analytics API

自然言語でデータに問いかける新しいアナリティクス体験をアプリに組み込む

2026年2月6日 | Google Cloud Tech - David Tamaki Szajngarten

①Conversational Analytics APIとは: データとの対話

  • 自然言語でデータに質問して正確な回答を即座に得られるAPIサービス
  • SQLを書く必要もダッシュボードをクリックする必要もない
  • Geminiスタックの一部として提供されている
  • 自分のアプリケーションに会話型インテリジェンスを直接埋め込める

②Lookerの内部エンジンを外部公開: API化の意義

  • Looker内部で動いている会話型アナリティクスエンジンと同じ技術をAPIとして公開
  • Webページ上だけでなく、どこからでもAPIを呼び出せる
  • 開発者が自由にアプリケーションに統合可能
Lookerの強力な分析機能を
自分のアプリに持ってこれる！

③データエージェント: スキーマを理解するAI

  • データエージェントを作成して、スキーマ・ビジネスロジック・データ接続を理解させる
  • 自然言語クエリを送ると、エージェントがSQLやLookMLに翻訳する
  • 構造化された結果を返すので、アプリ側で自由に表示できる
「先月の売上は？」と聞くだけで
SQLが自動生成される！

④システム指示: モデルのグラウンディング

  • 静的なモデルではなく、開発者がカスタマイズして制御できる
  • System InstructionsとAuthored Contextで独自の指示や例を定義
  • モデルをグラウンディングして、正確で一貫した回答を保証する
ブラックボックスではなく、
挙動を細かく制御できるのが強み！

⑤フィルタリングの制御: 特定フィールドの指定

  • エージェントに常に特定のフィールドでフィルタするよう指示できる
  • サンプルの自然言語質問と対応するクエリの例を提供できる
  • 重要な用語のグロッサリー（用語集）を定義してエージェントの理解を助ける
  • スキーマリレーションを使ったテーブル結合の方法を説明できる

⑥サンプル質問の定義: 正確な翻訳のガイド

  • 自然言語の質問と、それに対応するクエリのペアを事前に定義できる
  • エージェントが正しいクエリパターンを学習するための教師データになる 
// サンプル質問の例
Q: "先月の売上トップ10は？"
-> SELECT product, SUM(revenue)
   FROM sales
   WHERE month = LAST_MONTH
   GROUP BY product
   ORDER BY 2 DESC LIMIT 10

⑦用語集とスキーマ: ビジネスロジックの共有

  • グロッサリーで業界固有の用語やビジネス用語を定義する
  • テーブル結合の方法をスキーマリレーションで説明する
  • エージェントがビジネスコンテキストを正確に理解して、適切なクエリを生成できる
「MRR」「チャーンレート」など
社内用語も教えられる！

⑧会話管理のオプション: ステートフル vs ステートレス

  • ステートフルセッション: 既存のエージェントや会話オブジェクトで状態を管理する
  • ステートレスチャット: 動的に生成したインラインコンテキストで完全にステートレスに運用
  • ユースケースに応じて最適な会話管理方式を選択できる

⑨IAMアクセス制御: 組み込みのセキュリティ

  • IAMアクセス制御が組み込みで提供されている
  • データエージェントの作成・共有・チャットに対する権限を定義
  • ステートレスチャットの利用権限も事前定義ルールで管理する
  • エンタープライズレベルのセキュリティで安全にデータ分析を提供できる
誰が何をできるかを
きめ細かく制御できる！

⑩マルチターン・マルチソース推論

  • APIなのでGemini Live APIや他のLLMツールと自由に組み合わせられる
  • マルチターンの対話で段階的にデータを深掘りできる
  • 複数のデータソースにまたがる推論が可能
「もう少し詳しく」「地域別に見せて」
のような追加質問にも対応！

⑪Looker MCPサーバー: エージェント間の接続

  • Looker MCPサーバーを使って他のエージェントと接続できる
  • Agent Development Kit (ADK)との統合もサポート
  • ADKのAsk Data Insightsツールを使ったことがあれば、すでにCA APIを体験済み
MCPで他のAIエージェントと
シームレスに連携できる！

⑫静的ダッシュボードの先へ: 動的な分析体験

  • 従来の静的ダッシュボードを超えた分析が実現する
  • どのフィルタをクリックすべきか推測する必要がなくなる
  • 質問を動的に投げかければ、エージェントがクエリの構造を自動推論する
  • 会話のように感じるが、裏では安全でガバナンスの効いたデータ上で実行される

⑬BigQueryとの連携: クエリ自動生成

  • BigQueryをデータソースとして利用可能
  • エージェントが質問を解釈し、適切なBigQueryクエリを自動実行する
  • 自然言語のレスポンスと構造化された結果の両方を返す
BigQueryの巨大データにも
会話で簡単にアクセスできる！

⑭レスポンスの二重構造: NL + 構造化データ

  • 自然言語レスポンスで人間にわかりやすい回答を提供
  • 構造化結果でアプリが自由に処理・表示できるデータを返す
  • アプリ側でレスポンスの表示方法を自由にカスタマイズできる 
// レスポンスの構造
{
  "natural_language": "先月の売上は...",
  "structured_result": {
    "columns": ["product","revenue"],
    "rows": [["商品A", 15000], ...]
  }
}

⑮Pythonクイックスタート: すぐに始められる

  • クイックスタートリポジトリをクローンするだけですぐに動くサンプルが手に入る
  • Pythonで簡単にチャットメッセージを送信してレスポンスを処理できる 
# シンプルなPython呼び出し例
response = ca_client.chat(
    agent_id="my-data-agent",
    message="先月の売上トップ5は？"
)
print(response.natural_language)
print(response.structured_result)
