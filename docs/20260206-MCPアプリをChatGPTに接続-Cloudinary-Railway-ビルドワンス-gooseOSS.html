①Codename Gooseとは: Blockが開発するオープンソースAIエージェント

  • Codename GooseはBlock社が開発するオンマシン型のオープンソースAIエージェント。複雑なタスクを自律的に処理できる。
  • MCPベースのエクステンションでGitHub、JetBrains IDE、Google Driveなどと接続可能。カスタム統合も構築できる。
  • 様々なLLMプロバイダーに対応しており、好みのモデルを選んで統合できる。デスクトップアプリとCLIの両方で利用可能。
  • コードマイグレーション（Ember→React、Ruby→Kotlin）、テスト生成、未知のコードベースの調査など幅広いタスクに対応。
GooseはBlockが作る無料のAIエージェント！

③MCPアプリの核心: Build Once, Use Everywhere

  • MCPアプリの最大の利点は「一度構築すればどこでも使える」ポータビリティ。
  • GooseでもChatGPTでも、同じMCPアプリが最小限の変更で動作する。これがMCPの設計思想。
  • JSON-RPCがMCPの通信標準として使われており、プロトコルの標準化がポータビリティを実現している。
  • コードの書き直しなしに、異なるホスト間でアプリが動作することが最終目標。
コードの書き直しなしに
どのAIホストでも動く！理想形

④MCP UIとMCPアプリの違い: リソースとしてUIを送信

  • MCP UIではUIをインラインで送信するが、MCPアプリではリソースとしてUIを送信する。
  • リソースのcapabilities、resource listメソッド、resource readメソッドを追加する必要がある。
  • JSON-RPCが通信の標準化を担い、MCP UIサーバーからMCPアプリへの変換を可能にしている。
  • UIはiframe内で表示される。リソースURIがiframeのルートとして機能する。
UIをリソースとして送るのが
MCPアプリのポイント！

⑤静的URIへの変更: 動的から固定のリソースURIへ

  • 元のMCPアプリではアップロードのたびにランダムなURIページが動的に生成されていた。
  • ChatGPT対応のために、1つの安定した（static）URIに変更する必要がある。URIは変更されない。
  • この静的URIがiframeのルート名として機能し、ChatGPTがUIにアクセスする際のエントリポイントになる。
  • ファイル構成も変更: mcp-app.tsからindex.http.ts（ホスティング用）とstudio用の2ファイルに分離した。
動的URIから静的URIへの変更が
ChatGPT対応の第一歩！

⑥HTTPエンドポイントの作成: ローカルからリモートへ

  • ChatGPTアプリに接続するにはHTTPのMCPエンドポイントが必要。ローカルのままでは動作しない。
  • index.http.tsファイルを新規作成し、サーバーコードをホスティングできる形にした。
  • ローカルのstudioトランスポートも引き続き使えるように、別ファイルとして残している。
  • サーバーファイルの作成はAIエージェントに「ローカルMCPアプリをホストしたい」と伝えれば生成してもらえる。
ローカルからHTTPへ！
AIに頼めばサーバーコードは即生成

⑦Railwayでホスティング: GitHubから簡単デプロイ

  • 最も簡単な方法はGitHubからコードをプッシュしてRailwayに連携すること。
  • Railwayでnew projectを作成し、GitHubリポジトリを選択するだけでデプロイが始まる。
  • 環境変数としてCloudinaryのAPI Key、API Secret、Cloud Nameを登録する必要がある。
  • ポート8080で「running」のメッセージが確認できればデプロイ成功。Vercelなど他のサービスでも可能。
GitHubにプッシュ → Railwayで選択
環境変数を設定すればデプロイ完了！

⑧Railwayのハマりポイント: GitHub連携の罠

  • Railwayのsettingsで「リポジトリが見つからない」エラーが発生する場合がある。
  • 原因: RailwayアプリをGitHubに追加していない。Account Settings→下部にスクロールで設定できる。
  • GitHubとRailwayのアプリ連携を設定すると、リポジトリが正しく検出されるようになる。
  • Ebony氏もここで時間を取られた。Railwayを初めて使う場合は要注意のポイント。
Account Settingsの下部で
GitHubアプリを追加するのを忘れずに！

⑨ドメイン生成: RailwayからURLを取得

  • Railwayのsettings → networking セクションでドメインを生成する。
  • ポートは8080を指定する。コード内のポート設定と一致させること。
  • 生成されたURLがChatGPTからMCPアプリにアクセスするためのエンドポイントになる。
  • URLの先頭にhttps://を必ず付けること。付けないとChatGPTがURLを認識できない。
Settings → Networking で
ドメイン生成 → httpsを忘れずに！

⑩ChatGPTでの設定: デベロッパーモードを有効化

  • Settings → Apps → Advanced SettingsでDeveloper Modeをトグルオンにする。
  • MCPアプリを追加するにはDeveloper Modeが必須。通常は無効化されている。
  • 「Create App」をクリックし、アプリ名・説明・MCP Server URLを入力する。
  • 認証設定は「No Auth」を選択。セキュリティの警告が出るがそのまま進める。
Developer Modeをオンにしないと
MCPアプリは追加できないよ！

⑪ChatGPT接続の手順まとめ: 5ステップで完了

  • Settings → Apps → Advanced Settings → Developer Modeをオンにする。
  • 「Create App」でアプリ名を入力（例: Cloudinary MCP App）。
  • RailwayのURLをhttps://付きでMCP Server URLに貼り付ける。
  • 認証は「No Auth」→ 警告を承認 → 「Create」をクリックして完了。
たった5ステップで
MCPアプリがChatGPTで動く！

⑫リフレッシュボタンの罠: 変更が反映されない問題

  • コードを変更してもChatGPTに反映されない問題にEbony氏が長時間悩まされた。
  • 原因: Settings → Apps → アプリをクリック → 下にスクロールするとRefreshボタンがある。
  • コードを更新した後は必ずこのRefreshボタンを押す必要がある。これを知らないと変更が反映されない。
  • 「Don't be me」とEbony氏。同じ過ちを繰り返さないためにも覚えておくべきポイント。
Refreshボタンを見つけるまで

⑬CSS問題: UIスタイルが表示されない

  • 最初はUIのデータ（画像、Public IDなど）は表示されるが、CSSスタイルが全く反映されない状態だった。
  • 原因はタイプミス（typo）。CSSのインポートパスなど細かいミスがスタイル欠落を引き起こしていた。
  • ボタンや情報は正しく表示されるがデザインが崩壊しているので、見た目だけでは原因がわかりにくい。
  • MCP UIからMCPアプリへの移行時は、CSS関連のパスやインポートに特に注意が必要。
CSSのtypoで見た目が崩壊…
データは正しいのにスタイルがない！

⑭UI更新通知の欠落: データがあるのにUIに反映されない

  • detailsにはPublic ID、フォーマット、リソースタイプなど全情報が揃っているのにUIに画像が表示されない。
  • 原因: リソース取得後にUIを更新する通知（notification）の行が欠けていた。
  • リソースを読み込んでデータは保存されるが、UIに「情報が更新された」と通知しないと表示に反映されない。
  • この1行の通知コードを追加したら問題が解決。シンプルだが見落としがちな重要ポイント。
たった1行のnotificationで解決！
UIにデータ更新を通知するのを忘れずに

⑯ローカルファイルの制限: URL経由のみアップロード可能

  • ホスティングされたMCPアプリはローカルファイルシステムにアクセスできないため、ローカルファイルのアップロードは不可。
  • 現時点ではURL経由の画像のみアップロードに対応している。
  • ChatGPT固有の解決策としてwindow.openai.uploadFileメソッドが利用可能。
  • ホストに依存しない汎用的な解決をするにはより多くのコードが必要になるが、ChatGPT専用なら簡単。
ローカルファイルは現状不可
ChatGPT専用APIで対応は可能！

⑰ChatGPT拡張API: ホスト固有の機能

  • ChatGPTはMCPの標準にない機能を追加できる拡張APIを提供。Instant Checkout（即時購入）も可能。
  • File Upload、Post（投稿）、Modal（モーダルウィンドウ）など、ChatGPT専用の機能が利用できる。
  • ポータビリティとホスト固有機能のバランスが重要。汎用性を取るか、特定ホスト向け機能を取るかの判断が必要。
  • MCPアプリの標準機能で足りない場合のみ、ホスト固有APIを活用するのが推奨される使い方。
