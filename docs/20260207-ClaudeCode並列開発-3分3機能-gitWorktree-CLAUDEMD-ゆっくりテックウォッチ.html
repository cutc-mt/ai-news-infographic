⑥git worktreeによるフォルダ分離: 3体同時の秘密

  • 通常のブランチだとフォルダが1つで1体ずつしか動けない
  • worktreeならフォルダ自体が分かれるから3体が同時に動ける
  • 各フォルダが独立したブランチとして機能する 
# git worktreeの作成コマンド
git worktree add ../session-1 -b feature/json
git worktree add ../session-2 -b feature/base64
git worktree add ../session-3 -b feature/jwt

# 各フォルダで独立してClaude Codeを起動
cd ../session-1 && claude
cd ../session-2 && claude
cd ../session-3 && claude
フォルダが別れるから同時編集が可能に！

⑦キープアウト設計: 立入禁止がコンフリクトを防ぐ

  • セッション0が作った共通ファイルには絶対に触れないルールを設定
  • 各セッションは自分のフォルダだけを実装する
  • これがコンフリクトゼロの秘密。ファイルの重複を物理的に排除している
  • CLAUDE.MDに「触れてはいけないファイル」を明記することで徹底する
「ここは触るな」を最初に決めるのが
並列開発の鉄則！
