①概要: NotebookLMへのリンク収集を自動化

NotebookLMにリンクを一気にまとめて渡せるChrome拡張を発見し、自作してみる実験。複数のページからリンクを効率的に収集して、NotebookLMに一括で渡せる仕組みを作成します。

「知識をAIに読ませてツール化する」という新しい開発スタイルの実践
Chrome拡張 自動化

②元ネタ: 高菜こんぶさんのリンク収集ツール

Xで話題になった高菜こんぶさんのポストがきっかけ。既存のリンク収集ツールは無料版だと3個までという制限があります。そこで、より自由度の高いChrome拡張を自作することを決意。

「制限があるなら、自分たちで作ってしまおう」という発想から出発
Twitter発見 制限突破

③開発アプローチ: note記事をAIに学習させてスキル化

既存のツール開発記事をnoteから探し、その記事をAIに読ませます。そして、その知識からスキルを生成し、そのスキルを使ってChrome拡張を自作する。知識→スキル→実装という流れです。

「記事の知識を道具に変える」という斬新なアプローチ
知識活用 スキル生成

④知識を道具にする開発サイクル

「記事→AI→スキル→実装」という新しい開発スタイルを確立。従来のプログラミングの手順を完全にスキップして、AIの力を活用することで開発速度を飛躍的に向上させます。

「Antigravityがあれば、動画も撮影もいらない」という効率化を実現
効率化 新開発スタイル

⑤スキル生成プロセス: note記事から自動化

Antigravityにnote記事の原稿を読み込ませ、AIが自動的にChrome拡張スキルを生成します。この過程では、要件定義やコード生成、さらにはアイコン生成までが自動で完了します。

「記事を入力するだけでスキルが完成する」という高度な自動化
自動化 スキル化

⑥スキルの内容: 全自動開発フロー

生成されたスキルには、動画撮影ステップをスキップして、要件定義から始まります。その後、コード生成、アイコン生成、そしてコードレビューまで全自動で実行されるという優れた設計になっています。

「手作業が一切ない完全自動化フロー」を実現
完全自動 品質保証

⑦使用モデル: Claude Opus 4.6の高速性

Antigravity上で利用可能なClaude Opus 4.6を採用。このモデルは高速で品質が優れているという特徴があり、複雑なChrome拡張の開発を効率的に進めることができます。

「Opus 4.6の能力があれば、複雑なロジックも一瞬で完成」
Opus 4.6 高速生成

⑧バージョン1開発: 要件整理から完成まで

最初のバージョン開発では、要件整理を行い、コーディングを進めます。その後、Gemini 3 Pro Imageを使ってアイコンを生成し、最後にセルフレビューを実施。全工程が自動で進行します。

「人間がすることは最初の『お願い』だけ」という理想を実現

⑨バージョン1の課題: ポップアップ方式の限界

完成したバージョン1を試してみると、ポップアップ方式で1ページずつ移動する必要がありました。このUXはイメージと異なり、使いにくいことが判明します。

「実装後に気づく『こういうのじゃない感』がありました」
UX改善 課題発見

⑩バージョン1.1へ: サイドパネル方式に進化

AIのセルフレビューを受けて、ポップアップからサイドパネル方式に変更。さらに、リンク収集モードを追加することで、ページを閉じずに連続でリンク収集できるようにしました。

「AIのフィードバックをすぐ反映できるのがAntigravityの魅力」
UI改善 機能追加

⑪サイドパネルUI: Chrome Side Panel APIの活用

Chrome Side Panel APIを使って、ブラウザの右側に固定表示されるパネルを実装。これにより、常にリンク収集インターフェースにアクセス可能になり、ワークフローが大幅に改善されました。

「右側に常駐するパネルで、リンク収集が快適に」
UI/UX Chrome API

⑫リンク収集モード: クリックで自動収集

リンク収集モードをオンにすると、ページ上のリンクをクリックするだけで自動的に収集されます。複数のページを回遊しながら、効率的にリンクを集めることができるようになりました。

「ワンクリックでどんどんリンクが溜まっていく快感」
効率化 自動収集

⑬セルフコードレビュー: 品質の自動検証

Antigravity内のセルフレビュー機能を使って、セキュリティ、品質、パフォーマンスの観点から自動的にコードをレビュー。その結果、3つの修正が必要と判定され、自動的に改善されました。

「AIが自分のコードを厳しく審査して改善する」という新しい品質管理
品質保証 セキュリティ

⑭完成版の実演: ワンクリック一括収集成功

完成したChrome拡張を実際に使ってみると、noteの記事一覧からワンクリックでURL収集でき、NotebookLMに一括貼り付けが成功しました。当初のビジョンが完全に実現されました。

「見た目で期待した通りの動きをしてくれる完成度の高さ」
完成 実運用

⑮リッチコピー機能: タイトルとURLのセット

単純なURLコピーだけではなく、タイトルとURLの形式でコピー可能な「リッチコピー機能」を実装しました。これにより、NotebookLMへの貼り付けがさらに便利になり、アウトライン化が容易になります。

「タイトル付きで貼り付けられると、整理が格段に楽」

⑯YouTube対応: 動画一覧からも収集可能

noteだけでなく、YouTubeの動画一覧からもリンク収集できるように拡張されました。複数の情報源からシームレスに収集できるようになり、NotebookLMの資料集め効率が大幅に向上します。
