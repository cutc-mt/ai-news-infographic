OpenClaw セキュリティ問題と自作セカンドブレイン

185,000スターの大人気ツールの脆弱性と、Claude Codeで自作する方法

2026年2月12日 | Cole Medin

①OpenClawの爆発的人気: 185,000スター

GitHubで史上最速でスター数が爆増したオープンソースプロジェクト。N8Nを抜いて歴代最多の185,000スターを獲得。個人のAIアシスタントとして革命的なツール。

初めて「あなたを理解してくれる」エージェント！

②セキュリティ問題: ワンクリックRCE脆弱性

CVE-2026-25253として報告された重大な脆弱性。悪意のあるリンクを1回クリックするだけで、攻撃者があなたのOpenClawインスタンスから全データとAPIキーを盗める。

CVSS 8.8 RCE

③ClawHubの悪意あるスキル: 230以上のマルウェア

公開マーケットプレイスに230〜414個以上の悪意あるスキルが投稿されていた。76個がマルウェア確認済み、91%がハイブリッド攻撃を使用。APIキーやSSH認証情報を盗む。

公開スキルのダウンロードは危険！

④平文保存の問題: 認証情報が丸見え

全てのAPIキーとトークンが平文で保存されている。攻撃者がアクセスすれば、プラットフォームアダプターや他のサービスへの認証情報が全て盗まれる。

暗号化なし リスク大

⑤2時間で乗っ取り: 研究者が実証

ある研究者は2時間もかからずにOpenClawインスタンスを乗っ取ることに成功。42,665以上のインスタンスが公開されており、多くが攻撃対象になっている。

攻撃者にとっては簡単な標的！

⑥Ciscoの警告: セキュリティ悪夢

CiscoはOpenClawを「Absolute nightmare（絶対的な悪夢）」と表現。コードベースが大きく、エージェントに過剰な権限を与えていることが根本的な問題と指摘。

Cisco AI Defense

⑦自作のアプローチ: OpenClawの良い部分を抽出

解決策はOpenClawの革新的なアイデアを参考にして自分で構築すること。2日間で数1000行のコードで同等の機能を実現可能。

学んで、理解して、自分用にカスタマイズ！

⑧メモリシステム: Markdown駆動の仕組み

soul.md（アイデンティティ）、memory.md（コアメモリ）、agent.md（振る舞いルール）などのMarkdownファイルで管理。SQLiteデータベースで軽量なRAGを実装。

シンプル エレガント

⑨ハートビート機能: プロアクティブなエージェント

heartbeat.mdを使って30分ごとに自動実行。メール、カレンダー、タスクを確認し、ユーザーに通知。朝起きると「メールを下書きしました」などの報告を受け取れる。

指示しなくても勝手に動いてくれる！

⑩チャネルアダプター: どこからでもアクセス

WhatsApp、Telegram、Slack、Discordなど好きなプラットフォームからエージェントに話しかけられる。スレッド対応で複数の会話を同時進行可能。

Slack Discord

⑪スキルレジストリ: 単一ファイルで機能追加

新しい機能を1つのファイルで追加可能。「フロントエンドの作り方」「PowerPoint生成」などのガイドをスキルとして登録。自分だけのレジストリで安全に管理。

悪意あるスキルをダウンロードするリスクなし！

⑫Claude Code活用: OpenClawを参考に自作

OpenClawのリポジトリをローカルにクローンしてClaude Codeに向ける。「メモリシステムを説明して、自分用に構築して」と指示するだけで実装可能。

MITライセンス 自由に利用可能

⑬技術スタック: Obsidian + SQLite + Claude

Obsidianでメモリ同期、SQLiteでデータベース、Claude Agent SDKでプロアクティブ機能。PostgreSQLはリモートデプロイ時に使用。コスト効率の良い構成。

サブスク課金でAPI課金を回避！

⑭Anthropic規制: OpenClawユーザーがBAN

OpenClaw使用でAnthropicサブスクリプションがBANされたユーザーが多数。自作ならClaude CodeとClaude Agent SDKを直接使用するため規約に準拠。

規約違反回避

⑮カスタマイズ性: 自分に最適化

NanoClawなどの軽量版もあるが自分の要件に完全に合わせるなら自作がベスト。アダプターはSlackとターミナルだけで十分という人も多い。

必要な機能だけを自分用に構築！

⑯構築プロセス: 驚くほど簡単

クローンして、Claude Codeで読み込ませて、指示するだけ。良い例を与えるとコーディングエージェントは最高のパフォーマンスを発揮。OpenClawが脳の栄養になる。

数日で完成

⑰継続的な改善: 新機能も取り込める

OpenClawに素晴らしい新機能が追加されたら、変更をプルしてClaude Codeに「これを自分用に構築して」と指示。常に最新のアイデアを取り込める。

エコシステムの進化を活用！

PeterのOpenClawより優れているわけではないが自分が何を望んでいるか正確に把握して構築できる。それが自分にとって最高のセカンドブレインになる。
