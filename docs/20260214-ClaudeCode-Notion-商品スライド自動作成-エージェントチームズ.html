Claude Code + Notion + PptxGenJS。PptxGenJSを使うとパワポファイル形式で崩れずに生成可能。画像生成はNavi 7 Proを使用。

⑤全体フロー: 6ステップ

Notionから情報収集→ストーリー設計→図の画像生成→パワポ作成→レビュー→修正。仕事の進め方から任せる。

フロー設計もAIにお任せ！

⑥エージェントチームズとは

AIのチームを素早く組成できる機能。各担当に役割を分担し、必要な情報だけを持たせてフレッシュな状態で作業可能。

チーム編成役割分担

⑦従来の課題: 情報の汚染

1つのAIで全てやると関係ない情報を掴みすぎてアホになる。Notion検索時に余計な情報まで読み込む問題。

情報過多で品質低下！

⑧解決策: 役割ごとの担当

Notion情報収集担当は収集だけ。ストーリー設計担当は収集結果だけ受け取って設計。フレッシュな頭で作業可能。

分業鮮度維持

⑨ポイント1: レビュー担当を設置

出来上がったスライドに対して「これじゃダメだ」と跳ねる担当をつける。厳しく跳ねることで何度も作り直し、品質向上。

厳しくレビューして品質UP！

スライド作成に約450分（7.5時間）かかったが、人間は待っているだけ。AIが各担当に分かれて繰り返し作業。

デザインスキルで品質向上！

データベース情報蓄積
