Goosetown マルチエージェント可視化

Steve YeggeのGas Townアイデアを実装した分散サブエージェントシステム

2026年2月14日 | Goose OSS

①Gas Town: Steve Yeggeのアイデア

Steve Yeggeが提唱したマルチエージェントワークスペースマネージャーの概念。34分の長いブログ記事で詳細に説明。

エージェントの未来像！

②Goosetown: Gas Townの実装版

Tyler LongwellがGas TownのアイデアをGoose用に実装。メインオーケストレーターが複数のサブエージェントを指揮。

Goose実装

③システム構成: オーケストレーター+研究者+作業者+レビュアー

研究者が情報収集→作業者が実装→レビュアーが品質確認の3層構造でタスクを処理。

分業システム！

④研究者: GitHub+arXiv+ローカル+Reddit

GitHub、arXiv、ローカルコードベース、Redditなど複数のソースから情報を収集する研究者エージェント。

マルチソース

⑤Summon拡張: サブエージェント起動

summon拡張を有効にすると、スキルをロードしてサブエージェントにタスクを委任可能。

拡張機能で起動！

⑥サブエージェント: コンテキスト節約

サブエージェントは独自のミニセッションで作業し結果だけを報告。メインエージェントのコンテキストウィンドウを節約。

コンテキスト節約

⑦GT Wall: エージェント間通信

エージェント同士がGT Wall（テキストファイル）でメッセージを交換。タイムスタンプとエージェントIDで管理。

メールシステム的な通信！

⑧Beads: タスク管理システム

Beads拡張でGitベースのタスク管理。イシューを作成し、進捗を追跡し、完了したらチェックオフ。

Gitベース

⑨RPGスタイル可視化: ガチョウの町

各デリゲートをガチョウのキャラクターとして表示するRPGスタイルのピクセルアートタウンを構築。

見た目も可愛く！

⑩吹き出し表示: GT Wallの会話を可視化

ガチョウにホバーすると吹き出しでGT Wallのメッセージを表示。エージェント同士の会話を可視化。

インタラクティブ

⑪町の施設: 図書館・広場・市庁舎

研究者は図書館、広場にはタウンスクエア、中央にはプラザなど役割ごとに場所を分けて表示。

町としての構造！

⑫Crossfire: 複数モデルでレビュー

異なるモデルを使ったレビュアーを召喚して多角的な視点で品質確認。crossfireコマンドで実行。

マルチモデル

⑬モデル切り替え: サブエージェントごとに可能

各サブエージェントに環境変数でモデルを指定可能。Opus 4.6やGeminiなど好きなモデルを割り当て。

柔軟な設定！

⑭トークン消費: 20%程度で完了

研究・PRD作成・タスク分割・サブエージェント起動までコンテキストウィンドウの20%程度で完了。効率的。

効率的

⑮ボイスプロンプト: 話しかけるだけでOK

音声でプロンプトを入力可能。タイプせずに話しかけるだけでエージェントに指示を出せる。

手軽な入力！

⑯Code Mode: さらなるトークン節約

Code Modeを使うと必要な時だけツールをロードし、トークン消費をさらに削減可能。

Code Mode

⑰夜間実行: 寝ている間に完了

オーケストレーターに任せれば寝ている間にタスクを完了。朝起きたら結果を確認できる。

時間を有効活用！

⑱結論: Gas Townの実用化

Gas Townの概念をGooseで実装したGoosetown。拡張機能を有効にするだけでサブエージェントオーケストレーションが可能。
