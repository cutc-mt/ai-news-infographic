Playwright CLIでエージェントテスト

自然言語でブラウザテストを自動生成・実行

2026年2月15日

①Playwright CLIをグローバルインストール

まずPlaywright CLIをグローバルにインストール。VS Codeでプロジェクトを設定し、Playwrightをインストール。

セットアップ

②スキルをインストール

playwright-cli install --skillsコマンドでスキルをインストール。Gooseエージェントから利用可能に。

スキルで機能を拡張！

③インストールされるファイル

.gooseフォルダーにスキルファイルと参照フォルダーが作成。テスト生成・セッション管理・トレーシング・ビデオ録画など。

.gooseスキル

④ビデオ録画スキル

ブラウザ操作セッションをビデオとしてキャプチャ。デバッグ用に自動操作を記録可能。

操作を動画で確認！

⑤テストコード自動生成

全てのアクションが対応するPlaywright TypeScriptコードを生成。出力から直接テストファイルにコピー可能。

TypeScript自動生成

⑥ブラウザを開く: 自然言語で操作

「playwright-cli open deb.codes」と入力するだけでブラウザが起動してサイトに移動。

自然言語でブラウザ操作！

⑦スクリーンショット機能

「スクリーンショットを撮って」と指示すると自動でスクリーンショットを保存。VS CodeフォルダーにPNGで保存。

スクリーンショット自動保存

⑧アクセシビリティツリーをYAMLで保存

ページ全体のアクセシビリティツリーをYAMLファイルとして保存。エージェントが要素を見つけるためのref情報を含む。

ページ構造をローカル保存！

⑨トークン節約: MCPとの違い

ページ情報をコンテキストに入れないためMCPサーバーよりトークンを大幅に節約。ローカルに保存して参照。

トークン節約効率化

⑩--headedオプション: ブラウザを表示

通常はヘッドレスモードだが、--headedでブラウザを表示可能。エージェントには不要だが、人間が確認する際に便利。

人間には見えるモード！

⑪自然言語でページ操作

「動画ページに行ってMCPでフィルターしてテストを作成」と指示すると自動で操作を実行してテストを生成。

自然言語自動操作

⑫生成されたテストコードの品質

waitForPage、goto、expect、getByRoleなど適切なPlaywrightコードを生成。手動で書いたテストと同等の品質。

本格的なテストコード！

⑬テストの実行と検証

生成したテストをGoose内で直接実行して検証可能。「テストを実行して」と指示するだけで動作確認。

実行検証

⑭テスト結果の確認

「テストが通りました」と実行結果が表示。ナビゲーション・クリック・URL確認など全てのステップが成功。

テストが成功！

⑮できることを確認

「playwright-cli、他に何ができる？」と聞くと全ての機能一覧を表示。クリック、入力、タブ管理など豊富な機能。

機能一覧

⑯ビデオとトレースの作成

「ビデオとトレースを作成して」と指示すると操作の録画とPlaywrightトレースを生成。後で確認可能。

全てを記録！

⑰トレースビューアで詳細確認

トレースファイルを開くとタイムライン、ネットワーク、コンソールログを確認。各アクションのrefも記録。

トレースデバッグ

⑱まとめ: エージェントテストの新時代

自然言語でブラウザ操作、テスト生成、実行、ビデオ記録まで全てを自動化。PRに添付して動作証明も可能。
