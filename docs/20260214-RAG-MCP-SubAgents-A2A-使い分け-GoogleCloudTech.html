RAG・MCP・サブエージェント・A2A 使い分けガイド

スイスアーミーナイフのように選ぶAI技術の正しい選び方

2026年2月14日 | Google Cloud Tech

①問題提起: どの技術を使うべきか？

MCP、A2A、RAG、サブエージェントなど多数のオプションで混乱。どの技術をどの場面で使うべきか明確なガイドが必要。

選び方には明確なロジックがある！

②FixIt Fast: 架空の会社を例に

冷蔵庫や洗濯機を修理する技術者のネットワークを管理する架空の会社「FixIt Fast」を例に、各技術の使い分けを解説。

例題設定

③RAG: 技術者ヘルパーボット

20年前の洗濯機のエラーコードを調べるにはRAG（検索拡張生成）を使用。マニュアルのライブラリから正しい情報を検索。

マニュアルから必要な箇所を抽出！

④RAGの実装: Vertex AI Search

フロントエンドはモバイル/Webアプリ、バックエンドはADK（Agent Development Kit）を使用。修理マニュアルはVertex AI SearchまたはRAG Engineでインデックス化。

Google Cloud

⑤MCP: エグゼクティブ洞察ダッシュボード

「北東地域の売上と夏のマーケティング費用を比較」のような質問にはMCPを連合ルックアップツールとして使用。

各データベースにMCPサーバー！

⑥MCPの利点: セキュリティと保守性

1つのエージェントが全DBにアクセスするとセキュリティと保守が困難。MCPなら各チームが自DBのMCPサーバーを所有し、共有データを制御可能。

セキュリティ

⑦MCPでRAGを使わない理由

RAGはデータベース全体にアクセスする必要があるが、実用的でない場合や安全でない場合がある。MCPなら共有するデータを厳密に制御可能。

データアクセスを制御！

⑧MCP: サプライボット（アクション）

技術者が部品を注文するにはMCPをAIの手として使用。バックエンドシステムで注文を実行する標準化された方法を提供。

アクション実行

⑨MCP vs 通常のAPI呼び出し

MCPはエージェントがビジネスロジックを発見・トリガーする標準化された安全な方法を提供。通常のAPIよりも再利用性が高い。

標準化されたインターフェース！

⑩サブエージェント: トリアージ・派遣

「冷蔵庫が漏れてブーンという音がする」のような複雑なタスクにはサブエージェントを使用。1つのプロンプトでは処理できない。

複雑タスク

⑪サブエージェント構成: マネージャー+専門家

マネージャーエージェントが専門エージェントに委任。診断担当、カレンダー担当、技術者通知担当などがADKで状態を共有しながら協力。

分業システムで効率化！

⑫サブエージェント vs A2A

単一アプリケーション内ならサブエージェントの方がシンプル。A2Aは異なる組織のエージェントが協力する場合に使用。

使い分け重要

⑬A2A: 保証検証

修理がメーカー保証の対象かを確認するにはA2A（エージェント間通信）を使用。FixIt Fastは各メーカーのDBにアクセスできない。

異組織間のエージェント通信！

⑭A2Aの利点: プライバシー保護

プライバシー法によりメーカーは生データアクセスを提供できない。メーカーのエージェントがインテリジェントファイアウォールとして機能。

プライバシー

⑮A2A: 非同期処理対応

A2AはPub/Sub経由で非同期処理が可能。複雑な保証請求で人間の承認が必要な場合も対応できる。

非同期で柔軟に対応！

⑯RAGまとめ: コンテンツライブラリから情報取得

コンテンツライブラリから情報を知りたい場合はRAGを使用。マニュアル、ドキュメント、ナレッジベースからの検索に最適。

情報検索

⑰MCPまとめ: データアクセスとアクション

データアクセスを標準化するならMCPをルックアップツールとして使用。アクションを実行するならMCPをAIの手として使用。

読み取りと書き込み両対応！

⑱まとめ: 明確な使い分けルール

RAGは情報検索、MCPはデータアクセスとアクション、サブエージェントはアプリ内チームワーク、A2Aは異組織間通信。このルールで迷わず選択可能。
