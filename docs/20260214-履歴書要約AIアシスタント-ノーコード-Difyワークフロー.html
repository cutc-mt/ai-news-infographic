履歴書要約AIアシスタント構築（ノーコード）

DifyワークフローでHR業務を自動化する7ステップ

2026年2月14日

①ノーコードで構築可能

技術的背景がなくても誰でもこのワークフローを構築できる。既に社内のHR部門と共有済みで実用的なシステム。

プログラミング不要でAI活用！

②ノード1: 履歴書アップロード

Startノードでユーザー入力を選択。ファイル形式で履歴書をアップロード。単一ファイル・ドキュメント形式・必須項目に設定。

User InputFile Upload

③ノード2: Doc Extractor

PDFやWordファイルからテキストを抽出。LLMはPDFを直接読めないため、このステップでプレーンテキストに変換する。

PDF→テキスト変換！

④ノード3: Parameter Extractor

抽出したテキストからキーワードリストを作成。AIアシスタントの役割を定義し、検索用キーワードを構造化。

LLM構造化

⑤プロンプト設計: 役割と目標

「専門的な採用背景調査アシスタント」として役割を定義。履歴書を分析しカテゴリー別にキーワードを抽出する目標を設定。

AIに役割を与える！

⑥制約条件の追加

キーワードのみを出力する制約を追加。不要な情報を排除し、効果的で有用な検索リストを作成可能。

制約条件最適化

⑦ノード4: Iteration（反復）

トマトをスライスするように各キーワードに対して同じアクションを実行。キーワード1つずつを検索する反復処理。

リストの各要素に同じ処理！

⑧Tavily Searchで検索

Tavily APIを使用してキーワード検索。APIキー取得が必要だが、直感的で使いやすい検索ツール。

TavilyAPI

⑨検索パラメータ設定

Basic検索・一般トピック・7日以内の情報・最大5件の結果を設定。予算に応じて3〜4件に調整可能。

バランス良い設定で最適化！

⑩並列処理: 2つのLLMノード

検索結果を要約用と用語解説用の2つのLLMに分岐。並列で処理し効率的な実行を実現。

並列処理効率化

⑪ノード5: Summary（要約）

「10年経験のシニアリクルーター」の役割で候補者の要約を作成。学歴・経験・主要責任・採用判定を含む簡潔なブリーフ。

採用判定まで自動化！

⑫ノード6: Glossary Explanation

技術スタックやプロジェクト名の用語解説。类比（アナロジー）を使って分かりやすく説明する設定も可能。

用語解説アナロジー

⑬ノード7: 結合して出力

要約と用語解説の2つの出力を1つに結合。最終的な要約レポートとして候補者情報を一覧できる。

全ての情報を統合！

⑭Test Runで動作確認

テスト用履歴書でTest Runを実行。視覚的に各ノードの動作状況を確認でき、エラー箇所も特定しやすい。

デバッグ視覚化

⑮出力例: 背景情報

候補者のJordan Syntax - 学歴 - 会社（Stripe、台湾テック企業）など基本情報を整理。2年間のStripe勤務などを要約。

経歴をスッキリ整理！

⑯出力例: 技術用語解説

Docs as Code・GraphQL・CMSなどの専門用語を分かりやすく解説。アナロジーを使った説明で非技術者も理解可能。

技術用語可読性

⑰DSLファイルで共有

完成したワークフローはDSLファイルとして共有可能。インポートすれば同じ設定をすぐに利用できる。

ワークフローを再利用！

⑱プロンプト改善のコツ

自分でドラフトを作成し、AIに改善を依頼。効率的なプロンプトの書き方を学ぶことで、ワークフロー構築がよりスムーズに。

プロンプトエンジニアリング継続改善
