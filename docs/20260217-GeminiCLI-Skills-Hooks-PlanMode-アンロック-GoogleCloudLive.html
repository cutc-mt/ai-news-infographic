Skills、Hooks & Plan Modeでワークフロー自動化

Google Cloud Tech | 2026/02/17

①Gemini CLIとは: ターミナルネイティブAIエージェント

ターミナル内で直接動作するAIエージェント。ファイル、ツール、MCPサーバーにアクセスし、開発ライフサイクルの一部として機能する。開発者だけでなく誰でも使用可能。

②Memory Wallデモ: 20分で構築したアプリ

家族用のデジタル掲示板アプリ。Three.js + Reactで構築され、Firebaseホスティングとストレージを使用。Gemini CLIで約20-25分でアイデアからライブアプリに。

③Hooksとは: ライフサイクルで実行されるスクリプト

Gemini CLIのエージェントループの異なるライフサイクルポイントで実行されるスクリプト。セッション開始、ツール呼び出し前、エージェント後など様々なイベントでトリガーされる。

④Hello World Hooks: Lint自動実行

最も一般的なHook用途はコード生成時の自動Lint。Gemini CLIがコードを書く際にRuff（Python）やnpm run lint（Node）を自動実行し、エラーをその場で修正。

⑤決定論的動作: 不確実性への対処

Hooksはエージェントループに決定論的動作を追加できる。console.log文のチェック、Gitコミットのサマリー、開発サーバーの起動確認など、自動化したいタスクを定義可能。

⑥開発サーバーHook: セッション開始時チェック

settings.jsonで設定。ポート5173/5174でサーバーが実行中か確認し、システムメッセージで警告を表示。バックグラウンド実行対応で、CLIがブロックされなくなった。

⑦Skillsとは: 標準化されたバンドルされた専門知識

エージェントスキルは標準フォーマットで、プロバイダー間で互換性がある。Markdown形式で、YAMLフロントマターに名前と説明を含む。プログレッシブ開示が特徴。

⑧Custom Commandsとの違い: ユーザー vs エージェント

Custom Commandsはユーザーがトリガーする事前パッケージ化されたプロンプト。Skillsはエージェントがトリガーし、必要なときにだけ専門知識を読み込む（プログレッシブ開示）。

⑨プログレッシブ開示: コンテキス汚染防止

Skillsは「棚の本」のように待機し、トリガー時にだけ読み込まれる。100個のSkillsを持っていても、アクティベートされるまでコンテキストやトークンを消費しない。

⑪3JS Expert Skill: ベストプラクティス適用

Three.jsの専門知識をまとめたSkill。「3JS」という単語が含まれると自動的にトリガーされ、ベストプラクティスに基づいてコードレビューと改善提案を実行。

⑪ドキュメント作成Skill: 開発者が嫌う作業を自動化

「Doc Writer Skill」を対話的に作成。Gemini CLIが質問し、スタイルガイド、テンプレート、参照ファイルを自動生成。ワークスペーススコープでインストール。

⑫チーム共有: GitHubリポジトリでの管理

Skillsはagents/skillsフォルダに配置し、GitHubリポジトリでチーム共有可能。Verscellのnpx_skillsレジストリでは数千のSkillsが公開されている。

⑬Plan Mode: 読み取り専用プランニングモード

プレビュー中の機能（来週デフォルト有効化）。ファイル編集・書き込みを無効化し、調査→計画→フィードバックのサイクルを実行。あいまいなプロジェクトに最適。

⑭Plan Modeの活用シーン: Readonly MCPサーバー

GitHub MCPサーバーでIssueを読み取り、Gemini CLI拡張でGoogle DocsやSheetsからコンテキストを取得。実装前に全ての要件とコンテキストを確認可能。

⑮モード切替: Plan / Default / Auto Accept

Shift+Tabでモード切替。Plan（緑）は読み取り専用、Default（青）は手動承認、Auto Accept（黄）は編集を自動承認。プランの詳細度に応じて使い分け。

⑯Playwright MCP: ブラウザテスト自動化

UIを持つアプリケーションのテストにPlaywright MCPサーバーを使用。Gemini CLIがブラウザを開き、クリック、テスト、結果を報告。機能が動作しない場合のデバッグに活用。

⑰Ask User Tool: 対話的な質問機能

テキスト入力だけでなく、選択肢、Yes/No、複数選択など対話的な質問が可能。熟練した開発者のように要件を明確にしてから開発を開始できる。

⑱まとめ: Extensions、Skills、MCPの棲み分け

Extensionsは機能をバンドルしたパッケージ。Skillsはトリガーされる専門知識。MCPはツール統合。これらを組み合わせてGemini CLIをカスタマイズし、開発ワークフローを完全に自動化する。
