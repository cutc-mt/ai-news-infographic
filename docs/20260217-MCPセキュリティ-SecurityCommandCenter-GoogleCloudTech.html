①Security Command Center: AIセキュリティ管理の要

エージェントのセキュリティはワンステップのセットアップではなく継続的なプロセス。SECはAIセキュリティとエージェントエコシステム監査の中央ハブ。

②SEC AI Protection: AIワークロードのセキュリティ管理

AIワークロードのセキュリティポスチャを管理し、AI関連のリスク、脆弱性、脅威を単一のダッシュボードで管理。自動発見機能でAIアセットを特定。

③自動発見: AIアセットのインベントリ

モデル、データ、エージェント、MCPサーバーを含むAIアセットを単一のインベントリビューで表示。どのモデルとデータセットが使用されているか、エージェントエコシステムとの接続、潜在的なリスクを可視化。

④ポスチャ管理: 誤設定の継続的モニタリング

エージェントとMCPサーバーをホストするインフラの誤設定を継続的に監視。ツールを侵害の露出するMCPサーバーの脆弱性を検出してフラグを立てる。

⑤Model Armor保護の検証: エージェントインスタンス

基盤モデルやVertex AIエンドポイントがModel Armorで保護されていないインスタンスを特定。MCPサーバーからのデータに悪意のある内容が含まれていても、モデル処理前に検査されることを保証。

⑥実行時検知: ジェイルブレイク＆間接注入

Model Armorの実行時検知結果をSECに一元化。直接のジェイルブレイク試行だけでなく、ブロックされた間接注入やMCPツールレベルで検出された有害データも含まれる。

⑦可観測性: エージェントの思考と行動のログ

エージェントが背後で何を考えて何をしているかを理解する可観測性ツールが必要。全てのアクションをログし、エージェントID、セッションID、ペイロード詳細をキャプチャ。

⑧アイデンティティ検証: 最小限の権限原則

SECのポスチャ管理はエージェントのアイデンティティが最小限の必要な権限のみ維持していることを検証。プラットフォームセキュリティコントロールへのコンプライアンスも監視。

⑨CMEコンプライアンス: 顧客管理暗号化キー

モデルやデータセットなどのVertex AIアセットで顧客管理暗号化キー（CME）を要求するなどのプラットフォームセキュリティコントロールへのコンプライアンスを監視。

⑪アクション可能なガイダンス: 脆弱性の修正

SECは問題を検知するだけでなく、脆弱性を修正するための実行可能なガイダンスまたは提案を提供。ダッシュボードはインフラのボトルネック（Choke Points）を特定。

⑪ボトルネック修正: 一石二鳥のセキュリティ向上

1つのファイアウォール保護を修正するだけで、ダウンストリームの全MCPサーバーと他のAIアセットを一度に保護。数回の簡単なステップで最大のインパクトを持つ修正を優先。

⑫コントロールマッピング: Google Cloudへ

特定されたMCPセキュリティリスクをAIPとModel ArmorなどのGoogle Cloudコントロールにマッピングし、最大のインパクトを持つ修正を優先可能。

⑬SDPディスカバリー: ハードコードされたシークレット

Sensitive Data Protection（SDP）ディスカバリーを使用してMCPサーバーを定期的にスキャン。ビルドおよび実行時環境変数に露出したハードコードされたシークレット（APIキーなど）を検出。

⑭シークレットのローテーションとSecret Manager

発見後、シークレットをローテーションし、Secret Managerを使用して平文から隠蔽。ステップバイステップのラボと動画が提供されている。

⑴シリーズ構成: MCPセキュリティの3部作

Part 1: MCPアーキテクチャ、攻撃面、プロンプトインジェクション。Part 2: アイデンティティ、ネットワーク分離、実行時保護のセキュアパターン実装。Part 3（本動画）: SECによる可視化と自動化。

⑯まとめ: 継続的なセキュリティ管理

Security Command CenterはAIインベントリの可視化、アクティブな実行時脅威の検知、脆弱性を修正するための実行可能なステップを提供。エージェントセキュリティは継続的なプロセス。
