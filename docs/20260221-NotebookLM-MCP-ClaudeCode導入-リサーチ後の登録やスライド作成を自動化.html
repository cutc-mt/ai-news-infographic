NotebookLM MCP 解説

Claude Codeでリサーチ後の登録やスライド作成を自動化

2026-02-21

①NotebookLM MCP

ノートブックの作成、ソース追加、ディープリサーチ、コンテンツ作成が可能。Claude Codeでエージェントチームを使ってリサーチする場合に最適。

②WSL環境への導入

UVツールをインストールし、NotebookLM MCPをインストール。バージョン確認で正しく導入されたことを確認して進める。

③ログイン認証

Chrome拡張機能「Get cookies.txt LOCALLY」を使用してクッキーを取得。NotebookLM.google.comに保存して認証情報を設定。

④ファイルアップロード

Claude Codeがファイルを1回読んでしまう問題がある。ファイルをそのままアップロードする機能が今後の課題。

⑤MCPの利便性

定常的なアクションはMCPに追加すると便利。NotebookLM系は定常的な作業なので、追加しておくと作業効率が向上。

⑥リサーチ機能

NotebookLM MCPでディープリサーチが可能。リサーチ後の登録やスライド作成を自動化できる。

⑦スライド作成

スライドの自動作成が可能。手作業では時間がかかる作業を自動化することで効率化が図れる。

⑧設定ファイル

settings.jsonに使用するコマンドを登録。毎回確認がめんどくさいので、確認しなくてもいいように設定しておくと便利。

⑨一括処理

複数のファイルをまとめてアップロードできる。1個ずつ入れる必要はなく、まとめファイルを作成すれば一括処理が可能。

⑩エージェントチーム

Claude Codeのエージェントチームと組み合わせて使用。NotebookLM MCPと連携することで、より高度な自動化が可能。

⑪導入の苦戦

クッキー認証やファイルアップロードで少し手間取ったが、最終的には無事に導入できた。

⑫導入完了

NotebookLM MCPが追加され、便利に使えるようになった。時間はかかったが、一度導入すると非常に便利。

⑬自動化の効果

リサーチ後の登録やスライド作成が自動化されることで、作業効率が大幅に向上。手作業よりはるかに効率的。

⑭CLI対応

WSLのClaude CodeやCursorで利用可能。CLIベースで操作できるため、柔軟な使い方ができる。

⑮MCPの追加

これを追加した上で他のMCPもどんどん追加していく予定。定常的なアクションはMCPに追加していく。

⑯Bedrock連携

BedrockエージェントチームでMCPで使えるランキングを作っているので、NotebookLM MCPを追加していく。

⑰Cursorへの追加

CursorにMCPを追加して使うことも可能。設定に追加するだけで簡単に使えるようになる。

⑱まとめ

NotebookLM MCP導入でリサーチとコンテンツ作成が自動化。導入には少し苦戦したが、一度導入すると非常に便利。
