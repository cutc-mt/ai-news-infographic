Warp Oz 解説

OpenClawの後継・最強のクラウドAIコーディングエージェント

出典: WorldofAI - YouTube

①問題の解決: ローカルエージェントの限界を超える

  • 3〜4つのエージェントをローカルで実行するとマシンが遅くなる問題を解決
  • 10個のタブを開いてコンテキストスイッチングする煩わしさを排除
  • エージェントはクラウドで実行され、マシンを遅くしない

②無制限のエージェント: 好きな数だけ実行可能

  • 好きな数だけエージェントを実行できるスケーラビリティ
  • 端末だけでなくSlack、Linear、GitHub、cronジョブ、アプリケーションコードからトリガー可能
  • ほとんどのリポジトリで10分以内のセットアップ

③3つの導入方法: リスニング・スケジュール・Webインターフェース・ターミナル

  • リスニング・スケジュールトリガー：Slackで@Oz、Linearチケット作成、GitHub_issue、Sentry webhookで自動起動
  • Webインターフェース：oz.warp.devですべてのエージェントを管理し進捗を確認
  • Warpターミナル：oz agent runコマンドでローカルまたはクラウド実行

④環境設定: Gitリポジトリをスキャンして自動セットアップ

  • oz env setupコマンドでGitリポジトリにアクセス権を付与
  • コードベースをスキャンし必要な環境を自動検出
  • GolangとJavaScriptのプロジェクトならGoとNodeがプリインストールされたマルチステージDockerを作成

⑤統合機能: Slack・Linear・TypeScript・Python SDK

  • Slack・Linear：チームの誰でもがターミナルを使わずにコーディングエージェントを起動可能
  • GitHub Actions：issue、PR、何でもトリガーとしてビルド済み
  • TypeScript・Python SDK：独自のアプリケーションコードからOzを呼び出し可能

⑥セッション共有: チームで協力・リモートから参加

  • 単一のリンクでどのエージェント会話にも参加・観察・操作・協力可能
  • チームメイトにリンクを共有し、エージェントが何をしているかを正確に確認できる
  • クロスプラットフォーム対応：Mac、Linux、Windows、Webで動作

⑦設定可能エージェント: Skills.mdでカスタムワークフロー

  • Skills.mdファイルでエージェントを設定・他のAIコーディングツールと同様
  • 特定のタスクにカスタマイズ・プログラムしたエージェントを作成可能
  • 特定のワークフローやベストプラクティスを設定に組み込み可能

⑧使用例: Obsidian自動更新・サーバーモニター・Sentryアラート

  • Obsidian自動更新：GitHubリポジトリに置いたノートブックを週次でタグ付け・リサーチ・新規ノート追加
  • カスタムサーバーモニター：SSHでログをtailし、例外や使用レポートをSlackに転送
  • GitHub_issueトリアージ：受信issueを調査し、作業準備完了か追加詳細が必要かをチェックし自動タグ付け

⑨フルスタック対応: 複数リポジトリで単一の環境

  • マルチリポジトリ環境：フロントエンドとバックエンドのリポジトリを含む環境を構成可能
  • 両方のリポジトリにまたがるタスクに対してエージェントをトリガー
  • フルスタックプロジェクトでシームレスな開発を実現

⑩Warpの実績: 70万人以上の開発者・97%のコード差分が受理

  • 70万人以上の開発者がWarpを使用してソフトウェアを出荷
  • Warpが生成したコード差分の97%がエンドユーザーに受理される
  • ユーザーは1日平均1時間を節約、25%は1日2時間以上節約

⑪技術的背景: 100万行のRustコード・WarpでWarpを構築

  • 速度のために構築されたツール。100万行以上のRustコード
  • WarpエンジニアがWarpを使ってWarpを構築（良い兆候）
  • プロフェッショナル開発者向け：Cloud Code、Cursor、Windsurfと同じ複雑さのタスクに適用

⑫AIの未来: ヘルパーからチームへ・オーケストレーションが重要に

  • 手動で呼び出すAIヘルパーから、バックグラウンドで働くAIチームへ移行
  • チームには適切なオーケストレーションが必要
  • アンビエントトリガー・クラウド実行・セッション共有・エージェント管理パネルの組み合わせが画期的

⑬チームにとっての価値: 協調システム・重複作業の削除

  • 各自がローカルエージェントを実行して重複作業する代わりに協調システムを構築
  • ワークフローの実際のイベントによってエージェントがトリガーされる
  • チームにとってゲームチェンジャーとなる可能性

⑭主なコマンド: oz agent run・oz env setup・oz agent run-cloud

  • oz agent run "prompt"：プロンプトを渡してエージェント応答をストリーミング（ローカル実行）
  • oz env setup：Gitリポジトリへのアクセスを設定、環境を自動スキャン・Dockerfile作成
  • oz agent run-cloud "prompt" --env-id：クラウドでタスクを実行、デスクトップを閉じても完了

⑭セッション管理: SSH付きWarpセッション・リアルタイムストリーミング

  • クリックするとSSH付きのWarpセッションが読み込まれ、ローカル端末で会話しているような感覚
  • レスポンスがストリーミングで届く、または作業中にプロンプトを送ってエージェントを修正
  • 実行内容を共有し、他の人も確認できる

⑯Webインターフェース: oz.warp.dev・エージェント管理パネル

  • すべてのエージェントを一元管理
  • 進捗の追跡・セッションへの参加
  • ブラウザからクラウドエージェントを管理

⑰はじめに: Ozクレディット1000ボーナス・プロフェッショナル開発者向け

  • Warp Ozを試すと1000 Ozクレジットボーナスを追加で獲得可能
  • プロフェッショナル開発者向け：Flappy Birdクローンのような単純なアプリには過剰
  • Cloud Code、Cursor、Windsurfと同じ複雑さのタスクに適用
