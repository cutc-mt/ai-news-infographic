Claude Code実践入門

仕様詰め・コンテキスト管理・ステータスライン・CLAUDE.md設計

2026-03-07

①料金プラン

月額$20のPro〜$200のMaxプラン。毎日使うならMax $100が仕事でも十分使える。

②4つの利用方法

Web版・デスクトップアプリ・VS Code拡張機能・CLI。用途に合わせて選択。

③VS Code拡張がおすすめ

並列実行しやすく、Git操作やファイル確認が直感的。初心者はここから。

④CLIの利点

tmuxと組み合わせて並列実行が効率的。慣れたらCLIが使いやすい。

⑤仕様を詰めるのが重要

いきなり実装すると失敗しがち。事前に仕様を文章化すると品質が上がる。

⑥プランモード活用

Shift+Tab2回でプランモード起動。計画を立ててから実装。

⑦Ask User Questionツール

「曖昧な仕様を詰めて」と入力でClaude Codeが質問してくれる。

⑧並列レビュー

計画ファイルを別のClaude Codeでレビュー。Codexがレビュー得意。

⑨SPEC.mdとTASKS.md

公式推奨の管理方法。複雑な開発はこれらで進捗管理。

⑩計画はOpusで実装はSonnet

/modelでOpus Plannerを設定。計画は高精度で実装は経済的。

⑪コンテキストエンジニアリング

入力が長くなると精度が落ちる。コンテキストを意識して管理。

⑫ステータスライン設定

コンテキスト使用量を常に表示。7〜8万超えたら/compactで圧縮。

⑬/compactで要約

コンテキストが増えたら/compactで要約圧縮。精度を維持しながら継続。

⑭Playwright CLI推奨

Playwright MCPはトークン消費大。Playwright CLI Skillsの方が軽量。

⑮CLAUDE.mdの書き方

/initで自動生成するより、プロジェクト特有のルールだけ書く方が良いかも。

⑯/initは非推奨説

最新研究では何もない方が精度が高いという結果も。汎用ルールは不要かも。

⑰エージェントチームズ

複数モデルがコミュニケーションしながら処理。トークン消費は激しい。

⑱まとめ

仕様詰め・コンテキスト管理・ステータスライン表示が実践のポイント。
