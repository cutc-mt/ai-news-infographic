OpenClaw 7つのプロンプト

QMD Memory・Accountability・自己改善・コンテキスト最適化

2026-03-07

①QMD Memory

デフォルトのキーワード検索から意味検索（Semantic Search）へ進化。「疲れた」と「眠い」を関連付け。

②ローカルRAGパイプライン

ベクトル検索・意味検索・リランキングを搭載。正確な記憶呼び出しが可能。

③スペック要件

CPU only VPSでは動作が重い。Raspberry Pi・Mac Miniなら快適動作。

④Daily Morning Brief

Tavily Searchで毎朝のニュース概要を自動取得。Brave APIは有料化。

⑤YouTube Transcript活用

動画リンクからトランスクリプトを取得してプロジェクトに統合可能。

⑥Accountability Partner

「3ヶ月で10kg減量」などの目標を設定し毎日チェックイン・毎週レビュー。

⑦SQLデータベース活用

MDファイルではなくSQLデータベースに保存。コンテキスト肥大化を防止。

⑧ダッシュボード作成

進捗をHTMLダッシュボードで可視化。スマホから確認可能。

⑨自己改善メモリシステム

3回以上のパターンを学習して自動で指示を更新。1-2回は適用しない。

⑩Hot Memory

上位10個のルールをHot Memoryとして常に適用。

⑪安全プロトコル

プライベートな情報は保存しない。毎週メンテナンスで古いパターンを削除。

⑫OpenClaw Update Monitor

更新があれば24時間待機してバグ報告を確認。問題なければ自動更新。

⑬ヘルス＆セキュリティチェック

更新後にヘルスチェック・セキュリティスキャンを自動実行。

⑭Context Editor

すべての設定ファイルをレビューしてトークン削減提案。不要なコンテキストを削除。

⑮重複削除

同じ内容を複数箇所で記述している場合に統合提案。

⑯推奨モデル

Sonnet 4.6またはOpus 4.6で実行。複雑なプロンプトも正確に処理。

⑰ClawHubスキルの注意

サードパーティスキルはセキュリティリスクの可能性。自作プロンプトで代用推奨。

⑱まとめ

7つのプロンプトでOpenClawが実用的に。安く・賢く・使いやすく。
