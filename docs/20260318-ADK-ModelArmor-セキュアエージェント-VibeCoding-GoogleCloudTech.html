①今回のゴール

Model Armor Callbacksを使ったセキュリティチェックポイントの実装。

  • ユーザー入力の検査
  • モデル出力のバリデーション
  • ADKコールバックメカニズムの活用
Antigravityでテスト駆動のVibe Coding！
2

②ADK Callbackとは

エージェントの実行を監視・カスタマイズ・制御するフック。

  • 実行の観察
  • 動作のカスタマイズ
  • フローの制御
モデル呼び出しの前後に介入できる！
3

③Plan-Red-Green-Refactor

Antigravityのスコープを制約して進める。

  • Plan 設計フローの詳細化
  • Red 失敗するテストを書く
  • Green テストを通す実装
  • Refactor 整理してコンテキスト更新

④Step 1: 入力ガードレール

プロンプトインジェクション対策をModel Armorで実装。

  • before_model_callbackを使用
  • 悪意あるコンテンツの注入を防止
  • ポリシー違反時はリクエストをブロック
ユーザー入力がモデルに届く前に検査！

⑤Red Phase: 入力テスト

失敗するテストを先に書く。

  • 敵対的なユーザーをシミュレート
  • 悪意あるプロンプトの注入を試行
  • テストは「ブロックされる」ことを期待

⑥Green Phase: 入力実装

コールバックを実装してテストを通す。

  • Model Armor APIでサニタイズ
  • ポリシー違反時は特定のレスポンス返却
  • LLM呼び出しをスキップして安全な応答へ
デフォルト実行をオーバーライド！

⑦Step 2: 出力バリデーション

データリーク防止のための出力ガードレール。

  • after_model_callbackを使用
  • 好ましくないコンテンツの生成を防止
  • 機密情報のサニタイズ

⑧Red Phase: 出力テスト

モデル出力に機密情報が含まれるケースを想定。

  • 機密データを含む出力をモック
  • フィルタリング/編集されることを期待
  • 一貫したテストのためにモックを使用

⑨Green Phase: 出力実装

出力サニタイゼーション機能を実装。

  • 生のレスポンスを検査
  • 修正が必要な場合は新レスポンス返却
  • 元の出力を置き換え
モデルが生成した情報も検査・編集可能！

⑩Refactor Phase

セキュリティポリシーを統合。

  • Model Armorコールバック設定を集約
  • ADKエージェント定義を完成
  • コンテキストファイルを更新

⑪セキュリティチェックポイント

2つの重要な介入ポイント。

  • before_model: ユーザー入力 → モデル
  • after_model: モデル出力 → ユーザー
モデルとのやり取りの両端をガード！

⑫小さなバッチで進めるメリット

テスト駆動原則で安定したコードを書く。

  • 安定したコードの記述
  • 継続的な人間の検証
  • 各小さな増分を「良い状態」に保つ
