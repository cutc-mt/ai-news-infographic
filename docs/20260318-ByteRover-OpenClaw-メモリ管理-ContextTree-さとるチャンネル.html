ByteRover: OpenClawメモリ肥大化の解決策

Context Tree構造でスマートなメモリ管理

2026年3月18日 | さとるチャンネル@毎日AIニュース

①

①問題: MEMORY.mdの肥大化

  • MEMORY.mdが5000行以上に
  • セッション開始前に全ファイルを読み込む
  • コンテキストウィンドウの30-40%が消費
  • トークン課金の4割近くが無駄に
作業に使えるのは残り60%のみという状態
②

②原因: 構造化されていない

  • ただのマークダウンファイル
  • 情報がフラットに並ぶだけ
  • 同じ情報が何箇所にも散在
  • どれが最新か分からない
エージェントも混乱してしまう
③

③悪循環のメカニズム

  • エージェントが学んだことを書き込む
  • 同じ内容を何度も書き込み
  • ファイルが肥大化
  • コンテキストを圧迫→また書き込み
悪循環: 書込 → 肥大化 → 圧迫 → 書込...
④

④Compaction機能の限界

  • 残りトークン2万4000以下で起動
  • そこまで行く前に作業終わることも
  • Compaction自体にもトークン消費
  • 根本的解決になっていない
一時的な対処に過ぎない
⑤

⑤ByteRover: 解決策

  • NPMで2万6000以上インストール
  • 検索精度92.19%を達成
  • 必要な情報だけを検索で取得
  • Context Treeで階層管理
全て読む→必要だけ読むへ転換
⑥

⑥Context Treeの3層構造

  • Domain: フォルダー層 (アーキテクチャ/API/フロントエンド...)
  • Topic: サブフォルダー層
  • Context: ファイル層 (JWT-setup.md, Auth-flow.md...)
構造: Domain > Topic > Context File 各ファイルが独立→必要な情報だけ取得
⑦

⑦セマンティック検索

  • 単純キーワードではなく意味で検索
  • 「認証」で検索→login/JWTなども
  • レスポンス前に自動で検索実行
  • 関連情報だけがコンテキストに注入
5000行全部→必要な100行だけに
⑧

⑧自動キュレーション

  • 意味のある作業後に自動で保存
  • 適切なTopicに自動追加
  • 重複チェックが走る
  • 同じ情報を何度も書き込まない
OpenClawと違い膨らまない
⑨

⑨インストール3ステップ

  • npm install -g clawhub
  • npm install byterover
  • brv setup
たった3コマンドで完了！ ⚠️ ゲートウェイ再起動を忘れずに
⑩

⑩CLIコマンド

  • brv query "ユーザーについて"
  • brv curate 手動キュレーション
  • brv status 状態確認
自然文で検索可能
