CLI-Anything - Claude Codeを最強にする魔法のプラグイン

あらゆるソフトウェアをAIが直接操作可能に

2026年3月18日 | Generative AI and InfoBusiness Institute

1

①CLI-Anythingとは

香港大学HKUDSラボが開発した画期的なプロジェクト。

  • 公開からわずか4日で1万スター超え
  • 香港大学初のGitHubリポジトリー
  • ソフトウェアをAIネイティブに変換
現在11.5万スター以上！開発者コミュニティで大注目！
2

②何ができる？

あらゆるソフトウェアをCLI化してAIエージェントが操作可能に。

  • ソースコードを解析してCLI自動生成
  • AIエージェントが直接操作可能に
  • GUIソフトをCLIで制御
人間向けのGUIとAIエージェントの間の溝を埋める！
3

③コックピットの例え

AIは計器が見えないパイロットのような状態だった。

  • GUIは人間のために設計されている
  • AIには「手」がなかった
  • 窓の外だけ見て操縦する状態
AIがローカルのExcelを開いて！と言っても動けなかった！

④既存アプローチの限界

3つのアプローチすべてに問題があった。

  • 画面操作: 遅く、UI変更で壊れやすい
  • MCP: 誰かが構築する必要、コンテキスト圧迫
  • 直接編集: ファイルが壊れる可能性
見事な八方塞がりだった！

⑤MCP vs CLI-Anything

トークン消費を約1/30に削減。

  • MCP: 全スキーマを毎回コンテキストに読み込み
  • CLI: 必要な時だけヘルプを参照
  • コスト大幅削減
MCPは分厚いマニュアルを毎回読ませるようなもの！

⑥厨房の例え

お客さん用と厨房用の違い。

  • お客さん: 写真付き豪華メニュー表（GUI）
  • 厨房: プレーンテキストの注文チケット（CLI）
  • CLIこそAIにとって最もネイティブ

⑦7段階の自動化パイプライン

ソースコードからCLIを自動構築。

  • 1 分析: ソースコードをスキャン
  • 2 設計: コマンド構造を設計
  • 3 実装: Pythonパッケージ構築
  • 4 テスト計画: テスト計画作成
  • 5 テスト作成: テスト実装と実行
  • 6 文書化: ドキュメント作成
  • 7 公開: インストール実行

⑧内部APIの特定

AST解析で内部構造を把握。

  • GUIのボタンがどの関数を呼ぶか特定
  • ソフトウェアの内部サーフェスを解析
  • 本物のバックエンドを直接叩く
薄いラッパーではなく、本物のエンジンと直結！

⑨テスト駆動の自動化

テストコードも自動生成・実行。

  • ユニットテスト・E2Eテスト
  • 実際の出力ファイル（PDF/PNG）を検証
  • 1508個のテストで100%パス率

⑩対応ソフトウェア（11種）

複雑なアプリケーションでも動作。

  • Blender - 3Dモデリング
  • GIMP - 画像編集
  • LibreOffice - オフィス
  • OBS Studio - ライブ配信
  • Draw.io - 図解作成

⑪主要コマンド

4つのコマンドで操作。

  • cli-anything [path] - CLI構築開始
  • refine - 既存CLIを改良
  • list - 構築済みCLI一覧
  • test - テスト実行

⑫Refineコマンド

機能を見落としても大丈夫。

  • 既存のCLIと元ソースの差分を分析
  • 段階的に機能追加可能
  • システムを破壊せずに改善
1発で全機能網羅できなくてもOK！

⑬対応プラットフォーム

主要なAIコーディングツールに対応。

  • Claude Code - プラグインとして統合
  • OpenCode - 実験的コマンドセット
  • Codex - コミュニティスキル
  • Kilo CLI - 対応

⑭必要なモデル性能

Claude Sonnet 3.5以上の強力なモデルが必要。

  • ソース解析に高い推論能力が必要
  • 大規模ソフトはOpus 4.6推奨
  • 広いコンテキストウィンドウも重要
ローカルモデルでは精度が落ちる可能性！

⑮制限事項

注意点がある。

  • コンパイル済みバイナリは品質低下
  • 対象ソフトがインストールされている必要
  • UIとロジックが分離できないと不可

⑯MCPとの住み分け

公式MCPがあるならそれを使う。

  • Slack/GitHub等は公式MCPを優先
  • CLI-Anythingはロングテール向け
  • 自社独自システムの自動化に最適
MCPがない99%のソフトウェアに対するマスターキー！

⑰エージェントネイティブな未来

ソフトウェアのあり方を再定義。

  • 人間中心からAI・人間共存へ
  • あらゆるOSSが即座にエージェント操作可能に
  • 将来、人間向けGUIを作る必要はある？
歴史的な転換点を目撃しているかも！

⑱まとめ

プログラミング知識がなくてもソフトをAIに操作させられる。

  • ソースコード解析でCLI自動生成
  • トークン消費を1/30に削減
  • 社内システムやマイナーツールに最適
あなたの使っているソフトの裏側に眠るエージェントの可能性！
