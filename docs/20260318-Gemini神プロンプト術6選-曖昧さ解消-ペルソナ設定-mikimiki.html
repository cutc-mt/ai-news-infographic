プロンプト術 初級編（3選）

③初級1: 見本を1つ見せましょう

  • フォーマットや好みの文章を言葉で説明するのは意外と難しい
  • 理想的な過去の成果物を1つ見せる方が圧倒的に正確で早い
  • ステップ1: 理想の資料をアップロード
  • ステップ2: 「この形式に従って」と指示
ステップ1: 資料をアップロード
「この資料のフォーマットを分析して
特徴を整理して」

ステップ2:
「今後はこの形式に従って
アウトプットを出して」
見本を見せるとGeminiが「こう出せばいいのか」と理解できる！

④初級2: 曖昧さを数値と条件に変換

  • 「分かりやすく」「丁寧に」「短めに」は主観的すぎる
  • AIには具体的な物差しを渡す必要がある
  • 数値と条件をプラスすると精度が劇的に変わる
悪い例: 「いい感じにまとめて」
良い例: 「結論を冒頭に、各段落に見出しをつけて200文字以内で」

⑤初級2の実践例: 具体的条件指定

リモートワークのメリットと
デメリットを以下の条件で整理して:

- 項目数: メリット3つ、デメリット3つ
- 各項目: 50文字以内の要約 +
           100文字以内の補足説明
- 専門用語は使わない
- 読者: 入社2年目の一般社員
- 形式: 表形式
- 結論: どんな企業に向いているか
        1分で入れて
表形式で出力すればGoogleスプレッドシートにエクスポート可能！

⑥初級3: 誰目線のアウトプットか定義

  • ペルソナを設定すると口調・トン・専門性・視点が向上
  • 役割名だけではなく特徴や経験まで踏み込む
  • 読み手も同時に設定すると説明の詳細度が自動最適化
悪い例:
「あなたは人事担当者です」

良い例:
「あなたは大手メーカーの人事開発部に
15年在籍し、年50回以上の社内研修を
設計・登壇してきたベテラン講師です。
説明は具体的なエピソードを交える
スタイルで、親しみやすいトーンが
特徴です。」

⑦初級3の実践例: 新入社員研修

  • 役割設定なし → 一般的な内容のみ
  • 役割設定あり → 具体的なエピソード付きで実践的な内容
  • 対象読者の不安レベルまで指定可能
マナーと聞くとなんだか肩を固くされる...
失敗しても大丈夫。一番のマナー違反は失敗を恐れて不格好になることです。

プロンプト術 中級編（2選）

⑧中級1: タグでAIに教える

  • 長文の資料と指示が混在するとAIが見失う
  • XMLタグやマークダウン見出しで境界線を引く
  • 解釈の精度が跳ね上がる
  • データ資料を先に、指示を最後に
<role>
あなたはテクノロジー業界に精通する
経営コンサルタントです。
</role>

<reference>
[参考資料の内容]
</reference>

<constraints>
- 専門用語には括弧で補足をつける
- 各回答は150文字以内
</constraints>

<task>
上記の参考資料をもとに戦略的提言を
3つ上げてください。
</task>

⑨中級1の効果: タグ活用のメリット

  • どこまでが何の内容か明確に区別できる
  • プロンプトが長くなっても整理された状態を維持
  • 参考資料と指示の境界がはっきり
タグを使うだけで、複雑なプロンプトでも正確に伝わる！

活用場面 長文資料の分析 複雑な指示

⑩中級2: 骨組みから本文の2段階進行

  • 一発で完成を求めると重要ポイントが漏れ落ちる
  • 先に骨組み（アウトライン）を出してもらう
  • 合意してから本文の執筆に移る
レポート: 構成 → 本文
プレゼン: 目次 → スライド → 原稿
難しいタスクは区切るのがコツ！

⑪中級2の実践例: ステップ1（構成）

あなたは中小企業向けのDXコンサルタントです。

従業員50人規模の製造業が初めてDXに
取り組む際のロードマップというテーマで
レポートを作成したいです。

まずはアウトラインだけ作成してください:
- セクション: 5〜7個
- 各セクション: タイトル + 要点2〜3個
- 読者: 製造業の経営者

本文はまだ書かないでください。
構成案だけお願いします。
構成を見て気になるところがあれば壁打ちで修正！

⑫中級2の実践例: ステップ2（本文）

ありがとうございます。
この構成案で進めてください。

各セクションの本文を執筆してください:
- セクションの文字数: 500文字程度
- 抜け漏れなく網羅的に書いて
- 専門用語は必ず括弧で補足説明を入れて
- 後で人間が編集・削除する前提で
ステップで進めることでイメージに近い形で最終的なアウトプットを入手！
プロンプト術 上級編（1選）

⑬上級: AIから質問させる（盲点を炙り出す）
