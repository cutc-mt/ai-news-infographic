1

①Superpowersとは

GitHubで9万スターを超える大人気のAIコーディングエージェント向けスキル集。

  • 開発の進め方・フレームワークを提供
  • Claude Code、Codex、OpenCode対応
  • OSS界の歴戦のベテランが開発
プラグインを入れるだけで本格的な開発ワークフローが手に入る！
2

②7つの開発フロー

Superpowersに組み込まれた本質的な7つの流れ。

ブレスト 仕様レビュー Git Worktree
  • ブレインストーミングで仕様詰め
  • 仕様のレビューと確認
  • Git Worktreeで環境分離
3

③Git Worktree活用

複数環境で並列開発を可能にする仕組み。

  • メインブランチと同じ環境をコピー
  • AIエージェントが独立した環境で作業
  • 複数ブランチの動作確認が可能
人間が介入する短いタスクなら不要だが、長期開発には必須！
4

④計画とタスク分割

タスクを適切なサイズに区切って効率的に進める。

  • タスクを5分割程度に分割
  • 1タスク2〜5分で完了するサイズ
  • 最後にまとめフェーズで統合
上から順番にやって、最後に綺麗に統合していく！
5

⑤テスト駆動開発（TDD）

Red → Green → Refactorのサイクルで品質を担保。

  • Red: テストを先に書く（失敗状態）
  • Green: テストを通す実装
  • Refactor: コードを整理
コンテキストエンジニアリング的にも非常に強力なアプローチ！
6

⑥サブエージェント活用

複数のAIエージェントで並列作業を効率化。

  • タスクを区切って並列実行
  • 開発効率が大幅アップ
  • 各エージェントが独立して動作
サブエージェント駆動開発で効率化！
7

⑦2段階コードレビュー

実装後の2段階チェックで品質を確保。

  • 仕様に準拠しているか確認
  • コード品質のチェック
  • 必要に応じてループで改善
TDDだけでは追いつけない部分もカバー！

⑧インストール方法

Claude Codeのマーケットプレイスから簡単導入。

  • /plugins install superpowers
  • プロジェクトレベルでインストール可能
  • ユーザーレベルでも設定可能
Codex、OpenCodeを使っている場合も選択肢としてあり！

⑨ブレインストーミング

AIと対話しながら仕様を詰める最初のステップ。

  • 何が作りたいか要件定義
  • 思いつきで実装させない
  • 機能追加時にも活用可能
ざっくりとしたまま進めると「これ欲しくなかった」に！

⑩仕様レビュー

仕様書が本当に成り立つかAIに確認してもらう。

  • 問題がないかチェック
  • 成り立っていれば次のステップへ
  • 品質の良い仕様書を作成
AIエージェントが客観的にレビューしてくれる！

⑪主要スキル一覧

Superpowersに含まれる主なスキルを紹介。

  • /brainstorming - ブレインストーミング
  • /write-plan - 計画作成
  • /execute-plan - 計画実行
  • /subagent-driven-development
マルチプラットフォーム対応でコミュニティの評判も良い！

⑫ハーネスとの違い

自作ハーネスとSuperpowersの使い分け。

  • ハーネス作成の時間がない方におすすめ
  • 開発初心者にも最適
  • Superpowersを理解してから自作へ
環境が汚れないのもプラグインのメリット！

⑬TDDの重要性

何をやるべきかを明示できるのがTDDの強み。

  • 曖昧さがなくなる
  • テスト＝仕様書として機能
  • やりたいことを明確に表現
AIエージェントの方向性として最適なアプローチ！

⑭最終統合

ベースブランチへの統合で開発完了。

  • プルリクエストの作成
  • コードレビュー後の統合
  • 一連の流れが完了
7つの流れを守るだけで高品質な開発が可能！
