⑤エージェントマネージャー

  • 複数のアイデアを並行して管理

  • コンテキストを失わない

⑥design.md: デザインルールのMD化

  • エージェントフレンドリーなデザインルール
  • 他のデザイン・コーディングツールに移行可能
  • URLからデザインシステムを抽出
  • 一貫性と再利用性を担保
# design.mdの活用
- 既存サイトのビジュアルアイデンティティを抽出
- 毎回デザイン言語を説明する必要がなくなる
- 深いコンテキストとして使用可能

⑩Stitch → Claude Code ワークフロー

  • StitchでUI方向を決定
  • エクスポートしたアセットとdesign.mdを渡す
  • Claude CodeがReact/Next.js/Tailwindで実装
  • ルック＆フィールを維持
# ワークフロー
1. Stitchでフローを生成
2. ビジュアル方向を調整
3. design.mdをClaude Codeに渡す
4. 実際のアプリを構築

⑪Kilo CLIとの連携

  • ターミナルから画面ごとにフロントエンド構築
  • コンポーネントを改良
ターミナルから実装を反復！
