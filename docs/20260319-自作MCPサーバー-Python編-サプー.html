MCPサーバーとは

①MCPサーバーの概要

  • AIと外部ツールを繋ぐプログラム
  • 外部ツール: LINE、Google Drive、Git、メモ帳など
  • Claude、ChatGPT、GeminiなどのAIと連携
  • 多くの場合はローカルPC上で動作
対話の中で外部ツールを操作できる！

②MCPサーバーの活用例

  • 「今年度あと何日有給取れるの？」→ AIが勤怠管理システムを確認
  • 「在庫が少ない商品を教えて」→ AIが在庫管理システムを確認
  • チャットの中で社内データにアクセス
# 従来の方法
ログイン → 画面操作 → 確認
# MCPを使う方法
「有給あと何日？」→ AIが自動確認

③既存のMCPサーバー

  • PostgreSQL、GitHub、Google Driveなど
  • 多くのサービスが既にMCPサーバーを公開
  • 自分で作らなくても使えるものは多い
  • しかし自社システムには自作が必要

既存サーバー活用 自作も可能

なぜ自作するのか

④自作が必要なケース

  • 自社システムとAIを連携させたい
  • 社内データがAIから見えない問題
  • 独自のサービス・ツールをAI連携させたい
  • 業務効率化のためにAI活用したい
社内データをAI活用できていないケースが多い！

⑤MCPサーバーの役割

  • AIはユーザーの指示を理解して適切なツールを選択
  • MCPサーバーはAIと外部ツールの橋渡し
  • 外部ツールは実際のデータ操作
  • この3つが連携して動作

⑥なぜWebAPIだけじゃダメ？

  • Claude Codeなどは直接APIを叩ける
  • しかしClaude Desktopでは直接実行できない
  • APIが複雑・ドキュメント不備の場合も
  • MCPサーバーで抽象化できる
# MCPのメリット
- 非エンジニアでも使える
- 複雑なAPIを隠蔽
- Claude Desktopでも利用可能
実装手順

⑦手順1: 外部ツール（WebAPI）の準備

  • 在庫管理システムのWebAPIサーバーを作成
  • FastAPI + SQLiteで実装
  • 2つのエンドポイント: 商品検索、在庫更新
  • uvicornでサーバーを起動
プロジェクトフォルダー作成
↓
SQLiteでデータベース作成
↓
FastAPIでWebAPI実装
↓
uvicornでサーバー起動

⑧手順2: 環境構築

  • uv initでプロジェクト初期化
  • mcp[cli]をインストール（Python SDK）
  • httpxをインストール（HTTP通信）
  • Pipを使う場合: pip install mcp[cli] httpx
# uvを使う場合
uv init
uv add "mcp[cli]"
uv add httpx

# pipを使う場合
pip install "mcp[cli]" httpx

⑨手順3: MCPサーバーのコード実装

  • FastMCPクラスでインスタンス作成
  • 関数に@mcp.tool()デコレーターをつける
  • 関数名・引数名・docstringがAIに公開される
  • 関数内でWebAPIを呼び出す
関数名とdocstringが非常に重要！

⑩ツール定義の例

from mcp.server.fastmcp import FastMCP
import httpx

API_BASE = "http://localhost:8000"
mcp = FastMCP("在庫管理MCPサーバー")

@mcp.tool()
def search_products(
    category: str,
    max_stock: int,
    min_stock: int
) -> str:
    """在庫管理システムの商品を検索する
    カテゴリーと在庫数の上限下限で絞り込める"""
    params = {
        "category": category,
        "stock_le": max_stock,
        "stock_ge": min_stock
    }
    response = httpx.get(
        f"{API_BASE}/products/search",
        params=params
    )
    return response.text

⑪MCPサーバーの起動

  • mcp.run()で起動
  • transport="stdio"で標準入出力を使用
  • 最もシンプルな通信方式
  • SSEという方式もある
if __name__ == "__main__":
    mcp.run(transport="stdio")

⑫手順4: テスト（インスペクター）

  • mcp dev server.pyでテスト起動
  • MCP Inspectorでツール一覧を確認
  • 実際にツールを実行して動作確認
  • 引数を入力して結果を確認
ブラウザ上でテストできる！
Claude Desktop連携

⑬手順5: Claude Desktop設定

  • Claude Desktopをインストール
  • 設定 → 開発者 → 設定を編集
  • claude_desktop_config.jsonを編集
  • mcpServersに設定を追加
{
  "mcpServers": {
    "inventory": {
      "command": "uv",
      "args": [
        "--directory",
        "/path/to/MyMCPServer",
        "run",
        "server.py"
      ]
    }
  }
}
