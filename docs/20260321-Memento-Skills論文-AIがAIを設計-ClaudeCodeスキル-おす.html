AIがAIを設計する - パラメータ更新なしで継続学習

Memento-Skills Claude Code スキル おす

①衝撃的な論文: AIがAIを設計

2025年3月に発表されたMemento-Skills論文。AIが自分用のAIを作る、タスクに特化したスキルを自分で設計して保存する仕組みを提案。

②パラメータは更新しない

ファインチューニングとは異なり、パラメータは一切いじらない。代わりにマークダウンファイルにスキルを書き出して外部に保存。

③Claude CodeのスキルMDと同じ発想

Claude Codeのskill.mdと本質的に同じ。クロードコードユーザーは既にこの考え方を実践している。論文がそれを理論化した形。

④スキルベースのメモリ

タスクを実行した後に「こうやったらうまくいった」を自分でまとめてファイルに残す。次に似たタスクが来たらそのスキルを参照。

⑤リードライト反射学習

Readフェーズで適切なスキルを読み込み、Writeフェーズで経験を元にスキルを書き込む。このサイクルを回す。

⑥スキルルータの役割

新しいタスクが来るとスキルルータが動く。タスクを見て「このタスクにはこのスキルが適している」と振り分ける。

⑦スキルのメタ情報

スキルにはタグ付けがされている。「情報収集向き」「精度は高いが時間がかかる」など。選ばれる根拠になる。

⑧ライトフェーズの3つのパターン

①既存スキルの更新（バージョンアップ）②新規スキルの作成 ③何もしない（十分なスキルがあれば）

⑨エレメンタリースキルから複合スキルへ

最初は基本スキルのみ。タスクをこなすたびに複合スキルが生まれる。基本を組み合わせて応用スキルへ。

⑩実験結果: GAIAで26.2%向上

AIエージェントの総合能力を測るGAIAベンチマークで26.2%の相対的精度向上。体感できるレベルの改善。

⑪HLEで106.2%向上

「人類最後の試験」HLEベンチマークで106.2%の向上。ベースラインが解けなかった問題の半分以上が解けるように。

⑫なぜ専門問題に強いのか

専門的な問題には解法のパターンがある。物理なら図を描いて方程式を立てるなど。そのパターンをスキルとして蓄積。

⑬実践1: スキルルートを明確に

スキルMDを作る時に「このスキルはどんなタスクで使うべきか」を明確に書く。タスクとスキルの対応を明示化。

⑭実践2: 振り返りを習慣化

うまくいったワークフローがあったらすぐにスキルとして書き起こす。スキルの蓄積は性能を2倍以上にする可能性がある。

⑮実践3: 階層構造を意識

小さいスキルから始めて組み合わせる。いきなり大きなスキルを作らず、ファイルを読む、検索する、要約するなど小さく積み重ねる。

⑯将来: AIが自動でスキル生成

Memento-Skillsの仕組みはClaude Codeにも導入可能。タスクをこなすたびに自動でスキルMDを生成。使えば使うほど賢くなる。

⑰ファインチューニングより有利

ファインチューニングは数百万〜数千万円かかるが、スキル保存はコスト的にも現実的。知識を追加するたびに再学習不要。

⑱結論: スキル設計のスキルを磨く

「AIがエージェントを設計する」は当面のトレンド。スキルベースの考え方に慣れておくのが未来への準備になる。
