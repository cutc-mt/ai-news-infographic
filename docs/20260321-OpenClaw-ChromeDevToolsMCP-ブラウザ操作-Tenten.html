②Chrome DevTools MCPとは

GoogleのChromeDevToolsチームが公式にメンテナンスするMCPサーバー。PuppeteerをベースにAIエージェントからChromeを操作可能にする。

③統合のポイント

OpenClaw v2026.3.13-beta.1がChrome DevTools MCPの--autoConnectフローをexisting-sessionドライバーとして追加。設定変更1つで接続完了。

④3つのブラウザ制御モード

Managed mode（隔離されたChromium）、Existing Session mode（既存Chrome接続）、Headlessモードの3種類が利用可能。

⑤Managed mode: 最も安全

Cookie、履歴を共有しない完全に隔離された環境。Playwrightをベースにしているが、AIエージェントが使いやすい高レベル抽象化を提供。

⑥Existing Session: 高リスク・高機能

リアルタイムで動作中のChromeに接続。タブ、ログイン状態、Cookieすべて共有。公式ドキュメントは「より高リスク」と明記。

⑦Existing Sessionの設定手順

Chrome 146以降でchrome://inspect/#devicesにアクセスし、「Allow remote debugging」を有効化。127.0.0.1:9222で接続可能に。

⑧パフォーマンスプロファイリング

performance_start_traceツールでCore Web Vitals（LCP、CLS、INP）を分析。AIエージェントが人間と同じPerformanceパネル機能を使用可能。

⑨ネットワークデバッグ

リクエスト分析、CORS問題の検出、コンソールログエラーの読み取り。404やCORSヘッダーの設定ミスを特定可能。

⑩DOMとCSSの検査

ライブページに接続してDOM構造とCSSを検査。レイアウト問題の具体的な改善提案を取得。Elementsパネルで選択した要素を調査。

⑪環境エミュレーション

CPUスロットリング、ネットワーク速度、ビューポートサイズを変更可能。3G接続やモバイル環境でのページ挙動をテスト。

⑫セキュリティ警告: CVE-2026-25253

2026年3月2日に開示されたCVSS 8.8のゼロクリック脆弱性。悪意あるページを開くだけで認証トークンが漏洩。42,665インスタンスが暴露。

⑬プロンプトインジェクションリスク

エージェントが読み取るウェブページに悪意ある指示が隠されている可能性。browser.evaluateEnabled=falseで不要な時は無効化。

⑭Cookie・セッション盗難リスク

Existing Session modeは実際のブラウザセッションを共有。プロンプトインジェクションで悪意あるサイトに誘導されるとGmailや銀行セッションが暴露。

⑮ClawHubの市場汚染問題

2026年2月の監査で約20%のスキルに悪意あるペイロードを発見。認証情報盗難、データ流出、バックドアなど。スキル導入前にソースコード確認が必須。

⑯最低限のセキュリティ対策

Dockerコンテナでサンドボックス化、Gatewayをlocalhostにバインド、専用ブラウザプロファイルを使用、不要なツールは無効化。

⑰専用プロファイルの重要性

Existing Session使用時は日常使いのプロファイルではなく専用プロファイルを作成。問題が起きてもそのプロファイルのみ影響。

⑱結論: 最大のROIと最大のリスク

ブラウザ制御はAIエージェント自動化で最高のROIをもたらすが、同時に最も多くのセキュリティインシデントも発生。慎重な設計が不可欠。
