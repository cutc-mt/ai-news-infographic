① Claude Codeとは: AI駆動のターミナル開発ツール

Claude CodeはAnthropicが提供するコマンドラインAIアシスタント。ターミナル上で動作し、コードの作成・編集・デバッグを自動で行える。自然言語の指示だけで複雑な開発タスクを実行できる点が最大の特徴。

② 基本的な使い方: ターミナルでClaudeと対話

ターミナルで claude コマンドを起動するだけ。そのまま自然言語で指示を出せる。「このファイルのバグを修正して」「テストを書いて」といった日常的な指示で、Claudeがコードを理解して実行してくれる。

③ プロジェクト認識: 自動でコードベースを理解

Claude Codeは起動時にプロジェクトの構造を自動で読み取る。package.jsonやディレクトリ構成、依存関係を分析し、コンテキストを把握した上で回答を生成。CLAUDE.mdにプロジェクト固有の指示を書いておくと精度がさらに向上する。

④ ファイル操作: 自動でファイルの読み書き・編集

Read / Write / Editツールを使って、Claudeが自律的にファイルを操作する。コードの追加だけでなく、既存コードのリファクタリングやバグ修正も可能。git操作もサポートしており、コミットやプッシュまで自動化できる。

⑤ Bash実行: コマンドライン操作を自動化

ターミナルコマンドの実行もClaudeにお任せ。npm install、pytest、docker buildなどのコマンドを、Claudeが文脈に合わせて自動で実行。実行結果を読み取って次のアクションを判断する自律性が備わっている。

⑥ コード検索: GlobとGrepで効率的に探索

ファイル検索にはGlob（パターンマッチ）、コード内容検索にはGrep（正規表現検索）を使用。これらは専用ツールとして最適化されており、大量のファイルから正確に目的のコードを見つけ出せる。

⑦ スキル(Skills)とは: 再利用可能な専門能力

スキルはClaude Codeに特定のタスクを実行する能力を追加する仕組み。例えば「/commit」と打てばコミットを作成、「/review-pr」と打てばPRをレビューしてくれる。開発者頻出タスクをワンコマンド化できる。

⑧ スキルの仕組み: プロンプトが展開されて実行

スキルは本質的に展開されるプロンプト。「/commit」と打つと、そのスキルに設定された詳細なプロンプトがClaudeに送信される。複雑な手順をスキルとして登録しておけば、毎回同じ品質でタスクを実行できる。

⑨ カスタムスキルの作成: 自分専用のスキルを定義

.claude/skills/ ディレクトリにMarkdownファイルを配置するだけで独自スキルを作成可能。例えば「/deploy」でデプロイ手順を自動化したり、「/test-all」で全テストを実行したり。チーム共通のスキルをGit管理すれば標準化もできる。

⑩ フック(Hooks)とは: イベント駆動の自動化

フックは特定のイベントが発生したときに自動でシェルコマンドを実行する仕組み。ツール実行前後やセッション開始時などに処理を差し挟める。設定は settings.json に記述し、柔軟にカスタマイズ可能。

⑪ フックの実用例: 自動チェック・通知・整形

代表的な使い方として以下がある。

  • PreToolUse: ファイル編集前にリンターを自動実行
  • PostToolUse: コマンド実行後に結果を通知
  • Notification: 長時間タスク完了時にSlack通知
  • Stop: セッション終了時にサマリーを保存

⑫ 権限システム: 安全にツールを制御

Claude Codeには権限モードがある。Default、AcceptEdits、Planなどのモードがあり、Claudeの操作範囲を細かく制御可能。機密ファイルの編集や本番環境へのデプロイなど、リスクの高い操作を事前にブロックできる。

⑬ サブエージェント(Sub-agents): 並列で複数タスクを処理

Agentツールを使うと、Claudeがサブエージェントを起動して並列処理ができる。例えば、メインエージェントがコードを書きながら、サブエージェントがテストやドキュメント作成を同時進行。複数の調査タスクも並列で実行でき、作業効率が大幅に向上する。

⑭ サブエージェントの種類: 用途別に最適な選択

サブエージェントには複数の種類がある。

  • general-purpose: 汎用タスク、ファイル検索やコード調査に最適
  • Explore: 高速なコードベース探索に特化
  • Plan: 実装計画の設計に最適
  • worktree: 隔離された作業環境での安全な実行

⑮ MCP(Model Context Protocol): 外部ツールと連携

MCPはAnthropicが提唱するオープン規格で、Claude Codeと外部ツールを接続する標準プロトコル。GitHub、Slack、データベース、ファイルシステムなど、様々な外部サービスをClaudeから直接操作できるようになる。

⑯ MCPサーバー: 機能を拡張するプラグイン

MCPサーバーはClaudeに新しいツールを追加するプラグイン。例えばYouTube文字起こし、Web検索、画像分析など、MCPサーバーを追加するだけでClaudeの能力を拡張できる。claude mcp add コマンドで簡単に追加可能。

⑰ プラグイン生態系: コミュニティ拡張の可能性

MCP規格により、サードパーティのプラグインが急速に増加中。MCP Registryでは公開サーバーを簡単に検索・インストール可能。自分でオリジナルのMCPサーバーを開発して公開することもできる。

⑱ 全体像のまとめ: Claude Codeの拡張アーキテクチャ

Claude Codeの機能は「基本ツール ＋ スキル ＋ フック ＋ サブエージェント ＋ MCP」の5層で構成される。基本のファイル操作やコマンド実行を土台に、スキルで手順を再利用し、フックで自動化し、サブエージェントで並列処理し、MCPで外部連携する。これらを組み合わせることで、Claude Codeは単なるAIアシスタントを超えた強力な開発環境となる。
