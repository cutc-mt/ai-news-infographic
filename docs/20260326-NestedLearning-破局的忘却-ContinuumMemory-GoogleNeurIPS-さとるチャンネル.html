Nested Learning: AIの「学んでは忘れる」を終わらせる

Googleの発想転換 | Continuum Memory System | NeurIPS 2025 | さとるチャンネル@毎日AIニュース

① 破局的忘却（Catastrophic Forgetting）とは

AIが新しいことを学ぶと、以前に学んだ知識を忘れてしまうという深刻な問題。人間のように「積み上げて学ぶ」ことができないのが、現在のAIの最大の弱点の一つ。

② 従来の学習のジレンマ: 学ぶか忘れるか

新しいデータで再学習すると古い知識が上書きされる。知識を追加するたびに、以前の能力が低下してしまう。AI開発における長年の課題だった。

③ 従来のLLMの記憶は二極構造

現在のLLMには記憶が「短期記憶（コンテキストウィンドウ）」と「長期記憶（モデルの重み）」の二つしかない。人間のような段階的な記憶階層が存在しない。

④ コンテキストウィンドウの限界

短期記憶であるコンテキストウィンドウは、会話が終われば内容が消えてしまう。長期的な知識の蓄積には使えず、あくまで一時的な作業領域に過ぎない。

⑤ モデルの重みに刻まれる長期記憶

長期記憶はモデルの重みに保存されるが、一度固定されると部分的な更新が困難。新しい知識を追加するためには、モデル全体を再学習する必要がある。

⑥ 人間の脳との決定的な違い

人間には感覚記憶、短期記憶、長期記憶、作業記憶など、多層的な記憶システムがある。情報は徐々に安定化され、古い記憶を新しい学習で上書きしない。

⑦ Googleの発想転換: Nested Learning

Googleはこの問題に対して全く新しいアプローチを提案。記憶を二極ではなく「連続体」として設計するNested Learningというパラダイムを発表した。

⑧ Continuum Memory System（CMS）の導入

新たにCMS（Continuum Memory System）」という記憶アーキテクチャを導入。さまざまな更新頻度を持つ記憶モジュールの「連続体」として機能する。

⑨ 連続的な更新頻度の階層

CMSでは更新頻度の異なる複数の記憶モジュールが存在する。変化の激しい情報は頻繁に更新され、安定した知識はめったに更新されない。情報の性質に応じた最適な更新が行われる。

⑩ 古い知識の保護メカニズム

安定した知識を保持するモジュールは頻繁には更新されないため、新しい学習が古い知識を上書きするリスクを大幅に軽減。破局的忘却を根本から防ぐ仕組み。

⑪ 神経可塑性にヒントを得た設計

人間の脳における神経可塑性（Neuroplasticity）のメカニズムに着想を得ている。脳ではシナプス結合の強さが経験に応じて段階的に変化し、記憶が形成される。

⑫ ネストされた記憶の更新戦略

記憶モジュールは「ネスト（入れ子）」構造で配置される。外側のモジュールは高頻度で更新され、内側ほど安定性が高くなる。人間の記憶の段階的な安定化を模倣した設計。

⑬ 継続学習（Continual Learning）への道

これまでAIは学習を「終わらせる」必要があったが、Nested Learningは止まることなく学び続けるAIの実現に向けた重要な一歩。継続的に成長するAIの可能性を切り開く。

⑭ 実験結果: 性能の向上

CMSを組み込んだモデルは、従来のファインチューニングと比較して既存のタスクの性能を維持したまま新しいタスクにも対応可能であることが実験で示された。

⑮ 安定性と可塑性のバランス

AI研究における長年のトレードオフである「安定性（Stability）」と「可塑性（Plasticity）」のバランスを、CMSの連続的な階層設計によって解決するアプローチを提示している。

⑯ NeurIPS 2025で発表

この研究はAIのトップ会議であるNeurIPS 2025で正式に発表された。Google DeepMindを中心とする研究チームによる論文であり、AI学界で大きな注目を集めている。

⑰ 今後の展望: AIの進化の加速

継続的に学習できるAIが実現すれば、一度学んだ知識を維持しながら常に最新の情報を取り込み続けることが可能に。個人的なAIアシスタントの長期的な成長にも期待がかかる。

⑱ まとめ: パラダイムシフトの始まり

Nested Learningは、AIの記憶のあり方を「二極」から「連続体」へと根本的に転換する。破局的忘却という長年の壁を打ち破り、人間に近い学習能力を持つAIへの道を拓く研究だ。
