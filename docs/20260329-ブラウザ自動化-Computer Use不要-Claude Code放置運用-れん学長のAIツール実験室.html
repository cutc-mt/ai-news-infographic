完全放置でいけた！Computer Use不要のブラウザ操作自動化【Claude Code】

Claude Codeを使って、スクリーンショットを撮る従来のComputer Useとは全く違うアプローチでブラウザを自動操作する方法

れん学長のAIツール実験室

①Computer Useの課題とは

従来のComputer Useは、画面のスクリーンショットをAIが解析して操作する仕組みだ。一見便利そうに見えるが、実はいくつか大きな課題があった。 処理速度が遅い、トークン消費が多い、認識精度にムラがあるなど、実用化にはハードルが高かった。

つまり「目で見て操作する」のと同じやり方だから、コストも時間もかかってしまうのが問題だったんだ

②スクリーンショット方式の弱点

Computer Useの最大の弱点は、毎回スクリーンショットを撮って画像を解析する点だ。 1回の操作につき画像1枚をAIに送る必要がある。これはトークン消費量を爆発的に増やしてしまう。 また、視覚的な情報だけに頼るため、要素の特定に失敗することも多い。

画像1枚あたりのトークンコストが大きく、長時間の自動操作には不向きだった

③新しいアプローチ: DOM直接操作

そこで登場するのがDOM（Document Object Model）を直接操作するアプローチだ。 スクリーンショットを撮るのではなく、HTMLの構造を直接読み取って要素を特定し、操作を実行する。 人間がブラウザの開発者ツールで要素を検査するのと同じ考え方だ。

目で見るのではなく、ソースコードを読んで操作する。これがパラダイムシフトだ！

④Puppeteer + Claude Codeの組み合わせ

実際の実装では、Puppeteerなどのブラウザ自動化ライブラリをClaude Codeから操作する。 Claude CodeはMCP（Model Context Protocol）経由でブラウザを制御できる。 ページのHTMLをテキストとして取得し、Claudeが解析して次のアクションを決定する。

Claude Code → MCP → Puppeteer → ブラウザ、という流れで動く

⑤前提条件と環境構築

必要なものはClaude Code（Maxプラン）とNode.js環境だけだ。 Puppeteerはheadlessモードで動作するため、ディスプレイがなくても実行可能。 また、ChromeまたはChromiumがインストールされている必要がある。

  • Claude Code（Maxプラン）
  • Node.js v18以上
  • Chrome / Chromium
  • MCP対応のブラウザ操作サーバー

⑥MCPサーバーの設定方法

Claude Codeの設定ファイル（.claude/settings.json）にMCPサーバーを追加する。 ブラウザ操作用のMCPサーバーを設定することで、Claude Codeから直接ブラウザを操作できるようになる。 Puppeteer MCPやPlaywright MCPなどが利用可能だ。

設定例 
{
  "mcpServers": {
    "puppeteer": {
      "command": "npx",
      "args": [
        "-y",
        "@anthropic/mcp-puppeteer"
      ]
    }
  }
}

⑦Claude Codeへの指示方法

設定が完了したら、Claude Codeに自然言語で指示するだけだ。 「〇〇のサイトにログインして」「フォームに入力して送信して」など、 人間がブラウザ操作を頼むのと同じ感覚で指示を出せる。

指示の例 
$ claude
> Googleにアクセスして「Claude AI」で検索して、
  最初の検索結果を開いてページのタイトルを教えて
日本語で普通に頼めばOK！Claudeが自動的にHTMLを解析して操作を計画してくれる

⑧自動ログインの実践例

Webサイトへの自動ログインは最も基本的な活用例だ。 Claude CodeにURL、ユーザー名、パスワードを教えれば、自動的にログインページを開き、 フォームに入力してログインを実行してくれる。

入力フィールドのIDやname属性をClaudeが自動的に特定するため、 開発者ツールで要素を調べる手間も不要だ。

パスワードは環境変数などで安全に管理することを推奨

⑨フォーム自動入力の活用

問い合わせフォームや申込フォームの一括入力にも対応している。 複数フィールドがある場合でも、Claudeが各フィールドの役割を理解して適切な値を入力してくれる。 セレクトボックスやラジオボタンなどの操作も可能だ。

「名前は田中、メールはtaro@example.comで入力して」のように自然に指定できる

⑩ページ内データ収集

Webページ上のテキスト情報やデータを自動で抽出することもできる。 価格情報、商品の仕様、ニュース記事の本文など、 特定の要素を指定して収集が可能だ。

収集したデータはCSVやJSON形式で保存することもできるため、 データ分析やレポート作成の前処理にも活用できる。

定期的にデータを収集して推移を追跡するような使い方も面白そう

⑪完全放置で動く仕組み

この方法の最大の魅力は途中で人間の介入が不要な点だ。 Claude Codeが自動的にページのHTMLを解析し、次に何をすべきかを判断して実行する。 エラーが発生しても、Claudeが自動的にリカバリーを試みてくれる。

指示を出した後は、コーヒーを飲みながら終わるのを待つだけでいい。 数ステップの操作なら数分で完了する。

動画の中でも実際に放置で完了している様子を確認できる

⑫エラー時の自動リカバリー

ページが遷移してしまった場合や、予期せぬポップアップが出た場合でも、 Claudeが状況を分析して自動的に対応してくれる。 要素が見つからない場合は、代替のセレクタを探したり、ページを再読み込みしたりする。

人間が介在せずにエラーから復旧できるのが、この手法の強みだ

⑬Computer Useとの比較

⑭この手法のメリット

DOM直接操作方式には多くのメリットがある。

  • 高速 — 画像解析なしですばやく動作
  • 低コスト — テキストのみなのでトークン消費が少ない
  • 高精度 — DOM要素を直接特定できる
  • 完全自動 — 途中で放置しても完了する
  • スクリプト生成 — 操作をコードとして再利用可能

⑮デメリットと制限事項

もちろん万能というわけではない。

  • SPA対応に制限 — React/Vue等の動的レンダリングでタイミング問題が起きる場合がある
  • Captcha/認証 — 画像認証や二段階認証は突破できない
  • Maxプラン必須 — Claude Codeの利用にサブスクリプションが必要
  • 複雑な操作 — ドラッグ＆ドロップなどの直感的操作は苦手

⑯利用時の注意点

ブラウザ自動化を利用する際は、サイトの利用規約を必ず確認すること。 スクレイピングが禁止されているサイトでは、この手法を使わないようにしよう。

また、パスワードや個人情報を扱う場合は、環境変数で安全に管理することが重要だ。 Claude Codeの対話履歴に機密情報が残る可能性がある点にも注意が必要。

他者のサイトに負荷をかけないよう、リクエスト間隔にも配慮してね

⑰実践的な活用アイデア

この手法を活かせるシーンはたくさんある。

  • 競合サイトの価格情報の定期収集
  • 定型的な申請フォームの自動入力
  • サイトの更新通知の自動確認
  • E2Eテストの自動実行
  • 日次レポートの自動生成とメール送信
Claude Codeの自動化力をブラウザ操作に応用するアイデアは無限大！

Computer Useは不要。Claude CodeとPuppeteer（またはPlaywright）の組み合わせで、 ブラウザ操作の自動化は十分に実用レベルで動作する。

スクリーンショット方式に比べて、速度・コスト・精度すべてにおいて優位だ。 設定もシンプルで、自然言語の指示だけで動くため、プログラミングの知識がなくても利用できる。
