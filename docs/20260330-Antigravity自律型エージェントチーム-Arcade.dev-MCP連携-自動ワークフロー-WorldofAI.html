Antigravityを自律型AIエンジニアリングチームに！

Subagent × Arcade.devで、1つのプロンプトからリアルなワークフローを実行する方法

2026-03-30 | WorldofAI | YouTubeで見る

①AntigravityのMission Controlとは？

AntigravityのIDE内に組み込まれているミッションコントロールシステム。バックグラウンドでのタスク委譲、カスタムサブエージェントの構築、専門スキルを持ったAIモジュールのプログラミングが可能です。

要するに、AIエージェントを「プログラム可能なモジュール」として扱える画期的な機能です。

②サブエージェントによる自動タスク分割の実例

Mission Controlで「新しいデザイナーのオンボーディング」を依頼すると、自動的に3つのサブエージェントにタスクが分割されます。

  • エージェント1: 会社サイトをクローリングしてフォント・ロゴ・カラーパレットを抽出
  • エージェント2: 収集情報をまとめて社内ウォークスルーを作成
  • エージェント3: オンボーディングメールをGmailで下書き・送信

③Antigravityの最大の課題: 実行層の不在

Antigravityのエージェントシステムは計画（Planning）の段階で止まってしまうという問題があります。GitHub、Slack、Docs、メール、Webなどのツールをまたいだ実際のアクションを安全に実行する方法がありません。

つまり「計画はできるが、実行が脆い、ハックっぽい、あるいは危険」という状態です。

④Arcade.dev: AIエージェントのための実行レイヤー

Arcade.devは無料のMCPランタイムで、AIエージェントがリアルなツールに安全に接続してタスクを実行できる実行バックボーンです。認証管理、権限管理、監査ログを備えています。

Antigravityが「何をすべきか」を決めるなら、Arcadeは「実際にそれを実行する」役割を担います。

⑤Mission Control × Arcade.devの組み合わせ

AntigravityのMission ControlをArcadeのMCPランタイムに接続すると、エージェントはプランナーからオペレーターへと進化します。ツールへのログイン、リアルなアクションの実行、ワークフローの完了まで全て自動で行います。

⑥専門エージェントチームの具体的な動き

組み合わせにより、専門チームが自動で編成されます。

  • エンジニアリング: リポジトリ作成 → Issue登録 → 担当者割り当て → 初回コミット
  • マーケティング: ドキュメント作成 → リサーチ記入 → ローンチ日程設定 → スレッド作成
  • 運用: オンボーディング自動化、メトリクス管理

⑦Arcade.devのダッシュボード構成

Arcadeのアカウントを作成（無料のHobbyプラン or $25/月のProプラン）すると、以下の機能が利用できます。

  • Tool Catalog: AIエージェントが利用可能なリアルなツール一覧
  • MCP Gateways: AIエージェントと外部システムを安全に橋渡し
  • Servers: 制御された環境でツールをホスト・公開
  • Secrets: 各ツール・MCPごとの秘密情報を管理

⑧MCP Gatewayの役割と重要性

MCP Gatewayは、AIエージェントと外部システムを標準化・セキュアに接続するブリッジです。AntigravityのMission ControlからArcadeのプラットフォームに簡単に対話できるようになります。

OAuthは自動処理、コード内にAPIキー不要、スクレイピングも不要です。

⑨ステップ1: MCP Gatewayの作成

ArcadeでMCP Gatewayを作成し、エージェントに利用させるツールを選択します。

  • 1 Gatewayに名前を付ける（例: 「Antigravity Ops Dashboard」）
  • 2 GmailSlackGoogle CalendarGoogle Docs を選択
  • 3 48個のツールを一括選択（個別選択も可能）
  • 4 MCP Gatewayを作成してスニペットをコピー

⑩ステップ2: AntigravityにMCPを統合

コピーしたMCPスニペットをAntigravityの設定に貼り付けるだけです。

  • 1 Antigravityで別セッションを開く
  • 2 Additional Settings → MCP Servers → Manage MCP Servers
  • 3 Raw ConfigにMCPスニペットを貼り付け
  • 4 Mission ControlとArcadeが接続完了
Cursor Desktop、VS Code、Windsurf、Gemini CLIにも対応しています。

⑪AI Ops Dashboardの構築を開始

Mission Controlにコンテキストを提供して、モダンなWebアプリ「AI Ops Dashboard」の構築を依頼します。このダッシュボードはメトリクスのログ、KPIの表示、環境のコンテキスト確認ができるものです。

Antigravityが何をすべきか決定してコードを書き、Arcadeがエージェントをパワーアップして実際のツール操作を行います。

⑫フロントエンドとバックエンドの並行開発

Mission Controlの強力な機能として、複数のサブエージェントを同時展開できます。

  • サブエージェントA: フロントエンドのAI Ops Dashboard UIを構築
  • サブエージェントB: Arcade連携をサポートするバックエンドアーキテクチャを構築
複数のワークスペースを作成して、異なるサブエージェントに別々のタスクを並行して依頼できます。

⑬完成したAI Ops Dashboardの機能

構築されたダッシュボードはlocalhostでアクセス可能で、以下の機能を備えています。

  • ツールのアクティビティ管理（例: デザイナーのオンボーディング）
  • Arcadeのカタログに接続されたツール一覧
  • 実行結果の確認（イベント予定、メール送信、メッセージ投稿）
  • 見栄えの良いモダンなUIデザイン

⑭実行例: Gmailでのメール作成

ダッシュボード上で「オンボーディング」を実行すると、Arcadeを通じてサブエージェントがGmailにアクセスし、下書きメールを自動作成します。1回の実行につき1つのツール実行がカウントされます。

OAuth認証が必要ですが、初回の認証後はスムーズに動作します。

⑮実行例: Slackでのメッセージ送信

Slack OAuth認証を完了すると、サブエージェントはSlack上でメッセージの送信も可能になります。「World of AIチャンネルにAIデザイナーのオンボーディング完了を通知」を実行し、DMにメッセージが自動投稿される様子を実演。

⑯実行例: Google Docsの自動作成

ArcadeのMCP経由でサブエージェントがGoogleドキュメントを新規作成することもできます。実際に「AI Designer Onboarding Guide」というオンボーディングガイドが自動生成されました。

各サブエージェントがMCPを通じて「オペレーター」となり、リアルなツール操作が可能になります。

⑰Playgroundでツールをテスト

ArcadeにはPlayground機能があり、アプリに統合する前に各ツールを個別にテストできます。ツールがどのように機能するか、モデルとどう連携するかを事前に確認できるので、実装の品質が向上します。

⑱まとめ: プログラマブルなAIワークフォースの実現

Antigravity + Arcade.devの組み合わせにより、プログラマブルなAIワークフォースが実現します。1つのエージェントですべてをこなすのではなく、専門エージェントチームが各自の役割をMCP経由で実行。スクリプト、自動化、メール送信、ドキュメント作成、会議設定など、幅広いタスクが可能です。

プランナーからオペレーターへ。AIエージェントの真の可能性がここにあります。
