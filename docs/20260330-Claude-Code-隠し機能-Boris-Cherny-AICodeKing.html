Claude Code 2.0 隠し機能 & 裏技まとめ

開発者Boris Chernyが実際に使っている機能を完全解説
2026-03-30 | AICodeKing | YouTube

①概要: Claude Codeの隠し機能を大公開

  • Boris Cherny氏がX（旧Twitter）でシェアしたClaude Codeのお気に入り機能リスト
  • BorisはClaude Codeの開発者であり、実際に日常的に使っている機能を紹介
  • Anthropicは公式の利用率ランキングを発表していないが、開発者の使い方が参考になる
Claude Codeの開発者自身が推奨する機能だから信頼性抜群！

②モバイル対応: ターミナルだけじゃない

  • iOSとAndroidのClaudeモバイルアプリからコーディングが可能
  • Boris本人もモバイルから大量のコードを書いている
  • 多くの人は「ターミナル内のツール」と思っているが、実際はマルチプラットフォーム
通勤中や外出先でもコーディングできるのは強力！

③セッション移動: デバイス間でシームレスに

  • /teleport でクラウドセッションをローカルマシンに引き継ぎ
  • /remote-control でスマホやブラウザからローカルセッションを操作
  • モバイル → Web → デスクトップ → ターミナルの自由な移動が可能
  • 手軽な場所から始めて、パワフルな環境で続きを行うのが基本スタイル
電車で始めたコードを、オフィスで続きを書ける！

④/loop: 定期的なタスクを自動化

  • /loop で指定した間隔でタスクを自動実行
  • PRの監視、リベース、Slackのフィードバック収集などに活用
  • 見落としたレビューコメントのチェックや古いPRの整理も自動化
  • 30分ごとに同じチェックをする手間がゼロになる
コーヒーを飲んでる間にClaudeがPRを監視してくれる！

⑤/schedule: スケジュール実行で自律稼働

  • /schedule で特定の日時にタスクを実行
  • Claude Codeはもはやチャットツールではなく、自律型の共同作業者に進化
  • 反復可能なワークフローを「スキル + ループ」の組み合わせで構築
  • 有料プランをワンショットでしか使っていないなら費用対効果が悪い
ワンショットモード以外も活用すべき！

⑥Hooks: エージェントのライフサイクルを制御

  • Hooks でエージェントのライフサイクルの各ポイントにロジックを追加
  • セッション開始時にコンテキストを自動読み込み
  • Bashコマンド実行前のロギング
  • 権限プロンプトを別の場所にルーティングして承認
  • エージェントが停止した時に自動で再開させることも可能
Claude Codeをプログラマブルに操作できる超強力機能！

⑦出力の検証: Claudeに結果を確認させる

  • Boris曰く「Claude Codeを使う上で最も重要なヒント」は出力を検証させること
  • フロントエンド開発にはChrome拡張機能が最も信頼性が高い
  • AIが構築したものを視覚的に確認できないと、推測で作業するのと同じ
「見えないものを直すのは不可能」——だから検証が必須！

⑧デスクトップアプリ: 組み込みブラウザでテスト

  • ClaudeデスクトップアプリはWebサーバーを自動起動して組み込みブラウザでテスト可能
  • コードを書く → アプリを起動 → Claudeが結果を確認 → 修正のループ
  • 「コンパイルが通る」ではなく、見た目が良いまで反復できる
  • Web開発において特に強力なワークフロー
書く → 確認 → 修正 のサイクルを自動化！

⑨セッションフォーク: 分岐して実験

  • /branch で既存セッションをフォーク可能
  • コマンドラインでは --fork session でフォーク
  • AIが有用な方向に進んでいる時に、別のパスを試したい場合に最適
  • 元のコンテキストを破壊せずに新しい方向を安全に探索
Gitのブランチのようにセッションを分岐！

⑩/btw: サイドクエリで文脈を汚さない

  • /btw でエージェント作業中にサイドクエリを実行
  • 簡単な質問をして即座に回答を得ることができる
  • メインの作業ラインを汚染しないので、セッションの品質を維持
  • 無関係な思考を同じスレッドに混ぜるとAIの品質が低下する問題を解決
小さな疑問があっても作業を邪魔しない！QOL改善機能。

⑪Git Worktrees: 並行作業を完全分離

  • Git Worktrees で同じリポジトリ内の並行作業を分離
  • 複数のClaudeインスタンスがそれぞれ独立したWorktreeで作業
  • お互いの作業を上書きすることなく、別々の問題に取り組める
  • 同じコードベースに対して複数の機能開発を同時に進めるのに最適
複数のClaudeが足を踏まない！並行開発の最適解。

⑫/batch: 大規模変更を一括分散実行

  • /batch で大規模なチェンジセットを複数のWorktreeエージェントに分散
  • Claudeがまずインタビューで要件を把握してから作業を分割
  • 必要な数だけWorktreeエージェントにファンアウトして並列実行
  • 大規模なマイグレーションやコードベース全体の変更に最適
Claudeが課長になって作業を分散指示！

⑬--bare: 起動を高速化するフラグ

  • --bare フラグでClaudeの起動時の自動読み込みをスキップ
  • 通常は CLAUDEファイル、設定、MCPサーバーを自動検索する
  • 非対話型のスクリプト実行やSDK利用で起動オーバーヘッドを大幅削減
  • プログラマティックな実行でパフォーマンスを気にする場合に必須
CI/CDパイプラインや自動化スクリプトで大活躍！

⑭--add-dir: 複数フォルダを同時に参照

  • --add-dir でClaudeに複数のフォルダへのアクセスを許可
  • マルチリポジトリのプロジェクトで非常に有用
  • コンテキスト切り替えなしで、1つのセッションで広範なプロジェクト構成を理解
  • 現実のプロジェクトは単一リポジトリに収まらないことが多い
モノレポじゃない現代の開発に必須の機能！

⑮--agent: カスタムエージェントで専門化

  • --agent でClaude Codeにカスタムシステムプロンプトとツールセットを提供
  • .claude/agents フォルダにカスタムエージェントを定義
  • 用途例: リードオンリー分析 マイグレーション テスト ドキュメント
  • 一見小さな機能だが、ワークフローの構造化を大きく変える可能性を秘める
専門エージェントを作れば作業効率が爆上がり！

⑯/voice: 音声でコーディング

  • /voice で音声によるコーディングが可能
  • 「真面目なコーディングはタイピングでしかできない」という思い込みを打破
  • Boris自身も声でClaude Codeに指示を出してコードを書いている
  • 非常に underrated な機能だとBorisが強調
キーボードを叩かなくてもコーディングできる未来がここに！

⑱総括: Claude CodeはフルOS環境

  • パワーユーザーはClaude Codeを全体が1つのオペレーティング環境として扱っている
  • モバイル Web デスクトップ リモートコントロール Hooks ループ ブランチ Worktrees 検証 カスタムエージェント 音声 が全て連携
  • これは単なる「隠しトリック」ではなく、Claude Code開発者自身の実用的な使用マップ
ターミナルでシンプルなプロンプトしか使っていないなら、価値の大半を見逃しているかも！
