🎤 LiveKit Agents with Gemini Live API

Gemini 3 Flashでボイスエージェントを構築

LiveKit Gemini 3 Flash Voice Agent Google Developers

①Gemini Live APIの新機能

Gemini 3ベースの初のネイティブオーディオモデル。音声をテキストに変換せずに直接処理・生成する。

②単なるインクリメンタルアップデートではない

新しいネイティブオーディオモデルは命令追従、ツール呼び出し、話者ドリフトが大幅に改善。

③ネイティブオーディオ処理

音声をテキストに変換せず直接処理。自然な音声合成を実現し、より人間らしい対話が可能。

④プロジェクトセットアップ

uv init --bareで新規プロジェクト作成。uv addでLiveKit agentsとGoogle pluginを追加。

⑤必要な認証情報

cloud.livekit.ioからLiveKit認証情報、Google AI StudioからGemini APIキーを取得して.env.localに設定。

⑥agent.pyの基本構造

環境変数を読み込み、Agentクラスを初期化。システムプロンプトをinstructionsに設定する。

⑦Gemini 3.1 Flash Audioモデル

LLMとしてgemini-3.1-flash-audio（EAP）を指定。音声の種類も選択可能。

⑧システムプロンプトのベストプラクティス

ペルソナとスコープを明確に。オーディオ用に書く（メール用ではない）。短い文、自然な間。

⑨ガードレールをシステムプロンプトに

別の安全レイヤーを頼らずに、システムプロンプトに直接リダイレクト指示を含める。

⑩ディレクターズノートで声を制御

モデルは深い音声理解を持つため、「アイルランド訛りでゆっくり」など指定すると反映される。

⑪改善されたレイテンシーと信頼性

GoogleがLive APIインフラをアップグレード。数値精度、関数呼び出し、オーディオグラウンディングが向上。

⑫話者ドリフトの低減

長いセッションで声が徐々に変わる問題を解決。ペルソナと訛りの安定性を維持。

⑬多言語サポート

約70言語をサポート。会話中に自動で言語を切り替え可能。言語コード設定不要。

⑭サーバーサイド関数呼び出し

@function_toolデコレータでツールを定義。モデルが必要に応じて自動呼び出し。サーバー上で実行。

⑮Google Searchツール統合

google.tools.GoogleSearchを追加で、リアルタイムGoogle検索が可能。追加APIキー不要。

⑯クライアントサイドRPC

LiveKitのRPCシステムでブラウザやモバイルアプリ側の関数を呼び出し。UI制御やデバイス操作に最適。

⑰デプロイとスケーリング

LiveKit CloudにCLIコマンド1つでデプロイ。オートスケールでインフラ管理不要。

⑱まとめ: 完全なボイスエージェント

テキストを挟まないネイティブ音声対話。多言語、ツール呼び出し、検索統合を備えた本格的なエージェントを構築可能。
