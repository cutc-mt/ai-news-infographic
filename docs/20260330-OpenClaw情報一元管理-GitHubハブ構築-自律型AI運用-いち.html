① 使いこなせない唯一の理由: 情報の一元管理ができていない

OpenClawを使いこなせていない人に共通するのは、環境の悪さやプロンプトの下手さではない。情報の一元管理ができていないことだけ。これをやらない限り、どれだけ触っても「ちょっと高機能なAI」のまま終わる。

プロンプトの書き方やPCスペックを気にする前に、まず情報の整理から始めるのが正解です。

② 新入アシスタントの例え: 情報を渡さないと動けない

OpenClawを入れたばかりの状態は、会社に来たばかりの新入アシスタントと同じ。初日に「明日の打ち合わせ準備して」と言っても「誰との？資料はどこ？」となる。3ヶ月情報を共有し続けると、勝手に準備してくれるようになる。OpenClawもまったく同じ原理で動く。

③ Manus / Notion AIとの決定的な違い

ManusとNotion AIは「頼まれた作業だけこなす優秀な派遣社員」。毎回ゼロから説明が必要。一方OpenClawは「あなたの環境にずっと住んでいるもう一人の自分」になれる。プロジェクトや人間関係、過去の判断をすべて知っているから、聞かれてもいないことを勝手にやってくれる。

毎回説明が必要なら「派遣」、一度入れた文脈が残って勝手に提案されるなら「相棒」。これが見分け方です。

④ 「使える人」のOpenClawがやっていること

著者のOpenClawは次のようなことを頼んでもいないのに勝手にやってくる。「3時間後の打ち合わせの協業案を3つ作っておいた」「昨日伸びた投稿パターンで新作を生成した」「Slackの手動リマインドは非効率だから自動化した」。これこそがOpenClawの本領。

⑥ なぜGitHubを本拠地にするのか

すべての情報をGitHubリポジトリに集める。理由はシンプル。バージョン管理ができる、PC壊れても復元できる、別のPCからもcloneで即利用、Git historyの検索が可能。さらにGitHub Actionsで外部サービスのデータを自動でリポジトリに流し込める。NotionやEvernoteではこれができない。

Obsidian + GitHub管理 + GitHub Actions。この組み合わせが最強です。

⑦ GitHub Actionsで外部サービスを自動集約

GitHub Actionsを使えば、外部サービスのデータをリポジトリに全自動で流し込み続けられる。Gmail、Googleカレンダー、Google Tasks、X、経費データ、すべて毎時GitHubにpushされる。OpenClawはそのリポジトリをローカルで参照しているだけ。リポジトリに全部入っているから、OpenClawは全情報を把握できる。

⑧ Gmail同期: 毎時実行のメール自動分類

GitHub Actionsで毎時Gmailを取得し、優先度P0〜P3とノイズに自動分類してMarkdownでリポジトリに保存。OpenClawはこのファイルを読むだけで「今日返信すべきメールが1件ある」と教えてくれる。手動でメールをチェックする手間がゼロになる。

.github/workflows/gmail-sync.yml

→ メール取得 → 優先度分類 → markdown保存 → git push
