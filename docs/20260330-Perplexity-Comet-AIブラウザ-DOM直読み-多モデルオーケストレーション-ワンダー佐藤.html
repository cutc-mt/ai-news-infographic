Perplexity Comet（コメット）完全解説

2026年3月30日 - Claudeを超えた最強AIエージェント、その圧倒的な強さの秘密 | ワンダー佐藤

①Cometの全貌: 次世代AIブラウザ

  • 2025年夏に登場し、秋に無料開放されたChromeベースのAIブラウザ。
  • AIがブラウザの外部ツールではなく内部に直接組み込まれたアーキテクチャ。
  • 既存のAIツールとは根本的に異なる「AIネイティブ」の設計思想。

②脅威の構成: 19以上のAIモデル

  • Cometには19〜20個のAIモデルが搭載されている。
  • Claude Opus 4.6もその中の1つに過ぎない。
  • 1対19の構成で圧倒的な力を発揮する。
単体最強のClaudeですら、多勢には無力...！

③ネイティブ統合: AIのために生まれた

  • CometはAIのために最初から作られたブラウザ。
  • Claudeは後からブラウザ操作を応用として追加したもの。
  • ネイティブ統合こそがCometの強さの根源。

④ハイブリッド処理: ローカルとクラウド

  • 軽い処理はローカルのGPU（WASM / WebGPU）で実行。
  • 重い処理はクラウドで実行するハイブリッド方式を採用。
  • レスポンスタイムを300〜500ミリ秒短縮。

⑤ゼロレイテンシー: 思考のスピード

  • タブ切り替えを待つことなく、すべてのタブの文脈を一瞬で把握。
  • クロス参照や複雑なタスクを「思考のスピード」で処理。
  • タブ間の連携がシームレスに行われる。
待ち時間ゼロ！AIとの対話が呼吸のように自然に

⑥DOMの直接読み込み: 氷山の深層

  • ウェブページの骨組みであるDOM（Document Object Model）を直接読み込む。
  • Accessibility Treeという簡易YAML形式でページ構造を把握。
  • ページ構造を立体的かつ正確に理解する。

⑦スクショ vs DOM直読み: 決定的差

  • ClaudeのComputer Useはスクリーンショットを撮って座標を叩く仕様。
  • CometはDOM構造から要素の位置を正確に把握し、外れることがない。
  • 視覚的な「推測」ではなく、構造的な「理解」による操作。
目で見るか、骨格を知るか。Cometはページの「構造」を理解している

⑧氷山の一角 vs 氷山の深層

  • Claudeは海面に浮かぶ氷山の一角しか見えていない。
  • CometはJavaScriptの動的変化や隠れた要素など氷山の深層まで透視。
  • 正確な操作を実行できる理由は、ページ全体の構造を把握しているから。

⑨3層フレームワークの構造

  • バックエンドで計画を練り、タスクを分解・最適化。
  • SSE / WebSocketでリアルタイム通信するサイドカーが橋渡し。
  • ブラウザ拡張が実際のクリックやコード挿入を実行する3層構造。
バックエンド → サイドカー → ブラウザの流れで精密に連携！

⑩Opus 4.6を司令塔にした統率

  • 複雑なタスクの分解や計画立案をAnthropicの最高峰モデルが担当。
  • Claude Opus 4.6が指揮役としてコントロールする「Perplexity Computer」システム。
  • タスクの全体像を把握し、各モデルに最適な役割を割り振る。

⑪最強モデル陣: 各領域の専門家

  • Gemini 3.1 Proは深いリサーチとデータ合成に最適。
  • Grokは高速処理、GPT-5.2は長文・クリエイティブに最適解を出す。
  • Sonnetはコーディングに特化。それぞれが最適な役割を果たす。
適材適所の夢のオーケストレーション！

⑫画像・動画生成も適材適所

  • 画像生成にはNano Bananaを自動選択。
  • 動画生成にはVeo 3.1を並列で使い分け。
  • あらゆるタスクに最適なモデルを自動で選択して提供。

⑬Claude単体の決定的な弱点

  • Claudeは外側から画面をキャプチャして仮想マウスを動かす仕様。
  • エラーや意図のズレ、遅延が発生しやすい。
  • 反復ループのタスクで失敗が目立つのが最大の課題。

⑭実験: シューティングゲームのテスト

  • ClaudeのComputer Useにゲーム作成とテストプレイを指示。
  • スクショが真っ黒になり座標が当たらずテストプレイに失敗。
  • Chrome in Claudeでも同様に失敗。画面認識の限界が露呈。
Claudeには画面が「見えていない」...黒い画面では何もできない

⑮同じタスクをCometが一発完走

  • 同じプロンプトをCometに投げると、コードを書いて実行しテストプレイまで完了。
  • パズルゲームも自動生成 → 実行 → クリック操作 → クリア判定まで自律完走。
  • ループ処理で自律的にタスクを完遂する能力。

⑯スライドパズル: 全項目テスト

  • Cometはタイル移動、手数カウント、クリア判定、シャッフルを全項目確認完了。
  • Claudeが表面しか見ていないのに対し、Cometは構造を理解している。
  • DOMの直接読み込みが、インタラクティブなテストで圧倒的な差を生んだ。
タイルの移動も判定もすべて正確に把握！構造理解の差は歴然

⑰究極のコストパフォーマンス戦略

  • 自社でAIを開発するのではなく、各社の最高峰モデルをブラウザという「体」に移植。
  • 高価なOpus 4.6は司令塔に、安価なGrokやコスパの良いGPT-5.2を適材適所で配置。
  • コストとパフォーマンスの最適なバランスを実現する戦略。

⑱アルティメット悟飯 vs ゴジータ

  • Claude単体はアルティメット孫悟飯（単体最強）。
  • Cometはゴジータ / ベジット（複数融合で最強）。
  • AIエージェントはブラウザとの完全融合が最適解。
単独の力 vs 融合の力。ドラゴンボールで例えると一目瞭然！
