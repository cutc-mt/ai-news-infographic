①Cline Kanbanとは？

AIコーディングエージェント「Cline」の新機能。タスクをカンバン方式で管理し、複数のタスクを並列実行できるオーケストレーションツール。ターミナルから起動してブラウザで操作する。

Clineはプログラミング・コーディング系のAIエージェントとして有名なツールです。

②オーケストレーション開発とは

複数のAIエージェントが同時に別々のタスクを実行し、互いの依存関係を管理しながら開発を進める手法。1つのエージェントでは時間がかかる作業を分割して効率化できる。

  • タスクをフェーズごとに分割
  • 依存関係を自動で判定
  • 独立タスクは並列で同時実行

③今回のプロジェクト概要

コンテキスト（文字起こしやMarkdownファイル）を入力すると、AIが複数の視点からメモを自動生成するアプリを開発。裏側のAPIにはGLM-5を使用予定。

  • コンテキスト入力欄
  • ディレクター・プロデューサー視点など
  • 自分でメモも書ける
  • Markdownでダウンロード可能

④Cline Kanbanの起動方法

ターミナルからコマンドを実行すると、localhostでWebページが立ち上がる。ブラウザ上でカンバンを操作でき、左側にClaude Codeのチャットも表示される。

Web UIベースなので、直感的に操作できるのが魅力です。

⑤対応するエージェント

現在はClaude CodeとCodexの2つに対応。カンバン上でエージェントを選択して開発を進められる。

  • Claude Code - AnthropicのAIエージェント
  • Codex - OpenAIのコーディングエージェント
  • 今後はCursor CLIやComposer 2も対応予定？

⑥タスクの自動分解

要件定義書や技術スタックを確認させると、Claude Codeが自動的にタスクをフェーズに分解してカンバンに積んでくれる。ユーザーは「並列で実行してほしい」と指示するだけ。

あの会話してるClaude Codeが直接タスクを分解して入れてくれるんです！

⑦並列実行の仕組み

フェーズ1が完了すると、フェーズ2のタスクAとBが同時スタート。依存のないタスクは並列で同時に処理され、開発スピードが大幅に向上する。

  • フェーズ1 → 完了
  • フェーズ2A + フェーズ2B → 同時実行
  • フェーズ3A → 最終フェーズ

⑧依存関係の可視化

タスク間の依存関係が線で視覚的に表示される。タスクが進むにつれて線が減っていく様子は、まるでゲームのように楽しい。ぐにゃぐにゃ動くアニメーションも特徴的。

見てるだけで楽しい、気持ちいい！

⑨手動でタスクを操作

自動だけでなく、タスクの手動移動・削除も可能。カンバン上でドラッグ&ドロップのようにタスクの状態を変更できる。柔軟な運用ができる。

  • タスクの移動（手動）
  • タスクの削除
  • タスクの追加

⑩進捗の確認方法

実行中のタスクは開いて詳細を確認できる。トークンの消費量もリアルタイムで表示され、Claudeのセッション残量も把握できる。

トークン消費が激しいので、Claudeの残量には要注意です。

⑪コミット機能

各フェーズのタスク完了後にコミットボタンで確認・承認できる。コミットを行うと次のフェーズに自動的に移動し、開発フローがスムーズに進む。

CLAUDE.mdに設定を書いておけば、コミットも完全に自動化できる可能性があります。

⑫レビューフロー

タスク完了後に自動レビューが走る仕組み。ライブ配信時は手動で承認ボタンを押す形だったが、自動レビューも設定可能。レビュー → 完了の流れが整理されている。

  • 自動レビュー（設定次第）
  • 手動レビュー（承認ボタン）
  • レビュー完了でタスク終了

⑬セッションリミットに注意

並列実行するとトークンを大量に消費する。動画内ではClaudeのトークン残量が100%に達し、オートチャージが発動。コスト管理には注意が必要。

4月から消費税がかかるので、実質コストが上がります。

⑭類似ツールとの比較

視聴者からのコメントで紹介された類似ツールも存在する。

  • Vibes Kanban - 同様の並列実行機能
  • Superset - オーケストレーションが可能
  • Cline Kanban - Web UIで直感的に操作

⑮カスタマイズ可能性

Cline Kanbanはリポジトリが公開されており、フォークして自由にカスタマイズできる。動画内の配信でもライブでコードをいじるデモを実施。Cursor CLI対応なども期待。

  • オープンソースで公開中
  • フォークして自由に改造可能
  • 独自のエージェント追加も視野に

⑯開発したアプリのUI

完成したアプリはObsidianへの統合も考慮。コンテキスト入力 → AIメモ生成 → Markdownダウンロードの流れ。システムプロンプトで視点の切り替えも可能。

  • 文字起こしやMarkdown読み込み
  • 3～4視点でメモ自動生成
  • Markdown出力でObsidianに保存
  • 視点のカスタマイズ機能

⑰今回の成果

「見てるだけで楽しい！」ツールの完成度ではなく、体験の楽しさが重要。
