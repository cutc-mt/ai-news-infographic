Azure AI Foundry Agent Service
RAG不要で複数ファイルを秒で検索

Foundry Agent Service × File Search Tool × セマンティック検索

Shweta Lodha チャンネル

①Azure AI Foundry Agent Serviceとは

  • Azureが提供するフルマネージドのAIエージェント構築サービスです。
  • GPT-4o、Llama、Mistralなどの強力なモデルを組み合わせてエージェントを構築できます。
  • セキュアにビルド、デプロイ、スケールが可能な環境を提供します。
  • Code Interpreter、Function Calling、File Searchなどのツールを簡単に追加できます。
従来のRAGパイプラインを自前で構築する必要がなく、エージェントにツールを追加するだけでファイル検索が実現できるのが大きな特徴です！

②File Search Toolの概要

  • エージェントに外部ファイルの知識を簡単に追加できるツールです。
  • ファイルをアップロードするだけで、自動的にパース、チャンク分割、埋め込み生成を行います。
  • ベクトル検索とキーワード検索の両方を自動で実行します。
  • 対応フォーマット：PDF、Word、HTML、Markdown、JSON、Python、テキストなど多数。
RAGのパイプラインを自前で構築しなくても、このツールを追加するだけでセマンティック検索が使えるのが最大のメリットです！

③RAG不要の仕組み

  • 従来のRAGでは、チャンク分割、埋め込み生成、ベクトルDBの構築を自分で実装する必要がありました。
  • File Search Toolは、これらすべてをバックグラウンドで自動処理します。
  • ユーザーのクエリを自動で最適化し、複雑な質問は複数の検索に分解して並列実行します。
  • 検索結果を再ランキングして、最も関連性の高いコンテンツをモデルに渡します。
つまり、従来なら数週間かかるRAG構築が、File Search Toolなら数分で完了するということです！

④Basic vs Standardの2つのセットアップ

  • Basic セットアップ：Microsoft管理の検索・ストレージリソースを使用。ファイルのアップロードのみ対応。
  • Standard セットアップ：自分のAzure AI SearchとBlob Storageを使用。ファイルアップロードとBYO Blob Storageの両方に対応。
  • コードの違いはなく、リソースの保存先だけが異なります。
  • Standardセットアップでは、ファイルが自分のストレージに残るためデータ管理を完全にコントロールできます。
Blob Storageを使ったFile Searchは、Standardセットアップでのみサポートされています。Basicではローカルファイルのアップロードのみ可能です。

⑤Vector Storeの仕組み

  • ファイルを追加すると自動的にパース、チャンク分割、埋め込み、ベクトルDBへの保存が行われます。
  • 1つのベクトルストアに最大10,000ファイルまで保存可能です。
  • ベクトルストアはエージェントとスレッドの両方にアタッチできます（各1つまで）。
  • ファイルサイズは最大512MB、1ファイルあたり最大500万トークンまで対応。
ベクトルストアを作成したら、ポーリングで「completed」ステータスを確認しましょう。ファイルの処理が完了してからRunを作成するのが推奨されています！

⑥デフォルトの検索設定

  • チャンクサイズ：800トークン（1チャンク当たり）
  • チャンクオーバーラップ：400トークン（隣接チャンクとの重複部分）
  • 埋め込みモデル：text-embedding-3-large（256次元）
  • コンテキストに追加する最大チャンク数：20個
これらの設定は検索のベストプラクティスが反映されており、追加の設定なしで高品質な検索結果が得られます！

⑦Python SDKでプロジェクトクライアントを作成

  • azure-ai-projectsパッケージとazure-identityパッケージをインストール。
  • 接続文字列を使ってAIProjectClientを初期化します。 
from azure.ai.projects import AIProjectClient
from azure.ai.projects.models import (
    FileSearchTool, MessageAttachment,
    FilePurpose
)
from azure.identity import DefaultAzureCredential

credential = DefaultAzureCredential()
project_client = AIProjectClient.from_connection_string(
    credential=credential,
    conn_str=os.environ["PROJECT_CONNECTION_STRING"]
)

⑧ファイルのアップロードとVector Storeの作成

  • upload_file_and_pollでローカルファイルをアップロードして処理完了を待機します。
  • create_vector_store_and_pollでベクトルストアを作成し、インデックス構築を待ちます。 
# ファイルをアップロード
file = project_client.agents.upload_file_and_poll(
    file_path='./data/product_catalog.md',
    purpose=FilePurpose.AGENTS
)
print(f"ファイルID: {file.id}")

# ベクトルストアを作成
vector_store = project_client.agents.create_vector_store_and_poll(
    file_ids=[file.id],
    name="my_vectorstore"
)
print(f"ベクトルストアID: {vector_store.id}")
_and_pollメソッドを使うと、ファイルの処理完了を自動でポーリングしてくれるので便利です！

⑨エージェントにFile Searchを追加

  • FileSearchToolオブジェクトを作成し、ベクトルストアIDを渡します。
  • toolsとtool_resourcesの両方をエージェントに渡す必要があります。 
# File Search Toolを作成
file_search_tool = FileSearchTool(
    vector_store_ids=[vector_store.id]
)

# エージェントを作成
agent = project_client.agents.create_agent(
    model="gpt-4o-mini",
    name="my-agent",
    instructions="あなたは役立つエージェントです",
    tools=file_search_tool.definitions,
    tool_resources=file_search_tool.resources,
)

⑩スレッドとメッセージの作成

  • Thread（スレッド）は、ユーザーとエージェント間の会話セッションです。
  • メッセージにファイルを添付すると、スレッド専用のベクトルストアが自動作成されます。
  • Runの実行時、エージェントのベクトルストアとスレッドのベクトルストアの両方を検索します。 
# スレッドを作成
thread = project_client.agents.create_thread()

# ファイルをメッセージに添付
message_file = project_client.agents.upload_file_and_poll(
    file_path='product_info_1.md',
    purpose=FilePurpose.AGENTS
)
attachment = MessageAttachment(
    file_id=message_file.id,
    tools=FileSearchTool().definitions
)
message = project_client.agents.create_message(
    thread_id=thread.id,
    role="user",
    content="Smart Eyewearの機能は？",
    attachments=[attachment]
)

⑪Runの実行と結果の取得

  • create_and_process_runでスレッド上のエージェントを実行し、完了を待機します。
  • 結果は引用付きで返され、ソースファイルがどこに書かれているか確認できます。 
# Runを実行して結果を取得
run = project_client.agents.create_and_process_run(
    thread_id=thread.id,
    agent_id=agent.id
)
print(f"Run ID: {run.id}")

# メッセージ一覧を取得
messages = project_client.agents.list_messages(
    thread_id=thread.id
)
print(f"メッセージ: {messages}")

# クリーンアップ
project_client.agents.delete_vector_store(vector_store.id)
project_client.agents.delete_agent(agent.id)

⑫Azure Blob Storageとの連携

  • Standardセットアップでは、Azure Blob Storageに保存されたファイルを直接参照できます。
  • asset_uriを使えば、同じファイルを複数のエージェントで再アップロードなしで使い回せます。
  • VectorStoreDataSourceでBlob Storageのファイルをベクトルストアに追加します。 
# Blob Storageにアップロード
_, asset_uri = project_client.upload_file(
    "sample_file.md"
)

# ベクトルストアにBlob Storageを参照
ds = VectorStoreDataSource(
    asset_identifier=asset_uri,
    asset_type=VectorStoreDataSourceAssetType.URI_ASSET
)
vector_store = project_client.agents.create_vector_store_and_poll(
    data_sources=[ds],
    name="blob_vector_store"
)

⑬File Searchの検索プロセス詳細

  • Step 1：ユーザーのクエリを検索に最適な形に自動で書き直します。
  • Step 2：複雑な質問を複数のサブ検索に分解して並列実行します。
  • Step 3：エージェントとスレッドの両方のベクトルストアに対して、ベクトル検索＋キーワード検索を実行。
  • Step 4：結果を再ランキングして最も関連性の高いものをモデルに渡します。
クエリの自動最適化と並列検索により、複数ファイルからでも数秒で関連情報を見つけ出せる仕組みです！

⑭対応ファイルフォーマット

  • PDF、Word（.doc/.docx）、PowerPoint（.pptx）
  • Python（.py）、JavaScript（.js）、TypeScript（.ts）、C/C++（.c/.cpp/.cs）
  • Markdown（.md）、HTML（.html）、JSON（.json）、テキスト（.txt）
  • テキスト系MIMEタイプのエンコーディングはUTF-8、UTF-16、ASCIIに対応。
  • 1ファイル最大512MB、500万トークンまで。

⑮有効期限ポリシーでコスト管理

  • ベクトルストアのサイズに基づいて課金されるため、不要なストアは削除が重要です。
  • 有効期限ポリシーを設定して、自動的にベクトルストアを削除できます。
  • スレッドのベクトルストアはデフォルトで7日間の有効期限が設定されています。 
# 有効期限を7日間に設定
vector_store = project_client.agents.create_vector_store_and_poll(
    name="Product Documentation",
    file_ids=[file_1.id],
    expires_after={
        "anchor": "last_active_at",
        "days": 7
    }
)

⑯Azure AI Foundryポータルでの操作

  • コードを書かなくても、Azure AI FoundryポータルのGUIからFile Search Toolを追加できます。
  • 手順：エージェントの「Create and debug」画面 → Setup → Knowledge → Add → Files
  • エージェントのプレイグラウンドでリアルタイムにテストが可能です。
  • 「Create an agent」ボタンから始めると、GPT-4oモデルが自動デプロイされ、デフォルトエージェントが作成されます。
ポータルの基本セットアップはBasicのみ対応。Standardを使いたい場合は環境セットアップの記事を参照してください！

⑰エージェントの主要ツール一覧

  • File Search：ファイルのセマンティック検索（本動画のメインテーマ）
  • Code Interpreter：コードの実行とデータ分析、グラフ生成
  • Bing Grounding：Web検索で最新情報を取得
  • Azure AI Search：独自の検索インデックスと連携
  • Function Calling：カスタム関数をエージェントに呼び出させる
これらのツールは組み合わせて使うことができます。例えば、File Searchで文書を検索しつつCode Interpreterで分析、という使い方も可能です！
