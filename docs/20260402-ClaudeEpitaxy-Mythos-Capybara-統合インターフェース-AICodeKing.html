セクション1: Epitaxyリークの概要

①Epitaxy：最大のリークはモデルではない

  • 2026年3月31日、新たなAnthropicリークが流出。最も注目すべきはEpitaxyという新インターフェースの存在である。
  • Claude CodeのWebモード、あるいはClaude Desktopの新モードとして、ローカルとWebの実行を統合するインターフェースとみられる。
  • リークされたクリップの1つには、ブラウザのdevToolsが開かれた状態でclaude.ai/epitaxyというURLが確認できる。
モデルのリーク以上に重要かもしれない！
ユーザー体験を根本から変える可能性があるんだ。

②リークされたインターフェースの詳細

  • インターフェースの下部にLocal / Select Folder / Work Treeのコントロールが表示されている。
  • Auto-Accept Editsのトグルやプロンプトボックス、モデルピッカーも確認できる。
  • これはランダムなコンセプトアートではなく、実際に動作するビルドのように見える。
Claude CodeスタイルのUIがそのまま
ブラウザ上で動くイメージ！

③claude.ai/epitaxyルートが意味するもの

  • 1つのクリップはローカルのClaude Code風インターフェース、もう1つはclaude.aiのWebルート上で同じものが表示されている。
  • つまり、AnthropicはローカルとWebのどちらでもタスクを実行できる統合インターフェースを構築中と推測できる。
  • 現在のClaudeはCLI・Desktop・Webがバラバラだが、Epitaxyはそれを1つのプロダクトに統合する可能性がある。
製品を切り替えるのではなく、
実行先だけを切り替える世界が来るかも！

④統合タスクインターフェースの構想

  • ローカルで作業する場合は「Local」を選んでフォルダを指定し、Work TreeをトグルしてClaudeに実行させる。
  • Webやホスト環境にタスクを送りたい場合は、同じインターフェースから別の実行先を選ぶだけ。
  • プロダクトを切り替えるのではなく、実行ターゲットを切り替えるだけの世界観である。
CLI・Desktop・Webの3つが
1つのシームレスな体験に統合される！

  • インターフェースが「異なる状態」であることを明確に示すことで、ユーザーに適切な期待を持たせる設計である。

⑨MythosとCapybaraは同じモデル

  • 以前のリークでMythosとCapybaraという2つのコード名が登場したが、実際は同じモデルの別名と判明。
  • Anthropicが最終的なブランディングを決める前のドラフト名または並行命名だったとみられる。
  • 2026年3月26日、Fortune誌がAnthropic公式が新フロンティアモデルのテストを確認したと報じている。

⑩CapybaraはすでにV8に到達

  • リーク情報によると、AnthropicはすでにCapybara V8に到達している。内部で非常に高速に反復開発を進めている。
  • AnthropicはCapybaraをClaude Code自体の開発に使用しており、自社製品でドッグフーディングを行っている。
  • コーディング・推論・サイバーセキュリティでOpusを超えるステップチェンジと位置づけられている。

⑪Capybaraの弱点が信憑性を高める

  • リークノートには「過剰なコメント」と「誤った主張」の問題が記載されている。
  • フェイクのリークは全てを完璧に見せようとするが、「強いが不完全」という表現はリアルな内部メモらしい。
  • 強くて速く、巨大なコンテキストを持つが、まだ過剰コメントや誤情報の傾向が残っているという正直な評価。

⑫100万トークンコンテキストと高速モード

⑭モデルだけではない：プロダクトシフト

  • フロンティアモデル単体はベンチマークと見出しには良いが、実際の製品体験には不十分である。
  • EpitaxyはローカルやWebで有用なタスクを起動できる統合インターフェースであり、モデルの能力を製品体験に変える。
  • Anthropicは単なるモデル更新ではなく、Claude全体のあり方を変える可能性がある。

⑮ワークフロー断片化の問題を解決

  • 現在のClaudeはCLI・Desktop・Webが別々の体験になっており、ワークフローが断片化している。
  • ローカルタスクはファイルやターミナル、リポジトリのコンテキストが必要で、マシンを占有する。
  • リモートタスクはマシンを占有せずにバックグラウンドで実行したい場合に適している。

  • 単にチャットメッセージを送るのではなく、長時間実行タスクを起動するという意図をUIで伝える設計。
  • 燃える「Let Claude Cook」画面は、エージェントの実行中であることを直感的に伝える。
