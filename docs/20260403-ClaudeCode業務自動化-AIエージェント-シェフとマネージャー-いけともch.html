AIエージェントを使いこなす技術と実践デモ

Claude Code 業務自動化 AIエージェント いけともch

① Claude Codeは「優秀なシェフ」兼「マネージャー」

Claude Codeは一言で言うと、優秀なシェフでありレストランマネージャーでもあります。シェフとして毎回異なるリクエストに柔軟に応え、季節の食材のように状況に合わせて創意工夫ができます。さらにマネージャーとして、決まりきったレシピは他のスタッフに任せ、材料の仕入れや器具の手配までこなします。

② チャット型AIとエージェントアプリの違い

これまでのChatGPTやGeminiのチャット版は飲食店コンサルタントのような存在でした。アドバイスや設計図は作れるが、自分で厨房に立つことはできません。一方でClaude Codeは一緒にレストランを運営するパートナーであり、直接調理も店舗管理もこなす全く異なる存在です。

③ YouTube投稿をClaude Codeだけで完結

これまではGeminiやChatGPTを使って別々のツールを作り、手作業で繋いでいました。しかしClaude Codeだけで動画からアップロードまで全部完結します。動画ファイルを渡すと、音声抽出、文字起こし、タイトル作成、概要文作成、サムネイル画像作成、YouTubeアップロードまで一気に自動化されます。

④ YouTube自動化の具体的な流れ

全体の流れは以下の通りです。動画ファイルがあると、まずニュースかどうかを確認し音声をピックアップ。次に文字起こしをして、タイトル案と概要文を作成。内容から学習した上でサムネイル画像を生成し、最後に非公開設定でYouTubeにアップロードします。各ステップで「シェフの柔軟な対応」と「スタッフの固定処理」を組み合わせています。

⑤ 柔軟パートと固定パートの設計思想

自動化を設計する上で最も重要なのが、「柔軟パート」と「固定パート」の区別です。タイトルや概要文の作成は毎回内容が違うため、AIの柔軟な思考に任せます。一方で文字起こしやアップロードは毎回同じ処理なので、プログラム化して固定化します。この2つをどう組み合わせるかが自動化の要です。

⑥ 外部ツール連携の3つの方法（全体像）

Claude Codeが外部ツールを操作する方法は大きく3つあります。①AIが直接操作する手段（MCP/CLI/API）、②ブラウザを操作する、③デスクトップ自体を操作する。この3つの方法を理解しておくことが、自動化の設計において非常に重要になります。

⑦ MCP（コネクター）による連携

MCPは2024年11月にAnthropicが公開したオープンフォーマットです。Figma、Notion、Slackなど様々なツールとAIを繋ぐ仕組みで、コネクターを追加するだけで連携が可能です。手軽で簡単ですが、対応していないツールもあり、MCPを増やしすぎるとAIの負荷が上がりパフォーマンスが下がる弱点もあります。

⑧ CLI（コマンドライン）による高速操作

CLIは黒い画面からコマンドで操作する方法です。例えばGoogle Workspaceを操作するgwsコマンドを使えば、スプレッドシートからデータを取得したりドキュメントに書き込んだりできます。ブラウザを開かずに処理できるため高速で安定しています。AIはコマンドライン操作が非常に得意で、勝手にコマンドを考えて実行してくれます。

⑨ APIを活用したプログラム連携

APIはプログラムからツールを呼び出す昔からある仕組みです。Claude Codeは自分でプログラムを作成し、その中にAPI呼び出しを記述して実行できます。MCPがなくCLIも使えない場合でも、APIが公開されていれば操作が可能です。実際、会計ツール「freee」の自動化ではAPIが活用されました。

⑩ ブラウザ操作とデスクトップ操作

MCP・CLI・APIのいずれも使えない場合は、ブラウザ操作が選択肢になります。ClaudeのChrome拡張機能で画面を直接操作する方法で、URLが開けるサービスなら何でも自動化可能です。さらに2026年4月3日からWindows版でもデスクトップアプリの操作に対応し、動画編集ソフトなどブラウザ以外のアプリも操作できるようになりました。

⑪ 実践デモ：ニュース収集からSlack投稿まで

実際にClaude Codeを使ってWeb検索でAI情報を収集→Excelに蓄積→PowerPointレポートを生成→Slackに投稿する仕組みをライブデモで構築しました。Excelのテンプレートにデータを追加し、PowerPointのフォーマットを保ったまま中身を書き換え、最後にSlackのMCP経由でメッセージを送信する一気通貫の自動化です。

⑫ サブエージェントによる並行処理

デモでは3つのPythonスクリプトがサブエージェントとして並行して作成されました。Excel書き込み用、PowerPoint生成用、Slack投稿用のスクリプトがそれぞれ独立して作られ、プランに基づいて役割分担しながら処理が進みます。Claude Codeが自らマネージャーとして作業を割り振る仕組みです。

⑬ エラー発生時の自己修復

PowerPoint生成でエラーが発生した際、Claude Codeが自ら原因を分析して修正案を提示しました。スライドの複製が直接できない場合はXMLレベルで操作するという別のアプローチに切り替えるなど、試行錯誤しながら解決に導きます。エラーの内容と対応策はファイルにまとめておくことで、次回以降の学びとして活用できます。

⑭ CLAUDE.mdで仕組みを永続化

Claude Codeはフォルダーを開くたびにCLAUDE.mdを自動的に読み込みます。このファイルにプロジェクトの仕組みやルールを書いておけば、新しいチャットセッションを開始しても即座に文脈を理解します。CLAUDE.mdがないと毎回ゼロから全ファイルを確認する必要があり、時間もコンテキストも無駄に消費してしまいます。

⑮ progress.mdで進捗を管理

progress.mdに作業進捗をまとめておくことで、別のチャットセッションで再開する際にスムーズに作業を継続できます。コンテキストウィンドウの上限に達して新しいセッションを開く場合や、並列で複数のセッションを立ち上げる場合にも、進捗状況が分かれば効率的に作業が進められます。

⑯ メモリー機能で知見を蓄積

プロジェクトごとに得た学びはSpiceデータベースに保存して再利用できます。例えば「PowerPoint作成時のXML対応」の知見をメモリーに保存しておけば、別プロジェクトでPowerPointを扱う際に「前回のポイントを踏まえてやって」と指示するだけで、過去の経験を活かした効率的な処理が可能です。

⑰ セキュリティとバックアップの重要性

Claude Codeは様々なツールに接続できるため、データのバックアップと権限管理が重要です。会計データは登録前にスプレッドシートに履歴を残し、MCPの権限設定で書き込みに承認を必須にするなどの対策が必要です。またフック機能を使えば、特定フォルダー外への書き込み時に強制的にチェックを入れる仕組みも構築できます。

⑱ まとめ：チャットアプリからエージェントアプリへ

Claude Codeは飲食コンサルタントからシェフ兼マネージャーへの進化です。柔軟なAI思考と固定スクリプトを組み合わせ、外部ツールはMCP・CLI・API・ブラウザ・デスクトップの5つの手段で連携。エンジニアでなくても自動化が可能で、近い将来チャットアプリからエージェントアプリへの移行が加速するでしょう。
