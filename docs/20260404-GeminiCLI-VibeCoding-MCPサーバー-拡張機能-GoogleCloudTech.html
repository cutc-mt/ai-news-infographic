🚀 Gemini CLIでVibe Coding

コンテキストエンジニアリングからMCPサーバー・拡張機能まで

Gemini CLI Vibe Coding MCP Server Google Cloud Tech

①Gemini CLIとは: ターミナルベースのコーディングエージェント

直接ターミナルでコードを書けるAIツールで、Antigravity（IDEベース）とは異なるアプローチを採用している。ターミナル上で自然言語の指示を与えるだけで、ファイルの作成や編集、実行までを自動で行ってくれる。

②モデルの選択: 用途に合わせて使い分け

Flashモデルで高速処理、Proモデルで深い推論が可能。無料アカウントではGemini 1.5がデフォルトで利用できる。開発のスピードと精度のバランスを考えてモデルを選ぶことが大切。

③Antigravityとの違い: IDE vs ターミナル

Gemini CLIはターミナル主体のツールで、AntigravityはIDEベースのエージェントファースト体験を提供する。用途に合わせて使い分けが可能で、両方を組み合わせることでさらに強力な開発ワークフローを構築できる。

④コンテキストの管理術: gemini.mdを活用

gemini.mdファイルでプロジェクトレベルとユーザーレベルの文脈を管理する。プロジェクトごとに異なる定義を設定でき、コーディング規約や技術スタックの情報をAIに伝えることができる。

⑤メモリとスキルの違い: 常時読み込み vs 遅延読み込み

メモリは常にロードされるシステム指示として機能し、スキルは必要な時だけ遅延ロードされる動的コンテキストである。この使い分けがコンテキストウィンドウの効率的な利用に不可欠。

⑥ビルトインツール: 最初から使える機能群

Google検索、ファイル読み込み、フォルダ読み込み、スキルの有効化など、Gemini CLIに最初から組み込まれているツール群がある。追加の設定なしですぐに使い始めることができる。

⑦Gemini CLI拡張機能: パッケージマネージャー

MCPサーバーやスキル、フックを簡単に統合するパッケージマネージャー。extensions installコマンドで導入でき、コミュニティが開発した拡張機能をワンコマンドで追加可能である。

⑧MCPサーバーとは: エージェントとツールの統一通信

Model Context Protocolでエージェントとツールの通信を統一する仕組み。GiteaのようなツールをGemini CLIに接続でき、標準化されたプロトコルで様々な外部サービスを統合できる。

⑨settings.jsonの役割: 設定の一元管理

MCP接続、フック、認証を一元管理する設定ファイル。カスタマイズの中心となる重要なファイルで、ここにMCPサーバーの情報を追加するだけで外部ツールとの連携が可能になる。

⑩GiteaでSDLC管理: AIが開発サイクルを自動化

MCPサーバー経由でリポジトリ作成、イシュー発行、コードプッシュをAIが自動実行する。ソフトウェア開発ライフサイクル全体をAIで管理でき、開発効率が飛躍的に向上する。

⑪Nano Banana拡張: 高精細なテキストから画像生成

高精細なテキストから画像生成モデルで、4K、2K、1080の解像度と様々なアスペクト比に対応している。El Marinaのリーダーボードでトップクラスの品質を誇り、MCP経由でGemini CLIから呼び出せる。

⑫セッション管理: 柔軟な作業の中断と再開

各Gemini CLI起動で新しいセッションが作成される。セッションIDを使って前のセッションを再開することも可能で、作業の中断と再開が柔軟に行える。クリーンな状態で始めるのが基本。

⑬Vibe Codingの芸術: コンテキスト管理がスキル

AIは確率的な出力をするため、コンテキストの管理と指示の出し方がスキルとなる。エージェントを適切に監視することが重要で、良いコンテキストを与えるほど良い結果が返ってくる。

⑭YOLOモード: 権限プロンプトをバイパス

権限プロンプトをバイパスして迅速に開発できるモード。非クリティカルなファイルでの快速開発に最適だが、本番環境では推奨されない。開発スピードと安全性のバランスを考えることが大切。

⑮Shellモード: AI対話とシェル操作の切り替え

!を付けてコマンドを実行すると、Gemini CLIをバイパスして従来のシェルコマンドを直接実行できる。EscキーでAI対話モードに戻るため、シームレスな切り替えが可能である。

⑯AIで完結する開発サイクル: 全自動ワークフロー

イシュー作成から画像生成、コード修正、プッシュ、イシュークローズまで全てAIが実行する。驚くほどの自動化が実現しており、開発者は指示を出すだけで開発サイクルを回せる。

⑰Cloud Shell開発環境: どこからでも開発可能

Google Cloudの永続的なリモートVM環境で、VS Codeのようなエディタとターミナルを備えている。ブラウザからどこでもアクセスでき、Gemini CLIとの相性も抜群である。

⑱次回予告: さらに高度な内容を予定

エージェントスキル、フック、CI/CDパイプライン、Cloud Runへのデプロイ、ボスファイトなど、さらに高度な内容が予定されている。Gemini CLIの可能性はまだまだ広がっていく。
