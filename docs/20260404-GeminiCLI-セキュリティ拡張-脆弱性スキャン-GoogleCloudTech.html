セキュリティ拡張の概要と背景

① セキュリティ拡張とは

  • オープンソースのGoogle製セキュリティ拡張機能がGemini CLIに登場
  • AIの力でコードのセキュリティレビューを自動化する
  • Google Cloudと連携して強力なスキャンを実行可能
オープンソースだから誰でも自由に使える！

② AI時代のセキュリティ課題

  • AIの登場でコードのスピードと量が劇的に増加している
  • セキュリティレビューのペースも上げる必要がある
  • 手動レビューでは追いつかない速度にAIで対応する
AIが書いたコードをAIがチェックする時代が到来！

③ 5つの脆弱性スキャン

  • Secrets Management：シークレットの管理と検出
  • Insecure Data：安全でないデータ処理の検出
  • Injection：インジェクション脆弱性のスキャン
  • Authentication：認証の問題を検出
  • LLM Safety：LLMの安全性チェック
セキュリティの主要な5領域を網羅している！

④ OSVデータベース連携

  • GoogleのOSV（オープンソース脆弱性データベース）を使った依存関係スキャン
  • サードパーティライブラリの既知の脆弱性をチェック
  • サプライチェーンの安全性も確保できる
依存関係の脆弱性も見逃さない！

⑤ Google内部での実績

  • 多くのGoogle所有リポジトリがセキュリティ拡張を使用している
  • 実際のPull Requestで脆弱性を発見している
  • 本番環境のGoogleプロジェクトで信頼性が実証済み
Google自身が使っているから安心！

⑥ 実際に発見された脆弱性

  • Gemini CLI自身の任意ファイル読み取り脆弱性を発見
  • Project Chipの環境変数リダクション回避の脆弱性を発見
自分自身の脆弱性も発見する威力！
ローカルでの使い方

⑦ Flutterのパストラバーサル

  • Flutterリポジトリのパストラバーサル脆弱性も発見
  • タイムスタンプのハッシュコード使用も指摘されている
  • 大手オープンソースプロジェクトでも実証されている
Flutterのような有名プロジェクトでも見つかる！

⑧ ローカルでの使い方

  • 拡張機能をインストール後、すぐに利用可能
  • /ashコマンドでセキュリティカスタムコマンドを呼び出す
  • コマンド一覧からスキャンを実行できる 
# セキュリティコマンドを呼び出し
> /ash

# スキャンを実行
> セキュリティスキャンを開始
コマンド1つでスキャンが始まる！

⑨ 自然言語でカスタマイズ

  • 自然言語でスキャン対象を指定可能
  • 「全てのHTMLファイルをスキャンして」のような指示が使える
  • 用途に合わせて柔軟にスキャンをカスタマイズ
日本語で指示してもちゃんと動く！

⑩ YOLOモードで効率化

  • Ctrl+YでYOLOモードを有効にする
  • セキュリティ拡張は読み取り専用の操作なので安心
  • コードを変更せずにスムーズにスキャンを実行
読み取り専用だからコードが書き換わる心配なし！

⑪ 3ステップの分析フロー

  • まずTo-Doリストで監査範囲を定義する
  • 次にファイルを順番に分析していく
  • 最後に結果のサマリーを出力して完了
計画的にスキャンを進めるから見落としがない！
GitHub ActionsでのCI/CD統合

⑫ CI/CDへの統合

  • チーム開発ではGitHub Actionsでセキュリティ拡張をCI/CDに統合
  • 全員のコントリビューションを同じ基準でチェックできる
  • 「ローカルでスキャンし忘れた」という人的ミスを防止
チーム全員が同じセキュリティ基準をクリア！

⑬ GitHub Actionsの設定

  • 公式リポジトリからサンプルワークフローをコピーして簡単に設定
  • Workload Identity Federationで安全に認証
  • サービスアカウントキーを使わずに連携を実現 
# ワークフローをリポにコピー
cp security-workflow.yml \
  .github/workflows/

# Actions変数を設定
./setup-variables.sh
テンプレートをコピペするだけ！

⑭ PRでの自動セキュリティレビュー

  • 新しいPRが作成されると自動的にセキュリティ拡張がレビューを実行
  • AIがコードを解析して脆弱性を検出
  • 人的ミスを防止してセキュリティを担保
PRを出すだけで自動でセキュリティチェック！

⑮ 既存PRへのレビュー依頼

  • @gemini-cli/reviewコマンドでレビューを開始
  • 既存のPRにもセキュリティレビューを依頼できる
  • 過去のコードも遡ってチェック可能
コマンド1つでいつでもレビューを依頼！

⑯ セキュリティのシフトレフト

  • セキュリティを開発の初期段階に組み込む（Shift Left）
  • 問題が大きくなる前に発見・修正できる
  • 開発コストの削減と品質の向上を実現
早い段階で見つけるほど修正もラク！
