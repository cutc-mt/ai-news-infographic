ローカルでの使い方

⑦ ステップ1：拡張機能のインストール

  • まずGemini CLIのセキュリティ拡張機能をインストールする
  • プロジェクトディレクトリでGeminiを開く 
# セキュリティ拡張機能のインストール
gemini ext install security

# プロジェクトでGeminiを起動
cd my-project
gemini
インストールはコマンド1発で完了！

⑧ ステップ2：コマンドの実行

  • /securityコマンドでカスタムコマンド一覧を表示
  • リストから実行したいカスタムコマンドを選択
  • 自然言語でコマンドをカスタマイズ可能 
# セキュリティコマンドを呼び出し
> /security

# 自然言語でカスタマイズ
> Scan all my HTML files
「HTMLファイルを全部スキャンして」と日本語でもOK！

⑨ YOLOモードの活用

  • Ctrl+YでYOLOモードをオンにする
  • セキュリティ拡張機能は読み取り専用の操作しか行わない
  • コードを変更せずに安全にスキャンを実行可能
読み取り専用だからコードが書き換わる心配なし！

⑩ 分析の流れ：Todoリスト作成

  • まずTodoリストを作成して監査スコープを定義
  • どのファイルをスキャンするか、何をチェックするかを明確にする
  • 体系的に分析を進めるための計画を立てる
まずスコープを決めてから効率的にスキャン！

⑪ 分析の流れ：ファイルスキャン

  • スコープ内のファイルを順番に分析していく
  • タスクを完了するたびにTodoリストの項目をチェックオフ
  • 進捗を可視化しながら効率的にスキャンを実行
進捗が見えるから安心して任せられる！

⑫ 分析の流れ：結果サマリー

  • 最終的に発見事項のサマリーを生成
  • 脆弱性の重要度や影響範囲をまとめて表示
  • 修正提案も含めて報告される
最後にレポートが出力されるから見やすい！
GitHub ActionsでのCI/CD統合

⑬ なぜCI/CDに統合するのか

  • 個人のローカルスキャンは良いが、チーム全体で統一が必要
  • 複数のコントリビューターがいるリポジトリでは全員の貢献をチェック
  • 「ローカルでスキャンし忘れた」という人的ミスを防止
チーム全員が同じセキュリティ基準をクリア！

⑭ セットアップ手順

  • 公式のセキュリティ拡張機能リポジトリからワークフローの例をコピー
  • 自分のリポジトリにGitHub Actionsのワークフローとして配置
  • シェルスクリプトを実行してActions変数を作成 
# ワークフローをリポにコピー
cp security-workflow.yml \
  .github/workflows/

# Actions変数を設定
./setup-variables.sh
テンプレートをコピペするだけ！

⑮ Workload Identity Federation

  • GitHubからGoogle Cloudへの認証を設定

  • サービスアカウントキーを使わずに安全な連携を実現
キーを使わないからセキュア！

⑯ PRでの自動セキュリティレビュー
