Kilo Code 再構築版 VS Code 拡張機能

無料AIコーディングの新時代 - 並列エージェント・Worktree・MCP統合

2026年4月4日 | チャンネル: AICodeKing

①全面リニューアル: Kilo Codeが再構築

  • Kilo CodeのVS Code拡張機能が全面的に再構築された。旧版はVS Code内部に深く依存しており、他プラットフォームとの不整合が課題だった。
  • CLI、JetBrains、クラウド版、VS Code版が同じ基盤上で動く統合アーキテクチャに移行。
  • 機能のパリティが改善され、パフォーマンスも向上。保守が容易になり、新機能の展開スピードが大幅に高速化された。
単なるマーケティング的な「リライト」ではなく、実際のアーキテクチャ改善と評価できる内容！

②共有ポータブルコア: 統合アーキテクチャの実現

  • VS CodeをKilo CLIと同じポータブルコアの上に再構築。VS Codeはもはや「特別な実装」ではなく、同じシステム上のフロントエンドの一つに。
  • これにより、すべてのプラットフォームで一貫した機能が提供可能に。ある環境だけ機能が遅れるという問題が解消された。
  • 新機能の展開が全プラットフォーム同時に。メンテナンスの負荷も大幅に軽減。

③2026年4月2日: 正式版がローンチ

  • 2026年4月2日に "New Kilo for VS Code is live" が投稿され、ベータ版から正式なGA製品へと昇格。
  • 3月10日の再構築発表時は「プレビュー」だったが、4月の更新で実用的な欠落部分が埋まった。
  • 日常的なワークフローが明確になり、本格的な開発ツールとして使えるレベルに到達した。
「ベータだから様子見」と考えていた人にこそ、今こそ再チェックをおすすめ！

④並列処理の全面導入: パラレルツールコール

  • 並列ツールコールがサポートされ、1つの巨大なチャットスレッドで順次処理する必要がなくなった。
  • ファイルの読み込み、検索、コマンド実行がすべて並列で同時進行可能に。
  • 複数のエージェントを起動して、問題の異なる部分を同時に処理できる。
直列処理に比べてスループットが劇的に向上する！

⑤並列サブエージェント: 分業が可能に

  • エージェントAは機能の実装、エージェントBはテスト、エージェントCはレビュー、エージェントDは別アプローチの探索を同時に実行。
  • カスタムサブエージェントの定義も可能。チーム固有のワークフローに合わせてエージェントを構成できる。
  • これは「スループットを本当に変える」タイプの機能で、実用的な生産性向上に直結する。

⑥エージェントマネージャー: セッションを一元管理

  • エージェントマネージャーで、複数のライブセッションを1箇所で管理・切替・比較が可能に。
  • タブがカオスになるのを防ぎ、整理された状態を維持しながら複数のエージェントを同時に利用できる。
  • セッション間の比較や、作業の切り替えがスムーズに行える。
AIコーディングツールを本格的に使う人にとって最も重要な機能の一つ！

⑦Git Worktreeの統合: 作業を完全に分離

  • AIコーディングの最大のペインポイントは、異なる試行が互いに干渉してしまうこと。
  • Git Worktreeにより、各エージェントが独立したブランチやワークスペースで作業可能に。
  • すべてのエージェントが同じブランチに押し寄せる事態を回避し、安全に分離できる。
筆者が「絶対に好き」と言うほど推している機能。Worktreeは本当に便利！

⑧サイドバイサイド比較: モデルを並べて検証

  • 同じタスクを異なるモデルや戦略で実行し、出力結果を並べてクリーンに比較可能。
  • モデルの挙動はまだばらつきがあり、どのモデルが最適かは事前にわからないことが多い。この比較機能が非常に有用。
  • アーキテクチャに強いモデル、スピードに優れたモデル、コードレビューが得意なモデルなどを客観的に評価できる。

⑨インラインコードレビュー: 差分に直接コメント

  • チャットに巨大なフィードバック塊を出すのではなく、コードの差分に直接コメントを配置。
  • すべてのコメントはメインの会話にも要約して反映される。
  • 実際の開発者のコードレビューのやり方に非常に近い体験を提供。地味だけど実用的な機能。
小さく聞こえるけど、実際に使うと違いが歴然する機能！

⑩統一されたエージェント体験: 一貫性のあるワークフロー

  • 以前は断片的だったモード設定から、エージェント・セッション・レビューが一つの統合的なシステムに。
  • すべての機能が切り離された別々のものではなく、一つの一貫した体験として統合された。
  • 最悪のAIコーディングツールは「全部後付け」感がある。Kiloの新版は非常にまとまりがある。

⑪MCPマーケットプレイス: ネイティブサポート

  • MCPマーケットプレイスがネイティブでサポートされ、拡張機能の導入がスムーズに。
  • プロバイダー設定の直接編集、セッションのインポート、専用ターミナルなども追加。
  • CLIやクラウドとのセッション同期により、異なる環境間での作業継続がシームレスに。
ある場所で始めた作業を別の場所でそのまま続けられるのは本当に便利！

⑫無料ルート①: Kilo内蔵の無料モデル

  • Kiloにサインアップするだけで、内蔵プロバイダーから無料モデルにすぐアクセス可能。
  • 無料モデルには"free"ラベルが付いており、ドロップダウンから簡単に選択できる。
  • 最も簡単なパス。設定不要で、サインアップ直後から利用開始可能。

⑬無料ルート②: OpenRouterの無料モデル

  • OpenRouterをプロバイダーとして公式サポート。APIキーを拡張機能設定に入力するだけ。
  • 注意点: OpenRouterの設定で「Prompt Training」を許可する必要がある。無料モデルが表示されない場合はここをチェック。
  • 利用可能な無料モデルの例:
  • Qwen3 Coder Free
  • GLM 4.5 Air Free
  • DeepSeek R1 0528 Free
  • Kimi K2 Free
無料モデルが使えない場合はPrompt Training設定を確認！

⑭無料ルート③: NVIDIA NIMsによる開発者アクセス

  • NVIDIAのAPIカタログとサーバーレス開発者アクセスを利用。Kiloに公式ガイドがある。
  • 設定手順: NVIDIA APIキーを取得 → Kilo内のOpenAI互換プロバイダーでURLとキーを設定 → モデルを選択。
  • NVIDIAカタログ経由でKimi、GLM、MiniMaxなどのモデルが利用可能。開発・テスト用途で無料。
「無料」＝NVIDIAの現在の開発者プログラム条件下での無料。無期限の保証ではないので注意。

⑮無料オートコンプリート: Codestralを無料で利用

  • 日常的な利用で「無料」を感じるには、コード補完も無料であることが重要。
  • KiloのドキュメントでMistralのCodestralをオートコンプリートプロバイダーとして利用する方法が明記されている。
  • Codestralコード補完が完全無料。多くのツールはチャットは無料でも補完は有料だが、Kiloは両方無料にできる。
無料チャット＋無料補完のダブル無料はかなり珍しい！

⑯無料設定のベストプラクティス: おすすめの組み合わせ

  • まずKilo内蔵の無料モデルをドロップダウンから試す（最も簡単）。
  • 多様性が必要ならOpenRouterを接続して無料モデルをテスト。
  • NVIDIAカタログのモデルも試すならOpenAI互換フローで接続。
  • オートコンプリートはCodestralの無料ルートを活用。
すべての無料ルートが同じ再構築版のワークフロー内で使えるのが最大の強み！

⑰一貫性の強み: どのプロバイダーでも同じ体験

  • Kiloのモデル、OpenRouter、NVIDIAのどれを使っても、同じインターフェースで操作可能。
  • プロバイダーを切り替えても、エージェント・セッション・レビュー・Worktreeの設定はそのまま継続利用できる。
  • この一貫性こそが、Kiloの最も強力な特徴の一つ。
プロバイダーごとに別のUIを覚える必要がなく、学習コストが最小！
