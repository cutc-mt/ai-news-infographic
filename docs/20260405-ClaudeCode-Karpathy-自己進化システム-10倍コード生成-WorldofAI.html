LLM WikiがAIエージェントを自己成長型の知識ワーカーに変える

Claude Code Karpathy LLM Wiki 自己進化 Obsidian WorldofAI

①Karpathyの自己進化ナレッジシステム

Andrej KarpathyがAIモデルが自律的に動く自己進化型ナレッジシステムの構想を公開。人間がノートを書いたり知識を整理する代わりに、LLMが生データを読み込み、要約を作成し、構造を構築し、一貫性を維持しながら自己改善を続ける仕組み。

②LLMが代行する6つの機能

このシステムが実現するのは、生データの読み込み、要約の作成、構造の構築、一貫性の維持、質問への回答、自己改善の6つの機能。人間はアイデアを探索するのに長けているが、整理整頓は苦手。その「退屈な管理作業」をLLMがすべて引き受ける。

③Farza Pedia: 実際の事例

あるユーザーがこの構想を実装した「Farza Pedia」というシステム。日記、Apple Notes、メッセージなど約2,500件の個人データを投入すると、LLMが友人、アイデア、興味、創作インスピレーションを網羅する数百の構造化記事を持つ「個人版Wikipedia」を自動生成。

④エージェントのために構築されたシステム

重要なのは、このシステムは人間ではなくエージェントのために設計されているという点。すべてが構造化されたインターリンクファイルとして整理されているため、エージェントが簡単にナビゲートでき、文脈を引き出して実際のタスクに活用できる。過去のインスピレーションやアイデアを参照して、ランディングページのデザインを自動生成するようなことも可能。

⑤アイデアファイル: ソフトウェアからアイデアの共有へ

Karpathyはコードやアプリではなく、「アイデアファイル」という高レベルの設計図を公開。AIエージェントがこれを受け取り、ビルドしてワークフローに合わせてカスタマイズできる。Gistとして公開されており、誰でも自分のエージェントに渡して自己進化ナレッジシステムを構築可能。これは「ソフトウェアの共有」から「アイデアの共有」への大きな転換点。

⑥3層システム構造

LLM Wikiは3つのレイヤーで構成される。第1層は「Raw Sources」で生の記事、ノート、データを格納。第2層は「Wiki」でLLMが生成したマークダウンファイルがリンク付きで常に更新される。第3層は「Schema Rules」で、LLMに整理・更新・維持の方法を指示するルールセット。

⑦自己改善のループ

エージェントをWikiのindex.mdに接続するだけで自動ループが始まる。新しいソースの追加、質問、出力の生成を行うと、LLMが自動的にWikiを更新して改善し続ける。人間は探索が得意だが維持管理が苦手、LLMは逆にリンクや一貫性の維持などの地道な作業を得意とする。この補完関係がシステムの核心。

⑧Claude Codeが自己更新型ナレッジワーカーに

このLLM WikiをClaude Codeに接続すると、単なるコーディングアシスタントから自己更新型のコンテキスト認識ナレッジワーカーに進化する。構造化されたWikiを直接読み込み、ナビゲートし、推論できるため、複雑な質問に答えたりナレッジを使って出力を自動生成したりすることが可能に。

⑨Claude Codeのメモリ問題を解決

この仕組みはClaude Codeの最大の弱点である「メモリ問題」を根本的に解決する。これまでは文脈を忘れたり、矛盾した出力をしたりすることがあったが、LLM Wikiが常に整理された知識ベースとして機能するため、出力品質が大幅に向上する。

⑩セットアップ手順: Obsidianのインストール

Karpathyの推奨に従い、Obsidianをフロントエンドとして使用する。ObsidianはRawデータの可視化とWikiへのコンパイルの両方に役立つ。新しいVault（ローカルディレクトリ）を作成し、そこにすべてのRawファイルとLLM生成Wikiを格納する。

⑪Claude CodeでVaultを初期化

作成したVault内でClaude Codeのインスタンスを開く。VS Code内でClaude Codeを使うと、各要素の可視化と操作が容易になる。KarpathyのGistファイル（アイデアファイル）をコピーし、Claude Codeに貼り付けて実行する。

⑫2つのフォルダ構造

Claude Codeが「raw」と「wiki」の2つのフォルダを自動生成する。rawフォルダにはソースドキュメント、記事、ノート、画像がそのまま保存される（信頼できる情報源）。wikiフォルダにはLLMが生成したマークダウンファイルが格納され、要約、エンティティ、インターリンクされた概念ページが含まれる。

⑬実践例: フロントエンドデザインライブラリ

動画ではフロントエンドデザインに特化したVaultを構築。スクリーンショット、Figmaリンク、HTML/CSSスニペット、カラーやフォントのノート、デザインシステムなど、開発に役立つあらゆる素材をrawフォルダに追加。エージェントがこの内蔵メモリを参照するため、ハルシネーションを防げる。

⑭Obsidian Web Clipperでデータを追加

Obsidian Web Clipper拡張機能を使うと、ブラウザからワンクリックで新しいドキュメントやパッケージの情報をrawフォルダに追加できる。関連画像もローカルにダウンロードされるため、LLMが画像を認識可能。その後、Claude Codeに「新しいrawファイルをwikiにコンパイルして」と指示するだけで、要約の作成から概念の抽出、index.mdの更新まで自動で完了。

⑮実践デモ: CRMダッシュボードの生成

実際にCRMダッシュボードの作成でテストを実施。index.mdを参照させることで、提供したチャートやUIコンポーネントがすべて正しく実装された。Shadcnのチャートなど、rawフォルダに保存した外部ライブラリのコンポーネントも、クロスリンク参照によって自動的に活用される。

⑯トークン消費と時間の節約

この仕組みの大きなメリットはトークン消費の大幅な節約。必要なコンテンツだけをスクレイピングしてLLMに渡すため、無駄なプロンプトが減る。また、何を使うべきか、いつ使うべきかをエージェントが自動で判断できるため、開発時間も大幅に短縮される。

⑰自己修復機能: lintコマンドとヘルスチェック

最も重要なのが自己改善の仕組み。lintコマンドやヘルスチェックを実行すると、LLMが自分の過去の成果物を読み返し、矛盾や古い情報、欠落しているリンク、新しいつながりを発見して自動修正する。手動での編集は一切不要で、知識ベースが使えば使うほど賢くなる。

⑱まとめ: AIエージェントの未来

データを多く投入し、こまめにlintすればするほど、Claude Codeや他のAIコーディングエージェントの精度が向上する。自己進化するナレッジベースは、AIエージェントを単なるツールから「成長し続けるパートナー」へと変貌させる。Karpathyの構想は、AIコーディングのパラダイムを根本から変える可能性を秘めている。
