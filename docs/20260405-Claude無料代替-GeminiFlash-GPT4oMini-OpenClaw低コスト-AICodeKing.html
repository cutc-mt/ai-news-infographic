① AnthropicがOpenClawをサブスクリプション対象外に：2026年4月4日

2026年4月4日付で、Anthropicが正式にClaudeのサブスクリプション制限がサードパーティのハーネス（OpenClawなど）をカバーしなくなったことを発表しました。これにより、OpenClaw経由でClaudeを使い続けるには、サブスクリプションとは別の従量課金制の「追加利用」に切り替える必要があります。月額を払っていても、OpenClawの利用は別途請求される形になりました。

② サブスクリプション終了のシグナル：エージェント用途には向かない

この動きは単なる一時的な変更ではありません。本格的なエージェントワークフローにおいて、定額月額制は明らかに崩壊しつつあります。まずスロットルがかかり、次に共有キャップ、一時的なプロモ、追加利用、割引バンドルという段階を経て、最終的には「あなたのワークフローはもう対象外です」と通告される。それが今回の出来事の本質です。

③ ClaudeとClaude Codeは使用量を共有：5時間ごとにリセット

Anthropicのヘルプドキュメントによると、ClaudeとClaude Codeは同じ使用量プールを共有しています。この制限は5時間ごとにリセットされ、上限に達すると標準API料金に切り替えることが推奨されています。つまり、公式ドキュメント自体が「サブスクリプションではなくAPI課金へ移行せよ」と示唆している状態です。

④ API課金の時代が再来：なぜ今起こっているのか

本動画の核心的なメッセージは、APIベースの料金設定が戻ってきたということです。エージェントツールは運用コストが高く、ユーザーがカジュアルなチャットから本格的な自律タスクへ移行すると、定額制はプロバイダーにとって持続不可能になります。より多くの企業が同じ方向に進むと予想されています。

⑤ マインドセットの転換：サブスクリプションから選択肢とコスト管理へ

今後必要なのは、「今週どのサブスクリプションが一番太っ腹か」で全体を構築するのをやめることです。代わりに、選択肢の多さ（オプショナリティ）、ルーティング、コスト管理を軸にワークフローを設計すべきです。特定の企業の寛大さに依存する構造は、いつでも崩れる危険性があります。

⑥ Kilopass：API課金をより賢く管理する仕組み

KilopassはAPIベースの料金をより賢く扱うための仕組みです。支払いはクレジットに変換され、有料クレジットには有効期限がありません。さらにボーナスクレジットも付与されます。一社の寛大さに頼るのではなく、利用が透明で持ち運び可能な仕組みを構築できるのが大きな強みです。

⑦ Kilogateway：モデルを自由に切り替えてコストを最適化

Kilogatewayを使うと、すべてのタスクで高額な最先端モデルを使う必要がなくなります。ルーティンなOpenClaw作業（ログの要約、簡単なリファクタリング、デバッグ、ボイラープレート作成など）には、安価または無料のモデルを使用可能です。高価なAnthropicトークンを毎ステップで消費する必要がなくなります。

⑧ Qwen 3.6 Plus：Kilo経由で完全無料で利用可能

現在、Kiloを通じてQwen 3.6 Plusが無料で利用できます。日常的なエージェントループの処理には、必ずしもClaude OpusやGPT-5は不要です。無料または安価なモデルを通常業務に使い、本当に難易度の高いタスクにだけプレミアムモデルを投入する戦略が可能です。これは非常に画期的なことです。

⑨ Grok Code Fast：スピード重視の低コストオプション

高速かつ安価なオプションとして、Grok Code FastもKilo経由で利用可能です。スピードを犠牲にせず、コストを大幅に抑えられるため、素早い反復が必要な作業に最適です。エージェントのループ処理において、体感速度とコスト効率のバランスを取るための強力な選択肢となります。

⑩ Kilocloud：ホスト型OpenClawでインフラ管理から解放される

Kilocloudはホスト型のOpenClawサービスです。Mac Miniの準備やVPS管理、Dockerのセットアップ、更新対応、不意のクラッシュなど、面倒なインフラ作業をすべて不要にします。Kilogatewayスタックを利用し、膨大なモデルカタログにアクセスできて、エージェントの利用に集中できます。

⑪ Bring Your Own Key：Z AIやGLMを組み合わせてさらに節約

さらにコストを抑えたい場合は、KiloのBYOK（Bring Your Own Key）フローを使って、Z AIやGLMの設定をKilocloudに組み合わせることができます。自分のAPIキーを持ち込むことで、外部プロバイダーの料金体系を活かしつつ、Kilocloudの便利なホスト環境をそのまま利用可能です。

⑫ GLM Coding Planの圧倒的なプロンプト枠：Claudeと比較

Z AIのGLM Coding Planは驚異的なヘッドルームを持ちます。Lightプランで5時間あたり約80プロンプト、Proで約400、Maxで約1,600。一方、AnthropicのClaude CodeはProが5時間あたり約10〜40プロンプト、Max 5xでも約50〜200。GLMの方がはるかに多くのプロンプトを処理でき、料金も大幅に安くなります。

⑬ コスト比較の衝撃：GLM Coding Plan vs Claude Pro/Max

GLMのMaxティア（約1,600プロンプト／5時間）は、Claude Max 5x（約50〜200プロンプト／5時間）を大幅に上回る処理能力を持ちながら、料金はずっと安価です。KilocloudをGLM Coding Planスタイルのセットアップにルーティングすれば、ホスト型OpenClawでありながら、圧倒的に有利な経済性を実現できます。

⑭ Anthropicのモデル自体は優秀：問題はアクセスと価格

重要な前提として、Anthropicのモデル自体は依然として非常に優秀です。SonnetもOpusも素晴らしい性能を持っています。しかし、アクセスの安定性、価格の妥当性、利用の信頼性が重要です。サブスクリプションから第三者アクセスをいつでも削除できる企業に、ワークフローの全体を依存するのは危険だと言えます。

⑮ 推奨されるスマートな構成：5つのステップで最適化

推奨されるセットアップは以下の通りです。①Kilogatewayでモデルを自由に切り替え、②Qwen 3.6 Plusなどの無料モデルを日常業務に使い、③Grok Code Fastなどの高速低コストモデルでスピードを確保し、④本当に難しいタスクにだけプレミアムモデルを使い、⑤自前ホストが面倒ならKilocloudを利用する。この5ステップで最適化できます。

⑯ GLM Coding PlanをKiloに統合：さらなる価値を絞り出す

Kilocloudのホスト環境にGLM Coding Planを接続すると、ホスト型OpenClawでありながら圧倒的なプロンプトヘッドルームと低コストなモデルアクセスを獲得できます。サブスクリプションの当て外れに悩まされることなく、安定した経済性で作業を続けられるのが最大のメリットです。

⑰ 市場の明確な方向性：サブスクリプションからAPI課金への移行

今回のOpenClaw問題は厄介ですが、同時に市場の向かう先を明確に示しています。本格的なAIエージェント利用におけるサブスクリプションは段階的に廃止され、API課金が主流になる方向です。Kilopass、Kilogateway、Kilocloudの組み合わせは、コスト・モデル・ワークフローをすべて自分で管理できる強力な選択肢となります。

⑱ まとめ：Claudeに頼らない新しいエージェント運用の形

結論として、Kilopass + Kilogateway + Kilocloudの3本柱が、OpenClaw利用における最強の無料・低コスト代替策です。無料モデルを日常業務に使い、高速低コストモデルでスピードを確保し、GLM Coding Planを組み合わせれば、Claude Max/Proに頼らない強靭なエージェントワークフローが構築できます。Anthropicの寛大さを祈るより、自分でコストとモデルを管理する方が賢明です。
