Gemma 4 + Ollama + Hermes/OpenClaw

ローカルで動くAIエージェントがついに実用レベルに／完全無料で構築可能

Gemma 4 Ollama Hermes Agent OpenClaw NVIDIA NIM AICodeKing

① GoogleがGemma 4をリリース：ローカルAIのゲームチェンジャー

Googleが新オープンモデルファミリー「Gemma 4」をリリースしました。Gemini 3と同じ研究と技術から構築されており、自分のハードウェアで動かせる最も高性能なモデルファミリーだとGoogleは発表しています。Apache 2.0ライセンスで提供されており、商用利用も含めて自由に活用できるのが大きな特徴です。

② 4つのサイズ展開：用途に合わせて選べるラインナップ

Gemma 4は4つのサイズで展開されています。E2BとE4Bはエッジデバイス向けの軽量モデル、26BはMixture of Experts（専門家の混合）モデル、そして31BはDense（密結合）モデルです。26Bモデルは推論時に約3.8Bのパラメータしか活性化しないため、ローカル実行に最も適したバランスの良さを実現しています。

③ リーダーボードで大躍進：20倍のサイズ差を逆転

Gemma 4 31Bは現在、Arena AIのテキストリーダーボードでオープンモデル3位にランクインしています。26Bモデルも6位に入っており、自らの20倍ものサイズがあるモデルを上回るスコアを叩き出しています。ベンチマークがすべてではないものの、この数字は非常にインパクトのある結果です。

④ エージェント用途に必要な機能をすべて網羅

Gemma 4は単なるチャットボットではありません。高度な推論、関数呼び出し（Function Calling）、構造化JSON出力、ネイティブのシステム指示、長いコンテキストウィンドウ、マルチモーダル入力、140以上の言語対応をサポートしています。ローカルエージェントのワークフローに直接組み込める実用的な機能が揃っています。

⑤ 26B MoEモデルがおすすめのスイートスポット

多くのローカルユーザーにとって26B MoEモデルが最もおすすめです。Mixture of Expertsアーキテクチャのおかげで、推論時には約38億パラメータしか使用しません。そのため、本格的なモデルの性能を保ちながらも、巨大なGPUがなくても現実的にローカルで動かすことができます。コーディングや推論タスクでも十分に実用的なレベルです。

⑥ 31B Denseモデル：最高品質を求める人向け

最高の出力品質を求める場合は31B Denseモデルが最適です。すべてのパラメータを密に使用するため、26Bよりもさらに高い性能を発揮します。ただし、より多くのメモリと計算リソースが必要になるため、高性能なGPU環境がある人向けとなります。ハイエンド環境を持っている人には強力な選択肢です。

⑦ E2B・E4Bエッジモデル：軽量デバイスでも動く

E2BとE4Bはエッジデバイスやメモリの少ない環境向けに設計された軽量モデルです。ラズベリーパイのような小型デバイスや、古いノートPCでも動作するため、IoTや組み込み用途にも活用できます。フル機能のエージェント用途にはやや力不足ですが、基本的なチャットやテキスト処理には十分な能力を持ちます。

⑧ Ollamaですぐにローカル実行：コマンド1つで起動

OllamaはすでにGemma 4に対応しており、簡単なコマンドで即座にローカル実行が可能です。具体的には「ollama run gemma4:2b」「ollama run gemma4:4b」「ollama run gemma4:26b」「ollama run gemma4:31b」と入力するだけで各サイズのモデルを起動できます。ハードウェアが弱い場合は小さいモデルから試すのが賢明です。

⑨ Hermes Agent：Gemma 4を本格的なエージェントに変身させる

Hermes Agentは単なるチャットUIではなく、本格的なエージェントシェルです。ツールの操作、カスタムプロバイダー、MCPサーバー、メモリシステムなどに対応しており、Gemma 4を「脳」として活用することで、ローカル環境で完全なエージェントワークフローを構築できます。プライバシーを重視する用途にも最適です。

⑩ Hermes Agentのセットアップ手順：3ステップで完了

設定は驚くほどシンプルです。まず「ollama pull gemma4:26b」でモデルを取得し、次にOllamaをコンテキスト長32768で起動します（低コンテキストはエージェントの動作を著しく低下させるため重要）。最後にHermes Agentでカスタムエンドポイント「http://localhost:11434/v1」を指定し、モデル名を入力すれば完了です。

⑪ コンテキスト長がエージェントの命：小さいと「バカ」に見える

ローカルエージェントの性能においてコンテキストウィンドウの長さは極めて重要です。コンテキストが小さすぎると、エージェントはツールのスキーマや以前の指示を忘れてしまい、モデル本来の能力よりも遥かに劣った結果になってしまいます。Ollamaの起動時に必ず十分なコンテキスト長を設定することが、体感品質を左右します。

⑫ OpenClaw：オープンソースの個人AIアシスタント

OpenClawは現在最も注目されているオープンソースの個人AIアシスタントプロジェクトの一つです。ローカルモデルにもクラウドモデルにも接続でき、エージェントタスクの実行、ツールの利用、実用的な作業を自動化する能力を備えています。テキスト生成だけでなく、実際に役立つ作業をこなせるのが特徴です。

⑬ OpenClawのネイティブOllama対応：ツール呼び出しが安定

OpenClawはOllamaを単なるOpenAIクローンとして扱わず、OllamaのネイティブAPIを直接サポートしています。これにより、ストリーミングが改善され、ツール呼び出しが大幅に安定します。V1互換URLではなく、プレーンなOllamaベースURL（通常はhttp://127.0.0.1:11434）を使うことで、この恩恵を得られます。

⑭ OpenClawのセットアップ：自動検出で簡単接続

OpenClawのセットアップもシンプルです。OllamaをインストールしてGemma 4を取得した後、「openclaw onboard」を実行するだけです。プロバイダーでOllamaを選び、ベースURLを指定すると、自動的にローカルモデルを検出して一覧に表示してくれます。Gemma 4がすぐにデフォルトモデルとして選択可能になります。

⑮ NVIDIA NIM：GPUがなくても31Bを無料で試せる

ローカルでGemma 4を動かすハードウェアがない場合でも、NVIDIA NIMを通じてGemma 4 31Bを無料で試すことができます。プロトタイピング用途であれば、ホスト型のNIM APIを無料で利用可能です。OpenAI互換のChat Completionsエンドポイントを使うため、対応ツールやアプリからの接続も容易です。

⑯ 2つの道：ローカルかクラウドか、用途で選べる

Gemma 4の活用には2つのルートがあります。1つ目はハードウェアがある場合、Ollamaでローカルに実行してHermes AgentやOpenClawと組み合わせる方法。2つ目はハードウェアがない場合、NVIDIA NIMの無料APIで31Bを試す方法です。後者はローカルではありませんが、GPUなしで性能を体験できる便利な選択肢です。

⑰ Apache 2.0ライセンス：商用利用も自由自在

Gemma 4はApache 2.0ライセンスで提供されています。これは商用利用を含めて自由に使えることを意味しており、これまでのGemmaシリーズで懸念されていたライセンスの制約が大幅に緩和されました。オープンモデルを業務に組み込みたい企業や開発者にとって、非常に実用的な選択肢となっています。

⑱ まとめ：Gemma 4はローカルAIエージェントの新時代を切り開く

Gemma 4はGoogleが初めてサイズ、性能、エージェント対応、ローカル実用性のすべてを高いレベルで両立したリリースです。Ollama + Hermes AgentまたはOpenClawと組み合わせることで、単なるベンチマークモデルから「実際に使えるローカルAIスタック」へと進化します。ローカルでエージェントを動かしたい人にとって、今最も注目すべきオープンモデルです。
