①AIとの会話が消えている現実

半年間で1950万トークンものAIとの会話が蒸発している。Claude、ChatGPT、Cursorを全部合わせて計算するとこの膨大な量になる。毎日何時間もAIと話している人にとって、これは決して大げさな数字ではない。

②設計議論も決定の経緯も蒸発

消えているのは単なる雑談だけではない。設計議論、なぜこの実装にしたかの決定、レビューの経緯など、プロジェクトにとって重要な情報がすべてチャット履歴と一緒に消えてしまう。半年前に何を聞いたかなんてもう全く覚えていない状態だ。

③ChatGPTのメモリ機能では不十分

ChatGPTにもメモリ機能はあるが、AIが何を覚えるかを勝手に決める。ユーザーがコントロールできない。「この人はPostgreSQLが好き」という断片的な情報は残すが、「なぜPostgreSQLを選んだか」の理由は捨ててしまう。文脈がないと記憶の価値が激減する。

④MemPalace - 記憶の宮殿アーキテクチャ

MemPalaceは古代ギリシャで生まれた記憶術「場所法（Method of Loci）」をソフトウェアに実装したシステム。2500年前から使われている、場所と情報を紐付けることで大量の情報を引き出す技術を、AIのメモリ管理に応用している。

⑤Wing/Room/Drawerの3層構造

最上位にWing（翼）があり、人物やプロジェクトごとに分割される。Wingの中にはRoom（部屋）があり、トピックごとに分かれている（認証機能の部屋、デプロイ設定の部屋など）。Roomの中にはDrawer（引き出し）とCloset（クローゼット）があり、Drawerに元のテキスト、Closetに圧縮されたサマリーが格納される。

⑥HallとTunnelで横断的にアクセス

Hall（廊下）はメモリタイプ別に横断的にアクセスできる仕組み。さらにTunnel（トンネル）という概念もあり、異なるWing間をつなぐ通路として機能する。プロジェクトAとプロジェクトBで共通する話題があれば、Tunnelで結びつけることができる。

⑦構造化で検索精度が34ポイント向上

全部フラットに保存して検索すると精度は60.9%。しかしWingとRoomを指定して検索すると94.8%に跳ね上がる。34ポイントの改善は、構造化によってどこを探すかを絞り込めるためノイズが減るからだ。2階の書斎の左の棚と指定した方が見つかるのと同じ理屈だ。

⑧AAK圧縮言語 - 30倍圧縮を実現

AAKはMemPalace独自の圧縮フォーマット。膨大な会話を約30分の1に圧縮する。例えば5人分の役割と経歴が、わずか数行のAAK表現に収まる。しかも人間が読んでも意味が分かるという特徴がある。

⑨全LLM対応 - ベンダーロックインなし

AAKはClaude、GPT、Gemini、Llama、Mistralなど、すべてのLLMがデコーダーなしで直接読める。ファインチューニングも不要。テキストとして渡すだけでLLMが解釈する。今日はClaude、明日はGPT、来週はローカルのLlamaという使い方でも、同じメモリを参照できる。

⑩4層スタック構造（L0〜L3）

MemPalaceは4層のスタック構造を持っている。起動時はわずか170トークンでウェイクアップする。L0がアイデンティティ（誰であるか）、L1がクリティカル（絶対に忘れてはいけないこと）、これらだけが常時ロードされる。L2が最近のトピック、L3が全履歴からの検索で、必要な時だけオンデマンドで読み込む。

⑪L0とL1 - 最小限のコンテキスト

L0は「あなたはこういう人です」というアイデンティティ情報を約数トークンで保持。L1は仕事の前提条件（例：本番はAWSのECSで、PRレビュー必須など）のような絶対に忘れてはいけない事実を格納。これだけが常にロードされ、毎回説明し直す必要がなくなる。

⑫コンテキストウィンドウの95%を会話に使える

L2とL3はオンデマンドで読み込まれる。「認証の話をして」と言われたら認証のRoomから情報を取得し、「半年前のあの決定は？」と聞かれたらL3でサーチする。この仕組みにより、コンテキストウィンドウの95%以上を実際の会話に使える。メモリを持つこととコンテキストを圧迫することは別物だ。

⑬SQLiteでナレッジグラフを無料実装

通常ナレッジグラフにはNeo4jなどのグラフデータベースが必要だが、MemPalaceはSQLiteで完全ローカル・無料で同等機能を実装する。さらに時間軸にも対応しており、「2026年1月時点でこのプロジェクトはどういう状態だったか」という過去の特定時点の状態を再現できる。

⑭矛盾検出でAIの嘘を防ぐ

「総連がマイグレーションを完了した」と入力すると、「マヤが担当として登録されています。総連ではありません」と自動で警告が出る。事実関係のダブルチェックを自動で行い、矛盾には段階（ちょっと怪しい〜明確に矛盾）がある。チーム作業で誰かが間違った情報を入れても気づける。

⑮MCP統合で19個のツールを提供

MemPalaceはMCPサーバーとして動作し、19個のツールを提供する。代表的なものは、status（状態確認）、search（検索）、KG query（ナレッジグラフ検索）、diary write（セッション記録）など。推奨プロトコルは「起動→status → 応答前にsearch → セッション後にdiary write」というシンプルなサイクルだ。

⑯ベンチマーク結果 - ローカルで96.6%

標準的な評価指標MemEvalualeのR@5（上位5件に正解が含まれる割合）で完全ローカルで96.6%を達成。APIを一切使わずにこの数字。ハイブリッドモードでLLMを1回呼ぶと100%に到達する。競合のMem0（約85%・月額19〜249ドル）やZep（約85%・月額25ドル）を、無料で上回る性能だ。

1950万トークンをLLMに毎回渡すのは物理的に不可能。LLMでサマリーを作る方法だと年間約470ドル以上かかる。MemPalaceで同等のことをすると1日5回検索の想定で年間わずか10ドル。ローカルのみなら完全に0ドル。MITライセンスで商用利用も改変も自由だ。

⑱4コマンドで導入完了

導入は4ステップで完了する。pip install mempalaceでインストール、mempalace initで初期化、mempalace mineでチャット履歴を読み込み、mempalace searchで検索。Claude、ChatGPT、Claude Code、SlackのJSON、プレーンテキストなど主要フォーマットに対応。mineコマンドが自動でAAK圧縮も行う。
