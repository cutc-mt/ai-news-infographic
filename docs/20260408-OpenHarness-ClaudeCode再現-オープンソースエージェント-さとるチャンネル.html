OpenHarness - Claude Codeの97%を3%のコードで再現

香港大学HKUDSが開発したオープンソースAIエージェントフレームワーク

2026年4月8日 | MIT License

① OpenHarnessとは

  • 香港大学データサイエンス研究室（HKUDS）が公開したオープンソースのエージェントフレームワーク
  • MITライセンスで完全無料。ソースコードも全部見ることができる
  • 普通のLLMをClaude Codeのような自立型エージェントに変換できる
  • Anthropicに依存せず、任意のモデルでエージェント構築が可能
ハーネス（Harness）とは馬につける馬具のこと。LLMに「手足」と「記憶」と「ルール」を与えるインフラなんです！

② LLMの課題と解決アプローチ

  • LLMはそのままだと賢いけど何もできない存在
  • ファイルが読めない、コマンドを実行できない、外部APIも叩けない
  • OpenHarnessはLLMに3つの要素を付与する：
手足（Tools） 行動能力
記憶（Memory） メモリ管理
ルール（Rules） 権限制御
脳みそはあるけど体がない状態 → OpenHarnessがその体を提供する！

③ 中核機能①：エージェントループ

  • 思考→実行→結果受信→次の判断のサイクルを自動で繰り返す
  • 例：「このバグ直して」と頼むと → コードを読む → 怪しい箇所を特定 → 修正 → テスト実行 → 報告、まで全部自動
  • 指数バックオフ：APIエラー時に1秒→2秒→4秒と待ち時間を増やしてリトライ
  • ストリーミング通信に対応。結果が来るたびにリアルタイムで処理を進められる
人間が毎回指示を出さなくても、自分で判断して作業を進めるのがポイント！

④ 中核機能②：コスト追跡

  • APIの利用量とコストを自動で追跡してくれる
  • 課金爆発を防ぐ仕組みが組み込まれている
  • リアルタイムでトークン消費量をモニタリング可能
エージェントが自律的に動く分、コスト管理は超重要。安心して使える！

⑤ 中核機能③：43以上のツールキット

  • 43以上のツールが最初から組み込まれている
  • ファイルの読み書き、コードの実行、Web検索など
  • MCP対応（Model Context Protocol）で外部サービス連携を標準化
  • Gitの操作、Slack連携、GitHub連携なども対応
  • オンデマンドローディング：必要な時だけツールを読み込むのでメモリを節約
MCP = Model Context Protocol。SlackやGitHubなどのAPIを統一的に扱える仕組みです！

⑥ 中核機能④：2種類のメモリ

  • ① CLAUDE.mdファイル：プロジェクトのルートに置くと自動で読み込まれる（本家Claude Codeと同じ仕組み）
  • ② MEMORY.mdファイル：セッションをまたいで保存される永続メモリー
  • 例：「DB接続処理がある」「このプロジェクトの構造」などを記録して、次回から調べ直さずに済む
  • コンテキスト窓で自動要約。長くなりすぎたら自動でまとめる
メモリがないと毎回0からやり直しに！人間のメモ帳みたいな役割です

⑦ 中核機能⑤：ガバナンス（権限制御）

  • 3段階の権限モード：許可・確認・拒否を操作ごとに設定可能
  • ファイル読むのは「許可」、シェルコマンドは「確認」が必要
  • rm -rfなどの危険な操作は「拒否」に設定可能
  • 特定のパス（例：/etc）には触れないというルールも設定できる
AIが暴走したら怖い！システムファイルを守れるのは安心ですね

⑧ プレ／ポストツールフック

  • Pre-tool Hook：ツールを呼ぶ前にカスタム処理を挟める
  • Post-tool Hook：ツール実行後にカスタム処理を挟める
  • 例：git push前にメインブランチかどうか確認するチェックを追加
  • どのファイルを変更したかを全て記録する監査ログにも活用できる
企業での利用時に監査が必要なケースにも対応！安全性の設計が行き届いています

⑨ マルチエージェント調整

  • メインエージェントが必要に応じてサブエージェントを自動生成
  • 大きなタスクを専門家に分担。例：フロントエンドはA、バックエンドはB、セキュリティはC
  • 依存関係がなければ並列で同時実行可能
  • ツールレベルではなくエージェントレベルの並列分散（Claude Codeより上位の機能）
自分で部下を作るみたい！会社の組織のようにタスクを分担します

⑩ チームレジストリー

  • 各エージェントの役割・タスク・進捗を一元管理する仕組み
  • バックグラウンドで動いているタスクも追跡可能
  • プロジェクトのカンバンボードのように、誰が何をやってどこまで進んでいるかが一目でわかる
  • 1人で1時間かかる作業も3人で分担すれば20分で終わるという発想
複雑なプロジェクトでも破綻しにくい！マネジメント機能が充実しています

⑪ マルチモデル対応（2つの互換レイヤー）

  • レイヤー①：Anthropic互換 → Claude、GLM、MiniMax、Kimiなどが利用可能
  • レイヤー②：OpenAI互換 → GPT-4.5、o1、オープンソースモデルなどが利用可能
  • モデルを自由に切り替えて、用途に合わせて最適なものを選べる
  • 安いモデルでテスト → 本番は高性能モデル、という使い方も可能
中国のGLMやKimiなどのモデルでもエージェント構築ができるのが特徴です！

⑫ GitHub Copilot OAuth対応

  • GitHub CopilotのOAuth認証に対応している
  • デバイスフローで認証すれば、Copilotの契約でエージェントを動かせる可能性がある
  • 既存の開発環境とシームレスに統合しやすい設計
既にCopilotを契約しているなら、追加費用なしでエージェントが使えるかも！？

⑬ 技術スタック

  • ベース言語：Python 3.10以上が必要
  • ターミナルUI：React + Inkライブラリで構築
  • Inkを使うとReactのコンポーネントでターミナルUIを描画できる
  • 114件のテストが付属。品質もしっかり担保
ターミナルアプリなのにReactでUI構築！面白い技術アプローチです

⑭ 出力形式

  • テキスト：標準的なターミナル出力
  • JSON：構造化データとして他のシステムと連携しやすい
  • ストリーミングJSON：リアルタイムで結果を受け取れる
  • CI/CDパイプラインへの組み込みや、別アプリからの呼び出しにも最適
JSON出力があるから自動化パイプラインに組み込みやすい！

⑮ インストールと使い方

  • たった1行のコマンドでインストール可能
  • インストールスクリプトが依存関係を全て自動解決
  • 5分もあれば手元で動かせる
  • GitHubで「OpenHarness」と検索すれば見つかる 
# インストール例
curl -fsSL https://openharness.dev/install | bash
# または pip で
pip install openharness
試すハードルが低いのが魅力！すぐに始められます

⑯ MITライセンスの強み

  • 商用利用も含めて自由に改造・再配布が可能
  • 企業が自社向けにカスタマイズして使うことが想定されている
  • ソースコードが全て公開されているので、セキュリティ監査も可能
  • コミュニティでの共有と発展が期待できる
むしろ企業利用が想定されている！安心して導入できます

⑰ 学習教材としての価値

  • ブラックボックスだったエージェントの内部構造がコードレベルで全部見える
  • 「エージェントループってこう動くのか」という仕組みを理解できる
  • 「権限管理ってこう実装するのか」という設計パターンを学べる
  • 学んで改造して、自分だけのエージェントを作れる
Claude Codeの仕組みを学ぶ最高の教材！コードを読むだけで設計思想がわかります

⑱ まとめ：OpenHarnessの全体像

  • エージェントループ：思考→実行→判断の自動反復
  • 43+ツールキット：MCP対応のオンデマンドローディング
  • 2種類のメモリ：CLAUDE.md + MEMORY.mdで文脈を維持
  • ガバナンス：3段階の権限制御 + プレ/ポストフック
  • マルチエージェント：自律的なサブエージェント生成と並列実行
  • マルチモデル：Anthropic + OpenAI互換で自由にモデル選択
学んで改造して共有する。オープンソースの良さが詰まったフレームワークです！エージェント開発はこれから盛り上がります！
