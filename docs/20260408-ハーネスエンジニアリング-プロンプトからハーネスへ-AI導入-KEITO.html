AI活用の鍵がハーネスエンジニアリングへ

ハーネスエンジニアリング AI導入 KEITO Claude Code

①AI活用の3つの進化ステージ：プロンプトからハーネスへ

生成AIの活用では、これまで「プロンプトエンジニアリング」→「コンテキストエンジニアリング」→「ハーネスエンジニアリング」という3つの進化のステージがあった。プロンプトはAIへの指示の書き方、コンテキストは背景情報の設計、そしてハーネスはそれら全てを含む包括的な仕組み作りを指す。ハーネスを覚えたからといってプロンプトやコンテキストが不要になるわけではなく、全てが包含される関係にある。

②プロンプトエンジニアリング：AIへの「セリフ」を工夫する

プロンプトエンジニアリングとは、AIへの頼み方を工夫する手法だ。具体的には役割を明確に指定する、出力形式を定義する、手順を示す、例題を添えるといった基本的なルールが含まれる。例題に沿った形で生成させるFew-shotプロンプティングなどが代表的で、今でも非常に重要な基礎的なスキルである。この要素はハーネスの中にもしっかりと組み込まれている。

③コンテキストエンジニアリング：AIに渡す「事前共有資料」を整える

コンテキストエンジニアリングは、AIに渡す情報を設計していく考え方だ。背景情報としてプロジェクトの前提や目的を伝え、守るべきルールや制約を設定し、RAGによる参照データやドキュメントを準備する。また、手順書やテンプレート、最近流行っているスキルなども事前共有資料として組み込む。直近でも非常に重要視されている概念である。

④ハーネスエンジニアリングとは：AIモデル以外の「全て」

ハーネスエンジニアリングは、AIモデル以外の全ての道具と環境を整えるという新しい考え方だ。名前の由来は馬のハーネス（引き綱）で、AIの力を制御して信頼性の高い成果に変えるイメージ。プロンプトやコンテキストを含め、外部ツールとの接続、セキュリティルール、業務遂行のワークフローなど、AIを取り巻く全ての仕組みをハーネスと呼ぶ。

⑤なぜハーネスが大事になったのか：指示文だけではもう足りない

Claude CodeやOpen Crew、CodexなどのAIエージェントツールが実務を遂行するようになってきたことで、単なる指示文や背景情報だけでは業務と結びつかなくなった。最適なツールと連携したり、業務上のルールを与えたり、セキュリティリスクを管理したりと、仕組み全体を整える必要が生まれた。これがハーネスエンジニアリングという概念が誕生した背景である。

⑥ハーネスの3つの柱：ルール設計・ツール接続・オーケストレーション

ハーネスは大きく3つの柱で構成される。1つ目はルール設計で、システムプロンプトやCLAUDE.mdに書くようなAIの動きの定義。2つ目は外部ツールとの接続で、MCPやAPIを使って情報整理や業務遂行を実現する。3つ目はオーケストレーションで、ワークフローを組み立て、どういう順序で仕事を実行するかを定義する。これら3つを整えるのがハーネスの基本だ。

⑦細かな要素もハーネスの一部：ガードレールとコンテキスト管理

ハーネスにはさらに細かな要素も含まれる。入出力のフィルターとしてのガードレールを設定して、不適切な出力を防ぐ。コンテキスト管理のためのツールを導入し、スキルやMCPを活用する。実行前後のチェックやテスト自動処理のためのフック機能なども組み込む。これらを全て統合することで、高品質で安定した出力結果を再現性高く得られる仕組みが完成する。

⑧AIスロップ問題の解決策：不安定な出力を再現性の高いものへ

ハーネスの概念が求められた背景には「AIスロップ」問題がある。AIの出力が不安定で低品質なものになることが多々あり、同じ指示を出しても毎回違う結果が返ってくる課題だ。これを解決するために、ハーネス全体で制御と検証の仕組みを整えることで、出力の品質を安定させ、再現性の高い成果を得ることができるようになる。

⑨ハーネスの組み立て方ステップ1：ルールの設計

ハーネスを組み立てる最初のステップは、ルールをしっかりと設計することだ。例えば動画編集のAI活用を組む場合、「ジェットカットをして0.1秒の無音も許さない」「テロップを入れて重要な箇所は赤色にする」「赤色のテロップには効果音を入れる」「背景にBGMを入れる」など、具体的なルールを定義していく。このルール設計がハーネスの土台となる。

⑩ハーネスの組み立て方ステップ2：必要なツールとの接続

ルールを設計したら、次はそのルール通りに業務を遂行するために必要なツールと接続する。動画編集の例では、Remotion（動画レンダリング）、Runway（動画生成AI）、Whisper（文字起こし）、FFmpeg（音声抽出・空白検出）などのツールが活躍する。ルールを守るためには適切なツールとの連携が不可欠であり、これらをMCPやAPIで繋いでいく。

⑪ハーネスの組み立て方ステップ3：情報の整理と実行設計

ツールが揃ったら、どのツールがどのルールを守るべきなのかを整理し、データを構造化する。接続用のIDの管理場所や、ツールの呼び出し元を明確にする。そして最後に実行設計、つまり手順書を定義する。最近では「スキル」と呼ばれる形式で、いつ・どんな手順でツールを使っていくのかをしっかりと定義していくのがハーネスの完成形である。

⑫最も重要な概念：検証エージェントの導入

ハーネスの中で最も重要なのが検証エージェントの存在だ。ハーネスを整えても、それが再現性高く実行されているか、小さいタスク単位でちゃんと完了しているかをレビューする必要がある。例えば文字起こしが完了した後に、誤字脱字がないかをチェックする検証エージェントを入れることで、最終的に再現性の高い仕組みへと変わっていく。全体のワークフローに必ず組み込むべき要素である。

⑬実例で見るハーネス：動画編集スキルの全体像

KEITO氏が実際にClaude Codeで構築した動画編集スキルは、ハーネスの好例だ。スラッシュコマンド1つで全ての編集が完了する仕組みで、文字起こし、テロップ挿入、Bロール画像・動画の入れ込み、BGM追加まで全自動。ハーネスの中にはサブエージェントや検証ループが組み込まれ、エージェントチームとして文字起こしの構成確認や画像配置のタイミング管理なども行われている。

⑭検証・修正ループの仕組み：合格するまで最大5回繰り返す

動画編集スキルでは、レンダリング後に全体の検証修正ループが自動で実行される。テロップが15文字以内か、誤字脱字はないか、音声の重なりはないか、黒画面のフラッシュはないかなど、様々な項目をチェック。合格点に達するまで最大5回の検証ループを繰り返す。この仕組みによって、人間が逐一確認しなくても品質の担保された成果物が得られる。

動画編集スキルのハーネスを支えるツール群には、FFmpeg（音声抽出・空白検出）、OpenAI Whisper（文字起こし）、Remotion（動画レンダリング）、Runway（動画生成AI）などがある。さらに保存場所の管理や、過去の編集のリファレンスデータの参照、ツールからの指示の出し方などもハーネスの一部として組み込まれている。これら全ての要素をまとめてハーネスと呼ぶ。

⑯3つのエンジニアリングの比較：全てがハーネスに包含される

プロンプト・コンテキスト・ハーネスの3つを比較すると、ハーネスが最も包括的な概念であることがわかる。プロンプトは細かい指示の工夫、コンテキストは背景情報の設計、ハーネスは仕組み全体の構築を指す。ただしプロンプトやコンテキストが不要になるわけではなく、執筆作業での独自ドキュメントの読み込みや、自社Q&Aエージェントへの資料提供など、コンテキストも引き続き非常に重要だ。

ハーネスを設計する作業は、どのツールで、どういう権限で、どんな環境を作っていくのかという全体像を組み立てるディレクション業務に非常に近い。KEITO氏は自身の「AIディレクター」という肩書きが、実はハーネスの概念そのものだったと振り返る。エンジニアリングの細かなタスクよりも、全体像をどう組み立てていくかが重要視される時代になってきている。

⑱結論：モデルを変えるより環境を整える方が影響が大きい

ハーネスエンジニアリングの本質は、AIモデルを変えることよりも、周りの環境を整えることの方が成果への影響が大きいという考え方だ。仕組みの中にプロンプト、コンテキスト、外部ツール、セキュリティルール、検証エージェントなど全ての要素が含まれる。Claude Codeなどのツールに「このハーネスはどうですか？」と相談するだけでも、かなり良い設計を提案してくれるので、まずは試してみるのがおすすめだ。
