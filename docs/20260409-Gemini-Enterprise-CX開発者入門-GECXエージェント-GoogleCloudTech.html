Gemini Enterprise CX for Developers

Google Cloudの次世代カスタマー体験プラットフォームを開発者目線で徹底解説

Gemini Enterprise CX GECX CX Agent Studio Google Cloud AIエージェント

①AIエージェントのプロダクション運用が困難な理由

AIエージェントは非常に強力で、数分でかっこいいデモを作ることはできる。しかし、そのエージェントを本番環境に移行するのははるかに難しい。数百万人のユーザーにスケールしながら、低レイテンシ、セキュリティ、一貫したパフォーマンスを維持することは多くのプロジェクトがつまずくポイントだ。

②Gemini Enterprise CX（GECX）とは

GECXはプロトタイプからエンタープライズグレードの運用までを橋渡しする統合プラットフォーム。Dialogflow、Contact Center AI、Customer Engagement SuiteといったGoogle Cloudの会話ツールの次世代版として位置づけられており、生成AIファーストのワークフローにゼロから再構築された。

③Gemini EnterpriseとGECXの明確な違い

Googleには複数の「Gemini Enterprise」製品が存在する。Gemini Enterpriseは社内向けのエージェントプラットフォームで、企業のプライベートデータや社内ワークフローを横断するエージェントを展開する。一方、GECXは外部顧客向けに特化しており、検索、コマース、サービスをシームレスなカスタマージャーニーに統合する。

④GECXを構成する4つのコアコンポーネント

GECXは以下の要素で構成される。まずプリビルドコマースエージェント（小売や飲食注文などのターンキーソリューション）、CX Agent Studio（エージェント構築・改良の中核開発環境）、Agent Assist（人間のサポート担当者にリアルタイムのAIガイダンスを提供）、そしてVertex AI Search（RAG対応の高品質な検索エンジン構築サービス）。

⑤CX Agent Studioはブラックボックスではない

CX Agent Studioは単なるノーコードのUIではない。GoogleのオープンソースAIエージェントフレームワーク「Agent Development Kit（ADK）」の上に構築されたプラットフォーム。そのため、抽象化とプリビルドのボイラープレートのメリットを享受しながら、ブラックボックスに閉じ込められることはない。開発者は自由にカスタマイズできる。

⑥CX Agent Studioがカバーする開発ライフサイクル

CX Agent Studioコンソールは開発者の全ライフサイクルを管理する。「設計と開発」ではGeminiを直接使ってフローを設計可能。「マルチユーザーコラボレーション」はGoogle Docsのようにエージェントロジックを共同編集できる。「評価・モニタリング」には本番運用前のテストツールが組み込まれ、デプロイ後はGoogleのマネージドインフラで自動スケールする。

⑦マルチエージェントアーキテクチャで関心を分離

このサンプルエージェントはGoogle Cloud Next 2025のキーノートでプロダクトマネージャーPatrick Marlo氏がデモしたリファレンスアーキテクチャを採用。単一の巨大なプロンプトではなく、マルチエージェントアプローチで関心を分離。トップレベルのリテールエージェント、アップセルエージェント、スコープ外ハンドラーがそれぞれ独立した指示、モデル設定、ツールセットを持つ。

⑧PIF（プログラマティック命令追従）で構造化プロンプト

エージェントの指示にはPIF（Programmatic Instruction Following）を採用。これはXMLタグを使ってプロンプトを構造化する手法で、LLMがロジックやツール参照を解析しやすくなる。生のプロンプトがある場合は、コンソールの「魔法の杖」ボタンをクリックするだけで、自動的に推奨されるPIF形式にリファクタリングしてくれる。

⑨ツールのネスティングで決定論的な動作を実現

グリーティングツールは単なるテキストではない。CRMサービスを呼び出して顧客のカートと名前をコンテキストから取得するシーケンスとして動作する。このようにツールをネスティングすることで、エージェントの振る舞いが生成のランダムさに依存せず、決定論的かつ一貫性のあるものになる。各エンドポイントはOpenAPIスキーマ上の個別ツールとして機能する。

⑩画像認識×マルチステップチェーンの活用例

「商品レコメンド取得」ツールはマルチステップチェーンワークフローとして動作する。まずGoogle検索ツールでUSDA植物データベースにクエリを投げ、そのデータをCRMに渡してパーソナライズされたレコメンドを生成。画像からペチュニアなどの植物を特定し、土壌や肥料のアドバイスを1ターンで提供できる。

⑪本番レベルのセキュリティ：ガードレール機能

ガードレールアイコンからセキュリティ設定にアクセスできる。プロンプトガードはインジェクション攻撃からの保護を提供し、拒否、人間のエージェントへの転送、安全な応答生成などを設定可能。ブロックリストでは特定キーワードや正規表現パターンでユーザー入力とエージェント出力の両方をフィルタリングできる。

⑫セーフティフィルターとカスタムルール

セーフティフィルターはヘイトスピーチやハラスメントのフィルタリングなど、責任あるAI要件を処理する。カスタムルールでは、すべてのプロンプトに沿って実行される独自ロジックを記述できる。デモではモデルが異常な動作や脱線を始めた際に検出する「フライドポテトポリシー」というカスタムルールが含まれている。

⑬ガードレールは全エージェント共通でバックグラウンド実行

これらのガードレールはすべてバックグラウンドで自動実行される。サブエージェントやプロンプトごとに個別に設定する必要がない。グローバル設定ではデフォルトモデル、言語、テキスト読み上げの音声も管理可能。テレフォニー用途ではバージイン（ユーザーによる割り込み）、DTMF（キーパッド入力）、機密データのマスキングも有効にできる。

⑭評価機能：ゴールデンケースとダイナミックシナリオ

評価タブでは2種類のテストケースを利用できる。「ゴールデンケース」はハードコードされた正解シナリオで、エージェントが毎回同じパスをたどることを確認する回帰テストのようなもの。「ダイナミックシナリオ」はLLMが生成するテストで、様々なシナリオを通じてエージェントを新しい角度からテストする。後者はゴールデンケースに変換することも可能だ。

⑮各評価はタスク完了・目標達成・ハルシネーションをチェック

各評価ではタスクの完了度、ユーザーの目標達成率、ハルシネーションの有無を確認する。テストが失敗した場合、システムはLLMが生成した説明文で、なぜ期待に届かなかったのかを具体的に提示する。これにより、開発者はエージェントの改善ポイントを迅速に特定できる。

⑯デプロイ後のモニタリング：Conversational Insights

本番運用後のモニタリングにはConversational Insightsを利用。すべてのエージェントインタラクションのフルログとAI品質スコアを確認できる。プレビューエージェントウィンドウの「会話履歴」→「インサイト」からアクセス可能。運用中のエージェント品質を継続的に把握できる。

⑰MCP対応と豊富な統合オプション

統合面ではプリビルドコネクタのライブラリに加えて、MCP（Model Context Protocol）をフルサポート。Pythonコールバックや特定のツールランタイムも設定でき、厳密なコード定義ロジックが必要な処理にも対応する。また双方向ストリーミング音声により、Geminiネイティブオーディオモデル経由で低レイテンシのマルチモーダル入力と画像入力を処理できる。

⑱デプロイメントチャネルとホスト型MCPサーバー

デプロイメントでは3つのチャネルが利用可能。Webウィジェット（埋め込み可能なチャットUI）、テレフォニー（Twilio、Google Telephony、Google CCAIプラットフォームと直接接続）、API接続（RESTまたはクライアントライブラリでカスタムフロントエンドを構築）。IDE派の開発者向けにはホスト型MCPサーバーも用意され、Gemini CLIやCloud Codeから完全にコードベースでエージェントを構築・管理できる。
