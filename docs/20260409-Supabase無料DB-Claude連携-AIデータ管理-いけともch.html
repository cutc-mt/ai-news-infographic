①AI時代のデータ管理の課題：どこに何があるか分からない

AIツールを複数併用していると、「どこに何のデータがあるか分からない」という問題が必ず発生する。過去に分析した結果を使って次の作業をしたいのに、データを持ってくるのが面倒というケースが頻発する。データが整っていればAIがさらに賢くアウトプットを出せるため、データ管理のサイクルを回すことが非常に重要になる。

②データの2つの種類：非構造化データと構造化データ

データには大きく2つの種類がある。非構造化データは会議の文字起こし、メモ、リサーチ結果、画像、動画など。一方構造化データは売上データのように形式が整っていて、後から集計やグラフ作成がしやすいデータのこと。この2つをうまく管理・保存できる体制を作ることが重要だ。

③Supabaseとは：何でも入るExcelがクラウドにある

Supabaseはオープンソースのクラウドデータベースサービスで、非構造化データも構造化データも1か所にまとめて管理できる。自分でインストールして使うこともできるし、クラウド版で無料登録も可能。イメージとしては「何でも入るExcelがクラウドにある」ようなもので、テキストデータなら無料枠でも相当な量を扱える。

④Excelとデータベースの違い：大量データに強い仕組み

Excelやスプレッドシートは少量データなら便利だが、データ量が多くなると処理が遅くなり、柔軟な検索や集計が難しくなる。一方データベースは大量のデータを高速に検索・更新でき、AIがデータを読みやすく、複数人での管理も容易というメリットがある。Supabaseは裏側でPostgreSQLというデータベース仕組みを採用している。

⑤MCPコネクター：Claudeからワンクリックで接続

Supabaseが非常に優れている点はAIから操作しやすい仕様になっていること。Claudeの「コネクター」機能からSupabaseを検索し、連携ボタンをクリックするだけで接続完了。組織を選んでオーサライズするだけで、Claudeが自由にテーブルを作成したりデータを読み書きしたりできるようになる。

⑥Edge Function：毎日自動でデータを収集する仕組み

SupabaseのEdge Functionを使うと、プログラムを設定するだけで毎日決まった時間にデータを自動収集できる。パソコンを立ち上げていなくても、Xのフォロワー数やイベントのデータを定期的に取得して保存し続ける仕組みが作れる。これによりデータが溜まり続け、いつでもAIで分析可能になる。

⑦Supabase無料版の登録手順：数分で始められる

Supabase.comにアクセスし、メールアドレスでサインアップするだけ。パスワードは8文字以上で大文字・小文字・数字・特殊文字を含める必要がある。組織名を設定した後、プロジェクトを2つまで無料で作成可能。リージョンはアジアパシフィックを選ぶのがおすすめ。データベース用のパスワードも別途設定する。

⑧Claudeにテーブルを作ってもらう：AIにお任せ設計

接続後、Claudeのチャットで「学びを保存するテーブルを作って」と頼むだけで、AIがテーブル設計から作成まで全自動でやってくれる。ID、タイトル、本文、カテゴリーといった列を自動で考えて作成し、サンプルデータまで入れてくれる。SQLの知識がなくても、やりたいことを自然言語で伝えるだけでデータベースが構築できる。

⑨非構造化データの保存：学びやメモをAIに記憶させる

会話の中で「この学びを保存して」と伝えるだけで、ClaudeがSupabaseにデータを自動登録する。例えば「ClaudeとSupabaseを連携すると、どこでも使えるデータベースが作れる」といった気づきを保存。後から「以前こんな話をした気がする」と言えば、Claudeがデータベースから該当情報を検索して持ってきてくれる。

⑩マルチAI対応：Claude以外からも同じデータを読める

SupabaseのデータはClaude以外にもPerplexityやChatGPTなど複数のAIから活用可能。Perplexityの左下のコネクターからSupabaseを接続するだけで、同じデータベースにアクセスできる。データを特定のAIに依存させず、どんなAIツールからでも読み書きできるのが大きな強みだ。

⑪構造化データの実践：Xフォロワー数を毎日自動収集

「Xのフォロワー数を毎朝9時に保存する仕組みを作って」とClaudeに頼むと、テーブル作成からEdge Functionの設定まで全自動でやってくれる。X APIのベアラートークンを取得してSecretに設定するだけで、毎日自動的にフォロワー数が記録される。1週間・1ヶ月経てば推移の分析が可能になる。

⑫X APIの取得とSecret設定：外部データを安全に扱う

Xのデータを取得するにはdeveloper.x.comでAPIキー（ベアラートークン）を取得する必要がある。無料枠も一部あるが、基本的にはクレジット購入が必要（5ドル単位）。取得したトークンはSupabaseの「Secret」という機密情報管理機能に保存し、Edge Functionから安全に利用する。

⑬いけとも氏の実際の活用事例：複数テーブルでデータを整理

実際の運用では非構造化データとして、学び・メモ、イベントアンケート結果、リサーチ結果、記憶（メモリー）などをテーブル別に管理。構造化データとしてはLumaイベント、サークルコミュニティ、X、YouTube、投資・マクロデータを日々収集。必要な時にAIで分析して活用している。

⑭Claude Code / Coworkとの連携：レポート作成からメール作成まで

Supabaseにデータが蓄積されていれば、Claude CodeやClaude Coworkで自由にデータを引き出して分析やレポート作成ができる。例えばイベントデータを取得してHTMLレポートを作成したり、過去のアンケートデータと組み合わせてマーケティングメールを作成したりと、複数データの融合による高度な作業が簡単に実行できる。

⑮実践ワークフロー：イベントデータ＋アンケートでメール作成

実際のワークフロー例：Claude Codeで「4月9日のイベントのメールを作って」と指示すると、Supabaseからイベント情報と過去のアンケートデータを自動取得し、魅力的なメール文面を生成する。データを1つ1つ手作業で添付する必要がなく、データベースに全て揃っているため、AIエージェントへの指示だけで複雑な作業が完結する。

⑯セキュリティ注意点①：コネクター権限の設定

AIコネクターの権限設定には要注意。読み取りは常に許可しても良いが、書き込み（テーブル作成・データ削除など）は最初は毎回確認する設定がおすすめ。何でもかんでも自動許可にしていると、AIが誤ってデータを全消去するリスクがある。慣れてから定型作業のみ自動化するのが安全なアプローチだ。

⑰セキュリティ注意点②：RLS（Row Level Security）の重要性

RLSを設定していないと、外部からデータが丸見えになる深刻なリスクがある。実際に以前MiroTalkというサービスがSupabase上でRLS未設定のまま運用し、全ユーザーのデータが外部から閲覧できる事故が発生した。RLSを設定することで「読み取りのみ許可」「書き込み禁止」といった制御が可能。外部公開時は必ず設定すること。

⑱外部データベースに置く価値：AI乗り換えにも強い体制

データをClaude内（CLAUDE.mdやプロジェクト機能）に保存すると他のAIでは使いにくい。しかしSupabaseのような外部データベースに置けば、Claude、ChatGPT、GeminiなどどのAIからでもアクセス可能。AIの流行り廃りが激しい中、データの管理場所を独立させることで、いつでも最適なAIツールに乗り換えられる柔軟性が手に入る。バイブコーディングとの相性も抜群だ。
