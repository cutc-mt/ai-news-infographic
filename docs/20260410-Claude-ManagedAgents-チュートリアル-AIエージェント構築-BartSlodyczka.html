🤖 Claude Managed Agents フルチュートリアル

エージェント作成からセッション管理、環境設定、本番デプロイまで完全解説

Claude Managed Agents エージェント構築 n8n連携 セッション管理 Bart Slodyczka

①チュートリアルの概要：エージェント作成から本番デプロイまで

このチュートリアルではClaude Managed Agentsの全機能を一から解説する。まずClaudeコンソール上で新しいエージェントを作成し、セッション・環境・認証情報の概念を理解する。その後、コンソールでテストを行い、最終的にn8nを使った本番環境へのデプロイまでを実演する。実際に動くカスタマーサポートチャットボットを構築していく流れだ。

②エージェントの実演：カスタマーサポートチャットボット

チュートリアルではn8nをフロントエンドとし、バックエンドにClaudeエージェントを配置したチャットボットを構築する。ユーザーが「蛇口が壊れている」とメッセージを送ると、Claudeが適切に回答。コンソール側には30秒以内に新しいセッションが作成され、メッセージの送受信がすべて記録される。実際の顧客対応を想定したリアルなデモだ。

③セッションの仕組み：顧客ごとに独立した会話コンテキスト

セッションの最大の特徴は、1つのエージェントで複数の顧客と独立した会話を同時に進められること。新しい顧客がメールやチャットで問い合わせてくるたびに新しいセッションを作成するだけで、それぞれの会話の文脈がその顧客専用に保持される。同じエージェントを使い回しながら、プライバシーも確保できる優れた仕組みだ。

④セッションの継続性：同じ会話にメッセージを追加できる

一度作成したセッションには後からメッセージを追加できる。ユーザーが新しいメッセージを送信すると、既存のセッション内に追加されて文脈を引き継いだ回答が返ってくる。例えば「水を出してみたら動いた」と送ると、前の会話を踏まえた上で「解決してよかったですね」といった適切な返答が得られる。一問一答ではなく、対話が連続していく設計だ。

⑤アカウントの準備と課金の仕組み：クレジットのチャージが必要

Claude Managed Agentsを利用するにはplatform.claude.comでアカウントを作成し、クレジットをチャージする必要がある。Claude ProやMaxのサブスクリプションとは別物で、トークン単位の従量課金が適用される。つまり、サブスクリプション料金ではなく、実際に使った分だけを支払う仕組みだ。

⑥2つの課金方式：トークン料金とセッション実行時間

課金は大きく2つに分かれる。1つ目は使用したトークンのAPI料金で、これは通常のClaude APIと同じレートが適用される。2つ目はセッションの実行時間で1時間あたり0.08ドル。ただし重要なのは、エージェントが実際に動いている時だけ課金されるという点で、ユーザーの入力待ちやアイドル状態では一切料金がかからない。

⑦アイドル時間は無料：実行中のみ課金される仕組み

具体例を挙げると、最初のメッセージ送信からレスポンス受信までの5.7秒間の処理時間には課金されるが、その後ユーザーの返信を待っていた2分28秒間のアイドル時間は無料。次のメッセージを送ってからまたレスポンスが返ってくるまでの時間だけが課金対象になる。つまり、エージェントが仕事をしている時だけにお金がかかるのだ。

⑧トークンの内訳確認：入力・キャッシュ読み書き・出力を把握

コンソール上ではメッセージごとに消費されたトークンの詳細な内訳を確認できる。入力トークン数、キャッシュ読み取り（cache read）の量、キャッシュ書き込み（cache write）の量、そして出力トークン数がすべて表示される。これにより、どの処理でコストがかかっているかを正確に把握でき、最適化の指標として活用できる。

⑨エージェントの作成方法：テンプレート、白紙、会話ベースの3通り

エージェントの作成には3つの方法がある。既存のテンプレートを使う方法、白紙の状態から始める方法、そしてAIアシスタントに会話で指示して作らせる方法だ。動画では会話ベースのアプローチを採用し、「dualcut.com.auのカスタマーサポートチャットボットを作ってほしい」と指示。FAQ対応とトラブルシューティングができるエージェントを生成させている。

⑩エージェントへの指示内容：ClickUpへのエスカレーションも設定

作成するエージェントには具体的な要件を指定する。まずウェブサイトの情報を読み取ってFAQに回答できること。そして顧客が問題解決できなかったり人間へのエスカレーションを希望した場合は、ClickUpにタスクとして自動起票する機能も設定。URLを指定するだけで、エスカレーション先のリストを紐付けられる。

⑪環境の設定：エージェントが動くクラウドコンテナを構築

エージェントを作成したら次は環境の設定が必要だ。環境とはエージェントが実行されるクラウドコンテナのことで、オフィスに例えるなら「部屋を作る」ようなもの。その部屋の中にはエージェントが必要とするツールやファイル、すべてのリソースが揃っている。ネットワークアクセスも設定でき、特定のドメインのみに制限するセキュリティ設定も可能だ。

⑫ネットワーク制限：自社ドメインのみにアクセスを限定

環境設定ではネットワークアクセスの範囲を制限できる。このチュートリアルでは「limited」を選択し、Web検索の対象を自社のドメイン（dualcut.com.au）のみに制限している。これによりエージェントが不必要な外部サイトにアクセスすることを防ぎ、セキュリティとパフォーマンスの両面で安全性を確保できる。

⑬認証情報（Credentials Vault）：MCPツールのOAuth認証を管理

Credentials Vaultは、エージェントが利用する各種ツールの認証情報を一元管理する機能。このチュートリアルではClickUpのMCPサーバーに対してOAuth認証を設定している。認証を承認してワークスペースに接続するだけで、エージェントがClickUpのAPIにアクセスできるようになる。複数のツールの認証情報もここでまとめて管理できる。

⑭コンソールでのテスト：エージェントの動作をその場で確認

エージェントの準備ができたら、コンソール右側のテストパネルですぐに動作確認ができる。メッセージを送信すると、レスポンスやツール呼び出しの様子がリアルタイムで表示される。最初のメッセージはClaudeが自動生成するため、そのまま送信するだけでテスト開始。デバッグセクションでは、ツール呼び出しの結果（成功・失敗）も確認できる。

⑮エスカレーションのテスト：ClickUpへのタスク自動起票を確認

テストでは「人間にエスカレーションしたい」と伝えると、エージェントが名前やメールアドレスなどの情報を収集し、ClickUpに自動でタスクを起票する。デバッグ画面でツール呼び出しを確認すると「success: true」と表示され、実際にClickUp側にも「顧客エスカレーション：蛇口が開かない」というタスクが作成される。設定した通りの動作が完璧に機能していることが確認できる。

⑯エージェント管理画面：システムプロンプトやモデル設定の確認

作成済みのエージェントは管理画面でシステムプロンプトやデフォルトモデル（現在はSonnet 4.6）、MCP接続、スキル設定などを確認・編集できる。手動での編集と、AIアシスタントとの会話によるガイド付き編集の2つのモードが用意されている。複数のエージェントを作成することも可能で、例えばZenDesk連携のメール対応ボットや請求書処理ボットなど、用途ごとに独立して管理できる。

⑰n8nでの本番デプロイ：APIを組み合わせたワークフロー構築

n8nを使ったデプロイでは、わずか10個のノードでチャットボットが完成する。ワークフローは「新規セッション作成」と「既存セッションへの追記」の2ルートに分かれる。ユーザーからのメッセージを受け取ると、まずデータベースでセッションIDを検索。見つからなければ新規セッションを作成し、見つかればそのセッションにメッセージを追加する仕組みだ。Agent ID、Environment ID、Vault IDの3つを設定するだけで動く。

⑱本番運用のポイントとまとめ：APIリファレンスを活用しよう

n8nでの実装の鍵はClaudeの公式APIリファレンスを活用すること。エージェント・環境・セッション・認証の4つのAPIを組み合わせて構築する。メッセージ送信後は15秒待機してからリストAPIで結果を取得するが、実際の本番では長時間タスクを考慮したループ処理の実装が推奨される。ワークフローを公開してURLをコピーすれば、ブラウザからすぐにエージェントと会話できる。Claude Co-workプロジェクトにAPIドキュメントをアップロードしておくと、実装の助けになる。
