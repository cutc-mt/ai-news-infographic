⑤ ヘルメスエージェントとは: 自己進化するAI

  • Nurse Research（ナースリサーチ）が開発したオープンソースのAIエージェント。無料で誰でも使えます。
  • 最大の特徴は自己進化ループ。タスクをこなすたびに学び、上手くいったやり方をスキルとして保存します。
  • 技術的にはGEEP（Generative Experience-based Evolutionary Prompting）というアルゴリズムが使われています。
普通のAIは毎回ゼロから。ヘルメスは使えば使うほど賢くなる！

ベンダーロックインなし！自分に合ったモデルを自由に選べる

⑧ RAG（検索拡張生成）とは何か

  • RAG（Retrieval-Augmented Generation）は、質問が来たら資料を検索して関連部分をAIに渡す仕組みです。
  • NotebookLMもその一例。資料を登録してAIが検索しながら回答してくれます。
  • 便利な仕組みですが、知識が積み上がらないという根本的な問題があります。
質問のたびにゼロから検索。何百回使ってもシステムは賢くならない

⑨ 従来のRAGの限界: 使い捨ての知識

  • RAGは質問するたびに毎回ゼロから検索します。前回の会話で何を学んだかは関係ありません。
  • 何百回質問しても、システムそのものは賢くなりません。知識が蓄積されないのが問題です。
  • カパシーはこの「知識が使い捨てになる」構造に根本的な問題を感じました。
毎回図書館に行って調べるが、自分の本棚は増えない。それがRAGの限界

⑩ LLM Wiki: 知識が積み上がる仕組み

  • LLM Wikiはカパシーが2026年4月4日にGitHubに投稿した概念。数日でスター5000超え、フォーク2000以上。
  • 資料を追加するたびにAIが自分でWikiページを書き、内容を理解してマークダウン形式で整理します。
  • 新しい資料が来たら既存のWikiを確認し、更新すべき箇所を書き直し、矛盾があれば指摘します。
RAGは毎回図書館で調べる仕組み。LLM Wikiは読むたびに自分の本棚が育っていく！

⑪ LLM Wikiの三層構造

  • 第1層（下）: 生ソース。変更しない原本の資料です。
  • 第2層（中）: Wikiページ。AIが書いたマークダウンのファイル群です。
  • 第3層（上）: スキーマ。AIへの運用ルールを定義したファイルです。 
┌─────────────────────┐
│  スキーマ（運用ルール）│  ← 第3層
├─────────────────────┤
│  Wiki（マークダウン） │  ← 第2層
├─────────────────────┤
│  生ソース（原本資料） │  ← 第1層
└─────────────────────┘

⑫ LLM Wikiの3つの基本操作

  • 取り込み（Ingest）: 資料を読んでWikiを生成・更新します。
  • 質問（Query）: Wikiから検索して回答します。
  • 健全性チェック（Health Check）: 矛盾や抜け穴を発見します。
読んで、探して、検証する。この3つのループでWikiが育ち続ける

⑬ エンジニア向けの表現: コードベースとしてのWiki

  • カパシーはこう表現しています：「ObsidianはIDE、LLMはプログラマー、Wikiはコードベース」
  • コードベースが育つように、知識ベースが育つというアナロジーです。
  • エンジニアには直感的に伝わる表現で、ソフトウェア開発の概念を知識管理に応用しています。
Obsidian = IDE、LLM = プログラマー、Wiki = コードベース
プルリクのように知識が成長

⑭ プライバシー保護: 全てローカルに保存

  • LLM Wikiの全データはローカルのPC上に保存されます。
  • クラウドに送られないため、機密情報や個人情報も安全に管理できます。
  • データの主権がユーザーにあるという点で、エンタープライズ利用にも適しています。
クラウドに送らない！自分のPCで知識を育てる安心設計
