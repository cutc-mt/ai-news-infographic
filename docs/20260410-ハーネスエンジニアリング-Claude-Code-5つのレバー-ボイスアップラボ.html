①NTTドコモ調査：日本人の過半数が生成AIを利用

NTTドコモが発表した生成AI利用実態調査によると、2026年2月時点で日本人口の過半数が生成AIを利用していることが分かった。前回の2025年2月調査からわずか1年で利用者が倍増し、6500万人以上が日常的にAIを使う時代に突入した。15歳から70歳までを対象とした調査で、キャズムを完全に超えた感があるという。

②AI開発手法の進化：プロンプトからハーネスへ

AIの開発手法は大きく変化してきた。最初は「プロンプトエンジニアリング」が主流だったが、2025年夏頃から「コンテキストエンジニアリング」が注目されるように。そして今、一番ホットなのが「ハーネスエンジニアリング」だ。ただし定義がバラバラで、各自が思い思いの解釈で語っている状況がある。

③ハーネスエンジニアリングとは：「LLMがやること以外のすべて」

ハーネスエンジニアリングの定義はシンプルだ。LLM自身がやること以外のすべてを整備することがハーネスエンジニアリングである。AIエージェントを作ることも、ガードレールを設定することも、自然言語で操れるようにすることも、すべて含まれる。Claude Codeを作ることもハーネスエンジニアリングの一部だ。

④AIは魔法の箱ではない：新入社員と同じ環境整備が必要

LLMは賢いのである程度何でもやってくれるが、環境整備が必要な「手のかかる新入社員」と同じだ。毎回手順を忘れたり、会社の基本的なルールを知らなかったりする。何も設定しないと、LLMは賢いのかおばさんなのかよく分からない行動をしてしまう。だからこそハーネスエンジニアリングが重要なのだ。

⑤ハーネスの効果：生産性が40%以上向上する実験結果

ハーネスの有無による比較実験が行われており、同じLLMを使っていてもハーネスがちゃんとしていれば生産性が40%以上向上するという結果が出ている。LLMのモデル選びよりも、ハーネスの設定の仕方の方が重要だと言われるほどだ。この事実は見過ごせない。

⑥Claude Codeのハーネス5つのレバー：これが肝

ハーネスエンジニアリングには5つの重要な調整ポイントがある。①CLAUDE.md（設定ファイル）、②Hooks（イベント駆動のチェック）、③Skills（再利用可能なプログラム）、④MCP（外部ツール連携）、⑤Memory（セッション記憶）。この5つをうまく調整することこそが、ハーネスエンジニアリングの本質だ。

⑦レバー①CLAUDE.md：プロジェクトの憲法

CLAUDE.mdはプロジェクトのフォルダに置く設定ファイルで、Claude Codeが従うべきルールを記述する。出た当初は何を書けばいいか分からず、ネットにある完璧そうなものをコピーして使う人も多かった。しかし、書き方次第でClaude Codeの効き方が劇的に変わる最も重要なレバーだ。

⑧CLAUDE.mdの書き方：コンパクトにが鉄則

CLAUDE.mdで一番効果的なのは、できるだけコンパクトにすること。そして、コードからだけでは判断できない情報だけを入れるのが良い。例えば、AIが知らない独自のBashコマンド、コードベースに出てこない特殊な動作スタイル、明らかにハマりそうな注意点などだ。無駄な情報を削ぎ落とすことで、Claude Codeが迷わず動けるようになる。

⑨CLAUDE.mdに入れてはいけないもの

逆に入れてはいけないものもある。コードを見れば分かること（インデントルールなど）や、頻繁に変わる情報、なんとなくのルールはNGだ。「このコードには2つインデントを入れなさい」みたいなものは、コードを見れば分かるので書く必要がない。そういうものはHooksに任せるのが正しいアプローチだ。

⑩レバー②Hooks：イベント駆動で自動チェック

Hooksはイベント駆動で強制的に実行される仕組みだ。CLAUDE.mdがアドバイスをするものであるのに対し、Hooksはエラーチェックやリンターなどを自動的に発動させる。例えばコードを書いた後に自動でリントを実行し、その結果がコンテキストに流れるため、Claude自身がエラーを改善できるという強力なフィードバックループができる。

⑪Hooksの最大のメリット：自己改善ループ

Hooksで出たエラーはコンテキストに流れるのが最大のメリットだ。Claude自身がそのエラーを見て、自分で改善して修正してくれる。この自動フィードバックループは非常に便利で、人間がいちいちチェックしなくても品質が担保される。リンターやフォーマッターをHooksに設定しておくことで、コードの品質が自動的に保たれる。

⑫レバー③Skills：決まり切った作業をプログラム化

Skillsは、LLMに毎回やらせるのではなく、決まり切った作業をプログラムとして定義して使い回す仕組みだ。LLMは表の計算などで毎回微妙に異なる結果を出してしまうことがあるが、Skills化しておけばエラーが出ない限り毎回確実に同じ結果を返してくれる。同じプロンプトを何回も実行したい場合に特に有効だ。

⑬Skillsの多彩な活用法：デプロイからサブエージェントまで

Skillsは自分で手動で起動することもできるし、自然言語の中でClaudeが自動的に使うことも可能だ。例えばデプロイやリリースの操作をSkillsにしておけば、会話の中で「デプロイして」と言うだけで実行される。さらにサブエージェントのコンテキストの中でSkillsを使わせることもでき、部下の脳みその中で確実な処理を回すこともできる。

⑭レバー④MCP：外部ツールとの連携

MCP（Model Context Protocol）は、Skillsの外部化とも言える仕組みで、外部のツールやサービスをClaude Codeから利用できるようにする。Skillsが内部のプログラム化であるのに対し、MCPは外部ツールとの連携を担う。どちらも「LLMのやることを確実にする」という目的は同じで、ツールの使い分けが重要だ。

⑮レバー⑤Memory：コンテキストを賢く管理

Claude Codeのコンテキストには、CLAUDE.mdやシステムプロンプトなどがすでに入っており、命令を重ねるごとにコンテキストが圧迫されていく。「コンパクト」は早めに実行するのが良いとされている。コンテキストがいっぱいになってからでは遅く、余裕があるうちに整理しておくことで安定した動作を維持できる。

⑯サブエージェントを活用してコンテキストを汚さない

サブエージェントを使うことも効果的な手法だ。メインのClaude Codeとは別の「全く違う脳みそ」を使うため、メインのコンテキストを消費しない。サブエージェントに仕事を投げて、結果だけをメインに返すため、コンテキストが汚れず、トークンも大幅に節約できる。大きなタスクを分割する時に特に有用だ。

⑰ビジネスプロセスの自動化でハーネスが最も生きる

ハーネスエンジニアリングが最も威力を発揮するのは、ビジネスプロセスを自動化する時だ。画像を生成してスライドにする、文章の品質をチェックする、といった定型フローをSkillsで書いて、Hooksでチェックして、ダメなら再実行するというオートメーション化ができる。ボールを落としたら自動的に回転し続ける仕組みを作るのがハーネス設計の真髄だ。

⑱まとめ：ハーネスエンジニアリングは今後のエンジニアの必須スキル

ハーネスの設定は、エンジニアだけでなくDXやビジネス層の人でもできるのが魅力だ。コードを書くだけでなく、ハーネスでビジネスそのものを動かしていく時代が来る。CLAUDE.md、Hooks、Skills、MCP、Memoryの5つを適切に調整して、いかに自動的に動かすかを制御する力が、今後のエンジニアの価値を決めることになるだろう。
