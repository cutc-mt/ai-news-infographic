①日本人の過半数が生成AIを利用する時代に突入

NTTドコモが発表した生成AI利用実態調査によると、2026年2月時点で日本人口の過半数が生成AIを日常的に利用していることが分かった。前回の2025年2月調査からわずか1年で利用者が倍増し、15歳から70歳までを対象にした調査で6500万人以上がAIを使っている。キャズムは完全に超え、2人に1人以上がAIを日常使いする時代になった。

②AI開発手法の進化：プロンプトからハーネスへ

AI開発の手法は急速に進化してきた。最初は「プロンプトエンジニアリング」が主流だったが、2025年夏頃から「コンテキストエンジニアリング」が注目を集めるようになった。そして今、一番ホットな話題が「ハーネスエンジニアリング」だ。ただし定義がバラバラで、みんながそれぞれ思い思いの解釈で語っている状況があるという。

③ハーネスエンジニアリングの定義：「LLMがやること以外のすべて」

ハーネスエンジニアリングとは一言で言えば、LLM自身がやること以外のすべてを整備することである。AIエージェントを作ることも、ガードレールを設定することも、自然言語で操れるサービスを作ることも、すべてハーネスエンジニアリングに含まれる。Claude Code自体を作ることも、広義にはハーネスエンジニアリングの一部だ。

④AIは魔法の箱ではなく、手のかかる新入社員

LLMは賢いのである程度何でもやってくれるが、環境整備が必要な「手のかかる新入社員」と同じ存在だ。毎回手順を忘れたり、会社の基本的なルールを知らなかったりする。何も設定しないと、LLMは賢いのかおばさんなのかよく分からない行動をしてしまう。だからこそ、ハーネスで環境を整備することが不可欠なのだ。

⑤ハーネスの威力：生産性が40%以上向上する実験結果

ハーネスの有無による比較実験が行われており、同じLLMを使っていてもハーネスがちゃんとしていれば生産性が40%以上向上するという結果が出ている。LLMのモデル選びよりも、ハーネスの設定の仕方の方が重要だと言われるほどだ。この事実は、ハーネスエンジニアリングの重要性を明確に物語っている。

⑥Claude Codeのハーネス5つのレバー：全体像

ハーネスエンジニアリングには5つの重要な調整ポイント（レバー）がある。①CLAUDE.md（プロジェクトの設定ファイル）、②Hooks（イベント駆動の自動チェック）、③Skills（再利用可能なプログラム）、④MCP（外部ツール連携）、⑤Memory（セッション記憶）。この5つをうまく調整することがハーネスエンジニアリングの肝だ。

⑦レバー①CLAUDE.md：プロジェクトの憲法のようなもの

CLAUDE.mdはプロジェクトのフォルダに置く設定ファイルで、Claude Codeが従うべきルールを記述する。出た当初は何を書けばいいか分からず、ネットにある完璧そうなテンプレートをコピーして使う人も多かった。しかし、書き方次第でClaude Codeの効き方が劇的に変わる、最も重要なレバーの一つだ。

⑧CLAUDE.mdのコツ：コンパクトに、コードから分からない情報だけ

CLAUDE.mdで一番効果的なのは、できるだけコンパクトにすること。そして、コードからだけでは判断できない情報だけを入れるのが良いとされている。例えば、AIが知らない独自のBashコマンドや、コードベースに出てこない特殊な動作スタイル、明らかにハマりそうな注意点などが該当する。無駄な情報を削ぎ落とすことで、Claude Codeが迷わず動けるようになる。

⑨CLAUDE.mdに入れてはいけない情報とは

逆に入れてはいけないものもある。コードを見れば分かること（例えばインデントルールなど）や、頻繁に変わる情報、なんとなくのルールはNGだ。「このコードにはインデントを2つ入れなさい」みたいなものは、コードを見れば分かるので書く必要がない。そういうものはHooksに任せるのが正しい役割分担だ。

⑩レバー②Hooks：イベント駆動で強制的にチェックを実行

Hooksはイベント駆動で強制的に実行される仕組みだ。CLAUDE.mdが「こうしろああしろ」とアドバイスをするものであるのに対し、Hooksはエラーチェックやリンターなどを自動的に発動させる。例えばコードを書いた後に自動でリントを実行し、その結果がコンテキストに流れるため、Claude自身がエラーを確認して改善できる。

⑪Hooksの最大メリット：エラーがコンテキストに流れる自己改善ループ

Hooksで出たエラーはコンテキストに流れるのが最大のメリットだ。Claude自身がそのエラーを見て、自分で改善して修正してくれる。この自動フィードバックループは非常に便利で、人間がいちいちチェックしなくても品質が担保される。リンターやフォーマッターをHooksに設定しておくことで、コード品質が自動的に保たれる仕組みができる。

⑫レバー③Skills：決まり切った作業を確実なプログラムに変える

Skillsは、LLMに毎回やらせるのではなく、決まり切った作業をプログラムとして定義して使い回す仕組みだ。LLMは表の計算などで毎回微妙に異なる結果を出してしまうことがある。しかし、Skills化しておけばエラーが出ない限り毎回確実に同じ結果を返してくれる。同じプロンプトを何回も実行したい場合に特に有効だ。

⑬Skillsの多彩な活用法：手動起動からサブエージェントまで

Skillsは自分で手動で起動することもできるし、自然言語の中でClaudeが自動的に使い分けることも可能だ。デプロイやリリースをSkillsにしておけば、会話の中で「デプロイして」と言うだけで実行される。さらにサブエージェントのコンテキストの中でSkillsを使わせることもでき、「部下の脳みそ」の中で確実な処理を回すこともできる。

⑭レバー④MCP：外部ツールとの連携でClaudeの能力を拡張

MCP（Model Context Protocol）は、Skillsの外部化とも言える仕組みで、外部のツールやサービスをClaude Codeから利用できるようにする。Skillsが内部のプログラム化であるのに対し、MCPは外部ツールとの連携を担う。どちらも「LLMのやることを確実にする」という目的は同じで、用途に応じた使い分けが重要だ。

⑮レバー⑤Memory：セッションをまたいで記憶を維持する

Memoryはセッションの記憶を保持する仕組みで、コンテキストが変わっても前のことを覚えている。Claude CodeのコンテキストにはCLAUDE.mdやシステムプロンプトがすでに入っており、命令を重ねるごとに圧迫されていく。そのため「コンパクト」を適度に早めに実行することが推奨されている。100ターンぐらいで発動する機能だが、もっと早めにやった方が良いとされている。

⑯サブエージェント活用でコンテキストを汚さずトークンを節約

サブエージェントを使うことも効果的な手法だ。メインのClaude Codeとは別の「全く違う脳みそ」を使うため、メインのコンテキストを消費しない。サブエージェントに仕事を投げて、結果だけをメインに返すため、コンテキストが汚れず、トークンも大幅に節約できる。大きなタスクを分割する時に特に有用なアプローチだ。

⑰ビジネスプロセスの自動化こそハーネスが最も輝く場面

ハーネスエンジニアリングが最も威力を発揮するのは、ビジネスプロセスを自動化する時だ。例えば画像を生成してスライドにする、文章の品質をチェックするといった定型フローを、Skillsで書いて、Hooksでチェックして、ダメなら再実行するというオートメーション化が可能になる。「ボールを落としたら自動的に回転し続ける」仕組みを作るのがハーネス設計の真髄だ。

⑱まとめ：ハーネスエンジニアリングはエンジニアとビジネスの架け橋

ハーネスの設計は、エンジニアだけでなくDXやビジネス層の人でもできるのが魅力だ。コードを書くだけでなく、ハーネスでビジネスそのものを動かしていく時代が来る。CLAUDE.md、Hooks、Skills、MCP、Memoryの5つを適切に調整して、いかに自動的に動かすかを制御する力が、今後のエンジニアの価値を決めることになるだろう。
