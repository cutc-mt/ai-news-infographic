Claude Codeでナレッジ構築

Karpathy手法 × Obsidian × Claude Codeで自分専用Wikiを作る
2026年4月11日 | にゃんたのAIチャンネル

①Karpathyさんの話題の手法

OpenAIの共同設立者であるAndrej Karpathyさんが、自身のX（旧Twitter）で「個人用ナレッジベースの構築」についてツイートして話題になりました。Karpathyさんの最近のトークン処理量の大部分は、コードの操作ではなく知識の操作に費やされているとのことです。

AIを使ってナレッジを構築する手法が広く注目を集めています

②ナレッジベースとは何か

ナレッジベースとは、自分が調べたいトピックについての情報を整理して管理するシステムです。論文、記事、コードなどを集め、AIが自動的にWiki形式でまとめてくれます。

  • 調べ事や学習に使える自分専用のWiki
  • ノートアプリの代わりにもなる
  • 使えば使うほどパーソナライズされる

③Obsidian：高機能なメモ帳アプリ

Obsidianは無料で使える高機能なメモ帳アプリです。マークダウン形式でドキュメントを管理でき、リンク機能で文書同士を繋げられます。

  • 無料でダウンロード可能
  • ドキュメント間のリンク機能
  • グラフビューで知識の繋がりを可視化
  • 拡張機能が豊富
保管庫（Vault）という単位でフォルダーを管理します

④Web Clipperで簡単データ収集

ObsidianのWeb Clipper拡張機能を使うと、ネット上の記事をワンクリックで自分のパソコンに保存できます。Chrome拡張機能として追加して利用します。

  • 記事を一瞬でローカルに保存
  • rawフォルダーに直接格納可能
  • 設定で保存先フォルダーを指定できる
# Web Clipper設定
デフォルトのノートの場所 → raw
4ステップでナレッジ構築

⑤ステップ1：データを集める

調べたいトピックについての情報ソースをパソコン上に集めます。論文、記事、コードなどをrawフォルダーに格納します。

  • Web記事をクリップ
  • PDF論文を保存
  • コードを配置
Web Clipperを使えば一瞬で記事をrawフォルダーに追加できます

⑥ステップ2：データをコンパイル

集めた生データをAIを使って要約・整理します。コンパイルとは、生のデータファイルをAIが読みやすい形に変換する処理です。

  • AIが記事を要約
  • エンティティ（ノード）を抽出
  • リンクグラフを自動生成
  • index.mdを作成
/ingest ← このコマンドで実行

⑦ステップ3：質問して活用する

コンパイルされたWikiに対して質問や検索ができます。質問と回答はWikiに保存され、使えば使うほど知識が蓄積されていきます。

/query 質問内容
/query --save ← 回答を保存
質問すればするほどコンセプトページが増え、自分専用のシステムに育ちます

⑧ステップ4：ドキュメントを保守する

Wikiに矛盾がないかをチェックする定期保守コマンドが用意されています。情報が増えてきたら1日1回程度実行するとよいでしょう。

/lint ← 品質チェック
/lint --fix ← 自動修正
  • リンクの整合性チェック
  • 矛盾の検出と修正
  • 1日1回の定期実行がおすすめ
実際の構築手順

⑨Claude Codeでテンプレート導入

KarpathyさんのGitHub Gistにあるマークダウンテンプレートをコピーして、Claude Codeに渡します。Planモード（Shift+Tab×2）で計画を作成させます。

  • GistのRawボタンからテキストをコピー
  • Claude CodeにCtrl+Vで貼り付け
  • スキルとして定義するよう指示
  • Planモードで計画を作成・承認
計画を承認するだけで、自動的にシステムを構築してくれます

⑩フォルダー構造の全体像

システム構築後は以下のようなフォルダー構造が作られます。このフォルダーをObsidianの保管庫として開くことで、見やすく確認できます。

CCナレッジ/
├── raw/ ← 生データ
├── wiki/ ← Wiki化された情報
│ ├── concepts/
│ └── entities/
└── CLAUDE.md ← 指示書

⑪ObsidianとWeb Clipperの連携

Web Clipperの設定で、デフォルトのノート保存先をrawフォルダーに指定しておくことで、クリップした記事が直接rawフォルダーに保存されます。

  • Chrome拡張機能として追加
  • 宝石マークのアイコンから起動
  • 保存先をrawフォルダーに設定
これで情報収集→Wiki化の流れがスムーズになります

⑫/ingestで記事を取り込む

rawフォルダーに記事を入れたら、Claude Codeで/ingestコマンドを実行します。AIが自動的に記事を読み込んで、エンティティやコンセプトを抽出し、Wiki化してくれます。

  • index.mdが自動生成される
  • エンティティ（ノード）が抽出される
  • 情報ソースへのリンクが作られる
  • 操作ログが記録される
ナレッジの活用と仕組み

⑬/query --saveでQ&Aを知識化

Wikiに対して質問をすると、AIがWikiを参照して回答を生成します。--saveオプションを付けると、そのQ&Aがコンセプトページとして保存されます。

/query --save ハーネス設計に必要なスキルは？
  • Wikiの内容を参照して回答を生成
  • 回答が新しいコンセプトページとして保存
  • 次回の質問からこの情報も参照される

⑭エンティティとグラフビュー

Wiki化されるとエンティティ（ノード）が作成されます。グラフビューで確認すると、概念同士の繋がりが視覚的に分かります。

  • 各概念がドキュメントとして作成
  • 例：「Anthropic」に紐づく機能が一覧できる
  • 新しいことを学ぶ時に関係性が分かりやすい
記事をたくさん入れても綺麗なグラフ構造に整理されます

⑮RAGとの違い：シンプルな仕組み

一般的なRAGはベクトルデータベースやエンベディングの前処理が必要ですが、この方法はコンテキストウィンドウ内で直接処理します。

⑯100万トークンで十分に対応可能

現在のコンテキストウィンドウは100万トークンなので、100記事・40万個程度のデータでも普通に使えます。

  • エージェントが自動検索してくれる
  • MAX 20Xプランなら使い放題に近い
  • Proや5Xプランではトークン上限に注意
上限に達する場合はSonnetを使うか、CEXなどの他エージェントでも可能です

⑰/lintでWikiの品質を維持

/lintコマンドを実行すると、Wikiの品質チェックを行ってくれます。--fixオプションで自動修正も可能です。

  • リンクのエラーを検出
  • 矛盾をチェック
  • 整合性を自動で整える
  • 1日1回の定期実行を推奨
/lint ← チェックのみ
/lint --fix ← 自動修正付き

⑱コーディング以外のClaude Code活用

Claude Codeはコードを書くだけでなく、知識の整理や情報管理にも強力に使えます。質問を重ねるほど自分にパーソナライズされたシステムに成長します。

  • メール対応の文章作成
  • ログ参照で興味関心を推測
  • 自分の書き方を学習して返答文を生成
  • 使うほど体験が変わる
YouTube制作やメール対応など、日常のタスクにも活用できる可能性があります
