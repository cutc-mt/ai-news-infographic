Hermes Agent - 自己改善するAIエージェント

セッションをまたいでも忘れない、成長し続ける汎用エージェント

2026年4月11日

① Hermes Agentとは

  • Nous Researchが開発した自己改善型AIエージェント
  • セッションをまたいでも記憶を保持し、使い込むほど賢くなる設計
  • GitHubでオープンソースとして公開（nousresearch/hermes-agent）
  • 世界中で注目を集めており、OpenClawの強力なライバルと位置づけられている
使い込むほど進化するのが最大の特徴です！

② メモリ3層構造

  • セッションメモリ：現在の対話の文脈を保持
  • 永続メモリ：memory.mdとuser.mdでセッションを超えて記憶
  • 全文検索：過去の会話履歴から関連情報を検索・参照
  • この3層により、複雑なタスク後にナレッジを蓄積してスキルが向上
過去の会話から情報を引っ張ってこれる仕組みが秀逸！

③ 外部メモリ連携

  • SuperMemory：長期記憶の外部連携サービスと接続可能
  • Mem0：王道のメモリ管理サービスにも対応
  • ByteLoverなどの他の外部サービスとも連携
  • 外部APIキーを設定するだけで、ユーザー視点では複雑なことを考えずに長期記憶と接続できる
設定だけで自然に長期記憶が使えるようになります

④ スキルシステム

  • ゲーミング：ポケモンをプレイするスキルなどが存在
  • クリエイティブ：Excalidrawで図を描くスキルが利用可能
  • ソーシャルメディア：Jitter APIでX（旧Twitter）を操作するスキル
  • Apple系のスキルも存在し、OpenClawから持ってきたものもある
  • 公式ドキュメントのスキルズセクションで一覧確認可能

⑤ OpenClawとの比較

  • OpenClaw：リリース当初はハック的な作りが目立った
  • Hermes：王道でしっかり作り込まれた印象。よりスマート
  • どちらも「やるべきことを淡々とやる」方向性だが、作りの丁寧さが違う
  • OpenClawは先に市場を開拓した存在だが、現在では雑な部分も改善されている
まさおさん的にはHermesの方が好みとのこと！

⑥ CLIインターフェース

  • Claude Codeのようなチャット形式で操作できるCLIモード
  • コンテキストウィンドウが表示され、現在の状況をリアルタイムで確認可能
  • Claude CodeやOpen Codeの使い方に似ているので、開発者には使いやすい
  • ヘルプコマンドも充実しており、初心者でも確認しながら進められる
開発者視点で使い勝手が良いと高評価！

⑦ セットアップ手順

  • クイックインストールをポチッと押すだけで簡単に導入可能
  • フルセットアップでは対話形式で設定を進められる
  • バックエンドオプションとしてローカルやクラウドを選択可能
  • イテレーションの最大数やコンテキスト圧縮率なども設定できる
対話形式で進むので初めてでも安心！

⑧ 対応プロバイダー

  • OpenAI：最も無難な選択肢のひとつ
  • OpenRouter：複数モデルへのアクセスが可能
  • Anthropic：Claude Codeオプションもある（ただしサードパーティは禁止）
  • Google、Groq、GitHub Copilotも対応
  • カスタムエンドポイントでローカルLLMも利用可能

⑨ チャット連携

  • Discord：ボットとして連携可能。ゲートウェイコマンドで簡単設定
  • Slack：チーム内のAIアシスタントとして活用
  • Telegram：個人利用のボットとしても運用可能
  • セットアップ中にどれか選ぶだけで、すぐに接続が完了
秘書やボットとして低コストで運用できます！

⑩ 公式ドキュメントとメモリ設定

  • 公式ドキュメント：hermes-agent.nousresearch.com
  • メモリセットアップコマンドで、APIキーの入力や関連パッケージのインストールが可能
  • コマンドを一時的にでも覚えておかないと操作に戸惑う場面も
  • やはり公式ドキュメントを読むのが一番とのこと
ヘルメスチャットモードで質問するのもアリ！

⑪ 自己改善の仕組み

  • 複雑なタスクをこなした後にナレッジを蓄積していく学習ループ
  • マークダウンスキルで知識を整理し、スキルの向上を意識した設計
  • 使い込むほど自動で進化していく方向性がこのエージェントの強み
  • クローン機能や並列動作にも対応している
使い込むほど賢くなるのが本質！

⑫ GUIワークスペース

  • Hermes Workspace：GUIからの操作も一応可能
  • ただし、CLIからの操作がメインで、普段はCLIで十分に活用できる
  • スキルズ画面でスキル一覧を確認可能
  • 第三者が使いやすいためのGUIも提供されている

⑬ ブラウザツール連携

  • BrowserUse：ブラウザ操作のAPIを挿して自動化が可能
  • リモートブラウザー：クラウド上でブラウザを操作する機能
  • ライブセッションでブラウザの操作状況をリアルタイムで閲覧可能
  • 低スペックPCでもリモートブラウザーを使えばボットや秘書を構築できる
自然にブラウザツールが組み込まれているのが特徴！

⑭ 注意点・利用上の留意事項

  • Claude Code経由のサードパーティ利用はAnthropicによって禁止されている
  • 利用規約をしっかり読んで、自分の責任範囲内で利用することが重要
  • APIキーで使うのが基本。無難なのはOpenAIかOpenRouter
  • Hermesの情報共有はまだ少ないので、キャッチアップが少し大変

⑮ ソースコードの品質

  • ソースコードは量が多く、かなり作り込まれている
  • 内部実装を見ると、複雑だがしっかり設計されていることがわかる
  • コードを読み込むことで、AIエージェント設計の参考にもなる
  • 全部を一度に理解するのは難しいが、少しずつ使って学ぶのがおすすめ
めっちゃいいと思います！ — まさおさん

⑯ 最新トレンドとの関係

  • 自己学習して自立進化していくのが最新のAIトレンド
  • 各種研究やセンチベースのアプローチが盛んに
  • ハーネスエンジニアリング周辺のコミュニティでも注目
  • 情報が増えていく中で、キャッチアップして進化させられるかが重要なポイント

⑰ イベントフックと自動化

  • イベントフックで特定のタイミングに自動処理を仕込める
  • クローン機能でエージェントを複製可能
  • 並列での動作もサポートしており、複数タスクの同時実行が可能
  • コンテキスト圧縮率は0.5が標準設定。最近はコンテキストサイズも伸びてきた

⑱ まとめと総評

  • 基本欲しいツールが一通り揃っており、低コストで進められる
  • OpenClawよりも「しっかり作られている」という印象でおすすめ
  • OpenClawが市場を開拓した後を追い、より洗練された形で登場した存在
  • 汎用AIエージェントに興味があるなら、ぜひ試してみてほしいとのこと
