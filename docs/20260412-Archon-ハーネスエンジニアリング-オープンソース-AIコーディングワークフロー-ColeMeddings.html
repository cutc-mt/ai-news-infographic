Archon - オープンソースのハーネスビルダー

AIコーディングエージェントを wraps する独自ワークフローを構築
📅 2026年4月12日

①Archonとは: オープンソースのハーネスビルダー

  • Archonは、AIコーディングエージェントの上に構築する独自のハーネス（制御レイヤー）を誰でも作れるオープンソースツールです。
  • Cole氏が中心となり、RasmusとThomasの協力で開発。Claude Codeを使ってArchon自身を構築しています。
  • Stripe MinionsやRalph Loopのような社内ハーネスを、自分好みにカスタマイズして構築できるのが特徴です。
大企業は独自のハーネスを内部で構築している。Archonはそれを誰でも使えるようにする！

②進化の歴史: プロンプト → コンテキスト → ハーネス

  • 2022年: プロンプトエンジニアリング — GPT-3.5 Turbo登場。LLMにどう指示するかが鍵でした。
  • 2025年: コンテキストエンジニアリング — セッション全体の文脈を管理。Tobias LütkeやAndrej Karpathyが提唱。
  • 2026年: ハーネスエンジニアリング — コーディングエージェントを包み込む戦略レイヤーを構築する時代へ。
それぞれの進化は置き換えではなく積み重ね。プロンプトはコンテキストの一部、コンテキストはハーネスの一部です。

③ハーネスエンジニアリングの本質

  • ハーネスとは、コーディングエージェントの上に構築するツールとプロセスのレイヤーです。
  • モデル自体を改善するのではなく、エージェントを信頼性の高いプロセスでwrapします。
  • 同じモデルでも、ハーネスを構築すればPR承認率が6.7%からほぼ100%に跳ね上がったという研究結果もあります。
Stripeは社内ハーネスで週1,300件のAI生成PRをマージしている！

④Archonワークフローの全体像

  • トリガー — CLI、Web UI、Slackなどからワークフローを開始。
  • 計画 — コーディングエージェントにリサーチと計画を作成させる。
  • 実装 — 新しいセッションで実装を行う（バイアスを避けるため）。
  • 検証 — 決定論的なテスト実行で品質を担保。
  • レビュー → PR — 人間が承認してからプルリクを作成。

⑤PIVループ: 計画・実装・検証のサイクル

  • Plan（計画）: 特徴を分析し、リサーチを行って実装計画を立案する。
  • Implement（実装）: 計画に基づいてコードを書く。新しいセッションで行うのがポイント。
  • Validate（検証）: テストを実行して品質を確認。失敗したらフィードバックループで修正。
計画セッションでコンテキストが肥大化するとバイアスが蓄積するので、実装は必ず新しいセッションで！

⑥決定論的ノード: AIを使わない確実なステップ

  • Archonのワークフローでは、AIモデルを使わないノードを定義できます。
  • 対応スクリプト: Bash、Python、TypeScript
  • 「テストを実行して」と指示してもコーディングエージェントが無視する問題を、強制的に実行して解決します。
コーディングエージェントに「テストを実行して」と言っても無視されることがある。決定論的ノードなら確実に実行される！

⑦Human-in-the-Loop: 人間が関与する承認プロセス

  • ワークフローの任意の地点に人間の承認ゲートを挟める。
  • Web UI、CLI、Slackのすべてのインターフェースで承認に対応。
  • Ralph LoopのようなVibe Codingの問題（初期のエラーが増幅する）を防止。
長時間の自動実行でも、人間がいつでも介入できるのがArchonの強み。完全自動化ではなく、信頼できるプロセスを構築する。

⑧並列実行とWorktreeによる安全な処理

  • ArchonはWorktreeを使って複数のタスクを安全に並列実行します。
  • 8つのGitHubイシューを同時に修正しても、互いに干渉しない。
  • マージコンフリクトを気にせず、各タスクが独立してプルリクを作成。
Claude Codeから「この8つのイシューをArchonで修正して」と指示するだけで、全て並列で処理される！

⑨Web UI: ブラウザからワークフローを管理

  • ブラウザ上ですべてのワークフローを管理できるWebインターフェース。
  • リアルタイムでログを確認しながら、エージェントのツール呼び出しを監視可能。
  • チャット形式でエージェントに指示を出し、リポジトリを自動的にルーティング。
VPSにデプロイすれば、リモートからでもブラウザでワークフローを操作できる！

⑩複数のインターフェース: CLI / Web / Slack

  • CLI: Claude CodeやCodexから直接Archonを呼び出し、裏でタスクを委任。
  • Web UI: ブラウザ上で視覚的にワークフローを管理・実行。
  • Slack: スレッド単位で会話を管理。「このイシューを修正して」と指示可能。
すべてのプラットフォームで並列実行をサポート。Worktreeによる分離は自動で処理される。

⑪ノードごとのモデル選択機能

  • ワークフローの各ノードごとに使用するモデルを指定可能。
  • 分類や調査にはHaikuを使い、実装にはOpusを使うといった最適化ができる。
  • 必要なステップだけで高価なモデルを使うので、トークンコストを大幅に削減。
将来的には、エージェントが自動的にモデルを選択する機能も検討中！タスクの複雑さに応じて最適なモデルを割り当てる。

⑫トークン効率: ハーネスがモデルを超える

  • Archon + Sonnet の組み合わせが、単独の Opus よりも良い結果を出したという報告。
  • Anthropicのレート制限が厳しくなった中で、Sonnet中心の運用が現実的な選択に。
  • GSDワークフロー全体をSonnetで実行しても、Opusを1回使うより安価。
ハーネスがモデルを底上げする。高価なモデルに頼るよりも、包括的なプロセスで品質を担保するアプローチが勝つ！

⑬対応コーディングエージェント

  • Claude Code: Claude Agent SDKを通じてプログラム的に実行。完全対応。
  • Codex: OpenAIのSDK対応。ほぼ完成。
  • Pi: Pi Agent SDKのサポートを開発中。任意のモデルを実行可能に。
  • AMPやOpen Codeなどにも対応を拡大する計画。
Archonはコーディングエージェントの上のレイヤーなので、どのエージェントでも使えるのが理想！

⑭Anthropicサブスクリプションをそのまま利用可能

  • ArchonはClaude Agent SDKを使ってClaude Codeをプログラム的に実行。
  • 個人利用であれば、Anthropicのサブスクリプションをそのまま使える（Boris Churny氏が明言）。
  • OpenClawやOpenCodeのようなサードパーティの回避策とは異なり、規約違反にならない。
Cole氏は何ヶ月も問題なくサブスクリプションをArchonで使用中。ただし個人利用に限る。APIクレジットは不要！

⑮GSDワークフロー実例: 既存のハーネスを再現

  • 4つの並列リサーチエージェントが同時に調査を実行（技術スタック、依存関係、機能など）。
  • リサーチ結果を統合ドキュメントにまとめて、要件定義に進む。
  • 要件定義後に人間の承認ゲートで確認し、実装へ移行。
BMADやGSDのような既存のハーネスをArchonで再構築できる。自分好みにカスタマイズするのがポイント！

⑯YAML形式のワークフロー定義

  • ワークフローはYAML形式で定義。シンプルな構造で拡張性が高い。
  • コーディングエージェントに「Archon用のワークフローを作って」と指示すれば自動で生成。
  • ループや条件分岐、並列実行にも対応。フィードバックループも組み込める。 
# ワークフローの例
nodes:
  - id: plan
    type: agent
    model: sonnet
    prompt: "plan.md"
  - id: implement
    type: agent
    depends_on: [plan]
  - id: test
    type: script
    command: "npm test"

  • このライブストリームでは時間切れ。次回のライブストリームまたはYouTube動画で詳しく解説予定。
