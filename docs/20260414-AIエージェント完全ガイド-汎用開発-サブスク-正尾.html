汎用と開発の分け方

① 汎用と開発で分ける考え方

  • 汎用AIエージェント: 日常のタスクや秘書としての役割を担当
  • 開発AIエージェント: プログラミングやアプリ開発に特化
  • マクロ（全体管理）とミクロ（特定タスク）に分けて運用するのがおすすめ
  • 一つのエージェントに全てを任せると注意力が有限で破綻する
汎用と開発を分けるのが第一歩！一つに全部詰め込むのは NG です

② 汎用AIエージェントの役割

  • 自分の情報を覚えてもらうメモリシステム
  • カレンダーや予定の管理と提案
  • 会社に関する情報を覚えておく秘書機能
  • 自分好みの人格を育ててゲーム感覚で活用
  • マクロ側からミクロ側へ仕事を流す dispatch 機能

③ 開発AIエージェントの役割

  • アプリやWebサービスの開発に特化
  • プランモードでの設計から実装まで一貫して対応
  • バグ修正やリファクタリングも効率的に実行
  • 特定のタスクにドリルダウンして深く掘り下げる
開発は専用ツールに任せた方が、特化している分だけ効率が良いです！

④ 一つに求めないことが重要

  • 「開発もできて秘書もできる」を一つに求めない
  • AIエージェントは注意力が有限で、全部入れると破綻する
  • 特定のタスクはミクロ側に切り出すのが正解
  • マクロからミクロへ仕事を流すフローを構築する
全部入りを求めると結局どれも中途半端になる！適材適所がコツです
汎用AIエージェントのおすすめ

⑤ Hermes Agent（ヘルメス）

  • 汎用AIエージェントで一番おすすめ
  • Claude CodeのCLIのようなチャットベースで操作
  • 自立改善とメモリシステムをスマートに実装
  • 最新のトレンドを本質的に捉えた設計
  • 中級者〜上級者向け（CLIやターミナルに慣れている人向け）
作りが本当に良い！AIエージェントの本質的な要素を押さえています

⑥ OpenChore ChromeX（入門向け）

  • 入門者にはデスクトップアプリがおすすめ
  • インストールするだけですぐに使い始められる
  • OpenChoreの面倒なセットアップを簡略化
  • 慣れてきたらHermesに移行するのが理想
初心者はまず ChromeX のデスクトップアプリから始めるのがおすすめです！

⑦ Claude Agent SDKで自作

  • Claude Agent SDKを使って自作の汎用AIエージェントを構築可能
  • 自分の好きなようにカスタマイズできる
  • メリット: 自由度が高い / デメリット: 開発とメンテの負担が増える
  • 自分で作るよりセンスの良い開発者に任せる方が効率的な場合も

⑧ DiscordやSlackとの連携

  • DiscordやSlackにエージェントを常駐させてタスクを実行
  • Hermesを使うか、自作エージェントを連携させる方法がある
  • Anthropicが第三者でのClaude使用を制限しているため、汎用側ではCodexモデルの使用が現実的
Claudeのサブスクは公式以外で使えない方針なので、汎用側はCodexベースが良いです

Claude Code + Opus 4.6 の組み合わせが現在最強です！ハーネスエンジニアリングが卓越しています

無料で始めるなら Anti-gravity ですが、微課金から始めるのが圧倒的におすすめです
