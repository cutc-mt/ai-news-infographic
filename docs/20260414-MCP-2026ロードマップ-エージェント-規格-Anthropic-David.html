MCP創設者が明かす 2026年のAIエージェントロードマップ

2026年4月14日 | Anthropic - David (MCP共同創設者) 基調講演

①驚異的な成長: 月1.1億ダウンロードを達成

  • MCP SDKの月間ダウンロード数が1.1億以上に到達
  • Reactが普及に3年かかったのに対し、MCPはわずか16ヶ月で到達
  • オープンソースプロトコルとして歴史上最速の普及速度を記録
エージェント時代の「共通言語」として、React超えのスピードで世界中に広がっている！

②2024年11月のローンチと初期サーバー群

  • 2024年11月にMCPプロトコルを正式リリース
  • 初期のリファレンスサーバーとしてPostgres、SQLiteを提供
  • ファイルシステムとGitの統合サーバーも同時公開
  • 開発者がすぐに使える基盤を最初から用意した設計思想
シンプルな「始めやすさ」が、爆発的な普及の原動力になった！

③主要SaaSとの連携が続々と実現

  • Clickhouseとの連携でリアルタイムデータ分析が可能に
  • Notionと統合してナレッジベースへのアクセスを実現
  • Zapier経由で数千のアプリと自動連携できるように
エコシステムが急速に拡大して、実用的なツール連携がどんどん増えている！

④コミュニティの創造的な活用事例

  • Blenderと連携して3DモデリングをAIエージェントに操作させる
  • Abletonと接続して音楽制作をAIにアシストさせる試み
  • 3Dプリンターの制御をエージェント経由で実行
  • ファンタジープレミアリーグの管理までMCPで構築するユーザーも
公方が想定していなかった用途まで次々と誕生しているのがコミュニティの力！

⑤エンタープライズ導入と社内での活用

  • Salesforce CRMやJira、Snowflakeを企業ファイアウォール内で統合
  • セキュリティ要件の厳しい大企業でも導入が進んでいる
  • Anthropic社内でもナレッジベースとSlackをMCPサーバーとして構築
  • 自社製品を「dogfooding」して日常的に品質を検証
ファイアウォール内で動くのが特徴で、開発者自身も毎日使っているから改善が早い！

⑥プロトコルの進化: stdioからセキュリティまで

  • 当初はstdioによるローカル通信からスタート
  • 次にリモートサーバーへの接続機能を追加
  • 認可（Authorization）の仕組みをプロトコルに組み込み
  • セキュリティモデルの強化でエンタープライズ要件に対応
シンプルな仕組みから始めて、段階的に複雑な機能を追加する慎重な進化アプローチ！

⑦新しいプリミティブ: ヒアリング、構造化出力、長時間タスク

  • Elicitation（ヒアリング）でエージェントがユーザーに質問できる仕組み
  • 構造化出力（Structured Outputs）でデータの形式を厳密に制御
  • 長時間実行タスクを管理し、エージェント同士が非同期で通信可能に
  • タスクの進捗確認や中断、再開をプロトコルレベルでサポート
「分からないことは聞く」「時間のかかる処理も管理する」でエージェントが格段に賢くなる！

⑧SDKのティアリング: 安定性の明確化

  • 各SDKの安定性保証レベルを明確に tier 分け
  • 本番環境で使える「安定版」と実験的な「プレビュー版」を明示
  • 企業が導入を判断する際のリスク評価がしやすくなる
「どのSDKが本番で使えるか」が一目で分かるようになるのが嬉しい改善！

⑨MCP Apps: インタラクティブUIの提供

  • MCPサーバーが対話型UIをクライアントに提供する新しい概念
  • 単なるテキスト応答ではなく、リッチな操作画面を配信可能
  • エージェントの操作体験を飛躍的に向上させるパラダイムシフト
エージェントとのやり取りが「チャット」からフルUIへ進化する！

⑩Agentic AI財団への寄贈と中立的ガバナンス

  • MCPプロトコルをAgentic AI Foundationに寄贈
  • 単一企業に依存しない中立的なガバナンス体制に移行
  • 業界全体で標準プロトコルとして信頼できる基盤を構築
Anthropicが手放すことで、業界標準としての信頼性が一気に高まる！

⑪2026年の焦点: エンタープライズの本番運用

  • 2026年のテーマは「本番で動くエージェントシステム」
  • 実験段階から「企業で実際に使える」レベルへの移行を目指す
  • 信頼性、セキュリティ、スケーラビリティの3本柱を強化
「デモで動く」から「本番で動く」への移行が、今年最大の目標！

⑫トランスポートの進化: ステートレススケーリング

  • ステートレス通信でハイパースケーラーでの動作に対応
  • 2026年6月に仕様改訂を予定
  • AWS、Azure、GCPなど大規模インフラ上で効率よく動作する設計
数百万リクエストを捌くクラウド規模でもMCPが使えるようになる！

⑬クロスアプリアクセス: SSOのシームレス化

  • 企業のSSO（シングルサインオン）をシームレスに統合
  • 従来のOAuth認可フローなしでアプリ間連携が可能に
  • 異なるアプリケーション間でのスムーズなデータアクセスを実現
面倒な認証画面が不要になって、エージェントがシームレスに連携できる！

⑭トリガー（Webhook）: プロアクティブな通知

  • MCPサーバーが自発的にクライアントに通知する仕組み
  • 従来はクライアントからの要求に答えるだけだったのが、双方向に
  • 新しいデータが届いたら即座にエージェントが反応できるようになる
エージェントが「待つ」だけでなく、自ら動き出すための基盤！

⑮ネイティブストリーミング: ツール結果の段階的配信

  • ツールの実行結果を少しずつ段階的に配信する仕組み
  • 長時間の処理でもユーザーがリアルタイムに進捗を確認可能
  • ユーザー体験の大幅な向上が期待できる機能
全部終わるまで待つ必要がなくなる！インクリメンタルな結果表示で体感速度が向上！

⑯スキル配信: ドメイン知識をMCPでバンドル

  • ドメイン特化の知識をMCPサーバーと一緒に配信する仕組み
  • 数週間以内に拡張機能としてリリース予定
  • 「ツール」だけでなく「知識」もパッケージ化して配れるようになる
ツールの使い方だけでなく、ドメインの専門知識もエージェントに教えられる！

⑰SDK v2とプログレッシブディスカバリー

  • Python SDK v2とTypeScript SDK v2で使い勝手を大幅改善
  • 開発者のフィードバックを反映したエルゴノミクス改善
  • プログレッシブディスカバリーで必要なツールだけを検索して取得
  • コンテキストブロート（情報肥大化）問題を根本的に解決
SDKがもっと簡単になって、ツールが数百あっても検索で最適化できる！

⑱コンポーザビリティとコミュニティ駆動の進化

  • 構造化出力を使ってMCP呼び出し同士をチェーンで連結可能に
  • あるツールの出力を次のツールの入力として自動で接続
  • コミュニティからのリアルなフィードバックが仕様改善の原動力
  • 「ユーザーのためにユーザーと共に」作るオープンな開発文化
レゴのようにツールを組み合わせて、コミュニティの声で進化し続けるプロトコル！
