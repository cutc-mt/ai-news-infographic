Hermes Agent V0.9.0 - 大規模アップデート完全解説

ローカルWebダッシュボード・Android対応・メッセージング統合・高速モード
2026年4月15日 | AICodeKing

① V0.9.0リリース: プロジェクト史上最大のアップデート

  • 2026年4月13日にリリースされた V0.9.0
  • プロジェクト史上最も重要なアップデートの一つと評価
  • 単なるバグ修正ではなく、使い方と利用場所そのものを変える内容
  • オープンソースエージェントから完全なエコシステムへの進化
  • ユーザビリティ・移植性・アーキテクチャ・セキュリティの全てを改善
V0.9はHermesを「クールな実験」から「本格的なクロスプラットフォームエージェントシステム」へと昇格させるリリースです

② ローカルWebダッシュボード: 設定管理が劇的に簡単に

  • ブラウザ上で設定・セッション・スキル・ゲートウェイを管理可能に
  • これまでYAMLファイルを手作業で編集する必要があった設定をGUIで操作
  • セッションの監視、スキルの参照、ゲートウェイ管理もダッシュボードから
  • ローカルダッシュボードなのでクラウドへの強制移行なし
  • 新規ユーザーにとって最も重要な機能と言える
エージェントツールの最大の弱点は「設定が面倒」なこと。これが解消されます！

③ TermuxでAndroid対応: モバイルでHermesを利用

  • Termux経由でAndroid上でネイティブ動作が可能に
  • 小さい画面向けの最適化、TUIのモバイル対応
  • 音声バックエンドサポートで音声操作にも対応
  • スラッシュコマンドで画像生成もデバイス上で実行可能
  • モバイルルートは最速ではないが、オープンソースの携帯エージェントとして貴重
電話やタブレットからエージェントを利用できるのは、オープンソースとしては画期的！

④ iMessage対応: BlueBubblesでAppleメッセージと連携

  • BlueBubblesプラグイン経由でiMessageに接続
  • HermesがAppleのメッセージング世界にアクセス可能に
  • コールバックウェブフックで双方向通信をサポート
  • メッセージングベースのワークフローがより実用的に
  • DiscordやSlackに加えて、さらに柔軟なコミュニケーションチャネルを追加
実際のコミュニケーションが行われる場所でエージェントを利用できるのが最大の強み

⑤ WeChat・WeCom対応: 中国市場のツールにも対応

  • WeChatとWeComのコールバックサポートを追加
  • 多くのオープンソースツールが中国のツールを無視している中、貴重な対応
  • Mattermost、Home Assistant、DingTalk、Feishuなどもサポート
  • これによりHermesは合計16プラットフォームに対応
  • グローバルなチームや専門的なワークフローに対応する柔軟性を獲得
Telegram、Discord、Slack、WhatsApp、Signal、Matrixなど、主要プラットフォームのほぼ全てに対応！

⑥ Fast Mode: OpenAIとAnthropicの低遅延アクセス

  • スラッシュコマンド /fast で高速モードを有効化
  • GPT-5.4、Codex、Claudeなどの対応モデルで低遅延アクセス
  • OpenAIとAnthropicモデルを優先キュー・高速キューにルーティング
  • 各ステップの待ち時間を大幅に短縮し、素早い反復に最適
  • 高速モードは最安ルートではないが、スピード重視の用途には非常に有効
多くのクイックな操作を繰り返す場合や、メッセージング内でのやり取りでは低遅延が非常に重要！

⑦ バックグラウンドプロセス監視: 長時間タスクをリアルタイム追跡

  • Watchパターンでプロセス出力を監視し、重要なイベントを通知
  • サーバーがポートをリッスン開始した時やビルド失敗時に自動通知
  • 特定のエラーテキストや成功メッセージが表示されたら即座に反応
  • 長時間タスクの後に手動で確認する手間が不要に
  • 適切なタイミングで通知が届くので、作業の平行管理が効率化
バックグラウンドでタスクを開始して、結果が出たら通知。これがエージェントのあるべき姿！

⑧ プラガブルコンテキストエンジン: カスタムコンテキスト管理

  • コンテキスト管理をプラグイン可能なスロットとして設計
  • フィルタリング、要約、ドメイン固有のコンテキスト注入が可能に
  • コンテキストが圧縮されすぎていると重要な情報が欠落する問題に対処
  • 適切な情報を適切なタイミングで注入する仕組みを実現
  • 独自のカスタムロジックをプラグインとして差し替え可能
モデルが各ターンで何を見るかが変わると、エージェント全体が劇的に良くなります

⑨ セキュリティ強化: プロジェクト史上最深のハードニング

  • パストラバーサル保護でファイルシステムへの不正アクセスを防止
  • シェルインジェクション防止でコマンド実行の脆弱性を修正
  • Slack画像アップロードのSSRFリダイレクトガードを追加
  • Twilioウェブフック署名検証、APIキー認証、ボタン認可を実装
  • Git引数のサンドボックス化、サーバー認証強制、コマンドアクセス制御
複数プラットフォームで利用するなら、セキュリティはオプションではなく必須！

⑩ バックアップと復元: 設定の移植が簡単に

  • hermes backupとhermes importコマンドを追加
  • プロバイダー設定、ゲートウェイ設定、メモリをバックアップ可能
  • 別のマシンへの移行や新しい設定の試行が安全に
  • 環境構築に時間をかけた設定を一括で復元
  • Hermesをよりポータブルで管理しやすいツールに昇格
設定を全部調整した後に別マシンに移行したくなっても、バックアップがあれば安心！

⑪ デバッグツールキット: トラブルシューティングが劇的に改善

  • /debugコマンドとhermes debug shareを新規追加
  • エラー分類、フォールバック処理、モデル切り替え動作の改善
  • デバッグ情報を共有して問題を迅速に特定・解決
  • クラッシュ耐性の向上で、安定性が大幅に改善
  • 高度なユーザーにとって特に役立つ機能
強力なツールでもデバッグが困難なら、多くの人は使うのを諦めてしまいます。この改善は非常に重要！

⑫ プロバイダー柔軟性: 複数APIの選択肢を維持

  • 一つのAPIエコシステムにロックインされない柔軟性
  • OpenAI、Anthropicに加えてXAI（Grok）のネイティブサポート
  • Xiaomi MiMOのネイティブサポートも追加
  • プロバイダーが不安定な場合のフォールバック動作を改善
  • ローカル重視、スピード重視、コスト重視など柔軟なルート選択が可能
モデルを次々と試したい場合や、コストを最小限に抑えたい場合にも柔軟に対応！

⑬ エージェントの「実用性」への転換: 自動補完からアシスタントへ

  • Hermes V0.9はエージェントを本物のアシスタントに近づける
  • 「おしゃれな自動補完＋余計な手順」から脱却する方向性
  • 実際のコミュニケーションが発生する場所で利用できることが重要
  • メッセージング、ローカル、クラウドなど複数の利用形態を統合
  • 多くのオープンソース製品が「洗練された感じ」を出せていない中で差別化
ツールへのアクセスを与え、統合を提供し、セキュリティも確保。これぞ真のエージェント！

⑭ セットアップウィザード: 初期設定の簡素化

  • セットアップウィザードの統合で初期設定がより直感的に
  • 環境変数、設定ファイル、スキル、セッション状態の管理を統合
  • 多くの可動部分があるため、セットアップが面倒になりがちな問題に対応
  • 設定の摩擦を減らすことでツールの継続利用率が向上
  • ターミナルに慣れているユーザーは引き続きCLIも利用可能
設定ファイルを一生編集する必要はもうありません。ダッシュボードとウィザードで完結！

⑮ 16プラットフォーム対応: 幅広いコミュニケーションチャネル

  • Telegram、Discord、Slack、WhatsApp、Signal
  • Matrix、メール、SMS、Mattermost
  • DingTalk、Feishu、Home Assistant
  • WeChat、WeCom（コールバックサポート）
  • iMessage（BlueBubbles経由）
合計16プラットフォームに対応。グローバルなコミュニケーションツールをほぼ網羅！

⑯ コストとパフォーマンスのバランス: 自分に合ったルートを選択

  • コストを最小限に抑えたい場合はローカルルートが最適
  • スピード重視ならFast ModeでOpenAI/Anthropicを高速利用
  • ローカル重視でもメッセージング重視でも、強制的に一つに絞られない
  • 無料プロバイダールートも利用可能だが、Fast Modeは別ルート
  • 自分のユースケースに合わせて最適なバランスを選べる柔軟性
Hermesは一つのパスに強制しません。ローカル、メッセージング、スピード、コストから自由に選べます

⑰ 構造化API応答の改善: 実用性のさらなる向上

  • Hermesが構造化API応答の処理を改善
  • リアルワールドでのバリアント処理がより安定
  • エラー時のフォールバック処理がより堅牢に
  • モデル間の切り替え動作も改善され、プロバイダー間の移行がスムーズに
  • 全体的な安定性の向上で、長時間のワークフローにも安心して利用可能
バックエンドでの改善は地味ですが、日常的な使用体験を大きく向上させます

⑱ 総評: クロスプラットフォームエージェントとして本格始動

  • V0.9は単一の目立つ機能ではなく、全体の成熟度を高めるリリース
  • ダッシュボードとプラットフォーム拡大が最大のインパクト
  • セキュリティハードニングはプロジェクト史上最深レベル
  • Hermesはより柔軟でポータブルなオープンソースエージェントに成長
  • 完璧ではないが、実用的な作業がしっかり進んでおり、高く評価できる
Hermesはシリアスなクロスプラットフォームエージェントシステムへの道を歩み始めました。今後の進化が楽しみ！
