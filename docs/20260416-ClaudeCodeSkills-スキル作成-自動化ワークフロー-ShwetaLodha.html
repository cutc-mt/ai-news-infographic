Claude Code Skills - スキルを作ってワークフローを自動化

2026年4月16日 | Shweta Lodha

①Claude Code Skillsとは

  • 再利用可能な指示として、一度作成すれば何度でも使えるワークフロー
  • Claudeに新しい能力を教え込み、自動化や制約付きの処理を実現する
  • プロジェクトのトーンやブランド、アーキテクチャガイドラインに合わせて動作可能
  • AIエージェントのようなツールとしても活用できる
指示を書くだけで、Claudeがスキルを自動認識して実行するよ！

②スキルの基本的な仕組み

  • 各スキルは skill.md ファイルで定義される
  • ファイルにはメタデータ（名前・説明）と実行指示を記述
  • フォルダ構造で管理：.claude/skills/{スキル名}/
  • スクリプトやテンプレートも紐付け可能
Markdownファイル1つでスキルが完成するシンプルさ！

③理由①: 繰り返しタスクの自動化

  • よく使うワークフローをワンショットコマンドに変換できる
  • フォーマット処理や要約を自動化
  • データのクリーニングやコードパターンの生成も可能
  • 同じ作業を何度もプロンプトで書く必要がなくなる
面倒な繰り返し作業を一発で実行できるように！

④理由②: 新しい機能の追加

  • デフォルトではスクリプトを実行しないClaudeを拡張できる
  • API呼び出しやデータベースクエリを実行可能に
  • 特定のアクションをトリガーする機能を追加
  • 独自のツールやパイプラインとの統合が実現
Claudeのデフォルト機能を超えたカスタム能力を賦与！

⑤理由③: プロンプトのオーバーヘッド削減

  • スキルなしではすべてがプロンプトの一部になる
  • 長い指示を再利用可能なスキルにラップしてTokenを節約
  • ユーザーのクエリに応じて自動的にスキルを呼び出し
  • コンテキストウィンドウを大幅に削減できる
長いプロンプトを書き続けるとToken消費が増大するよ！

⑥理由④: マルチステップ推論の有効化

  • スキルを使うと複数のアクションをチェーンできる
  • 例：データ取得 → 分析 → クエリ作成 → ユーザー通知
  • 一連の処理をワークフローとして順次実行可能
  • スキルなしでは現在この連鎖実行ができない
複数ステップを自動でつなげて実行できるのが最大の強み！

⑦理由⑤: 実際のワークフローへの統合

  • 開発者・チーム・クリエイターに特に有用
  • Claudeを既存のシステムやパイプラインと連携させる
  • すでに稼働中のアプリケーションに統合可能
  • スピードと生産性が向上する
実際の開発現場のワークフローにClaudeを組み込める！

⑧理由⑥⑦: 生産性向上とカスタム動作

  • AIエージェントのように使えてスピードと生産性が向上
  • プロジェクト固有のカスタム動作を定義できる
  • Claudeの応答方法や思考プロセスを制御可能
  • 特定の環境での振る舞いをカスタマイズ
Claudeがどう考え、どう応答するかまで設定できる！

⑨スキル作成ステップ①: フォルダ構造

  • 1 プロジェクト直下に .claude フォルダを作成
  • 2 その中に skills フォルダを作成
  • 3 スキル名のフォルダ（例：greet-skill）を作成
  • 4 その中に skill.md ファイルを作成
📁 .claude/
  📁 skills/
    📁 greet-skill/
      📄 skill.md
      📁 scripts/
        📄 get_current_datetime.py
      📄 template.md

⑩スキル作成ステップ②: メタデータ定義

  • name: スキルの名前を定義（例：greet）
  • description: スキルの説明を簡潔に記述
  • できるだけ短く書くのが重要（Token削減のため）
  • 長すぎるとトークン消費が増大するので注意
---
name: greet
description: "A simple skill to greet user
  when they say hello or hi"
---
メタデータは「名前」と「説明」の2つが最小構成！

⑪スキル作成ステップ③: トリガーと応答の定義

  • トリガー: スキルを起動する条件を定義
  • 例：「ユーザーが hello または hi と言ったとき」
  • 応答: Claudeがどう返答するかを定義
  • レスポンスのトーンや内容を完全に制御可能
Use this skill whenever user greets
with hello or hi.

Respond with: "It's nice to meet you.
Congratulations, you are using your
very first skill!"

⑫スキルの動作確認方法

  • VS CodeのClaude Code拡張機能をインストール
  • チャットで「What skills do you have?」と質問
  • 内蔵スキルと自作スキルの両方が表示される
  • 「hello」と入力してgreetスキルが自動起動するか確認
自作スキルがリストに表示されたら成功！

⑬Pythonスクリプトの実行

  • skill.mdの中にPythonコードを直接記述可能
  • 例：現在の日時を取得して応答に追加
  • スクリプト実行時は許可プロンプトが表示される
  • Node.jsやJavaScriptなど他の言語も対応
Run the following Python code before
every single message and append the
output to your message.

```python
from datetime import datetime
print(datetime.now().strftime(
  "%Y-%m-%d %H:%M:%S"))
```

⑭スクリプトの外部ファイル化

  • コードが長くなるとskill.mdが煩雑になる
  • スキルフォルダ内に scripts/ サブフォルダを作成
  • Pythonファイル（例：get_current_datetime.py）を配置
  • skill.mdから相対パスで参照する
Run the script scripts/get_current_
datetime.py to obtain current date
time before every single message and
append the output to your message.
本番運用では外部ファイル化が必須！

⑮テンプレートの活用

  • template.md をskill.mdと同じ階層に作成
  • 毎回同じ内容でなく、状況に応じたテンプレートを使い分け
  • skill.mdから相対パスでテンプレートを参照
  • 10種類以上のテンプレートを条件付きで使い分けることも可能
Respond with template .template

# template.md
Respond with current date time:
{{output}}

⑯ワークフローの定義

  • スキル内に複数ステップのワークフローを定義
  • 1 初回の挨拶: スクリプトを実行してテンプレートで応答
  • 2 2回目以降: スマイリーだけを返す
  • 条件分岐で異なるアクションを実行可能
# Step 1: First time greet
If user says hello for the first time:
  Run the script scripts/get_current_
  datetime.py to obtain date time
  Respond with template .template

# Step 2: Other greets
Respond with a smiley 😊

⑰プログレッシブ・ディスクロージャー

  • 1 Advertise: メタデータ（名前・説明）だけを読み込み
  • 2 Load: スキルが一致したら詳細指示を読み込み（5,000トークン以下）
  • 3 Read Resources: テンプレートや補助ドキュメントを必要時のみ読み込み
  • 4 Run Scripts: 実際に必要な時だけコードを実行
スキルが10個あっても全ファイルを一度に読まないからToken節約！

⑱まとめ: Skills活用のベストプラクティス

  • メタデータは最小限に：トークン消費を抑える
  • フォルダ構造を活用：スクリプトは外部ファイル化する
  • ワークフローで条件分岐：場面に応じた応答を定義
  • テンプレートを使い分け：複数パターンの応答を準備
  • 段階的読み込みを意識してToken節約
