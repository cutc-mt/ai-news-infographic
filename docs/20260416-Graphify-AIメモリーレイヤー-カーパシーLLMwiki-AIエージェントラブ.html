Graphify: 記憶するAI、進化する思考

From Karpathy's LLM.wiki to Graphify - AIメモリーレイヤーの構築

  • GitHubで公開直後に2万7000スターを一気に獲得
  • Andrej Karpathyさんにも紹介され、海外で話題沸騰中
  • OpenAIの共同創設者が提唱したLLM.wikiの構想を実装したツール
  • アイデアから具体的なツールへと具現化された画期的なプロジェクト
スターヒストリーを見るとぶち上がっている！
今、最も注目のリポジトリだよ

②KarpathyのLLM.wiki構想とは

  • OpenAI共同創設者のAndrej Karpathyさんが提唱したアイデア
  • GitHub Gistに投稿された1つの提案メモから始まった
  • AIが情報を蓄積・整理し続ける永続的な知識基盤層を構築する構想
  • 使えば使うほど知識が深まる仕組みを目指している
LLM.wiki自体は「アプリ」ではなく、概念・アイデアの提案なんだ

③既存AIの「記憶喪失ループ」問題

  • 既存のAIアシスタントはセッションごとに全てを忘却してしまう
  • ファイルの読み込み → 質問 → 回答のたびにコンテキストがリセットされる
  • 膨大なトークンと時間を毎回浪費してしまう課題
  • 「さっきのプロジェクトの続き」と話しかけても、相手はポカンとする状態
コーヒーを入れに行って戻ると、
アシスタントが完全に記憶喪失している…！

④LLM.wikiが目指す永続的知識ベース

  • 都度0から情報を探すのではなく、LLM自身が知識を保有する
  • 持続的で総合的にリンクされた知識ベースを構築・保守する
  • 情報は単に保存されるだけでなく、互いにリンクして矛盾を解消する
  • 知識は雪だるま式に成長し、使えば使うほど賢くなる
理想のパートナー：情報労働者が
喉から手が出るほど欲しい存在だよ

⑤Graphify: AIメモリーレイヤーの誕生

  • コード、文章、動画などを解析してナレッジグラフを作成するツール
  • AIが全体の構造を即座に把握できるようにする
  • ソースコードの理解や研究調査の効率が飛躍的に向上
  • 回答制度の向上とトークン消費の大幅な削減を実現
Graphify = Graph + ify（グラフ化する）
あらゆる情報を知識グラフへ変換！

⑥4種類の構造化データ出力

  • graph.html: インタラクティブなグラフ可視化UI（コミュニティ探索・検索可能）
  • graph_report.md: ノードや関係性の繋がりをまとめた自然言語サマリー
  • graph.json: 永続化データ（再読み込みせずに数瞬間でクエリ可能なグラフDB）
  • cache/hash: 変更されたファイルのみを再処理するキャッシュ機能
4種類の生データが出力されて、
均一な知識ネットワークに統合されるよ

⑦抽出エンジン Path 1: AST抽出

  • 確定的なAST（抽象構文木）抽出を行う
  • ツリー（パーサー）を使用してコード構造を解析する
  • LLM不要でローカル環境で完結する
  • 関数やインポートの関係性を正確に抽出・解析する
AIを使わずにコード構造を解析！
軽量で高速な第一段階だよ

⑧抽出エンジン Path 2: Whisper連携

  • Whisperとの連携による音声・動画データの解析
  • 音声・動画データをローカルでテキスト化する
  • すべてローカル処理でプライバシーにも配慮
  • PNG、MP4などのメディアファイルにも対応
動画や音声も自動でテキスト変換！
ナレッジグラフに組み込まれるよ

⑨抽出エンジン Path 3: 並列抽出

  • Claudeのサブエージェントによる概念・関係性の抽出
  • 並列処理で複数の概念と関係性を同時に抽出する
  • 関係性の設計図を自動で作成する
  • 3段階の抽出エンジンを統合し、ネットワーク化していく
3つのPathを統合すると
意味の繋がりが自然な星団が形成される！

⑩エンベッディングからの脱却

  • 従来のベクターDBは距離に依存した単調なマッピングに過ぎない
  • Graphifyはグラフトポロジー空間を活用する
  • 自密度とLeidenコミュニティ検出で意味の結びつきを自然に形成
  • 情報が星団のように意味の繋がりでまとまる革新的なアプローチ
ベクターDBの「距離だけ」の世界から
意味の関係性の世界へパラダイムシフト！

⑪Leidenコミュニティ検出と情報の格付け

  • Leidenアルゴリズムで意味の星団を自動的に検出
  • Extracted（抽出済み）: ソースに直接記述あり、信頼性が高い
  • Inferred（推論済み）: AIによる推論的結論、Confidence Scoreを算出
  • 情報の出どころと確実性を明確に分離し、ブラックボックス化を防止
人間のレビューが必要なものは
スコアが低く表示されるよ！透明性が高い

⑫対応フォーマット: 20言語＋メディア＋文書

  • 20のプログラミング言語: Python、TypeScript、Goなどに対応
  • 論文・ドキュメント: MD、PDF、DOCXなど
  • メディアファイル: 画像（PNG）、音声、動画（MP4）など
  • あらゆる情報をシームレスに統合して知識ネットワーク化する
コード、論文、動画、画像を
全部ポイポイ放り込むだけでOK！

⑬デフォルトで賢いAIを実現する仕組み
