①OpenClawの最大の課題: メモリ問題

OpenClawはブラウジング、コード検索、ツール利用、タスク実行など、エージェントとしてできることは非常に優秀です。しかし、長期間使っていると重要なコンテキストを忘れたり、古いノートから間違った情報を取得してしまう問題が発生します。これにより、ワークフロー全体の信頼性が大きく低下していました。

エージェントの「能力」ではなく「記憶」に課題があったんです！

②ByteRoverとは: ステートフルメモリスキル

ByteRoverはOpenClawの公式プラグインとして統合されたステートフルメモリスキルです。フラットなノートやキーワードマッチ、一般的なベクトル検索に頼るのではなく、階層的なツリー構造でメモリを整理・管理します。エージェントが適切にクエリでき、人間が実際に確認できるメモリシステムを実現しています。

  • 公式OpenClawプラグインからインストール可能
  • フラットなノートからツリー構造へ進化
  • エージェントと人間の両方からアクセス可能

③メモリのキュレーション: ツリー構造での整理

ByteRoverはランダムなチャンクをストレージに投げ込むのではなく、知識をツリー構造にキュレーションします。プロジェクトエリア、機能、アーキテクチャの決定、ワークフロー、そしてそれらの関係性ごとに整理されます。

  • プロジェクトエリア ごとに分類
  • 機能 単位で整理
  • アーキテクチャ決定 を記録
  • ワークフロー を体系化
  • 関係性を明確にリンク
テキストの山をモデルに解釈させるより、構造化された方が圧倒的に確実です

④検索の革新: 段階的検索パイプライン

ByteRoverは通常のベクトル検索を段階的な検索パイプラインに置き換えます。ファジーテキスト検索から始まり、必要に応じてより深いLLM駆動検索へと移行します。

  • 第1段階: ファジーテキスト検索で高速に候補を絞り込み
  • 第2段階: LLM駆動の深い検索で精度を向上
  • 単なるメモリの保存ではなく、正しいメモリの取得が重要

⑤Loco Memo ベンチマーク: 92.2%の精度

ByteRoverの公開ベンチマーク资料によると、Loco Memo メモリベンチマークで約92.19%（四捨五入で92.2%）の精度を達成しています。これはメモリを保存するだけでなく、エージェントが必要な時に正しいメモリを取得できることを証明しています。

92.2%の取得精度 — 単なる保存ではなく「取り出す力」が違います

  • メモリを自由に管理できる
  • バックアップも簡単

  • ラップトップ → VPS への移行もシームレス
  • チーム全体で同じメモリベースを共有
  • ゼロからやり直す必要がなくなる
ローカルで始めて、必要に応じてクラウドへ。柔軟な運用が可能です
