② ParseBenchとは何か

ParseBenchはAIエージェントのための初のドキュメントOCRベンチマークだ。約2,000ページの人間が検証したエンタープライズ文書と167,000以上のテストルールで構成されている。テーブル、チャート、コンテンツ忠実性、セマンティック書式、ビジュアルグラウンディングの5つの能力軸で評価を行う。

  • 約2,000ページの人間検証済みデータ
  • 167,000以上のテストルール
  • 5つの能力軸で総合評価
LlamaIndex オープンソース

③ コンテンツ忠実性が最も重要な理由

ParseBenchの5つの評価軸の中で、コンテンツ忠実性（Content Faithfulness）は最も基本的かつ重要な次元だ。パーサーがテキストを正しく、正しい順序で、捏造せずにキャプチャできているかを検証する。エージェントのコンテキストが不完全または偽造されていれば、下流のすべての判断が危険に晒される。

コンテキストが間違っていれば、
エージェントは「誤った現実」で判断することに！
Content Faithfulness 最も重要な次元

④ 欠落（Omissions）の3つのレベル

パーサーがテキストを取りこぼす「欠落」には3つのレベルがある。単語レベル、文レベル、数字レベルでそれぞれテキストが欠落する。契約書で金額の桁が一つ抜けるだけで、意味が全く変わってしまう。数字の欠落は特にビジネス上の重大なリスクを伴う。

  • 単語レベル: キーワードが抜け落ちる
  • 文レベル: 文全体が消える
  • 数字レベル: 金額や数値が欠落
Omissions 単語・文・数字

⑤ ハルシネーション（幻覚）の危険性

OCRパーサーが原文書に存在しないコンテンツを勝手にでっち上げる現象を「ハルシネーション」と呼ぶ。LLMと同様に、パーサーも幻覚を起こすことがある。存在しない条項や金額を追加してしまえば、エージェントは事実に基づかない判断を下すことになる。

パーサーが勝手に文をでっち上げたら...
エージェントは「偽の現実」で動くことに！
Hallucinations 捏造コンテンツ

⑥ 読み順序違反（Reading Order Violations）

多段組レイアウトの文書では、パーサーが正しくない順序でテキストを線形化してしまうことがある。脚注が本文より前に来たり、段をまたぐ順序が崩れたりする。これにより意味が全く変わってしまい、エージェントは誤った文脈で判断を下す。

  • 多段組レイアウトで発生しやすい
  • 脚注が本文より先に出力される
  • 段をまたぐ順序が崩壊する
Reading Order 多段組レイアウト

⑦ 従来のメトリクスの限界

BLEUやROUGEなどの従来のテキスト類似性メトリクスは、タイポや小さな差異を検出するのは得意だが、エージェントにとって実際に重要な欠陥を見逃してしまう。文全体の欠落、存在しないコンテンツの幻覚、読み順序の崩れなどは、類似性スコアでは「良い」評価が出てしまう。

テキスト類似性が95%でも、
重要な文が欠けているかも！
BLEU ROUGE テキスト類似性の罠

⑧ ルールベース評価の仕組み

コンテンツ忠実性スコア（CFS）は、147,000以上の個別テストルールによるルールベースの評価システムを採用している。あいまいなテキスト類似性ではなく、人間が検証したMarkdownトランスクリプションから導き出された緻密なルールで厳密に評価する。エージェントのワークフローに適した堅牢なテスト基盤だ。

  • 147,000以上の個別テストルール
  • 人間が検証したMarkdownベース
  • エージェントワークフロー向けに設計
CFS ルールベース評価

ParseBenchの5つの評価軸

⑨ テーブル（Tables）の評価

エンタープライズ文書にはテーブルが頻繁に登場するが、実際のテーブルは単純なグリッドではなく、結合セルや階層ヘッダー、複数ページにまたがる構造を持つ。TableRecordMatchという新しいメトリクスで、列の並べ替えなどの無害な違いにはペナルティを与えず、ヘッダーの転置や列名の欠落などの重大なエラーには厳しく評価する。

TableRecordMatch 結合セル 階層ヘッダー

⑩ チャート（Charts）の評価

多くのパーサーはチャートをスキップするか、生のOCRテキストとして出力する。チャート評価では1チャートあたり最大10個のスポットチェックデータポイントを注釈し、数値・ラベル・許容誤差を検証する。明示的な値ラベルのあるチャートは完全一致を求め、軸から読み取る場合は1%の許容誤差を適用する。

チャートで50%を超えるのは
たった4社のみという厳しい結果！
スポットチェック 1%許容誤差

⑪ セマンティック書式（Semantic Formatting）

ほとんどのパーサーは書式を装飾として扱い、削除してしまう。しかし書式には意味がある。取り消し線の価格は現行価格ではなく、上付き文字の「1」は脚注参照であって数字の1ではない。エージェントが「$49.99 $39.99」を区別できなければ、旧価格を提示してしまうリスクがある。

  • 取り消し線 = 無効な価格
  • 上付き文字 = 脚注参照
  • 太字 = 重要な集計値
Semantic Formatting 書式の意味

⑫ ビジュアルグラウンディング（Visual Grounding）

抽出した全要素を元のページ上の位置に追溯到できるかを評価する。規制業界での監査可能性に不可欠だ。保険フォームから補償限度額を抽出した場合、その数値がページのどの位置から来たのかを正確に示す必要がある。局所化・分類・属性付けの3つを統合して評価する。

この数字はどこから来たの？
という問いに答えるのがグラウンディング！
Visual Grounding 監査可能性

ベンチマーク結果と分析

⑬ LlamaParse Agenticが84.9%で総合首位

14の手法を3つのカテゴリ（汎用VLM、専門パーサー、LlamaParse）で評価した結果、LlamaParse Agenticが84.9%で唯一5つの全次元で競争力のあるスコアを記録した。GPT-5 Mini、Haiku 4.5、Gemini 3 Flash、Qwen 3 VL、Textract、Azure Document AIなど強豪が揃う中での成果だ。

14手法 84.9% LlamaParse Agentic

⑭ コンテンツ忠実性はほぼ解決済み、しかし...

最良の手法は約90%の忠実性スコアに到達している。素晴らしいように聞こえるが、これは10ページ中1ページで意味のある欠落やハルシネーションが発生することを意味する。リスクの高いワークフローでは許容できない水準だ。ParseBenchはどの種類の文書に注意が必要かを理解するのに役立つ。

90%でも10ページに1回は
エラーが出るということ！
90%の壁 10ページに1回のリスク

⑮ チャートと書式で明暗が分かれる結果

チャートは大きな差別化要因だ。50%を超えるのはわずか4社のみで、ほとんどの専門パーサーは6%以下。書式についても、ほとんどのパーサーが取り消し線や上付き文字を装飾として扱い削除する。Doclingは1.0%、LlamaParse Agenticは85.2%と大きな開きがある。

  • チャート: 50%超えはわずか4社
  • 書式: Docling 1.0% vs LlamaParse 85.2%
  • グラウンディング: GPT-5 Mini & Haiku は8%未満
チャートの差別化 書式の軽視

⑯ 品質とコストのトレードオフ

VLMにより多くの計算リソースを投入しても、利益は逓減する。Geminiは最小思考から高思考に4倍のコストをかけても約5ポイントの向上に留まる。一方、LlamaParse Agenticは約1.2¢/ページで全プロバイダーを凌駕し、Cost Effectiveモードは0.4¢/ページ未満で競争力を維持している。

⑰ データセットの構築方法

約2,000ページはすべて実際の公開エンタープライズ文書（保険のSERFF申請、財務報告、政府文書など）から収集。簡単なものから意図的に難しいものまで幅広くサンプリングした。グランドトゥルースは2段階のパイプラインで生成: VLMによる自動ラベリング → 人間による全ページ検証と修正。

  • 保険・金融・政府の実際の文書
  • VLMによる自動ラベリング
  • 人間による全ページ検証・修正
2段階パイプライン 人間検証

⑱ オープンソースで自分でも試せる

ParseBenchは完全にオープンソースで、誰でもベンチマークを実行できる。90以上の事前設定パイプラインと全評価コードが含まれている。データセットはHuggingFace、コードはGitHub、論文はarXivで公開中。独自のOCRモデルの追加も簡単にできる。

git clone https://github.com/run-llama/ParseBench
cd ParseBench
uv sync --extra runners
uv run parse-bench run <pipeline_name>
uv run parse-bench serve <pipeline_name>
