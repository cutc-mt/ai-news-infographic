Claude Codeの便利な機能7選

知っておくと便利な機能や普段使っているものを徹底解説

CLAUDE.md、プランモード、エフォートレベル、フルスクリーン、コンパクト、ステータスライン、スキル

2026年4月18日

① Claude Codeの始め方4種類

  • Web版: GitHubと連携してクラウド上で使用
  • デスクトップアプリ: CoworkとCodeの2モード。Codeがおすすめ
  • VS Code拡張: エディター内でファイル確認がしやすい
  • CLI: 最新機能はほぼ全てここで使える
デスクトップアプリのCodeタブにはスケジューリング機能があって超便利！

② CoworkとCodeの違い

  • Cowork: 非エンジニア向け。機能が制限されている
  • Code: 全機能が使える。スラッシュコマンドも豊富
  • Coworkではスキルの使い勝手が悪い（サイドバー経由でないと使えない）
初心者はCoworkから始めて、慣れたらCodeに移るのがおすすめ！

③ CLAUDE.mdで常に読ませる指示設計

  • CLAUDE.mdに書いた内容は常にClaude Codeに渡される
  • プロジェクト単位とユーザー単位の2種類が存在
  • オートメモリー機能で会話から自動的に記憶を保存
  • AIにCLAUDE.mdを書かせるより自分で書いた方が品質が高いという研究結果
# 確認方法
/memory  → 現在読み込んでいるCLAUDE.mdを確認
/init    → 自動生成（品質注意）

④ CLAUDE.mdの階層構造を活用

  • サブフォルダにCLAUDE.mdを置くと親フォルダのCLAUDE.mdも同時に読み込む
  • 別々のルールで実験したい時にフォルダごとに切り替えが可能
  • 挙動がおかしい時はユーザー単位のCLAUDE.mdやオートメモリーを確認
データ分析などで複数の実験を別々に行いたい時に便利な設計！

⑤ プランモードで精度を上げる

  • Shift+Tabでプランモードに切り替え可能
  • Ask User QuestionツールでAIから質問が来て要件を自動整理
  • Codexでレビュー → Claude Codeで修正の使い分けが強力
  • ウルトラプランモードでクラウド上で計画を洗練も可能
「精度が欲しい時はプランモード」がにゃんたさんの鉄則！

⑥ エフォートレベル: Opus 4.7でXHigh登場

  • Low / Medium / High / XHigh(新) / Max の5段階
  • XHighが効率と精度のベストバランス（公式グラフで確認）
  • 3月下旬の「性能低下」はデフォルトがMediumだったのが原因かも
  • 環境変数で常時XHighに設定可能
# 環境変数で常時XHighに設定
export CLAUDE_CODE_EFFORT_LEVEL=xhigh

⑦ フルスクリーンレンダリング & フォーカスモード

  • /tui fullscreenでマウス操作が可能に
  • 出力をクリックで選択して簡単にコピー＆ペースト
  • /focusで差分表示を非表示にして作業に集中
  • Jump to Bottomボタンですぐに一番下に戻れる
ターミナルでClaude Codeを使っている人には必須の設定！

⑧ /compactでコンテキストを圧縮

  • 会話が長くなるとコンテキストロットで精度が低下
  • /compactで会話履歴を圧縮して精度を維持
  • 環境変数で自動圧縮を設定可能（例: 15万トークン以上で自動実行）
# 自動コンパクトの設定
export CLAUDE_CODE_AUTO_COMPACT=150000

⑨ ステータスラインで上限と負荷を可視化

  • 入力欄の下にモデル名、コンテキスト量、利用制限を表示
  • 間違ってSonnetを使っていた等の事故を防止
  • 色付きで見やすく設定可能。/statuslineコマンドでClaude Codeに設定してもらえる
5時間制限や7日間制限を監視しながら使えるから安心！

⑩ スキルでワークフローを自動化

  • AIエージェントにワークフローを設計できる仕組み
  • プログラムだけで判定が難しい処理もエージェントが自律的に判断
  • 動画編集での無音検知や言い直し検知を実装済み
  • 作った後に何度も改善できるのがローカル環境の強み

⑪ Obsidianスキルでナレッジ構築

  • ObsidianのCEOが開発した公式スキルが存在
  • コールアウト機能で読みやすい形式に自動整形
  • ハイライトや構造化も自動で設定してくれる
  • プラグインからインストールしてすぐに利用可能
ナレッジ構築 → Obsidianスキルで変換のワークフローが最強！

⑫ スケジューリング機能の活用

  • デスクトップアプリのCodeタブで定期実行タスクを登録
  • メールチェッカーを1時間ごとに自動実行する運用例
  • スタートアップ時実行に設定すればログイン時に自動起動
  • CLIのループ機能は長時間でトークン切れの問題あり

⑬ CodexとClaude Codeの使い分け

  • Claude Code: 日本語が分かりやすい。計画とやり取りに強い
  • Codex: レビュー能力と実装の正確性が高い
  • Claude Codeで計画 → Codexでレビュー → Claude Codeで修正の往復が最強
2つのツールを組み合わせると一発で高品質なアプリが作れる！

⑭ /initでCLAUDE.mdを自動生成する際の注意

  • /initコマンドでCLAUDE.mdを自動生成可能
  • しかし研究によるとAIに書かせたCLAUDE.mdは品質が低いとの結果
  • 人が書いたCLAUDE.mdの方が明らかにパフォーマンスが良い
楽をしてAIに書かせるより、自分でしっかり書くのが大事！

⑮ エフォートレベルの正しい選び方

  • XHigh: Opus 4.7のみ。効率と精度のバランスが最適
  • Max: Opus 4.7、4.6、Sonnet 4.6で利用可能。最高精度
  • XHigh → Maxの傾きが緩やか。XHighがコスパ最強
性能が下がったと感じたら、エフォートレベルをチェック！

⑯ スキルの真価: エージェントの自律性

  • 「言い直し」の検知は意味的理解が必要でプログラムだけでは困難
  • 並列で無音検知と言い直し検知を実行し、結果を自動で検証
  • 今までAPI従量課金でやっていたことがサブスク内で完結

⑰ 定期実行のベストプラクティス

  • デスクトップアプリのスケジューリング機能が最も安定
  • CLIのcronやループは長時間で認証トークンが切れる問題あり
  • スタートアップ時実行に設定して、ログイン時に自動開始
定期実行をさせたいならデスクトップアプリが一番安心！

⑱ まとめ: Claude Codeを使いこなす7つの機能

  • CLAUDE.md: 常に読ませる指示を設計（自分で書く！）
  • プランモード: Ask User Questionで要件整理
  • エフォートXHigh: 精度とコストのベストバランス
  • フルスクリーン: マウス操作で使いやすく
  • /compact: 長時間作業での精度維持
  • ステータスライン: 利用状況を可視化
  • スキル: ワークフローの自動化
