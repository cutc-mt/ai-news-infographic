Claude Designが登場！

会話をしながらデザイン世界観を作り込める新しいプロダクトを解説

デザインシステム → ワイヤーフレーム → リッチ化 → Claude Codeへの引き継ぎ

2026年4月18日

① Claude Designとは: 一言でまとめ

  • Claudeを使ってUI、スライド、プロトタイプ、LPなどを会話ベースで作成
  • 単なる画像生成ではなく、制作ワークフロー全体をカバーするプロダクト
  • ブランドやデザインシステムを適用しながら進められる
  • チームで共同編集や共有も可能
作って終わりではなく、対話でどんどん深掘りできるのが最大の価値！

② 単なる生成AIではなく「制作ワークフロー化」

  • 企画: ユーザーがClaude Designに意見を伝える
  • 骨組み: ワイヤーフレームで編集しやすい形で進める
  • 詳細化: どんどん細かく直してリッチに仕上げる
  • 共有: チームに共有してPDFやHTMLに出力
  • 実装: Claude Codeに渡してプログラムを書く
企画から視覚化・共有・実装まで繋がっているのが最大のポイント！

③ 非デザイナーにとって強力なパーツ

  • 頭の中の案を形として整えて作成できる
  • 対話しながらUIの方向を詰めていける
  • デザインの知識がなくても、質問に答えるだけでデザインが完成
デザインを普段やらない人ほど、このツールの恩恵が大きい！

④ デザイナーにとっても探索を加速するツール

  • 大量の探索パターンを高速に出力可能
  • モックを触れるプロトタイプに変換できる
  • 叩き台の作成を高速化して、本質的なデザイン作業に集中できる
  • ただしエディター機能がやや物足りず、実務への本格移行にはまだ課題あり
デザイナーの知人に聞くと「本格移行にはまだ早い」とのこと。でも将来性は抜群！

⑤ デザインシステムで統一感を担保

  • カラー、タイポグラフィー、コンポーネント、デザインルールを定義
  • デザインシステムを適用すれば、毎回バラバラの見た目にならない
  • CLAUDE.mdのような形でスキル定義が自動生成される
  • Organization Settingsの中でデザインシステムを管理
AIでUIを作ると見た目がバラバラになりがち。デザインシステムで統一感を出せるのが秀逸！

⑥ 実演: サウナタイマーのApple Watchアプリ

  • 森林テーマのデザインシステムを作成
  • その世界観でApple Watchのサウナタイマーアプリを設計
  • ワイヤーフレームからスタートしてプロトタイプを生成
  • 完成後、Claude Codeに渡して実装する流れがスムーズ
世界観を統一してからアプリを作ると、ブランド感が一気に伝わる！

⑦ スライドモードとプロトタイプモードの使い分け

  • スライドモード: プレゼン資料の完成図を作る場合に使用
  • ワイヤーフレームモード: 画面の骨組み・画面遷移を設計する場合に使用
  • 頭の中がざっくりしている場合はワイヤーフレームから始めるのがおすすめ
画面の骨組みをやるか、完成予想図を作るかでモードを使い分けよう！

⑧ 森林浴サイトのLPを対話で作成する実演

  • 「森林浴できるスペースのレンタルLPを作成して」と指示
  • Claudeが自動でTodoリストを作成して進行
  • グラデーションの方向やレイアウトを質問されて回答するだけで完成
  • 複数のパターンを出してくれて、その中から好きなものを選択可能
森林浴LPの出来栄えが「レベル高い」。質問に答えるだけでここまで作れる！

⑨ Tweak/Editモードで細部を調整

  • Tweak機能で「ここを変えて欲しい」と部分的な修正を依頼
  • Editモードで直接要素を編集可能
  • 「この部分だけフォントを変えて」といった細かな調整も対応
  • お絵かき機能で「ここをこうして欲しい」と図示して指示することも可能
レイアウトが崩れても対話で修正できるのが安心！

⑩ 共有・コメント・Send to Claude

  • Share機能でチームメイトと編集リンクを共有
  • コメント機能でフィードバックを残せる
  • Send to Claudeで対話を継続してさらにブラッシュアップ
  • プロジェクトのコピー、テンプレートとしてのコピーも可能

⑪ エクスポート: Canva、PDF、PPTX、ZIP

  • Canva連携でシームレスにエクスポート。Xでも話題に！
  • PDF出力で資料としてそのまま活用可能
  • PPTX出力でPowerPointスライドに変換
  • ZIP形式での一括ダウンロードにも対応
  • HTMLとしての書き出しも可能
Claudeのサブスクリプション内で全て完結。別のSaaSはもういらないかも！

⑫ Claude Codeへの引き継ぎで実装へ

  • デザイン画面からClaude Codeに直接引き継げる
  • コピーしたコマンドを実行するだけでデザインファイルを読み込み
  • デザイン → 実装のシームレスな流れが実現
# Claude Design → Claude Code の流れ
# 1. デザイン画面で「Send to Claude Code」
# 2. コマンドをコピーして実行
# 3. Claude Codeがデザインを読み込んで実装開始

⑬ 動的に生成される質問表が面白い

  • デザイン作成時に質問表が自動で動的に生成される
  • プロジェクトに応じて質問内容が変わる仕組み
  • グラデーションの方向やレイアウトの好みを質問してくれる
  • 様々なUIコンポーネントの選択肢が提示される
質問に答えるだけでデザインが出来上がる。非デザイナーにとって救世主！

⑭ 既存資料を新しい世界観で再現できる

  • 以前作成したOpusの資料を新しいデザインシステムで再現
  • 「このスタイルに変えて」と指示するだけで世界観ごとリッチに仕上げ
  • スライドのクオリティが驚くほど高い
既存の資料を別のデザインシステムで再現すると、見た目が劇的に変わる！

⑮ 「AIくさいUI」への一つの解決策

  • AIが生成したUIには独特の「AI臭さ」があるという問題
  • デザインシステムを適用することで、ブランド感のある独自のUIに
  • 自分のプロダクトの世界観に合わせたデザインが可能
  • お絵かき機能で「ここをこうして」と細かく指示できる
「AI臭いUIを見るのが辛い」という流れに対する一つの回答になりそう！

⑯ 現時点の限界とこれからの展望

  • エディター機能がやや物足りず、デザイナーの実務にはまだ早い
  • まだ「Claude感」が出るデザインになりがち
  • Webベースなのでスマホ・PCどちらでもアクセス可能
  • UIデザインが完結する時代が来たら嬉しいという期待
今後の進化に期待。Claude DesignでUIデザインが完結する日が来るかも！

⑰ まさおの開発フローへの組み込み方

  • まずデザインシステムを作成する
  • 「この世界観で行こう」という形を決める
  • 脳内を整理するように対話でデザインを作っていく
  • 完成したらClaude Codeに引き継いでプログラムを書く
個人開発で気軽に使える。デザイン→実装の流れが一気にスムーズに！

⑱ Claude Designの差別化ポイント

  • Claudeから気軽に始められるデザインツール
  • Claude Codeに引き継げて実装までシームレス
  • デザインシステムで統一感のある出力が可能
  • Canva / PDF / PPTX / ZIP / HTML の多彩なエクスポート
  • 対話で深掘りできるから作って終わりではない
別のSaaSはもういらないかも。Claude Designで十分ってなる日が来る！
