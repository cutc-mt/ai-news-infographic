Claude Designの使い方

Anthropic公式デザイナーが教える7つのTipsで爆速プロトタイプ
2026年4月18日 | さとるチャンネル@毎日AIニュース

①Claude Designとは

  • 対話型でプロトタイプやスライド、ワイヤーフレームを作成できるツール
  • DMやマーケター、エンジニア、個人開発者まで非デザイナー向けのツール
  • Figmaのようにマウスで操作するのではなく、会話しながらデザインが形になる
対話でUIが形になるのがClaude Designの最大の強み！キャンバスツールとは全然違う存在なんです。

②Ryan Mather - 1人で7つのプロダクト

  • Anthropicのデザインチームに所属
  • 1人で7つのプロダクトを担当している超人
  • Claudeをフル活用することで物理的に無理な量を実現
  • X（旧Twitter）で7つのTipsを公開
普通に考えたら無理だけど、Claudeをフル活用すれば1人で7つを回せる！

③7つのTipsは4カテゴリーに分類

  • セットアップ編（Tip 1）- デザインシステムの構築
  • コラボレーション編（Tip 2-4）- リアルタイム協業と連携
  • 活用編（Tip 5-6）- 本質の理解と実験
  • マインドセット編（Tip 7）- 速さと丁寧さの使い分け
この4つの柱でClaude Designを使いこなす秘訣を解説します！
セットアップ編

④Tip 1: 最初の1時間が全てを決める

  • 使い始める前に最初の1時間をデザインシステムの構築に投資
  • 色・タイポグラフィー・ボタンやカードなどのコンポーネントを定義
  • ここをサボると後で地獄を見る - 毎回「このボタンは角丸で色はこれで」と指示し直すことに
  • 最初に教えておけば以降は自動で適用される
最初の1時間の投資がその後の数十時間を節約する！ここが一番重要な初期投資です。

⑤コアスクリーンも一緒に作っておく

  • ログイン画面やダッシュボードなど、よく使うレイアウトの基本形を先に用意
  • テンプレートのようなものを事前に準備するイメージ
  • この初期投資があるから新しい画面を爆速で生成できる
  • Ryanさんが7つのプロダクトを1人で回せている理由はここにある
ベースが揃ってるから、新しい画面を作る時も爆速で生成できる！
コラボレーション編

⑥Tip 2: エンジニアとのリアルタイム協業

  • ミーティング中に新機能の話をしながらその場でデザインを作っていく
  • エンジニアの制約を聞いて即座にClaude Designに反映
  • 従来：ヒアリング→Figma作成→レビュー→修正の往復サイクル
  • Claude Design：1回のミーティングで完了
ミーティングが終わる頃にはもうデザインができてる！往復のラリーが1回で終わるのは革命的。

⑦Tip 3: ポイント&クリット機能

  • 画面上の特定の場所を指し示してフィードバックを書き込める
  • Figmaのコメント機能に似ているが、決定的な違いがある
  • Claudeがコメントを理解して自動で修正してくれる
  • 「このボタンもう少し右に」「余白を詰めて」と書くとその通りに修正
言葉で説明しにくい微調整も、指差すだけでできるから超便利！

⑧Tip 4: コネクター機能で自動化

  • SlackやGoogle Docsと連携が可能
  • Slackのミーティングノートを読み込ませてデザインソリューションを自動生成
  • 議事録からデザインが自動で生まれるという飛躍
  • ミーティング内容は「課題」と「解決したいこと」の構造なので要件として活用できる
プロンプトを撫でて散歩に行こう。戻ってきたら新鮮な目でデザインを確認できる！

⑨プロンプトを撫でて散歩に行く

  • プロンプトを入力して待ち時間を有効活用する手法
  • 戻ってきたら新鮮な目で生成結果を確認できる
  • 同期で張り付く必要がない非同期の進め方
  • マルチプロダクトを担当する人にとって必須のスキル
散歩して戻ってきたらデザインができてるって、なんか未来ですね。もうその未来は来てるんですよ！

⑩充実のエクスポート機能

  • PowerPoint - プレゼンで使いやすい
  • Figma - デザインツールへの移行も可能
  • HTML - Web用にそのまま活用
  • 編集権限の付与も可能で、チームへの共有とブラッシュアップも簡単
PPTで出せるのはプレゼンで使いやすい！チームに共有してブラッシュアップも簡単です。
活用編

⑪Tip 5: キャンバスツールではない

  • Claude DesignはFigmaのようなキャンバスツールとは根本的に違う
  • Ryanさんの表現：Claude Codeに近い存在
  • HTMLアーティファクトを生成でき、APIにもアクセス可能
  • プロトタイプの中でコンテンツ生成までやらせることも可能
プロトタイプの中でAIが動く！ビデオすら作れる。もう単なるモックアップじゃない！

⑫プロトタイプの中でAIが動く

  • プロトタイプ内でコンテンツ生成まで実行可能
  • ビデオすら作成できる
  • APIアクセスも可能なので外部サービスとの連携も
  • 従来のデザインツールの発想を捨てないと真価を発揮できない
発想の転換が必要！従来のデザインツールの常識を捨てることで真価を発揮する。

⑬Tip 6: 即興カスタムツールの実験

  • Claude Designで即興のカスタムツールを作らせる
  • Ryanさんの例：花と杯子を組み合わせたプロトタイプを実験
  • 仕事に直結しない実験から予想外のクリエイティブが生まれる
  • 従来のツールでは絶対に作れなかったものが作れるかもしれない
遊びから予想外のクリエイティブが生まれる！可能性を探る姿勢が大事なんです。

⑭限界を探る実験が鍵

  • Claude Designで何ができるか限界を探る姿勢が重要
  • 実験から予想外の発見やアイデアが生まれる
  • 従来のツールでは不可能だった領域にチャレンジ
  • 仕事に直結しなくても、可能性の幅を広げることが目的
仕事で使わないような実験も大事。そこから新しい発見があるから！

⑮Tip 7: 速さと丁寧さの使い分け

  • エージェント的デザインの超高速モードに吸い込まれがち
  • でも実は手作業で丁寧にやるべき領域がある
  • 手作業推奨①：アイコン - プロダクトの印象を左右する
  • 手作業推奨②：スポットイラスト - ディテールがインパクトを生む
  • 手作業推奨③：ネーミング - 言葉のニュアンスが大事
スローダウンするかを決めること自体がアートフォーム。全部AIに任せればいいわけじゃない！
マインドセットと使い分け

⑯正直なデメリットと注意点

  • 細かなレイアウト調整はFigmaに劣る
  • グリッドの微調整や複雑なインタラクション設計はまだ対応していない
  • 研究プレビュー段階なので不安定な面も
  • コメントが消えたり、たまにエラーが出たりすることあり
  • 本格的な実務移行はもう少し待った方がいいかも
万能じゃない！プロトタイプの初期段階で使うのがベストです。

⑰Claude CodeとClaude Designの使い分け

Claude Code

本番に近い感覚で使用

開発者ツールにアクセス可能

実装レベルの精度が必要な時に

Claude Design

高速なアイデア出しと探索に

プロトタイプ段階で方向性を確認

素早く見たい時に使う

探索はClaude Design、実装はClaude Code。この使い分けがポイント！

  • コネクターで自動ワークフローを試す。SlackやDocsと繋いでプロンプトを撫でて散歩に行く
