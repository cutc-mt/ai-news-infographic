Claude Design検証 — プロトタイプからエクスポート、コード品質まで

AnthropicのAIデザインツールを実戦投入して分かった真実

📅 2026年4月20日 | 🎬 Coding Crash Courses

①Claude Designとは

  • Anthropicが公開したAIデザインツール。デザインの知識がなくても会話するだけで本格的なデザインが作れる
  • アクセスURLは claude.ai/design
  • プロトタイプ、スライドデッキ、デザインシステムなど多彩な形式に対応
  • プロンプトと対話を通じて、アイデアを視覚的なデザインに変換する仕組み
デザイン経験ゼロでも、会話だけでプロ品質のレイアウトが生成される！

②プロジェクト作成の流れ

  • 左サイドバーで作成するものを選択する
  • プロトタイプ（ワイヤーフレーム or 高精細モックアップ）が選べる
  • スライドデッキやフォルダーからも作成可能
  • 白紙テンプレートから自由にスタートすることもできる
  • プロジェクト名を入力して「Create」をクリックするだけ
プロトタイプの精密度をワイヤーフレームと高精細から選べるのが便利！

③デザインシステム機能

  • 独自のデザインシステムを作成してブランド統一が可能
  • GitHubリポジトリを連携して、CSSファイルやアイコンを再利用できる
  • Figmaファイル、フォント、ロゴなどのアセットも登録できる
  • 今回はデザインシステムなしで実践検証を行った
GitHubやFigmaと連携すれば、チームのデザイン資産をそのまま活用できる！

④プロンプト入力

  • スクリーンショットをアップロードして参考画像として使用可能
  • 「modern, more hacky design」と指示してハッカー風のテーマを指定
  • Claudeがプロンプトに基づいて質問を返してくる（例：「ハッカー風のフレーバーは？」）
  • 全ての質問を「Decide for me」で自動判定することも可能
参考画像を添付すれば、好みの方向性がもっと伝わりやすくなるよ！

⑤Claudeからの質問

  • 「ターミナル風のモノスペース、黒背景に緑文字」など具体的な方向性を質問される
  • 全ての質問に丁寧に答えるか、「Decide for me」でClaudeに任せることも可能
  • 自分の好みを反映したい場合は、質問に丁寧に答えるのがコツ
  • 質問の数が多いほど、よりカスタマイズされた結果になる
全部自動に任せるのもありだけど、こだわりたい部分はちゃんと答えよう！

⑥生成結果の確認

  • 約10分でデザインが完成する
  • ホームページ、コースカタログ、Tweaksパネルが含まれる
  • 結果は「really good」と高評価。クオリティが期待以上
  • 密なレイアウトと余白のあるレイアウトを切り替えて比較可能
10分待つだけで、想像以上に完成度の高いデザインが出来上がる！

⑦Tweaksパネルで調整

  • 密度スライダーでdense（密）とairy（ゆったり）を調整可能
  • 背景画像の変更がスライダー1つで完了する
  • フォントの変更にも対応している
  • コンポーネントごとに色や高さのCSS編集もできる
  • デザインの微調整が直感的に行える
CSSを直接いじらなくてもスライダーでサクサク調整できるのが嬉しい！

⑧コメント機能で修正

  • デザイン上の要素にピン留めコメントを追加できる
  • 「この色を青に」「幅を50%に」など複数のコメントをまとめて送信
  • Claudeが一括で修正を適用してくれる
  • ただし必ずしも意図通りに動くとは限らない点に注意
まとめ指示は便利だけど、全部が思い通りになるわけではないので要注意！

⑨エクスポート機能

  • ZIP形式でまとめてダウンロード可能
  • PDFやPowerPoint形式での出力にも対応
  • Canvaへの直接エクスポートも可能。YouTubeサムネイルなどに活用できる
  • スタンドアロンHTMLやClaude Codeへのハンドオフも選択可能
Canvaに直接送れるのは超便利！サムネイル作成も一瞬で終わる。

⑩Claude Codeへの連携

  • 「Handoff to Claude Code」を選択するとAPI URL付きのプロンプトが生成される
  • VSCodeのClaude Codeに貼り付けるだけでローカル環境にデザインが展開される
  • Bypass permissionsで自動実行も可能に
  • デザインから実装へのシームレスな移行を実現する機能
デザインをそのままコードに落とせるのは、プロトタイプ→実装の流れがスムーズ！

⑪ローカル展開の結果

  • Claude Codeで実行すると約10分で完了する
  • index.html、style.css、2つのJSXファイルが生成される
  • しかしコンソールにエラーが表示される
  • そのままでは動かない場合があるのが現実
ファイルは生成されるけど、エラーが出てそのままでは動かない…ここが課題。

⑫エラー修正

  • Claude Codeに修正を指示すると、コードの再構成が始まる
  • 最終的に単一のHTMLファイルとCSSファイルに統合される
  • 動作確認はクリアして、デザインが正しく表示される
  • ただしコードの品質は別問題として残る
動くようにはなるけど、中身のコードはちょっと…というのが本音。

⑬コード品質の問題

  • HTMLファイルは700行以上の単一ファイルになってしまう
  • HTMLとCSSが混在していて、インラインスタイルが多用されている
  • 定数は使っているものの、プロダクション品質ではない
  • ReactJS等でリライトしないと実運用には耐えない
700行の1ファイル…これをメンテナンスするのは人間にもAIにも辛い。

⑭スパゲッティコードの実態

  • バイブコーディング（vibe coding）で生成されたコードは動くが構造が悪い
  • インラインに全て詰め込まれていて、可読性が非常に低い
  • コンポーネント分割やモジュール化がされていない
  • プロダクション環境では危険な状態。バグの温床になる
「動けばOK」の精神で作られたコードは、後で必ず技術的負債になる。

⑮Reactリライトの必要性

  • Claude CodeにReactJSでのリライトを指示することが可能
  • コンポーネント化してインライン記述を解消する
  • プロダクション品質のコードに昇華できる可能性がある
  • ただし追加の作業と時間が必要になる
プロトタイプをReactにリライトすれば、実務レベルに持っていける！

⑯プロトタイピングとしては優秀

  • アイデアを素早く形にする用途としては非常に便利
  • デザインの方向性を確認したり、ステークホルダーに見せたりする用途に最適
  • あくまで「プロトタイプ」ツールとして使うのが正解
  • 早期のフィードバック収集やコンセプト確認には高い効果を発揮する
プロトタイプとしては最高！でもそれ以上を期待するのは禁物。

⑰ハプ vs 現実

  • SNSでは「デザイナーの仕事が奪われる」「フロントエンドエンジニアは終わり」と騒がれている
  • 実際にコードを見ると、プロダクション品質ではないことが分かる
  • エンジニアの仕事はなくならない。むしろコード品質の担保がより重要に
  • 過度な期待は禁物。冷静に評価することが大切
SNSのバズを信じると痛い目を見る。実際のコードを確認するのが大事！

⑱TL;DR — 結論

  • Claude Designはプロトタイピングには便利なツール
  • ハプが主張するほど画期的ではない
  • フロントエンドエンジニアが仕事を失うことはない
  • デザインから実装への橋渡しとしては有用だが、過信は禁物
  • 適切な用途で使えば、開発ワークフローを加速できる優秀なツール
プロトタイプ作成には最高だけど、過度な期待はNG。正しい使い方が大事！
