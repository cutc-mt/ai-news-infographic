Hermes Agentとは

① サーバー常駐型のパーソナルAI

  • Hermes AgentはNous Researchが開発したオープンソースのAIエージェントフレームワーク
  • IDEやターミナルではなく、サーバー上に常駐して動き続けるのが最大の特徴
  • VPS、Docker、サーバーレス環境で駐在させ、メッセージアプリから指示を送る仕組み
  • PCがスリープしても関係なく、サーバー側でずっと処理を継続できる
LMシリーズのHermesとは別物！
エージェントを構築するための
フレームワークなんだよ

② PCを閉じても動き続ける

  • ローカルのエージェントはPCのスリープで一緒に止まってしまう宿命がある
  • Hermes Agentはサーバー側に駐在するため、PCの状態に依存しない
  • 長いタスクを任せて休憩しても、戻った時には処理が完了している
  • 寝ている間にPRのドラフトを作成しておくことも可能
PCを開きっぱなしにする
ストレスから解放されるのです！

③ MITライセンスで完全オープンソース

  • MITライセンスでフル公開されているため、商用利用も個人利用も自由
  • オープンソースなので中身も見えて、安心して使える
  • GitHubで公開されており、コミュニティによる貢献も可能
  • フォークして自分流にカスタマイズすることもできる
OSSなら透明性もあって
セキュリティ面でも安心なのです

④ LLMを自由に差し替え可能

  • 使うLLMを自由に選択できるのが大きな特徴
  • OpenAI、Anthropicなど主要プロバイダーに対応
  • ローカルモデルも使えるので、完全にプライベートな運用も可能
  • 特定のモデルにロックインされないから、コスト調整もしやすい
用途に合わせてモデルを
使い分けられるのは嬉しい！
多数のメッセージングプラットフォーム対応

⑤ 幅広いプラットフォーム対応

  • Telegram — セットアップが簡単で定番の選択肢
  • Discord — チーム開発やコミュニティでの利用に最適
  • Slack — ビジネス用途でそのまま活用できる
  • WhatsApp — 個人メッセンジャーから直接指示可能
  • Signal — セキュリティ重視のユーザーにも対応
  • iMessage — BlueBubblesブリッジ経由でiPhoneからもOK
  • WeChat — 中国圏のユーザーにも使いやすい

⑥ 単一ゲートウェイで一元管理

  • すべてのメッセージングプラットフォームを1つのゲートウェイプロセスで管理
  • プラットフォームごとに別々のサーバーを立てる必要がない
  • 運用がシンプルで、設定や管理の手間を大幅に削減
  • ステータス確認もダッシュボードから一括で可能
1箇所で全部管理できるなら
運用も楽チンなのです！

⑦ 端末を跨いで会話が継続

  • PCでSlackから話しかけて、外出先ではTelegramで続きを確認
  • 場所やアプリを選ばず、同じAIとの会話が継続できる
  • メモリ機能により、どのデバイスからでも文脈を引き継げる
  • 自宅でも外出先でも、いつでも同じアシスタントにアクセス
場所を選ばず同じAIと
話せるのは嬉しいのです！
経験から学ぶスキル自動生成とメモリ

⑧ 経験からスキルを自動生成

  • 複雑なタスクをこなした後に、その手順をスキルとして自動保存
  • 次に似たタスクが来たら、保存したスキルを使って効率よく処理
  • 毎回ゼロから考え直す必要がなく、人間の経験学習と同じ仕組み
  • 使えば使うほど賢くなるのが最大の魅力
使うほど賢くなるから
長期運用に向いているのです！

⑨ 閉じた学習ループで自己改善

  • 実行結果をフィードバックにして、より良い手順に書き換えていく仕組み
  • 公式ではこれを「閉じた学習ループ」と呼んでいる
  • 自分で作ったスキルを自分で育てていく成長型のアプローチ
  • 継続して使うほど処理の品質が向上していく
スキルが自己改善されていく
仕組みが面白い！

⑩ agentskills.io標準に準拠

  • スキルの形式はagentskills.ioというオープン標準に準拠
  • 他の人が作ったスキルを取り込んだり、自分のスキルを共有したり可能
  • コミュニティの資産を活用でき、車輪の再発明を防げる
  • 標準化されているため、エコシステムの成長が期待できる
スキルの共有で
みんなで賢くなれるのです！

⑪ 3層メモリで過去の会話を記憶

  • 短期メモリ: 今の会話の文脈を保持
  • 長期メモリ: 重要な情報を永続的に保存
  • エピソードメモリ: 過去の体験を記録
  • エージェント自身が「これは覚えておくべき」と判断して自動保存
  • 手動で整理しなくても、自律的にメモリを管理してくれる
人間の記憶みたいな
3層構造なのです！
検索・スケジュール・実行環境

⑫ 全文検索で過去の会話を横断検索

  • SQLiteのFTS5検索エンジンを使ってセッションを横断検索
  • 半年前に話した内容も、必要な時に思い出せる
  • LLMが関連する過去の文脈を要約して今の会話に活かしてくれる
  • Mem0（ユーザーモデリング）と連携して好みや文脈を深く理解
昔話したことを忘れないから
使い続けるほど便利になる！

⑬ Lunatic Cronで自然言語スケジュール

  • Lunatic Cronスケジューラで定期タスクを自動実行
  • 自然言語で設定できるのが特徴（cron式を覚えなくてOK）
  • 「毎朝9時にGitHubのニュースをまとめて報告」のような指示が可能
  • 結果を好きなプラットフォームに配信できる（Telegram/Discord等）
  • Claude Codeのルーティンズが発表される前から実装済み
寝てる間にPRが準備されてる
のは嬉しいかも！

⑭ 6種類のターミナルバックエンド対応

  • ローカル — 手元の環境でそのまま実行
  • Docker — コンテナ環境で安全に分離して実行
  • SSH — リモートサーバーに接続して実行
  • Daytona — 開発環境プラットフォームで実行
  • Singularity — HPC環境向けコンテナ
  • Modal — サーバーレス実行環境

⑮ サーバーレス永続化でコスト最適化

  • リクエストがない間はハイバネート状態になり、リソースを消費しない
  • TelegramやDiscordから指示が来たら自動で起き上がり処理を開始
  • アイドル時のランニングコストを大幅に抑制（ほぼ$0）
  • 同じコードで小さなVPSからGPUクラスタまでスケール可能
  • 環境を変えてもコードを書き直す必要がない
待機中は眠ってて呼ばれたら起きる
電気代の節約にもなるのです！

⑰ Claude Code・Cursorとの違い

  • Claude Code / Cursor: IDEやターミナルが起点。開発作業中のアシスタント
  • Hermes Agent: メッセージングアプリが起点。長時間常駐のパーソナルAI
  • 競合ではなく補完関係にあり、両方の併用がおすすめ
  • 昼はCursorでコーディング、夜はHermesに定期タスクを任せる役割分担が可能
どちらかを選ぶんじゃなくて
用途に合わせて使い分けるのがおすすめ！

⑱ 導入方法とまとめ

  • curlのワンライナーでインストール完了（Linux/macOS/WSL2対応）
  • hermes modelコマンドでLLMプロバイダーを対話形式で選択
  • hermes gatewayでTelegramやDiscordのトークンを設定
  • OpenCodeユーザーはhermes crow migrateで設定やスキルを引き継げる
$ curl -fsSL https://hermes-agent.nousresearch.com/install | bash
$ hermes model
$ hermes gateway
