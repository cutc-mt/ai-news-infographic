Agent Skills by Addy Osmani

22スキルでAIコーディングエージェントのパフォーマンスを10倍に引き上げる

2026年4月21日 | AICodeKing

① Agent Skillsとは: 単なるプロンプト集ではない

  • Addy Osmani（Google Chromeチームのエンジニア）が設計した構造化エンジニアリングワークフローです。
  • 単なるプロンプトの詰め合わせではなく、定義・計画・実装・検証・レビュー・簡略化・リリースという完全なライフサイクルを提供します。
  • コードレビュー、テストエンジニア、セキュリティ監査などの専門ペルソナも内蔵しています。
ベンチマークの数字ではなく、実際のプロセス改善こそが本当の価値です！

② 7つのコアコマンド: ライフサイクル全体をカバー

  • /spec - 要件を定義して仕様書を作成します
  • /plan - タスクを分解して実装計画を立てます
  • /build - インクリメンタルに実装を進めます
  • /test - TDD（テスト駆動開発）で品質を保証します
  • /review - コードレビューを実施します
  • /code-simplify - コードをシンプルに整理します
  • /ship - リリース準備を整えてデプロイします
spec → plan → build → test → review → simplify → ship の7ステップが基本フローです

③ /spec コマンド: アイデアを明確な仕様に変換

  • アイデアの段階から仕様駆動開発（Spec-Driven Development）を開始できます。
  • 機能要件・非機能要件・受け入れ基準を明確に定義します。
  • ユーザーストーリーやエッジケースも自動的に抽出してくれます。
  • 作成された仕様は/planコマンドにそのまま渡せます。
曖昧なアイデアを実装可能な仕様に変えるのが最初のステップです

④ /plan コマンド: タスクを細分化して優先順位付け

  • 仕様書をもとに実装タスクを段階的に分解します。
  • 依存関係を分析して、実装の優先順位を自動で決定します。
  • 各タスクには完了条件とマイルストーンが設定されます。
  • 並列実行可能なタスクと直列すべきタスクを判別します。
計画フェーズをしっかりやると、後の実装がスムーズになります！

⑤ /build コマンド: インクリメンタル実装の手法

  • インクリメンタル（漸進的）な実装を基本アプローチとして採用しています。
  • 一度に全てを実装するのではなく、小さな単位で積み上げていきます。
  • 各ステップで動作確認しながら進めるため、バグの早期発見が可能です。
  • 既存機能を壊さないように、常に安全な状態を保ちます。
一気に書くより少しずつ確実に進める方が結局早いです

⑥ /test コマンド: テスト駆動開発（TDD）をAIで実践

  • テスト駆動開発（TDD）のワークフローをAIエージェントに組み込んでいます。
  • Red（失敗するテストを書く）→ Green（最小限の実装）→ Refactor（リファクタリング）のサイクルです。
  • テストエンジニアの専門ペルソナがエッジケースも網羅します。
  • ユニットテスト・統合テスト・E2Eテストを自動生成します。
テストを先に書くことで設計の質も向上します

⑦ /review コマンド: AIコードレビューのベストプラクティス

  • コードレビューアーの専門ペルソナが客観的にコードを評価します。
  • バグの可能性、パフォーマンス問題、セキュリティリスクを指摘します。
  • コーディング規約への準拠やベストプラクティスの適用を確認します。
  • 改善提案を具体的に提示し、修正の方向性を示します。
人間のレビューとAIレビューを組み合わせるのが最強です

⑧ /code-simplify コマンド: コードをシンプルに保つ

  • 不要な複雑さを排除してコードの可読性を高めます。
  • 重複コードの統合、過剰な抽象化の削減を実施します。
  • 関数の長さや責任の分割を適切に整理します。
  • パフォーマンスの最適化も同時に考慮します。
シンプルなコードはバグも少なく保守しやすいです

⑨ /ship コマンド: 安全なリリースとデプロイ

  • リリース前の最終チェックリストを自動で実行します。
  • チェンジログの生成とバージョニングを管理します。
  • CI/CDパイプラインとの連携でデプロイを自動化します。
  • ロールバック戦略も含めて安全なリリースを設計します。
リリースは焦らず確実に。安全策こそが本番です

⑩ 3つの専門ペルソナ: レビュー・テスト・セキュリティ

  • コードレビューアー - コードの品質、可読性、保守性を評価する専門家です。
  • テストエンジニア - テストカバレッジを最大化し、エッジケースを発見します。
  • セキュリティ監査 - 脆弱性や攻撃ベクターを特定して修正を提案します。
3人の専門家がチームに参加するような感覚です！

⑪ コンテキストエンジニアリング: AIに適切な情報を与える

  • コンテキストエンジニアリングは、AIに必要な情報を的確に提供する技術です。
  • 不要な情報を省き、関連する情報だけを絞り込んで渡します。
  • 過去の決定事項やプロジェクトの歴史も含めて引き継ぎます。
  • 精度の高い出力を得るための最も重要なスキルの一つです。
プロンプトの書き方より何を渡すかが大事です

⑫ API設計スキル: インターフェースを適切に設計

  • RESTful APIの設計からGraphQLまで幅広くカバーしています。
  • エンドポイントの命名規則やHTTPメソッドの適切な使い分けを指導します。
  • リクエスト・レスポンスのスキーマ定義も自動生成します。
  • エラーハンドリングとバージョニング戦略も含まれています。
API設計が良ければフロントエンドも効率的に開発できます

⑬ フロントエンドスキル: UIコンポーネント設計

  • コンポーネント駆動開発で再利用性の高いUIを構築します。
  • レスポンシブデザインとアクセシビリティに配慮します。
  • 状態管理やパフォーマンス最適化のベストプラクティスを適用します。
  • CSSの設計手法（BEM、CSS Modules、Tailwindなど）にも対応します。
ユーザー体験を最優先にした設計ができます

⑭ デバッグスキル: 問題を体系的に特定して解決

  • 体系的なデバッグアプローチで問題の根本原因を特定します。
  • バイナリサーチ方式で問題箇所を絞り込んでいきます。
  • ログ分析やスタックトレースの読み解きも自動で行います。
  • 修正案を複数提示し、影響範囲も評価してくれます。
推測ではなく証拠に基づいたデバッグができます

⑮ セキュリティスキル: 脆弱性をスキャンして防护

  • OWASP Top 10に基づいたセキュリティ監査を実施します。
  • SQLインジェクション、XSS、CSRFなどの一般的な脆弱性を検出します。
  • 認証・認可の実装ミスも見つけ出します。
  • 依存パッケージの既知の脆弱性もチェックします。
セキュリティは後付けではなく最初から組み込みましょう

⑯ パフォーマンス最適化: ボトルネックを特定して改善

  • パフォーマンスボトルネックを自動的に特定します。
  • メモリリークの検出と不要な再レンダリングの最適化を行います。
  • Core Web Vitalsの改善提案を具体的に提示します。
  • レイジーロード、キャッシュ戦略、バンドル最適化もカバーします。
速いサイトはユーザー離脱も減らせるので一石二鳥です

⑰ ドキュメンテーション & CI/CD: 知識を共有して自動化する

  • APIドキュメントの自動生成とREADMEの整備を行います。
  • コミットメッセージの規約とチェンジログの自動生成を管理します。
  • GitHub ActionsをはじめとするCI/CDパイプラインの設定を支援します。
  • テスト・リント・ビルドの自動化ワークフローを構築します。
ドキュメントもコードの一部として扱いましょう

⑱ プレーンMarkdownでポータブル: どのツールでも動く

  • スキルはすべてプレーンMarkdown形式で記述されています。
  • 対応ツール: Claude Code、Cursor、Gemini CLI、Windsurf、OpenCode、Copilot
  • 特定のツールに依存しないため、いつでも別のツールに乗り換えられます。
  • オープンソースで無料。自由にカスタマイズして使えます。
ツールに縛られないフリーな設計が長期的な強みです

Verdent連携: verdent.mdでユニバーサルな習慣を設定

  • verdent.md - どのプロジェクトでも使う普遍的な習慣を定義します。
  • agents.md - プロジェクト固有のルールと制約を設定します。
  • Plan Rules - 仕様ファーストの計画手法を強制します。
  • カスタムサブエージェントでレビュー・テスト・セキュリティを担当させます。
  • 並列ワークスペースでライフサイクル全体をオーケストレーションします。
VerdentとAgent Skillsを組み合わせると相乗効果が生まれます！
