Claude Code Subconscious - 潜在意識が記憶する

セッションを閉じても忘れない。4つのフックと8つのメモリブロックでClaude Codeに記憶を与える
2026年4月20日 | さとるチャンネル@毎日AIニュース
Subconsciousの概要と解決する問題

①Claude Codeの「物忘れ」問題

  • Claude Codeのセッションを閉じると、すべての会話内容が消えてしまう
  • 次回起動時にまた同じ説明を繰り返す必要がある
  • ユーザーの好み、プロジェクトの文脈、過去のやり取りがすべてリセットされる
  • 「前回こうしました」というコンテキストの引き継ぎが一切できない
セッション閉じたら全部忘れちゃうの、毎回同じこと説明するの本当にめんどくさい...

②Subconsciousとは

  • Claude Codeに「記憶」を与えるプラグイン
  • Letta（旧MemGPT）のSleeptime Agent概念をClaude Codeに実装
  • バックグラウンドでメモリを管理して、セッション間で記憶を引き継ぐ
  • 人間の「潜在意識」のように、意識しなくても記憶が働く仕組み
人間が無意識に記憶を呼び出すのと同じように、Claude Codeも自動で記憶を活用できるようになる！

③Sleeptime Agentの概念

  • Letta（旧MemGPT）が提唱した概念
  • 人間が寝ている間に記憶が整理されるように、エージェントのアイドル時間に記憶を整理する
  • セッションが終わるたびに会話内容を分析してメモリを更新する
  • 次回のセッション開始時に必要な記憶を自動的に注入する仕組み
人間が寝てる間に記憶が整理されるのと同じ！起きたら前の日のことがちゃんと覚えてる感じ。

④導入は環境変数1つだけ

  • APIキーを環境変数に設定するだけで完了
  • 複雑な設定ファイルやデータベースの構築は一切不要
  • たった1つの環境変数ですぐに記憶機能が有効になる
  • 導入ハードルが極めて低く、既存のワークフローを変更する必要がない
環境変数1つ設定するだけ！複雑な設定なしですぐに使い始められるのが嬉しいね。
4つのフック：記憶のライフサイクル

⑤フック1: SessionStart

  • Claude Codeの起動直後に実行される最初のフック
  • エージェントの取得を行い、メモリ管理の準備をする
  • 新しい会話IDを作成してセッションを識別
  • CLAUDE.mdのクリーンアップを実行して、前回の残存データを整理
起動した瞬間に「どのエージェントか」「新しい会話か」を判定して、準備を整えるのが最初の役割！

⑥フック2: UserPromptSubmit（心臓部）

  • ユーザーがプロンプトを送信した時に実行される
  • メモリブロックから保存されている記憶を取得する
  • 差分を計算して、変更があった部分だけを注入する
  • Subconsciousの心臓部とも言える最重要フック
ユーザーが何か入力するたびに記憶を呼び出して、必要な情報を差し込む。まさに心臓の働き！

⑦フック3: PreToolUse

  • ツールが実行される直前に呼び出されるフック
  • ツールの実行コンテキストに追加のメモリ情報を注入する
  • 変更がなければサイレントに終了し、余計な処理を省く
  • パフォーマンスへの影響を最小限に抑えるスマートな設計
変更がなければ何もしない（サイレント終了）。無駄な処理を省いてパフォーマンスを守る賢い設計！

⑧フック4: Stop（終了時の非同期処理）

  • セッションの終了時に実行される最後のフック
  • セッションのトランスクリプト（会話記録）を送信
  • spawnSilentWorkerで親プロセスをブロックしない
  • 非同期で実行されるため、ユーザーの待ち時間が発生しない
spawnSilentWorkerで裏側にまわすから、ユーザーは待たされない。終了時にサクッと記憶を保存して終わる！
8つのメモリブロック：記憶の保管庫

⑨core_directivesとguidance

  • core_directivesはエージェントの役割定義を保存するブロック
  • 「あなたは〇〇の専門家です」のような基本的な役割と方針を記録
  • guidanceは具体的な指示を保存するブロック
  • 「関数にはJSDocを必ず書く」などのプロジェクト固有のルールを管理
core_directivesが「誰であるか」、guidanceが「どう振る舞うか」。人間でいう性格と行動指針みたいなもの！

⑩project_contextとuser_preferences

  • project_contextは技術スタックやプロジェクト情報を保存（最大3000文字）
  • 使用言語、フレームワーク、アーキテクチャなどのプロジェクト構成を記憶
  • user_preferencesはユーザーの好みを保存（最大3000文字）
  • 「変数名はキャメルケースで」「コメントは日本語で」などの個人の好みを記憶
「この人キャメルケース派だったな」とか「React派だったな」を覚えてくれるから、毎回指示しなくて済む！

⑪pending_itemsとself_improvement

  • pending_itemsは未完了のタスクを記録するブロック
  • 「あとでリファクタリングする」「バグ修正が残ってる」などを次回に引き継ぐ
  • self_improvementはエージェントの自己改善を記録するブロック
  • 過去のミスや改善点を蓄積して、徐々に賢くなっていく仕組み
自己改善ブロックが面白い！前回やったミスを覚えてて、次からは同じミスをしないようになる。

⑫recent_interactionsとclaude_md

  • recent_interactionsは最近のやり取りを要約して保存
  • 直近の会話の文脈を保持して、継続的な作業をスムーズに
  • claude_mdはCLAUDE.mdの内容をミラーするブロック
  • CLAUDE.mdの内容と同期して、プロジェクトの指示を常に最新に保つ
recent_interactionsで「前回どこまでやったっけ？」を解決。claude_mdでプロジェクト指示との整合性も保てる！
差分更新アルゴリズムと動作モード

⑬差分更新アルゴリズム

  • 毎回全ブロックを注入するのではなく、変更があった部分だけを更新
  • トークン消費を大幅に節約できるのが最大のメリット
  • 保存済みのメモリと新しい内容を比較して、差分のみを抽出して注入
  • APIコストを抑えつつ、必要な情報だけを効率的に配信
毎回全部注入してたらトークンが無駄に消費される。差分だけ注入すればAPIコストも抑えられて一石二鳥！

⑭whisperモード（デフォルト）

  • whisperは差分のみを注入する動作モード
  • 変更があったメモリブロックだけを追加して、最小限のトークンで済ませる
  • デフォルトの動作モードで、普段使いに最適
  • ユーザーにとってほぼ透過的に動作し、意識せずに記憶が活用される
ささやくように必要な情報だけを注入するから「whisper」。余計なノイズが入らないのが良い！

⑮fullモードとoffモード

  • fullモードはすべてのメモリブロックを毎回注入する
  • 初回起動時や、完全な文脈が必要な場合にフルコンテキストを提供
  • offモードはメモリ機能を完全に無効化する
  • 状況に応じて3つのモードを切り替えて使える柔軟な設計
普段はwhisper、最初はfull、一時的に無効にしたいときはoff。状況に合わせて選べるのが便利！

⑯SDK連携とread-onlyモード

  • SDK連携が可能で、外部からメモリブロックにアクセスできる
  • read-onlyモードが推奨されており、安全にメモリを読み取れる
  • 書き込みを制限することで、意図しないメモリの変更を防止
  • 他のツールや自動化パイプラインから記憶を参照する用途に最適
read-onlyで安全にアクセスできるから、CI/CDパイプラインから記憶を参照するような使い方も可能！
まとめ：Claude Codeが記憶を持つ未来

⑰非同期パターンの利点

  • spawnSilentWorkerで親プロセスをブロックしない設計
  • 記憶の保存処理がユーザー体験を邪魔しない
  • Claude Code本体のパフォーマンスにほぼ影響を与えない
  • バックグラウンドで記憶を処理するというアーキテクチャのベストプラクティス
バックグラウンドでこっそり記憶を整理してくれるから、ユーザーは待たされることなく快適に使える！

⑱まとめ

  • SubconsciousはClaude Codeに「記憶」を与えるプラグイン
  • 4つのフックでセッションの全ライフサイクルを管理
  • 8つのメモリブロックで役割、好み、文脈、タスクを記憶
  • 差分更新でトークン消費を節約する賢い設計
  • 環境変数1つですぐに導入可能
  • Claude Codeが自分の「潜在意識」を持つ未来がここから始まる
セッションを閉じても覚えてくれるClaude Code。毎回同じ説明を繰り返す日々はもう終わり！
