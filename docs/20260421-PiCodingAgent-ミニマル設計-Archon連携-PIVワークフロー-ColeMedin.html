Pi Coding Agent + Archon — ミニマルなコーディングエージェントで究極のワークフロー構築

BloatedなClaude Codeに代わる、最小限のコア + 拡張性の哲学。Archonと組み合わせたPIVワークフローを徹底解説

📅 2026年4月21日 | 🎬 Cole Medin

①Piとは: ミニマリズムのコーディングエージェント

  • PiはMario Zechnerが開発したオープンソースのコーディングエージェント
  • 最小限のコア機能だけを持ち、他のすべては拡張機能として追加する設計思想
  • Claude Codeのように肥大化せず、シンプルで予測可能な動作を実現
  • 必要な機能だけを自分でビルドして追加できる柔軟性が最大の強み
「最小の基盤を作り、あとは拡張で足す」
これがPiの核心理念です！

②なぜ「Bloated（肥大化）」が問題なのか

  • Claude Codeは当初シンプルだったが、機能追加が止まらない状態に
  • リリースごとにバグが頻発、プラットフォーム全体の保守が困難に
  • システムプロンプトが常に変わり、結果に影響するのにユーザーには分からない
  • コンテキストを自分でコントロールすることが事実上不可能になっている
Anthropicが出荷を急ぐあまり、
初期のミッションから逸脱してしまった…

③Mario Zechnerの「Slopの世界でPiを築く」

  • AI Engineer Conferenceでの講演 "Building Pi in a World of Slop"
  • AMP、Codex、Factory、Open Codeなど既存ツールを徹底的に検討
  • Open Codeはオープンソースだが、コードベースが大きく拡張が困難と判断
  • 「一切を削ぎ落とし、最小で拡張可能なコアを構築する」と決意してPiを開発
「シンプルで拡張しやすい」
これこそがエージェントの理想形！

④Pi最大の特徴: エージェントが自分自身を拡張する

  • Piに「この機能を追加して」と指示するだけで自分自身を拡張
  • 例: Claude Codeの新機能を見たらPiに「これをビルドして」と頼むだけ
  • コアがシンプルなので、コンテキストエンジニアリングの負担も最小
  • Claude Codeの機能を失わずに、ブロートなしで利用できる
「Hey Pi、MCPサポートを
追加して」→ それだけで完成！

⑤サードパーティ拡張マーケットプレイス

  • 拡張機能はnpm上に公開されており、誰でも検索・インストール可能
  • スキル、プロンプトテンプレート、テーマ、拡張など多様なカテゴリ
  • ワンコマンドでインストール: Piに「この拡張を検索して入れて」と頼んでもOK
  • サードパーティを信用しない場合は、自前でビルドするのも簡単
npmで「pi-package」を検索すれば
拡張がずらりと見つかります！

⑥インストールはワンコマンドで完了

  • READMEに記載のnpmコマンド1つで即座にインストール可能
  • インストール後に/loginでサブスクリプションを設定
  • Anthropic、GitHub Copilot、Antigravity、Codexなどに対応
  • /modelで利用可能なモデルから自由に選択できる
npm install -g @anthropic-ai/pi-coding-agent
# 起動
pi
# ログイン
/login
# モデル選択
/model
Anthropicのサブスク利用は
利用規約違反の可能性があるので注意！

⑦どんなLLMでも使える自由度

  • Mistral、Grok、Kimi K2、MiniMax M2.7など多数のモデルに対応
  • APIキーを使えば任意のプロバイダーのモデルを利用可能
  • Codexサブスクリプションを使えばGPT-5.3 Sparkなどのモデルも
  • Geminiモデルも利用可能で、プロバイダー間を自由に切り替えられる
モデルの選択肢が豊富なのが
Pi × Archonの強みです！

⑧Piに意図的に含まれていない機能

  • MCP、サブエージェント、プランモードなどは標準で搭載されていない
  • Mario自身が使わない機能はあえて除外している
  • 必要な人は自分でビルドするか、マーケットプレイスから取得可能
  • コアが小さいからこそビルドが簡単で、理解も早い
「削ぎ落とす勇気」こそが
Piの設計哲学の真髄です

⑨Archon: オープンソースのハーネスビルダー

  • ArchonはCole Medinが開発したAIコーディングのハーネスビルダー
  • エージェント工学のプロセス全体を再利用可能なワークフローにパッケージ化
  • 複数のコーディングエージェントセッションを並列実行してスケール
  • 当初はClaude CodeとCodexをサポート、Piが3番目の対応エージェントに
コンテキストエンジニアリングから
「ハーネスエンジニアリング」へ！
これがAIコーディングの進化です

⑩Pi × Archonの相性が最強な理由

  • ArchonでPiを使う場合、追加の認証設定が不要
  • Piで/login済みなら、Archonが同じ認証情報を自動利用
  • ワークフローのノードごとにプロバイダーとモデルを自由に切り替え可能
  • Piの拡張機能をArchonワークフローにそのまま統合できる
Piが急速に私のベストフレンドに！
あなたもそうなるかも？

⑪PIVワークフロー: Plan → Implement → Validate

  • Plan: 機能やバグ修正の計画を作成
  • Implement: コーディングエージェントで実装
  • Validate: PRレビューのような全体検証を実行
  • この3ステップのループを回すのがColeのAIコーディングの核心メソッド
Plan → Implement → Validate
このサイクルを回し続けることで
品質の高いコードが生まれます！

⑫Plannotator拡張: Web UIで計画をレビュー

  • 計画をWebサイトにレンダリングして視覚的に確認
  • インラインでコメントを追加してフィードバックを送信
  • フィードバックを送ると自動的に計画を修正して再表示
  • 承認するまで何度でも反復可能、Human-in-the-loopを実現
わざわざ行番号を指定して
修正指示する必要がなくなりました！

⑬ArchonワークフローをYAMLで定義

  • ワークフローはYAMLファイルで定義してGit管理
  • 明確化フェーズ → Pi計画 → Claude実装 → 検証のマルチノード構成
  • 各ノードでプロバイダーを個別にオーバーライド可能
  • ArchonのClaude Codeスキルが自動的にワークフローを構築をガイド
# ワークフロー構成例
nodes:
  - name: clarification
    provider: claude
  - name: planning
    provider: pi
    model: gpt-5.4-mini
  - name: implementation
    provider: claude
  - name: validation
