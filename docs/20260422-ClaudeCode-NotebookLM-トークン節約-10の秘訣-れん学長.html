1

Limited Reach問題: ある日突然止まる

  • Claude Codeを使っていると、突然「Limited Reach」というメッセージが出て作業が止まることがある。
  • 月ごとや週ごとに利用枠が決まっていて、使い切ると次のリセットまで数時間から数日待つことになる。
  • 思考のピーク時にこの制限に引っかかると、せっかくの勢いが消えてしまう。
Limited Reach 利用制限
「あー、せっかくうまくいってたのに…」って経験、結構ありますよね。
2

月12日問題: Proプランでも実質12日しか使えない

  • 年間200ドルのProプランでも、30日のうち実質12日しか使えないと海外では言われている。
  • 特定の人の問題ではなく、Claude Codeを本気で使っている人すべてが直面する構造的な問題。
  • 海外では「Day 12問題」と呼ばれて深刻な問題になっている。
Pro $200/年 30日中12日
PDFとか大量に読み込ませると、トークンがバケツみたいにガーっと減るんですよね。
3

解決策の本質: 重い仕事をNotebookLMに丸投げ

  • PDFの読み込み、重い分析、成果物の生成はNotebookLMに任せる。
  • Claude Codeは指示出しと最終編集に専念する。
  • この役割分担により、トークン消費を体感で約1/10に圧縮できる。

従来: Claude Codeに全部やらせる
トークン大量消費

新手法: 役割を分担
約1/10に圧縮
役割分担 1/10圧縮

NotebookLMの嬉しい特性: 出典付きで回答

  • NotebookLMは回答に出典を明示してくれる。どの資料のどの根拠を元に答えたかが一目瞭然。
  • 事実と異なる回答を出すハルシネーションのリスクを抑制してくれる。
  • トークン節約と回答の信頼性アップが同時に期待できる一石二鳥の効果。
出典明示 ハルシネーション対策
トークン節約だけでなく、回答の信頼性も上がるから一石二鳥なんです！

ユースケース1: 重い資料の理解と音声化

  • 業界レポートや専門書のPDFをAudio Overview機能で音声概要に変換。
  • スマホのNotebookLMアプリで、通学や通勤中に聞けるようになる。
  • 「アップロードしといて」とClaude Codeに頼めば、自動でファイルをアップしてくれる。
PDF読み込み 音声概要 スマホ学習
重い資料を理解したい時に、これがかなり使えるんですよ。

ユースケース2: ビジネス向けの議事録分析

  • 議事録をNotebookLMに入れて、「誰がいつ何をやるのか」を一覧で生成。
  • 担当、期限、未解決問題の推移がフラグ付きで出力される。
  • 前回から今回までの進捗が一発で分かるので、プロジェクト管理にも活用できる。
議事録 進捗管理 フラグ付き

ユースケース3: 競合リサーチとコンテンツ戦略

  • NotebookLMのWebリサーチ機能で競合記事を自動収集。ファストリサーチとディープリサーチの2種類がある。
  • ファストリサーチは広く浅く全体像を把握。ディープリサーチは時間をかけて特定テーマを深掘り。
  • 収集結果をそのままコンテンツ戦略の素材に活用できる。
Webリサーチ ファスト/ディープ コンテンツ戦略
Claude CodeのWeb検索でもできるけど、それだけでトークンめちゃくちゃ食うので注意。

ユースケース4: 過去の自分の資産を検索可能に

  • 過去に書いたノート記事を100本以上、NotebookLMに全部登録。
  • 新しい記事を書く際に、過去の自分の発言を自然な形で引用できるようになる。
  • 記事に厚みが出て、一貫性のある発信ができる。「この人、分かってるな」と思われるように。
過去記事検索 引用活用 一貫性
URLを100個ノートブックLMに登録して、って頼むだけで一発でできるんですよ。

ユースケース5: 古いノートブックの整理

  • NotebookLMの登録上限（約300）に達したら、古いノートを整理する必要がある。
  • Claude Codeから古いノートをリストアップして、一括削除やリネームができる。
  • 1個ずつポチポチ削除する手間がなくなり、一撃で指示するだけで完結する。

連携ツール: NLM MCP CLIでシームレス接続

  • NLM MCP CLIという非公式ツールで、Claude CodeとNotebookLMを連携。
  • インストールはターミナルでコマンドを打つだけ。Claude Codeに頼めば自動でやってくれる。 
# インストール例
uv tool install nlm-mcp-cli

# NLMコマンドで操作可能
nlm list
nlm create "ノート名"
NLM MCP CLI 非公式ツール シェル経由
クロードコードに頼めば、インストールから設定まで全部やってくれるので簡単ですよ。

実践ワークフロー: 競合リサーチを一発で

  • Claude Codeに「NotebookLMでファストリサーチして、結果をマークダウンでダウンロードして」と頼む。
  • 約30秒で完了し、結果がローカルにMDファイルとして保存される。
  • 保存されたリサーチ結果を、そのままClaude Code内で参照できる。
30秒完了 MD保存 ローカル参照
