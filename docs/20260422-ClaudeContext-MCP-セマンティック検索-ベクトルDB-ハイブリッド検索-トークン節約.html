Claude Context - セマンティックコード検索MCPプラグイン

Zilliz Techのオープンソースが、コードベース全体の意味検索をClaude Codeにもたらす

📅 2026年4月22日

①Claude Contextとは: セマンティックコード検索のMCPプラグイン

  • Zilliz Techが開発したオープンソースのMCPプラグイン
  • GitHubで6000+スコアを獲得するほどの人気プラグイン
  • コードベース全体をベクトルDBにインデックスして、質問に関連するコードだけを取得
  • つまり「RAG for your codebase」というコンセプトで動作する
コードを全部コンテキストに詰め込むのはもう不要！
必要な部分だけを自動で見つけてくれる！

②コアコンセプト: RAGをコードベースに適用

  • リポジトリ全体をベクトルデータベースにインデックスする
  • 質問を投げると、意味的に関連するコードだけを自動で取得
  • LLMのコンテキストウィンドウに、必要最小限のコードだけを渡す仕組み
  • 結果として約40%のトークン削減を達成できると主張
大規模リポジトリほど効果的！
全部を読み込む必要がなくなるのでコストも下がる

③ハイブリッド検索: 意味とキーワードの両方をカバー

  • 密なベクトル検索（Dense Vector）で意味的な理解を実現
  • BM25キーワード検索で正確な文字列マッチを確保
  • 2つの検索結果を統合（ハイブリッド）して高精度な結果を返す
  • 意味的理解と正確なマッチの両方に対応できる
ベクトルだけじゃなくて
キーワード検索も組み合わせるから精度が高い！

④ASTベースのチャンキング: コードを意味的に分割

  • AST（抽象構文木）を使ってコードを意味的な単位に分割
  • 関数やクラスなどの論理的なまとまりを保持したままチャンク化
  • 単純な文字数分割ではなく、コードの構造を理解して分割する
  • そのおかげで検索結果の精度と関連性が向上する
関数の途中でぶった切らないから、
検索結果がちゃんと意味をなす！

⑤対応言語: 12以上の主要言語をサポート

  • TypeScript / JavaScript
  • Python
  • Java / Kotlin
  • Go / Rust
  • C++ / C#
  • PHP / Ruby / Swift
主要な言語はほぼ網羅！
どんなプロジェクトでも使えるのが嬉しい

⑥Merkle tree差分インデックス: 変更分だけ再インデックス

  • Merkle tree（マークルツリー）でファイルの変更を追跡
  • コードが変更されたら、変更されたファイルだけを再インデックス
  • リポジトリ全体の再インデックスが不要になるので高速に更新
  • 継続的な開発でもパフォーマンスを維持できる
Gitみたいに差分だけ処理するから
大きなリポジトリでもサクサク！

⑦4つのMCPツール: シンプルなAPI構成

  • index_codebase - リポジトリをベクトルDBにインデックスする
  • search_code - セマンティック検索で関連コードを取得する
  • get_indexing_status - インデックスの進行状況を確認する
  • clear_index - 既存のインデックスをクリアする
たった4つのツールで完結！
シンプルなのが良いね

  • Claude Code - Anthropic公式のCLIツール
  • Codex CLI - OpenAIのコーディングCLI
  • Gemini CLI - GoogleのCLIツール
  • Cursor / Windsurf - AIコードエディタ
  • Cline など、MCPに対応するクライアントなら何でもOK
MCP対応なら何でも使える！
特定のツールに縛られないのが嬉しい

⑨セットアップ: Zilliz Cloud + OpenAI Embedding

  • Zilliz Cloud（マネージドMilvus）をベクトルDBとして使用
  • OpenAI Embeddingでテキストのベクトル化を実行
  • MCPクライアントの設定ファイルに数行追加するだけで完了 
// Claude Codeの設定例
{
  "mcpServers": {
    "claude-context": {
      "command": "npx",
      "args": ["-y", "@anthropic/claude-context"]
    }
  }
}
設定はシンプル！
数分で使い始められる

⑭インデックスの仕組み: 初回は全体、以降は差分のみ

  • 初回はリポジトリ全体をフルインデックスする
  • 2回目以降はMerkle treeで差分を検出して変更分だけ更新
  • ASTベースでコードをチャンクに分割してからベクトル化
  • ベクトル化されたチャンクをZilliz/Milvusに保存
初回だけちょっと時間がかかるけど
2回目からは爆速で更新される！

⑮検索の流れ: 質問からコード取得まで

  • ユーザーが質問やタスクを入力する
  • 質問をベクトル化してベクトルDBを検索
  • ハイブリッド検索（意味検索＋キーワード検索）を実行
  • 関連するコードチャンクだけをコンテキストに追加
  • LLMが精度の高い回答を生成する
たった5ステップで
ピンポイントなコードを見つけ出す！

⑯注意点: セッション横断の長期記憶ではない

  • Claude Contextはセッション横断の長期記憶を提供するものではない
  • あくまで現在のセッション内でコードを検索するためのツール
  • セッションをまたいだ会話の記憶が必要なら別のツールを検討
  • 目的は「コードの検索と取得」に特化している点を理解しておこう
記憶ツールじゃないよ！
検索ツールとして割り切って使おう
