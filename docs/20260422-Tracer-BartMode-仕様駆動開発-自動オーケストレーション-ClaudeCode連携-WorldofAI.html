Tracer Bart Mode - 仕様駆動開発を完全自動化

Spec ToolkitがVibe Codingを終わらせる。100倍効率的な開発フローを実現

2026年4月22日 | WorldofAI

① 従来のAIエージェント開発: 都度の監視と修正が必須

  • これまではAIエージェントを都度監視・チェック・修正する必要がありました。
  • エージェントがコードを生成するたびに、人間が確認して方向を修正する手間がかかります。
  • いわゆる部分的自動化の状態で、完全な自動化には程遠い状況でした。
  • Vibe Codingでは「AIに任せきり」になるか「人間が常に監視」のどちらかしかありませんでした。
AIを見張り続けるのは本質的ではない作業なんですよね...

② Bart Modeとは: スマートオーケストレーターの登場

  • Tracerの新機能Bart Modeは、エピック全体を自動実行するスマートオーケストレーターです。
  • エピックをタスクに分解して、並列バッチでAIエージェントが実行します。
  • 各バッチの実行後にレビューを行い、仕様との整合性を自動で確認します。
  • 必要に応じて計画を更新し、柔軟に適応しながら進めます。
人間が監視し続ける必要がなくなったのが最大の変化です！

③ Ralph loopではない: インテリジェントな適応型実行

  • Bart Modeは盲目的なリトライ（Ralph loop）ではありません。
  • 進捗を理解して、状況に応じて適応するインテリジェント層として機能します。
  • 失敗したら同じことを繰り返すのではなく、別のアプローチを試みます。
  • 人間の入力が必要な時だけエスカレーションして介入を求めます。
エージェントが学習しながら自律的に動くのがポイントです

④ Spec-Driven Development: 仕様がコードより先に来る

  • 仕様駆動開発では、コードを書く前に明確な仕様を定義します。
  • 何を作るか、どう動くか、どんな制約があるかを先に文書化します。
  • 仕様に基づいて自動的に検証できるため、品質が安定します。
  • Vibe Codingの「行き当たりばったり」とは対極のアプローチです。
仕様 → 計画 → 実装 → 検証の順序が鉄則です

⑤ Tracerの4つのモード: Epicで大規模プロジェクトを管理

  • Epicモード - 大規模プロジェクトを仕様やチケットに自動分解します。
  • Phasesモード - AIと対話しながら要件を明確化していきます。
  • Planモード - ファイル単位の詳細計画を立ててコーディングエージェントに渡します。
  • Reviewモード - デバッガーやレビューアーとして動作します。
プロジェクトの規模に合わせてモードを使い分けるのがコツです

⑥ Epicモード: 大規模プロジェクトを一気に自動化

  • エピック（大きな目標）を入力すると、自動的にタスクに分解します。
  • 各タスクはチケット化されて、優先順位と依存関係が付きます。
  • 複数のAIエージェントにタスクを割り当てて、並列実行を開始します。
  • 進捗はダッシュボードで一元管理でき、全体像を常に把握できます。
「このアプリを作って」と言うだけであとは全部お任せできます！

⑦ Phasesモード: AIと対話しながら要件を明確にする

  • AIとの対話を通じて、曖昧な要件を明確にしていきます。
  • AIが質問をしてくれるので、答えるだけで仕様が固まります。
  • 自分でも気づいていなかった要件の抜けをAIが見つけてくれます。
  • 明確になった要件は、そのままPlanモードに引き継げます。
AIと一緒に考えるから、仕様の質が段違いに上がります

⑧ Planモード: ファイル単位の詳細な実装計画

  • ファイル単位で実装計画を立てて、コーディングエージェントに渡します。
  • どのファイルから作るか、どんな順序で進めるかを自動で決定します。
  • ファイル間の依存関係を考慮して、最適な実装順序を導き出します。
  • 計画が完成したら、Claude Codeなどのコーディングエージェントに自動で指示を出します。
計画が正確なら実装も迷わず進むという当たり前の事実

⑨ Reviewモード: デバッガーとレビューアーを自動で実行

  • 自動デバッガーとしてバグを検出して修正案を提示します。
  • コードレビューアーとして品質とコーディング規約の準拠を確認します。
  • セキュリティの問題やパフォーマンスの懸念も自動で指摘します。
  • レビュー結果は次のバッチの計画にフィードバックされます。
人間のレビューとAIレビューを二重がけすると最強です

⑩ モデル選択: 用途に応じて最適なAIを使い分ける

  • Opus 4.7 - 最も高性能なモデルで複雑なタスクに対応します。
  • Balance - 速度とコストのバランスを取った中位モデルです。
  • Frontier - 最高品質の出力を優先する最上位設定です。
  • タスクの重要度に応じてモデルを適宜切り替えることでコストを最適化します。
全てを最高性能にするより用途別に使い分ける方が賢いです

⑪ エスカレーション: 必要な時だけ人間に通知

  • Bart Modeは人間の入力が必要な時だけエスカレーションを行います。
  • 仕様の曖昧さや判断に迷う箇所があると、人間に質問を投げかけます。
  • 不要な通知は一切なく、本当に重要な場面だけで介入を求めます。
  • 人間が寝ている間もエージェントは自律的に作業を継続できます。
朝起きたらプロジェクトが進んでいるという体験ができます

⑫ チームコラボレーション: リアルタイム共同作業が可能

  • リアルタイムでチームメンバーと共同作業ができます。
  • 複数人が同時に異なるタスクに取り組むことが可能です。
  • 進捗の共有やタスクの割り当てもダッシュボード上で完結します。
  • コメント機能でやり取りしながら、仕様の調整もスムーズに行えます。
一人で全部やるよりチームで分担する方が圧倒的に早いです

⑬ デモ: エージェント管理ダッシュボードを完全自動構築

  • 動画のデモではエージェント管理ダッシュボードをBart Modeで構築しています。
  • 認証機能を備えたログイン・ログアウトの仕組みを自動生成します。
  • API統合で外部サービスとの連携もシームレスに実装します。
  • デプロイ機能まで含めて、本番運用に必要な要素をすべて自動で作り上げます。
仕様を書くだけでフルスタックアプリが完成するデモは必見です！

⑭ エージェントCRUD: 作成・読取・更新・削除を自動実装

  • Create - 新しいエージェントを作成するフォームとAPIを自動生成します。
  • Read - エージェントの一覧表示と詳細ページを自動で作ります。
  • Update - エージェントの設定変更を安全に実行できる画面を作ります。
  • Delete - 削除時の確認ダイアログや安全な削除処理も含まれます。
CRUDはWebアプリの基本ですが、Bart Modeが全部やってくれます

⑮ Claude Code連携: Anthropicの強力なコーディングエージェントと統合

  • TracerはClaude Codeとシームレスに連携して動作します。
  • Claude Codeがターミナル上でコーディングを担当し、Tracerが全体をオーケストレーションします。
  • Planモードで作成した計画をそのままClaude Codeに渡して実装を開始できます。
  • Reviewモードの結果もClaude Codeにフィードバックされて、修正サイクルが回ります。
Tracerが司令塔でClaude Codeが実行部隊という役割分担です

⑯ 無料で始められる: Gemini CLIなども利用可能

  • Tracerは無料で使い始めることができます。
  • Gemini CLIなどの無料ツールと組み合わせれば、コストゼロで開発が可能です。
  • Cursor、VS Code、Windsurfなどの拡張機能も用意されています。
  • 自分の使い慣れたエディタでそのままTracerの機能を活用できます。
お金をかけずに始められるのが一番の魅力です

⑰ Vibe Coding vs Bart Mode: 決定的な違い

  • Vibe Coding - 行き当たりばったりで、AIに任せきり。品質の保証がありません。
  • Bart Mode - 仕様に基づいて計画→実行→検証のサイクルを確立します。
  • 100倍効率的と謳われるのは、監視の手間と手戻りのコストが劇的に減るからです。
  • 仕様との整合性を自動で確認するため、品質がブレません。
「雰囲気でコーディング」から仕様でコーディングへのパラダイムシフトです

⑱ まとめ: Bart Modeが開発の未来を変える

  • Tracer Bart Modeは仕様駆動開発を完全に自動オーケストレーション化します。
  • インテリジェントな適応型実行により、人間の監視負荷を大幅に軽減します。
  • Claude CodeやGemini CLIなど複数のAIツールと連携可能です。
  • 無料で始められて、CursorやVS Codeの拡張機能も充実しています。
  • Vibe Codingの時代は終わり、Spec Toolkitの時代が始まっています。
これからの開発は仕様を書く力こそがエンジニアの武器になります！
