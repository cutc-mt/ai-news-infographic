Claude Code「Proプランから削除」騒動の全真相

2026年4月23日

A/Bテストだったのか、それとも本気の市場調査なのか

騒動の発端

① スクショ1枚で炎上スタート

  • 2026年4月21日夜9:40、1枚のスクショがXに投稿された
  • 画像に映っていたのはClaudeの料金ページで、ProプランのClaude Codeがバツ印になっていた
  • 投稿は瞬く間に拡散し、150万ビューを超える大騒ぎに発展
たった1枚のスクショが、コミュニティ全体を揺るがすことになったんだよね。

② 料金ページに表示された変更内容

  • スクショにはProプランの列からClaude Codeが消え、バツ印が表示されていた
  • ProユーザーはMaxプラン（月100ドル以上）にアップグレードしないと使えなくなるという内容
  • 告知や公式発表が一切なかったことが最大の問題視された
「告知なしでこっそり書き換えた」という表現が、怒りにさらに油を注いだんだよね。

③ Xでの反応が爆発的に拡散

  • 「また値上げか」「せめて事前に告知してほしい」という不満が続々と溢れた
  • 「Anthropicは信用できない」という厳しい意見も多数
  • OpenCodeやCursorへの乗り換えを推す投稿も同時に急増
  • コミュニティ全体がかなりざわついた雰囲気に

④ 投稿者本人の立ち位置

  • 投稿者はMax月200ドルユーザーで、自分自身には影響がないと書いていた
  • 影響を受けていない立場からの告発という構図が、さらに話題を呼んだ
  • スクショ自体は本物だったが、全体像を捉えきれていなかったことが後に判明
自分は影響ないよって書いてるのが、なんかいい味を出してたんだよね（笑）
真相：実はA/Bテストだった

⑤ Anthropicが公式に発表

  • 投稿から約1時間後、Anthropicの内部の人間がXで公式のコメントを出した
  • 「新規Proサインアップのうち約2%に対して小規模なテストを実施していた」と回答
  • 「既存のProユーザーとMaxユーザーへの影響は一切ない」と明言
拡散されたスクショは、A/Bテストの画面だったというのが真相なんだよね。

⑥ A/Bテストとは何か

  • 一部のユーザーだけに違うページを見せて、どちらの反応がいいか測る実験手法
  • 今回は新規契約者のたった2%だけが、あのバツ印の画面を見ていた
  • 既存ユーザーには全く表示されておらず、機能への影響もゼロだった
Web開発ではよく使われる手法だけど、料金ページでやると勘違いを招くよね…。

⑦ 翌日に料金ページが復旧

  • 4月22日には料金ページが元に戻り、Proプラン欄にClaude Codeが含まれる表示が復活
  • アイルランドや日本のユーザーからも「普通にProで使えている」という報告が多数
  • 既存ユーザーは最初から何も変わっていなかったことが確認された

⑧ 「誤情報でした」で終わらせない

  • 表面的には「A/Bテストだった、ごめんね」で終わったように見える
  • しかし裏にはAnthropicのビジネスモデルの転換点が隠されている
  • プランのあり方が静かに変わりつつあるサインを見逃してはいけない
ここからが本当に大事な話なんだよね。
背景：なぜこのテストをしたのか

⑨ Claude Codeが通常チャットと違う理由

  • Claude CodeはターミナルやVS Codeから使える公式コーディングツール
  • プログラムのファイル全体を丸ごと読み込んで、何分も連続でAIが動き続ける
  • 処理するデータ量が通常のチャットとは桁違いになる
  • サーバーへの負荷が非常に大きく、コストも跳ね上がる構造
普通のチャットとは全然違う使われ方をするのがポイント！

⑩ Proプランの制限が厳しすぎる問題

  • Proプランには直近5時間の使用量リセットの仕組みがある
  • Claude Codeで重い作業をたった12回やると、もう制限に引っかかるという声が多数
  • 「Proで全然使えない」「すぐ止まる」という不満は、今回の騒動よりずっと前からあった
  • 「結局Maxに課金するしかないじゃん」という声がずっと強く存在していた

⑪ 4月4日の先行施策：サードパーティ遮断

  • 4月4日、AnthropicはOpenClawなどのサードパーティツールに対してサブスクリプションの枠を使えなくする変更を突然実施
  • OpenClawはClaude Codeをより便利に使えるようにした別会社のツール
  • こういったツール経由でがっつり使い続けるユーザーが増え、サーバー負荷が一気に膨らんでいた
サードパーティ遮断は、今回のA/Bテストの前段階だったんだよね。

⑫ エージェントの連続使用がリソースを圧迫

  • Claude Codeのようなエージェント型ツールは、AIが自律的に長時間動き続ける
  • 通常チャットの何倍ものトークンを消費し、サーバーリソースを圧迫する構造
  • Anthropic自身も「エージェントの連続使用がリソースを圧迫している」と公式に認めていた
「使い方が変わってしまった」のが根本的な問題なんだよね。
Anthropicの本音と方向転換

⑬ 公式が認めた「プラン設計が追いつかない」

  • Anthropicのメンバーが「使用量が変わりすぎて、今のプランの設計が追いつかない」と公式に発言
  • 月20ドルのProプランを想定して作った体系が、Claude Codeという予期せぬ登場で成り立たなくなってきている
  • エージェント型の使い方に対して、従来の料金設計ではコスト回収が困難に
これはかなり本音が漏れた発言だよね。

⑭ プラン定義が静かに変わっている

  • 以前：Proプラン月20ドルでもClaude Codeが使えるのが当然という前提
  • 現在：「Proは人間が軽く使うもの」という方向にシフト中
  • 「がっつりコードを動かしたいならMaxかAPIにどうぞ」というメッセージ
  • 「使えなくなる」ではなく「使いにくくなる」設計への移行が正確な表現

⑮ 本気の市場調査だったA/Bテスト

  • 今回のA/BテストはMax（月100ドル）にアップグレードするユーザーがどのくらいいるかを確かめる実験
  • 表面上はテストだったが、本気の市場調査だったのは間違いない
  • リソース圧迫の問題は何も解決していないため、同じような動きが今後も続く可能性が十分にある
「誤情報でした」で終わらせると、重要なサインを見逃すことになるんだよね。

⑯ ユーザーの声が割れている理由

  • 「Proで使える」派：軽い使い方なら制限に引っかからないので問題ない
  • 「Proじゃ実質使えない」派：ヘビーに使うとすぐ制限に引っかかる
  • この割れ方自体が、プラン定義の変化の表れ
  • 制限に引っかかる頻度はユーザーの使い方によって全然違うのがポイント
どっちの意見も正しい、というのが今回の難しいところ。
対策と今後の動き方

⑰ もしProから使えなくなった場合の4つの選択肢

  • Maxプラン（月100ドル）にアップグレード。ヘビーユーザーなら一番安定した選択肢
  • API利用で使った分だけ払う従量課金。月額の上限がなく、自分でコスト管理できる人向け
  • CursorやGitHub Copilotなど別のAIコーディングツールへの乗り換え。価格競争が激しい今は有力な選択肢
  • 短い作業に絞ることで、Proの制限内に収める方法。ライトな用途ならまだまだ十分使える
自分の使い方と照らし合わせて、どれが合うか考えてみてね！

⑱ 今後も予告なしの変更が続く可能性

  • Anthropicは今回のような予告なしの変更を繰り返してきている
  • こまめにClaudeの料金ページとAnthropicの公式Xをチェックしておくのがおすすめ
  • 月20ドルで全部賄えていた時代が、Claude Codeという強力なツールの登場でじわじわと終わりに近づいている
  • サブスクリプションは「人間が軽く使うこと」を想定。エージェントの連続使用はMax以上かAPIを前提にしてほしいという方向性が明確に
今回の騒動は、AIツールの料金体系が変革期に入ったことを象徴しているんだよね。
