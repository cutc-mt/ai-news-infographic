① Workspace Agentsとは：クラウドで動くデジタル同僚

ChatGPTのクラウド上で動く新世代エージェントが「Workspace Agents」だ。これまでのChatGPTはチャット形式での対話が中心だったが、Workspace Agentsはスケジュール実行や長時間タスク、Slack連携など、自律的に動く「デジタル同僚」として機能する。ビジネスプラン・エンタープライズ限定で、個人向けはまだ未対応となっている。

② クラウド上で24時間稼働するエージェント

Workspace Agentsの最大の特徴はパソコンを閉じていてもクラウド上で動き続けることだ。従来のChatGPTはブラウザを開いて操作する必要があったが、Workspace Agentsならエージェントに指示を出せば、バックグラウンドで長時間のタスクを自律的に実行してくれる。スケジュール実行にも対応している。

③ 料金体系：クレジット制の従量課金

Workspace Agentsはクレジット制の従量課金で提供される。ただし5月6日までは無料で利用可能なので、今のうちに試しておくのがおすすめだ。以降は使った分だけ支払う仕組みで、エージェントの実行時間やタスクの複雑さに応じてクレジットが消費される。

④ 機能1：エージェントを簡単作成

日本語の指示だけでエージェントを作成できる。豊富なテンプレートが用意されており、選ぶだけですぐに使い始めることも可能だ。さらに驚きなのが「エージェントを作るエージェント」の存在で、指示を出すと自動的に16のスキルを追加してくれる。プログラミングの知識は一切不要だ。

⑤ 機能2：プロンプトとファイルを追加可能

エージェントにはPDFや画像などのファイルを参照させることができる。たとえば自社のPPTテンプレートをアップロードしておけば、そのフォーマットに沿った資料を自動で作成してくれる。ブランドガイドラインや社内規定の文書を読み込ませれば、社内ルールに準拠した出力が得られる。

⑥ 機能3：MCPで700以上の外部ツールに接続

MCP（Model Context Protocol）で700以上のアプリに接続できる。Gmail、Outlook、Stripe、Notion、Googleカレンダーなど、ビジネスで使うツールの大半に対応している。さらにカスタムMCPも追加可能で、自社専用のAPIや社内システムとも連携できるのが強みだ。

⑦ 機能4：エージェントスキルでカスタマイズ

Claude Codeで作成したスキルをアップロードして、エージェントに追加できる。スキルの中にファイルを保持させることも可能で、たとえば自社フォーマットのPPTテンプレートをスキル内に組み込んでおけば、いつでもその形式でスライドを生成できる。エージェントの能力を自由に拡張できる仕組みだ。

⑧ 機能5：メモリー機能で賢く成長

エージェントにはメモリー機能が備わっている。ユーザーとのやり取りを通じて学習し、どんどん賢くなっていく。プロンプトを変更しなくても、エージェントが自律的にアップデートされるのがポイントだ。「前回こうしていたほうが良かった」という学習を自動で反映してくれる。

⑨ 機能6：Slack連携でチームに溶け込む

Slack上でエージェントを@メンションするだけで呼び出せる。Slackの画面を離れることなく、エージェントにタスクを依頼して結果を受け取れる。スケジュール実行にも対応しているので、毎朝9時にニュース要約を投稿する、といった自動化も簡単に設定可能だ。

⑩ 機能7：チーム共有で組織全体に展開

作成したエージェントはChatGPT上でチーム共有できるだけでなく、Slackのチャンネル経由でも共有可能だ。一度優秀なエージェントを作れば、チーム全員が同じデジタル同僚を利用できる。組織のナレッジやベストプラクティスをエージェントに込めて、横展開する使い方が期待できる。

⑪ ユースケース：秘書エージェントが毎朝のルーティンを自動化

カレンダー、メール、SNSを統合した秘書エージェントを作成できる。毎朝スケジュールの要約をSlackに投稿したり、重要なメールをピックアップして優先順位をつけて通知したりする。MCPで各種ツールに接続するので、情報の取りこぼしがないのが魅力だ。

⑫ ユースケース：リサーチエージェントが調査を代行

Web検索とストーリーライティングのスキルを組み合わせたリサーチエージェントを構築できる。テーマを指定すれば、自動的にWebを巡回して情報を収集し、読みやすいレポートにまとめてくれる。市場調査や競合分析など、毎回手動で行っていたリサーチ作業を自動化できる。

⑬ ユースケース：データベースQAでSlackからDBクエリ

Supabaseなどのデータベースと連携すれば、Slackから自然言語でDBにクエリを投げられる。「今月の売上トップ5を教えて」とSlackでメンションするだけで、エージェントがDBにアクセスして結果を返してくれる。SQLの知識がなくてもデータ分析が可能になる。

⑭ ユースケース：自社テンプレートで資料を自動生成

自社のPPTテンプレートをエージェントに読み込ませておけば、ブランドに統一されたスライドを自動生成できる。タイトルと大まかな内容を指示するだけで、デザインやレイアウトはテンプレートに沿って自動で適用される。営業資料や提案書の作成時間を大幅に削減できる。

⑮ 注意点1：Slack初期設定で権限の有効化が必要

Slack連携を始めるには、ユーザーグループ作成権限の有効化が必要だ。これはSlackの管理画面から設定する必要がある。また、エージェントを呼び出す際は@メインアカウントではなく、ハンドル名でメンションする必要がある点にも注意しよう。間違ったメンション先ではエージェントが反応しない。

⑯ 注意点2：バージョン管理の落とし穴

エージェントをアップデートしても、既存のチャットには新バージョンが反映されない。新しい機能や修正を利用するには、新しいスレッドを立ち上げる必要がある。この挙動を知らないと「アップデートしたのに反映されていない」と混乱しがちなので、チーム内で事前に共有しておくことが重要だ。

⑰ 注意点3：権限管理には専用アカウントを推奨

エージェントを共有すると、所有アカウントのデータが見えるリスクがある。個人アカウントでエージェントを作成して共有すると、個人のチャット履歴などが意図せず露出する可能性がある。そのため、チーム用の専用アカウントの作成が推奨されている。セキュリティ面で慎重な運用が求められる。

⑱ Claude Code vs Workspace Agents：使い分けが鍵

Claude Codeはパソコン上で動き、自由度が高く範囲が広い。自分で環境を設定して細かなカスタマイズができる。一方、Workspace Agentsはクラウドで動き、チーム共有やSlack連携が強みだ。どちらかが優れているわけではなく、似て非なるもの。用途に応じて使い分けることが2026年の正しいAI活用法と言える。
