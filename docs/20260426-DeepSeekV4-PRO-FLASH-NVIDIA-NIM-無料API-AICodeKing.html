② DeepSeek V4 Proのスペック：1.6兆パラメータの超巨大モデル

DeepSeek V4 ProはMixture of Experts（MoE）アーキテクチャを採用した超大型モデルです。NVIDIAのモデルページによると、総パラメータ数は1.6兆で、アクティブパラメータは約490億です。さらに、最大100万トークンのコンテキストウィンドウをサポートしており、これは非常に巨大です。ハードな推論タスク、コーディング、長文脈エージェント、ツール利用、ドキュメント分析など、最強のDeepSeek V4体験を求める場面に最適です。

⑤ NVIDIA NIMとは：ホスト型APIエンドポイントの仕組み

NIMは「NVIDIA Inference Microservice」の略称で、NVIDIAが提供するホスト型のAPIエンドポイントです。簡単に言うと、NVIDIAが自社のGPUインフラ上でモデルを動かし、それを開発者向けの通常のAPIとして公開してくれています。独自のGPU環境を構築する必要がなく、DeepSeek公式プラットフォームで即座にトークン単位の課金をしなくても、手軽に最新モデルを試せるという仕組みです。

自分でGPUを用意しなくても、NVIDIAがすべてホストしてくれるから超簡単！
NVIDIA NIM ホスト型API GPU不要

⑥ OpenAI互換API：既存ツールですぐに接続可能

最大の魅力は、APIがOpenAI互換であることです。あなたのツールがOpenAI互換プロバイダーをサポートしていれば、NVIDIAにかなり簡単に接続できます。ベースURLは integrate.api.nvidia.com/v1 で、チャット補完エンドポイントは integrate.api.nvidia.com/v1/chat/completions です。NVIDIA専用SDKは不要で、OpenAI SDKをそのまま使えます。システムメッセージ、ユーザーメッセージ、アシスタント応答など、おなじみのメッセージ構造がそのまま使えます。

base_url = "https://integrate.api.nvidia.com/v1"
model = "deepseek-ai/deepseek-v4-pro"
OpenAI互換 OpenAI SDK 専用SDK不要

⑦ モデル名の重要な違い：プロバイダープレフィックスに注意

モデル名の指定は非常に重要です。DeepSeek公式APIではモデル名は deepseek-v4-pro や deepseek-v4-flash ですが、NVIDIA NIMではプロバイダープレフィックス付きの名前を使います。具体的には deepseek-ai/deepseek-v4-pro と deepseek-ai/deepseek-v4-flash です。この間違ったモデル名を入れるとエラーになるため、NVIDIA NIM用の名前を正しく使う必要があります。

NVIDIA NIM用:
deepseek-ai/deepseek-v4-pro
deepseek-ai/deepseek-v4-flash
モデル名 deepseek-ai/ プレフィックス必須

⑨ コードからの利用方法：OpenAI SDKで簡単呼び出し

コードから利用する場合、最も簡単な方法はOpenAI SDKを使うことです。OpenAIパッケージをプロジェクトにインストールし、クライアントを作成する際にベースURLを integrate.api.nvidia.com/v1 に設定します。APIキーにはNVIDIAのAPIキーを使用し、モデルパラメータに deepseek-ai/deepseek-v4-pro または deepseek-ai/deepseek-v4-flash を指定するだけです。

from openai import OpenAI
client = OpenAI(
  base_url="https://integrate.api.nvidia.com/v1",
  api_key="nvapi-xxxxx"
)
response = client.chat.completions.create(
  model="deepseek-ai/deepseek-v4-pro",
  messages=[...]
)

⑩ Reasoning Effortパラメータ：推論の強さを3段階で調整

NVIDIAのドキュメントによると、DeepSeek V4のエンドポイントはreasoning_effortパラメータをサポートしています。設定値は none、high、max の3種類です。none は思考モードを無効にして高速な非思考モードで動作し、high は通常の推論モード（デフォルト）、max は最強の推論モードですが低速でトークン消費も多くなります。モデルを切り替えずに、同じモデルで推論の強さを調整できるのが便利です。

モデルを変えなくても、パラメータ1つで推論の強さをコントロールできる！
reasoning_effort none / high / max 推論調整

⑪ Reasoning Effortの使い分け推奨設定

Flashでは none か high を試すのがおすすめです。Proでは、通常のコーディングには high、本当に難しいタスクには max を使うのが推奨されます。この仕組みが便利なのは、タスクに応じて推論の強さを変えたい時に別のモデルに切り替える必要がないことです。同じモデルでパラメータだけ変更すれば、行動パターンを柔軟に変えられます。

Flash: reasoning_effort="none" or "high"
Pro:   reasoning_effort="high" (通常)
       reasoning_effort="max"  (難タスク)
推奨設定 Flash＝none/high Pro＝high/max

⑮ 最大出力トークン数の注意点：NIMでは16,384が上限

DeepSeekの公式APIドキュメントでは非常に大きな最大出力長が記載されていますが、NVIDIAのNIMエンドポイントでは現在max_tokensが最大16,384までとドキュメントされています。モデル自体が100万トークンのコンテキストウィンドウを持っていても、ホスト側の制限は異なる場合があります。特にAIコーディングツールでは、ツール自体が全リポジトリを送信しないことが多いです。ファイルをチャンク化したり、コンテキストを要約したり、独自のプロンプト制限を使ったりします。

100万トークンはモデルの能力であって、すべてのツールで使えるとは限らない！
max_tokens: 16,384 ホスト制限 注意点
