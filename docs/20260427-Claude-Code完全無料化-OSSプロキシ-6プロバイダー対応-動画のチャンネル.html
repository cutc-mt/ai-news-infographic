① Claude Codeを無料化するOSSプロキシ

GitHubリポジトリAlishahryar1/free-claude-codeが、Claude Code CLIとVSCode拡張を完全無料で利用可能にするオープンソースプロキシを公開した。このプロキシはAnthropic APIキーを一切必要とせず、複数のAIプロバイダーをバックエンドとして利用できる。Claude Codeの公式機能をそのままに、コストだけをゼロにする画期的なアプローチだ。

② Anthropic APIキーが不要な仕組み

通常、Claude Codeの利用にはAnthropic APIキーと従量課金が必要だが、このプロキシはリクエストを中継して別のAIプロバイダーにルーティングする。Claude Code側は自分がAnthropicと通信していると認識したまま、実際にはNVIDIA NIMやOpenRouterなどの無料モデルに接続される。APIキーの取得もクレジットカードの登録も一切不要だ。

③ 6つの対応プロバイダー

プロキシは6つのプロバイダーに対応している。NVIDIA NIM（40 req/min無料、推奨）、OpenRouter（数百のモデル対応、無料モデルあり）、DeepSeek（直接API）、LM Studio（完全ローカル）、llama.cpp（完全ローカル）、Ollama（完全ローカル）。クラウドから完全オフラインまで、用途に合わせて選択できる。

④ NVIDIA NIM: おすすめのプロバイダー

NVIDIA NIMは分間40リクエストまで無料で利用できる、プロジェクト推奨のプロバイダーだ。Kimi K2.5やDeepSeek R1などの高性能モデルにアクセス可能。モデルマッピングの例として、MODEL_OPUS=nvidia_nim/moonshotai/kimi-k2.5を設定すれば、Claude CodeのOpus呼び出しが自動的にKimi K2.5にルーティングされる。登録もAPIキー取得も無料で完結する。

⑤ OpenRouter: 数百モデルへのアクセス

OpenRouterは数百のAIモデルに統合アクセスできるハブサービスだ。無料で利用できるモデルも多数用意されている。設定例としてMODEL_OPUS=open_router/deepseek/deepseek-r1-0528:freeを指定すれば、DeepSeek R1の無料版がOpusの代替として動作する。モデルの差し替えも.envの変更だけで済むため、様々なモデルを試したい場合に最適だ。

⑥ LM Studio / Ollama: 完全ローカル動作

インターネット接続すら不要な完全オフライン環境での利用も可能だ。LM StudioではMODEL_OPUS=lmstudio/unsloth/MiniMax-M2.5-GGUFのようにローカルモデルを指定できる。Ollamaも同様にMODEL_OPUS=ollama/llama3.1で動作する。データが外部に一切送信されないため、セキュリティ要件の厳しい環境でも安心して利用できる。

⑦ ゼロコストで40リクエスト/分を実現

NVIDIA NIMを利用した場合、完全無料で分間40リクエストが可能だ。これは個人開発者の通常利用には十分なスループットであり、Claude Codeの月額$20〜$200のサブスクリプションと同等の体験が得られる。OpenRouterの無料モデルを併用すれば、複数プロバイダーを切り替えてレート制限を回避することも可能だ。コストを気にせずAIコーディングに没頭できる。

⑧ ドロップイン置換: 変更は環境変数2つだけ

Claude Code CLIやVSCode拡張のコード変更は一切不要だ。必要な設定は環境変数を2つ追加するだけ。ANTHROPIC_BASE_URL=http://localhost:8082を設定するだけで、プロキシ経由で免费プロバイダーに接続される。Claude Code自体は公式版をそのまま使い続けられ、アップデートにも影響しない。導入のハードルが極めて低い設計となっている。

⑨ モデルマッピングで柔軟なルーティング

Claude Codeが内部的に呼び出すOpus/Sonnet/Haikuの各モデルを、それぞれ別のプロバイダーやモデルにルーティングできる。たとえば、Opusの呼び出しにはNVIDIA NIMの高性能モデルを、Sonnetの呼び出しにはOpenRouterの高速モデルを、Haikuの呼び出しにはローカルのOllamaを、というように用途に応じて最適な組み合わせを自由に設定可能だ。

⑩ 思考トークンの自動変換

DeepSeek R1やKimi K2.5などのモデルは推論タグを使って思考プロセスを出力する。プロキシはこれらの推論タグを自動的に検出し、ネイティブなClaude思考ブロック形式に変換する。これにより、Claude Codeの拡張思考機能（Extended Thinking）と同等の体験を、無料モデルでも利用できる。推論の過程が可視化されるため、回答の信頼性も確認しやすい。

⑪ ヒューリスティックツールパーサー

無料モデルの中には、Claude Codeが期待する構造化されたツール呼び出し形式で出力しないものがある。このプロキシはヒューリスティックツールパーサーを搭載しており、モデルのテキスト出力からツール呼び出しを自動的に検出して構造化データに変換する。これにより、Claude Codeとの互換性が大幅に向上し、無料モデルでも実用的なコーディングアシスタントとして機能する。

⑫ リクエスト最適化で応答を高速化

プロキシは5カテゴリの自明なAPI呼び出しをローカルで拦截する。プロバイダーにリクエストを送るまでもない単純な処理は、プロキシ自身が即座に応答を返す仕組みだ。これにより、不要なネットワーク通信を削減し、レスポンスの待ち時間を大幅に短縮できる。無料プロバイダーのレート制限に対しても、リクエスト数を節約する効果がある。

⑬ スマートレート制限で安定運用

無料プロバイダーにはリクエスト数の制限があるが、プロキシはローリングウィンドウスロットルで制限に近づくことを防ぐ。さらに、429エラー（Too Many Requests）を受信した場合は指数バックオフで自動再試行する。ユーザーが意識することなく、レート制限の範囲内で最大限のパフォーマンスを引き出す設計となっている。

⑭ Discord/Telegram Botでリモートコーディング

プロキシにはDiscordとTelegramのBot機能が内蔵されている。メッセージプラットフォームから直接プロキシに指示を送ることで、リモートでの自律コーディングが可能だ。外出先からスマートフォンでコードの修正指示を出したり、CI/CDパイプラインに組み込んで自動修正させたりといった使い方ができる。BaseProviderとMessagingPlatformのABCにより、新しいプラットフォームの追加も容易だ。

⑮ サブエージェント制御でリソースを節約

Claude Codeは複数のサブエージェントを並列起動することがあるが、無料環境ではリソースの浪費につながる。プロキシはTask Tool拦截機能により、run_in_background=Falseを強制し、サブエージェントの並列実行を制御する。これにより、レート制限のある無料プランでも安定して動作し、リクエストの無駄遣いを防ぐことができる。

⑯ セットアップ手順（5ステップで完了）

導入はシンプルだ。まずPython 3.14をインストール（uv推奨）。次にリポジトリをgit cloneして.envファイルを設定。プロバイダーを選択（NVIDIA NIM推奨）し、uvicorn server:app --host 0.0.0.0 --port 8082でプロキシを起動。最後にClaude Code側でANTHROPIC_BASE_URL=http://localhost:8082を設定すれば完了だ。数分で無料のClaude Code環境が手に入る。

⑰ VSCode拡張とIntelliJでの設定

VSCode拡張版Claude Codeでも利用可能だ。claudeCode.environmentVariablesにANTHROPIC_BASE_URLとANTHROPIC_AUTH_TOKENを設定し、拡張をリロード。Anthropic Consoleをクリックしてauthorizeすれば動作開始する。IntelliJでもacp-agents/installed.jsonに設定を追加することで対応可能。主要なIDEすべてで無料Claude Codeが利用できる。

⑱ セキュリティと認証のオプション

プロキシにはANTHROPIC_AUTH_TOKENによるアクセス制限機能が用意されている。このトークンを設定すれば、プロキシへの不正アクセスを防止できる。公開ネットワークでプロキシを運用する場合は必ず認証を有効にすることが推奨される。ローカル環境での個人利用であれば不要だが、チーム共有やリモートアクセスを考慮するならセキュリティ設定を忘れずに行おう。
