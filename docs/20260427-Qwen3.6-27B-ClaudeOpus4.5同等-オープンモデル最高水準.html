Qwen3.6-27B が Claude Opus 4.5 並みの性能を達成

270億パラメータで397Bモデルを超える／オープンソースの最高水準を更新

Qwen3.6-27B Alibaba Apache 2.0 オープンモデル エージェントコーディング RTX 4090で動作

① Qwen3.6-27Bとは：オープンモデルの新フラグシップ

Qwen3.6-27Bは、AlibabaのTongyi Lab（Qwenチーム）が開発した270億パラメータのオープンソースモデルだ。2026年4月22日にリリースされ、Apache License 2.0で商用利用も可能となっている。画像と動画に対応するマルチモーダルモデルであり、思考モード（Thinking）と非思考モード（Non-Thinking）の両方をサポートする。

② 397B MoEモデルを27Bで超える衝撃

最も驚かされるのは、総パラメータ397億のQwen3.5-397B-A17Bを開発者にとって重要なタスクで上回ったことだ。わずか27Bという小型モデルで、前世代のフラグシップである巨大モデルを凌駕する性能を実現している。これはデプロイと運用が大幅に容易になることを意味する。

③ SWE-bench Verified：77.2%で397Bを超える

ソフトウェアエンジニアリングの実力を測るSWE-bench Verifiedで77.2%を記録した。これは397B MoEモデルのスコアを上回る結果であり、実務的なコーディング能力が非常に高いことを示している。なおClaude 4.6の81.2%には及ばないものの、Claude 4.5 Opus（2世代前）レベルに到達している。

④ Terminal-Bench 2.0：Claude Opus 4.5と同スコア

ターミナルでのコーディング能力を測るTerminal-Bench 2.0で59.3点を記録し、Claude 4.5 Opusと全く同じスコアを達成した。エージェント的なタスク遂行能力において、オープンモデルがプロプライエタリモデルの最前線に追いついたことを示す象徴的な結果だ。

⑤ GPQA Diamond：87.8でClaude Opus 4.5を上回る

高度な推論力を測るGPQA Diamondで87.8点を記録し、Claude 4.5 Opusの87.0点を上回った。専門家レベルの質問に対する理解力と推論能力において、27Bのオープンモデルがクローズドモデルを凌駕する結果を叩き出している。

⑥ 12ベンチマーク全てで前世代を上回る

Qwen3.6-27Bは12のベンチマークすべてで前世代のQwen3.5-27BとGoogle Gemma4-31Bを上回る圧倒的な成績を収めた。SWE-bench ProやSkillBenchでも397Bモデルを超えており、単一の得意分野だけでなく、幅広いタスクで総合的に強いモデルに仕上がっている。

⑦ RTX 4090で動作する実用的なサイズ

RTX 4090（24GB VRAM）で動作可能であり、24GBのMacでも問題なく動く。397Bモデルのような複数GPU環境が不要で、一般的なハイエンドPCでローカル実行が可能なのが大きな強みだ。FP8量子化版も公開されており、さらに軽量な運用ができる。

⑧ 量子化版でも高品質な結果を維持

4-bit量子化版でも良好な結果が確認されている。Simon WillisonがPelican benchmarkで検証した結果、4-bit量子化しても性能の劣化が最小限に抑えられていた。FP8版と4-bit版の両方が提供されており、ハードウェアリソースに応じた柔軟なデプロイが可能だ。

⑨ フラグシップ級のエージェントコーディング性能

Qwen3.6-27Bは「Flagship-Level Agentic Coding」を標榜しており、自律的にコーディングタスクを遂行する能力が非常に高い。エージェント的なタスク実行、コードの理解と修正、複数ステップの開発ワークフローをこなす能力において、オープンモデルとして最高水準を達成している。

⑩ 人気コーディングアシスタントとシームレス統合

OpenClaw、Claude Code、Qwen Codeなどの人気のコーディングアシスタントとシームレスに統合できる。既存の開発ワークフローにそのまま組み込めるため、新しいツールを学ぶ必要がない。開発者は使い慣れた環境のまま最新のオープンモデルを活用できる。

⑪ Claude 4.5 Opusとの head-to-head 比較

Terminal-Bench 2.0では同スコア（59.3）で並び、GPQA DiamondではQwen3.6-27Bが上回る（87.8 vs 87.0）結果となった。SWE-bench VerifiedではClaude 4.6の81.2%に及ばないが、Claude 4.5 Opusレベルに到達。クローズドモデルとオープンモデルの差が急速に縮まっていることを実証している。

⑫ Hugging FaceとModelScopeで重みを公開

モデルの重みデータはHugging Face（Qwen/Qwen3.6-27B）で公開されており、誰でもダウンロード可能だ。FP8版（Qwen/Qwen3.6-27B-FP8）も併せて公開されている。さらに中国のModelScopeでも配布されており、地域を問わずアクセスしやすい。

⑬ Alibaba Cloud Model StudioでAPI提供予定

ローカル実行だけでなく、Alibaba Cloud Model StudioでAPI経由での利用も予定されている。自前でGPUを用意できないユーザーでも、クラウド経由でQwen3.6-27Bの性能を活用できるようになる。商用利用もApache 2.0ライセンスで許可されているため、企業での導入ハードルも低い。

⑭ マルチモーダル対応：画像と動画を理解

Qwen3.6-27Bはテキストだけでなく、画像と動画の入力にも対応するマルチモーダルモデルだ。視覚情報を理解して推論や回答を行えるため、コーディング以外のタスクでも幅広い活用が期待できる。27Bのサイズでマルチモーダルを実現している点も特筆すべきだ。

⑮ 思考モードと非思考モードの使い分け

思考モード（Thinking）と非思考モード（Non-Thinking）の2つの推論モードを搭載している。複雑な推論が必要なタスクでは思考モードで深く考え、シンプルな質問には非思考モードで高速に回答するという使い分けが可能だ。タスクに応じた最適な推論戦略を選択できる。

⑯ Google Gemma4-31Bを全12ベンチマークで圧倒

GoogleのオープンモデルであるGemma4-31Bを全12ベンチマークで上回る結果を叩き出した。パラメータ数でも4B少ない27Bでありながら、全指標でGemma4を凌駕している。オープンモデルのリーダー交代を明確に示す結果と言える。

⑰ 開発チームの声明：エージェントコーディングの新世代

Qwenチームは「Qwen3.6-27Bは、はるかに大きい397BパラメータのQwen3.5-397B-A17Bを開発者にとって重要なタスクで上回りながら、デプロイと運用がより容易」と宣言。さらに「Qwen3.6シリーズは全スケールでのエージェントコーディングの世代を画す包括的なモデルスイートとなっている」と述べている。

⑱ オープンモデルの新時代：1/4のサイズで同等以上の性能

Qwen3.6-27Bが示した最大の意義は、397Bの1/4のサイズで同等以上の性能を実現したことだ。これにより、より多くの開発者が高性能なAIを手元の環境で利用できるようになる。オープンモデルとプロプライエタリモデルの差はほぼ消滅しつつあり、AIの民主化が大きく前進したと言える。
