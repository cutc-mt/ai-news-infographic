② DeepGEMM: LLM推論を劇的に高速化するFP8カーネル

DeepSeekが開発したLLM推論向けのFP8 GEMM（General Matrix Multiply）カーネル。TransformerEngineのFP8数値計算をわずか数百行のCUDAコードに圧縮し、同等以上の性能を実現している。シンプルでありながら極めて効率的な設計が大きな話題を呼んでいる。

GitHub

deepseek-ai/DeepGEMM

③ DeepGEMM: FP8xFP4混合精度でメモリ帯域を削減

DeepGEMMはFP8xFP4の混合精度モードをサポートしている。これによりMoE（Mixture of Experts）推論におけるメモリ帯域のボトルネックを効果的に解消できる。大量のパラメータを持つMoEモデルでは、メモリ転送が推論速度の最大の障害となるが、この混合精度アプローチで大幅な高速化が可能だ。

数百行
TransformerEngineのFP8計算をこの行数に圧縮

④ DeepGEMM: JITコンパイルと対応GPUアーキテクチャ

実行時JITコンパイルを採用しており、C++風の読みやすいコードで書かれている。対応するGPUアーキテクチャは以下の通り：

  • H100/H800/H200（SM90 Hopper）
  • B200/B300（SM100 Blackwell）
  • TMA（Tensor Memory Accelerator）に対応
統合済み

NVIDIA TensorRT-LLM、vLLMに既に統合されている。

⑤ DeepGEMM: なぜオープンソースコミュニティで注目されているか

DeepGEMMが注目される理由は、複雑だったFP8行列演算を極めてシンプルに実装した点にある。数百行のコードでNVIDIAの最適化ライブラリに匹敵する性能を出すことは、オープンソースコミュニティにとって大きな意味を持つ。誰でもコードを読み、学び、改善に参加できるのが最大の強みだ。

ポイント

DeepSeekのMoEモデル（DeepSeek-V3など）の推論を高速化する実用的な基盤技術として機能する。

⑥ MarkItDown: あらゆるファイルをMarkdownに変換

Microsoftが開発したPythonユーティリティ。PDF、PowerPoint、Word、Excel、画像（OCR）、音声（文字起こし）、HTML、CSV、JSON、XML、ZIP、YouTube URL、EPubなど、多彩なフォーマットをMarkdownに一括変換できる。LLMのパイプラインに最適化された設計が特徴だ。

GitHub

microsoft/markitdown

⑦ MarkItDown: 圧倒的なフォーマット対応力

MarkItDownが対応するファイルフォーマットは非常に幅広い：

PDF PowerPoint Word Excel 画像（OCR） 音声（文字起こし） HTML CSV JSON XML ZIP YouTube URL EPub

見出し・リスト・テーブル・リンクなど、重要な文書構造をMarkdownとして保持する点が大きな特徴。

⑧ MarkItDown: コマンド1行で変換が完結

MarkItDownはPython 3.10以上で動作し、コマンドラインから1行で変換が可能だ：

# インストール（全機能版）
$ pip install 'markitdown[all]'

# ファイルをMarkdownに変換
$ markitdown path-to-file.pdf > document.md

さらにプラグイン対応で拡張性も備えている。Markdownはトークン効率が高いため、LLMへの入力として最適なフォーマットに自動的に変換される。

⑨ LiteRT-LM: エッジデバイスでLLMを動かすGoogleの挑戦

Google AI Edgeが開発した、エッジデバイス向けの本番レディなオープンソースLLM推論フレームワーク。モバイルや組み込みデバイスでLLMを直接実行できるようにする高性能・クロスプラットフォームなソリューションだ。

対応モデル

Gemma 4モデルに対応。Hugging FaceでLiteRT-LM版のモデルが配布されている。

⑩ LiteRT-LM: スマホや組み込み機器でLLMを稼働

LiteRT-LMの主な特徴をまとめる：

  • クロスプラットフォーム対応で多様なデバイスにデプロイ可能
  • モバイル・組み込みデバイスでLLMを直接実行
  • Gemma 4モデルに標準対応
  • Hugging Faceで最適化済みモデルを配布
  • Google AI Edge Galleryでデモ体験が可能
利点

クラウド通信なしで端末上でLLMが動くため、プライバシー保護とレイテンシ削減が同時に実現する。

⑪ TimesFM 2.5: Google Researchの時系列予測基盤モデル

Google Researchが開発した事前学習済みの時系列予測基盤モデル。タスク特化の学習なしに、未知のデータセットに対しても高精度な予測を提供できる。時系列データの分析に革命をもたらす可能性を秘めている。

GitHub

google-research/timesfm

⑫ TimesFM 2.5: デコーダーのみのTransformerによる軽量設計

TimesFM 2.5はデコーダーのみのTransformerアーキテクチャを採用。200Mパラメータと軽量でありながら、16,000トークンのコンテキスト長をサポートしている。

200M
パラメータ数 — 軽量ながら高性能
16,000
トークンのコンテキスト長

オプションで30M連続分位点モジュールも利用可能で、予測の不確実性を定量化できる。

⑬ TimesFM 2.5: トレンド・季節性・残差に分解してモデリング

TimesFM 2.5は時系列データをトレンド・季節性・残差の3つの成分に分解してモデリングする。このアプローチにより、複雑な時系列パターンを効果的に捉えることができる。

トレンド（長期的な傾向）
季節性（周期的な変動）
残差（ランダムなノイズ）

JAXベースで実装されており、高速な計算が可能。

⑭ turbovec: 高速なベクトル検索に特化したオープンソース

turbovecは高速なベクトル検索と類似性検索に特化したオープンソースプロジェクト。RAG（Retrieval-Augmented Generation）や意味検索など、ベクトルデータベースが不可欠なAIアプリケーションにおいて重要な役割を果たす。

  • 高速なベクトル類似性検索
  • 大規模なベクトルデータの効率的な管理
  • オープンソースで透明な実装
  • AIパイプラインへの組み込みが容易
