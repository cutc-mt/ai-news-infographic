① CodeRabbitとは: プロ級レビューが自動で届く

PRを作成すると自動でコードレビューをしてくれるAIツール。GitHub Appとして導入するだけで、すぐに利用を開始できる。レビューの質と量は実務レベルの品質を確保しており、個人開発者からチーム開発まで幅広く対応している。

導入の魅力

設定不要でGitHubにインストールするだけ。数秒でPR内にレビューコメントが自動表示される。

② なぜCodeRabbitが必要なのか

個人開発ではレビュー相手がいないという根本的な課題がある。チーム開発でもレビュー負荷が高く、人的リソースの限界が来やすい。セキュリティホールやバグを早期に発見できる価値は非常に大きく、レビュアーにとってもPRの内容を素早く把握できるメリットがある。

個人開発 → レビュー相手が不在
チーム開発 → レビュー負荷が過大
CodeRabbit → 24/365自動レビュー

③ Claude Code Actionsとの違い

Claude Codeは`/claude`コマンドを手動で打つ必要がある。一方、CodeRabbitはPRを開いた瞬間にレビューが自動的に始まる。常にレビューが走るので見落としを防ぐことができ、コマンドを忘れるリスクもない。

Claude Code

手動でコマンド入力
忘れるとレビューなし

VS

CodeRabbit

PR作成で自動実行
見落としゼロ

インストール時に、全リポジトリに対して適用するか、特定のリポジトリのみに絞るかを選択できる。アクセス権限もこの段階で設定し、プライベートリポジトリにも対応している。

⑥ 導入ステップ3: .coderabbit.yamlの作成

リポジトリのルートに設定ファイルを追加することで、プロジェクト固有のレビュー観点を定義できる。これはCLAUDE.mdに近いイメージで、AIにプロジェクトのコンテキストを教えるための非常に重要なファイルだ。

位置づけ

CLAUDE.mdがClaude向けなら、.coderabbit.yamlはCodeRabbit向け。プロジェクトのルールや期待をAIに伝える橋渡し役。

⑦ .coderabbit.yamlの基本設定

言語からレビュートーンまで、細かくカスタマイズ可能だ。

language: 'ja' # 日本語でレビュー reviews: profile: 'chill' # マイルドなレビュー request_changes_workflow: false # 承認のみ high_level_summary: true # 全体サマリ生成 poem: false # 詩の生成をオフ collapse_walkthrough: false # レビュー全文表示
chill 以外にも strict や moderate が選べる

⑧ path_filtersでレビュー対象を制御

レビューのノイズを減らすために、不要なファイルを除外する設定が重要だ。lockファイルやnode_modules、自動生成ファイルを除外することで、レビューの精度と可読性が向上する。

reviews: path_filters: - "**/package-lock.json" - "**/node_modules/**" - "**/*.generated.*" - "**/.next/**" - "**/dist/**"
効果

ノイズを減らすことで、本当に重要な箇所にレビューが集中するようになる。

⑨ path_instructionsの活用

ファイルパスごとにレビュー観点をカスタマイズできる強力な機能だ。TypeScriptファイル、特定のディレクトリ、Prismaスキーマなど、それぞれに最適なチェック観点を設定できる。

reviews: path_instructions: - path: "src/**/*.{ts,tsx}" instructions: | Zodバリデーションを確認。 N+1クエリに注意。 - path: "prisma/schema.prisma" instructions: | マイグレーション影響を確認。 $queryRawUnsafeを禁止。

⑩ TypeScript/Next.js向けのレビュー観点

Next.js特有の落とし穴をCodeRabbitが自動で検出してくれる。

Critical 不要なuse clientディレクティブの検出
Critical Server Actionsのauth()呼び出し確認
Critical Route Handlersのセッションチェック
Warning Zodバリデーションの有無
Warning any型の使用・不要な型キャスト
Info TanStack Queryのキー適切性
Info PrismaのN+1クエリ検出

⑪ Prismaスキーマ向けのレビュー観点

データベーススキーマ変更は影響範囲が大きいため、慎重なレビューが不可欠だ。CodeRabbitは以下の観点を自動でチェックする。

Critical マイグレーションによる既存データへの影響
Critical $queryRawUnsafeの使用検出
Info 適切なインデックスの確認
Info リレーション設計の適切性

⑫ chat.auto_replyでPR内チャット対応

auto_reply: trueを設定すると、CodeRabbitがレビューコメントに自動で応答してくれる。レビューへの質問にAIが回答し、改善提案に対する議論がスムーズに進む。

chat: auto_reply: true # コメントに自動応答
活用シーン

「なぜこの指摘をしたの？」「どう修正すればいい？」といった質問に即座に回答が得られる。

⑬ PRサマリと図解の自動生成

PR全体のハイレベルサマリを自動生成してくれる。変更点の図解（チャートやファイル構造図など）も自動で作成され、レビュアーがPRの内容を素早く把握できるようになる。
