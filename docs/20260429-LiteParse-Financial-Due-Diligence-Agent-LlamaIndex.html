⑦ Financial Agentの全体構成：4つのフェーズで完結

このワークショップで構築するエージェントは、4段階のパイプラインで構成されている。

1. Document Loader
  LiteParseでPDFを読み込み

2. Data Extraction
  テーブルとテキストを分離して抽出

3. Analysis
  財務指標を計算・比較

4. Report Generation
  分析結果をレポートとして出力

LlamaIndexのAgentフレームワークを使用し、ReActパターンでツール呼び出しを行う設計になっている。

最初に必要なライブラリのインストールとAPIキーの設定を行う。数コマンドで環境が整う。

pip install llama-index-liteparse

# APIキーの設定

export LITEPARSE_API_KEY="..."
ワンポイント

Python 3.9以上を推奨。仮想環境（venvやconda）の使用を強くおすすめする。

⑨ 実装ステップ2：PDFのパース処理

LiteParseを使って財務PDFを読み込み、テーブルとテキストを分離して抽出する。

# LiteParseでPDFをパース
from llama_index_liteparse import LiteParse

parser = LiteParse()
result = parser.parse(
  "financial_report.pdf"
)

# テーブルとテキストを分離
tables = result.tables
text_blocks = result.text_blocks

パース結果は構造化されたデータとして保存され、後段の分析でそのまま利用できる。

⑩ 実装ステップ3：抽出データの構造化と正規化

抽出したテーブルを分析可能な形式に変換する。数値の標準化と日付の正規化が重要なステップだ。

変換項目変換前変換後
通貨記号$1,234,5671234567
桁区切り1.234.5671234567
日付形式Jan 15, 20252025-01-15
パーセント15.3%0.153
正規化の意義

データ形式を統一することで、LLMが数値を正確に比較・計算できるようになる。

⑪ 実装ステップ4：LlamaIndex Agentの構築

LlamaIndexのAgentフレームワークを使って、検索・分析・レポート生成のツールを定義する。

from llama_index.core.agent import
  ReActAgent

# ツールの定義
tools = [
  search_financial_data,
  analyze_revenue,
  generate_report
]

# エージェントの構築
agent = ReActAgent.from_tools(
  tools,
  llm=llm,
  verbose=True
)

財務分析に特化したプロンプト設計が精度の鍵を握る。
