Cloudflare x Stripe Projects

エージェントが自動でアカウント作成・ドメイン購入・本番デプロイを実行する新時代

2026年4月30日発表

①ついに解禁: エージェントによる本番デプロイ

  • CloudflareがStripeと共同設計した新プロトコルを発表
  • AIエージェントが自動でアカウント作成から本番環境へのデプロイまで完結
  • これまで人間が手動で行っていた作業がついに自動化された

②これまでの課題: 人間がボトルネックに

  • コーディングエージェントはソフトウェア構築が得意だった
  • しかし本番デプロイにはアカウント・支払い手段・APIトークンが必要
  • これらのセットアップは人間が手動で対応するしかなかった
  • エージェントの能力が活かしきれない状態が続いていた

③解決策: エンドツーエンドの自動化

  • アカウント作成 → 有料サブスク開始
  • ドメイン登録 → APIトークン取得
  • 本番デプロイまで一発で実行できるように
  • 人間は指示を出すだけで完了する

④Stripe Projectsとは

  • stripe projects initでプロジェクトを初期化
  • エージェントに自然言語で指示を与えるだけ
  • エージェントが自動的に必要なサービスを設定してデプロイ
  • 開発者はコードを書くことに集中できる

⑤プロトコルの3つの柱

  • Discovery: 利用可能サービスのカタログを参照
  • Authorization: StripeがIDプロバイダーとしてアカウントを自動作成
  • Payment: Stripeが安全に支払いトークンを提供

⑥Discovery: サービスカタログの参照

  • stripe projects catalogでREST API経由でサービス一覧を取得
  • 結果はJSON形式で返される
  • エージェントがユーザーの要件に基づいて最適なサービスを選択
  • 人間がサービスを調べる手間をゼロに

⑦Authorization: 自動アカウント作成

  • 新規ユーザーならCloudflareが自動でアカウントをプロビジョニング
  • 既存ユーザーならOAuthフローで認証
  • StripeがIDプロバイダーとして機能
  • 面倒な登録手続きが一切不要に

⑧Payment: 安全な支払い処理

  • 支払いトークンのみを共有、生のカード情報は非共有
  • クレジットカード情報はエージェントに一切渡らない
  • 予算上限を設定可能で使いすぎを防止
  • Stripeの信頼できる支払いインフラを活用

⑨コマンド例: ドメインを1行で追加

  • stripe projects add cloudflare/registrar:domain
  • たったこの1行でドメイン取得から設定まで完了
  • エージェントがコマンドを解釈して自動実行
  • 複雑なAPI呼び出しを抽象化している

⑩Stripe Atlas特典: $100,000のクレジット

  • 新規スタートアップに$100,000のCloudflareクレジットを提供
  • Stripe Atlasユーザー限定の特別な恩恵
  • 初期費用を大幅に削減できる
  • スタートアップの参入障壁をさらに下げる

⑪MCP対応: さらに強力なエージェント連携

  • Cloudflare Code Mode MCPサーバーに対応
  • Agent Skillsで機能を拡張可能
  • MCPプロトコル経由でエージェントとシームレスに連携
  • 既存のエージェントツールチェーンに統合しやすい

⑫企業向け: ゼロフリクション連携

  • サインイン済みユーザーを持つプラットフォームならどこでも連携可能
  • 既存のユーザーベースを活かしてCloudflareサービスを提供
  • ユーザーの追加登録手続きが不要
  • プラットフォーム独自の体験を維持したまま統合できる

⑬驚きのデモ: 2分でゼロから本番デプロイ

  • わずか2分でゼロから本番環境にデプロイ完了
  • アカウント作成 → ドメイン取得 → アプリデプロイまで自動
  • デモ動画が公開されており、実際の動作を確認可能
  • まるで魔法のような自動化体験

⑭利用可能サービス: 幅広いカタログ

  • Cloudflare Directoryの全製品に対応
  • Workers、Pages、Registrar、D1など幅広く利用可能
  • 他プロバイダーのサービスも追加予定
  • カタログは今後さらに拡大していく

⑮セキュリティ: 人間の承認プロセス

  • 人間の承認は引き続き必要（TOS同意など）
  • 支払い情報はStripe経由で安全に処理される
  • エージェントが勝手に高額な契約を結ぶことはない
  • 予算制限と承認フローで安全性を確保

⑯技術基盤: 既存の標準プロトコルを組み合わせ

  • OAuth: 安全な認証フローを提供
  • OIDC: ID管理の標準規格を活用
  • 支払いトークン化: カード情報を安全に保護
  • 新しい規格ではなく、既存の信頼できる技術の組み合わせ

⑰Cloudflare x Stripeの強力なタッグ

  • Cloudflare: グローバルインフラのリーダー
  • Stripe: インターネット経済の支払いインフラのリーダー
  • 両社の強みを掛け合わせてエージェント時代の新インフラを構築
  • AIエージェントが真に自律的に動ける世界への第一歩

⑱まとめ: エージェント時代の幕開け

  • AIエージェントが「アイデア」から「本番」まで一気通貫で実行できる時代へ
  • 人間はビジネスのアイデアに集中、インフラの設定はエージェントに任せる
  • ソフトウェア開発のスピードとアクセシビリティが劇的に向上
  • 次はあなたのアイデアが、2分で世界中に届く
