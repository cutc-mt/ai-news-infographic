Gemma Chat - Google公式ローカルVibe Codingアプリ完全解説

API不要・月額無料・完全オフラインで動くGemma 4搭載デスクトップアプリ

2026年4月30日 | Source: AICodeKing

①Gemma Chatとは: 完全ローカルのVibe Codingアプリ

Gemma ChatはGoogleのGemma 4モデルを搭載したオープンソースのデスクトップアプリです。Mac上でネイティブに動作し、APIキーやクラウド接続、月額サブスクリプションが一切不要です。

  • 初期モデルダウンロード後はインターネット不要
  • 飛行機やWi-Fiのない場所でも利用可能
  • プライベートなコードを外部サーバーに送信しない
「ローカル」というのは本当にローカル。APIキーもクラウドモデルも不要で、あなたのMacだけで完結します。

②開発者はGoogle AI Studioのプロダクトリーダー

このプロジェクトはGoogle AI Studioでプロダクトデザインを牽引するAmar Rashy氏がGitHubに公開したものです。彼はGoogle内でVibe Codingとエージェントワークフローを推進している人物です。

  • Google AI Studio側のプロダクトデザイン責任者
  • オープンソースとしてGitHubで公開
  • Google公式GemmaアカウントもSNSで紹介
GoogleのGemma公式アカウントが「オフラインVibe Coding with Gemma 4」として紹介。ただのサードパーティ製ラッパーではありません。

③Apple MLXでネイティブ動作する仕組み

Gemma ChatはApple SiliconのMLXフレームワークを活用して、モデルをMac上でネイティブに実行します。Electronアプリとして動作し、ローカルでMLXサーバーを起動してモデル出力をストリーミングします。

  • ローカルMLXサーバーを自動起動
  • Electronベースのデスクトップアプリ
  • すべての処理がMac上で完結
Apple Silicon MLX Framework Electron

④Build Mode: ローカルコーディングエージェント

Build Modeはプロンプトから実際の動くアプリを生成するモードです。Gemmaがファイルを作成・編集し、右側にライブプレビューが表示されます。ターミナルを一切触る必要がありません。

  • HTML、CSS、JavaScriptのマルチファイル生成
  • リアルタイムのライブプレビュー
  • サンドボックスワークスペースで管理
  • コマンド実行もエージェントが自動で処理
ランディングページ、Pomodoroタイマー、レトロゲームなど、小規模なWebアプリの構築に最適です。

⑤Chat Mode: ツール付きローカルAIアシスタント

Chat Modeは通常のAIチャットに加えて、Web検索、URL取得、計算、Bashコマンド実行などのツールが利用できます。オンライン時はより強力なアシスタントとして機能します。

  • Web検索ツール（インターネット必要）
  • URLフェッチツール（インターネット必要）
  • 計算ツール（オフラインOK）
  • Bashコマンド実行（オフラインOK）
完全オフライン時はBuild Modeとローカルチャットがメイン。Web検索などはオンライン必須です。

⑥モデル切り替え: 4つのGemma 4バリアント

マシンの性能や用途に応じて、4つのGemma 4モデルから選択可能です。推奨はE4Bモデルで、速度と性能のバランスが最適です。

  • E2B - 最軽量、高速動作向け
  • E4B - 推奨、約3GB、速度と性能のバランス
  • MOI - より高度な推論能力
  • 31B Dense - 最強性能、RAM要件大
8GB Mac → E2Bから、16GB以上 → E4B推奨、32GB以上 → 大きなモデルにも挑戦可能。

⑦Gemma 4: Google最先端のオープンモデル

Google DeepMindがリリースした最も強力なオープンモデルファミリーがGemma 4です。高度な推論、エージェントワークフロー、コード生成、長文脈、マルチモーダル入力、ローカル展開に対応しています。

  • 高度な推論とエージェントワークフロー
  • コード生成に最適化
  • 長文脈とマルチモーダル入力
  • ローカル展開を前提とした設計
Google DeepMind Open Model Apache 2.0

⑧内部仕組み: XMLスタイルのツールプロトコル

Build ModeではXMLスタイルのシンプルなツールプロトコルを採用しています。複雑な関数呼び出しの代わりに、テキストベースのアクションでファイル操作を行います。

  • write file - ファイル作成
  • edit file - ファイル編集
  • read file - ファイル読み込み
  • list files - ファイル一覧
  • run bash - コマンド実行
小規模なローカルモデルはJSON関数呼び出しよりXMLのほうが確実に処理できるという設計判断です。

⑨ライブファイルストリーミングの仕組み

モデルがファイルを生成中、数百ミリ秒ごとに部分ファイル内容をディスクに書き込み、プレビューがリアルタイムで更新されます。モデルがまだ書いている途中からアプリの動作を確認できます。

  • 数百ミリ秒間隔で部分書き込み
  • プレビューが書き込み中も自動リロード
  • デモ感覚でアプリが目の前に構築される
Chrome DinoゲームやSnakeゲームを作らせると、ファイルがリアルタイムに出現してプレビューが生き生きと動きます。

⑩音声入力: ブラウザ内Whisperで完全ローカル

transformers.jsとWhisperを使った音声入力にも対応しています。ブラウザ上で動作するため、音声認識も完全にローカルで処理されます。話しかけるだけでプロンプトを入力できます。

  • Whisperによるローカル音声認識
  • transformers.jsでブラウザ内動作
  • 音声データも外部送信なし
Whisper transformers.js Local STT

⑬セットアップ方法: 開発者向け簡単導入

セットアップはmacOS Apple Silicon、Python、Node.js 20+が必要です。リポジトリをクローンしてnpm install後、npm run devで起動します。初回起動時にMLXのセットアップとモデルダウンロードが自動で行われます。

  • macOS Apple Silicon必須
  • Python + Node.js 20以降
  • 初回はモデルダウンロードでインターネット必要
  • DMG形式の配布パッケージも生成可能
git clone <repo>
cd gemma-chat
npm install
npm run dev

⑭制限事項: 知っておくべきポイント

Gemma Chatは魅力的ですが、いくつかの重要な制限事項があります。現実的な期待値を持つことが大切です。

  • Mac専用 - MLX依存のためWindows/Linuxは対象外
  • 初回ダウンロード - セットアップ時にインターネットが必要
  • ローカルモデルの限界 - 最強のクラウドモデルには及ばない
  • Chat Modeの一部ツール - Web検索などはオンライン必須
本格的なSaaSアプリを1発のプロンプトで構築するのは期待しないほうが良いです。小規模なプロトタイプ向けと考えましょう。
