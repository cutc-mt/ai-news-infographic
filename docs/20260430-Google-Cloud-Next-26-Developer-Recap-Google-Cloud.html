⑫ Planner Agent（経路計画）

  • GIS空間エンジニアリングスキルで26.2マイル（42.195km）のルート計算
  • Google Maps MCPサーバーで地図データにアクセス

⑬ Evaluator Agent（評価）

  • 別モデルが「エージェントを評価するエージェント」として動作

  • 評価ループで品質を反復的に向上
  • 脆弱で予測不可能なループから厳密に評価されたエキスパートネットワークへ
Casey West：「壊れやすいエージェンティック・ループから、厳密に評価された専門家のネットワークへ」

⑭ Simulator Agent（シミュレーション）

  • ルート上にランナーとランダムな行動を配置してシミュレーション
  • 都市への影響をテスト・評価
  • シミュレーションの結果を記憶して次回に最適化
  • 何度も実行することで徐々に精度が向上
「エージェントが計画・シミュレーション・思考を助けて、大きな課題を解決する素晴らしい例」
スキル & ツール

⑮ スキルの仕組み

  • スキルはYAMLメタデータとMarkdown本文の2つの部分で構成
  • メタデータは短く、スキルの名前と機能の概要のみ
  • 起動時に全スキルのメタデータを読み込み、必要な時だけ本文を取得
  • コンテキストを軽量に保ちながら、必要な情報をポイントで取得
全てのスキルを常に読み込む必要なし！必要な時だけ深い情報を取ってくる賢い設計

⑯ GIS空間エンジニアリング

  • PythonスクリプトでGeoJSONデータを処理
  • 変動ルートを計算してマラソン基準を満たすルートを生成
  • 標高・給水所・距離など多様な条件を同時に計算
  • 情報を現実世界にグラウンディングするアプローチ
モデルに推測させるのではなく、コードで正確に計算。これが信頼性の鍵！

⑰ Google Maps MCP

  • Google Cloud Remote MCPでGoogle Mapsに接続
  • 車のルート情報・距離の概算を取得可能
  • エージェントが自律的に地図データを活用
  • MCP（Model Context Protocol）がツール接続の標準に
マラソンルートの計算にはGoogle Mapsだけでは不十分！より精密なGIS計算が必要

⑱ Range Director スキル

  • Google Workspace MCPサーバー経由でGoogleドキュメントを参照
  • 人間用のドキュメントを機械可読なフォーマットに変換
  • Geminiがテキスト変換を自動実行（要約・フォーマット変換）
  • マラソンの運営情報がドキュメントに記載されていたのを活用
「人間が読むGoogle Docを、機械が理解できるスキルに変換」— モデルの得意分野！
