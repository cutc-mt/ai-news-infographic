第1章：Gemma 4とは何か

① Gemma 4の登場

2026年4月2日にGoogle DeepMindがリリースした、史上最もインテリジェントなオープンモデルファミリーだ。Gemini 3と同じ世界クラスの研究・技術から生まれ、Apache 2.0ライセンスで公開されている。Gemmaシリーズは累計4億回以上ダウンロードされ、10万以上のバリアントが開発されている。

② 4つのモデルサイズ

Gemma 4は用途に合わせて4つのサイズを提供している。

E2B
エッジ・モバイル
E4B
軽量・高速
26B
MoE / 低遅延
31B
Dense / 最高品質

③ 26B MoEモデルの秘密

26B MoEはMixture of Experts（専門家混合）を採用している。総パラメータは260億だが、推論時に実際にアクティブになるのは38億パラメータのみ。つまり、大きなモデルの品質を小さなモデルの速度で実現できる。Arena AIテキストリーダーボードでオープンモデル世界第6位にランクインしている。

26B全体をメモリに読み込む必要があるため、VRAM要件は密な26Bモデルと同等。

④ 31B Denseの実力

31B Denseはオープンモデル世界第3位の実力を誇る。自分のサイズの20倍のモデルを上回るパフォーマンスを実現している。フューチャーレベルの推論能力を、サーバーグレードの環境なしにワークステーションで実行できるのが最大の魅力だ。

⑤ 広大なコンテキストウィンドウ

エッジモデル（E2B / E4B）は128Kトークン、大型モデル（26B / 31B）は256Kトークンのコンテキストウィンドウをサポート。リポジトリ全体や長文書を一度のプロンプトで処理できる。コーディングエージェントのセッション中に文脈が切れる心配が大幅に減る。

⑥ Apache 2.0で完全オープン

開発者からのフィードバックに応え、Apache 2.0ライセンスでリリースされた。商用利用も可能で、データ・インフラ・モデルの完全なコントロールが維持できる。データ主権を重視する企業や政府機関にも適した選択肢だ。

企業や主権組織は最高水準のセキュリティと信頼性を備えたモデルを利用可能。
第2章：コーディング能力の詳細

⑦ 高品質なローカルコード生成

Gemma 4は高品質なオフラインコード生成をサポートしている。ワークステーションをローカルファーストのAIコードアシスタントに変えることができる。インターネット接続が不要で、APIキーも不要。完全にオフラインでコーディングが可能だ。

⑧ ネイティブエージェント機能

Gemma 4はネイティブの関数呼び出し、構造化されたJSON出力、ネイティブシステムプロンプトをサポートしている。これにより、ツールやAPIと対話する自律型エージェントを構築し、複雑なワークフローを確実に実行できる。

  • 関数呼び出し（Function Calling）
  • 構造化JSON出力
  • システムプロンプト対応
  • 思考モード（Thinking Mode）

⑨ 高度な推論能力

マルチステップの計画立案と深い論理的推論が可能で、数学と指示追従のベンチマークで大幅な改善を示している。シンプルなチャットを超えて、複雑な論理とエージェントワークフローを処理できるのがGemma 4の強みだ。

⑩ ビジョンとオーディオ対応

すべてのモデルが動画と画像をネイティブに処理する。可変解像度に対応し、OCRやチャートの理解などの視覚タスクに優れている。さらにE2BとE4Bはネイティブ音声入力もサポートしており、音声認識や音声理解にも対応できる。

第3章：ローカル開発環境の構築

⑪ LM Studioでモデルを準備

LM Studioはデスクトップアプリで、モデルのダウンロード・量子化・ローカルAPIサーバーの起動を一手に引き受ける。OpenAI互換のAPIをローカルで公開するため、どんなエージェントツールとも接続可能だ。

LM Studioをインストール Gemma 4 26Bをダウンロード サーバー起動（localhost:1234）

⑫ 量子化オプションとVRAM要件

Gemma 4は量子化（GGUF形式）で消費VRAMを調整できる。

Q4_K_M
18GB
品質と速度のバランス
Q8_0
28GB
ほぼオリジナル品質

Q6_K（24GB）が高品質とVRAM効率の良い妥協点としておすすめ。MacユーザーはMLX形式がApple Siliconに最適化されている。

⑬ コンテキストサイズの調整

コンテキストサイズはVRAMに直接影響する。用途に応じて適切に設定しよう。

  • 小さな編集・単一ファイル: 16K（+1GB VRAM）
  • 通常のコーディングセッション: 64K（+4GB VRAM）
  • 複数ファイルのリファクタリング: 128K（+8GB VRAM）
  • リポジトリ全体: 256K（+16GB VRAM）
128Kがおすすめ！エージェントセッションでは文脈が蓄積されやすく、途中でコンテキスト切れになるのを防げる。

⑭ Piエージェントとの接続

PiはMario Zechnerが開発したミニマルなターミナルコーディングハーネスだ。4つのツール（read, write, edit, bash）のみで構成され、トークン効率が非常に高い。

  • npm install -g @mariozechner/pi-coding-agent
  • ~/.pi/agent/models.json にLM StudioのURLを設定
  • piコマンドで起動 → /modelでローカルモデルに切り替え
/compactで古いメッセージを要約、/newで新規セッション、/forkで分岐も可能。
第4章：Android Studio連携と実用性

⑮ Android Studioでのエージェントモード

2026年4月、Android StudioがGemma 4を公式サポート。LM StudioやOllamaをローカルで動かし、Agent ModeでローカルAIコーディングアシスタントを利用できる。KotlinとJetpack Composeを使ったベストプラクティスに基づいたコードを生成してくれる。

⑯ プライバシーとコスト効率

ローカル実行の最大のメリットはプライバシーの完全保護だ。コードがマシン内に留まり、外部に送信されない。APIキーも不要で、クオータ制限を気にせず複雑なエージェントワークフローを実行できる。オフライン環境でもコーディングが可能だ。

  • コードはローカルに完結
  • インターネット不要
  • API費用ゼロ
  • 企業のセキュリティ要件にも対応

⑰ 豊富なエコシステム対応

Gemma 4はリリース初日から主要なAIツール群に対応している。

  • vLLM / SGLang - 本番サービング
  • Ollama / LM Studio - ローカル実行
  • Transformers / MLX - Pythonエコシステム
  • LiteRT-LM - モバイル・エッジ
  • Google Cloud / Docker - クラウド展開
  • NVIDIA NIM / NeMo - GPU最適化
