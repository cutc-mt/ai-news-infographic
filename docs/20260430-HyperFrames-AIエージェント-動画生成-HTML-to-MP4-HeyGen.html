HTML→動画フレームワーク

HeyGenが公開したOSS「HTML in, MP4 out」の革命／AIエージェントと動画制作の新しい形

HeyGen HyperFrames オープンソース HTML → MP4 AIエージェント Apache 2.0

① HyperFramesとは

HeyGenが公開したオープンソースのHTMLベース動画レンダリングフレームワーク。コンセプトはシンプルに「HTML in, MP4 out」。HTML/CSS/JavaScriptで書いたウェブページをフレームごとにキャプチャして、MP4動画に出力する。Apache 2.0ライセンスでGitHubから誰でも利用できる。

HTML/CSS/JS フレームキャプチャ MP4動画
GitHub: heygen-com/hyperframes

② コアアイデア：HTMLが動画の「スコア」になる

オーサリングフォーマットはReactや独自DSLではなく、プレーンなHTML＋CSS＋JavaScript。data-*属性やスキーマで「ここがレイヤー」「ここがタイミング」を意味付けする。「ブラウザでレンダリングできるもの＝動画の1フレーム」という発想が根底にある。

  • data-*属性でレイヤーとタイミングを定義
  • プレーンHTMLなので学習コストが最小限
  • ブラウザの描画結果がそのまま動画フレームに
楽譜（スコア）のように、HTMLが動画の設計図になる

③ 対応するアニメーション技術

Web標準のアニメーション技術を広くサポート。Webで表現できるものはほぼそのまま動画フレームに変換できるのが大きな強みだ。

CSSアニメーション GSAP Lottie Three.js

GreenSock Animation Platform（GSAP）を使えば、複雑なトランジションやパスアニメーションも表現可能。Three.jsなら3Dコンテンツの動画化もできる。

④ レンダリングの仕組み

Headless ChromiumをPuppeteer経由で制御し、OSの実時間とは切り離された「仮想タイムコード」でBeginFrameを進める。各フレームのピクセルバッファを取得し、メモリ上でキャプチャしてFFmpegにストリームとして渡すことでエンコードする。

Headless Chrome Puppeteer制御 ピクセルバッファ FFmpeg
仮想タイムコードで実時間と独立！

⑤ 決定論的（Deterministic）レンダリング

同じHTML・CSS・JS・シードを入力すると、バイトレベルで同一の動画が出力される。ハードウェアや負荷状態に依存しないのが最大の特徴だ。

入力が同じなら
出力も同じ
ハードウェア
非依存
CI・自動テスト
バッチに最適
再現性が担保されるから、自動テストや大量レンダリングに最適

⑥ なぜ「エージェント向き」なのか

LLMはWebの学習を通じてHTML/CSSに非常に慣れている。「複雑な専用動画DSLを覚えさせる」より、得意なHTMLの方が安定してコード生成できる。Claude CodeやCursor、Gemini CLI向けの「スキル」としても提供されている。

❌ 専用DSLを教える
学習コスト高・生成エラー多
✅ HTMLを使う
既に得意・安定した生成
LLMの「得意分野」に合わせるのが正解！

エージェント環境にスキルをインストールするだけで、HyperFrames用のHTMLパターンを理解して動画生成が可能になる。

# スキルをインストール

# エージェントがHTMLパターンを理解
# /hyperframes コマンドで動画生成を指示

インストール後は/hyperframesコマンドで動画生成を指示できる。エージェントがHTMLファイルを生成し、自動的にレンダリングまで行う流れだ。

ワンコマンドでAIエージェントが動画クリエイターに！

⑨ 拡散モデル系（Runway, Pika）との違い

RunwayやPikaなどの拡散モデルは、プロンプトからピクセルを直接生成する確率的アプローチ。HyperFramesはレンダリングエンジンで、入力が同じならピクセルも毎回同じ決定論的パイプラインだ。

拡散モデル
確率的・再現性が低い
クリエイティブ用途に向く
HyperFrames
決定論的・再現性100%
自動生成・バッチに最適
用途に応じて使い分けが重要

⑩ React/Remotion系との違い

React系は高機能だが、JSXや状態管理などLLMの生成エラーが出やすい構造が増える。HyperFramesはプレーンHTMLのタイムライン制御に特化しており、エージェントが60行程度の単一HTMLファイルを書くだけで完結する。

React/Remotion
高機能・JSX・状態管理
LLMの生成エラー多
HyperFrames
プレーンHTML専用
60行で完結・エラー少
シンプルな方がAIには書きやすい！

After Effectsは手作業でタイムラインを編集してレンダリングする。HyperFramesは最初からAPI/CLI前提で設計されている。

After Effects
手作業・GUI操作
1本ずつ丁寧に
HyperFrames
API/CLI・自動化

CI/CDやエージェントオーケストレーションから叩きやすく、高頻度の自動レンダリングや差分更新に向いている。

自動化パイプラインに組み込みやすいのがポイント

⑫ ローカルCLIでの基本フロー

CLI一連のコマンドでプロジェクトの作成から動画出力まで完結する。

# プロジェクト作成
npx hyperframes init my-video

# ブラウザでライブプレビュー（ビルド不要）
npx hyperframes preview

# compositions/配下のHTMLを編集 → 即反映

# 動画を出力
npx hyperframes render \
  --composition product-promo \
  --output output.mp4
previewでリアルタイム確認しながら開発できる！

⑬ HTMLコンポジションの例

データ属性で動画の構造を定義する。プレーンHTMLなのでエージェントが生成しやすいのがポイントだ。

<div data-composition-id="product-promo">
  <div data-clip="title">
    プロダクト名
  </div>
  <div data-clip="description">
    説明テキスト
  </div>
</div>
  • data-composition-id で動画をマーク
  • data-clip でクリップ要素を指定
  • CSSやGSAPでアニメーションを付与
シンプルな属性定義だけで動画の構造が完成

WebページURL スクロール＆ズーム 紹介動画

⑱ プロンプトのコツ

より良い動画を生成するためのポイントを押さえよう。

  • 尺を必ず指定（例：10秒、30秒、5シーンで各3秒）
  • アスペクト比を明示（16:9 / 9:16 / 1:1）
  • 入力コンテキストを一緒に渡す（URL、PDF、CSV等のWarm start）
  • 1本目を作った後は小さく追加指示していく
良いプロンプトの例
「16:9で10秒。5シーンで各2秒。ブランドカラー#E91E63を使用。弊社プロダクトのLPから主要セクションをスクロール紹介して」
具体的な指定ほど、AIの生成精度が上がる！
