2026年4月30日 | GitHub: microsoft/BitNet

① BitNet b1.58とは: 重みをたった3値で表現するLLM

  • BitNet b1.58は、Microsoft Researchが開発した1-bit量子化LLMです。
  • モデルの重みを{-1, 0, 1}の3値だけで表現します。
  • 情報量は1.58 bit（log₂3 ≒ 1.58）で、名前の由来になっています。
  • 従来の16bit浮動小数点と比べて劇的なメモリ削減と高速化を実現します。
-1, 0, 1だけなのに精度が落ちない。
シンプル イズ ベスト！✨

② なぜ1-bitなのか: 従来LLMのボトルネックを打破する

  • 従来のLLMはFP16/BF16で重みを保持するため、メモリと計算量が膨大です。
  • 大規模モデルの推論にはGPUが必須で、コストも電力もかさみます。
  • 1-bit化により、乗算が加減算に置き換え可能になり、CPUでも高速推論が可能に。
  • マトリックス乗算をルックアップテーブルで代用するのが核心技術です。

③ ARM CPUパフォーマンス: モバイル・エッジでも爆速

  • ARM CPU環境での推論速度が1.37倍〜5.07倍に高速化されます。
  • 消費電力は55.4%〜70.0%を削減でき、バッテリー寿命が大幅に延びます。
  • スマートフォンやタブレットなど、エッジデバイスでのLLM実行が現実的に。
  • 発熱も大幅に抑えられ、ファンレスデバイスでも安定動作が期待できます。

④ x86 CPUパフォーマンス: サーバーでも劇的な改善

  • x86 CPU環境では2.37倍〜6.17倍の高速化を実現しています。
  • 消費電力は71.9%〜82.2%を削減し、運用コストを大幅に下げます。
  • GPUがなくても、一般的なCPU搭載サーバーで大規模LLMが動きます。
  • GPUサーバーのレンタルコストが不要になり、TCOが劇的に低下します。

⑥ ロスレス量子化: 品質を犠牲にしない圧縮技術

  • 1.58bitへの量子化を行っても、モデルの品質低下がほぼなしです。
  • 従来のINT8やINT4量子化とは異なり、完全にロスレスな性能を維持します。
  • これはBitNetが最初から1-bit用に設計・学習されているからです。
  • 事後量子化（Post-training Quantization）ではなく、学習時から3値化するアプローチが鍵です。

⑦ Lookup Table手法: 乗算をテーブル参照で置き換え

  • T-MAC論文で提案されたLookup Table（LUT）手法を採用しています。
  • 重みが3値 {-1, 0, 1} なので、行列の乗算をテーブル参照に変換できます。
  • 乗算演算が不要になることで、CPUの計算ユニットを効率的に活用できます。
  • llama.cppベースの実装で、既存のエコシステムとも互換性があります。
/* 通常の行列積: O(n²) の乗算 */
C[i][j] += A[i][k] * B[k][j]

/* BitNetのLUT: テーブル参照で置き換え */
C[i][j] += LUT[A[i][k]][B[k][j]]

⑧ 並列カーネル最適化: さらに1.15x〜2.1xの追加高速化

  • 最新の並列カーネル実装により、さらに1.15倍〜2.1倍の速度向上を達成。
  • マルチスレッド最適化とSIMD命令の活用で、CPUの全コアを効率的に使用。
  • 計算を細かく分割して並列実行することで、レイテンシを大幅に削減します。
  • ベースの高速化（最大6.17x）と組み合わせると、最大約13倍の総合高速化が可能に。
