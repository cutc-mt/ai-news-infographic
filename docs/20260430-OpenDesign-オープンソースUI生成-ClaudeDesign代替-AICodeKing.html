Open Design — オープンソースのローカルファーストUI生成ツール

Claude Designの代替となる、既存のコーディングエージェントを活用するデザインツール
2026年4月30日 | AICodeKing

① Open Designとは: オープンソースのUIデザイン生成ツール

  • Open Designはローカルファーストのオープンソースデザインアプリです。
  • 既存のコーディングエージェントに接続して、UIアーティファクトを生成します。
  • Apache 2.0ライセンスで公開されており、自由に利用・改造が可能です。
  • 独自のAIエージェントを同梱せず、既存のツールをデザインエンジンとして活用するのが特徴です。
自分のエージェントを使い回せるから、
別途デザイン用のサブスクリプションが不要！

② Claude Designの課題: クラウド依存の壁

  • Claude Designはクローズドで、有料、クラウドベースのサービスです。
  • Anthropicのエコシステムに完全に依存しており、他の環境では利用できません。
  • セルフホストやCodex、Cursor Agent、Gemini CLIとの連携が不可能です。
  • 自分のローカル環境でデザインワークフローを構築したい場合は、選択肢が限られていました。

③ 対応エージェント: 6つの主要コーディングツール

  • Claude Code — 最もストリーミング対応が優秀（構造化JSON出力）
  • Codex CLI — OpenAIのセットアップをそのまま利用可能
  • Cursor Agent — IDE統合型エージェントとして動作
  • Gemini CLI — GoogleのGemini設定に依存
  • OpenCode / Qwen Code にも対応
  • CLIが見つからない場合は、Anthropic APIのBYOK（Bring Your Own Key）フォールバックも利用可能です。

④ コスト構造: 既存エージェントの費用のみ

  • Open Design自体は無料でオープンソース（Apache 2.0）。
  • コストは選択したモデルやエージェントに依存します。
  • Claude Code → Claude Codeの料金、Codex CLI → OpenAIの料金という仕組み。
  • 学生、インディーハッカー、既にAIコーディングツールを契約している人に最適です。

⑦ DESIGN.md: デザイン指示をマークダウンで定義

  • DESIGN.mdはプレーンなマークダウンで書かれたデザインシステムファイルです。
  • 色、タイポグラフィ、スペーシング、レイアウト、コンポーネント、モーション、ブランドルールを定義。
  • アンチパターンも定義でき、よくあるAI UIの失敗を事前に防止します。
  • 「きれいにして」ではなく、LinearやStripeのような具体的なスタイルを指定できるのが大きな違いです。
# DESIGN.md（例）

## Colors
- Primary: #0984E3
- Accent: #00B894

## Typography
- Heading: Inter Bold
- Body: Inter Regular

## Anti-patterns
- ❌ 紫のグラデーション
- ❌ 汎用絵文字アイコン
- ❌ 偽のメトリクス

⑧ 5つのビジュアル方向: デザインの基盤を決定

  • Editorial Monochrome — 編集的でモノクロームな洗練されたスタイル

⑫ アーキテクチャ構成: シンプルで堅牢な設計

  • フロントエンド: Vite + React + TypeScript のモダン構成
  • バックエンド: Node + Expressのデーモン
  • データ: SQLiteでプロジェクト、会話、メッセージ、タブを管理
  • デーモンが権限プロセスとして動作し、CLIをプロジェクトフォルダ内で起動
  • エージェントは実際の作業ディレクトリでファイルの読み書きを実行します。

⑭ サンドボックスプレビュー: 安全な表示環境

  • 生成されたHTMLアーティファクトはサンドボックス化されたiframeで表示されます。
  • プレビューが不必要なアクセス権を持たないように制限されています。
  • レンダリング結果をリアルタイムで確認しながら、安全性を確保。
  • 生成物の確認とエクスポートを安全に実行できるのが重要な設計ポイントです。

⑮ アンチスロップ: AI UIの典型的な失敗を排除

  • 攻撃的な紫のグラデーションを禁止
  • 汎用的な絵文字アイコンの乱用を禁止
  • 左ボーダーアクセント付きの丸角カードの乱用を禁止
  • 偽のメトリクスや過剰な装飾要素を禁止
  • これらはAI生成UIが「安っぽく」見える典型的なパターンです。

⑱ 注意点と今後の展望: 現実的な期待値

  • まだ初期段階のプロジェクト。本格的なチーム導入には時間が必要。
  • 出力品質は基盤エージェントに大きく依存。Claude Codeが最もストリーミング対応が優秀。
  • 弱いモデルではデザインシステムがあっても良い判断にはならない。
  • コメントモード、サージカル編集、AIツイークパネルは今後の実装予定。
  • プロトタイプ、ランディング、ダッシュボード、デッキには最適。本番UIは手動でレビュー・調整を推奨。
