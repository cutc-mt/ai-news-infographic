AI Video Generation

URLを入れるだけで完全な動画が自動生成される — Archon + Claude Code + Remotion
Cole Medin
Cole Medinが1.5時間のライブストリームで、Archonの14ステップ・ワークフローを初公開。Claude Codeがプランニングからビルド、QAまで全工程を自律実行し、ElevenLabsが音声を担当、RemotionがReactで動画をレンダリングする。URLひとつで数分の動画が約15分で完成するエンドツーエンドのパイプラインを徹底解説。

入力はURLだけ

このワークフローが必要とする入力はたった1つのURLだけ。Hacker Newsの記事、ブログ記事、製品ページ、GitHubリポジトリのいずれでも動作する。
URLを投げるだけで、ソースの取得、シーンの計画、ナレーション生成、音楽と効果音の作成、TypeScriptコードの記述、品質チェック、自動修正、最終的なMP4レンダリングまですべて自動で完了する。手動での作業はゼロ。
ライブデモではHacker Newsのトップ記事（TI-84 Evo電卓）を使って、一発パスで動画を生成。イテレーションなしでもそれなりの品質が確認された。

14ステップのパイプライン

Archon上で動く14ステップの自律ワークフロー。すべてが単一のYAMLファイルに定義されている。
パイプラインの流れ：
① プリフライトチェック → ② ソース取得 → ③ スラッグ導出 → ④ Remotionセットアップ → ⑤ 動画プラン作成 → ⑥ ナレーション音声生成 → ⑦ 効果音生成 → ⑧ 音楽生成 → ⑨ コンポジション構築 → ⑩ レジストリ再生成 → ⑪ QAレビュー → ⑫ イテレーション（必要時） → ⑬ 最終検証 → ⑭ レンダリング
各ステップは独立したノードとして定義され、ノード間でコンテキストがシームレスに受け渡される。

Archonとは？

オープンソースのハーネスビルダー。AIコーディングプロセスをYAMLファイルで定義し、確実かつ再現可能なものにする。「Dockerfile for AI coding」という表現がぴったり。
GitHub starsは20,500以上。ループ、コンテキスト管理、プロバイダーの混合（Claudeで実装、Codexで検証など）をサポート。Human-in-the-loopにも対応し、interactive: trueでプラン承認やコード検証を挟める。
大きなワークフローをスキルだけで実行すると信頼性が下がるが、Archonなら複雑なプロセスも確実に実行できる。決定的なステップはスクリプトで、エージェントが必要な箇所だけAIに任せるハイブリッド構成が可能。

Claude Codeが主役

プランニング、ビルド、QAの全工程でClaude Codeが活躍する。Archonは裏でClaude Code（またはCodexなど他のコーディングエージェント）をディスパッチする。
シーン構成の設計からTypeScriptコードの記述、品質チェックまで自律的に実行。1つのセッションで全工程を行うとコンテキストが溢れるため、Archonが複数のセッションを連結して1つのワークフローとして実行する。
実装とレビューを同じセッションで行わないのが重要。「子供に自分の宿題を採点させる」のと同じだから、レビューは別セッションで実行する設計になっている。

Remotionで動画生成

Reactベースの動画フレームワーク。TypeScriptでコンポジションを記述し、プログラマティックに動画を生成する。
リアルなトランジション、動的デュレーション、適切なReactフックを実装。単なるスライドショーではなく、プロフェッショナルなアニメーション付きの動画が生成される。Remotionが自動的にセットアップされるため、事前のインストールも不要。
ColeはHyperframes（Remotionの代替ツール）への乗り換えにも関心を示しており、スキルを差し替えるだけで移行可能。Google Trendsで比較するとRemotionの方が依然として人気が高い。

39ルールのベストプラクティス

Remotion用の39のベストプラクティスルールをスキルとしてビルドノードに注入。Remotionの公式スキルをベースにカスタムビルドしたもの。
AIが生成する動画が「AIっぽく」ならないよう、リアルな演出と品質を保証。トランジションの使い方、動的デュレーションの設定、フックの正しい実装方法など、実践的なルールが含まれている。
さらにダイアグラム生成スキルも別途注入され、図解が必要なスライドも自動で作成される。YAMLファイル内に51箇所以上「Remotion」が言及されており、プロンプトが非常に詳細。

ElevenLabsで音声生成

ナレーションはElevenLabsまたはCartesiaが担当。ElevenLabsの音声クローン機能を使えば、自分の声をベースにナレーションを生成できる。
Coleがライブで調整したパラメータ：Speed 1.0、Stability 0.5、Similarity 0.8、Style Exaggeration 0.25。v2モデルが推奨（v3は品質が落ちる）。声のイントネーションや話し方まで忠実に再現された。
Cartesiaもサポートされており、.envファイルのAPIキーを切り替えるだけで利用可能。パラメータはもともとハードコードされていたが、Coleがライブで環境変数化するリファクタリングを実演した。

音楽と効果音の生成

ElevenLabs APIでBGMと効果音（SFX）も自動生成。ナレーション音声の生成と並行して音楽も処理される。
パラレル実行で効率化：音声生成と効果音生成は並列で実行され、その後に音楽生成が続く。音声が完了してから音楽を生成する依存関係をArchonが自動で管理。
すべての音声アセットはアーティファクトディレクトリに出力され、後続ノードから即座にアクセスできる。generate_voiceover.pyスクリプトがアーティファクトディレクトリから台本を読み込み、APIを呼び出して音声を出力する。

スキル注入の仕組み

各ノードに必要なスキルを注入する仕組みがArchonの柔軟性の鍵。ビルドノードにはRemotionベストプラクティススキルとダイアグラム生成スキルが注入される。
ツールの乗り換えも容易：RemotionからHyperframesへの移行は、スキルを差し替えるだけ。ワークフロー自体の変更は最小限。プランニングや音声同期は同じ仕組みをそのまま使える。
Archon自体に直接統合をビルドする必要はなく、スキルやMCPサーバーとして注入するだけで拡張できる。プリフライトチェックでスキルの存在確認も自動で行われる。

アーティファクトディレクトリ

ノード間のコンテキスト渡しを支える仕組み。Archonのワークフロー実行ごとに専用のディレクトリが作成される。
2つの方法でノード間の情報共有：① アーティファクトディレクトリにマークダウン文書を配置して共有。② node_id.outputで前段ノードの出力に直接アクセス。後続ノードはすべて同じディレクトリを参照する。
記事本文、生成された音声、動画プラン、効果音ファイルなど、すべてのアセットが1箇所に集約される。前段のノードがリソースをフェッチしてarticle_bodyとして保存すれば、以降のノードは全て同じデータにアクセス可能。

品質チェックと自動修正

ワークフローの末尾に自動QAパスが組み込まれている。音声と映像の同期チェック、アニメーション品質確認などを自動で実行。
QAレビューは構造化出力で判定結果を返す。Passならそのままレンダリングへ、Needs Iterationなら新しいセッションでフィードバックを渡して自己修正を実行。実装セッションとは別のコンテキストで行われるため、客観的な品質評価が可能。
この検証ノードの追加はArchonの大きな強み。コード生成でも動画生成でも、最後にバリデーションを簡単に組み込める。最後にサマリーを出力して完了。

ライブ実演

ColeがArchon自身のマーケティング動画をライブで生成。マーケティングモードを選択し、GitHubリポジトリのREADMEから製品情報を抽出して動画を作成。
約15分で数分の動画が完成。マーケティングモードは「問題提起 → 機能紹介 → コールトゥアクション」の流れで動画を構成。ArchonのGitHubリポジトリURLを投げるだけで、READMEから製品の要点を抽出してピッチ動画を生成する。
Web UIでノードの進行状況をリアルタイムに確認でき、各ノードのログとClaude Codeの出力を直接閲覧可能。途中でWindowsがフリーズするハプニングもあったが、リトライで無事に完了。

イテレーションの実践

90%から100%への最後の壁。YAMLの発音問題、効果音の調整、速度の最適化など、細かなイテレーションが必要な課題をライブで実演。
問題をClaude Codeに渡すと自律的に解決。ElevenLabsのパラメータ調整（Speed、Stability、Similarity、Style Exaggeration）を.envファイル経由で設定し、音声品質を改善。v2モデルへの切り替えもライブで実施。
ワークフロー自体の改善もその場で行い、パラメータをハードコードから環境変数に変更するリファクタリングを実演。ColeのチームメンバーThomasはすでにHyperframesに移行しており、最新の動画はHyperframesで生成されている。

Dark Factory

完全自律進化するコードベースのコンセプト。Archonワークフローを使って、GitHub Issuesから実装、レビュー、テストまで全自動で回す。
フロー：GitHub Issuesで機能要求 → Archonワークフローがトリアージ → 実装 → PRレビュー → 週次リグレッションテスト。Claude Code（Kimik 2.6）で運用中。
この動画生成ワークフロー自体も、将来的にDark Factoryの一部として自己改善していく構想。ColeのチームメンバーThomasとRasmusが共同で開発を進めており、リポジトリはGitHubで公開中。

並列実行と独立性

Archonはworktreeとisolationをサポートしており、複数のワークフローを並列で実行できる。
トークン効率も良い：エージェントにすべてのステップを任せるのではなく、決定的なステップはスクリプトで実行し、必要な箇所だけエージェントを使う。ステップごとにモデルとプロバイダーを指定でき、例えばClaudeで実装してCodexで検証という組み合わせも可能。
各ノードは独立したセッションで実行されるため、コンテキストの溢れを防げる。大きなワークフローでも信頼性を保てるのがArchonの大きな利点。

柔軟な統合

Jira、Linear、Confluence、Git Nexusなどの外部ツールはスキルやMCPサーバーで統合。Archon自体に直接統合をビルドする必要はない。
ユーザーからのよくある質問「ArchonはJiraやConfluenceをサポートしていますか？」に対する答えは常に「スキルを作ればOK」。必要なノードにスキルを注入するだけで、任意の外部サービスと連携できる。
Slack、GitHub、Web UIなど、複数のアダプターを標準サポート。ローカル実行もDockerを使ったVPSでの本番運用にも対応。Cole自身はローカルでの利用を推奨している。
