AI動画生成ワークフロー: Claude Code + Remotion + Archon

2026年5月3日 | Cole Medin

①Archonとは: オープンソースのハーネスビルダー

  • ArchonはAIコーディングプロセスをYAMLファイルとして定義できる、世界初のオープンソースハーネスビルダーです。
  • エージェントを使ったあらゆるプロセスを、ノード単位のワークフローとして信頼性高く再現可能にします。
  • ループ、コンテキスト管理、モデル・プロバイダーのノードごとの指定、ヒューマンインザループなど豊富な機能をサポート。
  • 確定的なステップとエージェントのステップを組み合わせて、トークン効率の良いワークフローを構築できます。
エージェントにワークフローを組み込むのではなく、ワークフローにエージェントを組み込むという逆転の発想！

②AI動画生成の全体アーキテクチャ

  • 14ステップのワークフローが、URLひとつから完全な動画を生成します。
  • ソース取得 → シーン計画 → ナレーション生成 → 音楽・効果音 → TypeScriptコンポジション → 品質レビュー → 自動修正 → MP4レンダリング
  • Archonがオーケストレーション、Claudeが実装とQA、ElevenLabsとCartesiaが音声を担当。
  • 数分の動画の生成に約15分かかりますが、すべて完全自動で動作します。
記事、製品ページ、Hacker Newsストーリーなど、どんなURLからでも動画を作れる！

③Remotion: Reactベースの動画生成

  • RemotionはReactを使ってプログラマブルに動画を作成できるオープンソースツールです。
  • ワークフローには39のベストプラクティスルールがスキルとして注入され、本物のトランジションや動的なデュレーションを実現します。
  • AI特有の「安っぽい」見た目を避けるため、適切なフックとリアルなアニメーションを使用。
  • 将来的にはHyperframesへの乗り換えも検討中。スキルの差し替えだけで対応可能です。
Remotionのスキルを差し替えるだけで、別の動画生成ツールに切替えられるのがArchonの柔軟性！

④ElevenLabsによる高品質な音声生成

  • ElevenLabsまたはCartesiaのAPIを使って、ナレーション音声を生成します。
  • 音声パラメータ（速度・安定性・類似度・スタイル誇張）を環境変数で調整可能。
  • コール氏自身の声のクローンも作成。V2モデルで声の抑揚まで再現できました。
  • ナレーション、音楽、効果音は並列で生成され、効率的に処理されます。
V3モデルよりV2の方が自然な音声に近いという結果も！パラメータ調整が重要です。

⑤YAMLワークフローの構造

  • ワークフロー全体が単一のYAMLファイルで定義されており、資産としてコードベースに保存できます。
  • プリフライトチェック → ソース取得 → スラッグ導出 → Remotionセットアップ → プラン作成 → 音声生成 → コンポジション構築
  • 各ノード間のデータ受け渡しは、アーティファクトディレクトリとノード出力の2つの方法で実現。
  • プロバイダーの混合が可能：実装はClaude、検証はCodexという使い分けもできます。
# ワークフローの例
nodes:
  - id: fetch_source
    type: agent
    model: claude
  - id: plan_video
    type: agent
    model: claude
  - id: generate_audio
    type: script
    command: python generate_voiceover.py

⑥スキルの注入で柔軟に拡張

  • Archonはスキル（Skills）を通じてコーディングエージェントに能力を注入します。
  • Remotionのベストプラクティス、図表生成、Archon自体の操作などがスキルとして用意されています。
  • Jira、Linear、Confluence、Google Driveなどの外部ツール連携も、スキルやMCPサーバーの注入で対応。Archon本体の変更は不要です。
  • RemotionからHyperframesへの乗り換えも、スキルを差し替えるだけで基本的には完了します。
「Archonに〇〇の統合をビルドして」と聞く必要はない。スキルを作って注入するだけ！

⑦ソース取得と動画の計画

  • GitHubのREADME、Hacker Newsの記事、製品ページなどから自動でコンテンツを取得します。
  • 取得した情報をもとに、マークダウンドキュメントで動画の計画を作成。シーン構成、テキスト、表示内容を設計します。
  • マーケティングモードでは、問題提起 → 機能紹介 → CTA（コールトゥアクション）の構成で動画を構築。
  • Hacker Newsのトップ記事をURLとして渡すだけで、テストから本番まで簡単に実行できます。
デモではTI-84 EVO計算機のHacker News記事から動画を生成。ランダムなトピックでもしっかり動作！

⑧コンポジション構築ステップ

  • 計画に基づいて、RemotionでTypeScriptのコンポジションをゼロから構築します。
  • すべてのワード、スライド、アニメーション、アイコンが完全に自動生成されます。
  • 音声と映像の同期、シーン間のトランジション、動的なデュレーション設定も自動で処理。
  • このステップが最も時間がかかる部分。複数シーンのアセット生成を順次実行します。
「AIスロップ」にならないよう、39のベストプラクティスルールで品質を担保！

⑨品質レビューと自動反復

  • QAレビューノードが、音声と映像の同期、アニメーション品質などを自動チェックします。
  • レビュー結果はPass / Needs Iterationの判定として構造化出力されます。
  • 「要反復」の場合は、新しいコンテキストウィンドウでフィードバックを渡して修正を実行。自己修正の仕組みです。
  • 実装とレビューを別々のエージェントセッションで実行することで、客観的な品質評価を実現。
自分の宿題を自分で採点するようなものだから、別セッションでのレビューが重要！

⑩ノード間のデータ受け渡し

  • アーティファクトディレクトリ: ワークフロー実行ごとに固有の作業ディレクトリが作成され、マークダウンや音声ファイルを共有。
  • ノード出力参照: derive_slug.outputのように、前段ノードの最終出力に直接アクセス可能。
  • 確定的ノードは標準出力を、非確定的ノードはエージェントの最終メッセージを出力として取得。
  • コンテキストの受け渡しが簡単なので、複雑なマルチステップワークフローも構築できます。
ノード間でデータを渡せないと、ワークフローの構築は事実上不可能。これがArchonの鍵！

⑪確定的ステップの活用

  • エージェントに覚えておいてほしくない処理は、確定的なスクリプトとして実行します。
  • スラッグの生成、コンポジションレジストリの更新などはBashスクリプトで確実に実行。
  • 実装後のバリデーションコマンドなど、必ず実行したい処理をエージェントに忘れさせるリスクを排除。
  • すべてをエージェントに任せないことで、トークン消費を大幅に削減できます。
エージェントに全部覚えさせるより、ルールはコードで、創造性はエージェントに。この分担が勝利の鍵！

⑫コーディングエージェントを超えた活用

  • Archonはコーディング以外のワークフローにも使えることが、この動画生成で証明されました。
  • Claude CodeやCodexなどのコーディングエージェントは、第二の脳としても活用可能。
  • GitHub Issueの修正、PRのレビュー、バグ修正など、従来のコーディング用途はもちろん対応。
  • ワークツリーによる並列実行もサポート。複数のワークフローを同時に実行できます。
「コーディングエージェントはコーディングだけ」の常識が覆る！エージェント＝汎用ワーカー。

⑬セットアップの簡単さ

  • 5分でセットアップ完了。必要なのはElevenLabsまたはCartesiaのAPIキーだけ。
  • Remotionはワークフローが自動でインストールしてくれるので、事前準備不要。
  • Archonのスキルディレクトリにワークフロー用スキルが含まれており、Claude Codeがすぐに利用可能。
  • archon workflow run remotion-from-hacker-newsをタイプするだけで動画生成が始まります。
# セットアップ手順
1. ElevenLabsまたはCartesiaのAPIキーを設定
2. Archonをインストール
3. コーディングエージェント（Claude Code等）を準備
4. ワークフローを実行！

⑭Archon Web UIでの可視化

  • ローカルホストでArchon Web UIを起動して、ワークフローの進行状況をリアルタイムで確認可能。
  • 各ノードのステータス、Claude Codeの出力ログ、アーティファクトの内容を閲覧できます。
  • 本番利用にはDockerを使ったVPSデプロイもサポート。SlackやGitHubアダプターも利用可能。
  • 推奨はローカル実行。開発環境と同じ場所でArchonを動かすのが最も便利です。
Web UIでノードごとのログを見られるから、どこで止まっているか一目で分かる！

⑮マルチセッションアーキテクチャの利点

  • Archonは複数のコーディングエージェントセッションを一つのワークフロー実行に紐付けます。
  • 単一のセッションで全工程を処理すると、コンテキストが溢れて品質が低下。
  • 実装セッションとレビューセッションを分離することで、客観的な品質評価が可能に。
  • ディスパッチするだけで、エンドツーエンドの完全自動化が実現します。
大きなワークフローを一つのスキルに詰め込むと信頼性が下がる。セッション分割こそが正解！

⑯今後の展望: Hyperframesへの移行

  • HyperframesはRemotionの代替として注目されている、Claude統合の動画生成ツール。
  • RemotionとHyperframesのヘッドツーヘッド比較も今後のコンテンツとして計画中。
  • 乗り換えは簡単。動画の計画方法や音声同期は同じで、注入するスキルを変更するだけ。
  • Google Trendsでは現在Remotionの方が人気だが、Hyperframesの台頭にも注目。
ワークフロー自体はほとんど変更なしでツールを乗り換えられる。これがArchonの真価！

⑰デモ実績: TI-84 EVO計算機

  • Hacker Newsのトップに掲載されたTI-84 EVO計算機の記事から動画を生成するデモを実施。
  • 3倍速いプロセッサ、50%大きなディスプレイ、USB-C、新しいインターフェースの内容を自動で動画化。
  • 初回パス（反復なし）でもそこそこの品質の動画が完成。フィードバックを与えればさらに改善可能。
  • 音声に若干の不自然さはあるものの、全体としてAI生成とは思えないクオリティ。
ランダムなトピックでも動作するのがポイント。実用性の高さを証明！
