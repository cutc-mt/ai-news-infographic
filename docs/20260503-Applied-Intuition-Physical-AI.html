Applied Intuitionとは？

2017年YC出身。フィジカルAIのためのテクノロジープラットフォーム。NVIDIAやAMDがチップを作るように、彼らは車・トラック・建機・農業機械・防衛システムに「頭脳」を提供する。
現在1,000人のエンジニア、全世界非中国トップ20自動車メーカーの18社が顧客、評価額$15B。

フィジカルAI vs 画面AI

LLMはチャットやコーディングで間違えても許されるが、無人トラックや建機のミスは許されない。
現在日本でL4無人トラックを実際に運行中。エラー＝人の命に直結する安全クリティカル環境でのAI展開が本命。

車載OS：Android以前のスマホ

現在の車載ソフトウェアスタックはAndroid以前のスマホと同じ状態 — OSが分散していて統一されていない。
車をbrick化することはiPadを壊すより遥かに深刻。リアルタイム制御、センサーストリーミング、ミリ秒レベルのレイテンシ、フェイルセーフ — 求められる信頼性は桁違い。

シミュレーション：仮想↔現実の相関

複雑なソフトウェアを動くマシンに載せるにはテストが不可欠。シミュレーション＋実世界テストの組み合わせが鍵。
どんなシミュレーターも現実世界を完全には再現できない。Sim-to-Realの検証プロセスが必須。実世界テストは永遠に消えない。

Neural Sim：次世代シミュレーション

Applied Intuitionが発表したNeural Simは、カメラ・LiDARなどセンサーレベルでシミュレーションを実行。
クローズドループで被験システムが自車の動作を変更できる。RLを実際に使えるレベルに高速かつ安価にするのが目標。

社内AIツール：Cursor × Claude Code

社内ではCursor、Claude Codeが全面的に導入。内部採用リーダーボードまで存在。
組込みシステムや安全クリティカルソフトウェアの開発でさえ、AIツールがエンジニアリングワークフローを劇的に変えている。

オンボード vs オフボードAI

オフボード（データセンター）
巨大・低速OK
制約が少ない
オンボード（車載）
ミリ秒レベル
低消費電力・小型必須
車載モデルには蒸留のような効率化が必要。モデルの知能だけがボトルネックではない — リアルハードウェアへの展開が本当の壁。

統計的検証へのシフト

自律走行の検証は「通過/不合格」の二項テストから、「何個の9」の信頼性かという統計的アプローチへ。
MTBF（平均故障間隔）がKPIに。Cruiseの教訓、Waymoが業界の信頼基準を高めている。規制当局との対話が極めて重要。

世界モデル：物理AIのコア

ハイドロプレーニング 建設機械 視覚的手がかり 因果関係学習
世界モデルは物理的因果関係の理解に役立つが、それだけでは不十分。ハードウェア制約の壁を乗り越える実践的アプローチが不可欠。

レガシー自律走行 vs 次世代

テクノロジースタック：2年ごとの進化

Applied Intuition自身の技術スタックは約2年ごとに完全に入れ替わる。過去10年で4回の完全進化。
研究発表に応じて適応し続ける。30以上のプロダクトを展開し、シミュレーション・OS・AIモデルのポートフォリオを拡大。

デモ ≠ プロダクション

ロボティクスデモを見ると、次に直面する20の問題が予測できる — 10年の経験から。脆い最後の1%が研究と実装の壁。
ヒューマノイドの信頼性、DARPA Grand Challenge的な課題。「動いた」だけではビジネスにならない。

Plan Mode for Physical Systems

「プランモード」がロボタクシー・鉱業・防衛に適用可能。アクションが世界の状態を変える多次ステップ物理タスク。
次トークン予測の普遍性：テキストだけでなく物理世界の次の状態も予測する。これが自律走行の基盤。
