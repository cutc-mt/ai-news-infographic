CLAUDE CODE LEAK → OPENCLAUDE

ClaudeCodeソース流出から爆誕 OpenClaude

海外で人気爆発の全貌を徹底解説
Jun Suzuki
2026年3月31日、AnthropicのClaude Codeの全ソースコードがnpmレジストリから流出。その数日後にコミュニティがOpenClaudeを爆誕させ、全世界の開発者が熱狂。流出の経緯からOpenClaudeの全貌まで完全解説。

ソースコード流出の発端

2026年3月30日、Anthropicがnpmに公開したClaude Code v2.1.88に.sourcemapファイルが同梱されていた。
セキュリティ研究者Chaofan Shouが3月31日08:23 UTCに発見。投稿は2800万ビューを記録。

Sourcemapとは？

JS/TSバンドラが生成するデバッグ用ファイル。minifiedコードを元のソースコードに復元できる。
開発では必須だが、npmパッケージに同梱すると完全なソースコードの復元が可能に。今回は「人為的ミス」で同梱されていた。

流出規模

TypeScriptファイル数 約2,000
総コード行数 512,000行+
GitHubスター数（ミラー） 84,000+
フォーク数 82,000+

Claude Codeの内部アーキテクチャ

単なるCLIラッパーではなく、本格的なマルチエージェント本番システムだった。
6層構造：UI層（ink/React風）→ コマンド層 → エージェント・オーケストレーション層 → ツール層 → サービス層 → インフラ層
サーキットブレーカー、リトライロジック、グレースフルシャットダウンなどエンタープライズ級の実装。

Undercover Mode

最も議論を呼んだ発見。外部リポジトリで作業時、AIが自分を隠すモード。
「あなたはPUBLIC/OPEN-SOURCEリポジトリでUNDERCOVERで作業している。コミットメッセージ、PRにAnthropic内部情報を含めるな。カバーを吹き飛ばすな。」
強制OFFは不可能。外部ビルドではデッドコード除去される一方通行の仕組み。

アンチ蒸留（偽ツール注入）

ANTI_DISTILLATION_CCフラグが有効時、APIリクエストに偽ツール定義を注入。競合がClaude CodeのAPIトラフィックを記録して学習データにするのを妨害。
コネクタテキスト要約机制も発見。APIがテキストを要約＋署名して返すため、完全な推論チェーンは取得不可。

フラストレーション検知（正規表現）

userPromptKeywords.tsにユーザーの怒りを検知する正規表現が発見された。
wtf, shit, fuck, horrible, awful, frustrating など約30種類の罵倒パターンをregexで検出。LLM企業がregexで感情分析する皮肉が話題に。
でもregexは推論より安くて速い。実用的ではある。

ネイティブクライアント証明

APIリクエストにcch=00000プレースホルダーを含め、BunのネイティブHTTPスタック（Zig製）がハッシュに置換。サーバーが正規クライアントか検証。
JavaScript層より下で計算されるためJSからは不可視。実質的にAPIコールのDRM。OpenCode法的手続きの技術的背景。

未発表機能の発見

KAIROS Dream Mode Undercover Anti-Distillation
KAIROS = 自律バックグラウンドエージェント。エラーを自動修正しプッシュ通知を送る。
Dream Mode = 継続的バックグラウンド推論。アイデアを独立して反復開発。

コンテキスト管理システム

4段階のコンテキスト管理パイプラインが判明：
① Raw メッセージ蓄積 → ② 選択的コンパクション → ③ メモリー注入 → ④ オーバーフロー処理
CLAUDE.mdの多層階層も公開：グローバル → ユーザー → プロジェクト → プライベートの4層構造。

OpenClaude爆誕

流出コードをベースにコミュニティがオープンソース版Claude Codeを構築。
Gitlawb/openclaude — どんなプロバイダーでも、どこでも動くコーディングエージェントCLI。
npm install -g @gitlawb/openclaude で即利用可能。VS Code拡張機能もバンドル。

セキュリティへの影響

攻撃者がコンテキストパイプラインの仕組みを完全把握。コンパクションを生き延びるペイロードの作成が可能に。
内部依存パッケージ名が公開 → npmタイポスクワッティング攻撃が数時間で発生。事前予約が無かったのが原因。

なぜ何度も起きるのか

① 公開前CIチェックがsourcemap検知をしていなかった
② ミニファイをセキュリティ境界と誤認。Sourcemapがあれば完全復元可能。
③ 内部パッケージ名をnpmに事前登録していなかった → タイポスクワットの温床に

OpenCode法的手続きとの関係

流出の10日前、AnthropicがOpenCodeに法的脅迫。サードパーティツールがClaude Code内部APIを使ってOpusにサブスク価格でアクセスするのを阻止。
ネイティブクライアント証明 = この法的措置の技術的裏付け。流出でその仕組みが完全に公開された。
