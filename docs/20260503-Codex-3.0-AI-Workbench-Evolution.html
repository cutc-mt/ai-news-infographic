Codex 3.0

CLIからフルAIワークベンチへの進化
AICodeKing
OpenAIのCodexが、単なるCLIラッパーから本格的なAI開発ワークベンチへと進化を遂げた。デスクトップアプリの大幅刷新、ブラウザ利用、コンピューターユース、CLI v0.122〜0.125の連続アップデート、そしてGPT-5.5の統合により、Codexは最も完成度の高いコーディングプラットフォームになりつつある。

①デスクトップアプリの大幅刷新

Codex Desktopが単なる「CLIのラッパー」から本格的なワークスペースへと生まれ変わった。これまでAIコーディングツールはファイルを編集できても、実際の見た目を確認する機能が弱かった。
アプリ内ブラウザでローカルサイトや公開ページを開いてフィードバックを残し、Codexに修正を依頼できる。さらにmacOSコンピューターユースでネイティブアプリの操作も可能になり、GUI限定のバグにも対応できるようになった。
チャット機能も追加され、プロジェクトフォルダなしで研究、執筆、計画、分析ができる。スレッドの自動化で定期実行も可能に。

③Codex Pets — 便利な浮遊コンパニオン

Codex Petsはオプションのアニメーション付きコンパニオン機能で、設定から有効にできる。一見すると「可愛いだけのギミック」に見えるが、実用的な価値がある。
フローティングオーバーレイが他のアプリを使っている間もアクティブなスレッドを可視化。Codexが実行中か、入力待ちか、レビュー準備完了かを一目で確認できる。バックグラウンドでCodexを動かす開発者にとって非常に便利な機能。
/hatch-pet スキルで自作キャラクターの作成も可能。

④CLI v0.122.0 — 基盤の強化

派手な機能ではなく、信頼性と制御性の向上に焦点を当てた重要なアップデート。
/side コマンドでサイド会話が可能に。メインの作業を止めずに「このファイルは何をするのか？」「この依存関係はまだ必要か？」と質問できる。これは実際の開発ワークフローで非常に役立つ。
キュー入力も改善され、Codexの動作中に次の指示を入力できるようになった。エージェントが止まるのを待つ必要がなくなった。

⑤プランモードの大幅改善

プランモードが大きく進化した。新しいコンテキストで実装を開始でき、移行前にコンテキスト使用量を確認できる。
長い計画セッションでは議論や代替案、古いアイデアでコンテキストが汚染されがち。クリーンなコンテキストで実装を開始できることで、実際のコーディング部分がより的確になる。これは非常に優れた改善。
スタンドアロンインストールも自己完結型になり、WindowsやIntel Macでも正しく動作するようになった。

⑥プラグインエコシステムの成長

v0.122.0でプラグインが大幅に改善された。タブブラウジング、インラインの有効/無効トグル、マーケットプレイスからの削除、リモートクロスリポやローカルマーケットプレイスソースのサポートが追加された。
プラグインがCodexの最も重要な方向性の一つに。スキルやMCPは便利だが、適切にパッケージングしてこそチームや一般ユーザーに再利用可能になる。プラグインが簡単になれば、CodexはクローズドなAIコーディングツールよりはるかにカスタマイズ性が高くなる。

⑦セキュリティと権限の強化

Codexに企業向けの権限管理が導入された。サムネイルにはならないが、実際の運用では極めて重要なアップデート。
読み取り拒否（deny-read）ポリシーでエージェントが読めるファイルを制限。信頼済みワークスペースで安全な実行境界を定義。サンドボックスの強化で分離を保証。秘密鍵や環境ファイル、顧客コードを含むリポジトリでこそ必要な機能。
プロジェクトフックとexecポリシーが信頼済みワークスペースを要求するようになり、セキュリティレベルが底上げされた。

⑧v0.123.0 — Amazon Bedrock対応

Amazon Bedrockがビルトインのモデルプロバイダーとして追加され、AWSプロファイルでの認証が可能に。
これは重要な意味を持つ。CodexがOpenAIモデルだけでなく、AWSエコシステムにも対応した。AWS中心の企業ではBedrockサポートは大きな意味を持つ。
さらに/mcp verboseでMCPサーバーの完全な診断情報（リソース、テンプレート）を表示可能に。MCPのセットアップは時にトラブルシューティングが難しいため、これはパワーユーザーに嬉しい機能。

⑨v0.124.0 — 推論制御とマルチ環境

ターミナルUIにクイック推論制御が追加された。
Alt+, （カンマ） 推論を軽く
Alt+. （ピリオド） 推論を重く
メニューを開いたり設定を変更したりせずに、現在のタスクの重さを即座に調整できる。マルチ環境サポートで1つのセッションが複数の環境と作業ディレクトリを管理可能に。Hooksが安定版となり、MCPツールや長時間のBashセッションの監視にも対応。

⑩GPT-5.5 — Codex推奨モデルに

GPT-5.5がCodexの推奨モデルとして利用可能になった。実装、リファクタリング、デバッグ、テスト、検証、ナレッジワークのアーティファクトに適している。
モデルピッカーにGPT-5.5が表示されない場合は、CLI、IDE拡張機能、またはアプリのアップデートが必要。ロールアウト中はGPT-5.4がフォールバックとして利用可能。
対象のChatGPTプランでは、高速サービスタイアがデフォルトに。別料金ではなく、既にChatGPTを利用しているユーザーにとっては価値が向上し続けている。

⑪ブラウザ利用 — UI検証の自動化

Codexがアプリ内ブラウザを実際に操作できるようになった。ローカル開発サーバーやファイルベースのページで、レンダリングされたUIをクリックして確認できる。
コーディングエージェントがUIを見られないのは常に限界だった。CSSを変更できても、結果が本当に良いかどうかが分からない。ブラウザ制御はこのフィードバックループを閉じる重要なステップ。
視覚的なバグの再現やローカル修正の検証が自動で行える。

⑫自動承認レビュー

Codexが対象となる承認プロンプトを自動レビュー担当エージェントに渡す仕組みが導入された。
要するにAIが別のAIの行動をレビューしてから実行する仕組み。最初は奇妙に聞こえるが、危険なコマンド、ファイルアクセス、リスクの高い操作に対しては意味がある。「エージェントがこれをやりたい」から「実行された」までの間に、もう一つの防護レイヤーを挟むことができる。
レビュー状況とリスクレベルがアプリに表示され、承認・拒否・停止・タイムアウトの状態が確認できる。

⑭画像生成とツール検索

v0.122.0からツール検索と画像生成がデフォルトで有効になった。MCPやJavaScript REPLからの画像出力のメタデータと高解像度画像処理も改善。
UIの構築やデバッグに役立つだけでなく、アプリのブラウザやアーティファクトビューアーの方向性とも相性が良い。OpenAIはCodexでプレーンテキストと差分以上のものを扱えるようにする意図を明確にしている。

⑮権限プロファイルの統一

権限プロファイルが双方向セッション、ユーザーターン、MCPサンドボックス、シェル昇格、アプリサーバーAPIを横断して一貫性を保つようになった。
技術的に聞こえるが、シンプルな意味はこうだ — CLI、アプリ、MCP、シェル承認、アプリサーバーのすべてが「何が許可されているか」で合意する。Codexが動くあらゆる場所で権限状態が一貫していることが重要。

⑯多数のバグ修正と安定性向上

各バージョンで多くの修正が行われている：
古い承認の自動消去 トークン使用量の即時反映 ログアウト時のトークン失効 ファイルウォッチャーの新規ファイル検知
その他、ロールバック後のコピー、スタック状態の解消、VS Code WSLでのUnicode入力などが修正され、モデルメニューやスキルリストのUI改善も行われた。/reviewの中断時や双方向終了時のスタックも解消された。
