Codex App ペット機能

自作キャラで癒される使い方を解説
まさおAIじっくり解説ch
OpenAIのCodex Appに新機能「ペット機能」が登場！デスクトップ上にアニメーション付きコンパニオンを表示し、作業状況をリアルタイムで確認できる。自作キャラクターの作成方法から便利な使い方まで完全解説。

Codex Petsとは？

OpenAIのCodex Appで利用できるアニメーション付きコンパニオン機能。デスクトップに浮遊するオーバーレイとして表示される。
他のアプリを使っている間も、Codexの作業状況を視覚的に確認できる常駐型UI。

フローティングオーバーレイ

ペットはデスクトップにフローティングオーバーレイとして常駐。好きな位置に配置可能。
アクティブなスレッド、実行中・入力待ち・レビュー準備完了などの状態をリアルタイム表示。

8体のデフォルトペット

設定の「外観→ペット」から選択可能。全8体の公式ペットが用意されている。
Codex 手掛かり発見
Fireball ビルド監視
Rocky 安定した足場
BSOD エラー対策

起動方法は簡単

3つの起動方法：① /pet（なでる）コマンド ② Ctrl+K / Cmd+K ③ 設定→外観→ペット
右クリックで「ペットを閉じる」も可能。しまいたい時は設定→ペットをしまうから操作。

カスタムペットの作成手順

① $skill-installer hatch-pet を入力してスキルをインストール
② Codex Appを再起動
③ /hatch コマンドで作成開始
④ 画像を渡すとデフォルメキャラを自動生成
⑤ 設定→外観→ペットで選択

画像からペット生成

ベース画像を渡すだけで、GPT Image 2を使ってゲーム素材風のデフォルメキャラを自動生成。
画像はCodex App内のイメージツールでも作成可能。外部ツール不要で完結する。

ペットファイルの保存先

カスタムペットはローカルに保存される。
.codex/pets/<pet-name>/ に pet.json と spritesheet.webp として保存。

なぜペット機能が嬉しい？

AI駆動開発やエージェント作業を続けていると頭痛や精神的疲労を感じることがある。そんな時にキャラクターに癒される効果。
機能としてはシンプルだが、長時間のAI開発において心理的なサポートになる。

オートメーション機能

Codex Appには定期的な自動実行も可能なオートメーション機能が搭載。
スレッド内で「自動化を追加」を選ぶと、指定間隔でセッションを定期実行できる。

アプリベースの使用感ならCodex、CLIベースの探索ならClaude Code。両方使うのが最適。

ドキュメントの充実

Codexの公式ドキュメントが非常に充実している。英語だが翻訳しながら読めば、機能の全容を把握できる。
クロードコードのドキュメントも充実しているという認識が広まっているが、Codexも負けていない。

GPT-5.5モデルの進化

GPT-5.5およびGPT-5.5 Proは文章の品質も高く、モデル単体の使い勝手も非常に良いと好評価。
Opus 4.7も悪いモデルではないが、環境込みで評価するとGPT-5.5+Codexが有利な場面が多い。

コミュニティでスキル共有

hatch-petのような補助スキルがコミュニティで共有されている。自分で作るだけでなく、他者が作ったスキルも活用可能。

AI開発体験の新時代

ペット機能は地味だが、AIとの長時間作業において心理的快適さをもたらす。OpenAIが「開発者体験」に注力している表れ。
機能性だけでなくエモーショナルな体験も重視する方向性が明確に。
