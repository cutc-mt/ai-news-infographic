Codex Petとは？

Codexアプリに追加されたオプションのアニメーションコンパニオン機能。
ペット自体はコーディングを行わないが、Codexが作業中のスレッドを可視化し、タスク完了や入力待ちの通知を表示するフローティングオーバーレイ。

ペットの召喚方法

/petコマンドを入力するだけでペットの表示/非表示を切り替え可能。Settings → Appearance → Petsからも選択できる。
キーボードショートカット：Cmd+K / Ctrl+K から「Wake Pet」「Tuck Away Pet」を実行。

8種類の内蔵ペット

デフォルトで8種類のビルトインペットが用意されている。可爱い動物からロボットまで、自分の好みに合わせて選択可能。
内蔵ペット カスタム可 AI生成 共有可能

/hatchコマンドで生成

AIを使って自分だけのオリジナルペットを生成できる！
/hatchコマンドでAIにペットのデザインを指示。例えば「可愛いゴブリン」や「ドラゴンの赤ちゃん」など、自由な発想で生成可能。

hatch-petスキルのインストール

カスタムペットを作成するにはhatch-petスキルが必要。
$skill-installer hatch-petでインストール後、Cmd+KでForce Reload Skillsを実行。その後スキルにペット作成を指示。

フローティングオーバーレイ

ペットは他のアプリの上に浮遊して表示される。
エディタやブラウザを開いたまま、ペットがCodexの状態をリアルタイムで表示。スレッドの進行状況をアプリを切り替えずに確認可能。

通知機能

ペットが知らせる3つの状態：
Codex実行中 Running
入力待ち Waiting
レビュー可能 Ready

ペットの共有

アーリーアダプターが既に大量のカスタムペットをアップロード。Microsoft Clippyのバージョンまで存在し、コミュニティで共有が活発化中。
Settings → Appearance → ThemeのShare機能で友達にカスタムテーマを共有可能。

Codexエージェントとの連携

Codexアプリ内のエージェントはCLIやIDE拡張機能と同じ設定を継承。config.tomlで統一的に管理。ペットはエージェントの状態を視覚的にフィードバック。

CLIからの利用も可能

Codex CLIでも同じ設定とスキルが利用可能。アプリ版とCLI版の間でシームレスな体験を提供。ターミナル派でもペット機能を活用できる。

ペットの今後の展開

現在はオーバーレイ表示のみだが、将来的にはよりインタラクティブな機能が追加される可能性。ペットがコーディングの提案をしたり、学習をサポートしたりする進化が期待される。
