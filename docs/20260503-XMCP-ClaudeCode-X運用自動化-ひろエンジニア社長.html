XMCPでClaude CodeからX運用を自動化

X公式のMCPサーバーを使って、ツイート・分析・DMをAIに任せる方法

📅 2026年5月3日 | ひろエンジニア社長 / AI大全

①

X運用の課題とXMCPの登場

  • ツイートのネタ探し、投稿、分析、DM対応などを全てやると1日数時間を消費してしまう。
  • X公式の開発プラットフォームからXMCPサーバーがリリースされ、AIでのX運用が可能に。
  • 第三者ツールではなくX公式ツールなので信頼性が高い。
XMCPはX公式のMCPツールの一種。Claude CodeのようなAIエージェントと連携してXを操作できる！
②

XMCPの仕組みと連携イメージ

  • 人間がClaude Codeに指示を出す。例：「最近のAIトレンドを調査してツイートして」。
  • Claude Codeが自動で情報を調査・分析する。
  • XMCPサーバーを通してXのAPIを操作し、投稿や分析を実行。
人間の指示 → Claude Code → XMCPサーバー → X（ツイート投稿完了！）
③

この動画の対象者

  • ソロプレナー・個人事業主：Xの運用に時間を使いすぎている人。
  • SNSマーケター：AIの自動化で情報発信をスケールさせたい人。
  • AIエージェント利用者：Claude CodeやCursorを普段から使っている人。
  • ターミナル操作が出てくるため完全ノーコード派には少し難しいが、勉強としては有用。
④

XMCPでできること一覧

  • ツイートの自動化：トレンド取得から投稿まで全自動。
  • フォロワー分析：関心分野、アクティブ時間、エンゲージメント傾向を分析。
  • DMの自動送信：新規フォロワーへの挨拶DMを自動化。
  • 画像の投稿やリポストなどにも対応。
  • 検索、トレンド取得、アカウントのインサイト分析も可能。
実は100以上のツールが利用可能！かなり細かい操作まで自動化できる。
⑤

ツイート自動化の流れ

  • Claude Codeに「AIトレンド1位を取得して投稿して」と指示。
  • XMCPのgetTrendByツールでトレンドを自動取得。
  • 取得したトレンドから投稿案を自動作成。
  • XMCPのcreatePostツールでXに投稿完了。
投稿URLが発行され、実際にツイートがXに反映される！
⑥

デモ：トレンドツイートの自動投稿

  • Claude Codeを立ち上げてプロンプトを貼り付け、エンターで実行。
  • XMCPの各ツール使用の許可画面で「Yes」を押していく。
  • トレンド1位はGeminiのファイル直接生成機能と特定。
  • 「ChatGPT、Claude、Geminiのファイル直接生成が3者揃った」という投稿案を自動作成。
  • オッケーと入力すると自動でツイート投稿完了。

ツイート自動化の応用アイデア

  • AIにトレンドを調べさせて自動ツイートする流れ作業の自動化。
  • 投稿内容を修正したい場合は、Claude Codeに修正指示を出すだけで調整可能。
人間が一切考えることなく、AIが自動で投稿してくれる！SNS運用の効率が劇的に変わる。

⑪

DMの自動送信機能

  • 最近フォローしてくれた人に丁寧な挨拶DMを自動作成・送信。
  • いきなり送信せず、送信前に必ず確認するフローを設定可能。
  • 「丁寧で自然な日本語」「営業感を出さない」「100文字以内」などの条件を指定。
「初めまして、ひろです。フォローありがとうございます。」みたいな自然なDMを自動作成！
⑫

DM送信デモの流れ

  • 「最新フォロワーに挨拶DMを作成して」とClaude Codeに指示。
  • XMCPで最新フォロワーの情報を自動取得。
  • DMの案を自動作成して確認を求めてくる。
  • オッケーと入力すると自動でDM送信完了。
全て日本語の指示だけで完結。コードを書く必要はない！
⑬

セットアップ手順①：APIキーの取得

  • X Developer Console（開発者用管理画面）を開く。
  • 新規アプリを作成（例：XMCP Sample）。EnvironmentはDevelopmentでOK。
  • 以下の3つのAPIキーが発行される：
  • Consumer Key（コンシューマーキー）
  • Consumer Secret（コンシューマーシークレット）
  • Bearer Token（ベアラートークン）
発行された3つのキーをメモに貼り付けて保管しておこう！
⑭

セットアップ手順②：ユーザー認証設定

  • 作成したアプリを選択し、「ユーザー認証設定」のSetupボタンを押す。
  • アプリの種類はWeb Appを選択。
  • 以下の2つのURLが必須：
  • Callback URL：認証後に戻るURL（ローカル開発環境のURLを指定）
  • Website URL：アプリのベースURL（同じくローカルURLを指定）
2つのURLを入力して「変更を保存」を押せば認証設定完了！
⑮

セットアップ手順③：MCPサーバーの準備

  • GitHubからXMCPレポジトリをクローンしてPCに持ってくる。
  • クローンしたフォルダに移動してPython仮想環境を作成。
  • 仮想環境を有効化して、必要なライブラリを一括インストール。
  • 環境変数ファイルに3つのAPIキー（Consumer Key、Secret、Bearer Token）を設定。
  • Pythonのサーバーファイルを起動してXMCPサーバーを起動。
⑯

セットアップ手順④：Claude Codeとの連携

  • Claude CodeにXMCPを登録するコマンドを実行： 
claude mcp add x-mcp ...
  • 出力に「@mcp-server/x-mcp」と表示されれば追加成功。
  • Claude Codeで「XMCPは利用可能？」と質問して確認。
  • 「XMCP利用可能です」と返ってくればセットアップ完了。
セットアップの難易度は高め。詳細はnoteの記事を参照！
