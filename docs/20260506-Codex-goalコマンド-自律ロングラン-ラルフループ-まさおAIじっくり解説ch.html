Codexの /goal コマンド徹底解説

自律ロングラン開発を実現する必須技術 - Ralph Loopとの比較からgoal-forgeまで

2026年5月6日

① /goal コマンドとは

  • Codex CLI 0.128.0 から公式に追加されたコマンド
  • 指定したゴールに到達するまで自律ループを回し続ける仕組み
  • 目的から逆算して、そこにたどり着くまで粘り強く実行し続ける
  • 行き当たりばったりではなく、事前に仕様を徹底的に作った上で順番に実行する設計思想
Codex使いなら絶対に押さえておきたいコマンド！
ロングランさせるほど効果が出るよ

② Ralph Loopとの共通思想

  • Ralph Wiggum プラグイン（Ralph Loop）とアイデアは似ている
  • どちらも「ゴールに到達するまで走り続ける」という発想は同じ
  • Ralph LoopはClaude Code界隈で流行したプラグイン
  • Ralph LoopはAnthropicのオフィシャルプラグインにも採用された実績あり
まさおさん自身、Ralph Loopはめちゃくちゃ愛用しているそう！

③ 6時間の自律開発が可能に

  • ブログ報告では6時間44分の連続自律開発に成功
  • 夜寝ている間にAIが作業してくれる「夜間バッチ」としても活用可能
  • 大量リファクタや多数テスト追加に最適
  • 今では6時間どころか、何十時間も動かしている人もいるとのこと
寝て起きたらコードが完成している...
まさに未来の開発スタイル！

④ /goal の導入方法

  • ユーザーディレクトリ下の .codex 隠しフォルダーを開く
  • config.toml ファイルを編集する
  • features の下に goals = true を追加 
# config.toml の設定例
[features]
goals = true
設定後、/goal と入力すると
ゴールアクティブ状態になるよ！

⑤ 実例: 哲学的ツイートを100点に

  • /goal で「哲学的なツイートを100点取るまで作って」と指示
  • 「Goal Active」状態になり、ゴールを満たすまで自動で作り続ける
  • 「100点」という基準がアバウトすぎると、AIが勝手に100点相当と解釈して終了してしまう問題も
  • 評価基準をより細かく設定することが効果的な使い方のコツ
「100点」の基準をどう定義するかが
一番のポイントだね！

⑥ ゴールクリアと再設定の小技

  • ゴールクリア で途中でゴールを完了させることが可能
  • 新しいゴールをスマートスラッシュコマンドで再設定できる
  • 途中で方向転換したい場合にも柔軟に対応
  • 成功基準を変更せずに、離席中に走らせたい場面で特に有効
柔軟にゴールを切り替えられるから
実用的な使い方ができるよ！

⑦ Ralph Wiggumとの決定的な違い

  • /goal は1つのコンテキストウィンドウ内で走り続ける
  • Ralph Loopは毎回新しいフレッシュなコンテキストで再起動する方式

⑧ コンテキストウィンドウの思想

  • /goal はCodex公式がサポートしている機能
  • Ralph Loopはプラグインとして実装され、コードを自分で編集・カスタマイズ可能
  • コンテキストをどう扱うかという設計思想の違いが現れている
  • Ralph Loopの方がカスタマイズ性が高い一方、/goalの方が公式の安定感がある
どちらが良いかはユースケース次第。
両方使えるのがベスト！

⑨ ロングランに適したユースケース

  • 大量リファクタリング - 離席中にコード全体を整理
  • 多数テスト追加 - テストカバレッジを自動で向上
  • 夜間バッチ処理 - 寝ている間にAIが作業
  • ハーネスが仕上がるほど、自分の代わりにAIエージェントに仕事をさせられる
単発タスクには過剰かも...
反復的で評価基準が明確な作業に最適！

⑩ 解像度の高いゴール設計が鍵

  • ゴールの基準をどれだけ解像度高く設定できるかが成功のカギ
  • サブエージェントに評価させる基準を明確に定義すると効果的
  • ゴールの基準を細分化することで、ループの精度が上がる
  • 評価基準の解像度を上げるほど、出力の品質も比例して向上
ぼんやりしたゴールだと
AIが勝手に「できた」と判断しちゃう...

⑪ 自己評価の罠に注意

  • 最初から自己評価が「96点です！」と高く出てしまい、即終了する問題
  • 基準が曖昧だと、AIが勝手に「十分な品質」と判断してしまう
  • 対策として外部評価や具体的チェックリストを用意する
  • サブエージェントによる客観的な評価基準を組み込むのが有効
「本当に100点なの？」と
自分で疑える基準作りが大事！

⑫ goal-forgeの登場

  • goal-forge は /goal をサポートするサードパーティのCodex Skill
  • ユーザーのラフアイデアをSPEC.md → GOAL.mdに変換してくれる
  • ハードゲートが設定されており、品質を高める工夫がされている
  • GitHubで公開されているオープンソースプロジェクト
goal-forgeを使うと
/goalがさらに強力になる！

⑬ goal-forgeの導入方法

  • GitHubのリポジトリから導入コマンドをコピー
  • ターミナルにペーストするだけでCodexのグローバル領域にインストール
  • /goal-forge でスキルを呼び出せるようになる 
# goal-forge の導入イメージ
# ターミナルにコピペするだけ
$ codex skill install goal-forge

⑭ 実演: goal-forgeでテトリスを作る

  • /goal-forge で「美しいテトリスを作る」と指示するだけ
  • 自動でSPEC.md（仕様書）とGOAL.md（目標書）を作成
  • お任せで実行すると、ゴールに到達するまで自動で走り続ける
  • アプリ構成を一緒に詰めてくれるので、品質の高い成果物が期待できる
「テトリス作りたい」と言うだけで
仕様から完成まで自動でやってくれる！

⑮ ラフアイデアからゴールへの変換

ラフアイデア SPEC.md GOAL.md /goal 実行
  • ユーザーのラフなアイデアを構造化された仕様に変換
  • SPEC.md で何を作るかを明確に定義
  • GOAL.md でゴールの達成基準を具体化
  • これをもとに /goal が確実に実行される

⑯ ハードゲートによる品質担保

  • ハードゲートが設定されており、各段階で品質をチェック
  • SPEC.mdの品質が不十分なら先に進まない仕組み
  • 確実性の高い成果物を作るための工夫が組み込まれている
  • プランの段階にトークンをかけることで、目的からのズレを防ぐ
品質ゲートがあるから
変なゴールで走り出さない！
