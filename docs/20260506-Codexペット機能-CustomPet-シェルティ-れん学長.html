想像以上に便利！Codexのペット機能を本気で試した結果

OpenAI Codexアプリで自分だけのカスタムペットを設定する完全ガイド

2026年5月3日

① Codexアプリが急激に進化中

  • 最近のCodexアプリはファイル操作や画像生成など、機能が大幅に向上している。
  • 操作性も改善され、Claude CodeからCodexへの乗り換えを検討するユーザーも増加中。
  • そんなCodexアプリに、なんとペット機能が追加された。
実用的というより癒し要素だけど、個人的には結構好きな機能です！

② ペット機能とは何か

  • Codexアプリの設定 → 外観の一番下に「ペット」という項目がある。
  • デフォルトで用意されたペットキャラクターから選択可能。ペットを起こすと右下に常駐する。
  • ペットに話しかけると、考え中のアニメーションと共にAIが応答してくれる。
  • ドラッグで移動も可能。ジャンプやアニメーションも楽しめる。
「こんにちは」と挨拶すると「何か手伝いましょうか？」と返してくれます！

③ デフォルトで選べるペットたち

  • コデックス君 — デフォルトの公式キャラクター。
  • 雨粒みたいな「ピチョン君」 — なんか可愛い。
  • ファイヤーボール君 — 炎のキャラクター。
  • ロッキー — 岩石風のキャラクターなど複数種類。
  • 設定画面から選ぶだけで、すぐにアプリ内に表示される。
標準ペットだけでも種類豊富で、どれも可愛い！

④ ペットとの対話が面白い

  • チャット欄で「こんにちは」と入力すると、ペットが考え中のアニメーションを開始。
  • 「Codexのペット機能について最新情報を教えて」と頼むと、真面目に調査してくれる。
  • 「最新情報なのでまず公式情報を優先して確認します」のような思考プロセスも表示される。
  • 下部のバーを押すと処理中の内容が表示され、クリアすると完了状態になる。
ペットが考えてる姿を見るだけで癒される…！

⑤ hatch-petスキルをインストール

  • カスタムペットを作るには、まずhatch-petというスキルをインストールする必要がある。
  • スキルインストーラーを使って「hatch-pet」を検索・インストール。
  • インストール後、Codexを再起動すると新しいスキルが読み込まれる。
  • 再起動後に「force-load-skill」を実行して、hatch-petを有効化する。
再起動はCommand+Qでアプリを終了してから再度立ち上げるだけ！

⑥ シェルティ君をカスタムペットに

  • 普段グラレコで使っているシェルティ君（シェルティードッグの研究員キャラクター）をペットに設定する実験。
  • 「この画像を元にサブエージェントを使ってペットを作って」と指示するだけでOK。
  • 生成されたペットは ~/.codex/pets/sheltie/ に自動保存される。
  • Custom Pet URLを使ってGIF画像を設定することも可能。
自分の好きなキャラクターをペットにできるのが最高！

⑦ hatch-petの実行プロセス

  • 親エージェントが実行フォルダーとベースを管理し、ジョブマニフェストを作成。
  • Pythonスクリプト（pjstatus.py、prepare-pet-run等）が複数実行される。
  • 各ポーズの生成はサブエージェントに委任されて並列処理される。
  • 生成されたベース候補を目視確認し、使えるものだけをマニフェストに記録。
親は管理、子は生成という役割分担が秀逸！

⑧ カノニカルベースとは

  • カノニカルベースとは、ペット制作における正解の見た目の基準画像のこと。
  • 元画像をそのまま使うのではなく、Codexペット向けの小さいスプライト風に変換する。
  • ドラクエのドット絵のような可愛いサイズ感に変換される。
  • このベース画像を基準にして、各アニメーションポーズが生成される仕組み。
シェルティ君がドット絵になると予想以上に可愛い！

⑨ サブエージェントによる並列生成

  • サブエージェントを使うことで、複数のポーズ画像を同時並列で生成できる。
  • アイドル、ランニングライト、ランニングレフト、ジャンピングなどに分割して処理。
  • 最大6つのエージェントが同時並列で実行可能（環境の上限に依存）。
  • 同時エージェント上限に達すると「プロセスを起動できませんでした」と表示されることも。
サブエージェントの名前付き管理もできるから見やすい！

⑩ 生成される9種類のポーズ

  • アイドル — 待機中のポーズ。何もしていない時の基本姿勢。
  • ランニングライト — 右方向に移動する時のアニメーション。
  • ランニングレフト — 左方向に移動する時のアニメーション。
  • ジャンピング — ジャンプする時のポーズ。
  • ウェイビング — 手を振るポーズ。
  • ウェイティング — 待っている時のポーズ。
  • フェイルド — エラー失敗時のポーズ。
寝てるポーズや甘えてるポーズまで作ってくれます！

⑪ スプライトシートの完成

  • 9種類のポーズ画像を1枚のスプライトシートにまとめて出力。
  • シェルティ君の特徴をしっかり踏まえた上で、ドット絵風に変換。
  • 背景の透過処理も自動で行われ、CODexアプリ上で自然に表示される。
  • 完成したスプライトシートは自動的にCodexのペットフォルダに保存される。
9種類のポーズが1枚にまとまって可愛い！

⑫ Codexアプリでペットを選択

  • 設定 → 外観 → ペットの順に開き、生成されたSheltieが選択肢に表示される。
  • Sheltieを選択して「ペットを起こす」をクリックするだけ。
  • アプリに戻ると、右下にシェルティ君が常駐している。
  • これで自分だけのカスタムペットの設定が完了！
設定画面に自分のキャラが表示されるのは感動的！

⑬ 別アプリでもペットが常駐

  • Codexアプリ上だけでなく、PC上の別アプリにもペットが常駐する。
  • 作業中にブラウザやエディタに切り替えても、ペットが見える状態を維持。
  • 処理が完了した時などにリアルタイムで確認できるのが大きなメリット。
  • 別ウィンドウで作業していても、癒しの存在が常にそばに。
他のアプリでも見えるのは想像以上に便利！

⑭ AI作業中の癒し効果

  • AIで作業していると、心が削られる瞬間がある。そんな時にペットが癒しをくれる。
  • 自分の好きなキャラクターをペットとして設定できるから、愛着もひとしお。
  • 考え中のアニメーションを見るだけで、作業のストレスが軽減。
  • 実用性というより遊び心だが、モチベーション維持に効果的。
シェルティ君と一緒に作業するのが楽しくなります！

⑮ サブエージェントの可能性

  • Codexアプリでもサブエージェントが使えるようになった（Claude Codeと同様）。
  • サブエージェントには名前（James、Ball等）が付けられ、管理が見やすい。
  • ペット生成以外にも、並列処理の活用場面は多岐にわたる。
  • CodexアプリがClaude Codeに近い機能を備えつつある。
サブエージェントのUIも見やすくて好印象！

⑯ hatch-petの全体ワークフロー

  • スキルインストーラーでhatch-petをインストール。
  • Codexアプリを再起動してスキルを読み込む。
  • 画像を見せて「サブエージェントを使ってペットを作って」と指示。
  • カノニカルベースが生成され、各ポーズがサブエージェントで並列作成。
  • スプライトシートが完成し、~/.codex/pets/に保存。
  • 設定画面でカスタムペットを選択して完了。
たった6ステップで自分だけのペットが完成！

⑰ Codexアプリの画像生成も優秀

  • Codexアプリ内でGPT Image 2を使った画像生成が可能。
  • サムネイル画像などもCodexアプリで作成でき、使用感も良好。
  • GIF画像を設定すれば、動くペットとして常駐させることもできる。
  • 画像生成からペット設定まで、Codexアプリ内で完結するのが便利。
画像生成とペット機能の相性が抜群！
