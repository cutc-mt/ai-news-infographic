Gemini ファイル生成機能まとめ

チャットから一撃でドキュメント・PDF・Excelを作成

2026年5月6日

①公式発表: Geminiのファイル生成機能

  • 2026年4月27日付けでGoogleの公式ブログで発表
  • Geminiのチャット画面からプロンプトだけで、さまざまなファイル形式のドキュメントを直接作成可能に
  • アイデアをそのまま綺麗に整ったファイルに変換できるという機能
  • コピペや別アプリでの整形といった手間がほぼ不要に
これまでもSNSで「できるかも？」と一部で話題だったけど、今回は公式機能としての堂々たるアナウンス！

②対応ファイル形式はほぼ全部

  • Google系: ドキュメント、スプレッドシート、スライド
  • PDF: PDFファイルも直接生成可能
  • Microsoft系: Word、Excel、CSV
  • その他: RTF、プレーンテキスト、Markdown
ほぼ全形式に対応！チャット画面から出ないで完結するのが本当に便利 ✨

③誰でも使える: 全Googleユーザー対象

  • 対象: Googleアカウントを持つ全員（年齢問わず）
  • Google Workspace法人プランの方も対象
  • 個人の無料Googleアカウントの方も全員利用可能
  • 現時点では1回のプロンプトにつき1ファイルまで
無料アカウントでも使えるのが嬉しいポイント！制限はあるけど十分すぎる便利さ

④進化のスピード: 3ステップから1プロンプトへ

  • 従来: 文章マニュアル作成 → スクショツール作成 → 合体（3ステップ）
  • 現在: たった1つのプロンプトで一撃完成
  • 以前は何度試しても「動画からのスクショは無理」と言われていた
  • 公式アナウンス前後でGemini側が急激に進化したと推測
AIの世界は本当に進化が速い。前回の動画で3ステップを学んだ方にはごめんなさい！

⑤活用例①: 動画からスクショ付きマニュアルを一撃作成

  • Geminiのチャット画面にMP4動画をアップロード
  • プロンプトは「この動画から画像スクショ付きのマニュアルをワードで作成して」だけ
  • Wordファイルが回答内に直接表示される
  • 各操作ステップに画面のスクリーンショットが自動挿入される
操作録画の動画さえ撮っておけば、マニュアル作成で消耗しなくて済む！

⑥マニュアル作成の手順: 4ステップで完了

  • Geminiのチャット画面を開く
  • 左下のプラスアイコン → Googleドライブから動画をアップロード
  • プロンプトに「動画から画像スクショ付きのマニュアルをワードで作成して」と入力
  • 送信後しばらく待つとWordファイルが表示される → クリックして確認
社内向けマニュアルや新人への引き継ぎ資料に最適！

⑦デモ結果: 高品質なWordマニュアルが完成

  • 「NoteLMガイド.docx」という名前でWordファイルが生成
  • 各ステップにその時点の画面スクリーンショットが挿入
  • 操作手順とキャプチャ画像が正確にペアリングされている
  • 前回の3ステップで作っていたものと同等の品質
以前は3ステップで苦労していたものが、たった1行のプロンプトで！

⑧マニュアル作成の実務活用シーン

  • 新人研修: 操作手順の録画動画から即座に教材を作成
  • 引き継ぎ資料: 退職者の手順動画からマニュアル化
  • カスタマーサポート: よくある質問を動画→マニュアル化
  • マニュアル作成の工数が劇的に削減される
操作の様子を録画するだけでOK。あとはGeminiが全部やってくれる！

⑨活用例②: スプシデータから請求書PDFを自動作成

  • スプレッドシートの明細データを元に請求書PDFを自動生成
  • ユーザーがチャットで条件を指定すると、Geminiが自動でPDFを作成
  • 毎回同じレイアウトの請求書を作るためにGem（カスタムボット）を活用
  • 裏側でPythonコードが実行されてPDFが生成される仕組み
経理や事務の方に特におすすめの実用例！

⑩Gem（カスタムボット）の仕組み

  • Gem = 決まった指示を仕込める専用チャットボット
  • 知識欄に参考ファイル（テンプレート等）を添付可能
  • カスタム指示にプロンプトを仕込んで再利用
  • 毎回プロンプトを書き直す手間がなくなる
特定の仕事専用のカスタムチャットボットのようなイメージ！

⑪テンプレートは画像ファイルがおすすめ

  • PDFテンプレートは画像ファイル（JPEG）に変換して知識欄に添付
  • 画像の方がGeminiのデザイン再現精度が高くなる傾向
  • テンプレート内に {請求企業名} のように変数を設定
  • 変数部分はスプレッドシートから都度読み取った値に置き換わる
検証結果: PDFそのままよりJPEGの方が見た目の再現度が高い！

⑫@Workspaceでスプレッドシートを参照

  • プロンプトの冒頭に@Workspaceと書くことでデータ参照が可能
  • スプレッドシートを直接読みに行ってくれる
  • プロンプト内にスプレッドシートIDを指定
  • @Workspaceを書かなくても必要に応じて参照してくれるが、書いた方が確実
スプレッドシートのIDは自分の環境に合わせて書き換えて使うのがポイント！

⑬変数の仕組み: 動的データと固定値

  • 動的変数: 請求先名、購入品、金額 → スプシから自動取得
  • 固定値: 発行日、自社名 → カスタム指示の中で直接指定
  • テンプレートでは {変数名} の形式で変数を定義
  • 生成されたPDFの明細・合計金額がスプシのデータと完全一致を確認
消費税10%の計算から合計まで自動で正確に出力される！

⑭Canvas機能でスプシをインライン表示

  • Canvas機能を使ってGemini画面内にスプシを表示
  • 表示だけでなく、そのまま編集やフィルター操作も可能
  • iframeタグを使ったHTMLでスプシを埋め込む仕組み
  • GemのデフォルトツールでCanvasをオンにしておくのが重要
タブを行ったり来たりしなくて済むから、地味にストレスが減る！

⑮2段階プロンプトの仕組み

  • ステップ1（起動トリガー）: 空のまま送信 → Canvasにスプシを表示
  • ステップ2（PDF生成）: 条件を指定 → @Workspaceでスプシ参照 → PDF作成
  • 重要: PDF生成時はCanvasを一旦オフにする（混同を防ぐため）
  • Canvasをオフにしてもスプシは消えず、後から再表示可能
CanvasがオンのままだとGeminiが混乱してPDF生成に失敗することがあるよ！

⑯複数PDF同時生成の可能性と注意点

  • 複数の会社をまとめて指定すると、会社ごとに別々のPDFを生成可能
  • 注意: 複数同時生成は動作が不安定になることがある
  • 3月と指定したのに4月分も作られたり、明細が間違うことも
  • 公式では1プロンプト1ファイルがルールとされている
複数同時生成は今後の改善に期待！今は1社ずつが安全。

⑰実務での推奨運用方法

  • 最初は1社ずつ作成するところから始める
  • 生成されたPDFは必ず目視で確認する
  • 元データのスプレッドシートと明細が一致しているか検証
  • 安定性を確認してから段階的に業務に組み込む
AIは便利だけど、最後は人間の目が必須。まずは確認作業から！

⑱まとめ: Geminiファイル生成で変わる働き方

  • プロンプト1行でWord・PDF・Excelなど各種ファイルが作成可能
  • 動画からスクショ付きマニュアルの一撃作成が実現
  • スプシデータからの請求書PDF自動化で経理作業が大幅に効率化
  • Gem + Canvas機能の組み合わせで強力なワークフロー自動化が可能
事務作業が一気に楽になる、まさにGeminiの大進化！これからの進化も期待！
