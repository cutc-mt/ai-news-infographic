③Software 3.0とは何か

  • Software 1.0: 人間が明示的にロジックを書く従来のプログラミング
  • Software 2.0: データから学習するニューラルネットワーク（画像認識など）
  • Software 3.0: LLMにプロンプト・マークダウン・知識文書を読み込ませる新しい開発手法
例: Bashスクリプトからマークダウンスキルへ
環境設定やインストール作業をエージェントが自動で試行錯誤して実行できるようになった

  • 非エンジニアの人たちが続々とソフト開発を始めている
  • バイブコーディングは底を広げた存在＝参加のハードルを下げた
  • 1年前はまだ困難だったが、現在はかなり高品質な結果が出る
やる人とやらない人で大きく2つに分かれているけど、やる人はすごい勢いで色んなものを作り始めている！
エージェンティックエンジニアリングとは

⑤エージェンティックエンジニアリングの定義

  • 複数のAIエージェントを組み込んだ作業ワークフローを設計する技術
  • 人間が仕様・検証・品質責任を握ったまま成果物を作る
  • マークダウン形式でエージェントの振る舞いを定義し、制御できる
  • プロ品質の成果物を作れるようになったことが大きな変化

⑥バイブコーディングとの違い

  • エージェンティックエンジニアリングならプロ品質の成果物が可能
最近はバイブコーディングという感じはしない。マークダウンで振る舞いを定義できるので、おかしなことになりにくい！

  • 記事作成でもAI活用で数倍の生産性向上を実感（まだ10倍には未達）
エージェントを複数繋げて様々な作業をさせることで、生産性の桁が変わる

AIの能力のムラと人間の役割

⑨AIが得意なこと

  • 強化学習で鍛えやすい領域
  • 数学・コーディングなど正解を検証できる作業
  • 調査・要約・比較・コード生成・論点整理
  • 検証可能な領域では報酬を与えやすく、モデルの能力が急速に伸びる
ポイント: テスト可能な領域ではAIの能力が爆発的に向上する

⑩AIの「能力のムラ」（Jagged Intelligence）

  • ものすごいことができる反面、どうしようもないこともできない
  • Karpathy氏の例: 「50m先にカーウォッシュがある」のに「歩いて行くべき」と回答
  • 高速道路を助手席から見ているようなムラのある能力
カーウォッシュに行くのに歩く必要ある？こんな簡単なことでもAIは間違えることがある！

⑪人間とAIの役割分担

AIに任せる
調査・要約・比較
コード生成・論点整理
人間の責任
仕様・設計・判断
美意識・品質管理
  • 強化学習で報酬を与えて勝手に良くするということができない領域がある
  • しかし、この役割分担は固定ではない（評価環境が整えば変わりうる）

⑫人間の脳とAIの脳は根本的に違う

  • 動物の脳: 進化の過程で形成された感情・欲求・好奇心・恐怖・身体感覚に基づく
  • 機械の脳: 事前学習で大量データを読み込み、強化学習で後付けで機能を持たせる
  • AIには恐怖心がない。間違いを指摘されても「そうですか」としか反応しない
AIに「もっと頑張れ」と言っても頑張らない。怒っても意味がない。全然違う脳なんだ！
「理解」というボトルネック

⑬「思考」は外注できるが「理解」は外注できない

  • AIが調査・要約・比較・実装をできるようになればなるほど...
  • 人間の理解がボトルネックになる
  • Karpathy氏: 「私はまだの一部ですし、情報は何らかの形で私の脳に入らなければなりません」
AIがリサーチを一瞬で終えても、それを正確に理解するのに人間は1〜2時間かかってしまう。このアンバランスがボトルネック！

⑭責任を持つことの重み

  • AIが作成した文章を自分が責任を持って公開するためには全てを理解しなければならない
  • 間違った評価を受けたら仕事がなくなるかもしれないという恐怖が人間にはある
  • AIには恐怖心がないので、間違いを指摘されても「すいません」と口先だけで済ませる
  • AIが出したものを正確に理解するのに1〜2時間かかるのが現実

⑮理解がないと次の方向性も決められない

  • 自分の理解がなければ次に何をすべきか判断できない
  • 「何が重要なニュースか」を判断するには、何が起きているかを理解している必要がある
  • AIは全く違う潮流を出してくる。人間のフィルターが必要
AIについていくためには、どんどん勉強しないといけない時代になった

⑯LLMナレッジベースで理解を加速

  • 従来のナレッジベースはキーワード検索のみだった
  • LLMナレッジベースはデータの共通性やパターンを認識し、意味を理解してくれる
  • 理解が早くなる → 人間のボトルネックが解消される
  • 個人の理解支援ツールとして、自分専用の「教科書」を作ることも可能に
子供の試験の成績を入れると、その子専用の学習アプリを作って楽しく学べるようになる！LLMで学びの仕組みも進化している
エージェントネイティブエコノミーへ

⑰エージェントネイティブエコノミーへの移行

  • エージェントが読めるような文章に設計する必要がある
  • エージェント中心に設計していくことがこれからの標準になる
  • RAGからエージェント型サーチへ。社内の文章検索も横断的に分析できるように
バイブコーディング エージェンティック エージェントネイティブ経済
少し前まで「RAGどうする？」が話題の中心だったのに、もう別のフェーズに入っている。変化のスピードがすごい！

⑱AI大手の動きを読まないと未来は予測できない

  • AI大手（Anthropic、Google、OpenAI）は経済的価値が高い領域から順に強化学習を進める
  • コーディングが一番経済的価値があるので、まずそこから始めている
  • AI大手が方針を変えれば、残る仕事の予測も変わる
  • 「AIにはこれができない」と決めつけるのは危険。過去に何度も覆されている
未来を読むには: AI大手が次に何を訓練するのかを見ることが不可欠。今のAIの苦手なことだけを見て未来を予測してはいけない。

⑲まとめ: Karpathy氏が伝えたかったこと

  • バイブコーディングは入り口に過ぎない。エージェンティックエンジニアリングが本番
  • AIは検証できる領域で急速に強くなる。検証できない領域は今は人間の役割
  • 役割分担は固定ではない。評価環境が整えばAIの得意領域は広がる
  • 理解して指揮する力は人間に残る。進化的経験をAIはしていないから
  • LLMナレッジベースが理解のボトルネックを解消する鍵になる

⑳解説者の所感

  • 「AIにはこれができない」と決めつける人が多いが、未来は読めない
  • AI大手の最新の動きを追っていない人が未来を予測するのは不可能
  • 「人間の理解がボトルネック」という視点が非常に面白かった
  • AI歴25年のエンジニアも「AIにはできない」と決めつけるのは危険と実感
  • 10年前は「AIに東大の問題は解けない」と言われていた時代から劇的な変化
10年前に「人間は読解力を身につけないと」と言われていたのは何だったんだろう...（笑）
