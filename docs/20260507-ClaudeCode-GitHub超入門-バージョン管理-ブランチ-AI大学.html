第一章：GitHubのしくみ

①GitHubとは何か

  • GitHubはバージョン管理システムと呼ばれるサービスです
  • ファイルの変更履歴を丸ごと記録・管理してくれるため、いつ・どのように変更したかを後から確認しやすくなります
  • チーム開発には欠かせないツールとして広く使われています
  • 最近ではClaude Codeと連携して、GitHub上のファイルを自動でレビューしてくれる機能も実装されています
Claude Codeを使いこなすなら、GitHubはぜひ導入したいツールです！

②GitとGitHubの違い

  • Git ＝ 自分のパソコンの中でファイルの変更履歴を管理する仕組み
  • GitHub ＝ Gitで管理しているデータをインターネット上で保存・共有できるクラウドサービス
  • この2つを組み合わせることで、チームでの開発が進めやすくなります

Git

ローカル（PC内）で履歴を管理する仕組み

GitHub

クラウド上で履歴を共有・管理する場所

よく混同されがちですが、役割は異なります！

③ローカルリポジトリとは

  • Gitは自分のパソコン内でファイルの変更履歴を丸ごと記録・管理してくれます
  • ファイルや履歴をまとめて管理する場所をリポジトリと呼びます
  • パソコン内に存在するためローカルリポジトリとも言われます
  • Gitを使うことで、好きなタイミングで過去の状態に巻き戻すことも可能です

④コミットとは

  • Gitを使うとゲームのセーブのように、ある時点の状態を記録して残すことができます
  • その記録のことをコミットと呼びます
  • コミットすることで変更履歴がローカルリポジトリに記録され、いつ何を変更したかを後から振り返れます
ゲームのセーブポイントみたいなもの！
いつでもその時点に戻れます

⑤リモートリポジトリとは

  • GitHubはローカルリポジトリの内容をインターネットのクラウド上に保存するサービスです
  • クラウド上で管理する場所をリモートリポジトリと呼びます
  • リモートリポジトリにより、チームメンバーとデータを共有・管理できるようになります

⑥ブランチの仕組み

  • ブランチは日本語で「枝」という意味です
  • チームの1人1人が枝分かれするように作業を分けて進める仕組みです
  • 元の完成版はそのまま残しつつ、新しい機能の追加や不具合の修正ができます
  • 途中で失敗しても元の安定した状態に影響を与えにくいのが特徴です
本番データを守りながら安全に別作業を進めるための仕組みです！

⑦マージとは

  • ブランチ上での作業が終わったら、変更内容を元のブランチに合流させます
  • この合流のことをマージと呼びます
  • ブランチは本番のデータを守りながら安全に別作業を進めるための仕組みです 
ブランチ（枝）で作業
        ↓
  作業が完了
        ↓
マージ（合流） → 元のブランチに統合

⑧第一章のまとめ

  • Gitは自分のパソコン内でファイルの変更履歴を記録・管理する仕組み
  • GitHubはその履歴をインターネット上で保存・共有できるクラウドサービス
  • チームで作業する際は、各々が自分のPC上で安全に作業を進めてからGitHubにアップロードして合流させます
この流れを抑えておけばGitHubの基本はばっちりです！

第二章：GitHubの登録方法

⑨GitHubアカウントの作成

  • 前提として、パソコンにGitを導入しておく必要があります
  • GitHubには有料プランもありますが、個人で使うなら無料プランでも十分です
  • GitHubの公式サイトで「Sign up」ボタンを押して登録を開始します
  • Google、Appleアカウント、またはメールアドレスを使ってサインアップします
  • 他のユーザーと被らないユーザーネームと居住国を設定します

⑩ユーザー名とメールアドレスの設定

  • Gitをインストールした際に入力したユーザー名とメールアドレスをGitHubの登録情報に合わせる必要があります
  • 異なっている場合は、再度ユーザー名とメールアドレスを設定し直す必要があります
  • GitHubのユーザー名は、ログイン後の画面の右上アイコン → Settings で確認できます
  • メールアドレスはEmailsセクションで確認できます
合わせないとうまく動かないことがあるので注意！

第三章：Claude CodeでGitHubを使う ～基本操作～

⑪リモートリポジトリの作成

  • GitHubにログイン後、左側のCreate repositoryボタンを押します
  • Repository nameにリモートリポジトリの名前を入力します
  • Public（誰でも見れる）かPrivate（自分のみ）かを選びます
  • Create repositoryボタンを押して完了です
今回はPrivateで作成して進めます

⑫GitとGitHubの接続方法

  • まずローカルにフォルダーを作り、git initコマンドでローカルリポジトリ化します
  • ファイルをコミットしておきます。最初のコミットでmasterブランチが自動作成されます
  • リモートリポジトリページのHTTPSのURLをコピーします
  • Claude Codeでgit remote add originの後にURLを入力して接続します 
# ローカルリポジトリの初期化
git init

# リモートリポジトリと接続
git remote add origin https://github.com/...

⑬Git Pushでアップロード

  • GitのデータをGitHubにアップロードすることをプッシュ（Push）と呼びます
  • git push origin masterコマンドでローカルの内容をリモートに反映させます
  • 初回はGitHubへのログインとメール認証が求められます
  • プッシュが完了すると、GitHub上にローカルのファイルが反映されます 
# GitHubにプッシュ
git push origin master

# → GitHub上にファイルが反映！

第四章：Claude CodeでGitHubを使う ～ブランチ編～

⑭ブランチの作成と切り替え

  • git branchの後にブランチ名を入力すると、新しいブランチが作成されます
  • git switchの後にブランチ名を入力すると、そのブランチに切り替えできます
  • ブランチ上でファイルに変更を加えてコミットします
  • git push originの後にブランチ名を入力してGitHubにプッシュします 
# ブランチの作成
git branch branch-test

# ブランチの切り替え
git switch branch-test

# GitHubにプッシュ
git push origin branch-test

⑮プルリクエストとは

  • プルリクエスト（Pull Request）とは、チームメンバーにデータチェックを依頼することです
  • 他のメンバーが変更内容をレビューして、問題ないかを確認します
  • レビューの際、タイトル・変更内容・レビューして欲しい内容を記載します
  • レビューする人を指定して、Create pull requestボタンを押します
チーム開発では、自分の変更を他の人にチェックしてもらうのが基本フローです！

⑯コードレビューとマージ

  • データをチェックすることをコードレビューと呼びます
  • レビューが完了し問題なければ、Merge pull requestボタンを押してマスターに統合します
  • マージ後、不要になったブランチはDelete branchボタンで消去できます
レビュー → マージ → ブランチ削除の流れが基本パターンです！

⑰ローカルの更新とクリーンアップ

  • リモートで更新されたマスターブランチをローカルに反映するにはgit pullを使います
  • git pull origin masterでリモートの更新内容をローカルに反映します
  • 不要になったローカルブランチはgit branch -dで削除できます 
# マスターブランチに移動
git switch master

# リモートの更新を取得
git pull origin master

# 不要なブランチを削除
git branch -d branch-test

⑱Claude Codeで日本語でGit操作

  • Claude Codeなら日本語で指示するだけでGit操作ができます
  • 「マスターブランチに移動して」「このブランチをGitHubにプッシュして」と指示するだけで対応してくれます
  • 他の人のリポジトリをダウンロードするにはgit cloneコマンドを使います
  • プロジェクトによってはSSHでの接続が必須の場合もあり、その際はPC上で鍵を作成する作業が必要です
  • Claude Codeにはプルリクエストを自動レビューする機能も実装されています（一部プランのみ）
コマンドを覚えなくても日本語で指示すればOK！
Claude Codeがよしなにやってくれます
