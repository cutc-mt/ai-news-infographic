Code with Claude 開発者会議まとめ

Anthropicが描く、AIコーディングの未来とは

📅 2026年5月7日 | 🎬 WorldofAI

会議の概要

① Code with Claude 会議の全体像

  • Anthropicが開催したCode with Claude開発者会議。AIコーディングエージェント、エージェントワークフロー、そしてソフトウェアエンジニアリングの未来に焦点を当てた大規模イベント。
  • ライブデモ、新しいClaudeの機能、そして今後の大きな進歩のプレビューが披露された。
  • 会議全体を通じて最大のテーマは長期的なエージェント知能の実現だった。
シンプルなチャットボットから、完全に自律したソフトウェアエンジニアリングシステムへ。Anthropicの野望が見えた会議でした！

② 会議の主要な6つの発表

  • 次世代モデルの3つの開発焦点を発表
  • Claude Codeのレート制限を全プランで2倍に引き上げ
  • SpaceXとの巨大なコンピュート提携を発表
  • マルチエージェントオーケストレーション機能を導入
  • ドリーミング機能でエージェントが自己学習する仕組み
  • Webhookサポートで外部ツール連携を強化

③ Anthropicが目指す未来のビジョン

  • チャットボット → 完全に自律したソフトウェアエンジニアリングシステムへの移行を目指す。
  • エージェントが過去の経験から学習し、長期的なタスクを自律的に遂行する世界を描いている。
  • 複数の専門エージェントが協調して動く「一人の開発チーム」のような存在を目指す。
単なるAIアシスタントではなく、自律的に動く「開発組織」を作ろうとしているのがAnthropicの戦略です。

④ 次世代モデルの3つの開発焦点

  • 高い判断力とコードのセンス：ベンチマークの数字だけではなく、実際のアーキテクチャ理解と保守性を重視。
  • 無限コンテキストウィンドウ：リポジトリやプロジェクト、セッションをまたいで大量の記憶を保持し、推論できる。
  • マルチエージェント協調：リードエージェントが専門エージェントに並列でタスクを委譲する仕組み。
この3つが揃えば、Claudeは「何も忘れない一人の開発チーム」になります！

次世代モデルの詳細

⑤ 高い判断力とコードのセンス

  • これまでのAIモデルはベンチマーク性能で評価されてきたが、次世代モデルは違うアプローチをとる。
  • アーキテクチャの理解と保守性を実際に評価できるモデルを目指す。
  • シニアエンジニアのような「コードのセンス」を持つモデルを目指しており、単なる性能競争から脱却する方向性。
ベンチマークの数字で競う時代が終わり、実際の「エンジニアリング力」で評価される時代が来るかもしれません。

⑥ 無限コンテキストウィンドウとは

  • エージェントがリポジトリ全体、プロジェクト、セッション、ツールにまたがる膨大な長期記憶を保持・推論できる。
  • コンテキストを頻繁に失うことなく、一貫した処理が継続可能に。
  • 大規模なコードベースでも、全体を把握した上で的確な提案や修正ができるようになる。
「忘れないAI」が実現すれば、長期間にわたるプロジェクトでも最初から最後まで一貫したサポートが可能です！

⑦ マルチエージェント協調の仕組み

  • リードエージェントが全体を統括し、タスクを適切に振り分ける。
  • 専門エージェントがフロントエンド、バックエンド、デバッグ、リサーチなど並列で作業。
  • 複雑なタスクを効率的に分割し、並行処理することで大幅なスピードアップを実現。
リードエージェント＝プロジェクトマネージャー、専門エージェント＝各分野のスペシャリスト、という役割分担です。

⑧ Claude 5のリリース時期は？

  • 予測市場Poly Marketでは、Claude 5は9月頃にリリースされるという予測が優勢。
  • 5月や6月のリリースを示唆する情報はなく、まだ先になりそう。
  • SonnetとHaikuモデルは長期間アップデートされていないため、新モデルへのアップグレードも期待される。
  • 次期モデルは「Claude 5」ラインナップなのか、噂の「Mythos」システムなのかは未確認。
早ければ9月、でもSonnetとHaikuのアップデートが先かも？待ち遠しいですね。

レート制限とインフラ強化

⑨ Claude Codeのレート制限が2倍に

  • 全有料プラン（Pro、Max、Teams、Enterprise）で5時間あたりのレート制限が2倍に引き上げられた。
  • Pro・Maxユーザーのピーク時間帯の制限削減も撤廃された。
  • Claude OpusのAPI制限も大幅に引き上げられ、大規模なエージェントワークフローを構築する開発者に大きな恩恵。
ついにClaude Codeを再び使えるように！コミュニティのフィードバックに応えた嬉しい対応です。

⑩ SpaceXとのコンピュート提携

  • AnthropicがSpaceXのColossus 1データセンターの全コンピュート能力を利用する契約を締結。
  • 翌月中に300メガワット以上のコンピュートパワーと22万枚以上のNvidia GPUにアクセス可能に。
  • この新たな能力により、Claudeユーザーの可用性とパフォーマンスが直接向上する。
22万枚のGPUは規格外の規模！これがレート制限緩和の背景にあります。

⑪ マルチギガワット級のインフラ提携

  • Anthropicは複数のマルチギガワット級のコンピュート契約を進行中。
  • 提携先にはGoogle、Amazon、Broadcom、Microsoft、Nvidia、Fluid Stackが含まれる。
  • AIインフラの大規模拡張により、モデルの学習から推論まで全フェーズでの能力強化を狙う。
ビッグテック各社と広く提携することで、Anthropicはコンピュートリソースの寡占リスクを回避する戦略です。

⑫ 軌道上AIコンピューティング構想

  • Anthropicが将来的にSpaceXとの軌道上AIコンピュートインフラの開発に興味を示している。
  • 地球上のデータセンターに依存しない、宇宙空間でのAIコンピューティングの可能性。
  • 現時点では「興味」の段階だが、将来的な構想として非常に野心的。
宇宙でAIを動かすなんてSFの世界！でもAnthropicとSpaceXなら実現するかも？

エージェント機能の進化

⑬ マルチエージェントオーケストレーション

  • リードエージェントが全体を統括し、複数の専門エージェントを並列で動かす仕組みを導入。
  • 複雑なジョブを複数のエージェントに分割し、それぞれが並行処理で作業を進める。
  • フロントエンド、バックエンド、テスト、リサーチなど、各分野の専門エージェントが協調動作。
人間の開発チームと同じように、役割分担して並行で作業できるようになります。

⑭ アウトカムループシステム

  • エージェントが自身の出力を事前定義されたルーブリックに対して評価する仕組み。
  • 反復的な自己修正システムにより、継続的に結果を改善していく。
  • 長期間のタスクでも、品質を自動的に担保しながら遂行できるようになる。
自分で品質をチェックして直す→またチェックする、このループで精度がどんどん上がっていく仕組みです。

⑮ ドリーミング機能で自己学習

  • エージェントが過去のセッションや経験を振り返り、次回のパフォーマンスを改善する機能。
  • 以前の失敗から学習し、将来の意思決定と結果を向上させる。
  • 時間を超えた学習が可能になり、エージェントが長期的に賢くなっていく仕組み。
人間が寝ている間に夢を見て学習するように、AIも「ドリーミング」で成長するなんて面白い発想！

⑯ Webhookサポートの追加

  • Claudeエージェントを外部ツールやアプリに簡単に統合できるようになった。
  • 自動化ワークフローへの組み込みがよりシームレスに。
  • CI/CDパイプラインやプロジェクト管理ツールなど、既存の開発環境との連携が強化。
WebhookでGitHubやSlackなどと連携すれば、開発フロー全体をClaudeで自動化できます。

まとめと今後の展望

⑰ Claude Managed Agentsの進化

  • 以前はシンプルなAIチャットボットだったものが、自律的な恒久的なワークフォースシステムへと進化しつつある。
  • 長期間稼働するタスクを自律的に管理・遂行できるようになった。
  • マルチエージェント協調、アウトカムループ、ドリーミング機能が組み合わさり、圧倒的に高度な自律システムに。
Managed Agentsは以前から存在していましたが、今回の機能追加で一気に実用的になりました。

⑱ コミュニティの反応と期待

  • 直近ではモデル品質の低下や厳しいレート制限に対するコミュニティの不満が大きかった。
  • しかし今回のレート制限緩和と新機能発表により、コミュニティの意見を聞き入った姿勢が評価されている。
  • 次世代モデルの高い判断力や無限コンテキストへの期待が高まっており、今後の復調を望む声が多い。
  • 「 Anthropicが方針を変え、コミュニティに耳を傾け始めた」と、評価の変化が見られる。
ユーザーの声に応え始めたAnthropic。次世代モデルでの「恩返し」に期待が集まっています。
