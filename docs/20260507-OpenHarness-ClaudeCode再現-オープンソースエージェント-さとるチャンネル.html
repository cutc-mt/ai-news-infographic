OpenHarness - Claude Codeの97%を3%のコードで再現

香港大学が公開したオープンソースのAIエージェントフレームワーク

2026年4月8日 | github.com/HKUDS/OpenHarness

①Claude Codeを自作できる時代が到来

  • AnthropicのClaude Codeは強力だが、中身がどう動いているかはブラックボックス
  • コードを編集したりファイル操作したりする仕組みが一切見えない
  • 香港大学データサイエンス研究室（HKUDS）がその仕組みをオープンソースで公開
  • MITライセンスで完全無料、商用利用も可能
ブラックボックスだったエージェントの内部構造が、コードレベルで全部見れる時代に！

②OpenHarnessとは何か

  • 香港大学HKUDSが開発したオープンソースのエージェントフレームワーク
  • 普通のLLMをClaude Codeのような自立エージェントに変換できる
  • Claude以外のモデルでも動作するマルチプロバイダー対応
  • curlコマンド1行でインストール完了、5分で動かせる 
# 1行でインストール
curl -fsSL https://openharness.dev/install | bash

③名前の由来「ハーネス」とは

  • ハーネスは馬につけるくつわのような制御具
  • 馬にくつわをつけると人間が乗って制御できるようになる
  • LLMも同様に、そのままだと賢いけれど何もできない存在
  • チャットGPTも「これやって」と言っても話すことしかできない
ハーネスはLLMに「手足」と「記憶」と「ルール」を与えるインフラなんです！

④設計哲学「モデルがエージェント、コードはハーネス」

  • 手足＝ファイルの読み書き、シェルコマンドの実行、Web検索などの行動能力
  • 記憶＝メモリ管理。どこまで作業したか、過去に何を学んだかを保持
  • ルール＝権限管理。システムファイルを勝手に削除しないように防ぐ
  • この3つがないと、どんなに賢いLLMでも実用的なエージェントにはならない
脳みそはあるけど体がない状態 → ハーネスがその体を提供する！

⑤中核機能①エージェントループ

  • 思考→ツール呼び→結果受信→次の判断のサイクルを自動で繰り返す
  • 人間が毎回指示しなくても、自分で判断してタスクを進める
  • 例：「バグ直して」と頼むと → コード読む → 怪しい場所特定 → 修正 → テスト → 報告
  • この一連の流れがすべて自動で完結する 
# エージェントループの流れ
while not task_complete:
    thought = model.think(context)
    action = select_tool(thought)
    result = execute(action)
    context = update(result)

⑥エラーハンドリングとコスト追跡

  • 指数バックオフで自動リトライ。1秒→2秒→4秒と待ち時間を倍増
  • サーバーが混んでいても諦めずに待機してリトライを継続
  • ストリーミング型のツール呼び出しをサポート
  • 全結果を待つのではなく、リアルタイムで処理を進めて体感速度が向上
  • APIの利用量とコストを自動追跡して課金爆発を防止
コスト管理もバッチリ！課金爆発を防げるのは安心ですね

⑦中核機能②ツールキット（43+ツール）

  • ファイルI/O：ファイルの読み書き、コードの編集
  • シェルコマンド：ターミナルコマンドの実行
  • Web検索：インターネット上の情報を検索
  • MCP対応：外部サービスとの連携を標準化
  • Git操作：バージョン管理の操作も可能
43個ものツールが最初から内蔵。コーディングから検索まで何でもできる！

⑧MCP対応とオンデマンドローディング

  • MCP（Model Context Protocol）は外部サービス連携の標準規格
  • SlackやGitHubなど、サービスのAPIを統一的に扱える
  • 43個のツールはオンデマンドローディングで必要な時だけ読み込む
  • 使わないツールでメモリを消費しない、実用的なエージェントに欠かせない設計
必要な時だけ読み込むから軽量・高速！メモリの無駄遣いなし

⑨中核機能③メモリ管理 - CLAUDE.md

  • CLAUDE.md：プロジェクトのルートに置くと自動で読み込まれる
  • 本家のClaude Codeと同じ仕組み
  • プロジェクト固有のルールや設定を記述（使用ライブラリ、言語など）
  • 毎回「うちはTypeScriptを使ってる」と言わなくて済む 
# CLAUDE.mdの例
- このプロジェクトはTypeScript + React
- テストはJestを使用
- ESLintの設定はstrict mode

⑩永続メモリ - MEMORY.md

  • MEMORY.mdはセッションをまたいで保存される永続メモリ
  • エージェントが学習したことを記録できる
  • 例：「データベース接続処理がある」「APIの構造情報」などを記録
  • 次に同じプロジェクトで作業する時に調べ直さなくて済む
  • 長くなりすぎたら自動で要約してコンテキスト窓に対応
人間のメモ帳みたいなもの。一度覚えたら忘れない！

⑪ガバナンス - 権限管理の3段階

  • 許可：ファイルを読むなど安全な操作は自動で実行
  • 確認：シェルコマンドなどは実行前にユーザーに確認
  • 拒否：rm -rfなどの危険な操作は完全にブロック
  • 操作ごとに個別に権限レベルを設定できる
  • パス規則も設定可能（特定のパスは触れないようにするなど）
AIが暴走しても怖くない！システムファイルを守れるのは安心

⑫プレ/ポストツールフック

  • プレツールフック：ツール呼び出し前にカスタム処理を挟める
  • ポストツールフック：ツール実行後にカスタム処理を挟める
  • 例：Git Push前にメインブランチかどうか確認するチェックを入れられる
  • どのファイルを変更したかをすべて記録する監査ログも可能
  • 企業で使う際の監査要件にも対応できる設計
事前に危険を検知 + 事後のログ取得も可能。安全な運用設計！

⑬中核機能④マルチエージェント調整

  • メインエージェントがサブエージェントを自動生成
  • 大きなタスクを専門家に分担する仕組み（自分で部下を作るイメージ）
  • 例：Webアプリのテスト → フロントはA、バックエンドはB、セキュリティはC
  • 依存関係がなければ同時並列で処理できる
  • Claude Codeの並列呼び出しより上の、エージェントレベルの並列分担
1人で1時間の作業も3人で分担すれば20分で終わる！

⑭チームレジストリーで進捗管理

  • チームレジストリーが各エージェントの役割と進捗を一括管理
  • 誰がどのタスクをやっていて、どこまで進んでいるかを可視化
  • バックグラウンドで動いているタスクも追跡可能
  • プロジェクトのカンバンボードのようなイメージ
複雑なプロジェクトでも破綻しない！進捗が一目で分かる

⑮対応プロバイダー - マルチモデル対応

  • 2つの互換レイヤーで幅広いモデルに対応
  • Anthropic互換：Claude、GLM、MiniMaxなどが使える
  • OpenAI互換：GPT-4.5、O1、DeepSeekなどが利用可能
  • モデルを自由に切り替えて用途に合わせて選べる
  • 安いモデルで試して本番は高性能モデルに、という使い方も可能
コストと性能のバランスを自分で調整できる！

⑯GitHub Copilot認証にも対応

  • GitHub CopilotのOAuth認証に対応している
  • デバイスフローで認証すればCopilotの契約でエージェントを動かせる可能性
  • VS Codeで使っているあのCopilotを、エージェントとしても活用できる
  • 既存の開発環境との統合が容易
すでにCopilot契約している人なら、追加費用なしでエージェントが使えるかも！

⑰技術スタックとテスト

  • Python 3.10以上が必要、シンプルな環境
  • ターミナルUIにはReact + Inkを使用
  • InkライブラリでReactのコンポーネントとしてターミナルを描画
  • 回帰テストで114件のテストケースを完備
  • 出力形式はテキスト、JSONストリーミング、JSONに対応
