OpenHarness - Claude Codeの97%を3%のコードで再現

香港大学発・オープンソースのAIエージェントフレームワークを徹底解剖

2026年4月8日 | さとるチャンネル@毎日AIニュース

①Claude Codeを自作できる時代が到来

  • AnthropicのClaude Codeのような自立型AIエージェントを自分で作れる時代が来た。
  • Claude Codeは便利だが、中身がどう動いているかブラックボックスのまま使っている。
  • オープンソースなら中身が全部見える。仕組みを学びながら改造できる。
賢いLLMも、そのままでは「話すことしかできない」存在。 手足と記憶とルールが必要！

②香港大学発のオープンソース

  • 香港大学データサイエンス研究室（HKUDS）が公開したフレームワーク。
  • MITライセンスで完全無料。商用利用も自由に可能。
  • GitHubで公開されており、誰でもコードを読んで改造できる。
  • 普通のLLMをClaude Codeのような自立型エージェントに変換する。
curlコマンド1行でインストール完了！ 5分あれば手元で動かせる。

③設計哲学：モデルがエージェント、コードはハーネス

  • ハーネスとは馬につける轡（くつわ）のこと。馬にハーネスをつけると人間が制御できるようになる。
  • LLMも同じ。そのままだと賢いけど何もできない存在。
  • ハーネスはLLMに手足と記憶とルールを与えるインフラ。
  • 脳みそはあるけど体がない状態 → ハーネスが体を提供する。
LLM = 賢い馬 ハーネス = 制御装置 この組み合わせでエージェントが完成！

④エージェントに必要な3つの要素

  • 手足（行動能力）：ファイルを読む、シェルコマンドを実行する、Web検索するなどの行動。
  • 記憶（メモリ管理）：今どこまで作業したか、過去に何を学んだかを保持する仕組み。
  • ルール（権限制御）：勝手にシステムファイルを消さないよう、危険な操作をブロック。
  • この3つがないとどんなに賢いLLMでも実用的なエージェントにはなれない。
AIが暴走したら怖いですからね。 ルールがあってこそ安心して使える！

⑤中核機能①エージェントループ

  • 思考 → ツール → 結果受信 → 次の判断のサイクルを自動で繰り返す。
  • 人間が毎回指示を出さなくても、自分で判断して進むことができる。
  • 例：「このバグを直して」と頼むと、コードを読む → 怪しい箇所を特定 → 修正 → テスト実行 → 報告、まで全部自動。
人間が介入しなくても 自分で考えてタスクを完結できる それが「自立型エージェント」！

⑥エラー対策とストリーミング

  • 指数バックオフを搭載。最初は1秒待ってリトライ、ダメなら2秒、次は4秒と待ち時間を倍に増やす。
  • サーバーが混んでいても諦めずに待って再試行してくれる。
  • ストリーミング型のツール呼び出しもサポート。結果が来るたびにリアルタイムで処理を進める。
  • 体感速度がかなり向上。全部終わるまで待つ必要がない。

⑦APIコスト管理

  • APIの利用量とコストを自動で追跡してくれる。
  • 課金爆発を防げる。安心してエージェントを動かせる。
  • トークン単位での消費量を把握できるので、コスト最適化も可能。
安いモデルで試して本番は高性能モデルに、 という使い分けもできるよ！

⑧中核機能②ツールキット（43+ツール）

  • ファイルI/O：ファイルの読み書きや編集。
  • シェル実行：コマンドラインの操作。
  • Web検索：インターネット上の情報を検索。
  • MCP対応：モデルコンテキストプロトコルによる外部サービス連携。
  • Git操作：コミットやブランチの管理。
合計43以上のツールが最初から内蔵！ SlackやGitHubのAPIも統一的に扱える。

⑨MCPで外部サービスと統一連携

  • MCP = Model Context Protocol（モデルコンテキストプロトコル）。
  • 外部サービスとの連携を標準化する仕組み。
  • Slack、GitHubなどのサービスのAPIを統一的に扱える。
  • 新しいサービスを追加するのも、MCPに対応すればスムーズに統合可能。

⑩オンデマンドローディング

  • 43個のツールが常にすべて読み込まれているわけではない。
  • オンデマンドローディングで必要な時だけツールを読み込む。
  • 使わないツールでメモリを消費しない。効率的な設計。
この効率性が実用的な エージェントには欠かせない！ 賢い設計ですね。

⑪中核機能③メモリ管理（2種類）

  • ①CLAUDE.md：プロジェクトのルートに置くと自動で読み込まれる。プロジェクト固有のルールや設定を記述。
  • ②MEMORY.md：セッションを跨いで保存される永続メモリ。エージェントが学習したことを記録。
  • 例：DB接続処理がある、構造情報などを記録 → 次回の作業で調べ直す必要がない。
  • 長くなりすぎたら自動で要約する仕組みも搭載。コンテキスト制限にも対応。
人間のメモ帳みたいなもの。 毎回0からやり直さなくて済む！

⑫マルチレベルの権限管理

  • 3段階の権限：許可・確認・拒否を操作ごとに設定できる。
  • ファイルを読むのは「許可」、シェルコマンドは「確認が必要」。
  • rm -rf は「拒否」のように危険な操作をブロック。
  • ルールベースの規則も設定可能。例：「システムファイルは触れない」。
システムファイルを守れるのは安心。 企業での運用でも安全第一！

⑬プレ/ポストツールフック

  • Pre-Tool Hook：ツールを呼ぶ前にカスタム処理を挟める。
  • Post-Tool Hook：ツール実行後にカスタム処理を挟める。
  • 例：git pushの前にメインブランチではないか確認するチェックを入れられる。
  • どのファイルを変更したかを全部記録する監査ログも可能。
企業で使う時や監査が必要な ケースに対応できる設計！

⑭中核機能④マルチエージェント調整

  • メインエージェントが必要に応じてサブエージェントを生成できる。
  • 例：Webアプリのテスト → フロントエンドはA、バックエンドはB、セキュリティチェックはCに分担。
  • 依存関係がなければ並列で同時処理が可能。
  • Claude Codeの並列呼び出しより上のエージェントレベルの並列分担。
自分で部下を作るみたい！ 1人で1時間 → 3人で20分に短縮できる。

⑮チームレジストリーで進捗管理

  • チームレジストリーで各エージェントの役割と進捗を一元管理。
  • 誰が何をやっていて、どこまで進んでいるかが一目でわかる。
  • バックグラウンドで動いているタスクも追跡可能。「あれどうなった？」がすぐ解決。
プロジェクトのカンバンボードみたい！ 複雑なプロジェクトでも破綻しにくい。

⑯マルチプロバイダー対応

  • Anthropic互換レイヤー：Claude、GLM、MiniMaxなどが使える。
  • OpenAI互換レイヤー：GPT-4.5やo1などもエージェント構築に利用可能。
  • モデルを自由に切り替え可能。安いモデルで試して本番は高性能モデルに、という使い方も。
  • GitHub CopilotのOAuth認証にも対応。VS Codeの契約でエージェントを動かせる可能性も。
既存の開発環境との連携がしやすい！ Claudeだけじゃないのが良いですね。

⑰技術スタック

  • Python 3.10+が必要。Pythonベースのフレームワーク。
  • ターミナルUIにReact + Inkを使用。InkライブラリでReactコンポーネントとしてターミナルを描画。
  • 回帰テストで114件のテストを搭載。品質もしっかり管理。
  • 出力形式：テキスト / JSONストリーミング / JSONに対応。他システムとの連携も容易。
ターミナルアプリなのにReact？ Inkのライブラリが面白い仕組みを実現！

⑱まとめ：エージェント自作の未来

  • curlコマンド1行でインストール完了。依存関係の処理もインストールスクリプトが全部やってくれる。
  • ブラックボックスだったエージェントの内部構造がコードレベルで全部見える。学習教材としても優秀。
  • 学んで改造して、自分だけのエージェントを作れる。さらにコミュニティで共有して発展。
  • MITライセンスなので、企業が自社向けにカスタマイズして使うことも想定。
GitHubで「OpenHarness」と検索！ エージェント開発はこれから どんどん盛り上がりそうですね。
