知っておくと生産性が爆上がりする実用的なTips集

2026年5月8日  |  Shin Coding Tutorial

①

Claude Codeの
最新バージョンを確認

  • まずは最新版かどうかを確認することが重要です
  • ターミナルでClaudeを一度閉じてから確認コマンドを実行します
  • 差分がある場合はインストールコマンドで最新版に更新できます
ターミナル
# バージョン確認
claude update

# 最新版に更新
claude install
頻繁にアップデートされているので、毎回確認する癖をつけるのがおすすめです！
②

/effortで
努力量を調整する

  • /effortでClaude Codeの出力の努力量を決定できます
  • デフォルトはhighですが、xhighが推奨のバランス設定です
  • 仕様書の作成や複雑な実装ではmaxが効果的です
  • 出力スピードとインテリジェンスのバランスを調整できます
デフォルトでxhighに設定しておくのがおすすめ。複雑な実装時だけmaxに切り替えよう！
③

/modelで
モデルを確認・変更

  • /modelで現在使用中のモデルを確認できます
  • デフォルトはOpus 4.7で100万トークンのコンテキストが利用可能です
  • 複雑な要件はOpusで、実装だけはSonnetに変更する使い分けも可能です
トークンを節約したい場合はSonnetに切り替えるのもアリ。でも基本的にはOpusで十分です！
チャット＆セッション管理
④

/btwで
待ち時間に質問

  • Claude Codeの処理を待っている間に/btwで質問ができます
  • コンテキストを汚染することなく質問できるのが最大の特徴です
  • GitHubワークフローについて教えて、などちょっとした学習に最適です
  • 上下キーでスクロールでき、ESCで終了します
処理待ちの時間を有効活用して学習できるので、非常によいコマンドです！
⑤

/resumeで
セッションを引き継ぐ

  • /resumeで過去のセッション会話を遡って再開できます
  • 複数のClaude Codeを立ち上げる際、記憶領域を共有するのに便利です
  • /renameでセッションに名前をつけることも可能です
  • claude --continueで前回の会話から続けることもできます
毎回claudeコマンドで立ち上げると記憶がリセットされます。継続したい場合は--continueを使いましょう！
⑥

/rewindで
出力を巻き戻す

  • /rewindでコードも会話も前の状態に戻すことができます
  • ESCキーを2回押しても実行可能です
  • 期待と違う出力になった場合に、好きな時点まで巻き戻せます
デザインの変更で微妙な結果になった時などに即座に元に戻せるので超便利！
/resumeは「過去のセッションに再接続」
/rewindは「今の会話を巻き戻し」
← この違いを理解しておこう！
デザイン＆並列処理
⑦

DESIGN.mdで
デザイン方向性を指定

  • DESIGN.mdファイルでデザインの方向性をデザイントークンとして定義できます
  • getDesign.mdなどのサイトからデザイントークンを取得して活用できます
  • Resend、Linear、Vercelなど有名サービスのデザインを再現可能です
  • アプリ開発の最初に指定するのがベストプラクティスです
デザイン特化のエージェントを作って、DESIGN.mdを読み込ませるとさらに高品質に！
⑧

/batchで
並列処理を実行

  • /batchで似たような実装を並列でまとめて実行できます
  • リファクタリング、ダークモード対応など、複数ファイルへの同一実装に最適です
  • テストコードの作成も各コンポーネントに対して並列で実行できます
バッチコマンドを使うと自動でGit Worktreeが立ち上がって並列処理してくれます！
⑨

Git Worktreeで
安全に並列開発

  • Git Worktreeは「別々の部屋」を作って独立して作業する仕組みです
  • 同じ場所で作業するとリアルタイムで競合・上書きが発生するリスクがあります
  • Worktreeなら各部屋で競合なしに並列作業が可能です
  • 最終的に各部屋の成果を1つにマージする必要があります
.claude/のworktreeディレクトリ下にアプリが複製されます。npmだと重いのでpnpm推奨です！
権限＆セキュリティ
⑩

Auto Run Modeで
自動実行を有効化

  • Auto Run Modeは安全な自動実行モードです
  • 危険な操作は人間に確認し、安全な操作は自動で実行してくれます
  • /configのデフォルトパーミッションモードでAutoに設定可能です
  • Shift+Tabで随時切り替えができます
bypass-permissionsより安全で、毎回権限を聞かれる手間も省けるので最近はこれ一択です！
⑪

/fewer-permission-promptsで
権限を自動設定

  • 過去の許可履歴を分析して自動でパーミッション設定してくれます
  • 例えば「Tursoデータベースの読み取りクエリを83回許可した」などを検出します
  • settings.jsonのallowリストに自動追加してくれます
危険なコマンドはdenyに設定を！
.envファイルの読み取りや
git pushのリモート実行は必ずdenyにしましょう！
⑫

/permissionsで
きめ細かく権限管理

  • /permissionsで1つずつ許可・拒否を手動設定できます
  • deny設定推奨: .env読み取り、git pushのリモート実行
  • セキュリティに厳しい企業開発では必須の設定です
  • Add new ruleからルールを追加して、settings.jsonのdenyリストに反映されます
deny推奨コマンド
# 本番環境へのプッシュを防止
Bash: git push

# 環境変数の読み取りを防止
Read: .env
分析＆ターミナル環境
⑬

/insightsで
使い方を分析

  • /insightsでClaude Codeの使い方を分析して改善提案をしてくれます
  • うまくいっている点を評価してくれます
  • 足を引っ張っている点も具体的に指摘してくれます
  • すぐに実践できる改善案も提示してくれます
日本語でレポートを出力してくれるので、1回は必ず実行してみてください！
⑭

ccmux-cliで
ターミナルを分割

  • ccmux-cliは無料のオープンソースCLIでターミナル分割が簡単にできます
  • npm installで簡単に導入可能です
  • Claude Codeを複数立ち上げて独立した並列作業ができます
  • IDEに依存しないので軽量な開発環境を構築できます
Cursorなどのリッチなエディタは重くなりがち。ccmuxなら軽量に並列開発が可能です！
⑮

!でターミナル
コマンドを実行

  • 先頭に!をつけるとClaude Code上でターミナルコマンドを実行できます
  • !npm run devでローカルサーバーを立ち上げ可能です
  • 下キー→Enterで現在動いているプロセスを確認できます
  • Claude Codeがログを認識するので、コピペ不要でエラーを把握できます
Ctrl+Bでバックグラウンドタスクに移動できるので、並行作業もラクラクです！
応用テクニック
⑯

/planで
プランモードを活用

  • /planでプランモードに切り替えて、実装前に計画を立てられます
  • Shift+Tabでもプランモードと実行モードを切り替えられます
  • 仕様書を固める時はプランモードをオンにするのが推奨です
  • effortをmaxにしてプランモードを使うと、より深く考えてくれます
プランの承認後はYes auto modeで実行。フェーズごとの依存関係に注意して並列化を考えよう！
⑰

並列開発の
正しい使い方

  • NG: 依存関係があるフェーズを別々のClaudeで同時実行する
  • OK: 実装用と分析用など、完全に独立したタスクを並列で実行
  • 別コンテキストで立ち上げると会話履歴は引き継がれない点に注意
  • 記憶を共有したい場合は/resumeでセッションを合わせましょう
1つは実装、2つ目は分析や改善など、役割を分けて並列化するのが正解です！
⑱

まとめ: 中級者から
上級者への道

  • 今回の10選を完璧に理解すれば上級者と言っても過言ではありません
  • 他にもteam-onboarding、power-up、/loop、/scheduleなど多数存在
