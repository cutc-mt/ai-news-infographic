Verdent Manager - AI CTOが開発を自動管理する新機能

アイデアから本番デプロイまで、ワンプロンプトで完結する次世代のAIコーディング体験

2026年5月9日 | AICodeKing

① Verdent Managerとは：AI CTOの登場

  • Managerは、Verdentの新機能で、通常のエージェントワークフローの上に位置する存在です。
  • 単なるコーディングツールではなく、プロジェクト全体を統括する役割を持ちます。
  • ユーザーが「達成したいゴール」を伝えると、それを計画に変換して実行します。
  • 計画を細かいタスクに分解し、ワーカーを並列で動かして進捗を管理します。
「アシスタント」「チームメイト」「共同創業者」という言葉は色々なAI企業が使っているけど、Managerは本当にその通りに動くんです！

② 従来のAIコーディングの問題点

  • AIツールを開いて、モードを選んで、プロンプトを貼り付けて、出力を確認する…
  • 何か忘れていることに気づいて、もう一度聞き直して、モードを変えて…
  • ユーザーがプロジェクトマネージャー、QA担当、CTOをすべて兼任する状態に。
  • さらに「精神的なサポート係」までこなすことに…
従来のワークフローは、結局すべてを人間が管理しないといけないんですよね。疲れます…。

③ Managerの基本的な仕組み

  • ユーザーが成果物を伝える（ゴールの指定）
  • Managerがゴールを計画に変換する
  • 計画を小さなタスクに分解する
  • ワーカーを並列実行で動かす
  • ボード上で進捗を追跡する
  • 完了したらレビューを依頼する
「バックエンドを作って」「フロントエンドを作って」「テストを追加して」…が全部自動で進むんです！

④ 具体例：ベビー用品レコメンドアプリ

  • プロンプト例：「ベビー用品ストアのアプリを作って」
  • 親がいくつか質問に答えて、商品をレコメンドされる仕組み
  • パーソナライズされたチェックリストのためにメールを送信
  • クリーンでモバイルフレンドリーなデザイン、小規模なローンチ-ready
非技術的な創業者がアイデアを素早く検証したい場合に、最適なアプローチです。

⑤ タスクの自動分解とフェーズ分け

  • Managerはゴールを見て、自動的にフェーズに分割します：
  • 要件定義 → 何を作るかを整理
  • UI構造 → 画面レイアウトとコンポーネント
  • レコメンドロジック → 推薦アルゴリズムの実装
  • メール取得 → フォームとバックエンドの連携
  • テストと検証 → 最終的な品質確認

⑥ 並列実行で高速な開発を実現

  • ワーカー1：ランディングページとフォームのレイアウト
  • ワーカー2：レコメンドロジックの実装
  • ワーカー3：データストレージのセットアップ
  • ワーカー4：テストとクリーンアップ
1つの会話で1ステップずつ進むのではなく、複数の作業が同時に進むのでまるで小さな開発チームを率いている感覚です！

⑦ 「指示するだけ、あとは出荷」の哲学

  • あなたの仕事：ビジネスを考える
  • これが正しいオファーか？顧客は誰か？
  • 収益化は可能か？ユーザーは本当に何を必要としているか？
  • Managerの仕事：ソフトウェアを前に進める
この「ビジネスに集中、技術はManagerに任せる」という分担こそが最大の価値です！

⑧ CTO級の能力①：長期記憶

  • ほとんどのAIツールは、ユーザーのことを忘れてしまいます。
  • スタック、命名スタイル、好みを説明しても、3回会話したらまた最初から…
  • Managerの長期記憶は以下を覚えています：
  • 好みのスタック（TypeScript, Tailwind, Supabase, Vercelなど）
  • コードスタイル、過去のフィードバック、プロジェクトの詳細
使えば使うほど、繰り返しの説明が減っていくのが嬉しいポイントです。

⑨ CTO級の能力②：スキルシステム

  • スキルは、特定のワークフロー用の再利用可能な知識パックです。
  • テスト：品質保証の自動化
  • デプロイ：本番環境への公開
  • フロントエンド：UIデザインの改善
  • セキュリティ：脆弱性のレビュー
  • GitHub：ワークフローの統合

⑩ 適切な専門家を自動で呼び出す

  • Managerは状況を理解して、必要なスキルを自動で呼び出します。
  • 「QAチェックリストを使って」「セキュリティレビューをして」と指示する必要がありません。
  • これはCTOがチームを率いる方法と同じです。
  • 将来的には「Ship it」と言うだけで、スタイル・テスト・デプロイ・レビューの期待を理解して実行できるように。
1人で全部やるのではなく、適切な専門家を適切なタイミングで投入する。これが本当のチームマネジメントです！

⑪ CTO級の能力③：リモートワーク（Slack）

  • ManagerはSlackに接続して、どこからでも指示を出せます。
  • 会議中に「ダッシュボードの変更をお願い」とメッセージを送るだけでOK。
  • 「最新のPRを確認して、リスクな変更があれば教えて」という依頼も可能。
ブラウザを開いてログインして…という手間が不要です。Slackから直接エンジニアに指示する感覚！

⑫ Telegramからのリモート指示

  • お昼休みにバグを見つけたら、スマホからTelegramで報告。
  • 寝る前に「明日やるタスク」を準備しておくことも可能です。
  • 「最新版をステージングにデプロイして、完了したら報告して」という指示もTelegramで完結。
寝る前にメッセージを送って、朝起きたら完了している…未来の開発スタイルですね。

⑬ リモートワークが特に便利なケース

  • 小さなタスクの依頼
  • 検証の実行と結果の報告
  • PRのサマリーの確認
  • 社内ツールの軽微な修正
  • ランディングページの更新
  • プロトタイプの作成
もちろん、重要な変更は自分でレビューしましょう。銀行アプリを確認なしでデプロイするのはNGです！

⑭ パーソナライゼーション機能

  • Managerにニックネームとアバターを設定できます。
  • チームメンバーが「誰が応答しているか」をすぐに認識できます。
  • SlackやTelegramに導入すると、アイデンティティが重要になります。
  • ただのボットではなく、自分のビジネス向けに設定した技術パートナーとして扱えます。
見た目のカスタマイズに思えるかもしれませんが、チームに導入するなら認識性は本当に大事です！

⑮ Eco Modeでコストを節約

  • Eco Modeは、クレジット消費を抑えながら作業を続けられるモードです。
  • 選択された低コストモデルを使用します。
  • 長時間のセッションや反復的な作業、探索的なタスクに最適です。
  • コピーの調整、アイデアの探索、軽微な変更に向いています。
Eco Modeは最も高性能なモデルを使うのとは異なります。難しいアーキテクチャタスクには高性能モードを推奨します。

⑯ BYOK（独自APIキーの持ち込み）

  • BYOK = Bring Your Own Key（自分のキーを持ち込む）
  • 対応プロバイダー：Anthropic、OpenAI、OpenRouter
  • キーを追加すると、チャットモデル、プリセット、サブエージェント、レビュー設定で利用可能に。
  • 他でクレジットを持っている場合や、モデルのコストを直接管理したい場合に便利です。
一部の組み込み機能はBYOKに対応していませんが、コスト管理とモデル選択の自由度は大幅に向上します！

⑰ Verdentの複数の使い方

モード 用途
通常クレジット 高性能な組み込みモデルを使いたい時
Eco Mode 安く長く反復作業を続けたい時
BYOK プロバイダーを直接制御したい時
Manager コードだけでなく、プロジェクト全体の実行を任せたい時
この4つの組み合わせこそが、Verdentの本当のストーリーです。用途に合わせて柔軟に使い分けられます。

⑱ まとめ：Managerが変える開発の未来

  • 従来はユーザーが技術マネージャーだった。Managerはそれを反転させます。
  • ゴールを与えれば、作業を整理して、重要なことを覚えてくれます。
  • SlackやTelegramからリモートで指示を出せます。
  • 適切なスキルを自動で呼び出し、タスクを並列で実行します。
  • Eco ModeとBYOKでコスト管理も充実。
  • 技術者にとっても、プロジェクト管理の負担を大幅に軽減できる存在です。
ManagerはVerdentのこれまでのアップデートの中で、最も重要なものと言えるかもしれません！
