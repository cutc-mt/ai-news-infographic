Codex 3.0 最新アップデート完全解説

Chrome拡張 × v0.128/0.129 × エージェントワークスペースの進化

2026年5月11日 | AICodeKing

① Codex for Chrome: 最大の新機能

  • CodexがChrome拡張機能を通じて実際のブラウザを操作できるようになった
  • ログイン済みのサイト（Salesforce、Gmail、LinkedInなど）でそのまま作業が可能
  • バックグラウンドで並列タブ実行に対応し、ブラウザセッションを占有しない
  • タスクごとにChromeタブグループが作成され、整理された状態を維持
従来の「アプリ内ブラウザ」とは異なり、実際のブラウザセッションにアクセスできるのが最大の違い！

② アプリ内ブラウザ vs Chrome拡張の使い分け

  • アプリ内ブラウザは localhost確認、公開ページのテスト、ファイルプレビューに最適
  • Chrome拡張は認証が必要なサイト、社内ツール、SaaS製品での作業に最適
  • 別ブラウザで再ログインが困難な場面でChromeが圧倒的に便利
  • 用途に応じて適切なブラウザモードを選択することが推奨される
「コードを書いて、ブラウザで確認する」という開発ループがCodex内で完結するようになった

③ パーミッション制御と安全性

  • デフォルトでは新しいサイトごとに確認を求める設計
  • 許可リスト（Allow List）とブロックリスト（Block List）でホスト単位の管理
  • 許可のスコープは「今回のみ」「常に許可」「拒否」から選択可能
  • ブラウザ履歴へのアクセスには「常に許可」オプションなし（タスク内のみ）
ブラウザ履歴には社内URLや検索履歴が含まれる可能性があるため、慎重な取り扱いが重要

④ プラグイン連携でChromeを起動

  • CodexのプラグインシステムにChromeが統合されている
  • セットアップ: プラグインを追加 → 拡張機能をインストール → 権限を承認 → 接続確認
  • @Chromeコマンドで直接指示可能（例: 「Salesforceを開いてこのアカウントを更新して」）
  • Chromeが起動していない場合、自動で開いてタブグループを作成 
// 使い方の例
@Chrome Salesforceを開いて、
メモの内容からこの
アカウントを更新して

⑤ 実務での活用シナリオ

  • バグ再現: ステージング環境にログインし、実際のダッシュボードでバグを再現
  • 修正ループ: コードを修正 → ローカルで実行 → ブラウザで検証を自動化
  • 会議メモから関連するIssue、CRMエントリ、管理画面を自動更新
  • 社内SaaSツールとコードベースの境界領域の作業を自動化
これまではコードとブラウザの境界で人間が介入する必要があった。Codexがその隙間を埋める！

⑥ Chrome拡張の権限と注意点

  • 拡張機能に必要な権限: ページデバッグ、Webサイトアクセス、通知、ダウンロードなど
  • 閲覧活動はタスクのコンテキストとしてのみ保存（常時録画ではない）
  • 確認プロンプト、ホスト承認、許可/ブロックリストの多層防御
  • 放置厳禁: 機密ダッシュボードや個人情報では必ず人間が確認を推奨
「適当にブラウザ作業をして」と放置するような使い方はNG。常に人間の監視が前提

  • IDEコンテキスト注入で開発環境の情報をシームレスに統合
  • ワークスペース認識の差分を自然言語で表示
古い作業に戻って、そこから新しい分岐を作るのが簡単になった！

  • ターミナルのポライトが向上し、よりプロフェッショナルな感覚に
派手ではないが、毎日使うツールの細部の磨き上げが重要！

⑩ v0.129: プラグイン管理の進化

  • ワークスペース共有と共有アクセス制御を追加
  • ソースフィルタリング、ローカル共有パス追跡、マーケットプレイスのアップグレード
  • リモートバンドル同期と管理者による無効化ステートに対応
  • プラグインの標準化でチーム利用がしやすくなった
「各開発者がバラバラな設定」から「承認済みプラグインの統一」へ進化。チーム導入のハードルが下がった
