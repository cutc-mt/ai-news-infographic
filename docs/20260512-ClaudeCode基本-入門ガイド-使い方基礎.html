Claude Codeの全体像
1

① Claude Codeは「エンジン」である

  • Claude Codeとは厳密に言うとエンジンそのもの
  • そのエンジンをどう使うかによって、様々なインターフェースが存在する
  • GoogleやOpenAIも同様にCodexやAntigravityを「エンジン」と位置づけている
  • ツールの名称やモデルの名称が混在するため、話がややこしくなりがち
「Claude Code＝ターミナル」という
認識はもう古い！エンジンなんだよ
2

② Claude Code GUIとは（乗用車）

  • Claude Code GUIはディーラーで買った乗用車のようなもの
  • 一般的な方でも乗りやすく、誰でもすぐに使い始められる
  • チャットタブ、コワークタブ、コードタブなどを統合したデスクトップアプリ
  • 昔は「Claude Desktop」と呼ばれていた単なるチャットアプリが進化した姿
ディーラー製の乗用車みたいに
誰でも安心して乗れる！
3

③ Claude Code CLIとは（レーシングカー）

  • Claude Code CLIはレーシングカーのようなもの
  • カスタマイズ自由度が高いが、乗りこなすには知識が必要
  • 1年前は「Claude Code」と言えばこのCLIのことだった
  • サスペンションやハンドルなど、自由にチューニングが可能
レーシングカーみたいに
チューニング自在だけど
乗るのがちょっと大変！
Claudeへのアクセス方法
4

④ スマホアプリとWebブラウザー

  • スマホアプリからでもClaudeにアクセスできる
  • Webブラウザー経由でAnthropicにログインして利用可能
  • 基本的なチャット機能はどの方法でも同じように使える
  • ただし機能の充実度はインターフェースによって異なる
スマホでもブラウザーでも
どこからでもClaudeにアクセスOK！
5

⑤ デスクトップアプリ（Claude Code GUI）

  • WindowsやMacにダウンロードしてインストール可能
  • チャットタブ：会話・戦略会議・ブレインストーミング
  • コワークタブ：業務効率化・定期的なタスク実行
  • コードタブ：開発作業・CLIと同じエンジンを呼び出し
チャット・コワーク・コードの
3つのタブが統合されてる！
6

⑥ CLIの複数の呼び出し方法

  • ターミナルから claude コマンドで直接呼び出し
  • デスクトップアプリのコードタブからもCLIと同じエンジンを利用
  • VS Code拡張機能やCursorなどのエディターからも呼び出し可能
  • どの方法でも呼び出されるのは同じCLIエンジン
どこから呼んでも中身は同じ！
入り口が違うだけ
GUIの各タブと機能詳細
7

⑦ チャットタブ：対話のメイン

  • チャットタブは純粋な対話に特化したインターフェース
  • ユーザーのことを覚えてくれる永続的なメモリーが有効
  • 戦略会議やブレインストーミングに最適
  • あなたのビジネスやSNS活動なども覚えておいてくれる
メモリーが有効なのは
このチャットタブだけ！
重要なポイントだよ
8

⑧ コワークタブ：業務効率化

  • コワークタブは業務効率化に特化した機能
  • スケジュール機能で定期的なタスクをローカルマシンで実行
  • ローカルファイルにフルアクセス可能
  • セッション内で状態が保持される
ローカルで動くから
自分のPCのファイルに
ガッツリアクセスできる！
9

⑨ コードタブ：開発環境

  • コードタブはローカルPCにインストールされたCLIそのもの
  • MCPの設定などがすべて反映される
  • セッションログが残るのが便利なポイント
  • ルーティン機能でクラウド上での定期実行が可能
コードタブの中身はCLIと
まったく同じエンジン！
メモリーの影響範囲と問題点
10

⑩ メモリーはチャットタブに限定される

  • アカウントに紐づく永続的なメモリーは存在する
  • しかし影響範囲はチャットタブに限定されている
  • コワークやコードを使う時は常に「初めまして」の状態
  • あなたのビジネス情報や文脈が引き継がれないのが問題
チャットで覚えてくれても
コワークでは「初めまして」！
ここが大きな落とし穴
11

⑪ サイロ化の問題点

  • 製品間でメモリーや設定が共有されない（サイロ化）
  • チャットで教育した内容がコワークやコードに反映されない
  • 定期実行のワークフローを組んでも、毎回同じ教育が必要
  • AIの成果物に対するレビューやチューニングができない
毎回同じことを教えるのは
かなり非効率…
これがサイロ化の問題！
12

⑫ MCP・プラグインでカバーする方法

  • MCPやプラグインやスキルで対応可能
  • GitHubのリポジトリーで様々なツールが公開されている
  • Roo Flow、RTK、Everything Claude Code などのツール
  • これらをインストールすることでClaude Codeを強化できる
コミュニティのツールで
メモリー問題をカバーしよう！
ルーティン vs スケジュールの違い
13

⑬ ルーティン（コードタブ）の特徴

  • 実行環境はAnthropicのクラウド環境
  • 24時間365日ずっと動かし続けることができる
  • トリガーはスケジュールまたはAPI / GitHub Webhook
  • 毎回状態がリフレッシュされる（状態は保持されない）
PCをシャットダウンしても
クラウドで動き続ける！
それがルーティンの強み
14

⑭ スケジュール（コワークタブ）の特徴

  • 実行環境はローカルマシン
  • マシン起動時のみ動く（PCを閉じたら止まる）
  • ローカルファイルにフルアクセス可能
  • セッション内で状態が保持される
自分のPCで動くから
ローカルファイルに
ガッツリ触れるのが魅力！
15

⑮ ルーティンとスケジュールの比較

  • 実行環境：ルーティン＝クラウド、スケジュール＝ローカル
  • 稼働：ルーティン＝24/365、スケジュール＝マシン起動時のみ
  • ファイル：ルーティン＝ローカル不可、スケジュール＝フルアクセス
  • 状態：ルーティン＝毎回リフレッシュ、スケジュール＝保持される
  • 回数制限：ルーティン＝5〜25回/日、スケジュール＝基本無制限
ほぼ全部が逆になってる！
面白い对比だよね
まとめと実践的な使い方
16

⑯ 用途に応じた使い分け

  • ルーティンは夜間の自動レビューや定期レポートに最適
  • GitHub Actionsと同様のセットアップで自動化が可能
  • スケジュールはローカル環境依存の重い処理に適している
  • 自分のメインPCでがっつり処理したい時はスケジュールで
用途によってルーティンと
スケジュールを使い分けよう！

  • エディター上でセッションログが残るのが大きな利点
  • 過去の対話履歴を後から確認・振り返りができる
  • MCPの設定もそのまま反映されるためシームレス
