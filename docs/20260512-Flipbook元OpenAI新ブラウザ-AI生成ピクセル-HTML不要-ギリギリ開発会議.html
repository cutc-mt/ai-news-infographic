Flipbook - 元OpenAIチームが作る次世代AIブラウザ

HTML・CSS・JSは不要。画面のすべてのピクセルをAIがリアルタイム生成する、未来のブラウザ体験

2026年5月11日

ギリギリ開発会議 by ボイスアップラボ

①Flipbookとは何か: 次世代AIブラウザの概要

  • Flipbookは、画面のすべてのピクセルをAIモデルがライブストリーミングする新しいブラウザ
  • HTML、CSS、JavaScriptといった従来のWeb技術を一切使わずに画面を表示
  • テキストも画像もすべてAIが生成したピクセルとして表現される
  • 画面上のどこでもクリック可能で、クリックすると新しいページがその場で生成される
従来のWebは「コードで画面を作る」けど、Flipbookは「AIが画面を描く」という全く逆のアプローチ！

②開発チームの背景: 元OpenAIとAppleのエンジニア

  • ザさん - 元OpenAIのエンジニア
  • エディさん - 元HuggingFaceのエンジニア
  • ルーさん - 元Appleのエンジニア
  • トップテック企業出身の3人が主体となって開発を進めている
OpenAIとAppleの出身者が組んでるから、AIとUXの両方に強いチームだね！

③技術アーキテクチャ: LTX Videoと拡散トランスフォーマー

  • ユーザーの操作を受け付け、LTX Videoというモデルで画像を生成
  • LTX Videoはイスラエル発のオープンソース拡散トランスフォーマー動画生成モデル
  • 拡散モデル（ボヤボヤから徐々に鮮明に）とトランスフォーマー（情報を順次処理）の融合
  • 画像だけでなく動画生成の技術を使っているのがポイント
拡散モデルは「少ない情報から徐々に画像を固めていく」、トランスフォーマーは「次の情報を順次生成する」。この2つを合体させてるんだ！

④Modal Lab: サーバーレスGPUで動くインフラ

  • Modal Labというプラットフォーム上でサーバーレスGPUを利用
  • ローカルではなくリモートのGPUリソースにアクセスして計算を実行
  • WebSocketのストリーミング技術を使ってリアルタイムに画像を配信
  • ユーザー操作 → サーバー側でAI生成 → WebSocketでクライアントに配信という流れ
重いGPU処理は全部クラウドでやってるから、ローカルのPCスペックは関係ない設計！

⑤デモ体験: 猫を検索してみる

  • 「cat」と入力して検索を実行すると、AIが猫の情報ページを画像として生成
  • 結果が表示されるまで少し時間がかかる（AIが画像を生成しているため）
  • 表示された画像のどこでもクリックできて、例えば「猫の耳」をクリックすると猫の耳の詳細が生成
  • 「猫の尻尾」をクリックすると尻尾の図解が表示されるなど、自由に探索可能
実際に試してみたらちょっと重かったけど、概念としてはすごく面白い！

⑥どこでもクリック可能: 柔軟なハイパーリンク

  • 従来のWebではリンクの場所があらかじめ決まっていたが、Flipbookでは画面のどこでもリンク
  • AIがクリックされた場所を理解して、次に表示すべき情報を判断する
  • ハイパーリンクが固定ではなく、フレキシブルに生成されていく仕組み
  • 情報の探索がより直感的で自由な体験になる可能性
全然知らないことを調べるのにすごく良さそう！知りたい部分を直接タップできるからね。

⑦テキストもピクセル: 文字の概念が変わる

  • 従来のブラウザではテキストはテキストノードとして扱われていた
  • Flipbookではテキストも画像内のピクセルとして表示される
  • 人間には文字に見えるが、コンピュータにとってはただの画像データ
  • HTML、CSS、JSのレンダリングエンジンが完全に不要になる
画像だからテキストのコピーや検索ができない...そこはまだ課題だね。

⑧従来のWebとの根本的な違い

要素 従来のWeb Flipbook
画面構成 HTML + CSS + JS AI生成画像
レイアウト レイアウトエンジン モデルが自動配置
操作 事前定義のボタン・リンク どこでもクリック可能
テキスト テキストノード 画像内ピクセル
従来のWebの仕組みを全部ひっくり返す、パラダイムシフト的なアプローチだね！

⑨デモの実際の様子: 公式デモ vs ライブ体験

  • 公式のデモ動画は非常にスムーズに動いているように見える
  • ミュージアムのような美しいUIで、まるで触りたくなるような世界観
  • しかし実際にライブで試すとかなり重いことが判明
  • 公式デモは編集済みで、待ち時間をカットしているとのこと
動画で見ると「すごい！」ってなるけど、実際に触るとまだまだ...というギャップがある。

⑩課題①: 生成速度が遅い

  • クリックしてから画像が生成されるまでかなりの待ち時間が発生
  • 従来のハイパーリンクなら即座に画面が切り替わるが、AI生成では待たされる
  • もし30FPSでストリーミングできれば、もっと滑らかな体験が可能
  • ネットワークの速度とGPUの性能がボトルネックになっている
結局、すべては「スピードの問題」。速くなれば一気に実用的になるはず！

⑪課題②: テキストの品質と読みにくさ

  • テキストが画像として生成されるため、読みやすさが若干劣る
  • AIが文字を描画する精度はまだ完全ではなく、ぼやける場合がある
  • テキストのコピーや検索ができないという実用上の問題も
  • モデルの精度が上がれば、将来的には改善される可能性がある
文字が画像だから、Ctrl+Fで検索できないのは実用上かなり不便かも...

⑫課題③: 状態管理が存在しない

  • 同じ場所をクリックしても毎回違う画像が生成される
  • キャッシュの仕組みがないため、同じコンテンツでも再生成が必要
  • セッションや状態の保持ができないため、アプリケーション的な使い方は困難
  • まだ実験段階（ベータ）なので、今後の改善に期待
戻るボタンを押したら違う画面が出ちゃうかも...？状態管理はWebの基本だからね。

⑬GPUコストと計算リソースの課題

  • 1回クリックするだけでも莫大なGPU計算コストがかかる
  • 電気コストとGPUリソースの消費が非常に大きい
  • サーバーレスGPU上で動いているため、利用量に応じてコストが増大
  • 従来のWeb表示に比べて何桁もコストが高い可能性
1クリックがすごい電気とGPUを使ってると思うと、ちょっと申し訳ない気分になるね（笑）

⑭MetaのNeural Computer概念との比較

  • 以前Metaが提唱したNeural Computerとほぼ同じ概念
  • Meta: コンピューター全体をAI生成にする構想
  • Flipbook: ブラウザに特化したAI生成の実装
  • Metaの構想は研究段階だが、Flipbookは実際に動くプロトタイプがある
Metaは「概念」だったけど、Flipbookは「動くもの」を見せてる。そこが大きな違い！

⑮拡散モデル × トランスフォーマー: 二つのAIの融合

  • 拡散モデル: ノイズから徐々に鮮明な画像を生成する仕組み
  • トランスフォーマー: LLMに使われる、情報を上から下に順次処理するモデル
  • この全く別の二つのモデルを融合させたのがDiT（Diffusion Transformer）
  • 画像のような「空間的な生成」とテキストのような「逐次的な生成」のいいとこ取り 
# 拡散モデル: ノイズ → 次第に鮮明に
 noise → blur → shape → clear

# トランスフォーマー: 順次生成
 token1 → token2 → token3 → ...

# DiT: 両方を組み合わせたアプローチ

⑯現在の立ち位置: クライアントに提案するプロトタイプ段階

  • まだ実験段階（ベータ）であり、本番運用ではない
  • 「クライアントに提案するプロトタイプ」の段階
  • 概念を実証するもの（PoC）としては十分に動いている
  • 実用化に向けては、速度、品質、コストの3つの壁をクリアする必要がある
「プロトタイプを提案した段階」というのが一番しっくりくる表現かも。

⑰将来展望: 2〜3年後には実用化しているかも

  • 2〜3年後にはこのようなブラウザが実際に動いている可能性
  • ネットワークの高速化とGPUの進化が実用化の鍵
  • ローカルGPUとリモートGPUを分担する仕組みができるかもしれない
  • ニューラルコンピューター同士が連携する未来も予想される
パーソナルコンピューターのGPUがすごくなれば、ローカルでも動くようになるかも！

⑱まとめ: Webの未来を垣間見る体験

  • FlipbookはHTML・CSS・JSを使わない全く新しいブラウザの概念
  • 元OpenAI・Appleのエンジニアが開発する、高い技術力を持つチーム
  • 現在はまだ実験段階で、速度・品質・コストに課題が残る
  • 拡散トランスフォーマーの技術を使って、AIが全画面を生成する世界観
  • 2〜3年後の実用化に期待。ニューラルコンピューターの未来が近づいている
「行動が不要な世界」への第一歩。プログラミングしなくても画面が作られる未来が来るかも！
