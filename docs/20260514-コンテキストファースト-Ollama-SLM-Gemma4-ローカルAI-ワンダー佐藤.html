AIの致命的な弱点「Lost in the Middle」
4

④ Lost in the Middle現象とは

  • 研究論文「Lost in the Middle」で言語モデルの処理弱点が証明された
  • 情報の先頭と末尾には強く注目できるが…
  • 中間の情報を無視してしまうという致命的な問題
長い文章の真ん中にある情報
AIは読めてないことが多い
これが「ミドルロスト」！
5

⑤ U字型の性能曲線

  • 性能曲線はU字型を描く。先頭と末尾が高く、真ん中が低い
  • 位置によって情報の処理精度が大きく変動する
  • プロンプトの構造を意図的に設計しないと精度が落ちる
U字型ってことは
真ん中は要注意ゾーン
大事な指示を置いちゃダメ！
6

⑥ SLM特有の「最後優位性」

  • SLMは先頭情報の処理能力がさらに弱く、一番最後の情報を極端に重視
  • 指示を先頭に置いて長文を読み込ませると肝心の指示が中間に押し流される
  • AIが指示を無視したり関係のない出力をする原因はここにある
SLMは「最後の言葉」を
一番大切に聞くタイプ
そこを逆手に取る！
コンテキストファースト戦略の全貌
7

⑦ コンテキストファーストとは

  • 外部知識や過去の会話などの長い文脈をすべて先に提示
  • 今回実行したい具体的な指示を一番最後の末尾に配置
  • SLMの最後優位性（Recency Bias）を直接活用する設計アプローチ
資料を先に出して
最後に「で、これ要約して」
この順番が正解！
8

⑧ 配置の比較: 絶対に覚えよう

❌ プロンプトファースト
[指示] → [長い文脈]
指示が中間に押し流される！
✅ コンテキストファースト
[長い文脈] → [指示]
指示が末尾に残り確実に認識！
  • 基本ルール: 長文＝コンテキストファースト、短文＝プロンプトファースト
  • 短文処理ならプロンプトファーストでもOK
長い資料を渡す時は
必ず「資料→指示」の順番
これを徹底しよう！
9

⑨ プロンプトの「鮮度」問題

  • プロンプトが末尾にあるほど「鮮度」が高くAIに強く伝わる
  • 中間に埋もれると指示の効果が薄れ、AIに伝わらない
  • 末尾配置は「熱い指示」をAIに届ける方法と理解できる
指示も「鮮度」が大事
古い指示＝冷めたコーヒー
最後が一番熱い！
Ollamaの構造と注意点
10

⑩ OllamaのModelfile構造

  • 派生モデル作成時のModelfileのSYSTEM命令が最上流に強制配置
  • Ollama特有の構造: SYSTEM（最上位固定）→ Context → Prompt
  • これによりプロンプトがミドルロストしやすい環境になる
Ollamaはシステムプロンプトが
必ず最初に来る仕組み
だからコンテキストファーストが
さらに重要に！
11

⑪ ChatGPT / Gemini / Ollama の比較

  • ChatGPT型: System Prompt → Context → User Input
  • Gemini型: 独自の配置ルールを採用
  • Ollama: Modelfile SYSTEM（固定）→ Context → Prompt
  • 各プラットフォームの構造の違いを理解した上でプロンプトを設計する必要がある
プラットフォームごとに
プロンプトの置き場所を
調整するのがプロのやり方！
