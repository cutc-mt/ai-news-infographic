①ゴールドフィッシュ問題とは

  • AIエージェントが2分前の会話も忘れてしまう問題のことを「ゴールドフィッシュ問題」と呼ぶ
  • 多くの開発者がモデルとツールばかりに注目して、記憶を忘れがち
  • ユーザーを感動させるアプリを作るには、「完全な脳」を使うエージェントが必要
モデルの賢さだけでは不十分。記憶があってこそ、エージェントは本当に役に立つ存在になるんだよ！

②エージェント記憶の6つのパターン

  • 基本 セッションステート（会話中の一時記憶）
  • 基本 マルチエージェントステート（エージェント間の共有記憶）
  • 基本 永続的メモリ（長期保存の記憶）
  • 高度 コールバック（自動で記憶を更新）
  • 高度 カスタムツール（構造化データの読み書き）
  • 高度 マルチモーダルメモリ（画像・音声・動画の記憶）
前回の動画で基本3つ、今回は高度な3つを解説！

③基本の復習: セッションステート

  • 会話中だけ有効な一時的な記憶
  • エージェントが会話のコンテキストを保持する仕組み
  • セッションが終了するとリセットされる
ブラウザのタブを閉じると消えるデータと同じイメージだよ

④基本の復習: マルチエージェントステート

  • 複数のエージェント間で状態を共有する仕組み
  • あるエージェントが学んだことを別のエージェントが利用可能
  • チームワークのような協調動作を実現
エージェントAが調べたことをエージェントBが引き継ぐイメージ！

⑤基本の復習: 永続的メモリ

  • セッションをまたいでデータを長期保存する仕組み
  • アプリを再起動しても記憶が残る
  • データベースやファイルシステムに永続化
基本パターンは前回の動画で詳しく解説済み。今回はここからが本番！

⑥高度パターン①: コールバックとは

  • 会話に基づいて自動的に記憶を更新する仕組み
  • スパイのように会話を見守り、メモを取るイメージ
  • エージェント自身を複雑にせずにコンテキストを保存できる
専属のアシスタントが常にノートを取りながら話を聞いてくれるようなもの！

⑦コールバックの仕組み

  • エージェント実行の前後にカスタムロジックを追加可能
  • モデル呼び出しの前後で処理を挟める
  • ツール呼び出しの前後でも処理可能
  • コールバックがエージェントのライフサイクルを横取りする形で動作
「before_model_callback」「after_tool_callback」など、フックポイントが複数あるよ

⑧コールバック実装例: 旅行プランニング

  • ユーザーが「美術館」をリクエスト → アクティビティタイプをCulturalに記録
  • レストランをリクエスト → タイプをFoodに記録
  • 公園をリクエスト → タイプをOutdoorに記録
  • 連続して同じタイプの場合、別の種類を提案して重複を防ぐ
「美術館」の後にまた「美術館」を提案しないように、自動で制御できるんだ！

⑨コールバックのコード解説

  • after_tool_callbackでツール実行後に処理を追加
  • 呼び出されたツールに応じてアクティビティタイプを自動設定
  • タイプをツールコンテキストに書き込む 
// after_tool_callback のイメージ
if tool == "museum_expert":
    activity_type = "cultural"
elif tool == "restaurant_finder":
    activity_type = "food"

// プランナーの指示で前回のタイプを確認
if last_activity == "cultural":
    ban_tool("museum_expert")

⑩高度パターン②: カスタムツールとは

  • 非構造化テキストから構造化データへの進化
  • チャットログではなく、ユーザープロファイルに保存
  • 例: 「ヴィーガンです」と言ったら → 食事の好みフィールドを自動更新
膨大なチャットログから探すより、プロフィール欄を見た方が速いよね！

⑪非構造化から構造化データへの進化

  • Before 全てがテキストのチャットログに埋もれている
  • After ユーザー属性をフィールドとして整理して保存
  • エージェントが必要な情報に瞬時にアクセス可能
  • 辞書オブジェクトとして保存し、データベースに書き込む
「ヴィーガン」という発言 → dietary_preference: "vegan" として保存

⑫カスタムツールのユーザー体験例

  • 初回: 「夕食を計画して」→ 好みを知らないので質問される
  • ユーザー: 「ヴィーガンです」→ プロファイルに自動保存
  • スクリプトを再起動
  • 再回: 「夕食を計画して」→ もう質問なしでヴィーガンレストランを提案
一度伝えれば、次からは自動で覚えていてくれる。これが構造化データの力！

⑬カスタムツールの実装: 2つのツール

  • save_user_preference: ユーザーの好みをデータベースに保存
  • recall_user_preferences: 保存済みの好みを読み出し
  • エージェントの指示に必ず最初に recall を呼ぶと設定 
// ツールの定義
save_user_preference({
    preferences: {
        "dietary": "vegan",
        "budget": "moderate"
    }
})

// recall は保存されたレコードを読み出すだけ
prefs = recall_user_preferences()

⑭高度パターン③: マルチモーダルメモリとは

  • これまでの記憶はテキストだけだった
  • 人間は写真の雰囲気や声のトーンも覚えている
  • 優れたエージェントは写真・動画・音声も記憶すべき
  • 異なるメディアタイプを横断的に接続する能力
テキストだけじゃなく、写真や音声の「感覚」も記憶するのが次世代のエージェント！

⑮マルチモーダルメモリの体験例

  • 旅行アシスタントに写真・動画・音声をアップロード
  • 「共有したメディアに基づいて、どこに旅行すべき？」と質問
  • エージェントが異なるメディアを横断して好みを推測
  • 過去の体験と結びつけて最適な旅行先を提案
写真の雰囲気＋動画の風景＋音声の好み = すべてを総合して提案するよ

⑯マルチモーダルメモリの実装

  • 写真・動画・音声を含むセッションをメモリバンクに保存
  • preload_memoryツールでメモリバンクから会話へ読み込み
  • Vertex AI Agent EngineのMemory Bankを活用 
// メモリバンクにセッションを保存
memory_bank.save({
    session_id: "travel_001",
    media: [photo, video, audio]
})

// エージェントがメモリをプリロード
preload_memory(session_id="travel_001")

⑰3つの高度パターンの振り返り

  • コールバック: エージェントを複雑にせずに、自動で記憶を更新する仕組み
  • カスタムツール: ユーザープロファイルなどの構造化データを読み書き
  • マルチモーダル: 写真・動画・音声を含む豊かな記憶を実現
この3つを組み合わせれば、ユーザーを感動させるエージェントが作れる！
