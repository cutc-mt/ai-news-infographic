AIコーディングのオープンソース・マーケットプレイスが開幕 - コミュニティ主導でワークフローを共有

2026年5月15日 | Cole Medin

①マーケットプレイスがローンチ: コミュニティへのワークフロー投稿が開始

  • Archon Workflow Marketplace が正式にオープンした。コミュニティメンバーが自作のワークフローを投稿できるようになった。
  • ライブストリームで第1波のワークフローがArchonの自動レビューパイプラインを通じて公開された。
  • レビュー担当のAIエージェントもArchon自身が構築しており、マーケットプレイス上で自動審査が行われる仕組みだ。
AIが構築したレビュアーが、AIの構築したマーケットプレイスを審査するという自己完結型エコシステムが実現！

②Archonとは: AIコーディングを再現可能にするオープンソースエンジン

  • ArchonはAIコーディングエージェント向けのワークフローエンジンであり、開発プロセスをYAMLで定義できる。
  • Dockerfileがインフラを再現可能にしたように、GitHub ActionsがCI/CDを標準化したように、ArchonはAIコーディングのワークフローを決定论的に再現可能にする。
  • AIに「このバグを修正して」と頼むと、モデルの気分で結果が変わる問題を解決する。プロセスは固定化し、知能は各ステップでAIが担当する。
n8nのようなワークフロー自動化ツールのソフトウェア開発版と考えると分かりやすい

③YAMLワークフロー定義: 開発プロセスをコード化する仕組み

  • ワークフローは.archon/workflows/ ディレクトリにYAMLファイルとして定義する。
  • 各ノードは依存関係（depends_on）でつながり、DAG（有向非巡回グラフ）を形成する。
  • ループノードを使えば、テストが通るまでAIが自動で繰り返し修正を試みる反復処理も可能だ。 
nodes:
  - id: plan
    prompt: "コードを調査して実装計画を作成"
  - id: implement
    depends_on: [plan]
    loop:
      prompt: "計画に基づいて実装"
      until: ALL_TASKS_COMPLETE
  - id: approve
    depends_on: [implement]
    loop:
      until: APPROVED
      interactive: true

④マルチプラットフォーム対応: CLI、Web、Slack、Telegram、Discord、GitHub

  • CLI - ターミナルから直接ワークフローを実行。Claude Codeと統合されて動作する。
  • Web UI - ダッシュボードでリアルタイムにチャットや進行状況を監視できる。
  • Slack - チャットから直接ワークフローを起動（設定15分）。
  • Telegram / Discord / GitHub Webhooks にも対応済み。
すべてのプラットフォームからの会話が1つのダッシュボードに集約される！

⑤Git Worktreeによる分離実行: 並列で安全に開発を進める

  • ワークフローを実行するたびに、専用のGit Worktreeが自動で作成される。
  • 5つのバグ修正を同時に走らせても、コンフリクトなしで並列処理が可能だ。
  • ワークフローを起動したら放置OK。戻ってくるとレビュー付きの完成したPRが待っている「Fire and Forget」スタイル。
ワークフローを起動して別の作業へ。完了時にはPRが準備されている！

⑥Plan-Implement-Validate: 人間の承認を挟む開発ループ

  • Plan - AIがコードベースを調査して実装計画を作成する。
  • Implement - 計画に基づいてAIが実装を進める。タスク単位で反復的に処理。
  • Validate - テストの実行とコードレビューで品質を担保。
  • 各フェーズ間にHuman-in-the-Loopの承認ゲートを設置でき、AIが暴走するのを防ぐ。
AIに任せつつ、关键な判断は人間がコントロールできる！

⑦自動レビューパイプライン: AI審査員が品質を担保

  • マーケットプレイスに投稿されたワークフローは、AIレビュアーが自動で審査する仕組み。
  • レビュアー自体もArchonで構築されており、自己ブートストラップ型のエコシステムを実現している。
  • 品質基準を満たさないワークフローは自動で差し戻され、投稿者が修正できる仕組みだ。
Archonが構築したレビュアーが、Archonが構築したマーケットプレイスを審査するメタ構造！

⑧17種類のデフォルトワークフロー: すぐに使える豊富なテンプレート

  • archon-fix-github-issue - GitHub Issueを自動分類して調査・実装・PRまで完結。
  • archon-idea-to-pr - アイデアから計画・実装・バリデーション・5並列レビュー・自己修正まで。
  • archon-comprehensive-pr-review - 5人の専門レビューエージェントが並列でPRを審査。
  • archon-refactor-safely - 型チェックと動作検証付きの安全なリファクタリング。
  • ほかにもarchon-assist、archon-architect、archon-resolve-conflictsなど全17種を搭載。

⑨マーケットプレイス注目ワークフロー: コミュニティの傑作たち

  • video-generic (@lamachine) - URLやリポジトリから音声付きアニメーション動画を自動生成。3つの承認ゲート付き。
  • archon-idea-to-wo (@lraphael) - アイデアを8ノードの対話型ワークフローでBKM形式のWork Orderに変換。
  • archon-smart-mr-review - GitLab MR対応の適応型コードレビュー。Haikuが分類して並列レビュー後に自動承認。
  • archon-ralph-dag - PRD実装ループ。ストーリーを分割して完了まで反復的に実行する。
archon workflow install <名前> でワンコマンドインストール！

⑩Web UIダッシュボード: リアルタイムでワークフローを管理

  • Chat - リアルタイムストリーミング付きの会話インターフェース。ツールコールの可視化も可能。
  • Dashboard - 実行中のワークフローを一覧表示。プロジェクト・ステータス・日付でフィルタリングできる。
  • Workflow Builder - ドラッグ＆ドロップでDAGワークフローを視覚的に作成できるエディタ。
  • Workflow Execution - 実行中または完了したワークフローのステップごとの進行状況を表示。
archon serve でWeb UIがワンステップ起動！

⑪アーキテクチャ全体像: 6層構造のシステム設計

  • Platform Adapters層 - Web UI、CLI、Telegram、Slack、Discord、GitHubの6つの入力ソース。
  • Orchestrator層 - メッセージのルーティングとコンテキスト管理を担当。
  • Command Handler - スラッシュコマンドの処理。
  • Workflow Executor - YAMLワークフローの実行エンジン。
  • AI Assistant Clients - Claude、Codex、Piの3つのAIアシスタントに対応。
  • データ層 - SQLiteまたはPostgreSQLで7テーブルを管理。

⑬カスタムワークフローの自作: 自分好みのプロセスをYAMLで定義

  • デフォルトワークフローは.archon/workflows/defaults/からコピーしてカスタマイズ可能。
  • 同じ名前のファイルをリポジトリに置けばデフォルトを上書きする仕組み。
  • Gitにコミットすれば、チーム全員が同じプロセスで開発を進められる。
  • 決定論的ノード（bash、テスト、git操作）とAIノード（計画、コード生成、レビュー）を自由に組み合わせ可能。
AIが価値を追加する部分だけにAIを使い、それ以外はスクリプトで確実に実行！

⑭マルチAI対応: Claude、Codex、Piの3つのアシスタント

  • Claude - AnthropicのClaude Codeと直接統合。最も推奨されるアシスタント。
  • Codex - OpenAIのCodexにも対応。AIアシスタントを柔軟に選択可能。
  • Pi - Inflection AIのPiにも対応済み。
  • 設定ファイルでbinaryPathを指定すれば、インストール済みのAIツールをArchonから利用できる。
ワークフローは共通だから、AIを入れ替えても同じプロセスで動く！

⑮コミュニティ投稿の流れ: ワークフローを世界に共有

  • 作成 - .archon/workflows/ にYAMLファイルを配置して独自のワークフローを定義。
  • 投稿 - マーケットプレイス（archon.diy/workflows）に提出。
  • 審査 - AIレビュアーが自動で品質チェックを実行。
  • 公開 - 審査通過後にコミュニティ全体に公開。インストール可能になる。
コミュニティが作ったワークフローが、全員の生産性を底上げするエコシステム！

⑰テレメトリとプライバシー: 最小限の匿名データ収集

  • Archonはワークフロー起動時に1つの匿名イベントのみを送信する（workflow_invoked）。
  • 個人情報は一切収集しない。コード、プロンプト、メッセージ、トークンは送信対象外。
  • 収集されるのはワークフロー名、説明、プラットフォーム、バージョン、ランダムなUUIDのみ。
  • オプトアウトは環境変数 ARCHON_TELEMETRY_DISABLED=1 または DO_NOT_TRACK=1 で可能。
PostHogを自前ホストしたり、独自プロジェクトに変更することも可能！
