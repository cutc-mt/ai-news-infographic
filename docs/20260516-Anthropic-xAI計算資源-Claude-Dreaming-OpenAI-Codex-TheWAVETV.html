Anthropic × xAI 計算資源調達

① AnthropicとxAIの大型契約

  • 5月6日、AnthropicとxAIが大型契約を締結した
  • xAIは社名を「xAI」から「Space xAI」に変更した模様
  • ライバル同士だった両社の急接近に業界が驚愕
  • イーロン・マスクは「Anthropicの連中と話したがいいやつだった」とX上で発言
以前はマスクがAnthropicをボロカスに言っていたのに、手のひら返しで提携！双方に必要があったからこそ実現した取引です。

② Colossus 1の全容量をAnthropicに

  • xAIのデータセンター「Colossus 1」の全容量をAnthropicに提供
  • 月に300MW超の電力、22万台のNVIDIA GPUにアクセス可能に
  • xAIはColossus 2を別途構築済みで、Grokの開発はそちらに移行
  • Colossus 1が余ったため Anthropicに貸し出す形に
Colossus 2が完成したことで1が空いた。マスクにとっては「電力会社」のようなビジネスモデルに移行しつつあるのかも。

③ Claudeの利用制限が緩和された理由

  • Anthropicはこれまで計算資源不足で利用制限をかけていた
  • Claude Codeが大爆発的な人気で、需要が急増
  • xAIからの計算資源調達で制限を緩和できるように
  • A16Zのマーク・アンドリーセンは「トークンが当たらないから制限しているのでは」と指摘していた
ユーザーは「この程度で制限？」と不満が上がっていたが、裏で計算資源の確保が進んでいた。
AI導入支援に参戦するAnthropicとOpenAI

④ AI導入支援にAnthropic・OpenAI・Googleが参戦

  • 5月4日、Anthropicが金融系企業とジョイントベンチャーを設立
  • 翌週、OpenAIが同じようなAI導入支援会社を設立
  • さらにGoogleも同様の取り組みを開始
  • 企業はソフトウェア1ドルに対してサービス約5ドルを払うビジネスモデル
AI導入支援は今最もホットな領域。アクセンチュアやデロイトなどのコンサル市場を侵食していく可能性がある。

⑤ フォワードデプロイエンジニアという新職業

  • Anthropic・OpenAIが設立した会社は「フォワードデプロイエンジニア」を派遣するモデル
  • AIの知識と業界ドメイン知識の両方を持つ人材が求められている
  • BoxのCEOも「今卒業する大学生はこの職業を目指せ」と推奨
  • 従来の駐在型エンジニアのAI版とも言える新しい職業
これからAIを活用する人としない人の差がめちゃくちゃ開いていく。その橋渡し役になる人材が今一番必要とされている。

⑥ サム・アルトマンの「奇妙な結果」発言

  • 2026年1月、サム・アルトマンがインタビューで興味深い発言をしている
  • 「AIはすでに賢い。でも社会はその賢さを十分に活用できていない」
  • 「このことは数年前には予測できなかった奇妙な結果をもたらすだろう」
  • 「オーバーハング」＝AIの賢さが溢れて、受け止める器（人間側）が小さい状態
AIの賢さをそのまま生かすための仕組み（ハーネス）作りが重要。AIを使える人と使えない人の差がどう開くのか、まだ分からない部分も多い。
2つのニュースの背景

⑦ xAIからの人材流失

  • 2〜3月頃、xAIの創業メンバーが次々と退社
  • 右腕と言われた人物や有名研究者が続々と離脱
  • Grok 5はQ1リリース予定だったが、5月時点でも未発表
  • 2月に「Xは基礎から作り直す」、3月に「コーディングで負けている」とマスクが発言
珍しくマスクが「負けている」と認める発言。フロンティアモデル開発競争から事実上撤退したのではという見方も。

⑧ Anthropicがデータセンターに投資しない理由

  • 2月のポッドキャストでダリオ・アモデイCEOが語った
  • 「2027年末に1兆分の計算資源を買える。売上が8000億ドルなら倒産を止められない」
  • 「需要が伸びなかったら1年で倒産する」と慎重な姿勢
  • 成長率が10倍ではなく5倍なら倒産リスクがあると警告
Claude Codeが予想外の大成功を収め、状況が急変。アモデイ氏の慎重さが裏目に出た形だが、今回の提携で一気に解決へ。

⑨ 勝負は「電力」ではなく「モデルとハーネス」

  • これまで「データセンターを持てば勝つ」と言われていた
  • しかしAnthropicはデータセンターを持たなくても計算資源を確保できることを証明
  • 勝負の鍵はモデルの性能とハーネス（実行環境）の質
  • お客さんをしっかり取っていれば、計算資源はどこからでも買える
Anthropicは良いモデル（Claude）とハーネス（Claude Code）を作ったから、計算資源は後からでも確保できた。本質的な競争力はインフラではない。
自律型実験室とエネルギーの未来

⑩ 自律型実験室「PoLARIS」が発光ナノ材料を最適化

  • ノースカロライナ州立大学の研究チームが自律実験室を構築
  • AIが合成レシピを選び、マイクロ流体実験装置が自動で材料を合成
  • 「探索→合成→評価→改善」のサイクルを自動化
  • 12時間で120回の実験を実施（人間なら数年かかる作業）
デジタルの世界だけでなく、物理的な実験室もAIが自動化し始めた。新素材発見のスピードが劇的に加速する。

⑪ エネルギーが「無料」になる未来

  • Google DeepMindのハサビス氏は「エネルギーが無料になるのはそう遠くない」と主張
  • その根拠の一つが、自律型実験室での新素材発見の加速
  • 太陽光燃料などの発見スピードが格段に向上
  • 多くの人が「そんなわけない」と反応するが、技術的には裏付けがある
自律型実験室があれば、人間が数年かかる実験を半日で完了。ハサビス氏の主張は決して妄想ではない。
Android 17とハーネス戦略

⑫ Android 17は「スマホOS」から「AI実行環境」へ

  • Androidに「Gemini Intelligence」というAI基盤が統合される
  • 「フィラーゴ」機能：会話の「えー」「あのー」などのフィラー語を自動削除
  • 文章指示だけでGoogleアプリのカスタムウィジェットを生成
  • AndroidとiPhoneのファイル共有機能も拡大
  • セキュリティ強化も同時実装
これまでのAndroidは「アプリを起動する土台」。これからは「ユーザーの意図を理解して作業を進めるAIレイヤー」に進化する。

⑬ 統合戦略 vs 分離戦略

  • 統合戦略：モデルとハーネスを一体化（Google、Anthropic、OpenAI）
  • 分離戦略：モデルとハーネスを分離（Apple、NVIDIA、Microsoft、Amazon）
  • スマホ：Googleが統合、Appleが分離
  • B向け：Anthropic・OpenAI・Googleが統合、NVIDIA・Microsoft・Amazonが分離
  • どちらが良いのかはまだ分からない
統合の方が性能は良いが、分離の方が安くて柔軟。AIの世界でもこの二つの戦略が競合していく。
AIコーディングエージェント競争

⑭ OpenAI Codexの人気がエンジニア間で急上昇

  • エンジニアコミュニティ「Latent Space」でCodexの評価が高まっている
  • GPT-5.5の性能向上に伴い、Codexのハーネスも大幅に改善
  • OpenAIは値段が安く、外部ツール利用の課金も手頃
  • Anthropic側は計算資源不足で実質的な値上げやAPI不安定が続く
投資家や経営者はまだ「Claude Code一強」だと思っているが、エンジニアの間では「Codexもすごい」という声が増えている。市場の認識にギャップが生じている。

⑮ AIコーディングエージェントの競争軸

  • 競争の軸は「どのハーネスと融合するか」「外部ツール利用をどう課金するか」
  • Anthropic：Claude単体は安いが、外部ハーネス利用で高額になる
  • OpenAI：外部ツール利用も安く、OpenClaudeの開発者も在籍
  • 実務では両方を使い分けるケースも増加中
  • 「書くエージェント」と「レビューするエージェント」を分ける手法が人気
一つのAIに任せると偏りが出るので、別々のAIで相互レビューする使い方が実用的。
Claudeの「眠る」機能

⑯ Claudeは「眠る」ことで賢くなる

  • Claude Managed Agentsの新機能「Dreaming」が登場
  • エージェントが過去の作業セッションを振り返り、メモリを整理する
  • 重複・矛盾・古い情報を見つけて修正案を作成
  • 作業や失敗の傾向を抽出して学習する
人間の脳は睡眠中に記憶を整理する。これと同じ機能がAIにも実装された。ハサビス氏が「この機能が必要」と言ってから1週間で実現！

⑰ AI開発のスピードが異常なことになっている

  • 「こんな機能があったらいいな」→1週間後に実装というスピード
  • 開発自体もエージェントを使って行われている
  • エンジニアの間でClaude Managed Agents（CMA）の評判が非常に良い
  • 既に実際のプロジェクトでCMAを導入する企業も登場
ハサビス氏が「半年後には出るかな」と言った機能が1週間後に出た。AIがAIを開発する時代、開発サイクルは加速する一方。

⑱ まとめ：Anthropic中心の週だった

  • xAIから計算資源調達で利用制限の課題を一気に解決
  • AI導入支援に参戦し、新たなビジネス領域を開拓
  • 「Dreaming」機能で技術的にも最先端を走り続ける
  • ただしCodexの台頭でAIコーディング競争は激化
  • OpenAIはサム・アルトマンが裁判で忙しく、動きが鈍い状況
今週は文字通り「Anthropic一色」。計算資源、ビジネス、技術のすべてでAnthropicが主役の週でした。
