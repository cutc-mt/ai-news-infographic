①スキルとは何か: AI用の教材セット

  • スキルとはAI用のマニュアルのようなもの。作業の進め方や手順をまとめた教材セット。
  • クロードコードに「どうして欲しいか」を伝えるための仕組み。
  • ブログ執筆、リサーチ、画像生成など、作業ごとに専用のスキルを作成できる。
  • スキルなしでも動くが、あると自分の思い通りに動かせるようになる。
スキルを使っていないと、
クロードコードを使っていないのと
同じくらいもったいない！

②スキルがもたらす進化: 初心者脱却の鍵

  • スキルなしの状態では、ただ賢いだけのAI。業務のことは知らない。
  • スキルを与えることで、こちらがどうして欲しいかが伝わる。
  • スキルなしでは毎回説明が必要。スキルがあれば一発で動いてくれる。
  • スキルを使いこなせるかどうかが、初中級者以上の分かれ目。
クロードコードの真価は
スキルを使って初めて発揮される！

③CLAUDE.mdとスキルの違い: 読み込み方式

  • CLAUDE.mdは会話開始時にすべて読み込まれる設定ファイル。
  • スキルは必要がある時だけ読み込まれる。
  • CLAUDE.md: 「日本語で会話して」などの基本設定に最適。
  • スキル: 「記事執筆」「リサーチ」など作業ごとの専用手順に最適。
リサーチする時に記事執筆の
スキルは不要だからね！
必要なものだけ読むのが効率的。

④コンテキストウィンドウの節約: なぜ必要時のみ読むのか

  • AIにはコンテキストウィンドウという記憶容量の限界がある。
  • 関係ないスキルまで全部読み込むと、容量が圧迫されて内容を忘れる原因に。
  • 必要な時だけ必要なスキルを読む方式で、会話の途中で内容を忘れない。
  • 「記事執筆して」と頼むと、記事執筆に必要なスキルだけを自動で読み込む。
AIの記憶容量は有限！
不要なものは読み込まないのが
一番の効率化なんです。

⑤スキルの配置場所: 2つの置き方

  • 方法1: プロジェクト配下に設置 → そのプロジェクトでだけ使える。
  • 方法2: ホームディレクトリに設置 → 全プロジェクト共通で使える。
  • プロジェクト専用のものはプロジェクト内に、共通のものはグローバルに配置する。
  • 例: コミット＆プッシュのスキルはグローバルに置くと便利。
置き場所で使える範囲が変わる！
目的に合わせて使い分けよう。

⑥正しいフォルダ構成: スキルの作り方

  • プロジェクト配下: .claude/skills/ に設置する。
  • スキル名でディレクトリを作成し、その中に skill.md を配置。 
.claude/
  └── skills/
      └── commit-push/
          └── skill.md
      └── make-content/
          └── skill.md
  • skill.mdはMarkdown形式のテキストファイル。内容を自由に記述。
ディレクトリを作ってから
skill.mdを入れるのが正解！

⑦プロジェクト配下がおすすめ: 管理のしやすさ

  • グローバルに置くとスキルの存在を忘れがちになる。
  • 忘れたグローバルスキルが原因で、思った通りに動かないことがある。
  • プロジェクト配下なら見やすく管理もしやすい。
  • AIが勝手にグローバルに作ることがあるので、「プロジェクト配下に作って」というスキルも作っておくと安心。
グローバルに置いたスキルは
AIが勝手に読みに行って
混乱の元になるかも…！

⑧スキル作成後は再起動が必要な場合も

  • スキルを作った直後は再起動が必要な場合がある。
  • 環境によって異なるが、新しいセッションを立ち上げると認識される。
  • 「スキルを作ったのに使えない」場合は、まず再起動を試すこと。
  • 再起動後にスキルが正しく読み込まれるか確認する癖をつけよう。

⑭スキル作成のコツ①: シンプルに始める

  • まずはSkill Creatorで土台を作ってからカスタマイズ。
  • 最初から凝ったスキルを作ろうとするとうまくいかないことが多い。
  • 2回以上やる作業を見つけたら、そのままスキル化するのがコツ。
  • 「これをスキルにして」と言うだけで、AIが自動でスキル化してくれる。
うまくいった手順は

⑮スキル作成のコツ②: ディスクリプションを具体的に

  • AIが自動でどのスキルを使うか判断してくれるのが最大のメリット。
  • ディスクリプションを具体的に書くと、AIが正確にスキルを選んでくれる。
  • 手順を順番に書くことも大切。「これをやって、次にこれをやって」のように。
  • 強い命令口調でなくてもOK。ちゃんと「こうして欲しい」を明確に書くことが重要。
「コンテンツを作って」と言えば

⑯詰め込みすぎに注意: 品質低下の原因に

  • スキルを詰め込みすぎると忘れたり実行しないことがある。
  • スキルがバッティングして意図しない動作を起こす可能性も。
  • AIが進化すると、昔のスキルが邪魔になることがある。GPT-5.5などの新モデル登場時に顕著。
  • 「ざっくり投げてAIに任せる」方が良い結果が出ることも増えている。

⑰他人のスキルは危険: セキュリティリスクに警鐘

  • 他人のスキルをそのまま入れるのは絶対にNG。中身が不明だから危険。
  • 「ブログ記事用スキル」に見えても、ハッキングや情報漏洩のコードが書かれている可能性がある。
  • 入れる前に必ずAIに内容を確認させる。「このスキルは大丈夫？」と聞くこと。
  • 野良MCPも同じリスク。実際にGoogle広告のMCPが原因で乗っ取られた事例も。
  • 理解できないものは入れないという原則を守ること。

⑱スキルは定期的に見直す: AI進化への対応

  • AIが進化すると昔のスキルが使えなくなることがある。
  • 他人のスキルを入れていると、内容を理解できないから問題に気づけない。
  • 消すのも怖い → 理解して作ったスキルだけを管理するのが安全。
