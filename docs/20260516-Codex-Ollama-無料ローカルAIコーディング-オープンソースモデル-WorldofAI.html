Codex × Ollama = 無料で使い放題のAIコーディング

OpenAIのCodexがOllamaに対応。オープンソースモデルをローカルで無料実行できる時代が到来

2026年5月16日

①Codex × Ollama 統合とは

  • OllamaがOpenAIのCodexアプリに正式対応した
  • オープンソースモデルをローカルで無料実行してCodexのAIコーディング機能が利用可能に
  • APIコストなし、クラウド制限なし、完全ローカル環境で動作
  • ローカルAIコーディングワークフローにおける大きな転換点となる統合
Claude Code + Ollamaに続き、Codexでもローカルモデルが使えるように！

②Codexとは

  • OpenAIが提供するAIコーディングエージェント
  • アプリの構築・編集・レビュー・リリースをAIで支援するツール
  • 現状の市場で最も優れたAIコーディングエージェントという評価
  • 現在はmacOSとWindowsに対応（Linux版は近日公開予定）
  • アプリ自体は完全無料でインストール可能

③Ollamaとは

  • オープンソースモデルを自分のPC上でローカル実行できるツール
  • シンプルなコマンドでモデルのダウンロードと実行が完了
  • データが外部に送信されないためプライバシーも安心
  • DeepSeek、Gemma、Qwenなど多数のモデルに対応
インストールもコマンド1行で完了するほどシンプル！

④なぜ無料なのか

  • モデルを自分のGPUとCPUで実行するためAPI料金が発生しない
  • クラウドプロバイダーの利用制限なし
  • インターネット接続が不要なため、オフライン環境でも利用可能
  • 利用量に応じた課金がないため使い放題
必要なのは自分のPCとGPUだけ！

⑤対応モデル一覧

  • DeepSeek V4 — 高性能なコーディング特化モデル
  • Gemma 4 — Googleのオープンモデル（複数サイズ展開）
  • Qwen 3.6 — Alibabaの多言語対応モデル
  • Kimi K2.6 — ビジョン対応のマルチモーダルモデル
  • JLM 5.1 / Neotron 3 Super

⑥推奨モデル（有料API版）

  • Kimi K2.6 — ビジョンサポート付きの高性能モデル
  • JLM 5.1 — 汎用的なコーディングタスク向け
  • Neotron 3 Super — 最高品質の生成結果
これらはOllama Cloudのサブスクリプションが必要な有料版です

⑦推奨ローカルモデル（無料）

  • Gemma 4 — 複数のモデルサイズが選べる
    自分のPCスペックに合わせて最適化可能
  • Qwen 3.6 — 様々なバリアントを提供
    軽量版から高性能版まで自在に選択
  • 自分のコンピュートリソースで完全無料でホスト可能
自分のGPUに合わせてモデルサイズを選ぶのがコツ！

⑧前提条件とシステム要件

  • Codexアプリをインストール済み（macOS / Windows）
  • Ollamaをインストール済み（バージョン0.24以上が必須）
  • 十分なVRAMとRAMを搭載したGPU
  • CPUコア数もモデルの実行速度に影響する
Ollamaが古いバージョンだと動かないので要確認！

⑨Can I Run AI — 互換性チェック

  • canirun.aiで自分のPCスペックを入力するだけで対応モデルを確認
  • GPU、VRAM、RAM、コア数を指定して診断
  • 各モデルの実行可能性と推奨サイズが一覧表示される
  • Gemma 4などはPCスペック次第で複数バリアントから選択可能
モデルを入れる前にまずはここでチェック！

⑩モデルのインストール手順

  • Ollamaを起動してバックグラウンドで実行しておく
  • OllamaのモデルカードからCLIコマンドをコピー
  • コマンドプロンプトに貼り付けてバリアントを指定 
# Gemma 4の軽量版をインストール
ollama run gemma4:e2b

# 4Bパラメータ版をインストール（約9.6GB）
ollama run gemma4:e4b
モデル名の後にコロン（:）でバリアントを指定するよ！

⑪Ollamaバージョンの重要性

  • Ollamaはバージョン0.24以上が必要
  • 古いバージョンではCodexとの連携が動作しない
  • インストール前に最新版へアップデートすること
  • アップデート後はOllamaのチャット画面でモデルが認識されているか確認可能
動画内でもアップデート忘れでエラーが出たよ！要注意

⑫CodexをOllamaで起動する方法

  • 新しいコマンドプロンプトを開く
  • 起動コマンドを貼り付けて実行
  • 推奨モデル一覧とローカルインストール済みモデルが表示される
  • 使用したいローカルモデルを選択してEnter 
# Ollama経由でCodexを起動
ollama launch codex-app
Codexアプリに「Powered by Ollama」と表示されたら成功！

⑬ビジュアル編集機能

  • Codexの内蔵ブラウザでローカルサーバーやサイトを直接読み込み可能
  • ページ上に直接アノテーションを描き、チャットで変更指示
  • オープンソースモデルを使ってビジュアル編集をローカルで実行
  • ほぼすべてのキャンバスをビジュアル編集できる柔軟性
画面に直接書き込んでAIに修正させる画期的なワークフロー！

⑭コードレビュー機能

  • Codex内でコードレビューをローカルモデルで実行可能
  • ワークスペースから離れずにコメントを残して反復可能
  • ローカルホストされたオープンソースモデルを活用
  • コードが外部に送信されないためセキュアなレビューが可能
社外秘のコードも安心してレビューできるね！

⑮デモ: SaaSランディングページ作成

  • フロントエンドタスクとしてSaaSランディングページの作成を指示
  • Gemma 4（4Bパラメータ）モデルで実行
  • 生成されたコードをコピーしてHTMLビューアーで確認
  • 4Bパラメータのローカルモデルでも高品質なコードを出力
わずか4Bパラメータでこのクオリティはすごい！

⑯ローカルモデルの実力

  • Gemma 4 E4Bは約9.6GBのダウンロードサイズ
  • ローカルモデルでもSaaSランディングページレベルの実用的な出力が可能
  • クラウドモデルと比較しても、軽量タスクなら十分な品質
  • Codexのハーネス（ツール群）をすべて活用可能
モデルが小さくてもCodexの強力なハーネスでカバー！

⑰元のCodexプランに戻す方法

  • Ollama連携から元のCodex体験に簡単に復元可能
  • コマンドプロンプトで復元コマンドを実行するだけ 
# 元のCodexプランに復元
ollama launch codex-app --restore
いつでも元に戻せるから安心して試せる！
