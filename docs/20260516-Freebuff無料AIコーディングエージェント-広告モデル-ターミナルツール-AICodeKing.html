Freebuff - 完全無料のAIコーディングエージェント

ターミナルで動く無料AIコーダー - 広告モデルで完全無料を実現

2026年5月16日 | AICodeKing

① Freebuffとは

  • ターミナルで動く完全無料のAIコーディングエージェント
  • Codebuffプラットフォーム上に構築されている
  • サブスクリプションなし、設定不要、数秒で起動可能
  • Mac・Linux・Windowsに対応
「無料」と聞くと怪しいと思うけれど、仕組みはシンプル。CLI内テキスト広告で運営されているんだ！

② 無料の仕組み: 広告モデル

  • 月額課金の代わりにテキスト広告をCLI内に表示
  • クレジット購入やサブスク登録が一切不要
  • ターミナル内のテキスト広告と引き換えに、フル機能が使える
  • 特に学生や個人開発者にとって魅力的なトレードオフ
後から制限が増える偽りの「無制限プラン」より、最初から広告モデルと明示している方が正直だよね！

③ インストールと起動

  • ターミナルを開いてコマンドを実行するだけ
  • 前提条件はNode.jsがインストール済みであること
  • プロジェクトフォルダに移動してコマンドを実行 
# インストール
npm install -g freebuff

# プロジェクトフォルダで起動
cd your-project
freebuff
わずか数秒でセットアップ完了。設定ファイルもルーターも不要！

④ シンプルな設計思想

  • モードなし、設定ファイルなし、すぐ動く
  • 多くのツールがモード・プロバイダー・ルーター・スイッチで複雑化している
  • Freebuffは逆方向へシンプル化している
  • 高速モデルと素早いコンテキスト収集で2〜5倍の高速化を謳う
ベンチマークの正確な数値は要検証だけど、設計思想は理にかなっているよ。

⑤ 9つの特化型サブエージェント

  • File Picker: 関連ファイルを自動で特定
  • Code Reviewer: コードの品質を自動レビュー
  • Browser Use: ブラウザを開いてアプリをテスト
  • Thinker GPT: ChatGPT連携で深い思考を実行
大きなタスクでは、ファイル検索・編集・レビューを別々のエージェントが分担するよ。ブラウザテストもできるのがすごい！

⑥ 対応モデル一覧

  • メインエージェント用:
  • DeepSeek V4 Pro、Kimi K2.6、DeepSeek V4 Flash、MiniMax M2.7
  • ファイル検索・リサーチ用: Gemini 3.1 Flashlight
  • 深い思考用（要ChatGPT契約）: GPT 5.4
単一の安いモデルじゃなくて、タスクごとに最適なモデルを使い分けているのがポイント！

⑦ ファイルメンション機能

  • @ファイル名で特定のファイルを指定可能
  • @エージェント名で特定のエージェントを呼び出し
  • コンテキストを絞り込んで効率的に作業できる 
# 使い方の例
@auth.ts このファイルの
  認証ロジックを説明して

@code-reviewer このPRを
  レビューして
メンションを使うと、エージェントが的確に必要な情報にアクセスできるよ！

⑧ プライバシーの注意点

  • コードはユーザーのものと公式に表明
  • ただし一部のモデルは学習用データ収集に使用される可能性
  • DeepSeekの一部オプションには「学習データとして収集」の明記あり
  • 会社のプライベートコードを扱う場合はモデル選択に注意
無料だからといって盲信しないこと！モデルのラベルを必ず確認しよう。

⑨ 推奨ワークフロー

  • /interview: 最初に質問して要件を明確化
  • /plan: 実装計画を策定
  • /implement: 計画に沿って実装
  • /review: コードをレビューして品質確認
大きな機能追加には質問→計画→実装→レビューの順が推奨。いきなり全権を渡すのは危険！

⑩ まずは低リスクで試そう

  • まずはコードの説明を頼んでみる
  • 次に小さな変更を依頼してdiffを自分で確認
  • 信頼できたら大きなタスクを段階的に任せる 
# ステップ1: プロジェクトの理解
「このプロジェクトで認証が
  どこで処理されているか教えて」

# ステップ2: 小さな変更
「この関数のエラーハンドリングを
  改善して」
最初から最重要プロダクションのリファクタリングを任せないこと。これは全コーディングエージェントに共通する鉄則！

⑪ ブラウザテスト機能

  • Browser Useエージェントがブラウザを操作
  • 自動でページを開いてクリック操作が可能
  • Webアプリの動作テストを自動化
  • 実装後にUIが正しく動くか確認できる
ターミナルからブラウザ操作までできるのは、無料ツールとしては超優秀！

⑫ フォローアップ提案

  • 各レスポンス後に3つのクリック可能な提案を表示
  • 「次にテストを走らせる？diffを確認？クリーンアップする？」
  • エージェントが終了した後の次のアクションに迷わずに済む
小さな機能に見えるけど、作業の流れが途切れないから実はめちゃくちゃ便利！

⑬ 有料代替案: GLM Coding Plan

  • Lightプランは月額約$18から
  • 5時間あたり約80プロンプトの利用可能枠
  • Claude Code・Cline・Open Code・Kilo Codeで利用可能
  • GLM 5.1は難易度の高いコーディングやエージェント的作業に最適
広告なしでたっぷり使いたい人はGLMプランがコスパ最強の選択肢！

⑭ 有料代替案: Verdant

  • Plan → Code → Verifyのワークフロー
  • Git Worktreeでワークスペースを分離
  • カスタムAPIキー対応（Anthropic・OpenAI・OpenRouter）
  • 外部エージェントランタイム（Codex・Claude Code）も接続可能
プロ向けの本格的なコーディングワークスペース。複数エージェント＋分離環境＋レビューループが完備！

⑮ ツール比較まとめ

  • 完全無料なら → Freebuff
  • 広告なし＋ generous枠なら → GLM Coding Plan
  • 計画・検証・Worktree・柔軟性なら → Verdant
GLM + Claude Code を Verdant に組み込む構成も可能。組み合わせて使うのが最强かも！
