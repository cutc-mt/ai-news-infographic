Kimi WebBridge - AIエージェントからいつものブラウザを操作

2026年5月16日 | Moonshot AI 提供

① Kimi WebBridgeとは

  • Moonshot AIが提供するChrome拡張機能
  • AIエージェントとブラウザを橋渡しするブリッジとして機能
  • 普段使っているブラウザ（Braveなど）をそのまま操作可能
  • 新しくブラウザを立ち上げる必要がなく、既存の環境を活用できる
エージェントからブラウザを操作できるようにする、
それがKimi WebBridgeの役割です！

② 対応するAIエージェント一覧

  • Claude Code - AnthropicのCLIツール
  • Hermes - 汎用AIエージェント
  • Codex - OpenAIのコーディングエージェント
  • Cursor - AI搭載エディタ
  • Kimi Code - Moonshot AIのコードツール
様々なAIエージェントから
統一的にブラウザを操作できるのが強みです

③ セットアップ手順（Chrome拡張）

  • kimi.com/features/webbridgeにアクセス
  • 「拡張機能」ボタンを押してChromeに追加
  • ブラウザにWebBridgeのプラグインが表示される
  • BraveなどのChromium系ブラウザでも動作可能
Chrome拡張のインストールは
クリック1回で完了します！

④ セットアップ手順（コマンド実行）

  • 「Connect WebBridge to your Agent」をクリック
  • 表示されたコマンドをコピー
  • ターミナルに貼り付けてEnterで実行
  • スキルがグローバル領域に自動インストールされる

⑤ 実際の使用例 - Hacker Newsを開く

  • Claude Codeで/webbridgeを実行
  • 「ハッカーズニュース開いて」と指示を入力
  • WebBridgeが裏側で確認しながら自動操作
  • ブラウザが自動的にHacker Newsを開く
自然言語で指示するだけで

⑥ ログイン認証を使えるのが強み

  • 普段使っているブラウザのログイン状態をそのまま活用
  • Cookieやセッション情報が引き継がれる
  • 新しいブラウザを立ち上げてログインし直す必要がない
  • この機能が最大のメリットと解説者が評価
Playwrightなどは新規ブラウザ起動なので
ログイン情報が引き継がれないんです

⑦ 裏側の仕組み（アーキテクチャ）

  • 3層構造で動作：AIエージェント→デーモン→Chrome拡張
  • デーモンがローカルで常駐してリクエストを中継
  • デーモンとChrome拡張がインタラクション
  • 結果としてブラウザ操作が実行される 
AIエージェント
    ↓ HTTP POST
デーモン（ローカル常駐）
    ↓ インタラクション
Chrome拡張
    ↓ 操作
ブラウザ（Brave等）

⑧ HTTP POSTリクエストの仕組み

  • AIエージェントがデーモンに対してHTTP POSTでリクエストを送信
  • 開発者ツールでリクエストのやり取りを確認可能
  • デーモンがリクエストを受け取り、Chrome拡張に命令を渡す
  • Chrome拡張が実際のブラウザ操作を実行
HTTPリクエストの流れを
開発者ツールで覗けます！

⑨ Playwrightや他ツールとの比較

  • Playwrightは新規ブラウザを起動する方式
  • Playwrightはログイン状態が引き継がれない
  • WebBridgeは既存のブラウザを借用する方式
  • 動作確認だけならPlaywrightでも十分。ログインが必要な場合はWebBridgeが有利
用途に応じて使い分けるのが
一番の正解みたいです

⑩ Claude in Chrome / Codex in Chromeとの違い

  • Claude in ChromeはClaude専用のブラウザ統合機能
  • Codex in ChromeはCodex専用の同様の機能
  • WebBridgeは複数エージェントに対応する統一的なアプローチ
  • スキルとして配布されるため、自作エージェントにも組み込みやすい
専用機能ではなく、
どんなエージェントでも使えるのがポイント！
