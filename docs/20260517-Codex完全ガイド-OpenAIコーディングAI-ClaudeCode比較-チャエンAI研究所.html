初めてのCodex — Claude Codeとどっちを使うべきか完全解説

Codexとは何か

① Codexの正体：ChatGPTとの違い

  • ChatGPTは質問に答えるだけのアシスタント
  • Codexは実際にお願いして、動かして、完成までさせるエージェント
  • コードを書いて、エラーを直して、実際に実行までできる
  • メールを作って送信するといった行動まで責任を持って実行できる
ChatGPTは「答える人」、Codexは「やってくれる人」ってイメージだね！

② 4つの利用環境が用意されている

  • アプリ：デスクトップアプリで手軽に使える
  • IDE拡張機能：VS CodeやCursorの拡張機能として統合
  • CLI：Claude Codeのようなターミナルでの利用
  • クラウド：ブラウザ上でも利用可能
初心者にはアプリ版が一番おすすめ！

③ アプリ版が初心者に最適な理由

  • OpenCodeの開発者が実際にCodexのアプリ版でAIエージェントを作成したと公言
  • OpenAIが約200億円でOpenCodeを買収したとの噂
  • ターミナルにこだわる必要はなく、デスクトップアプリでも十分に実用的
  • チャエン氏自身もClaude Codeをターミナルからアプリ版に移行したと話している
ターミナルが苦手な人でも大丈夫！アプリ版で十分なんだよ
Claude Codeとの比較

④ Claude Code vs Codex：違いはある？

  • 基本的にはどちらも同じような機能を提供している
  • 最大の違いは裏側のモデル
  • Claude CodeはClaude Opus 4.7やSonnetを使用
  • CodexはGPT 5.5などのChatGPTモデルを使用
機能の差はほぼないけど、モデルの差が品質に直結するよ！

⑤ 画像生成ができるのはCodexだけ

  • AnthropicのClaudeには画像モデルや動画モデルがない
  • Claudeで画像生成するにはAPIを経由する必要がある（1回約20円）
  • CodexはGPT Image 2を直接使えて追加課金なしで画像生成が可能
  • GPT Image 2の精度は非常に高く、資料作成にも活用できる
画像生成はCodexの大きな強み！Claudeにはない機能だよ

⑥ プレビュー機能が地味に便利

  • 作成したExcelやPowerPointをその場でプレビュー表示できる
  • Claude Codeにはないプレビュー機能がCodexにはある
  • スプレッドシートもガッツリ作成可能。Claude CodeよりExcel作成に強い印象
  • 成果物をその場で確認できるため、修正のやり取りがスムーズ
作ったファイルをすぐ確認できるのは地味に便利だよね！
GPT 5.5モデルの登場

⑦ GPT 5.5がCodexを強力にしている

  • GPT 5.5が登場し、Codexの能力が大幅に向上
  • 処理速度が高速で、開発能力も非常に高いと評価されている
  • 界隈での評価は高めで、チャエン氏も個人的にお気に入りと話している
  • OpenAI全体として開発スピードが加速しており、Claude陣営も対抗が必要に
GPT 5.5は速くて賢い！Codexを使うならこのモデルがおすすめ

⑧ モデル選び：迷ったらGPT 5.5

  • Codexでは複数のモデルから選択が可能
  • GPT 5.5が最新かつ最高品質のモデル
  • 良いモデルほど使用リミットが早く来る点に注意が必要
  • ファストモード：速度は1.5倍だが、クレジット消費は2倍になる
品質を重視するなら5.5、節約するなら下位モデルでバランスよく！
料金プランとインストール

⑨ 料金プラン：追加課金なしで始められる

  • 無料プランも一応使えるらしい（チャエン氏は未検証）
  • Plus プラン（$20/月）が推奨。ChatGPT既存の課金者は追加なしで利用可能
  • Pro プラン（$200/月）は事業者向けで制限が大幅に緩和
  • Claude Codeがメインの人はPlusプランで十分とのこと
ChatGPTを既に有料で使ってる人は、追加課金なしでCodexが使えるよ！

⑩ インストールはたった3ステップ

  • ChatGPTアカウントを用意（無料または有料）
  • 公式サイトからCodexアプリをダウンロード
  • サインインしてすぐに利用開始
  • 必ず公式サイト（openai.com）からダウンロードすること。偽サイトに注意！
変なサイトからダウンロードしないでね！公式サイトをちゃんと確認して！

⑪ 権限設定：最初は安全に始めよう

  • 自動レビューが最初は安全。人間が確認しながら進められる
  • フルアクセスは自分のパソコン全部に触れる権限。リスクが高い
  • 慣れてからフルアクセスに移行するのがおすすめ
  • 設定画面でデフォルト権限のオン/オフも変更可能
セキュリティが心配なら、まずは自動レビューで安全に始めよう！
基本操作と実務活用

⑫ 基本操作：日本語でOK、プログラミング不要

  • プログラミング用語は一切不要。普通の日本語で依頼するだけ
  • 「このフォルダーの中身を整理して」といった自然な指示でOK
  • 「サイトを作って」と頼むだけで、Webサイトを自動生成
  • 基本的にはChatGPTと同じ感覚でチャットして使う
普通に話しかけるだけで動いてくれる。すごい時代になったね！

⑬ ファイル参照でコンテキストを追加

  • チャット欄の＋ボタンからファイルやプラグインを選択可能
  • 自分のローカルファイルを直接参照させることができる
  • スプレッドシートも読み込ませて、データ分析や処理が可能
  • ファイルを渡して「これをもとに資料を作って」と依頼するだけで完成
ファイルを直接読み込ませるのがコツ！精度が全然違うよ

⑭ オートメーションで自動化も可能

  • 左サイドのオートメーションから自動実行の設定ができる
  • プロジェクト：自分のフォルダーを指定して作業するモード
  • チャット：通常のチャット形式で気軽に使うモード
  • プロジェクトやモデル、時間などを選んで定期実行も設定可能
Claude Codeにはチャットモードがないから、初心者にはCodexが使いやすいかも！
設定画面の活用とClaude Codeからの移行

⑮ 設定画面：コーディング向けと日常業務向け

  • コーディング向け：開発者向けの設定プロファイル
  • 日常業務向け：プログラミングが苦手な方向けの設定
  • 設定でフルアクセスをオフにしておくと安全に使える
  • Claude Codeより設定画面が丁寧で、UIが分かりやすい
最初は日常業務向けで始めて、慣れたらコーディング向けに切り替えよう！

⑯ サンドボックスで安全に作業する

  • サンドボックスは架空の環境で安全に作業できる仕組み
  • 「承認しない」に設定すると、作業フォルダーの外には一切アクセスできない
  • リクエストごとにサンドボックスを適用するかどうかも選べる
  • リードオンリー（読み取り専用）モードもあり、最も安全な設定
初心者は「リクエストごと＋リードオンリー」が一番安全！

⑰ Claude Codeからの移行が超簡単

  • インポート機能を使えば、Claude Codeの環境をそのまま移管できる
  • CodexやClaude Codeのツール、チャット、プロジェクトをすべて移行可能
  • 設定の一般タブにあるインポートボタンから実行
  • 移行後はほぼClaude Codeと同じ環境で作業できる
Claude Codeから乗り換えたい人も安心！環境ごと移せるよ

⑱ まとめ：どっちを使うべきか

  • チャエン氏のメインは引き続きClaude Code
  • 両方を併用するのが現実的なアプローチ
  • 画像生成やスプレッドシート作成はCodexが得意
  • 今後はMCPやフック、メモリー機能などエンジニア向けの詳細設定も第2弾で解説予定
どちらか一方ではなく、用途に合わせて使い分けるのが正解！両方試してみよう
