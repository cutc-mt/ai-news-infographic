Google Antigravity 2.0

無料エージェントIDEが大幅アップデート - より安全で使いやすく進化

2026年5月17日

① Antigravity 2.0の全体像

  • GoogleのエージェントIDE「Antigravity」が大幅にアップデートされた。
  • 多くのバグが修正され、以前よりも圧倒的に使いやすくなっている。
  • 一見すると更新がないように見えるが、内部的には着実に進化している。
  • 完全無料で利用可能で、フリーティアでもすべてのモデルにアクセスできる。
Google I/OでJewelsとの統合が発表される可能性が高い！AI StudioもすでにAntigravityベースで動いている。

② 大幅なポリッシュアップ

  • 以前の厄介なバグの多くが修正され、快適に動作するようになった。
  • モデルの動作が正確で安定している。
  • エージェントの仕組みもスムーズに動作するよう改善。
  • 全体的なUI体験が向上し、プロフェッショナルな使い心地に。
初回リリース時の粗さがほぼ解消。実際の作業フローに耐えうる品質に到達！

③ Google I/Oで発表予定

  • Google I/OでAntigravityとJewelsの統合が発表される見込み。
  • 統合によりオンラインでエージェントを起動できるようになる。
  • Google AI StudioはすでにAntigravityベースで稼働中。
  • Google Cloudの一部サービスもAntigravityで動いている。
Google全体のエージェント戦略の中心にAntigravityが位置づけられている証拠！

④ Strict Mode（厳格モード）

  • Strict Modeを有効にすると、エージェントの自律的な攻撃実行を防止。
  • すべてのエージェントアクションに人間のレビューが必須になる。
  • エージェント設定パネルから簡単に切り替え可能。
  • セキュリティ重視の環境で安心して使える。
以前のSecure ModeからStrict Modeに名称変更。機能はより洗練された！

⑤ レビューポリシーと権限管理

  • Review Policyでエージェントの行動を細かく制御できる。
  • ターミナルコマンドの自動実行設定をカスタマイズ可能。
  • Terminal Sandboxで安全な実行環境を提供。
  • シェル統合やパーミッション設定も充実。
コーディングエージェントにおいて権限管理は最も重要。デモで終わらず実務で使えるかどうかの境界線！

⑥ 通知設定の追加

  • Notification Settingsが新しく追加された。
  • 作業が完了した時などに通知を受け取ることができる。
  • 通知の種類やタイミングを自由にカスタマイズ。
  • バックグラウンドでエージェントを動かす時に非常に便利。
長時間タスクを投げっぱなしにできるから、作業効率が劇的に向上する！

⑦ モデル画面：レート制限の透明化

  • 新しいModels画面でレート制限が完全に透明化された。
  • 各モデルのレート制限を個別に確認できる。
  • 制限を超える場合はAI Creditsを追加して利用可能。
  • レート制限は約4時間ごとにリフレッシュされる。
フリーティアでも全モデルが利用可能で、制限が見えるから計画的に使える！これは大きな改善。

⑧ フリーティアで全モデル利用

  • 完全無料で複数のAIモデルにアクセスできる。
  • 各モデルに固有のレート制限が設定されている。
  • 利用状況を透明に把握できるので計画が立てやすい。
  • 必要に応じてAI Creditsを追加購入することも可能。
無料でここまで使えるのはかなり太っ腹。他の有料ツールと比較してもお得感が強い！

⑨ スキルとMCPサーバー対応

  • CustomizationsタブでスキルとMCPサーバーを設定。
  • Google関連のMCPサーバーが直接利用可能。
  • ディレクトリを指定してスキルを自動検出できる。
  • ClaudeやOpenCodeのスキルを追加して連携も可能。
MCPが正常に動くことはエージェントIDEにとって基盤。外部ツール連携の要！

⑩ ブラウザコントロールの新機能

  • Browser Toolsを有効にすると、エージェントがブラウザを操作。
  • URLを開いてWebページを読み込み、コンテンツと対話できる。
  • 重要な知識の取得やバリデーションに役立つ。
  • タブやエディタのオプション設定も追加された。
ブラウザ操作ができることで、エージェントの自律性が大きく向上！

⑪ JavaScript実行ポリシーとURL許可

  • Disabled: JSコードを一切実行しない（最も安全）。
  • Request Review: 実行前に必ず許可を求める。
  • Always Proceed: 自動で実行（最大の自律性だがリスクあり）。
  • URL Allow Listでアクセス先を限定できる。
Always Proceedは便利だがセキュリティリスクが最も高い。実案件ではRequest Review推奨！

⑫ エージェントマネージャーの改善

  • Agent ManagerのUIが大幅に刷新された。
  • 使用するモデルの選択と変更が簡単に。
  • Codexと同様にプロジェクトフォルダの変更がワンクリックで。
  • 環境設定は現在Localのみだが、将来的にRemoteも追加予定。
プルダウンメニューが用意されているので、リモート環境の追加は間近かも？

⑬ 会話履歴とチェンジペイン

  • Conversation Historyですべてのワークスペースと会話を閲覧。
  • 新しいChanges Paneで変更内容を一覧表示。
  • セッション全体の変更を一目で確認できる。
  • 会話の名前の変更も可能で、整理しやすくなった。
変更内容が可視化されることで、チーム開発でも安心して使える！

⑭ レビュー機能で品質管理

  • Review機能でウォークスルー中にコメントを追加。
  • コメントに基づいてエージェントが自動的に修正を実行。
  • コードレビューの対話型ワークフローが実現。
  • 人間とAIのコラボレーションがよりスムーズに。
指摘→修正→確認のループがシームレス。これが実務で使える最大の理由！

⑮ Linux対応：カスタムプロバイダー

  • Linuxユーザーがカスタムプロバイダーを設定可能に。
  • Ollamaなどのローカルモデルツールを統合できる。
  • 独自のモデルを自由に設定して活用可能。
  • サンドボックス機能もLinuxに対応済み。
Ollama統合で完全ローカルのエージェント環境が構築できる！コストゼロで動く。

⑯ チェンジログ：活発な開発の証

  • 4月16日ビルド: MCPサーバー読み込み修正、ワークスペース設定修正。
  • 4月7日: 統一エージェント権限システムを追加。
  • 3月: Linux向けサンドボックス、MCP認証改善、チャットUI簡素化。
  • 2月: ファイル削除バグの修正など重要な修正を含む。
派手な新機能だけでなく地味な基盤部分を着実に改善。これが成熟の証！

⑰ セキュリティとサンドボックスの進化

  • 12月: Secure Mode追加。全アクションに人間レビュー必須化。
  • 1月: macOS向けターミナルサンドボックスを追加。
  • 3月: Linux向けサンドボックスも対応完了。
  • 2月: 変更取り消し時にファイルが削除されるバグを修正（重要）。
エージェントが実際のファイルを編集する以上、信頼性の確保は最優先課題。着実に前進中。

⑱ 総評：静かに熟成されるエージェントIDE

  • 権限管理、サンドボックス、MCP信頼性、ルール、ワークフローなど基盤が着実に強化。
  • まだVerdantのレベルには達していないが、着実に追いついている。
  • Windsurf由来の古い要素のリフレッシュが期待される。
  • 「派手な再ローンチ」ではないが、実務で使えるツールとして確実に進化している。
初回リリースで試して離れた人にももう一度試す価値あり。退屈な改善こそが本物の進歩！
