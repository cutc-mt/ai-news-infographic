Codex × Ollama 〜 小規模言語モデルに推論を実装 〜

二段階プレ出力システムでSLMを超ブーストする画期的アプローチ

2026年5月18日

① プロジェクトの目的：ローカルSLMに推論を実装

  • Codex（コーディングAI）を使って、Ollamaのローカル小規模言語モデルに推論能力を実装する試み。
  • 現在のリーズニングモデル（o1など）のような高度な思考プロセスを、小規模言語モデルでも再現できるかが最大の見どころ。
  • ローカルモデルはノイズが少なく安全性が高いが、複雑なタスクが苦手という弱点がある。
  • アプリケーション側で推論を強化し、大規模言語モデルに匹敵する深みのある出力を目指す。
小さいモデルでも工夫次第で深い回答が出せるようになる！

② 小規模言語モデルの弱点とCoT不足問題

  • 小規模言語モデル（SLM）は軽量で安全だが、CoT（思考の連鎖）が組み込まれていない。
  • 現在の大規模モデルには内部的にリーズニング機能が組み込まれているが、SLMにはそれがない。
  • 自己回帰モデルは「次の単語を予測する」仕組みのため、段階的な推論が苦手。
  • 複雑なタスクをこなせなかったり、深い回答が出なかったりするのが大きな課題。
一気にまとめて答えを出してしまうため、分解する力がないのが弱点

③ 文字カウント問題で見るSLMの限界

  • 「スターゲートプロジェクト」の中にTはいくつある？正解は3つ。
  • SLMは「1つ」や「2つ」と間違えて回答。自己回帰モデルの典型的な弱点。
  • 対策として、Pythonを使って文字を分解してカウントする方法も紹介。
  • プレ出力（事前思考）を使うと、タスクを分解してから考えるため正解に到達できる。
トークン境界に依存する処理ミスを防ぐには、事前にタスクを分解させるのが効果的。

④ 二段階プレ出力推論システムの仕組み

  • 第1段階：プレ出力（事前思考）— AIに論理プロセスを構築させる。
  • 第2段階：メイン出力（本番出力）— プレ出力の結果をもとに最終回答を生成。
  • エージェントループの技術を応用し、役割の異なるプロンプトを自動で連続投入。
  • プレ出力のプロンプトはユーザーには見えず、最終結果のみが表示される。
AI自身に思考プロセスを強制させる環境を構築！

⑤ O1型推論をSLM上で再現する発想

  • 2024年9月、OpenAIがo1モデルを発表。4o（自己回帰）とo1（推論）の二本立て。
  • このo1のような推論機能を、ローカルの小規模言語モデルで再現しようという発想。
  • 内部的には「事前プレ出力 → メイン出力」の二段構え。
  • 大規模モデルと同等の推論能力を、軽量なローカル環境で実現することを目指す。
小規模言語モデルをo1のようにしていくというイメージ！

⑥ エージェントループによる自己改善・反復推論

  • AI共創Local Novaでは、最大9999回までエージェントループが可能。
  • 「前回の出力をフィードバックして改善・ブラッシュアップ」を自動で繰り返す仕組み。
  • ループを重ねるごとに、出力の深みと精度が向上していく。
  • ループ回数は3回・4回・5回と自由に調整可能。
# エージェントループの流れ
1. ユーザー入力 → AI出力
2. フィードバック自動投入
3. ブラッシュアップ出力
4. 2-3を指定回数分繰り返し
→ 最終的な深みのある回答

⑦ 論理プレ出力モード：深掘り型推論

  • タスクの方向性が明確な場合に使用するモード。
  • 単純なCoT（ステップバイステップ）ではなく、まずタスク自体を細かく分解する。
  • 分解した要素から論理的に深掘りを行い、精緻な回答を構築していく。
  • プロンプトは設定画面から書き換え可能で、自分好みの思考プロセスを設計できる。
1つのテーマを徹底的に深掘りしていくのが論理モード！

⑧ 探索プレ出力モード：仮説生成型推論

  • 答えが導き出しにくい難解なタスクや創造的なアプローチが必要な場面で使用。
  • Tree of Thoughts（ToT）の概念を取り入れ、表形式で5つの仮説を並列出力。
  • 直列思考に陥るのを防ぎ、多角的に可能性を探索してから最適な枝を深掘りする。
  • 仮説を評価し、最適なものを選択して最終出力を生成する。
いきなり深掘りせずに、まず広く探索してから絞り込む！

⑨ 論理と探索の使い分け戦略

  • 論理プレ出力は、タスクがすでに分かっている場合に有効。1つの方向を深く追求。
  • 探索プレ出力は、未知のタスクに最適。広く仮説を立ててから方向を決める。
  • 未知タスクで論理モードを使うと、最初の方向が間違っていた場合に間違ったまま深掘りしてしまう危険がある。
  • おすすめの流れ：探索 → 当たりを見つける → 論理で深掘り。
探索して当たりを見つけてから深掘りすると確率と精度が上がる！

⑩ 並列思考で直列の罠を回避する仕組み

  • 箇条書き形式で仮説を出させると直列思考になってしまう（上から順に出力される）。
  • 対策として表形式で横並びに仮説を出力。各仮説が独立して生成される。
  • いきなり1つの答えを出さず、まず仮説のタイトルだけを並列で出力させる。
  • 最初の仮説に引きずられず、真に多角的な探索が可能になる。
自己回帰モデルは最初の確率が最も高いものを選ぶため、直列だと1つの方向に偏ってしまう。並列表形式で多様性を確保する工夫が重要。

⑪ AI共創Local Novaとたぬきモデルの紹介

  • AI共創Local Novaは、Ollamaの出力を投映する専用アプリ。GitHubから無料ダウンロード可能。
  • たぬきモデルは松尾研が作成したフルスクラッチの日本語モデル。
  • たぬきの特徴は柔らかく共感的で温かみのある文章が書けること。
  • 完全ローカル動作。ネットを切っても動作するオフライン環境。
GitHub: AICo-Creation/localnova からダウンロード！たぬきは名前も可愛いよね

⑫ 通常出力と論理プレ出力の品質比較

  • 通常出力：「人生とは成長と変化の連続」というSF風の回答。速いが深みに欠ける。
  • 論理プレ出力：発達心理学的に「幼少期→思春期→成人期→退職後→終焉」とライフサイクルで構成。
  • 論理出力ではエリクソンの発達段階も引用されるなど、学術的な深みが出る。
  • 生成速度は推論モデルとほぼ同等。プレ出力のオーバーヘッドは最小限。
同じたぬきモデル・同じプロンプトでも、プレ出力の有無でここまで違う！

⑬ たぬきモデルの日本語表現力の特徴

  • 日本語ネイティブで作られたモデルなので、非常に自然な日本語が出力される。
  • 「人生とは一言で言えば成長と変化の連続と言えるでしょう」のように丁寧で温かい文体。
  • 共感的な文章を書きたい時にたぬき + 論理プレ出力の組み合わせが強力。
  • 温かみのある文章にロジックが加わることで、説得力と深みが両立する。
たぬきの温かみ＋論理の深さ＝最強の組み合わせ！

⑭ 「人生とは？」を論理モデルで深掘り

  • 論理プレ出力では「人生」をタスク分解してから思考を開始。
  • 幼少期の経験 → 思春期の葛藤 → 成人期の責任 → と段階的に深掘り。
  • 通常出力にはない「意識と自己意識」「瞑想や内省」といった深い概念が登場。
  • 最後に全要素を統合した結論で締めくくる構造的な文章。
行けば行くほど深いテキストが出てくる！

⑮ 「人生とは？」を探索モデルで多角的分析

  • 5つの仮説を並列生成：①個人的成長と変化 ②人間関係の複雑さ ③ストレス管理 ④自己実現と幸福感 ⑤時間の有限性。
  • 評価の結果、仮説①と④を総合した回答を選択。
  • さらにマズローの自己実現欲求や、時間の有限性も採用され多角的な文章に。
  • 複数の仮説を掛け合わせることで、単一方向では出ない深みのある文章が生成される。
自己回帰モデルなら仮説1だけだが、探索モデルは複数を組み合わせる！

⑯ 推論モデルではなく論理モデルと呼ぶ理由

  • 本来、自己回帰モデルの方が「次の言葉を予測する」という意味で推論的。
  • CoTを組み込んだリーズニングモデルは、タスクを分解して論理的に構築するもの。
  • 「Reasoning」の訳語として「推論」を使うのは概念のずれが生じると指摘。
  • 本来の言葉の意味に即すると、論理モデルという呼称が適切だと主張。
「他力本願」や「お客様は神様」と同じように、言葉の意味がどんどんずれていく…

⑰ 概念設計と表記揺れがAI長文生成に与える影響

  • 表記揺れ（同じ概念を異なる言葉で表すこと）は、AIエージェントを走らせるとコンシステンシーが崩れる原因になる。
  • 1冊の本を書くような長時間タスクでは、定義がずれていくと矛盾が蓄積していく。
  • 概念エンジニアリング：純粋出力で定義を確定させ、表記揺れを防ぐ考え方。
  • 概念設計ができないと、長文生成やエージェントループでの品質が安定しない。
定義をしっかり確定していくことが、AIを使う上でも大事！

⑱ 推論の民主化とローカルモデルの未来

  • 今後、大規模言語モデルが高価なものになる可能性がある。推論には膨大な電力と計算リソースが必要。
  • 一般庶民が使えなくなったり、モデルの格差が拡大する懸念がある。
  • ローカルモデルで推論を実装できれば、誰もが無料で高度なAIを利用可能に。
  • 推論の民主化、論理思考と探索思考の普及が、このプロジェクトの大きな意義。
小さいモデルでも工夫すれば、皆が高度な推論を使えるようになる！
