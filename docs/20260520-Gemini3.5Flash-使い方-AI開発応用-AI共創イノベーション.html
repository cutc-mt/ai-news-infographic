Gemini 3.5 Flashの使い方とAI開発への応用

Google I/O 2026で発表の新世代モデル「動的な実行力」を徹底解説 | 2026年5月20日

①Gemini 3.5 Flashとは

  • 静かな賢さよりも動的な実行力が特徴のモデル
  • 「頭」ではなく「手足」として動いてくれる強さがある
  • タスクの実行やツールの利用に特化した設計
  • 実行知能においてはGPT5.5やClaude Opus 4.7をも凌駕
抽象的な推論よりも、現実のタスクを完遂させることに最適化されたAIなんだ！

②Google I/O 2026で発表

  • Google I/O 2026で多数のアップデートとともに発表
  • 従来の「軽量モデルは賢くない」という常識を完全に覆すモデル
  • 上位モデルのGemini 3.1 Proすら凌駕する実行知能
  • Antigravityやブラウザ操作にも標準搭載されている
迷ったらまず3.5 Flashを使ってみるのがおすすめ！

③Sparse MoEアーキテクチャ

  • 巨大な1960億パラメータを持ちながら推論時はわずか110億だけ活性化
  • 全ての専門家を呼び出すのではなく、必要なものだけを呼び出す仕組み
  • 軽くて賢く、軽いから早いという相乗効果
  • サクサク動いてインタラクティブにもエージェンティックにも対応
196B→11Bに絞り込むから高速で低コスト！

④MTP-3技術で爆速出力

  • 出力速度は毎秒280トークン以上で他社の約4倍
  • MTP-3（Multi-Token Prediction）で複数トークンを同時に予測
  • 高速かつ無駄のない出力が可能に
  • ロボットやフィジカルAIにも応用できるスピード
素早い対応はこれからのAI開発でますます重要になる！

⑤ベンチマーク比較

  • Terminal-Bench 2.1: GPT5.5に迫るコーディング性能
  • MCP Atlas: エージェントのツール利用で1番高いスコア
  • GDPval-AA: ビジネス価値を測る指標でもトップクラス
  • ファイナンスエージェントやエキスパートタスクでも1位
学術的な知識より「現実のタスクを完遂する力」が圧倒的！

⑥コーディング性能が優秀

  • Terminal-BenchでGPT5.5に次ぐ高スコアを記録
  • ミスをせずにちゃんとコーディングしてくれる
  • UIコントロールもGPT5.5レベルで高評価
  • SoftBenchでも高スコアでコーディング全般に強い
初動のコーディングに使うと非常に優秀な結果が出るよ！

⑦エージェント性能がトップ

  • ツール利用のスコアが全モデル中で最高レベル
  • MCP Atlasでマルチステップのツール呼び出しに最適化
  • エラー修正の反復ループも高速でこなす
  • 専門的な知識にも対応できるエキスパートタスク性能
何かを実行させたい時はこのモデルを使うのが正解！

⑧マルチモーダルも強力

  • 画像を入力して「これは何？」と聞くのに最適
  • 動画の入力にも対応してマルチモーダル性能はトップ
  • 音声入力も強力で従来からGeminiの得意分野
  • ロングコンテキスト1Mトークンで超高速処理が可能
画像・動画・音声を突っ込んで質問する使い方が得意！

⑨思考レベルの最適化

  • 4段階で思考の深さを調整できる：Minimal・Low・Medium・High
  • 通常の実務にはデフォルトのMediumが最適
  • 単純作業はMinimalやLowでスピード優先
  • 複雑なバグ修正や高度な推論にはHighを設定
拡張モードにするとReasoning Thinkingが使えるよ！

⑩虫の目と鳥の目のバランス

  • 虫の目（スライディングウィンドウ）: 直近の文脈を細部まで読み取る
  • 鳥の目（グローバルアテンション）: 全体の文脈を俯瞰する
  • 黄金比は3:1（ローカル:グローバル）
  • このハイブリッド構造が100万トークンの超高速処理を実現
直近だけ見ると全体が崩れる。全体だけだと重すぎる。3:1が最適解！

⑪コンテキストキャッシュでコスト削減

  • API入力価格は100万トークンあたり$1.50
  • キャッシュ機能を活用すると90%オフの$0.15まで圧縮
  • 巨大なコードベースや社内マニュアルを一度読み込ませれば長時間低コストで運用可能
  • 長時間の自律タスクでもコストを気にせず実行できる
キャッシュを使えば10分の1のコストで済むから超お得！

⑫API価格とコスト比較

  • 内部推論（Thinking）の計算量が増加しているため従来よりAPI価格は高め
  • しかしGPT5.5と比較すると大幅に安い
  • 推論の計算量が多い主な理由は内部の思考（Thinking）プロセス
  • 使うところを工夫すれば十分にコストパフォーマンスが良い
高性能モデルの Thinking はどれも計算量が増えている傾向があるよ。

⑬URL管理アプリの実演

  • URLを保存するとサムネイルアイコンが表示される管理アプリを作成
  • ドラッグ＆ドロップでアイコン移動、名前の変更も可能
  • アイコン形式とリスト形式の切り替えにも対応

  • 圧倒的に3.5 Flashが優秀という結果に
