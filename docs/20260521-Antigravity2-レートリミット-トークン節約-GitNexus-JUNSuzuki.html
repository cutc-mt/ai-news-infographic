【Antigravity2】利用制限とリフレッシュのルール＆トークン節約術

①Antigravity2マスタークラスが開講

  • 無料のオンライン学習コースがリリースされました。ユーザー登録するだけで誰でも受講可能です。
  • 全16章・24レッスン・78のクイズが用意されています。
  • 進捗管理機能も備わっており、どこまで学習したかを一目で確認できます。
  • 初心者向けに設計されており、英語のドキュメンテーションを日本語で体系的に学べます。
CLI、IDE、SDK、スキル、プラグイン、アーティファクトなどの基本概念を1から理解できるのが魅力です！

②レートリミットの基本メカニズム

  • 制限はチャット回数ではなく、トークン使用量とモデルの種類に依存します。
  • コードベースが大きくなるほど、消費トークンも増加して制限に達するサイクルが短くなります。
  • 制限に達すると、設定画面の「モデル図」で次のリフレッシュまでの時間を確認できます。
  • 同じ時間使っていても、プロジェクトの規模や選んだモデルによって到達タイミングが大きく異なります。
シンプルに「何回使えるか」ではなく、トークン消費量で決まるのがポイントです！

③プラン別のリフレッシュサイクル

  • ウルトラ純粋な枠が提供され、5時間ごとにリフレッシュされます。
  • プロ同様に5時間ごとにリフレッシュ。ただし週間制限に達すると次の週まで待つ必要があります。
  • 無料5時間ごとのリフレッシュがなく、週間制限に達すると次の週まで完全に利用不可です。
ウルトラとプロは5時間ごとに回復するのが強み。無料プランは週単位の回復なので注意です。

④モデルラインナップの新体系

  • Flash Liteが最も高速で安価なモデル。今までの「Flash」の位置づけに近いです。
  • Flash（3.5 Flash High）は新しいミドルクラスのモデル。低価格ハイスピードではなくなりました。
  • Proがハイレベルなモデルとして位置づけられています。
「Flash」という名前が以前と同じでも、位置づけが変わったので要注意です！

⑤Antigravity 2.0の4つの提供形態

  • デスクトップ版（Antigravity 2.0）はCursorやCodexのデスクトップアプリに近いUI設計です。
  • CLIはコマンドラインから操作するツールです。
  • SDKは開発者向けのソフトウェア開発キットです。
  • IDEはVSCodeフォークとして提供されるエディタ。エクステンションや日本語化に対応。
古いバージョンをアップデートすると2.0デスクトップ版になってしまうので注意！

⑥旧IDEからの移行で起きた混乱

  • 既存のAntigravity（旧バージョン）をアップデートすると、2.0デスクトップ版に強制移行されてしまいました。
  • IDEの設定やカスタマイズ内容が消えてしまうケースも報告されています。
  • 新しいバージョンはまだ英語のままなので、日本語ユーザーには使いにくい状況です。
  • 解決策として、別途IDE 2.0をダウンロードすれば旧環境に近い使い方が可能です。
海外でも不評が多く、ダウンロードリンクの間違いなどもあったようです。初期のつまずきポイントですね。

⑦設定画面でリフレッシュ状況を確認

  • 左下の「Setting」からモデル図を開くと、トークン使用量を確認できます。
  • リフレッシュまでの残り時間（何時間後・何日後）が表示されます。
  • どれくらいトークンを消費したかの累計も確認可能です。
  • リフレッシュサイクルは5時間が基本となっています。
制限に達する前にこまめにチェックして、作業のペースを調整するのがおすすめです！

⑧制限に達したらどうする？ツール切り替え戦略

  • 1つのツールで制限に達したら、別のツールに切り替えるのが最もシンプルな対策です。
  • Antigravity → Claude Code → Codex → OpenCodeのようにローテーションさせます。
  • 複数のサブスクを組み合わせることで、各サービスの利用枠を有効活用できます。
  • 月額$10〜$30程度のライトユーザーでも、ツールを渡り歩けば継続して作業が可能です。
ただし切り替え時にコードベースを読み直すトークンの無駄遣いが発生するのが課題です…。

⑨AionUI：エージェントのオーケストレーター

  • AionUIは各種コーディングエージェントを統合管理するオーケストレーションツールです。
  • OpenCode、Claude Code、Codex、Antigravityなどを1つのUIで統合管理できます。
  • Claude Codeをオーケストレーターにして、各エージェントに指示を出す使い方も可能です。
  • 複数エージェントを協調させることで、トークンの節約にもつながります。
エージェント同士の「共同作業部屋」を作るようなイメージです！

⑩GitNexus：コードベースのナレッジグラフ化

  • GitNexusはコードベースをナレッジグラフ化するツール。「神経ネットワーク」のようなものです。
  • CLIとMCPが標準装備されており、主要なコーディングツールからアクセス可能です。
  • インデックスはローカルPCに保存され、グローバルレジストリーで一元管理されます。
  • どのエージェントがアクセスしても、同じ構造理解でスムーズに作業を開始できます。
どのツールからアクセスしてもコード構造を完全に理解した状態からスタートできるのが最大のメリットです！

⑪GitNexusの事前計算による一括回答

  • インデックス作成時に、コードの構造・クラスタリング・実行フローのトレース・関連性スコアリングを事前計算。
  • AIエージェントを1回呼び出すだけで、完全に構造化されたコンテキストを取得可能です。
  • 呼び出し元の数、関連度の強度、変更の影響範囲を一発で把握できます。 
従来: LLMに生データを渡し、何度もクエリを重ねる
GitNexus: 事前計算済みのナレッジグラフから一括取得
「この変更をするとどこのファイルのどの処理に影響するか」が一発でわかるのがスゴイ！

⑫トークン効率が大幅に向上する理由

  • 従来の問題点：1つの関数を理解するだけで、10回もクエリを連鎖させる必要がありました。
  • GitNexusの解決策：事前計算により、1回の呼び出しで完全なコンテキストを取得できます。
  • クエリの連鎖が不要になるため、トークン消費を大幅に削減できます。
「呼び出し元を探す」「対象ファイルはどれか」「テストコードを除外する」…こういうステップが全部不要に！

⑬ツール間の作業引継ぎがスムーズに

  • Antigravityで開発 → 制限到達 → Claude Codeに引き継ぎという運用が可能です。
  • 共通インデックス構造により、ツールを切り替えても文脈をロスしません。
  • 「さっきまで別のエディタで触っていた変更はどこに影響が出る？」と尋ねると、1発で回答が得られます。
  • 通常なら関連ファイルを何回も読み直す時間とトークンを大幅に節約できます。
引き継ぎがスムーズかつ低コストで行えるのが、この仕組みの最大の強みです。

⑭GitNexusに対応する主要ツール群

  • Claude Code・Codexはプラグインで簡単に接続可能です。
  • OpenCode・Kilocode・Hermesなど、CLI・MCP対応ツールなら広くアクセスできます。
  • Antigravity自身にも対応しており、同じナレッジグラフを共有可能です。
  • CLIとMCPが標準装備されているため、対応ツールの幅が広いのが特徴です。
MCP対応が前提なので、今後新しいツールが登場しても対応しやすい設計です。

⑮コードベースが大きいほど効果を発揮

  • 小規模なプロジェクトでは恩恵が薄いですが、大規模プロジェクトで真価を発揮します。
  • バージョンアップや機能追加でコードベースが肥大化するほど、トークン節約効果が増大します。
  • 制限に達するサイクルが縮まってくる体感がある大規模プロジェクトほど導入メリットが大きいです。
実際にゲームアプリを作りながら検証。機能追加を繰り返すほど制限への到達が早くなることを体感しました。

⑯ライトユーザー向けの賢い運用戦略

  • 月額$10〜$30程度のサブスクを複数組み合わせることで、十分な開発環境を構築可能です。
  • GoogleのサブスクとAnthropicのサブスクを両方活用してツールをローテーションさせます。
  • $100や$200の高額プランに入らなくても、GitNexus＋ツール切り替えで効率的に運用できます。
  • 制限に達したら待つのではなく、即座に次のツールに切り替えて作業を継続するのがコツです。
高額プランに頼らず、工夫で乗り切るのがライトユーザーの戦い方です！

⑰異なるAIの得意分野を組み合わせる

  • 各コーディングツールには得意な分野があります。Antigravityの強力な自律実行など。
  • コンテキストをロスすることなく、異なるAIの強みを取り合わせることができます。
  • GitNexusが裏にあることで、共通のナレッジグラフを通じたシームレスな協調が実現します。
この「コンテキストをロスしない協調」こそが、GitNexusアーキテクチャーの隠れた最大の強みです！

⑱まとめ：トークン節約の総合戦略

  • STEP 1 自分のプランのリフレッシュサイクル（5時間/週間）を把握する。
  • STEP 2 制限に達したら別のツールに切り替えて作業を継続する。
  • STEP 3 GitNexusでナレッジグラフ化し、ツール間の引継ぎをスムーズに。
  • STEP 4 AionUIで複数エージェントをオーケストレーションして効率化。
Antigravity2マスタークラス（無料）も併せて受講して、基礎からしっかり学びましょう！
継続して学ぶのが一番の近道です
