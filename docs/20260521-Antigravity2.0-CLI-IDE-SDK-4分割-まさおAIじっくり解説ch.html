Antigravity 2.0 徹底解説

CLI・IDE・SDK の4分割でGoogleの本気を見る | 2026年5月21日

① Antigravity 2.0 登場: 4製品への分割

  • Antigravityが大型アップデートで4つの製品に分割された
  • Antigravity 2.0 — デスクトップアプリ（Codexのような管理画面）
  • Antigravity CLI — ターミナルで動く新コマンド「agy」
  • Antigravity SDK — コードレベルでエージェントを統合
  • Antigravity IDE — VS Codeベースのエディタ拡張
アップデートしたら使いにくくなったという声も...
実は「分割」されたという理解が正しいみたい！

② Antigravity CLI (agy): ターミナルの新顔

  • agy コマンドでターミナルから起動
  • Claude Codeのような対話形式でGeminiモデルとやり取り
  • インタラクティブモードとワンショット実行の両方に対応
  • ファイルパスの補完や @ 参照など、基本機能は一通り揃っている
Geminiモデルを使うならGemini CLIじゃなくて
Antigravity CLI（agy）が本命！

③ agy の主要コマンド一覧

  • /agents — サブエージェントの管理と配置
  • /skills — グローバル/ワークスペース領域で管理
  • /goal — 自律的なゴール設定と実行
  • /grill-me — インタビュー形式の深掘り
  • /schedule — スケジュールタスクの設定
  • /plan — プランニングとタスク整理
  • /fork — 会話の分岐と再利用

④ agy -p: ワンショット実行コマンド

  • agy -p でプロンプトを一発投げて結果を得られる
  • claude -p や codex exec と同じ使い方ができる
  • パイプラインやスクリプトへの組み込みにも便利
  • /continue で前回の会話を継続可能
シェルスクリプトに組み込んで
自動化ワークフローに使えるのが便利！ 
$ agy -p "このコードのバグを見つけて"

⑤ Gemini CLI 廃止: Antigravity CLI へ移行

  • Gemini CLI は徐々に廃止される方向
  • 2026年6月18日にリクエスト提供が停止予定
  • 公式から Antigravity CLI への移行案内が出ている
  • Gemini Code Assist の有料プランも移行対象
Gemini CLI ユーザーは要注意！
早めに agy への移行を検討しよう

⑥ Go言語で書き直された並列エージェント設計

  • Antigravity CLI はGo言語で一から書き直された
  • 並列エージェント前提で再設計されている
  • 高速なシミュレーション実行とマルチエージェント協調に最適化
  • Flash モデルの高速性を活かした設計思想
速度が速いと細かい確認タスクや
サブエージェントとの協調で有利になる！

⑦ Antigravity IDE: エディタでの開発体験

  • 従来のVS CodeベースのAntigravityがIDEとして独立
  • コードを見ながらその場で編集したい時に便利
  • VS Code拡張として普通にインストール可能
  • CLIとIDEで用途に応じて使い分ける設計
エディタ的な使い方をしていた人は
この IDE に移行すればOK！

⑧ Antigravity 2.0 デスクトップアプリ

  • Codex的なデスクトップアプリとして提供
  • スケジュールタスクの設定が可能（実行日時を指定）
  • プロジェクトの選択と管理
  • 画像生成などのタスクもデスクトップから実行
  • ただし制限が厳しくなった印象があるとのこと

⑨ Antigravity SDK: コードレベル統合

  • Claude Agent SDKと同様の位置づけ
  • 自分のサービスの中にエージェントを組み込める
  • プログラムレベルでAntigravityの機能を統合
  • エージェント機能を自作アプリに組み込む需要に対応
サービスにAIエージェントを組み込みたい
開発者にとっては嬉しいSDK！

⑩ サブエージェントの設計パターン

  • Generator（生成）と Evaluator（評価）を分離する設計
  • 自分で生成して自分で正しいと思っても、客観的には正しくない場合がある
  • 別のエージェントにレビューさせることで品質を担保
  • 初めから Research と Browser のサブエージェントが内蔵
  • 関心ごとを分離しながら、それぞれのサブエージェントに注意点を定義
生成と評価を分けるのは
AIエージェント設計の基本パターン！

⑪ ネスト制限: 10層までのオーケストレーション

  • サブエージェントのネストは最大10層まで可能
  • 無限ループを防ぐための制限
  • 10層あれば十分なオーケストレーションが実現可能
  • Teamwork Preview は Ultra プラン限定
  • Geminiもマルチエージェント協調を意識して進めている
無限ループ防止は必須だけど
10層あればかなり複雑な処理もOK

⑫ スキル・フック・MCP対応

  • Skills — 再利用可能なタスク定義
  • Hooks — イベント駆動の自動処理
  • MCP — 外部ツール連携の標準プロトコル
  • グローバルとワークスペースの2つの領域で管理
これ、Claude Codeで見た景色...！
かなり意識してる感がすごい

⑬ Windsurfチームが開発を牽引

  • Antigravityの開発チームはWindsurf出身者が中心
  • 元々Windsurfで培った知見が活きている
  • エディタやAIコーディングのノウハウが豊富なチーム
  • Google傘下でリソースを得て本格的に開発を加速
Windsurfチームの実力があるから
今後に期待できる根拠がある！

⑭ Googleの後追い戦略とコスパの行方

  • CodexやClaude Codeの進化を後追いしながら追走
  • モデルの差がなくなった時にコスト優位性で勝負する狙い
  • 「Codexよりずっと使わせますよ」でユーザーを取り込む戦略
  • 共通ハーネスを持ってインターフェースだけ変える手法
モデルが追いついた瞬間に
コスパでひっくり返しに来るかも！

⑮ Gemini 3.5 Flash の現状評価

  • 評判も使用感ももう一歩という評価
  • 速度は速いが、それだけでは不十分という声が多い
  • GPT-5.5と戦えるかというと厳しいとの見方
  • 高速性はサブエージェントとの協調では有利に働く可能性
  • Flashモデルはあくまで高速・低コストが売り

⑯ Gemini 3.5 Pro への期待

  • 来月リリースと言われている Gemini 3.5 Pro が鍵
  • GPT-5.5より優れていれば選択肢に上がる
  • 同じ性能でコストが安くて速いなら移行の理由になる
  • 界隈の流れをひっくり返す可能性は十分にある
3.5 Pro次第で
Antigravityの評価が大きく変わる！

⑰ レートリミットと制限の厳しさ

  • Antigravityの制限が厳しくなったという感想が多い
  • 使いすぎるとリミットに到達してリセット待ちに
  • リセットまでの待ち時間が長いという課題
  • 無料枠と有料枠の差が大きい可能性
試しに使ってみたけど
すぐリミット来ちゃったとのこと...

⑱ 結論: 今は様子見だが将来に期待

  • 今すぐ使う必要はないが、将来の選択肢として有力
  • ハーネスの方向性は面白く、Claude Codeライクな設計
  • 移行時のストレスを減らす努力が積み上がっている
  • Gemini 3.5 Pro のリリースが大きな転換点になる
  • Googleが本気を出せば界隈の流れを変える可能性は十分
ファーストリリースなので
これからのアップデートに注目！
