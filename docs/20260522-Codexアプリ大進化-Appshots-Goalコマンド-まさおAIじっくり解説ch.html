Codexとは？なぜおすすめなのか

①Codexアプリに新機能が続々登場

  • OpenAIのCodexアプリに便利な新機能が複数追加された
  • 最近のCodexは非常に勢いがある。Claudeから乗り換えるユーザーも増加中
  • 今回解説する新機能は全部で5つ
  • アプリの使用感と作り込みがすばらしく、初心者にもおすすめ
身内に「5000円でAIを勉強するなら？」と聞かれたら、Codexをおすすめしています！

②なぜCodexをおすすめするのか

  • シンプルで使いやすいUI設計が魅力
  • 初心者が迷い込んでも、なんとなくアプリケーションが作れる仕組み
  • オートメーション機能も便利で、日常のエージェントタスクに最適
  • 現在の環境では最高のアプリケーションと評価
ClaudeのMaxプラン（$100〜$200）でOpusを使いたい人にはClaudeもおすすめ。基本はCodex！

③ClaudeとCodexの立ち位置

  • Claude Codeはハーネスの作り込みに強い
  • CodexはアプリのUXと使いやすさが抜群
  • ハーネスを深掘りしたい → Claude
  • 手軽にアプリを作りたい → Codex
  • $20プランでもCodexは十分に使えるコスパの良さ
$200プランなら思想の差が出てくる。最先端のハーネスを掘るならClaudeも有力な選択肢！

④新機能①: Appshotsとは

  • スクリーンショットをCodexにサクッと渡せる新機能
  • Macなら左Command + 右Commandの同時押しで起動
  • 単なるスクリーンショットではなく、構造化データも一緒に送信可能
  • アプリと連携することで、よりリッチなコンテキストを共有
「テキストを表示」を押すと、画面の内容が構造化データとして展開される。ただの画像じゃないのがすごい！

Appshotsとサイドパネルの詳細

⑤Appshotsの使い方・操作手順

  • 見たい画面を開いた状態でCommand+Commandを同時押し
  • Codex側にコンテキストとしてスクリーンショットが送信される
  • 「テキストを表示」を押すと構造化データが展開
  • そのままCodexアプリ内で指示を出して作業を進められる 
# Appshotsの操作イメージ
画面を開く → Cmd+Cmd → Codexに送信
→ テキスト表示 → 構造化データを確認
→ そのまま指示入力

⑥Appshotsの構造化データがすごい

  • 画面の見た目だけでなく、中身のデータ構造も把握
  • AIがコンテキストを正確に理解できるため回答精度が向上
  • 連携アプリに応じて、より詳細なコンテキスト共有が可能
  • 「スクショ撮ってCodexに聞こう」というユーザー体験が最高
サイドからパッと出して、そのままCodexで進められる。このUXが本当にすごい！

⑦新機能②: サイドパネルで使い勝手向上

  • ターミナルモード: コマンドライン操作に対応
  • ブラウザモード: アプリ内でWebページを表示
  • サイドチャットモード: 複数のチャットウィンドウを持てる
  • これら3つのモードを切り替えながら作業できる
ブラウザでTodoリストを表示しながら、ターミナルも置ける。CodexをメインにしてClaudeも呼び出せる！

⑧マルチモデル併用のワークフロー

  • メインをCodexアプリにしつつ、下に他のモデルを配置
  • 下パネルからOpenCodeのK2.6などを呼び出し可能
  • Hermesなど、いろんなツールからモデルを呼び出せる
  • 初心者にもなんとなく操作できるUIが実現
CodexがメインUI。その下にサブモデルを置いて、必要に応じて使い分けるのがおすすめ！

Goalコマンドとブラウザ機能

⑨新機能③: /goalコマンドでロングラン

  • /goalコマンドで目標を設定すると、達成するまで走り続ける
  • 「アプリを作ってデモまで完成させて」のような長時間タスクに最適
  • 目標を入力して「目標として送信」を押すだけで起動
  • CodexとClaudeで仕組みが若干異なるが、どちらも利用可能 
# /goalコマンドの使い方
/goal 長文のアプリを作って
    デモまで作ってください

→ 目標として送信
→ 達成まで自動で走り続ける

⑩/goalコマンドの活用テクニック

  • ゴールは具体的に設定するほど高品質な結果に
  • 「磨き込んでください」といった品質向上の指示も有効
  • マクロ気味に使うことで、大規模なタスクもこなせる
  • 「クリアしたら品質が良くなる」という難易度の高い目標を設定するのがコツ
ロングランタスクをやりたい時に、/goalコマンドと適切な目標設定を組み合わせるのがおすすめ！

⑪サブエージェントレビューで品質担保

  • サブエージェントレビューでメインエージェントの自己消化を防止
  • メインが走っている最中にレビューを挟みながら進行
  • 細かい基準を明確に提示することが品質向上の鍵
  • OpenAIとClaudeで仕組みの違いがあるので理解しておくのが大事
エージェントが同じことを繰り返す「自己循環」を防ぐために、レビューを挟むのが重要です！

⑫新機能④: ブラウザのアノテーション機能

  • Codexブラウザ内にアノテーション機能が追加された
  • 画面上の要素に対して色を変えたり指示を出したりできる
  • SVG形式でアノテーションを送信可能
  • 指示に対してCodexがアップデートを重ねて反映
「ここをレインボーのアニメーションにして」とか「背景を白にして」とか、視覚的に指示を出せる！

アノテーション詳細とComputer Use

⑬アノテーションの具体的な使い方

  • リストの背景色を変更（白や黒などに変更可能）
  • チェックボックスにチェックを入れることもできる
  • プレースホルダーにSVG形式でアノテーションを配置
  • アノテーション付きで送信すると、反映されたアップデートが返ってくる 
# アノテーション機能の流れ
ブラウザを開く
→ 要素をアノテーション（色変更・SVG追加）
→ 送信
→ Codexがアップデートを重ねる

⑭ブラウザ表示の柔軟性

  • ブラウザパネルを全面表示に切り替え可能
  • 今後も機能がどんどん増えていく可能性が高い
  • サイドパネルとの併用で効率的なワークフローを構築
  • ブラウザ上でのリアルタイム確認ができるのが大きな強み
ブラウザをパッと表示して、そこにアノテーションもできる。Codexアプリ、本当にいいなあ…！

⑮新機能⑤: Macロック中のComputer Use

  • 設定のComputer Useセクションに新オプションが追加
  • Macがロック状態でもCodexが操作を継続できる
  • 設定をオンにするだけで有効化。とても簡単
  • 長時間タスク実行中に席を離れても安心
Mac miniでCodexを走らせておいて、ロック中も作業を続けてもらえるのは便利！

⑯Mac mini連携の実践ワークフロー

  • Mac miniを複数台設置してCodexを常時稼働
  • 「こういう仕事をしてほしい」とリモートで指示可能
  • スクリーンショットを取りまくって、レポートを自動作成
  • 成果物をまとめて共有する仕組みを構築
アプリ連携機能とMac miniの組み合わせが、個人的に一番気に入っている使い方です！

使い分けとまとめ

⑰ClaudeとCodexの使い分け戦略

  • CLIツールや効率化デスクトップアプリ → Codexで作成
  • ハーネスのベース開発 → Claudeで実装
  • 散歩から帰ってきたら両方を組み合わせることで品質を高める
  • 用途に応じて適切なツールを選択するのがベストプラクティス
Codexでサクッと作って、Claudeで品質を高める。この二段構えが今の最適解！

  • $20プランでも十分に活用できるコスパの良さ
  • Appshots、サイドパネル、/goal、アノテーション、Computer Useの5つの新機能
  • ハーネスエンジニアリングを深掘りするならClaudeも併用を検討
