Codex制限削減の実態
①

① Codex制限が50%削減された

  • OpenAIがCodexの利用制限を2分の1に削減したことがユーザーの報告で発覚
  • Twitter上で多くのユーザーが「以前よりはるかに早く制限に達する」と不満を表明
  • これまで2〜3回あった週次レートリミットリセットも、ユーザー増加に伴い停止済み
従量課金ユーザーにも関わらず、使える量が半分以下に…これは困った！
②

② OpenAIチームの「沈黙」が意味するもの

  • 問題が報告されてから11〜12時間、OpenAIチームから一切の回答なし
  • バグであれば迅速に対応するはずだが、この沈黙は意図的な変更を示唆
  • かつてClaude Codeを嘲笑するためにリミットリセットを行っていたが、それも終了
これだけ大きな問題で沈黙を守るなら、恒久的な変更の可能性が高い
③

③ OpenAI Deployment Companyの影

  • OpenAIが19のグローバル投資会社・コンサルティング会社と提携を発表
  • 専用デプロイメントモデルを企業向けに提供する新パートナーシップ
  • そのコンピュートリソースはどこから来る？ → Codexユーザーから転用の可能性大
企業向けにコンピュートを割り当てるため、個人ユーザーの枠を削減するのは理にかなっている
④

④ Codexは「赤字事業」だった

  • Codexの推論コストは収益を上回る赤字状態で運営されていた
  • 本来の目的はユーザー獲得と、企業クライアントの獲得のための「お試し期間」
  • 典型的なベンダーロックイン戦略：まず使わせて、後で制限を厳しくする
「高すぎて信じられないサービス」は、結局データかお金、またはその両方を取られる
なぜCodexは制限を下げたのか
⑤

⑤ ユーザー獲得のための「餌」だった

  • Claude Codeの制限に不満を持つユーザーをCodexに乗り換えさせるのが狙い
  • 高い制限は一時的もので、ユーザーが定着した時点で徐々に引き下げる計画だった
  • かつて週2〜3回のレートリミットリセットでClaude Codeを挑発していたのも戦略の一部
ユーザー獲得が一段落したら、制限を下げてコスト削減…見事な計画だ
⑥

⑥ エンタープライズ優先の戦略転換

  • 新たな「OpenAI Deployment Company」は企業向け専用デプロイメントを提供
  • 19の投資会社・システムインテグレーターと提携し、専用モデルを運用
  • 個人向けCodexのコンピュートをエンタープライズ向けに転用
個人ユーザーは切り捨て、企業から利益を取る — ビジネスとしては正しいが…
⑦

⑦ AI業界の「典型的なパターン」

  • フェーズ1：破格の条件でユーザーを獲得（赤字覚悟）
  • フェーズ2：ユーザーが依存したら制限を厳しくする
  • フェーズ3：条件の1行書き換えでサードパーティ利用をブロック
次の新興企業が「新しいCodex」と名乗り出て、同じことを繰り返す…
⑧

⑧ 影響を受けるのは「個人ユーザー」だけ

  • エンタープライズユーザーは以前と同じ制限を維持。影響なし
  • 個人・非エンタープライズユーザーだけが制限削減の直撃を受ける
  • 月額$200を払っても、Codexでは使える量が半減。コストパフォーマンスが悪化
結局、個人の開発者が一番損をする構造…これが現実
