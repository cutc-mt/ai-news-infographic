GitNexus - AIエージェントのための神経系

GitNexusの全体像

① GitNexusとは: コードベースをナレッジグラフ化するツール

  • GitNexusはリポジトリ全体をナレッジグラフに変換するローカル完結型のエンジンです
  • MCP（Model Context Protocol）とCLIの両方が提供されています
  • Claude Code、Cursor、OpenCode、Antigravityなど、主要なAIコーディングエージェントに対応しています
  • データは全てローカルPCに保存され、外部サーバーへの送信はありません
コードベースを読み直す必要がなくなり、
ツール間の作業引き継ぎがシンプルになります！

② AIエージェントの構造的な盲点

  • 現在のAIエージェントは生のテキストを1ファイルずつ読み込んでいるだけです
  • 特定の関数の中身は見えても、他のファイルとの依存関係は把握できません
  • ある関数の戻り値に依存する別の関数が47個あるような場合、人間でも把握しきれません
  • 3つのマイクロサービスにまたがるアーキテクチャ全体は全く見えていない状態です
本のランダムなページだけめくって、
物語全体を理解しようとするようなもの...

③ ナレッジグラフの仕組み: ノードとエッジ

  • ノード（点）: 関数、クラス、メソッド、インターフェースなどを全て点として扱います
  • エッジ（線）: 依存関係や関数呼び出しを線として結びつけます
  • 点と点を繋いでコードベースの完全な地図を作成します
  • 構造データはRaylib DBというグラフデータベースに永続化されます
DeepWikiが「理解して記述する」ツールなら、
GitNexusは「分析して関係性を把握する」
データベースとして機能します！

④ コード変更の爆風被害（Blast Radius）

  • AIにリファクタリングを頼んで完璧なコードが出力されても、全く別のサービスがクラッシュすることがあります
  • 美しい家を建てたつもりが、地下の水道管が破裂していたような状態です
  • AIは変更の影響範囲を計算できず、システムを壊してしまいます
  • これはモデルの知能不足ではなく、物理的な制約の問題です
コンテキストと構造的認識が
決定的に欠けている状態なんです！

解析技術と仕組み

⑤ Tree-sitterによる高精度な構文解析

  • Tree-sitterを使ってAST（抽象構文木）を解析しています
  • 単なる文字列検索ではなく、文法としてコードを読み解くアプローチです
  • 主語と述語の関係性までしっかり理解する感覚で解析します
  • Unreal Engine C++特有のマクロストリッピングにも対応しています
ゲーム開発のコードまで解析できる
のは驚きですよね！

⑥ 14の主要プログラミング言語に対応

  • TypeScript / JavaScript
  • Python
  • C / C++（Unreal Engineマクロ対応）
  • Java / Go / Rust / Ruby など
  • 合計14言語をサポートしており、カバー範囲が非常に広いです
最新のアップデートで
マクロストリッピングにも対応！

⑦ Leidenアルゴリズムによる自動クラスタリング

  • 開発者が手動でラベルを貼る必要は一切ありません
  • 背後でLeidenアルゴリズム（コミュニティ検出アルゴリズム）が自動実行されます
  • 本の中身を読まずに「この50冊は頻繁に引用し合っているから同じジャンルだろう」と自動判定します
  • 依存関係や参照の頻度を計算して、自然な機能クラスターを見つけ出します
「ここは認証系」「ここはデータベース系」
みたいに自動で整理してくれます！

⑧ 事前計算によるトークン効率の大幅向上

  • 従来のGraph RAGは、AIに生のグラフデータを渡して何度もクエリーを重ねる必要がありました
  • GitNexusは事前計算（Pre-computed Relational Intelligence）を採用しています
  • インデックス作成時に、構造・クラスタリング・実行フローのトレース・関連性スコアリングを全て完了させます
  • 1回のツール呼び出しで完全な構造化データが取得できるため、トークンの無駄遣いを削減します
Graph RAGはAIにレシピとスーパーの行き方を教えて
自分で買い出しに行かせるようなもの...
GitNexusは下ごしらえ済みのフルコースを
目の前に出すようなものです！

16のMCPツールと機能

⑨ MCPで提供される16個のツール

  • 単一リポジトリ用に11個のツールが提供されています
  • マルチリポジトリ用に5個のグループツールが提供されています
  • MCP（Model Context Protocol）経由で、Claude CodeやCursorなどのエージェントに接続します
  • ツールを切り替えても、同じインデックスから作業を継続できます
CLIとMCPが標準装備されているので、
AntigravityからClaude Codeに
スムーズに切り替え可能です！

⑩ Impact Tool: 変更前の影響範囲分析

  • 最も強力なツールがImpact Toolです
  • エージェントがコードに変更を提案する前に、その変更が及ぼす爆風被害を分析します
  • 影響の深さと信頼度スコア付きで結果を教えてくれます
  • 専属の超優秀なシニアエンジニアが横でコードレビューしてくれているような感覚です
変更する前にリスクが分かるから、
安心してリファクタリングできます！

⑪ Detect Changes: 変更検出と影響マッピング

  • AIのdiff（差分）を読み込んで影響を分析します
  • 変更された行がシステムのどの実行プロセスやクラスターに属しているかを自動マッピングします
  • 「この書き換えによってログインプロセス全体に影響が出ますよ」のようなコミット前の影響分析が可能です
  • 単に「ここが書き変わりました」ではなく、実行フローのコンテキストを持った結果を返します
コミットする前に影響範囲を
丸ごと把握できるのが最大の魅力です！

⑫ RRF検索: キーワード検索と意味検索の融合

  • Reciprocal Rank Fusion（RRF）で2つの異なる検索のいいとこ取りをします
  • BM25: キーワードの完全一致を重視する厳格な検索
  • セマンティック検索: 変数名は違くても処理の意味は同じだと読み取る柔軟な検索
  • 2人の審査員のスコアカードを受け取って、総合的に最も正確な答えを導き出します
  • プロセスごとにグループ化して結果を返すため、関連行動を見失いません
厳格な審査員と柔軟な審査員、
二人の意見をレフェリーが
ジャッジするイメージです！

安全性とエコシステム

⑬ 安全性の仕組み: リードオンリーとミュータティング

  • リネームツールは、グラフ構造に基づいて確実に安全な変更と手動確認が必要な変更を明確に区別します
  • 100%安全なものと、AIの推測が入っているものの間にちゃんと線を引いてくれる仕組みです
  • 全MCPツールにRead-Only（読み取り専用）とMutating（変更を伴う）の安全アノテーションが追加されています
  • CIパイプラインで自立的に動くエージェントが安全な探索だけを行うのか、変更権限を持つのかを制御できます
AIが勝手に放送して全部壊す...
という心配は不要です！
非常に丁寧に設計されています。

⑭ Claude Codeが受け取る特別待遇: フック機能

  • Claude Codeはツール＋スキル＋フックを取得しており、他のエディタより統合が深いです
  • Pre-Tool Hook: ツール使用前にグラフ情報で検索を自動強化します
  • Post-Tool Hook: コミット後にインデックスの古さを即座に検出して、再インデックスを促します
  • 常に最新の状態に保たれる、自動更新されるカーナビのような存在です
新しい道が作られたり工事中になった瞬間に
自動的にアップデートされる
最強のカーナビなんです！

⑮ マルチリポジトリ対応とグローバルレジストリ

  • グローバルレジストリを使って、1度のセットアップで全リポジトリに対応します
  • ローカルのregistry.jsonで一元管理される仕組みです
  • 複数リポジトリをまたいだAPIの境界を抽出するグループツールも存在します
  • あるサービスの変更が別のサービスにどう影響するかを横断的に把握できます
マイクロサービス環境で
誰も全体像を把握しきれていない状況で
こそ威力を発揮します！

⑯ ツール間の作業引き継ぎ: インデックス共有

  • Antigravityで作業して制限に達したら、Claude Codeに引き継いで作業を継続できます
  • 共通インデックス構造により、記憶が共有化・共通化されています
  • 従来はツールを切り替えると、最初からコードを読み込ませるトークンの無駄遣いがありました
  • GitNexusが裏に入っていれば、コードの構造を完全に理解した状態から即座にスタートできます
異なるAIツールの得意分野を、
コンテキストをロスすることなく
シームレスに活用できるんです！

導入方法とソフトウェア開発の未来

⑰ Web UIとCLIの2つの利用モード

  • CLI＋MCPモード: ネイティブで実行され、大規模コードベースに向いています
  • Web UIモード: ブラウザ上で完結し、インストール不要です（最大約5,000ファイル）
  • WebAssembly（WASM）技術でブラウザ内処理が完全に完結します
  • ブリッジモード: ローカルバックエンドとWeb UIをつなぐセキュアな環境も用意されています
  • Publish機能: インデックス化されたナレッジグラフを共有レジストリに公開可能です
データは一切外部サーバーに送信されないから、
企業でも安心して使えます！
