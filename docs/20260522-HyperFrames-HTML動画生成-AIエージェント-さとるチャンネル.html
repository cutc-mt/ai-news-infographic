HyperFrames爆誕：HTMLで動画、エージェント向けの新常識？

2026年5月22日

HyperFramesとは

① HeyGenが新OSSを公開

  • HeyGenがHTMLで動画を作るフレームワーク「HyperFrames」をOSS化
  • GitHubスターが2万超えを達成、リリースからわずか1ヶ月で爆増
  • 2026年4月17日にオープンソースとして公開
  • 元々はHeyGen社内で使われていたツールを外部に公開した経緯
社内で磨き上げられてから公開されたから、実用性が高いのが特徴！

② キャッチコピーの真意

  • キャッチコピーは「HTML, Render, Video, Built 4 Agents」
  • 「4 Humans」ではなく「4 Agents」 — AIエージェントのために設計
  • 普通の動画ツールは人間が操作する前提だが、HyperFramesは最初からAIに書かせる前提
  • 「HTMLを書いて動画を出せ」というシンプルな命令でエージェントが動作
AIが書く前提で設計されているのが、他のツールと決定的に違うポイント！

③ 動画編集ソフトではない

  • HyperFramesは動画編集ソフトではない — タイムラインをマウスでドラッグするUIはない
  • やることはHTMLとCSSとJSで動画を書くことだけ
  • Premiere ProやDaVinci Resolveのイメージで触ると確実に積む
  • 正しくはコード実行の動画ツール
「書く言語がHTMLかReactか」に見えるけど、実は設計思想の根幹に関わる違い！

④ Apache 2.0で完全無料

  • ライセンスはApache 2.0 — 商用利用も完全無料
  • チーム人数が増えても追加コストはゼロ
  • 自宅でクライアントに納品する用途でも問題なし
  • 導入の決裁が1段軽くなる — 課金を気にする必要がない
4人だろうが400人だろうが同じ条件で使えるのは大きい！
仕組みとパイプライン

⑤ HTML → MP4の変換パイプライン

  • HTML + CSS + JSで動画の内容を記述
  • Puppeteer（ヘッドレスChrome）で1フレームずつスクリーンショットを撮る
  • FFmpegでフレーム画像を連結してMP4に変換
  • ブラウザで動いている見た目をそのまま動画に変換する仕組み 
// パイプラインの流れ
HTML/CSS/JS
    ↓  Puppeteer（スクリーンショット連射）
フレーム画像（連番）
    ↓  FFmpeg（エンコード）
MP4 動画ファイル
CSSアニメもThree.jsの3DもPotreeも、画面に出るならそのまま動画になる！

⑥ フレームアダプター方式

  • アニメーションを1フレームずつドル（$）で停止させてキャプチャ
  • 例：4秒地点のフレームを取りたい時は、タイムラインをシークで瞬間移動させてキャプチャ
  • 時間を流すのではなく切る — 同じHTMLを入れたら同じMP4が出る
  • 決定論的レンダリング — 再現性100%の動画ビルド
普通の動画ツールはタイミングが揺れるけど、HyperFramesは毎回同じ結果が保証される！

⑦ 前提条件と対応環境

  • Node.js 22以上が必要（LTSが22.xなので対応）
  • FFmpegが必須 — ここで詰まる人が一定数いる
  • Macなら brew install ffmpeg で終わり
  • WindowsはPATHを通すのに1手間かかる
  • HDR出力にも対応 — Remotionがまだ対応していない部分 
# セットアップ確認
node -v   # v22以上であること
ffmpeg -version  # インストール済みか確認
FFmpegのインストールでつまずく人が多いので、最初に確認しておくのがおすすめ！

⑧ Web制作者に刺さる理由

  • CSSアニメーションがそのまま動画になる
  • Three.jsの3D表現も画面に出ればそのまま動画化
  • PotreeなどのWebGLコンテンツも対応可能
  • 普段の制作物がそのまま素材になる — これは大きい
Webの強い人にはめちゃくちゃ刺さる！既存のスキルがそのまま動画制作に活かせる！
Remotionとの徹底比較

⑨ 言語の違い: React vs HTML

Remotion
TSX（React）で記述
HyperFrames
HTMLで記述
  • RemotionはReactの状態管理もプロップスもエコシステムごと持ち込める
  • HyperFramesはHTML1枚で完結 — シンプル
  • Webアプリを作っているチームにはRemotionが馴染みやすい
言語の違いは設計思想の違いに直結 — 次のカードで解説！

⑩ LLMとHTMLの親和性

  • 学習データ的にHTMLが一番深い — LLMの出力が安定しやすい
  • Reactは状態管理や依存関係が多く、AIが雑に触ると壊れやすい
  • HTMLなら1枚絵として固められる — 出力品質が安定
  • ここが「Built 4 Agents」の意味に直結する
「このHTMLとCSSを出して」の方が、AIにとって成功率が高いのが分かる！

⑪ ライセンスの決定的な違い

Remotion
4名以上で有料ライセンスが必要
HyperFrames
Apache 2.0で常に無料
  • Remotionはスタートアップだと4人ですぐに条件に引っかかる
  • HyperFramesは4人でも400人でも追加コストゼロ
  • 上司や法務への説明コストが全然違う
ライセンスは意思決定に直結する重要ポイント — 触る側のスピード感も変わる！

⑫ 成熟度と分散レンダリング

  • Remotionは2021年から稼働 — 圧倒的に成熟している
  • RemotionはAWS Lambdaでの分散レンダリングが安定動作
  • HyperFramesはプレビルダーの未実装 — まだゼロ.x台
  • HyperFramesは破壊的変更があり得る前提で触る必要がある
プロダクションに本気で載せるなら、メジャーバージョンが出るまで様子見が堅実！

始め方とAIエージェント統合

⑰ 3コマンドで始める

  • プロジェクト作成 — テンプレート付きで初期化
  • プレビュー起動 — HMRが効くのでHTMLをいじるだけで即反映
  • MP4書き出し — レンダリングして完成 
# ① プロジェクト作成（テンプレート付き）
npx hyperframes init my-video --example kinetic-typography

# ② プレビュー（HMR対応）
npx hyperframes preview

# ③ MP4書き出し
npx hyperframes render --output output.mp4
init → preview → render の流れがWebフレームワークの感覚そのまま！

⑱ AIエージェントとの統合

  • Claude Code / Cursor / Gemini CLI に公式スキルとして登録可能
  • 登録はコマンド1発 — npx skills.com/hyperframes
  • 「URLから動画化」スラッシュコマンドが標準で搭載
  • 音声生成や背景画像もHyperFrames CLIで一括管理
  • 最初からエージェント側に手を出してきているのが決定的な違い
ウェブサイトを動画に変えるスラッシュコマンドが標準でくるの、発想として強すぎる！
