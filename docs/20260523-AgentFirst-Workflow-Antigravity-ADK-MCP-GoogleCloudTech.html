3つのエージェントファーストワークフロー

  • ① デバッグ: エージェントが障害を自動調査し、原因を特定して修正コードを生成。

  • ③ 自己修復: 障害を自動検知して修正し、CI/CDパイプラインを自律的に実行。
プロトタイプを本番レベルに引き上げる3つのアプローチ！

第4部: 自己修復システムとDinoエージェント
13

Dinoエージェント: 3つの自律型エージェント

  • Remediation Agent: 常時稼働のSRE。障害を自動検知・調査し、根本原因を修正。
  • CI Agent: テスト実行、セキュリティスキャン、イメージビルドを担当。
  • CD Agent: デプロイメント管理、カナリアリリース、自動ロールバックを実行。
すべてCloud Run上でホスト。アプリと同じプラットフォームで低レイテンシ・安全・コスト効率良好！
14

自己修復の仕組み: Event-driven自動修復

  • アプリが Cloud Logging にログを記録（デフォルト動作）。
  • エラー検知時、Eventarc + Pub/Sub がRemediation Agentを起動。
  • Geminiで問題を調査・修正し、GitHub PR を自動作成して Slack で通知。 
# 自己修復フロー
App → Cloud Logging → Eventarc
  → Pub/Sub → Remediation Agent
    → Gemini (調査・修正)
      → GitHub PR + Slack通知
15

Agent Development Kit (ADK) の活用

  • 状態管理、ツールルーティング、リトライロジックの ボイラープレートが不要。
  • Skill Toolset でカスタムドメインプレイブックをロードし、知識を再利用可能なワークフローに。
  • 各エージェントは IAMセキュリティ を継承。CI Agentはデプロイ権限を持たず、Remediation Agentはビルド権限を持たない。
秘密情報はすべてSecret Managerで管理。セキュリティを犠牲にしないエージェント設計！

Agent CLI: エージェント開発を数秒で開始

  • Agent CLI でエージェントのひな形を数秒で生成。テスト、評価、コンテナ化ファイルも自動作成。
  • エージェントコード、指示ファイル、ツール定義、テストスイートがすべて含まれる。
  • デプロイ用の コンテナ化ファイル も自動生成。Cloud Runへのデプロイがスムーズ。

第5部: エージェント間連携と自律的CI/CD
17

Agent-to-Agent (A2A): エージェント間の協調動作

  • エージェント同士が A2Aプロトコル で直接対話。チームメンバーのように協調。
  • Agent Registry で各エージェントの場所、呼び出し方、性格（説明）、アクセス可能なツールを管理。
  • 静的パイプラインと違い、エージェントが 自律的に判断。テスト内容やセキュリティスキャンの要否を自動決定。
CI Agentが間違ったイメージを指定すると、CD Agentが「これ存在しないよ」と指摘。自動で再実行まで行う！
18

自律的CI/CDとSpend Capsによる安全な運用

  • CD Agentが リスクスコア を評価し、カナリアリリースの割合を自動決定。
  • デプロイ中に障害を検知すると、自動ロールバック を実行して安全を確保。
  • Spend Caps（Private Preview）: 予算上限に達するとトラフィックを一時停止。翌朝に予算を破壊される心配なし。
  • SlackからCI/CDを起動可能。どこからでもパイプラインを実行できる。
Cloud Runはアイドル時にスリープ。エージェントはエラー発生時だけ起動して、終わったら停止。無駄な課金なし！
