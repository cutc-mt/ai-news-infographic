Antigravity 2.0 徹底解説

GoogleのAIコーディングツールが4製品に分割 - CLI・IDE・SDKの全貌

2026年5月23日

① Antigravity 2.0 の概要: 4製品への分割

  • GoogleのAntigravityが大型アップデートを実施し、1つのアプリから4つの製品に分割された
  • 構成はデスクトップアプリ、CLI、IDE、SDKの4つ
  • ユーザーからは「アプリをアップデートしたら使いにくくなった」との声も上がっているが、実際は製品の分割によるもの
  • 共通のハーネス（基盤）を持ち、インターフェースを変えているイメージで理解すると分かりやすい
1つのアプリだったものが4つに分かれて、
それぞれの用途に特化した形になったよ！

② 4つの製品構成: 役割と位置づけ

  • Antigravity 2.0: デスクトップアプリ。Codex的なUIでスケジュールタスクやプロジェクト管理が可能
  • Antigravity CLI: ターミナルで動く「agy」コマンド。Claude Code的な使用感
  • Antigravity IDE: VS Codeベースの拡張機能。エディタ内でコードを見ながら編集
  • Antigravity SDK: コードレベルでエージェントを統合。Claude Agent SDKに近い位置づけ
同じGeminiモデルを
使う場面に応じて使い分ける設計！

③ Antigravity CLI (agy): ターミナルの新顔

  • agyコマンドでターミナルから起動。Claude Codeと同じような使用感で操作できる
  • ターミナルに「agy」と打ち込むだけで、Antigravity CLIが起動する
  • 対話形式でGeminiモデルとやり取り。例えば「ハロー」と打てば応答が返ってくる
  • 今回のアップデートの中で最も注目すべきはこのCLIだと解説者が評価
Claude Codeユーザーなら
違和感なく使える設計！

④ CLIのコマンド一覧: 充実の機能ラインナップ

  • /agents: サブエージェントの管理。グローバル領域とワークスペース領域の両方に配置可能
  • /skills: スキルの管理。グローバルとワークスペース領域が存在
  • /goals: ゴールコマンドを実装。タスクの目標を設定できる
  • /schedule: スケジュールの設定が可能
  • /plan: プランニング機能やリージョン、リウンド等も搭載
一通り欲しそうなコマンドが
しっかり揃っている印象！

⑤ CLIの拡張機能: フック・MCP・アーティファクト

  • /hooks: フック機能が存在。Claude Codeで見た仕組みに酷似
  • MCP: プラグイン的な概念としてMCP（Model Context Protocol）をサポート
  • アーティファクト: Antigravity独自の特徴的な機能も残されている
  • /fork: フォーク機能も利用可能
Claude Codeの影響を
強く感じる拡張機能のラインナップ！

⑥ BTWコマンド: 気軽な質問モード

  • BTWコマンドは、ちょっと気軽に質問したい時に使える便利な機能
  • 解説者自身も実際に日常的に使用しているとのこと
  • コマンド実行時には注意が必要で、意図しない操作を避けるために確認が推奨される
「ちょっと教えて」的な
気軽な使い方ができるのが嬉しい！

⑦ Gemini CLIの廃止: Antigravity CLIへ移行

  • Gemini CLIは徐々に廃止され、Antigravity CLIへの移行が推奨されている
  • 2026年6月18日にGemini CLIのリクエスト提供が停止予定
  • Gemini Code Assistの有料版などで提供されていた機能も対象
  • 公式から移行案内が出されており、新しいCLIへの移行がスムーズに進められている
6月18日が期限！
早めの移行を検討しよう。

⑧ Go言語で再設計: 並列エージェント前提のアーキテクチャ

  • Antigravity CLIはGo言語で書き直されている
  • 並列エージェント前提で再設計されており、マルチエージェント協調を意識した構造
  • Gemini Flashの高速性を活かしたシミュレーション高速化が期待できる
  • 程度の低いタスクでも抜け漏れを確認する用途で、高速化は非常に実用的
Go言語の並列性を活かして
エージェントを高速に動かす設計！

⑨ サブエージェントの設計パターン: 生成と評価の分離

  • ジェネレーター（生成）とエバリュエーター（評価）を分ける設計パターンが効果的
  • 自分で作ったものを自分で評価すると客観性が欠けるため、別のエージェントにレビューさせるのが重要
  • 関心ごとを適切に分離しながら、サブエージェントごとに注意点を定義して進める
  • デフォルトでresearchとbrowserのサブエージェントが導入済み
作る人とチェックする人を
分けるのがコツだよ！

⑩ ネスト制限とマルチエージェント: 最大10層まで

  • サブエージェントのネストは最大10層まで可能
  • ネスト制限は無限ループを防ぐために設けられている
  • 10層あれば、かなり複雑なオーケストレーションが実現可能
  • チームワークプレビューというスラッシュコマンドがUltra限定で存在
10層もネストできれば
実用的なマルチエージェントが組める！

⑪ agy -p コマンド: ワンショットでモデルを呼び出す

  • agy -pで非対話モードでGeminiモデルを呼び出し可能
  • Claude Codeの「claude -p」と同じ使い方で、ワンショットで結果を返してくれる
  • -c（コンティニュー）コマンドで、前回のセッションを継続することもできる
  • ファイルパスの保管（タブ補完）機能もあり、実用的なCLIとして完成度が高い
claude -p と同じ感覚で
Geminiが使えるようになった！

⑫ Antigravity IDE: VS Codeベースのエディタ統合

  • 従来のVS CodeベースのAntigravityが、Antigravity IDEとして独立したアプリに
  • コードを見ながらちょっと編集したい時に便利な選択肢
  • 普通に拡張機能としてインストールして利用できる
  • 「IDEが分割された」という事実を知っておくだけで、用途に応じた使い分けができる
エディタ派の人は
このIDEを使い続けよう！

⑬ デスクトップアプリ 2.0: Codex的なスケジュール管理

  • スケジュールタスクの設定が可能。プロジェクトを選んで実行タイミングを指定できる
  • プロジェクトの作成や管理をUI上で行える
  • 画像生成などのタスクも試すことができる（ただしGeminiモデルへの学習データとして使われる可能性に注意）
  • 制限が以前より厳しくなっている印象があるとのこと
Codex的なUIで
タスクの自動実行ができる！

⑭ Antigravity SDK: コードにエージェントを組み込む

  • Antigravity SDKでコードレベルでエージェントを統合できる
  • Claude Agent SDKと同様の位置づけで、サービスの中にエージェントを組み込める
  • CLI、IDE、SDKすべてが共通のハーネスを基盤として動作
  • エンタープライズ用途でのエージェント組み込みにも対応
アプリの中に
エージェントを埋め込める時代！

⑮ Windsurfチームの参画: 開発の裏側

  • AntigravityのチームにはWindsurf（旧Codeium）の元中心メンバーが参加している
  • Windsurf出身のチームが手がけていることもあり、今後の発展への期待が持てる
  • 既存のAIコーディングツールのノウハウが活かされている可能性が高い
Windsurfの経験者が
開発を牽引しているのは心強い！

⑯ 後追い戦略とコスパの行方: Googleの本気度

  • CodexやClaude Codeが進化する中、Googleは後追いしながら最後に逆転を狙う戦略と推測
  • モデルの差が感じられなくなった時に、コスト面で優位性を出せばユーザーが移る可能性がある
  • 「GeminiがCodexよりたくさん使わせますよ」という訴求ができれば流れが変わる
  • 現在はスレッドベースのクローンが存在しないなど、まだ追いつくべき機能がある

  • 速度の速さはサブエージェントとのマルチエージェント協調では有利に働く可能性がある

  • 3.5 ProがGPT 5.5より優れている、あるいは同等でコストが安ければ、界隈の流れが変わる可能性がある
