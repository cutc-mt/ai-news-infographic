「初動はAntigravityで手早く、本格化はClaude Codeで！」この組み合わせが鍵だよ！

  • Vibeコーディングを始めたばかりの初心者開発者

  • 複数のAI開発ツールを効果的に組み合わせたい人
  • プロトタイプを短期間で作りたいアイデアマン
毎月のAPI代が怖い…って人には特にオススメの内容！

③全体ワークフロー

  • AntigravityでGemini 3.5 Flashを使いプロトタイプを爆速生成
  • Claude Code / Codexで機能拡張・品質向上
  • Ollamaでローカルモデルを組み込みテストを無料化
  • 本番はAPIモデルに切り替えて最終品質確認
「高速→高品質→低コスト」の黄金パターンぽん！
Antigravity ─ 初期開発の爆速エンジン

④Antigravityの最大の強み

  • Gemini 3.5 Flash搭載でめちゃくちゃ速い初期開発
  • 自然言語の指示だけでAIが自律的に計画・環境構築・コーディング・動作確認を実行
  • プロンプトを入力するだけでアプリが完成する体験
  • 基本無料から利用可能（GmailアカウントでOK）
「作りたいもの」を伝えるだけでAIが勝手に作ってくれる！夢のツール！

⑤Antigravityの実行プラン機能

  • 実行前にステップごとの計画を自動生成してくれる
  • 思考プロセスも表示され、AIが何を考えながら作業しているかが可視化
  • 各ステップごとに許可ボタンでユーザーが確認可能
  • 自動モードもあり、許可なしで一気に実行もできる
思考プロセスが見えるから、AIの判断に安心感があるね！

⑥Antigravityの注意点

  • 複雑なアプリ開発にはまだ少し弱い部分がある
  • レート制限に達すると一時停止（有料ユーザーでも発生）
  • Always Proceedモードは意図しないファイル削除のリスクあり
  • Gemini 3.1 Proの登場で精度向上が期待される
基本は「Review Driven Development」モード推奨！安全第一！
実践デモ ─ ノートベースELアプリの開発

⑦ノートベースEL ─ 検索機能の追加

  • 元々は検索機能がないノートアプリだった
  • 検索機能を追加（部分一致・完全一致対応）
  • 大文字小文字を区別しない設定も実装
  • 特定キーワードを含むスレッドだけを絞り込んで表示
1発で検索機能を作れた！Antigravityの爆速感すごい！

⑧ダッシュボード形式への拡張

  • スクロールではなくクリックで開く形式に進化
  • 出店・日付などのメタ情報をダッシュボード表示
  • ドラッグ＆ドロップで自由に要素を移動可能に
  • レスポンシブ対応でスマホサイズでも動く
Claude Code / Codexで拡張したら一気に本格的なアプリになった！

⑨AIサイドチャット機能

  • アプリ内にAIチャットを統合（サイドバーに表示）
  • チャットログの保存機能を追加（前回は残らなかった）
  • クリアボタンで会話履歴をリセット可能
  • 過去のやり取りが要約されて残る仕組み
ログが残るようになったから過去の会話も振り返れる！便利！
Claude Code × Codex ─ 拡張と機能追加

⑩Claude Code / Codexの役割

  • Antigravityで作ったプロトタイプを拡張するためのツール
  • Claude Code MaxやCodexなら本格的な機能追加が可能
  • 初期開発はAntigravityで手早く、本格化は高品質モデルで
  • 小さなアプリならAntigravityだけで十分な場合もある
最初からClaude Code Maxで始めてもいいけど、初動を楽にしたいならAntigravity！

⑪拡張で追加した機能群

  • 自動モード / 保護モードの切り替え機能
  • 保護モードは安全重視（プロンプトインジェクション対策）
  • 解析中のキラキラアニメーションで処理状況を可視化
  • 停止ボタンで処理を中断できる機能
セキュリティ面もしっかり考えられてる！プロンプト攻撃対策もバッチリ！

⑫Amazon画面操作のデモ

  • プロンプト：「Amazonページに行ってうどん1kgのコスパいいものを検索」
  • 実行プランが自動生成：検索→選択→カート→購入画面で停止
  • 許可ボタンでステップバイステップで進行
  • 自動制限（8回上限）で無限ループを防止
「続き」と入力で再開できる！安全装置も万全！

⑬OllamaでローカルAI構築

  • Ollama = 自分のパソコン上でAIモデルを動かすツール
  • インターネット不要でローカル環境で実行可能
  • QwenやLlamaなど人気モデルを簡単ダウンロード
  • GPUがあれば高速、CPUだけでも動作する
ローカルAIのセットアップは意外とカンタン！

⑭API料金を劇的に抑えるコツ

  • 開発中はローカルAIで動作確認（機能が動くか？）
  • 最終確認はAPIモデルで品質確認（精度が十分か？）
  • 役割分担がコスト削減の鍵
  • テスト回数が多いほど節約効果が大きい
ローカル＝動作確認、API＝品質確認。この分け方が大事！

⑮DOM解析の仕組み

  • DOM = Webページの構造をプログラムから読み取る仕組み
  • 「どんなボタン」「どんなテキスト」「どんな構造」を把握
  • AIが画面操作するために構造理解が不可欠
  • 悪意ある指示のプロンプトインジェクション対策も重要
セキュリティと使いやすさのバランスが大事！

⑯安全性と使いやすさのバランス

  • 保護モード＝安全性重視（APIキー保護・外部サイト注意）
  • 自動モード＝便利だけどリスクあり（許可確認なし）
  • 基本は自動モードOFFで操作ごとに許可確認
  • 安心感を優先すれば保護モード推奨
便利さも大事だけど、安全が第一！

⑰開発フローのまとめ

  • Antigravityでサクッとプロトタイプを作る
  • Codex / Claude Codeで機能拡張・本格化
  • OllamaのローカルAIで低コストにテスト
  • APIモデルで本番品質を最終確認
小さく作って、大きく育てる！この流れが最強！

⑱今後のAI開発のポイント

  • ツールをひとつに絞る時代は終わった
  • 「どれが最強か」ではなく「どの場面で何を使うか」
  • 個人開発や小チームではコスト管理が超重要
  • AI共創開発は試行錯誤のサイクルを高速回すことが鍵
ツールの得意分野を見極めて、役割分担させるのがコツ！
