社内AI活用の9割が期待ハズレ

「汚れたデータ」が最大のボトルネック。AI-Readyデータの整え方で年間3.9億円の無駄を削減する方法

①AI活用の9割が期待ハズレという現実

  • 企業のAI活用プロジェクトの9割が期待した成果に届いていないという厳しい現実がある
  • Sansanの鳴海佑紀さんと、ソフトバンクの平岡拓さんがPIVOT番組「& questions」でこの問題に切り込んだ
  • 原因はAIの性能不足ではなく、入力するデータの質に問題があることが多い
  • 多くの企業が「AIを導入すれば魔法のように効率化する」と誤った期待を持っている
AIの性能じゃない、データの質が問題なんだ！ここを見落とす企業が多すぎる

②登壇者紹介：Sansan × ソフトバンク

  • 鳴海佑紀さん：Sansan SDI推進部 副部長。保険・EC業界でBtoB営業とマーケティング責任者を歴任
  • 2019年にSansanへ入社。DX・AI活用を止めないデータ品質の実装に取り組んでいる
  • 平岡拓さん：ソフトバンク satto事業開発本部 本部長。最年少本部長として活躍
  • 生成AIエージェント「satto workspace」のベータ版を提供開始した異色の経歴の持ち主
データ品質のプロと生成AIのプロが対談。実践的な知見が詰まっている！

③期待ハズレの本当の原因を探る

  • AI活用が期待通りにいかない最大の原因は、企業が持つデータが「汚れている」こと
  • 表記揺れ更新漏れ重複登録など、日々の業務でデータが劣化していく
  • 部署間でデータが統一されておらず、同じ取引先が別名で複数登録されているケースも頻発
  • AIに汚いデータを入力しても、正確な出力は得られない。ゴミを入れたらゴミが出てくる
GIGO（Garbage In, Garbage Out）の法則。AIに限らずデータ活用の鉄則だ！

④「汚れたデータ」が生まれる背景

  • 人間が手作業でデータを入力する限り、表記揺れや入力ミスは避けられない
  • 部署ごとに別々のシステムを使っていると、同じ顧客情報がバラバラに管理される
  • 担当者の異動や退職に伴う更新漏れが、データの鮮度を低下させる
  • 「誰が管理すべきか」という責任の所在が曖昧になっている企業も多い
「（株）」と「株式会社」で別レコードになる...こんなことが当たり前に起きている

⑤年間3.9億円もの無駄とは？

  • データの不整合による年間3.9億円の無駄が発生している企業の実例が紹介された
  • 重複データによって同じ顧客に複数回アプローチしたり、見落として商機を逃したりする損失
  • データの確認・修正に費やす人件費も、見えないコストとして膨大に積み上がっている
  • AI導入前にデータ整備のコストを見積もらない企業が多いのが問題
3.9億円
データ不整合による年間の無駄なコスト（実例）

⑥個人の時短で終わらせるな

  • 多くの企業がAI活用を「個人の業務効率化（時短）」のレベルで終わらせている
  • 本当の価値は組織全体のプロセス変革を起こすことにある
  • 個人レベルの効率化では、組織としての大きな成果には繋がらない
  • 「個人の便利ツール」から「組織の意思決定を支える基盤」へ昇華させる必要がある
ChatGPTでメールを書くのは時短。でも組織全体を変える話じゃないんだよね

⑦AI活用が出来ていないマイナス面

  • データが整っていない状態でAIを導入すると、誤った洞察や判断を生み出すリスクがある
  • AIの出力が信頼できないと、現場からの不信感が募り導入自体が頓挫するケースも
  • データクレンジングに想定以上の時間がかかり、AI導入計画が大幅に遅延する事態も
  • 結局「AI使えないじゃん」と現場のモチベーションが低下して、DXの足かせになる悪循環
一度失敗すると「次も失敗する」という先入観が残って、二度と導入できなくなることも...

⑧表記揺れの具体例と影響

  • 表記揺れの典型例：「㈱」「（株）」「株式会社」が同じ会社を指すのに別レコードになる
  • 全角・半角の違い、スペースの有無、大文字小文字の違いなどが重複を生む
  • 名寄せ（同一実体を統合する作業）が不十分だと、正確な顧客分析が不可能に
  • AIが重複データを別々の顧客として扱うことで、誤った予測や提案をしてしまう
例: 同じ取引先が3つに分裂
①Sansan株式会社
②(株)Sansan
③サンサン株式会社
→ AIは「3社」と認識してしまう！

⑨データ負債とは何か

  • データ負債＝技術的負債のデータ版。過去の運用で蓄積された不適切なデータが組織の足かせになる
  • 「後で直す」と放置されたデータが雪だるま式に膨れ上がり、修正コストが跳ね上がる
  • データ負債の利息は目に見えない形で毎日発生している。気づいた時には手遅れに
  • 技術的負債と同様に、早期に対処しないと修復コストが指数関数的に増大する
借金と同じ。放置すればするほど返済が大変になる。早めの対策が不可欠！

⑩汚れたデータへの対処法

1現状把握：まず自社のデータがどれくらい汚れているかを可視化する
2優先順位付け：影響の大きいデータ領域から段階的に整備していく
3自動化：表記揺れや重複を自動検出・修正する仕組みを導入
4運用ルール：データ入力のルールを定め、継続的な品質維持を目指す
  • Sansanの取引先データ診断を活用すれば、現状把握が手軽にできる

⑪AI-Readyデータとは？

  • AI-Readyデータ＝AIが正確に処理できる状態に整えられた高品質なデータ
  • 条件は主に3つ：正確性一貫性鮮度。この3つが揃って初めてAIが活きる
  • 単にデータを綺麗にするだけでなく、AIの入力として最適な形式に変換することも含む
  • AI-Readyデータが整うことで、生成AIの精度と信頼性が劇的に向上する
正確 × 一貫 × 最新。この3拍子揃ったデータこそがAIの強力な燃料になる！

⑫データクオリティマネジメントの実践

  • データクオリティマネジメント（DQM）＝データ品質を継続的に管理・改善する仕組み
  • データ品質の監視・測定・改善をループとして回し続けるアプローチ
  • 一度整えれば終わりではなく、日々の運用の中で品質を維持し続けることが鍵
  • Sansanは「AI活用を止めないデータ品質」の実装に注力している
継続は力なり。データ品質は一度やればOKではなく、日々のケアが大事なんだ

⑬Sansan Data Intelligenceの取り組み

  • SansanのData Intelligence部門が、企業のデータ品質改善を支援するサービスを展開
  • 取引先データ診断で、自社データの問題を客観的に把握できる
  • 名寄せや表記揺れの自動修正、データクレンジングの自動化を実現
  • 名刺管理の強みを活かして、BtoB領域のデータ品質に特化したソリューションを提供

⑭ソフトバンク「satto workspace」の挑戦

  • ソフトバンクが開発中の生成AIエージェント「satto workspace」はベータ版を提供中
  • AIエージェントが自律的に業務を遂行する次世代のワークスペース
  • 平岡本部長はノーコードツール開発と生成AIスタートアップの経験を活かして開発を主導
  • データ品質とAIエージェントの性能は切っても切れない関係にある
AIエージェントが自律的に動くには、基盤となるデータが信頼できることが大前提！

⑮データが整うと何が変わるのか

  • AI-Readyデータが整うと、AIの精度が飛躍的に向上してビジネスインパクトが増大
  • 部署間のデータサイロが解消され、組織全体でシームレスな意思決定が可能に
  • 予測分析や需要予測など、高度なAI活用が初めて実用レベルで機能し始める
  • 結果として組織のDXが加速し、競争力の向上に直結する
変革の連鎖
データ整備 → AI精度向上 → 業務改革 → 組織変革 → 競争力強化

⑯データ整備へのアプローチ戦略

  • まずは小さく始めて成功体験を作ること。全社一斉ではなく一部門から
  • ビジネスインパクトが大きいデータ領域を優先して整備する
  • 手作業のクレンジングではなく、ツールによる自動化を早期に導入する
  • 現場への教育と意識付けも並行して行い、データ品質の文化を根付かせる
成功の方程式：
現状診断 → 優先領域特定 → ツール導入 →
自動クレンジング → 運用ルール策定 → 継続改善

⑰「余白」のある社会への提言

  • 番組では「余白」のある社会というテーマも語られた。AIが効率化を進める中で
  • 人間にしかできない創造的・感情的な仕事に時間を割ける社会を目指すべき
  • AIに任せる業務と人間が担う業務の適切な分担を設計することが重要
  • データ整備はその「余白」を生み出すための土台作りと言える
AIが効率化してくれた時間を、人間らしさを活かす仕事に使う。それが理想的な未来
