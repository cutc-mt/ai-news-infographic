Claude Codeを自分専用に育てる

対話履歴から興味や学習内容を抽出してパーソナライズする方法を解説
📅 2026年5月31日
💬 対話ログ解析 × 🎯 興味抽出 × 📚 学習自動化 三つの軸で育てる

①Claude Codeを「育てる」とは？

  • 「Claude Codeを育てる」＝使えば使うほど自分のことを理解して、パーソナライズされた返答が返ってくる状態にすること
  • コードを書かせるだけでなく、調べ物情報収集提案など多彩な用途でClaude Codeを活用
  • にゃんたさんはYouTubeコンテンツ制作や勉強にもClaude Codeを使いこなしている
  • 自分の興味関心に近い情報を集めてもらったり、会話で提案してもらうことが可能に
Claude Codeは単なるコーディングツールじゃない！自分専用のAIアシスタントに育て上げよう！

②興味関心を記憶する仕組みの全体像

  • Claude Codeと対話すると、パソコンに会話のログが自動的に蓄積されていく
  • この会話ログを解析すれば、自分がどういった疑問や関心を持っているかが分かる
  • 会話ログから興味をプロファイルし、interest.mdというファイルに定期的にまとめさせる
  • このファイルを参照させることで、自分の興味に沿った情報収集や学習教材の作成ができる
会話 → ログ蓄積 → 興味抽出 → プロファイル化 → パーソナライズ活用。この流れが全てだ！

③Andrej Karpathyとの関連

  • Andrej Karpathy（OpenAI創業者の一人で最近Anthropicにジョイン）が話題にした「LLMの知識ベース」
  • 自分専用のコンテンツ管理システムを作る方法として以前話題になった
  • コーディングエージェントで質問した内容を参照させて知識を整理するもの。使うほど情報が溜まる
  • 今回は知識ベースとは別のアプローチ。興味関心そのものを定期抽出してパーソナライズする

④interest.mdで関心プロファイルを活用

  • 「私が最近興味あることは？」と入力すると、Claude Codeが自分の興味を理解して教えてくれる
  • 例えばエージェントハーネスの評価やClaude Code育成など、実際の関心事が的確に出力される
  • このファイルを参照して「私の興味に沿って調べて」と頼めば、自分好みの情報を集めてくれる
/interest.md を参照して、私が興味ありそうなことや
学ぶべきことを調べてください
Claude Codeが自分の関心を理解して提案までしてくれる。まるで相棒みたいだ！

⑤Claude Code対話ログの保存形式

  • Claude Codeを使うと、ローカルファイルに会話履歴が自動保存される
  • JSONL形式で、セッションごとに会話履歴が蓄積されていく構造
  • Webアプリ版のClaudeやChatGPTの会話履歴エクスポート機能を使えば過去のログも移行可能
  • Claude Codeなら直接ローカルにログが溜まるので手間いらず
Claude Codeの最大のメリット：ログが自動的に手元に残る！外部サービス頼りじゃないのが強み

⑥interest-profileスキルの処理フロー

1会話ログから未分析部分のみを抽出（全ログ参照は重すぎるため）
2ノイズ除外：挨拶や雑談、スキル実行時のコマンドなどを除去
3深掘りスコア：何回も質問しているトピックほど高く評価
4時間の重み付け：直近の関心ほど重視してスコア算出
  • 最終的にinterest.mdを生成し、興味関心が高い順に並べる設計
  • 最小限のコードはGitHubで公開済み（nyanta012/cc-personalize）

⑦ノイズ除外と深掘りスコアの仕組み

  • ノイズを除外しないと「こんにちは」などの雑談まで興味に含まれてしまう
  • Claude Codeでスキルを使う際のコマンド入力もユーザー入力として扱われてしまうので除外が必要
  • 深掘りスコア：1回で理解できる質問より、何回も質問したトピックをより興味が強いと評価
  • 「難しかった」か「興味が高かった」かの区別はつかないが、どのトピックに強い関心があるかを表す指標としては有用
挨拶やスキルコマンドがノイズになるのを見落としがち！ここをちゃんと除外するのがポイント

⑧時間の重み付けとスコア算出

  • 興味は日々変わるので、直近の関心ほど重視する必要がある
  • 直近で聞いたことと1ヶ月前に聞いたことでは、直近の方が参照したい情報になることが多い
  • 時間経過による減衰スコアを組み込んで、最新の関心が上位に来るように設計
  • 深掘りスコア × 時間重みで総合的な興味スコアを算出し、interest.mdに蓄積していく
スコア算出
深掘り度 × 時間の新しさ ＝ 興味関心の強さ

⑩interest.mdの中身を見てみる

  • 生成されたinterest.mdには、例えば「この人について」というセクションで関心事が整理される
  • 最新機能の追跡用途の深い理解実際の利用シーンの確認などが記述される
  • 「クロードコードの最新機能や特定機能の用途理解」のような具体的な関心事が抽出される
  • 興味関心が高いものから順番に並べる設計になっている

⑪パーソナライズされた検索体験

  • 「interest.mdを参照して私の興味あることを調べて」と指示すると自分の関心に沿った検索結果が返ってくる
  • 例えばClaudeエージェントやアップデートの用途別整理など、関心事ベースでまとめてくれる
  • YouTubeのネタ探しにも活用。Claude Codeが「喜びそうな提案」をしてくれる
  • 以前の動画の続編として紹介すべき論文なども的確に提案される
「用途に関心がある」というニュアンスまで理解してくれた！すごく的確な検索体験ができる

  • プログラミングを学んでいるなら、自分が質問した内容から復習問題を自動生成できる

⑭Claude Webアプリを使わなくなる体験

  • この仕組みを作ると、ClaudeのWebアプリやChatGPTをあまり使いたくなくなっていく
  • Webアプリの会話履歴エクスポート機能を使えば、過去のログをClaude Codeに移行することも可能
  • ただしClaude Codeはログが直接ローカルに溜まるので、エクスポートの手間が不要
  • ローカル完結でプライバシーも安心。外部サービスに依存しないのが最大の魅力

⑮Claude Desktopルーティンで定期実行

  • 興味は日々変わるのでinterest.mdを定期的に更新する必要がある
  • 推奨はClaude Desktopアプリのルーティン機能を使う方法
  • 好きな時間にinterest-profile syncを自動実行するスケジュールを設定可能
  • Desktopアプリのスタートアップ時に実行をオンにすれば、PCを開くだけで自動起動
ルーティン設定例：
「/interest-profile sync を実行して、
情報収集や動画のネタを考える際に
参考になるコンテキストを作成する」

⑯定期実行のその他の方法と注意点

  • claudeコマンドのloopコマンドもあるが、毎回セッションを立ち上げる必要があって面倒
  • cronなどを使った定期実行も検討したが、Claudeの認証情報が期限付きで切れる問題があった
  • 結果的にClaude Desktopアプリのルーティンが一番確実との結論
  • Desktopアプリは認証済みのセッションを使うため、認証切れのリスクなし
cronでやろうとしたら認証で失敗した…Desktopアプリならその問題もクリア！
