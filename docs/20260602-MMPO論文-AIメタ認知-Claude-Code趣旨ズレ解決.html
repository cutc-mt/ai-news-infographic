セクション1：Claude Codeが趣旨をズラす理由

① Claude Codeの「趣旨ズレ」現象

  • Claude Codeで長いタスクを任せると、途中から趣旨がズレる現象が起きる
  • 例えば「コードのリファクタリング」を頼んだのに、途中で機能追加を始めてしまう
  • タスクが長くなるほど、本来の目的から遠ざかる確率が高まる
  • ユーザーは「なぜAIが道を見失うのか」と戸惑う場面が多い
  • これはLLMエージェントに共通する構造的な問題である
「これ、頼んでないよ…」
Claude Code使いなら
一度は経験があるはず！

② 根本原因：中間記憶の劣化

  • エージェントはタスクの進行中に重要な情報を忘れていく
  • 中間記憶の劣化：長時間タスクでは記憶容量が限界を迎える
  • 情報圧縮の過程で重要な細部が失われる
  • 要約すればするほど、元のニュアンスや文脈が欠落していく
  • 記憶が「薄っぺらになる」ことで、判断の質が低下する
情報をギュッと詰めるから
大事なピースが
落ちてしまうんだ。

③ 最も重要な点：AIが「忘れたことに気づいていない」

  • AIは「自分が忘れていること」に気づいていない
  • 自分の記憶の不足や劣化を自覚する能力がない
  • 人間なら「あれ、なんか忘れてるかも」とメタ認知が働く
  • AIは忘れたことすら忘れているので、自信たっぷりに間違った方向へ進む
  • これこそが「趣旨ズレ」の真犯人
「忘れたことを忘れる」
これが一番怖い。
人間なら気づけるのに！

④ Target Drift ＆ コンテキスト制約

  • Target Drift：本来の目的から徐々に離れていく現象
  • コンテキストウィンドウの制約：1Mトークンでも長時間タスクでは足りない
  • 過去のやり取りを要約して保存するが、要約自体が情報を失う
  • タスク時間が長いほどドリフトは大きくなる傾向がある
  • 悪循環：記憶劣化 → 誤判断 → 余計な作業 → さらなる記憶圧迫

⑪ Belief Entropyの4段階フロー

1 推定：現在の記憶状態からタスクの潜在状態を推定する
2 計算：その推定の不確実性（エントロピー）を計算する
3 判定：エントロピーが高い ＝ 記憶が不十分 or 品質が悪い
4 学習：高エントロピーの要約にペナルティ → より良い記憶政策を学習
  • このサイクルを繰り返すことで記憶の品質が向上していく
