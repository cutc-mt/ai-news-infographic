セクション1: Qoderとは

① Qoder プラットフォーム概要

  • Alibaba系の包括的AIプラットフォーム
  • AI IDE から自律的開発デスクトップへと進化を遂げる
  • Qoder 1.0マイルストーン達成、本格的な安定運用フェーズへ
  • 単なるコード補完ではなく開発全体を自律化する環境
  • コミュニティ版＋エンタープライズ版の二層構造で展開中
「コードを書く補助」から
「開発を丸ごと任せる」
への大転換がここにある！

② 対応モデルと料金体系

  • 対応モデル: Claude / GPT / Gemini / Qwen3.5-Plus / Qwen3.7-Max / MiniMax M3
  • コミュニティ版でBYOK（Bring Your Own Key）に対応
  • 無料ティアでQwen3.7-Maxを1日200回まで利用可能
  • 自身のAPIキーを持込できるためコスト管理が柔軟
  • Ultimateモデルティア半額プロモ実施中（4/26〜期間限定）

  • エージェントが自動で計画→コーディング→検証を自律実行
  • タスクの分解・優先順位付け・実行まで人間の介在を最小化
  • 複雑なマルチステップタスクも自然言語一発で起動
  • 進捗はリアルタイム可視化、必要な箇所だけ人間がレビュー

セクション2: Qoder CLIの進化

④ Qoder CLI: ターミナルAI開発の革新

  • 2025年10月に正式リリースされたコマンドラインツール
  • 超高速起動：70ms未満。待ち時間ゼロで即座に利用可能
  • IDEを開かずターミナルから直接コードレビュー・修正・リファクタリング
  • 開発ワークフローにシームレスに統合できる
  • WSL接続サポートでWindows環境でも問題なく動作

⑤ TUI / Bash モードの2面性

  • TUIモード: チャットインターフェース＋スラッシュコマンドで対話的に操作
  • Bashモード: シェルコマンドをAIが直接実行。自動化パイプライン構築に最適
  • 2つのモードをシームレスに切り替えながら作業可能
  • チャットで指示を出しつつ実際のコマンド実行もAIに任せられる
  • 熟練エンジニアのターミナル中心ワークフローに完全フィット

セクション3: デジタル従業員アプリ「QoderWake」

⑨ QoderWake: 「常に稼働する」AI従業員

  • 2026年4月30日に招待制プレビュー公開（Alibaba）
  • 「常に稼働し、働き続ける」AI従業員の概念を体現
  • 従業員と作業環境を分離設計し、環境が変わっても継続稼働
  • 役割: ソフトウェアエンジニア・運用担当・アナリストなど多岐にわたる
  • ビジネスプロセスに合わせてカスタマイズ可能な従業員を配置
AI従業員が休まず働く。
環境が変わっても
記憶とスキルを持ち続ける！

⑩ QoderWake 5レイヤー構造

  • QoderWakeは5つのレイヤーで構成される独自アーキテクチャ
ID（アイデンティティ） — 従業員の固有ID
Memory（記憶） — 長期・短期記憶
Skills（スキル） — 業務能力セット
Division of Labor（分業） — 役割分担
Permission Red Line（境界） — 権限の赤線
人間の組織と同じように
5層でAI従業員を定義。
権限の赤線が安全の要！

⑪ 環境分離と継続稼働の仕組み

  • 従業員と作業環境を分離: プラットフォーム間を横断して稼働
  • 環境が変わってもアイデンティティと記憶を保持し続ける
  • プロジェクトをまたいでも一貫したパフォーマンスを発揮
  • チーム全体でAI従業員を共有し、分業体制を構築可能
  • 新規メンバー追加時も既存の記憶を引き継ぎ、即戦力として稼働
環境が変わっても
記憶を持ったまま働ける。
これが本当の「従業員」だ！

⑫ カスタム役割とビジネス適用

  • ソフトウェアエンジニア: コーディング・デバッグ・レビューを自律実行
  • 運用担当（DevOps）: インフラ監視・トラブル対応・デプロイ自動化
  • アナリスト: データ分析・レポート生成・ビジネスインテリジェンス
  • ビジネスプロセスに合わせてカスタマイズ可能。役割は無限に拡張
  • 人間の従業員と協働する形で組織に自然に統合できる
