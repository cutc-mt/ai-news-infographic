セクション1：エージェンティックアーキテクチャとは

① エージェンティックアーキテクチャの概要

  • AIがコードを「提案するだけ」から「自律的に実行」するパラダイムシフト
  • 2025〜2026年は「エージェントモデル爆発」の波が本格化
  • 従来の単一エージェント（autocomplete）からマルチエージェント協調へ進化
  • 自然言語プロンプトがマルチステップの自律的コーディングアクションに変換される
  • 開発者が「監督者」になり、AIが計画・実行・テスト・検証を担う世界線
「AIに書かせる」じゃなくて
「AIに開発させる」。
ここが決定的に違う！

② 単一エージェント → マルチエージェントへの進化

  • 第一世代: Copilot的なオートコンプリート — コードの「提案」に留まる
  • 第二世代: 単一エージェントがファイルの編集・コマンド実行まで担当
  • 第三世代: マルチエージェント協調 — 複数のAIが役割分担して開発を並列推進
  • 設計エージェント・コーディングエージェント・テストエージェントが同時に稼働
  • 人間は「オーケストレーター」として全体を監督するスタイルに移行
Copilot → Agent → Multi-Agent
進化のスピードが
加速している！

③ 核心サイクル：Plan → Execute → Test → Verify

Plan → Execute → Test → Verify
  • Plan（計画）: 自然言語の指示からタスク分解・アーキテクチャ設計を生成
  • Execute（実行）: 計画に基づいてコード執筆・ファイル生成を自動実行
  • Test（テスト）: 自動でユニットテスト・統合テストを作成・実行
  • Verify（検証）: 結果が期待通りかを判断し、必要なら修正ループを回す

AIが自律ループで
完結する仕組みだよ。

セクション2：Google Antigravity

⑤ Antigravityとは：エージェントファーストIDE

  • 2025年11月リリースの「エージェントファーストIDE」。Googleが本気で挑む開発ツール
  • VS Codeフォークベースで馴染みのある操作感を維持しつつ、エージェント機能を全面採用
  • マルチエージェント協調を単一エージェントのオートコンプリートより優先する設計思想
  • 対応モデル: Gemini 3.1 Pro / Claude Sonnet 4.6 / Claude Opus 4.6
  • VS Code拡張としても提供され、既存ワークフローへの段階的導入が可能
「エージェントファースト」=
AIが先に動くIDE。
VS Codeがベースで移行も楽チン。

⑥ Antigravityのエージェント横断動作

  • エージェントがエディタ・ターミナル・内蔵ブラウザを横断して動作
  • エディタでコードを書きつつ、ターミナルでコマンド実行し、ブラウザでUIを検証
  • 並列処理で同時に複数のタスクを推進する仕組み
  • エージェント同士がコミュニケーションを取りながら協調的に開発を進める
  • 安全性も重視：エージェントの権限管理・承認フローが組み込まれている
エディタ・ターミナル・ブラウザ
全部またいで
AIが動くんだよ！

⑦ Antigravityの並列タスク処理

  • アーキテクチャ設計エージェントが全体の構造と設計を立案
  • コード執筆エージェントが設計に従って各モジュールの実装を並列で進行
  • テストエージェントが自動テストの生成・実行を同時に行う
  • UI検証エージェントが内蔵ブラウザでレンダリング結果をチェック
  • Agent-first思想: 人間は監督者、エージェントが作業。人間の負担を劇的に削減
4つのエージェントが同時に走るから
開発スピードが何倍にもなる！

⑧ Antigravityの使いどころ

  • 初期設計フェーズに最適。マルチエージェントが全体のアーキテクチャを並列検討
  • 複数ファイルにまたがる大規模リファクタリングで力を発揮
  • UI/UXの実装では内蔵ブラウザで即座に検証できるのが強み
  • チーム開発ではエージェント間のコミュニケーションが品質を担保
  • 注意: Gemini / Claude / GPTなど複数モデルの組み合わせで使うのが推奨

セクション3：Claude Code（Anthropic）

⑨ Claude Codeとは：ターミナルベースのエージェント

  • Anthropicが開発したエージェンティックコーディングツール（ターミナルベース）
  • コードベースとの対話・ファイル編集・コマンド実行・Gitワークフロー管理を網羅
  • 自然言語で操作。「この関数をリファクタリングして」と伝えるだけでOK
  • IDEの拡張機能ではなくターミナルで直接動くため、あらゆる環境で利用可能
  • VS Code / JetBrains等のIDEとも統合可能。柔軟性が高い
ターミナルを開いて
「clauode」と打つだけ。
エージェント開発の入口。

⑩ Claude Codeの搭載モデル：Opus 4.7

  • 搭載モデル: Claude Opus 4.7 — 高度なソフトウェアエンジニアリングに特化
  • 長時間タスクの実行に強化。複雑な開発タスクでも文脈を維持して処理
  • 100万トークンのコンテキストウィンドウで大規模コードベースに対応
  • プロジェクト全体を把握した上で、的確な変更提案と実行が可能
  • Opus 4.7は推論力とコーディング能力がトップクラス。SWE-benchで好成績

⑪ Claude Codeのワークフロー強み

  • 反復開発に最適。試行錯誤を高速に繰り返しながら品質を向上
  • 問題探求能力に優れる。バグの根本原因を深く掘り下げて特定
  • 新解決策の発見。既存コードを見直してより良いアプローチを提案
  • Gitワークフローをネイティブで管理。コミット・PR・レビューを自動化
  • ターミナル中心のワークフローに最適。Vim/Neovim使いにも馴染む

⑫ Claude Codeの使いどころ

  • 実装フェーズに最適。深度あるコーディングで複雑なロジックを確実に実装
  • バグ修正・デバッグで大規模コードベース全体を分析して原因特定
  • 大規模リファクタリングで影響範囲を正確に把握しながら安全に変更
  • オープンソースへのコントリビューション作業でも快適に使える
  • 100万トークンのおかげで巨大リポジトリでも文脈を失わないのが最大の強み

セクション4：OpenAI Codex

⑬ OpenAI Codexとは：高速エージェントCLI

  • オープンソースのCLI + IDE拡張。OpenAIがコミュニティに公開
  • ChatGPTワークフローにエージェント機能を統合。使い慣れた環境で利用可能
  • GPT-5.5でエージェント性能が大幅改善。推論速度も向上
  • ワンショットでのアプリ生成・高速プロトタイピングが得意
  • マルチプラットフォーム対応: クラウドWeb / CLI / VS Code / Cursor / macOSデスクトップ
オープンソースで無料。
GPT-5.5のスピードで
サクッとプロトを作れる！

⑭ Codexの強み：高速・低コストな実行

  • 推論速度が速い。短時間で大量のコード生成が可能
  • 低コストで実行できる。小規模〜中規模のタスクにコスパ最強
  • ターミナル・システムレベルタスクで効率的。環境構築・スクリプト作成に最適
  • Cursor / VS Code拡張としても動作。既存のIDE環境にシームレス統合
  • macOSデスクトップアプリもあるため、GUI環境でも快適に利用可能
速い・安い・手軽。

⑮ Codexの使いどころ

  • プロトタイピングに最適。アイデアを即座に動くコードに変換
  • 既知のプロセス・定型作業の自動化。反復的なコード生成で威力発揮
  • コードレビューで素早く修正提案を生成。PRのレビュー効率を向上
  • セキュリティチェック・リントなど定型レビューの自動化にも向いている
  • マルチプラットフォームなのでどこでも同じ体験で使えるのが便利
