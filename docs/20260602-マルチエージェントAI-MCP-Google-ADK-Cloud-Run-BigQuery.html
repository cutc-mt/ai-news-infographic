③ Google ADK（Agent Development Kit）

  • Googleが開発したオープンソースのAIエージェントフレームワーク（Python製）
  • LlmAgentクラスでLLM + ツール + 会話フローを1つのオブジェクトに統合
  • 推論モデルにはGemini 2.5 Flashを採用。高速かつ低コストで動作
  • 複数エージェントの役割分担が可能：要件収集・リソース評価・提案生成など
  • 各エージェントにinstruction（システムプロンプト）を渡して専門性を定義
LlmAgent 1つで
LLM + ツール + 会話
が全部まとまる。超シンプル。

④ MCP（Model Context Protocol）

  • AIエージェントが外部ツールと標準的に通信するためのオープンプロトコル
  • Cloud API Registry経由でBigQueryのマネージドMCPサーバーに接続可能
  • 自前MCPサーバー不要。Google側が管理済みのサーバーを提供
  • 利用可能なツール：execute_sql・get_table_info・list_dataset_ids・list_table_ids・get_dataset_info
  • エージェントはMCPツールをAPI Registryのget_toolset()で簡単取得
自前でMCPサーバーを
立てなくてOK。
Googleがマネージドで用意済み！

⑤ Cloud Run: サーバーレスでエージェントをデプロイ

  • エージェントをDockerコンテナ化してサーバーレスにデプロイ
  • adk api_serverコマンドでREST APIを自動公開
  • ゼロスケール対応でアイドル時はコストゼロ。リクエストに応じて自動スケール
  • Swagger UI（/docs）が自動生成。APIのテストと確認が手軽
  • 自前でスケーリング・監視を管理。シンプルで低コストな運用が可能

  • プロジェクト関連データの中央リポジトリとして活用

⑦ コード例: わずか数行でエージェント完成

  • Pythonのコード量が極めて少ない。ADKとAPI Registryの組み合わせだけで構築可能
  • Python依存はgoogle-adkとgoogle-cloud-aiplatformの2つだけ
from google.adk.agents import LlmAgent
from google.adk.integrations.api_registry import ApiRegistry

tool_registry = ApiRegistry(PROJECT_ID, location="global")
toolset = tool_registry.get_toolset(
  f"projects/{PROJECT_ID}/locations/global/"
  f"mcpServers/google-bigquery.googleapis.com-mcp"
)

agent = LlmAgent(
  model="gemini-2.5-flash",
  name="project_intake_agent",
  instruction=SYSTEM_INSTRUCTION,
  tools=[toolset],
)

⑩ 要件収集エージェント: インプットの整理

  • 整理した要件を次のエージェント（リソース評価）に引き渡す

⑪ リソース評価エージェント: BigQueryで過去データ分析

  • チーム規模・工数・予算・成功/失敗パターンを定量的に評価

  • データ駆動の意思決定。勘や経験ではなく過去の実績に基づく提案

⑰ MCP経由のBigQueryツール一覧

  • execute_sql: SQLクエリを実行。自然言語から自動生成されたSQLをBigQueryで実行
  • get_table_info: テーブルのスキーマ情報を取得。LLMが適切なSQLを組むために活用
  • list_dataset_ids: プロジェクト内のデータセット一覧を取得
  • list_table_ids: データセット内のテーブル一覧を取得
  • get_dataset_info: データセットのメタデータを取得。エージェントがデータ構造を理解するのに利用

BigQueryを完全操作。
