SECTION 2: 長時間稼働エージェントの仕組み

⑥ ステートレスAIモデルの限界を突破

  • 従来のAIはステートレス：リクエストごとに記憶がリセットされる
  • 長時間のタスク（数時間〜数日）ではコンテキスト喪失が致命的
  • Google Agentic Stackは状態を永続化する新しいパラダイム
  • 外部ストレージにセッション状態を保存・復元する仕組み
  • 時間をまたいだ継続的なタスク実行を可能にする
従来: 「毎回最初から」
Agentic Stack: 「続きから再開」
この差は大きい！

⑦ セッション永続化の仕組み

  • 状態とコンテキストを外部ストレージに長期間保持
  • エージェントが途中で停止しても、再開時に正確に状態を復元
  • 履歴・判断・進捗の完全なトレーサビリティ
  • 並行稼働の複数エージェントが独立したセッションを維持
  • Cloud Spanner等のマネージドDBに自動保存
実行中 保存 停止 復元 再開
人間の「仕事を途中で保存」
と同じ。エージェントにも
「続きから」ができる！

⑧ イベント駆動のスリープ＆ウェイクアップ

  • アイドル時にエージェントは「スリープ」状態に入る
  • 外部イベント（メール受信・API応答等）で「ウェイクアップ」
  • イベント駆動アーキテクチャでリソースを効率的に活用
  • スリープ中はコンピューティングコストがほぼゼロ
  • Pub/SubやCloud Tasksで外部トリガーと連携
待つ間は寝て待つ。
何かあったら起こして。
省コストで賢い設計！

⑨ Human-in-the-Loop (HITL) Inbox

  • 制限アクション（送信・削除・決済等）でエージェントが自動一時停止
  • 人間に承認リクエストをInboxに通知
  • 人間が承認すると自動的に実行再開
  • HRオンボーディング・営業アプローチなど数日〜数週間のタスクに対応
  • 自律性と人間の管理を最適なバランスで両立
「勝手に送信しないで」
→ 人間がOKを出してから
実行再開。安心安全！

⑩ Gemini Spark: 常時稼働する自律エージェント

  • Cloud VM上で24/7常時稼働する自律エージェント
  • バックグラウンドで継続的なタスク処理を実行
  • 異常検知・フラグ機能で問題を自動検出
  • 重要事象を人間に通知して判断を委ねる
  • 自己修復・リトライ機能で耐障害性を確保

SECTION 4: MCP & A2A通信プロトコル

⑯ MCP: Model Context Protocol

  • エージェントと外部ツール・データソースを接続する標準プロトコル
  • ファイルシステム・API・DBなどあらゆるツールに統一アクセス
  • オープン標準なので、独自のツールも簡単に追加
  • エージェントが自ら必要なツールを発見・選択して利用
  • 「ツールごとに自前の連携」を不要にする
エージェント MCP ツール群
USBみたいに繋ぐだけ。
エージェントの能力が
ツールの分だけ広がる！

⑰ A2A: Agent-to-Agent Protocol

  • エージェント同士が直接通信・協調するための標準プロトコル
  • メッセージの送受信・タスク委任・結果共有を標準化
  • 異なるフレームワークで作られたエージェントとも相互運用
  • 組織の壁を越えたエージェントのオーケストレーション
  • 認証・権限・監査ログでセキュアな通信を保証
