⑰ HTMLを選ぶ5つの理由

  • 表やSVG画像、コードなど豊かに表現できる情報密度の高さ
  • 構造を階層的に整理し、モバイルにも対応できる読みやすさ
  • リンク1つで誰でも開ける共有のしやすさ
  • デザインを調整し、その内容をプロンプトとしてClaude Codeに戻せる双方向性
  • MCPやWebブラウザからの情報を取り込み、HTMLレポートを作れるデータ統合力
HTMLの方がトークン数は増えるが、表現力と人間の理解可能性は全体としてHTMLが勝る。

⑱ HTMLの主な活用例と使い始める方法

  • 複数の設計案を1枚のHTMLページに並べて比較する手法
  • アプリやウェブサイトのプロトタイプを作る手法
  • 複数の情報源から読みやすいHTML形式のレポートを作る手法
  • 始め方は簡単：「HTMLファイルを作ってください」または「HTMLアーティファクトを作って」と頼むだけ
大事なのはHTMLを作らせること自体ではなく、何のために使うのかを考えること。比較・説明・レビュー・編集によって作るべき形が変わります。
