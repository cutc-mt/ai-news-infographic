Codexに「Sites」機能が登場

Webサイトを生成して公開まで全部やってくれる新機能を徹底解説
2026年6月3日

① Codex「Sites」とは — 新機能の概要

  • OpenAIのCodexにプラグインとして「Sites」機能が追加された
  • これまでのCodexでもウェブサイトは作れたが、作成→ビルド→公開までを一貫して自動でやってくれるのが新機能のポイント
  • OpenAIのサーバー上に公開される仕組み。生成したサイトはすぐにプレビュー可能
  • ワークスペースのメンバーと共有することもできる。チーム開発にも対応
従来のAIコーディングは「コードを書く」まででしたが、Sitesは「公開までやる」のが大きな違いです。

② 利用可能プラン — ビジネス以上が必要

  • 現状はビジネスプランまたはエンタープライズプランのみ利用可能
  • 一般のプランへの提供時期はまだ未定。OpenAIの動向次第
  • ビジネスプランならチームメンバーを追加して共同利用が可能
  • ケトさんはビジネスプランに課金済みのため、実際に触って検証を実施
今回の検証はビジネスプランで実施。
一般プランへの展開が最大の注目ポイントです。

③ 基本的な使い方 — 超シンプル

  • プラグインの中にある「Sites」をクリックして起動
  • サイトコマンドを入力して、音声入力で作りたい内容を伝えるだけ
  • ほとんど勝手にローカル上にウェブサイトを構築してくれる
  • 「Google Chromeで確認する」を押すと、完成したサイトがブラウザで表示される
入力 → 自動生成 → プレビュー
の3ステップだけでサイトが完成。
基本操作はこれだけです。

④ 公開先と共有機能

  • 生成されたサイトはOpenAIのサーバー上に自動公開される（ドメイン: chatgpt.team など）
  • 左上の「Sitesタブ」が新設され、公開済みサイトの一覧を確認可能
  • 招待された人やワークスペースにリンクしている人全員と共有できる
  • Codex上で何度でも編集・更新作業が可能。リアルタイムで反映される
ただし独自ドメインの設定は
現状ではできないのが大きな制約。
公式ドメインでの運用のみ。

⑤ 環境変数・シークレットキーの管理

  • Sitesの「Settings」画面から環境変数を保存できる
  • APIキーやシークレットキーといった機密情報を安全に管理可能
  • URL、機能スイッチなどの設定値も保存できる。Web制作に必要な変数を一元管理
  • AI機能が組み込まれたアプリを作る際、APIキーを事前にセットしておくことでアプリが動作する仕組み
OpenAI APIキーをここにセットすれば、
AI連携アプリもコデックス上で
完成から動作確認までできるようになります。

⑥ 生成されるサイトのクオリティ

  • ケトさんの評価：「全然文句なしのクオリティ」。いい感じにまとまっている
  • クオリティはプロンプトの書き方に大きく依存する。詳細な指示ほど良い結果が得られる
  • 曖昧なお願いだとデザインが微妙になりがち。明確なイメージや画像を添付するのがおすすめ
  • 画像の添付も対応。参考デザインがある場合、それを渡すだけで精度が上がる
プロンプト次第で天と地。
単純な指示だとシンプルな結果に。
具体的な指示が鍵です。

⑦ デザインプラグインとの連携 — 画像からコードへ

  • デザイン系プラグインと組み合わせるのがベストプラクティス
  • Step 1: 画像生成でデザインを作成する
  • Step 2: そのデザイン画像をSitesに渡してコーディングさせる
  • デザインがはっきりしている状態でSitesがコーディングを開始するため、精度が大幅に向上
例えば「イメージLPビルダー」スキルで
デザイン画像を生成 → Sitesで再現
の流れが最強の組み合わせです。

⑧ Creative Production — AIが企画を考えてくれる

  • 左上の「Creative Production」プラグインがクリエイティブの方向性を提案
  • 「訴求の軸は何にしますか？」「ビジュアル言語は？」「避ける表現は？」と対話形式でディレクション
  • 質問に答えると、デザイン案や画像を自動生成してくれる
  • 企画立案 → デザイン生成 → Sitesでコーディングの全自動フローが構築できる
素人でもプロ相当のディレクションを
AIがサポートしてくれるのが強力。
プラグインの組み合わせがここで光ります。

⑨ コストの現実 — 1サイト約1万円

  • 1回のLP作成で約600クレジット（約1万円）を消費した実例
  • 生成には約30分の時間を要した。かなり時間もかかる
  • 推論レベル（reasoning effort）を「非常に高い」に設定したのが高額の要因
  • 「高」や「中」に設定すればコストを抑えられる可能性があるが、品質とのトレードオフ
ケトさん：「びっくりするぐらいお金が
かかった。現状即座のおすすめは
できないかもしれない。」

  • 完成したサイトをChromeで開いて確認。シンプルな構成でまとまっている
  • 画像はAI生成のもの。品質はプロンプトの詳細度に比例する傾向
  • 完成後はSitesタブに自動保存。いつでも再編集・更新が可能

⑪ Vercelプラグイン — 独自ドメインで運用

  • 「Vercel」プラグインを使えばCodexから外部のデプロイ環境へ移行可能
  • 「Vercelに公開してください」と指示するだけで、サクッと公開作業を完了
  • Vercel側でドメインを購入（年間約$9）し、独自ドメインをセットすれば本格運用が可能
  • app.vercel.app形式のURLで即座に公開され、後から独自ドメインを割り当てられる
Codexでサクッと作る →
Vercelに移行 → 独自ドメイン
で運用するのが現実的な流れです。

⑫ Codex→Vercelのデプロイ手順

  • CodexのSitesでウェブサイトを生成・作成する
  • Vercelプラグインを起動して「Vercelに公開してください」と指示
  • Vercel上に自動デプロイ完了。app形式のURLで公開される
  • Vercelダッシュボードでドメインを購入・設定して独自ドメインと連携
この4ステップで独自ドメインの
ウェブサイト運用が可能に。

⑰ 現状の評価 — 即おすすめとは言えない

  • コストが高額（1サイト約1万円）で現状は即座にはおすすめできない
  • 独自ドメインに対応していないのが最大の弱点。実運用には不向き
  • プレビューとしてサクッと公開できる点は便利。テスト環境としての価値はある
  • 生成に20〜30分かかる。推論レベルを下げれば短縮可能だが品質への影響は未知数
