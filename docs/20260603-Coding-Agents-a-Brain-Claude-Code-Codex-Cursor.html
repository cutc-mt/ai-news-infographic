コーディングエージェントに「脳」を与える

Agent Memory で Claude Code / Codex / Cursor がセッションをまたいで記憶を保持する仕組み
2026年6月3日

① コーディングエージェントはすぐに忘れる

  • どのエージェントもコンテキストウィンドウが一杯になると全てを忘れてしまう
  • プロジェクトの構成、コーディング規約、過去の意思決定を毎回ゼロから説明し直す必要がある
  • 同じような指示を何度も繰り返す非効率なサイクルが発生
  • Claude Code Codex Cursor — どのエージェントでも共通の課題
「いつも同じことを教えてる…」
この不満を解決するツールが登場！

② CLAUDE.md だけでは不十分

  • CLAUDE.md は「付箋」のようなもの — 毎回コンテキストウィンドウに読み込まれる
  • すぐに情報が古くなる（stale）。プロジェクトが進むと内容と現実が乖離していく
  • ファイルが大きくなるとコンテキストを圧迫し、他の重要な情報を追い出してしまう
  • 検索性がなく、関連情報だけをピンポイントで引き出すことができない
静的なファイル vs 動的なデータベース。
ここが根本的な違いです。

③ Agent Memory が解決策

  • 背後に検索可能なデータベースがあり、動的に情報を管理
  • エージェントの行動を自動的にキャプチャし、関連する情報だけを引き出す
  • Claude Code、Cursor、Codex などすべてのエージェントで動作
  • MCP（Model Context Protocol）サーバーとして動作するアーキテクチャ
付箋ではなく、
エージェント専用の「記憶」を構築！

④ 対応エージェント一覧

  • Claude Code — AnthropicのCLIエージェント。MCPプラグインで直結
  • OpenAI Codex — OpenAIのコーディングエージェントにも対応
  • Cursor — VS CodeベースのAIエディタでも利用可能
  • MCPサーバーとして動作するため、MCPに対応していればどんなエージェントでも接続可能
エージェントを問わず、
1つの記憶基盤で統一できるのが
大きな魅力です。

⑤ インストール手順 — npm で一発

  • グローバルインストールを実行：
    npm install -g agent-memory
  • インストール後に agent-memory と打つだけでサーバーが起動
  • Windows・Macのどちらでも基本は同じ手順
  • ただしWindowsでは接続コマンドが使えない場合があり、手動設定が必要
$ npm install -g agent-memory
$ agent-memory
// サーバーが起動します
npm一発でインストール完了。
とてもシンプルです。

⑥ Docker で動かす方法

  • Dockerを使うのが特におすすめ。特にWindowsではトラブルが少ない
  • 環境変数を設定：AGENT_MEMORY_USE_DOCKER=true
  • 設定後に agent-memory を実行すると、Dockerイメージが自動ダウンロードされる
  • プロンプトでEnterを押すだけでコンテナが立ち上がる
$ export AGENT_MEMORY_USE_DOCKER=true
$ agent-memory
// DockerイメージがDLされて起動
Windows勢は迷わず
Dockerを選びましょう！

⑨ Claude Code にプラグインを追加

  • Claude CLIでplugin marketplaceコマンドを実行
  • Agent MemoryのGitHubリポジトリを追加:
    plugin add <repo-url>
  • 追加後に plugin install agent-memory でユーザースコープにインストール
  • インストール完了でClaude CodeがAgent Memoryのツールを使えるようになる
$ claude
> plugin marketplace
> plugin add agent-memory
> plugin install agent-memory --scope user

⑩ MCP設定ファイルで接続

  • MCP.jsonファイルを作成してClaude CodeにAgent Memoryを接続
  • Agent Memoryは裏側でMCPサーバーとして動作している
  • サーバーのURLをMCP設定に記述すると、Claudeが自動的に検出して利用
  • Windowsでは connect コマンドが使えないため、この手動JSON設定が必要
// mcp.json の中身
{
  "mcpServers": {
    "agent-memory": {
      "url": "http://localhost:3113"
    }
  }
}
Windowsユーザーは注意！
connectコマンドが使えないので
JSONファイルを手動で作成しましょう。

⑬ Hooks が本番の魔法を発動

  • MCP接続だけではバックグラウンド記録はまだ有効にならない
  • Hooksが実際の「魔法」を担当。すべてのアクションを自動で記録する仕組み
  • 各アクションごとに発火する小さなスクリプトが、メモリサーバーに情報を送信
  • 設定するとエージェントの作業が全自動で記憶に蓄積されていく
MCPは「道具」を渡すだけ。
Hooksで「自動記録」をONにして
はじめて本物の記憶が完成します！

⑭ Hooks のセットアップ手順

  • まずグローバルnode_modulesのパスを取得：
    npm root -g
  • パスを元にAgent Memoryのスクリプトディレクトリを特定
  • settings.jsonを作成し、プロジェクトローカルにHooksを設定
  • スクリプトへのフルパスを正確に指定するのが重要なポイント
// settings.json の例
{
  "hooks": {
    "PreToolUse": [...],
    "PostToolUse": [...],
    "Notification": [...]
  }
}

⑰ Strength — 記憶の強度システム

  • 各記憶には「Strength（強度）」という重要な指標が付与される
  • よく使われる有用な記憶は強度が上がり、システムからの信頼度が増す
  • 使われない記憶は徐々に減衰し、最終的に自動削除される
  • これにより記憶が肥大化せず、常にシャープな状態を維持する仕組み
人間の記憶と同じ！
よく使うことは覚えて、
使わないことは忘れていく。
