① Hermes Agent Desktop とは？

  • オープンソース・MIT ライセンスの AI エージェントデスクトップアプリ。以前から CLI 版があったが、デスクトップ版で一気に使いやすくなった
  • macOS / Windows / Linux対応。OS を自動検出して適切なダウンロードを表示
  • 公式サイトのキャッチコピーは「The agent that grows with you」。ユーザーと共に成長するエージェントという理念
  • 現在のバージョンは v0.5.x。急速に開発が進んでおり、頻繁にアップデートが配信されている
単なるチャットアプリではなく、
本格的なエージェントプラットフォーム。
ターミナル苦手な人でも使える！

② デスクトップ UI の全体像

  • 左パネル：セッション管理。新しいセッションの作成、検索ボックス、ピン留め、過去のセッション履歴を一覧表示
  • 中央：メインのチャットエリア。「バグやファイルパスを貼り付ければテスト付きで修正」というメッセージが表示される
  • 下部：メッセージ入力ボックス、コンテキスト追加、音声コントロール付きのチャットインターフェース
  • ターミナルツールではなく、ちゃんとしたアプリケーションとしての体裁を備えている点がポイント
エージェントを使い続けていると
セッションがどんどん溜まる。
検索とピン留めで整理できる！

③ ボトムバーで全体をコントロール

  • Gateway：Hermes をデスクトップアプリと他のインターフェース・ワークフローにつなぐ役割。正常に接続されると「Gateway is ready」と表示
  • Agents：サブエージェントの「Spawn Tree（スポーンツリー）」を表示。メインエージェントが作業を委譲したときの進捗がリアルタイムで分かる
  • Cron：スケジュールされたタスクの管理。毎朝のレポートや定期チェックなど、自動化の要
  • Model Picker：プロバイダーを問わずモデルを切り替えられる。現時点で最大の魅力ポイント
このボトムバーがHermesを
単なるチャットUIから
エージェントプラットフォームに変える。

④ コーディングとローカルタスクに特化

  • メインのチャットエリアに「バグを貼り付ければテスト・差分・PR付きで修正」と明記。開発者向けの設計意図が明確
  • ファイルパスを投げるだけでエージェントが読み込んで対処。コードベースの理解・修正がスムーズ
  • テストの自動生成、差分の作成、プルリクエストの作成まで一貫して対応。開発フローに直接組み込める
  • CLI ツールとしての側面も残しつつ、GUI で見える化することで、開発現場の導入ハードルを大幅に下げている
単純なチャットボットが欲しい人には
機能過多かもしれない。
でもエージェント用途なら最強。

⑤ 108個のスキルカタログをトグルで管理

  • スキル＆ツールページには検索ボックスとスキル／ツールセットのタブ。108個ものスキルがカテゴリ別に一覧表示される
  • カテゴリ例：Apple / Creative / GitHub / MLOps / Productivity / Research / Software Development など多岐にわたる
  • 各スキルには短い説明とトグルスイッチが付いている。ONにするだけでエージェントがそのスキルを使えるようになる
  • 具体的なスキル：Apple Notes、Apple Reminders、iMessage 検索、macOS のコンピュータ操作、Claude Code、Codex など
設定ファイルをいじる必要なし。
トグル一つでスキルをON/OFF。
直感的で分かりやすい設計！

⑥ ツールセットでワークフローを構築

  • スキルとは別にツールセットのタブも存在。複数のツールをグループ化してまとめて有効化できる
  • Hermes Agent 自身や、Kanban ボード、CEX Lane など、プロジェクト管理系のツールも揃っている
  • スキル＋ツールの組み合わせで、エージェントの能力を自分好みにカスタマイズできるのが大きな魅力
  • カテゴリでのフィルタリングが可能で、大量のスキルの中から必要なものを素早く見つけ出せる

⑨ Cron でエージェントの定期実行

  • 「Create First Cron」フォームで名前・プロンプト・頻度・出力先を設定。定期タスクを数ステップで作成
  • 自然言語で頻度を指定可能。「every 15 minutes」と入力すると自動的に cron 式に変換される
  • 用途例：未読 Slack スレッドの要約、日次レポート、リポジトリの監視、毎朝のエンジニアリングブリーフィング
  • 出力先は「このデスクトップ」だけでなく、メッセージング統合と組み合わせればSlack や Telegram に結果を送信可能
定期タスク＋メッセージングで
本物の自動化ハブに。
「毎朝レポートを Slack に」が一発！

⑩ Spawn Tree でサブエージェントを可視化

  • Agents ページには「Live Sub-Agent Activity」が表示される。現在のターンで稼働中の子エージェントの状態をリアルタイムで確認できる
  • メインエージェントがコーディング、テスト、リサーチ、レビューなどの作業を子エージェントに委譲したとき、その進捗がここに表示される
  • マルチエージェントのワークフローではブラックボックス化を防ぐ可視性が不可欠。Spawn Tree がその役割を果たす
  • 親子関係のツリー構造で表示されるため、どのタスクが誰に委譲されたかが一目で分かる
エージェントがタスクを委譲するとき、
「何が起きているか見えない」のは怖い。
Spawn Tree で透明性を確保！

⑪ モデルピッカー — プロバイダーロックインなし

  • モデルピッカーはプロバイダー別にグループ化されている。Anthropic / ローカル Llama / Nous Portal / OpenAI Codex / OpenRouter など
  • 軽いタスクには無料や安価なモデル、難しいタスクには強力なクラウドモデルという使い分けが可能
  • ローカルモデルも選択肢に含まれる。プライバシーを重視するタスクや、ランニングコストを抑えたいシーンで活用
  • 一つのピッカーで全てを切り替え可能。これこそが Hermes 最大の強みだと動画内で強調されている

⑫ Hermes を支える 6 つの柱

  • Connect：Telegram、Discord、Slack、WhatsApp、Signal、Email、CLI など多数のプラットフォームに接続
  • Remember：永続的なメモリを持ち、プロジェクトを学習してスキルを自動生成する能力
  • Schedule：Cron と自動化。定期タスクを自然言語で設定し、任意の宛先に結果を配信
  • Delegate：サブエージェントと Spawn Tree によるマルチエージェントの委譲と可視化
  • Search：ウェブ検索、ブラウザ自動操作、ビジョン、画像生成、TTS、マルチモーダル推論に対応
  • Experiment：ローカル、Docker、SSH、Singularity、Modal など多様なバックエンドで分離されたサンドボックスを提供

⑭ 安全なサンドボックスでコードを実行

  • Modal などのクラウドバックエンドも利用可能で、ローカルリソースに縛られない実行ができる
  • 開発者にとって安全に試行錯誤できる環境は必須。Hermes はその点をしっかりカバーしている
