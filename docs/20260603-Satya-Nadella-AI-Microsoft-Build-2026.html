Satya Nadellaが語るAIの未来

No Priors × Latent Space クロスオーバー特別編 — Microsoft Build 2026

📅 2026年6月3日

① No Priors × Latent Space クロスオーバー

  • No PriorsとLatent SpaceがMicrosoft Build 2026会場から合同配信を実施
  • ホストのSarah GuoとElad GilがSatya NadellaにAI戦略を深掘り
  • 3時間にわたるキーノートの直後にインタビュー。Microsoft本社から生放送
  • Nadella「両方のポッドキャストをいつも聞いている」と喜びの声
AI業界をリードする2つのポッドキャストが合同でMicrosoftのCEOに直撃。業界の最前線を生々しく語る貴重な機会です。

② AIはエコシステムプレイ — 単一モデルではない

  • Nadellaの核心：AIは単一モデルでも単一プラットフォームでもない、エコシステムのゲーム
  • Microsoftで4つの主要プラットフォームシフトを経験した視点から分析
  • プラットフォームの定義：プラットフォームが生み出す価値が、プラットフォーム上で捉えられる価値を上回ること
  • 他社AIも使うが、自社がAIを作れる道筋を用意するのがMicrosoftの役割
「どんな企業も自社が作ったAIを指し示せる存在でありたい」— Nadellaのビジョン

③ MAIモデルの訓練戦略 — クリーンリネージ

  • MAIモデルは優れたリネージ（血統）から出発。高品質データで事前学習
  • 多くのオープンモデルはベンチマークでは優れるが実務で振るわない問題を指摘
  • 系統的な検証（ablation）を重ねて堅牢なモデルを構築するアプローチ
  • Mustafa Suleymanが語った「great lineage」の概念がモデル品質の要
オープンモデルの弱点を正面から受け止め、クリーンなデータ系統から堅牢なモデルを作るMicrosoftの哲学が光ります。

④ 丘陵登りスキャフォールドで専門家モデルを構築

  • Hill Climbing Scaffold：ベースモデルの周りに登り用の足場を組み立てる概念
  • 汎用モデルをベースに各企業が独自の専門家モデルを作れる仕組み
  • 5B（50億パラメータ）の小さな推論モデルが丘陵登りで驚くべき性能を発揮
  • 「認知のコア（Cognitive Core）」を見つけることが究極の目標
小さなモデルでも足場さえあれば高みに登れる。だから5BモデルがFD（基礎モデルエンジニア）たちに愛されているのです。

⑤ プライベート評価が企業の最大のIPに

  • 公開ベンチマークはすべてマックス可能。真の評価は企業固有のプライベートEval
  • 各企業が独自の評価セットを持ち、モデルの価値を独自に測る必要がある
  • エージェントのトレース、コンテキスト、評価をループさせて改善
  • 「プライベートEvalこそが最大の知的財産かもしれない」とNadellaが強調
ベンチマークハックの時代は終わり。企業独自のEvalがAI競争の真の差別化要因になります。

⑥ GitHub Copilotの進化 — コーディングを超えて

  • GitHub Copilotはコーディングで圧倒的な成功を収めたが、今は他領域にも拡大
  • エージェントの管理、セキュリティ、フロントエンドなど複数のエンジニアリング分野に対応
  • 今後は「エージェントを管理する人」が雇用の主軸に。FTE（フルタイム相当）ではなく
  • コーディングエージェントのレバレッジが、他の仕事にも波及する効果を実証
コーディングの成功は序章。次はエージェントが自律的に動く世界が到来します。

⑦ フルスタックビルダーの台頭

  • フルスタックビルダーという新しい役割が登場。1人で全工程を担当
  • 設計、プロダクト、バックエンド、フロントエンドなど分断されたワークフローが統合
  • 従来は専門家の連携が必要だった工程を1人がAIの力でカバー
  • Microsoft内でMike Rendalが書いたエッセイがこの概念の起点
1人でゼロからアプリを構築できる世界。かつては想像もできなかったことが現実になっています。

⑧ ハーネスがエージェント戦略の要

  • ハーネス（Harness）の選択がエージェント戦略で最も重要な決定
  • ハーネス＝モデルを訓練する枠組み。ツール＋トレース＋Evalを統合
  • GitHubハーネス（オープン）と、M365ハーネス（エンタープライズ）など状況に応じて使い分け
  • モデルAとモデルBを切り替えて結果を比較。マルチモデルで最適な組み合わせを追求
ハーネスが良ければどんなモデルでも丘陵登りできる。これがMicrosoftのプラットフォーム戦略の核です。

⑨ マルチモデルハーネスの現実

  • 実運用ではすべてがマルチモデルハーネス。1つのモデルだけに依存しない
  • GPT-4.1やMAIモデルなど、目的に応じて最適なモデルをルーティング
  • オープンハーネスでLlamaなどのサードパーティモデルも組み込み可能
  • 「to rule them all（1つですべてを支配する）」ではなく、状況に応じた使い分け
特定のモデル崇拝はしない。最適な道具を最適な場面で使うのが正しいアプローチです。

⑩ 企業のAI適用 — 誰もが一等参加者に

  • AIネイティブ企業も伝統的企業も「一等参加者」としてAIエコシステムに参加できる
  • 「どうやるか？ 何が必要か？ 何が価値なのか？」という明確な道筋がMicrosoftの使命
  • Lando Lakesのデモ：社内データを基にAIが自律的に作業計画を作成
  • エージェントがバグや脆弱性を発見し、修正計画を立案。セキュリティ分野での実証例
AIを作る企業だけでなく使う企業も、エコシステムの主人公になれる。これが真の民主化です。

⑪ 独立系フロンティア研究所との協業

  • 独立系フロンティア研究所の存在が極めて重要。OpenAIとの提携がその証明
  • Microsoftはハイパースケーラーとしてインフラを提供し、研究所は研究に集中
  • グリッドが良くなればエネルギー消費も効率化。より良い計算インフラがAIの進展を加速
  • 「限界費用で何かを取得するなら、その価値はプラットフォームの価値になる」という経済論
OpenAIとの提携は投資ではなく、エコシステム構築の戦略的判断。その哲学が他のパートナーにも適用されています。

⑫ Work IQ — Microsoft 365におけるAIの実力

  • Work IQ：M365の組織のコンテキストを理解するAIエージェント
  • メール着信を監視し、優先度を判断して担当者に振り分ける自律型エージェント
  • Teams、SharePoint、PowerPointなどの企業データを横断的に活用
  • 「誰がエージェントを管理するのか？」— その人たちが次世代の重要な役割に
エージェントが私の「チーフ・オブ・スタッフ」になる世界。メールを管理し、計画を立てて実行までしてくれる。

⑬ データセンターとインフラの莫大な投資

  • Microsoftは過去最大規模のデータセンター投資を実施中
  • エネルギーと水の消費は懸念されるが、計算効率の向上で相殺を狙う
  • グリッドの改善、ファイバー事業者との連携でインフラ全体を最適化
  • 「Jensenとのパートナーシップ」— NVIDIAとの2年にわたる協業が実を結んでいる
AIのインフラ投資は単なるコストではなく、未来への投資。計算効率の改善が持続可能な成長の鍵です。

⑭ AIがもたらす雇用と教育の変革

  • AI時代の新しい役割が次々と創出されている。「アイデア人（Idea Person）」の時代
  • 汎用能力（Generalist）のレバレッジが上がり、野心とエージェンシー（主体性）がより重要に
  • 教育の再考が必要。Alpha Schoolの創業者たちからのインスピレーション
  • 資格やスキルの価値が変化。トークンを使って価値を生み出せる能力が新しい指標に
「これがCEOたちと投資家に音楽的な話」— Nadella。雇用が減るのではなく、新しい形の雇用が生まれています。

⑮ SaaSビジネスモデルの大転換

  • 従来のユーザーごとの課金モデル（per-user pricing）が根本から問い直されている
  • 新たな成果ベースの課金（outcome-based pricing）とトークンベースの課金が台頭
  • GitHubで最近パーユーザー課金を発表したが、これは最初のバンドリングに過ぎない
  • 「最終的には4つの課金モデルが共存する」とNadellaが予測
SaaSのビジネスモデルは根本的に変わる。消費ベース、成果ベース、トークンベースが混在する新しい世界が到来します。

⑯ トークンの会計処理と知的財産

  • トークンの会計基準（accounting standards for tokens）が新たな課題として浮上
  • AIが生成した成果物の知的財産（IP）は誰に帰属するのかが議論の的
  • 企業が自社のデータでモデルを訓練した場合、その改良分の権利は企業にあるべき
  • SEC（証券取引委員会）もトークンの会計処理について新たな枠組みを検討する必要性
トークンは単なる消費ではなく、企業のバランスシートに載る価値ある資産。その会計処理が次の大きな課題です。

⑰ 社会的影響 — AIの広範な恩恵

  • AIの広範な恩恵（broad benefit）が経済成長と生産性向上を駆動する
  • 医療分野でAIがより良い健康 outcomes を実現。建設中・建設後の安全管理にも活用
  • 給付金や手続きなど、市民サービスの効率化で社会全体が恩恵を受ける
  • 「AIが富を創造するか、それとも富を分配するか」— 富の分配メカニズムが鍵
技術の進展だけでは不十分。社会全体が恩恵を受けられる仕組みを構築することが不可欠です。

⑱ 今後の展望 — 次の12〜18ヶ月

  • 次の12〜18ヶ月でAIエージェントの採用が爆発的に加速するとNadellaが予測
  • エージェントのトレースが最大の推進力。企業が実績を重ねて波及効果が拡大
  • 「世界は懐疑的だが、結果が証明すれば話は変わる」という確信
  • Microsoftの目標：すべての企業がAIを自社の成果物として指し示せる世界の実現
「6ヶ月後には全員が『wow』と言っているだろう」— Nadellaの強い自信。エージェントの台頭は避けられない未来です。
