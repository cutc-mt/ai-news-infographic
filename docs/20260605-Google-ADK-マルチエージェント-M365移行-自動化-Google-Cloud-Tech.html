M365からGoogle Workspaceへの移行を自動化する実践ガイド

①エンタープライズ移行の課題: 手作業の苦痛

  • Microsoft 365からGoogle Workspaceへの移行評価は企業にとって大きな負担
  • ドキュメントを読んで機能マッピングを整理するのに数日かかる
  • ライセンスとアドオンを一つ一つ手作業で比較する純粋な肉体労働
  • マッピング、最新の変更、クリーンな出力の整形...すべて手動
数日かかる退屈な作業...これをAIに任せたい！

②動画の主題: ADKで信頼性のあるエージェント構築

  • AIエージェントの構築は巨大で脆いプロンプトとの格闘になりがち
  • 出力結果が予測不能で、運用に耐えないことが多い
  • Google Agent Development Kitで信頼性の高いエージェントを構築可能
  • 実際のワークフロー自動化のデモを通じて解説
巨大プロンプトに別れを告げよう！

③エージェント構築の3原則: 地面のルール

  • 状態管理（State Management）— データの流れを明確に制御
  • ルーティング（Routing）— 適切な処理に振り分ける仕組み
  • ガードレール（Guardrails）— 危険な出力や誤動作を防止

これら3つが信頼性のあるエージェントの核となる要素。

APIを叩くだけじゃダメ！設計が大事！

④デモ概要: ライセンス移行マッピングエージェント

  • Microsoft 365のライセンスをGoogle Workspaceの同等機能にマッピング
  • 結果として10チャプターのレポートを出力
  • ADKのオーケストレーションパイプラインで自動化
数日の作業が数分に！魔法みたい！

⑤パイプライン入力: テキストまたは画像

  • テキスト — ソフトウェアライセンスの一覧を直接入力
  • 画像 — ライセンス一覧のスクリーンショットも対応
  • 入力後にリサーチフェーズが自動で実行される
  • 最終的に構造化されたデータとして出力される
スクショを投げるだけでOK！

⑥マルチエージェント: 巨大プロンプトの終焉

  • 1つの巨大プロンプトではなく複数の特化型エージェントに分割
  • ADKのSequential Agentで直列に接続
  • ワークロードを focused なサブエージェントに分解
  • それぞれが独立したシステムプロンプトを持つ
分担作業で efficiency up！

⑦入力パーサー: ライセンスデータの抽出

  • テキストや画像からライセンス情報を正確に抽出
  • ノイズを除去して構造化フォーマットに変換
  • 次のエージェントに渡せる形にデータを整形
ゴミを取って綺麗にしてくれる！

⑧リサーチャー: ドキュメント検索と照合

  • 抽出されたライセンスに基づいて社内ドキュメントを検索
  • Microsoft 365とGoogle Workspaceの機能を比較
  • 正確なマッピング情報を収集して次に渡す
ドキュメントを読んで比較...地味だけど重要！

⑨レポーター: 10チャプターのレポート生成

  • リサーチ結果をもとに整形された10チャプターのレポートを作成
  • 章立てされた構造で読みやすい出力を実現
  • エンドユーザーに意味のある情報として提示
プロのレポーターが書いたみたい！

⑩ハルシネーション対策: 特化で削減

  • 各エージェントが非常に具体的なタスクだけを担当
  • 独立したシステムプロンプトで役割が明確
  • もっともらしい嘘を大幅に削減
  • デバッグも格段にやりやすくなる
嘘が減る！開発者も嬉しい！

⑪自己修正サイクル: Loop Agentの活用

  • ワンショットで完璧なレポートを出すのはほぼ不可能
  • Loop AgentでMapperとReviewerをループ構成に
  • レビューアーが機能の欠落やフォーマット崩れをチェック
  • 問題があればMapperに戻して修正。完了まで繰り返す
自己修復ループ！これが本物のエージェント！

⑫レビューアー: 品質チェックとフィードバック

  • 欠落した機能がないかを確認
  • 不要な絵文字やフォーマットの崩れを検出
  • 問題箇所を特定してMapperにフィードバック
  • 品質基準を満たすまで修正を繰り返す
絵文字が混ざってたら修正される！

⑬データソース: Search＋ナレッジベース

  • Google Searchをリアルタイムのグラウンディングツールとして活用
  • ローカルに保存されたエンタープライズ機能マトリックスのナレッジベース
  • 2つのソースを組み合わせることで正確性を担保
リアルタイム＋自社データの最強コンボ！

⑭コンテキストウィンドウの課題: 巨大CSV問題

  • 大量のローカルCSVを毎回プロンプトに注入すると大問題
  • コンテキストウィンドウを消費してしまう
  • APIコストが急激に上昇する
  • 本番運用では許容できない問題
お財布に優しくない...！

⑮コンテキストキャッシュ: ADKの built-in 解決策

  • ナレッジベースを1回だけロードしてキャッシュ
  • ADKがTTL（Time To Live）とトークン制限を自動管理
  • レイテンシを削減して毎回の実行コストを節約
  • 開発者は何もしなくても自動で最適化される
1回ロードすればOK！速い！安い！

⑯ロギングと可観測性: プラグインで簡単

  • ロギングプラグインを追加するだけでプロンプトとレスポンスのデータをストリーミング
  • ロギングシンクへ直接データを送信可能
  • 本番環境での可観測性（Observability）をサポート
  • 後から問題の追跡やデバッグが容易になる
本番でも安心！何が起きたか全部見える！

⑰本番へのデプロイ: Agent Engineで1コマンド

  • ローカルIDEから本番環境への移行が超簡単
  • カスタムDockerファイルを書く必要がない
  • 複雑なホスティングインフラの構築も不要
  • Google Agent Engineへのデプロイは literally 1コマンド
1コマンドでデプロイ！夢みたい！

⑱HTTP 429対策: 優先推論（Priority Inference）

  • HTTP 429リソース枯渇エラーは構築中の最大の学び
  • たった2つのHTTPヘッダーを追加するだけの解決策
  • 優先推論（Priority Inference）を利用可能に
  • モデルエンドポイントで保証された優先スロットを取得
2つのヘッダー追加だけで解決！
知っておくと超便利！
