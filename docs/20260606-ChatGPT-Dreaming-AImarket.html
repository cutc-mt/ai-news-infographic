2026年6月6日

① 今日話す3つのポイント

  • Dreamingとは何か — 従来のSaved Memoriesを完全刷新した新メモリシステム
  • どんなシーンで便利か — 旅行・仕事・趣味…実用例で確認
  • 注意点と使い方 — プライバシー対策と完全削除の方法
2026年6月4日発表。ChatGPT Plus・Proユーザーから先行展開中！

② 「Dreaming」とは何か

  • 会話していない時間に、過去の全履歴を背景で自動整理・更新するシステム
  • 従来の「覚えておいて」指示が不要に。AIが自主的に記憶を合成・管理
  • 計算リソースを約5倍効率化。将来の無料ユーザー展開に向けた布石
  • 2024年4月 → 2025年4月 → 2026年6月のV3へ。2年間の進化の集大成

③ 2年間の記憶機能の進化

  • V1（2024年4月） 手動メモ。自分で「覚えて」と言う必要あり
  • V2（2025年4月） 自動抽出開始。会話中に重要情報をピックアップ
  • V3（2026年6月） Dreaming登場。非アクティブ時にも記憶を整理・統合
毎回ゼロからやり直す時代は
もう終わったんだよ！

④ 従来版との決定的な違い

  • 旧システム 手動管理。ユーザーが明示的に記憶を追加・削除
  • Dreaming 自動合成＋ユーザー編集のハイブリッド
  • AIがバックグラウンドで記憶を整理するから、ユーザーは何もしなくていい
  • 必要な時だけMemory Summaryで手動修正も可能

⑤ 公式事例：水中写真

  • 水中写真の機材相談があったユーザーのケース
  • 過去に使ったカメラ・レンズ・ライトのセットアップを自動で記憶
  • 次回の相談で「再説明不要」でピンポイント提案が来る
  • 「前回と同じ構成で、今回は暗い場所向けに」— これが一発で通じる
趣味の機材も全部覚えてくれる。

⑥ 公式事例：旅行計画

  • 旅行前：計画段階では予算・日程・好みを自動反映して提案
  • 旅行中：「今ここにいる」を優先してリアルタイム情報を提供
  • 旅行後：写真整理や感想のまとめまでサポート
  • 時期に応じて回答モードが自動切替されるのがポイント

⑦ 公式事例：テイクアウト提案

  • 過去の食事情報から好みを把握している状態
  • でも提案は「今・ここ」を優先。古い記憶より現在地ベース
  • 記憶と現在の文脈をうまくバランス。過剰な記憶依存を避ける設計
  • 「毎回辣いのが好きって言わなくていい」← これが地味にすごい

⑧ 相性抜群：長期プロジェクト

  • ダイエット 食事記録・運動履歴・目標体重を跨いで追跡
  • 副業 プロジェクト進捗・クライアント情報・やることリストを自動管理
  • 数ヶ月単位の継続がDreamingの真価

⑨ Memory Summaryページ

  • アクセス：設定 → Personalization → Memory
  • ChatGPTがあなたについて記憶している情報を一覧表示
  • バラバラの会話履歴から「意味のある情報」に自動整理済み
  • ここで確認・修正・削除がすべてできる

⑩ Memory Summaryでできる4つのこと

  • 追加 新しい情報を手動で追加
  • 更新 既存の記憶情報を修正
  • 深掘り 特定トピックについてチャットで詳しく聞く
  • 強制更新 AIに記憶の再整理を促す
「自動でやってくれる」けど
「自分でも触れる」。完璧なバランス！

⑪ Custom Instructionsとの違い

  • Custom Instructions ユーザーが書いた固定プロンプト
  • Dreaming 会話から自動抽出する別物の機能
  • 両方は併用可能。Dreamingが会話ベース、CIがルールベース
  • 「絶対に省略形で答えて」「担当はエンジニア」はCI。「好きな食べ物」はDreaming

⑫ プライバシー：Temporary Chat

  • デリケートな話は「Temporary Chat」で
  • 会話終了後に内容が一切残らない一時モード
  • Dreamingの記憶にも保存されない安全な会話
  • 機密情報・個人情報を扱う時は必ずこれを使う癖をつけよう
夢見るAIも見ちゃダメな話はある。
これ知っとくと安心！

⑬ 完全削除：4カ所すべて必要

  • Memory Summary → 記憶情報を個別に削除
  • チャット履歴 → 該当会話を削除
  • ファイル → アップロードしたファイルを削除
  • 接続アプリ → 第三者アプリのデータ連携も解除
4カ所全部やらないと完全に消えない。
これは超重要！

⑭ 削除しても30日間保持される

  • 削除したメモリはデバッグ目的で最大30日保持される場合あり
  • OpenAIの内部システム上に一時的に残存する可能性
  • 完全に消したいなら、削除後にしばらく利用を控えるのが最安全
  • サービス改善 vs プライバシーのトレードオフ。透明性は評価すべき
