【神アプデ】ChatGPT のメモリが進化！
『Dreaming』とは何かを解説

箇条書きの3つの弱点が消える理由
📅 2026年6月6日 | 出典: まさおAIじっくり解説ch

①今回のテーマ: ChatGPT「Dreaming」

  • OpenAI がChatGPT の新しい記憶機能「Dreaming」を公開した
  • 箇条書き型のSaved Memoriesが、裏で自動合成される文章プロフィールへと進化
  • 情報が古くなる・似た事実と取り違える・毎回覚えさせる、という3つの弱点がすべて消える
「Dreaming」＝ AI が夢を見るように
記憶を裏で整理して再合成する仕組み

②ChatGPT メモリのこれまで

  • 2024年4月: Saved Memoriesが初登場。「覚えておいて」と頼めば記憶する仕組み
  • 会話中に書き込まれる箇条書きのメモで、強いきっかけがないと記憶されなかった
  • メモに書かれていないことは忘れる＝「メモを取る人間」と同じ問題
  • 時間が経つと情報が古くなったり、無関係になったりする問題もあった
「メモは取ったけど内容が古くなってる…」
ユーザー自身でメンテが必要だった

③覚書から文章プロフィール型へ

  • これまで: 「ユーザーは猫を2匹飼っている」など箇条書き
  • これから: 複数の会話の文脈を裏で合成して文章形式のプロフィールを自動生成
  • AI がユーザーの好み・制約・文脈を裏で自動的に学習して事実精度を改善
  • 「覚えて」と言わなくても、自然な会話から情報を吸い上げる
もう「覚えておいて」って
言わなくていいのね！

④裏で自動学習して精度が改善

  • Dreaming はバックグラウンドプロセスとして動作
  • 過去の多くの会話から学習し、最新のコンテキストを常に提供
  • 明示的な「覚えて」がなくても、自然な発言から文脈を記憶
  • 推論コストが大幅に削減され、Free ユーザーにも提供可能になった
裏で AI が一生懸命
あなたのプロフィールを書き上げている！

⑤文章型のメリットと箇条書きの欠点

  • 箇条書きの欠点①: 情報が古くなる（旅行が「行く予定」のまま残る）
  • 箇条書きの欠点②: 似た事実と取り違える（前回の旅行と今回の混同）
  • 箇条書きの欠点③: 毎回覚えさせる必要がある（合図がないと記憶されない）
  • 文章型は時系列・関連性・ニュアンスを自然に保持
箇条書きは「点」の記憶だけど
文章は「線」の記憶！

⑥忘却ではなく「再合成」という発想

  • 従来のアプローチ: 古い記憶を忘却曲線で削除する手法が主流だった
  • Dreaming の発想: 古い記憶を捨てずに再合成する
  • 「7月にシンガポールに行く」→ 旅行後は自動的に「行った」に更新
  • まさお氏も汎用エージェントの長期メモリを自作しようとしており感銘を受けた
捨てるんじゃなくて
「書き直す」のか！理にかなってる

⑦CLAUDE.md に例える Dreaming

  • CLAUDE.mdはユーザーの指示や好みを記述したテキストファイル
  • Dreaming は似た発想: 文章プロフィールを裏で作成・更新
  • 違いはユーザーが手動で書かなくても AI が自動生成する点
  • Memory Summary ページで合成された記憶の要約を確認・編集できる
CLAUDE.md の AI 自動生成版
みたいなイメージで合ってるかな？

⑧公式提供と自作という選択肢

  • OpenAI 公式: Plus・Pro ユーザー（米国）から本日提供開始
  • 今後数週間で Free・Go ユーザーと追加の国々にも展開予定
  • 自作派: 汎用エージェント向け独自の長期メモリを実装する選択肢
  • 公式に乗るか自作するかは用途とコントロール要件次第
あなたは公式派？それとも
自作派？コメント欄で教えて！

⑨3つの弱点が消える理由

  • 情報の鮮度: 時間経過に応じて自動更新。「行く予定」→「行った」に変更
  • 事実の正確性: 複数会話から合成するので似た事実が混ざらない
  • スケーラビリティ: 明示的な「覚えて」がなくても自然な会話から文脈を吸い上げる
  • 再合成によって一度記憶した情報を書き換えることができるのがポイント
忘却ではなく再合成
この発想の転換がすごい！

⑩ベンチマークと精度の向上

  • OpenAI は3つの評価軸でメモリの性能を測定している
  • ①コンテキストの伝達: 過去の情報を新しい会話で正しく再現できるか
  • ②好みの追従: ベジタリアンならベジタリアン向けの提案をするか
  • ③時刻の正確性: 「今はシンガポール」が旅行後に解除されるか
  • Dreaming V3 は全評価軸で大幅にスコアが向上している
2024年 Saved Memories → 2025年 V0 → 2026年 V3
段階的に進化してきたんだね

⑪計算コストの大幅低減

  • 最新の改善で Dreaming の計算コストが約5分の1に削減された
  • これによりFree ユーザーへの提供が可能に
  • Plus・Pro ユーザーのメモリ容量も増加
  • 数億ユーザー・数年スパンのスケーラビリティに対応できる基盤が完成
コスト5分の1！無料ユーザーも
高度なメモリ体験ができるようになった

⑫プライバシー制御と提供範囲

  • Memory Summary でAI が知っている自分の情報を確認できる
  • 情報の追加・更新・削除や、話題の制御指示が可能
  • 「Stan の話を持ち出さないで」などのネガティブな指示も反映される
  • 段階的な展開: 米国 → 追加の国 → Free・Go ユーザー
何を覚えられてるか確認できるのは
安心だね！自分でコントロールできる

⑬実用例: カメラ機材の提案

  • 過去にカメラのセットアップについて話した場合…
  • 「私の写真セットアップに互換性のある商品を教えて」と言うだけでOK
  • Dreaming は過去の会話を参照してあなたのカメラ機材に合った商品を提案する
  • ゼロから説明しなくていいので時間の節約になる
「私の写真セットアップに合うもの」
これだけで最適な提案が来るなんて！

⑭実用例: シンガポール旅行

  • 旅行2ヶ月前: 旅程の作成を依頼した時点で過去の旅行の好みを参照
  • 野生動物写真が好き・強力なエアコンのホテルを好む・静かな食事を好む…
  • 旅行後: 「シンガポールに行く予定」から「行った」に自動更新
  • 帰国後はホームの地域とタイムゾーンに基づいた提案に戻る
旅行が終わったら自動で
記憶が更新されるのが便利すぎる

⑮実用例: 食事の好み

  • 「私はベジタリアンです」と一度言えば記憶に残る
  • 後日「食事のアイデアを教えて」と頼むと、ベジタリアン向けの提案が自動で出る
  • 好みは明示的（ベジタリアン）と暗黙的（サンフランシスコ在住）の2種類
  • Dreaming はどちらの好みも適切に記憶・適用する
一度「ベジタリアン」と言えば
もう二度と肉料理を提案されない！
