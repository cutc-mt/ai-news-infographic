AIコーディングの質が激変する「Grill Me」

たった3行のプロンプトでClaude Codeが「詰め役」に変わる神スキルを徹底解説

② Claude Codeの現在地: 作れるけど判定できない

  • AIコーディングは今「ちゃんと上手にできているのか」の判定が難しい時代
  • 開発はできているし、コードも書けている。でも品質の良し悪しを評価する基準が曖昧
  • Claude Codeにはプランモードがあり、計画を立ててから実装する流れが一般的
  • プランを立てるスキルそのものが次の課題になりつつある
「もっと上手な人はいるかもしれないけど、その判定ができていない」まさに今のAI開発の現在地です。

③ Grill Meとは: AIが「詰め役」に変わるスキル

  • Matt Pocock が作成。TypeScript教育で有名な元Vercelエンジニア
  • GitHubで7万〜10万スターを獲得した大ヒットAgent Skill
  • 名前の由来は「問い詰める・尋問する」（英語のgrill）から
  • 2026年2月3日に公開され、5月には一大ブームに。SNSで話題騒然
わずか3行のプロンプトでAIの挙動がガラッと変わる。シンプルイズベスト

④ たった3行のプロンプトが全て

Interview me relentlessly about every
aspect of this plan until we reach a
shared understanding. Walk down each
branch of the design tree, resolving
dependencies between decisions one-
by-one. For each question, provide
your recommended answer. Ask the
questions one at a time. If a question
can be answered by exploring the
codebase, explore the codebase instead.
  • SKILL.mdにこの3行を書くだけで有効化。/grill-me で呼び出せる
  • Agent Skillは数十〜150行が普通だが、grill-meは異例の短さ
毎回長いプロンプトは不要。SKILL.mdに置けばツールキットに自動追加されます。

⑤ 使い方: /grill-meで壁打ちを開始

  • Claude Codeのプロンプトに/grill-me を付けてざっくりした要件を書くだけ
  • AIが質問＋2〜5つの選択肢を提示し、1つずつ答えていく形式
  • 設計ツリーの各分岐をたどり、決定事項間の依存関係を順番に解決
  • コードベースを調査すれば答えられる質問は、AIが自動でソースを読んで解決
「これくらいの質問、すぐ答えられるわ」と思っているとどんどん深掘りされていきます…！

⑥ 質問の深さ: 18〜24回の徹底的な詰め

  • 「その仕様で本当にいいのか？」「例外処理はどうする？」「依存関係は？」
  • 平均18回、最大24回にわたって徹底的に質問攻めにあう
  • 設計の分岐を枝の先まで一つひとつ辿り、判断の抜け漏れを潰す
  • 長いセッションだと約45分の対話になることも。終わる頃には認識が完全に揃う
「もうええでしょ！」と言いたくなるほどめちゃくちゃ聞いてきます

⑦ 推奨回答機能: AIが自分の意見も提示

  • 質問だけでなく、AIが推奨する回答も一緒に提示してくれる
  • 「聞かれるだけ」ではなく「前に進みやすい」形に最近アップデート
  • 推奨案を選ぶことも、自分で別案を出すことも自由
  • 人間が判断を下す主体性を保ちつつ、AIとの共通理解を迅速に形成
尋問されるだけじゃなくAIが「これがおすすめです」と提案してくれるのが心地よい！

⑧ プランモードとの決定的な違い

  • プランモード: AIが計画を立ててから実行。一部変更で計画全体の書き直しに
  • grill-me: 質問で認識を揃えてから実行。壁打ちセッションが実装計画に
  • プランモードの罠：「修正→計画全書き直し→修正→…」の無限ループ
  • grill-meは質問段階でズレが潰れるので、実装後の手戻りが格段に少ない
プランモード←→通常モードの切り替えより、grill-meはシームレスな体験です。

⑨ AI版ラバーダッキング: 思考を整理する革命

  • ラバーダッキング = 誰かに向かって考えを話して思考を整理する手法
  • 相手がAIなので遠慮がなく、抜け漏れも拾いやすい
  • 「まあ作りながら考えよう」という曖昧な開発を許してくれない
  • 1つずつ聞いてくるので、考えが強制的に整理される仕組み
人間相手だと気を遣うけど、AI相手なら全部ぶっちゃけられる。それが心地よい壁打ちになります。

⑩ ボトルネックは「書く速度」から「詰める速度」へ

  • AIでコードを書く速度は格段に上がったが、手戻りは減っていない
  • ボトルネックが「何を作るか」を詰める速度にシフトしている
  • 実装前にAIが質問攻めにしてくれるので、仕様の穴が事前に潰れる
  • 速くコードを書く人より、速く仕様を詰められる人が強い時代に
AIはもう「書く役」だけでは弱い。
先に問いを立てる相棒として使うのが正解です。

⑪ おすすめのモデル設定: Opus＋Medium effort

  • Opus 4.6（1Mコンテキスト） がgrill-meの最適解
  • effortはhighではなくmediumで十分。賢さと速度のバランスが良い
  • 100万コンテキストなら数十回の壁打ちでも余裕。ウィンドウ不足の心配なし
  • Sonnet 4.6でも大体問題ないが、時々深掘りが浅いと感じることがある
まずはOpusで始めるのがおすすめ。コンテキストが長いほど良い結果が出ます。

⑫ 注意点: とにかく疲れる

  • 何度かセッションを進めると、脳がめちゃくちゃジーンになる
  • 壁打ち開始から実装完了までの時間はかかる。45分の質問セッションもあり得る
  • でも壁打ち不足のまま計画→実装しても結局さらに修正することになる
  • トータルでかかる時間は同じくらいかも。45分の質問で手戻りが減るなら十分元が取れる
でも、それくらいよく考えないと良いものは作れないということですね。疲れるのは証拠

⑬ 効く場面: 要件定義からデータ設計まで

  • 要件定義: 曖昧な数行のアイデアからでも始められる
  • 画面設計: UIの構成要素や遷移をAIが深掘りして整理
  • データ設計: テーブル構造やリレーションを質問で詰められる
  • 権限設計・命名: 実装前の迷いをAIが体系的に解消
コーディング以外にも使える。
Matt Pocock本人が推奨する汎用スキルです。

⑭ Claude Code Skillsの仕組み

  • SKILL.md をプロジェクトに置くだけでツールキットに追加される
  • 関連性があればAIが自動で使用。/skill-name で直接呼び出しも可能
  • 毎回長いプロンプトを書かなくても、考え方そのものを再利用できる
  • 実装用・レビュー用・仕様深掘り用など、役割ごとのスキルを揃える時代へ
grill-meは単なる便利ネタではなく、AIとの付き合い方を一段進める事例です。
