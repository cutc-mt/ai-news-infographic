② コードベースの画像生成 — 全く新しい仕組み

  • 従来の画像生成はガチャ。Reve 2.0はまずコードを生成し、構造を定義してから画像を描く
  • 画像内の要素がレイヤーとして明確に分離される。バナナ、椅子、背景が個別に認識・選択可能
  • 裏側ではGeminiベースの推論が空間を把握し、レイアウトを生成してからレンダリング
  • LLMが計画して、ディフュージョンが描くという2段階構造

③ ゼロ劣化 — 選択部分だけ変える革命

  • 女の子のドレスだけを赤に変更しても、背景や他の部分は1ピクセルも変わらない
  • 従来のAIは編集するたびにノイズが蓄積して崩れていくが、Reve 2.0は裏側のレイアウトコードを書き換えて再レンダリング

⑫ Microsoft Image 2.5 vs Reve 2.0 — フィニッシャー対決

  • Reve 2.0は画像生成2位だが、編集部門は9位に下がる。プロンプトで角度が変わってしまう弱点

  • Reveは行動意識（コードベース）で生成すれば崩れない。プロンプト主体だと角度が変わりやすい
  • Arenaのスコアだけでは編集力は測れない。評価方法には限界がある

⑰ Reve 2.0の技術アーキテクチャ

  • 入力: テキストプロンプト、参照画像、行動指示
