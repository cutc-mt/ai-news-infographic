ADK入門: マルチエージェントシステム構築ガイド

Google Agent Development Kit でエージェント開発を始めよう

2026年6月8日 | Greg Baugues | 12分

ADKの基本を理解する

①

ADKとは何か？

  • ADK = Agent Development Kit の略称
  • Googleが提供するエージェント構築フレームワーク
  • ボイラープレートコードを削減し、開発をシンプルにする
  • LLMにツールを装備し、ループで実行する仕組みを提供
  • 開発中のテスト用インターフェースも内蔵
エージェント開発の基本をGoogle公式フレームワークで学べる！
②

エージェントとは？

  • エージェント = LLM + ツール + ループ
  • タスクが完了するまでループで自動実行
  • ツールを追加することでLLMの能力を拡張
  • ADKがタスク完了の判定と停止を自動管理
LLM
+ ツール
+ ループ
エージェント
LLM単体ではできないことを、ツールとループで実現できる！
③

開発環境のセットアップ

  • AI StudioでGemini APIのキーを取得
  • Pythonパッケージのインストール
  • 仮想環境での作業を推奨 
# セットアップ手順
# 1. AI StudioでAPIキーを取得
# 2. 仮想環境を作成
python -m venv .venv
source .venv/bin/activate

# 3. ADKをインストール
pip install google-adk
APIキーとADKパッケージがあればすぐ始められる！
④

最初のエージェントを作成する

  • コマンド1つでエージェントの雛形を生成
  • モデルはGemini 3 Flash Previewを選択可能
  • 最小限のコードでエージェントを定義 
# エージェントを作成
adk create first_agent
モデル・名前・説明・指示の4つのパラメータだけでOK！

エージェントのコード構造

⑤

エージェントのコード構造

  • google.adk.agentsをインポート
  • 4つのパラメータでエージェントを定義
  • model / name / description / instructions 
from google.adk.agents import Agent

agent = Agent(
    model="gemini-3-flash-preview",
    name="getting_started",
    description="初めてのエージェント",
    instructions="You are an assistant
        who is helpful but brief."
)
⑥

CLI でエージェントをテスト

  • adk runコマンドでローカルテスト
  • 対話形式でエージェントとチャット可能
  • すぐにレスポンスを確認できる 
# ターミナルで実行
adk run first_agent

# 対話例:
> Hello. How are you today?
> I'm doing well! How can I help you?
コマンドラインから手軽に動作確認！
⑦

Web インターフェースでデバッグ

  • adk webでブラウザベースのテスト環境を起動
  • ローカルサーバーが 127.0.0.1:8000 で立ち上がる
  • スタックトレースを確認できるのが最大の魅力
  • エージェント構築時のデバッグに非常に有用 
# Web インターフェースを起動
adk web

# ブラウザでアクセス:
# http://127.0.0.1:8000
内部処理の流れが可視化されて超便利！
⑧

ツールの基本概念

  • LLMの学習データにはカットオフ日が存在する
  • 「今何時？」「今日の日付は？」に正確に答えられない
  • ツールを追加すると最新情報にアクセス可能
  • ツールによりLLMの能力をほぼ無限に拡張できる
LLM単体では「現在時刻」すら分からない。ツールが必須！

カスタムツールと組み込みツール

⑨

カスタムツールの作成

  • Python関数としてツールを定義するだけ
  • 例: get_current_timeで現在時刻を取得
  • 定義した関数をエージェントのtoolsリストに追加 
from datetime import datetime

def get_current_time():
    """現在の日時を返す"""
    return datetime.now()

agent = Agent(
    tools=[get_current_time],
    ...
)
普通のPython関数がそのままツールになる！
⑩

ツールの組み込み方

  • エージェント定義のtoolsパラメータにリストで渡す
  • リスト形式なので複数のツールを追加可能
  • Time Keeperエージェントの実例を確認 
# toolsパラメータにリストで渡す
agent = Agent(
    model="gemini-3-flash-preview",
    name="time_keeper",
    description="時刻を知っているエージェント",
    instructions="...",
    tools=[get_current_time]  # ← ここ！
)
⑪

組み込み Google Search ツール

  • Google Search ToolはADKに組み込み済み
  • インポートするだけでそのまま使える
  • リアルタイムのWeb検索結果を取得
  • bypass_multiple_tools_limitで複数検索を許可 
from google.adk.tools import google_search

agent = Agent(
    tools=[google_search],
    # 複数回検索を許可
    bypass_multiple_tools_limit=True
)
総合的な結果を出すには複数回検索が必要だからこの設定が大事！
⑫

検索結果の課題と対策

  • 検索ツールを使うとレスポンスに時間がかかる
  • 返されたソースURLの一部が404エラーになる
  • LLMはハルシネーションでURLを捏造することがある
  • 検索結果の検証メカニズムが必要
検索はしているのに、URLが存在しないことがあるのが問題！

マルチエージェントシステムの構築

⑬

URL検証エージェント

  • fetch_urlツールでWebページの内容を取得
  • 検証結果は3つのステータスで返す
  • いわゆる「LLM as a Judge」パターン

検証
  • verified = 検証済み
  • not verified = 未検証
  • inconclusive = 判定不能
⑭

LLM as a Judge パターン

  • あるLLMの出力を別のLLMで検証する仕組み
  • 検索エージェントの結果をURL検証エージェントで確認
  • 情報の正確性を担保できる

⑮

リサーチャーエージェント（統括）

  • オーケストレーターとして機能するエージェント
  • ツールとして他のエージェントを装備
  • 検索エージェントで情報を収集
  • 検証エージェントで結果を確認
Researcher
Search
Verify
エージェントのツールにエージェントを渡すのがポイント！
⑯

マルチエージェントの実行フロー

  • ユーザーが質問を入力
  • ResearcherがSearch Agentで検索を実行
  • ResearcherがURL Verifierで検索結果を検証
  • URLが実在し、内容が正確かを確認
  • 検証済みの結果だけをユーザーに返す
複数のツール呼び出しが自動で連携して動く！

マルチエージェントのメリットとまとめ

⑰

マルチエージェントのメリット

  • 各エージェントに単一の役割を割り当てて管理しやすい
  • タスクに応じて異なるモデルを使い分け可能
  • 簡単なタスクには安価なモデル、複雑なタスクには高性能モデルでコスト最適化
  • パフォーマンス・メモリ・コストを個別に管理
  • ハルシネーションを減らしデータの正確性を向上
1つの巨大エージェントより、複数の専門エージェントの方が効率的！
