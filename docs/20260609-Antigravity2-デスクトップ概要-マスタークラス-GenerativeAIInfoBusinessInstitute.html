アーキテクチャと全体構成

①アーキテクチャの全体像

  • Electronベースのクロスプラットフォームアプリ
  • Windows、macOS、Linuxの全OSに対応
  • 4層構造でモジュール化された設計
  • Geminiモデルと通信するエージェントアーキテクチャ
Electron採用により、どのOSでも同じ体験が可能です！

②4層アーキテクチャの詳細

  • UI Layer：React/TypeScriptのチャット・エディタ・ファイルツリー
  • Agent Layer：Shared Agent Harness経由でGeminiと通信
  • Tool Layer：ファイルシステム・ターミナル・ブラウザへのアクセス
  • Storage Layer：プロジェクト設定・会話履歴・Rules/Skillsのキャッシュ
各レイヤーが独立しているので、拡張性が高い設計になっています

③UIレイヤーの技術スタック

  • フロントエンドはReactで構築
  • TypeScriptによる型安全な開発
  • チャットインターフェースがメインの対話手段
  • ファイルツリーでプロジェクトの構造を可視化 
// 技術スタック
UI: React + TypeScript
Runtime: Electron
AI: Gemini Model
Storage: Local File System

④エージェントレイヤーとGemini連携

  • Shared Agent Harnessを介してモデルと通信
  • GoogleのGeminiモデルを利用
  • 自然言語で指示してアプリを自動生成
  • Tool Layer経由でローカル環境にアクセス
Geminiモデルの力で、プログラミング知識なしでもアプリが作れます！
メインインターフェースの構成

⑤画面の3パネル構成

  • 左パネル：セッション履歴・プロジェクト管理・設定
  • 中央：メインのチャットウィンドウ
  • 右パネル：ファイル変更・アーティファクト・エージェント情報
左右のパネルはトグルスイッチで開閉できます。両方閉じるとチャットに集中できます！

⑥左サイドパネルの機能一覧

  • New Session：新しいチャットセッションを開始
  • Session History：過去のチャット履歴を確認
  • スケジュールタスクの管理も可能
  • Projects：作成済みのプロジェクトに即座にアクセス
  • Settings：歯車アイコンから設定画面を開く
プロジェクトごとに会話履歴が保存されるので、とても便利です！

⑦チャットウィンドウ：操作の中心

  • メインの操作はすべてチャットで完結
  • 自然言語でメッセージを送信してタスクを実行
  • 左下のプラスボタンでファイルを追加可能
  • アプリの作成や修正を自然言語で指示
基本はチャットにメッセージを送るだけ。分からないことがあれば何でも聞けます！

⑧右サイドパネルの機能一覧

  • Overview：プロジェクトの概要を表示
  • File Changes：作成・変更されたファイルを確認
  • Artifacts：タスクリストやエージェント情報を表示
  • ソースコードの閲覧も可能（編集はIDEで）
IDEのような本格的な編集機能はないので、コードを直接いじりたい場合はAntigravity IDEを使いましょう
プロジェクト管理

⑨プロジェクトの作成方法

  • 左パネルのNew Projectボタンをクリック
  • 既存のローカルフォルダーを選択してプロジェクトに設定
  • 新しくディレクトリーを作成することも可能
  • プロジェクトごとにフォルダーを紐付け
あらかじめパソコンにフォルダーを作っておいて、それを選ぶのが一番簡単です！

⑩プロジェクトごとの管理機能

  • ローカルファイルシステムのディレクトリーへのリンク
  • プロジェクト固有のRulesとSkills
  • エージェントとの会話履歴がプロジェクトごとに保存
  • 権限設定やセキュリティ設定も個別に管理
各プロジェクトの設定と履歴が独立しているので、整理整頓がしやすいです

⑪ファイル変更の確認方法

  • バイブコーディングで作成されたファイルが一覧表示
  • 変更が加えられたファイルもここに表示
  • ファイルをクリックすると内容をプレビュー
  • プレビューは閉じることも可能
自然言語で指示してどんなファイルができたか、すぐに確認できます！

⑫アーティファクトとタスク管理

  • Taskタブでエージェントの作業内容を確認
  • 指示した内容がチェックマークで進捗表示
  • AgentタブでエージェントのJSON情報を確認
  • エージェントがどのように動いているか可視化
タスクのチェックマークを見れば、何が終わって何が残っているか一目瞭然です！
エージェント起動モードとコマンド

⑬ローカルモード：基本の作業方式

  • 現在のワーキングディレクトリーに直接変更を加える
  • プロジェクトフォルダーを指定して作業を開始
  • 基本的なモードとして推奨される方式
  • アプリやウェブサイトの作成を直接実行
基本はこのローカルモードで十分。フォルダーを作ってプロジェクトを始めるのが一番簡単です！

⑭ワークツリーモード：安全な実験環境

  • GitのWorktree機能を活用した独立ワークスペース
  • メインブランチを汚さずに実験的な変更が可能
  • プロジェクトがGitリポジトリであることが前提
  • 大胆な変更も安心して試せる環境
前提条件：プロジェクトがGitリポジトリとして管理されている必要があります

⑮2つのモードの比較

ローカルモード ワーキングディレクトリーに直接変更
ワークツリー 独立したワークスペースで変更
ローカル 前提条件なし。すぐに開始可能
ワークツリー Gitリポジトリが必要
ワークツリー メインブランチを汚さない利点
Base44やClaude Codeで作ったプロジェクトをGitHub管理している場合は、ワークツリーモードが便利です

⑯スラッシュコマンドの使い方

  • チャットウィンドウで/を入力してコマンド呼び出し
  • /goal：高レベルのゴールを設定するモード
  • /grillme：要件の明確化のための質問を行うモード
  • アップデートでコマンドが追加・削除されることも
/planなどのコマンドは既に内部処理に統合されて、スラッシュコマンドからは消えている場合もあります
IDE連携とバイブコーディング

⑰Antigravity IDEとの連携

  • 右パネルにAntigravity IDEの起動ボタンがある
  • デスクトップアプリとIDEをシームレスに切り替え可能
  • ソースコードを直接編集したい場合はIDEを活用
  • 未インストール時は「Install IDE」ボタンが表示
デスクトップは「見る」中心、IDEは「編集する」中心。用途に合わせて使い分けましょう！

⑱バイブコーディングで開発を進める

  • 基本は自然言語でチャットしながら開発
  • 「こういうアプリを作って」「ここを修正して」と指示
  • ファイルの作成や変更は右パネルで確認
  • プログラミングの知識がなくてもアプリ開発が可能
分からないことがあればチャットで聞けばOK。とにかく試して進めるのがコツです！

⑲対応プラットフォームとインストール

  • Windowsに対応
  • macOSに対応
  • Linuxに対応
  • Electronベースでクロスプラットフォーム対応
現在のUIは英語メインですが、操作自体はシンプルなので日本語でチャットすれば大丈夫です！

⑳マスタークラス第2章のまとめ

  • 画面は左・中央・右の3パネル構成
  • プロジェクトごとにフォルダーを紐付けて管理
  • ローカルとワークツリーの2つの起動モード
  • スラッシュコマンドで /goal と /grillme が利用可能
  • 基本はチャットで自然言語によるバイブコーディング
まずはプロジェクトを作って、チャットで何でも聞いてみるのが始め方のコツです！
