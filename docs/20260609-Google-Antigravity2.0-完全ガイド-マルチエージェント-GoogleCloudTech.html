Google Antigravity 2.0 完全開発者ガイド

①Antigravity 2.0とは: IDEからエージェントファーストへ

Google Antigravity 2.0は、従来の単一IDEから4つの独立コンポーネントへ進化したエージェントファーストプラットフォームです。I/O 2026で発表された大型アップデートにより、開発者は自分のワークフローに合わせて最適なツールを選択できるようになりました。

  • Agent Manager デスクトップでのエージェント管理
  • CLI ターミナルから高速に操作
  • IDE コードレビューと編集に特化
  • SDK Pythonで独自ワークフロー構築
すべて別々にダウンロード可能で、必要なものだけインストールできるのがポイント！

②Rody Davis: Googleのトップエージェントエンジニア

Rody Davisは、GitHubの米国トップ100 OSSコントリビューターの一人であり、18のプログラミング言語を操るスーパーエンジニアです。Project IDXからFirebase Studio、そしてAntigravity 2.0の立ち上げに大きく貢献してきました。

  • 18言語でコーディング可能（AIなしでも！）
  • GitHub US Top 100 OSSコントリビューター
  • Project IDX → Firebase Studio → Antigravity 2.0
  • AIの活用で「認知の負担」を減らすことに注力
AIは学習プロセスも構築も研究も加速させる。プロトタイプを何度も作り直せるから、最適なフレームワークにたどり着きやすい！

③スキルとは: エージェントのための「カンニングペーパー」

Antigravityにおけるスキルは、モデル向けにコンテキストを圧縮した「カンニングペーパー」です。テストに例えると、教科書を開きっぱなしにするよりも、1枚にまとめたメモの方が圧倒的に速く答えを見つけられるのと同じ仕組みです。

  • コンテキストを圧縮してモデルに提供
  • 再利用可能なスクリプトやワークフローを定義
  • デザインシステムを丸ごと格納可能
  • ワンコマンドでトリガー可能なタスク化
オープンブックテストは遅い。でも、1ページにまとめたメモなら超高速。スキルはまさにこれ！

Antigravity 2.0では、MCPサーバーやカスタムスキルでエージェントの能力を拡張できます。RodyはFlutter/Dart用のMCPサーバーを常時有効化し、ホットリロードやDevTools連携を実現しています。

  • Android CLI コマンドラインからAndroid開発
  • Chrome Extensions Web開発のモダン化
  • DevTools 開発者ツール統合
  • Modern Web Guidance モダンCSS手法のガイド
  • MCPサーバーでホットリロード＆DevTools連携

⑤盆栽アプローチ: コードアーキテクチャの哲学

Rodyはコーディングを「盆栽アーティスト」に例えています。不要なものを常に剪定し、シンプルで美しい構造を目指すのです。シンプルは簡単ではなく、むしろ高い技術力が必要だと強調しています。

  • フラットなアーキテクチャを採用
  • 状態・UI・データを明確に分離
  • エージェントが逸脱したらすぐに気づける構造
  • 複数フレームワーク・言語で共通パターンを適用
エージェントがフォルダを間違えたら、すぐに軌道修正できる。これが良いアーキテクチャの力！

⑥コードレビューの戦略: 100%レビューするべきか？

コードレビューについては、領域によってレビューの深度を変えるのが正解だとRodyは答えています。マーケティングサイトのHTML/CSSは見た目重視、バックエンドのGoコードはAPI契約を厳密にレビューします。

  • マーケティングサイト → 視覚的にチェック。コード自体は気にしない
  • バックエンド（Go） → API契約・インターフェースを厳密にレビュー
  • 最初のパターンを自分で書き、エージェントに拡張させる
  • テストが通れば、実装の詳細は柔軟に許容
APIのスキーマ変更は影響が大きい。間違えても「ごめん、戻すね」では済まない。だからこそ契約は厳重に！

⑦認知の負担を減らす: 拡張機能の構築

Rodyは「認知の負担（Cognitive Toil）」を減らすために、Antigravityで様々なツールを構築しています。Finder用の時間ベースタグ付け拡張やChrome拡張機能は、その代表例です。

  • Finder拡張: タイムボックスでファイルを整理
  • 「今日」「昨日」「今週」「今月」でフィルター
  • 同じ機能をChrome拡張としても再構築
  • Swiftで完璧に構築 — プロンプトだけで生成
複数のGitプロジェクトを跨ぐ作業で「先週どこを触ったっけ？」が解消。思考を設計とレビューに集中できる！

⑧AIエンジニアも手書きコードを書く理由

Rodyは「AIがあっても、自分でコードを書くのが大好き」と断言しています。それは、コンピュータの仕組みを深く理解することで、将来のプロジェクトに活かせるビルディングブロックが蓄積されるからです。

  • Go言語の文法がシンプルで理解しやすい
  • 古いハードウェアに新ソフトウェアを書く楽しさ
  • インターフェースや標準ライブラリを深く学ぶ
  • 研究段階の知識が2年後に役立つことがある
AIがコードを書いてくれるからこそ、人間は「なぜ動くのか」を深く理解する時間が取れる！

⑨Gemma 4でオフラインAI: 個人サイトの自動レコメンド

Rodyの個人サイトはGemma 4を活用して完全オフラインで動くレコメンドシステムを搭載しています。LM Studioでローカル実行し、コンパイル時にベクトル化して関連記事を自動生成する仕組みです。

  • 機内でもエージェントが使える（API不要！）
  • Gemma 4で記事を要約 → 埋め込みでベクトル化
  • コンパイル時に自動で「関連記事」を生成
  • サーバー不要。静的サイトで完結

2. Embedding Gemma → 要約をベクトル化
3. コンパイル時 → 関連記事を自動計算

⑩マルチエージェント実演: 音声プロンプトでアプリ構築

デモでは1つの音声プロンプトからマルチページ多言語アプリを構築しました。Vite + Go + SQLite + Docker Composeを使ったノートアプリを、サブエージェントたちが並列で構築しています。

  • 音声入力でプロンプトを指示（自然言語認識が優秀）
  • 5言語対応: 英語・ヒンディー語・スペイン語・中国語・日本語
  • DevOps・QAエンジニアのサブエージェントが自動起動
  • Docker Composeでサービス全体をオーケストレーション
  • Markdownプレビュー付きノートアプリが完成
音声入力がGeminiアプリの自然言語処理を使っているから、プログラミング用語も正確に認識！

⑪IDEのアンバンドリング: 4つの独立ツールへ分離

Antigravity 2.0の最大の変更点は、単一IDEから4つの独立コンポーネントへの分離です。これにより、非開発者にはAgent Managerだけを、CLI派にはCLIだけを提供できる柔軟性が生まれました。

  • Agent Manager 非開発者向けのデスクトップアプリ
  • CLI SSH接続先のサーバーにも簡単インストール
  • IDE コードレビューと編集に最適化
  • SDK Pythonで独自オーケストレーション構築
共有設定とプラグインは統一。でもチャット履歴は各ツールで独立しているから、コンテキストが混ざらない！

⑫マルチフォルダプロジェクト: 自由な構成を実現

以前は1ワークスペース＝1フォルダーでしたが、Antigravity 2.0では1つのプロジェクトに複数フォルダーを紐付け可能になりました。プロジェクトの構造を変えずに、そのままAntigravityに統合できます。

  • フロントエンド・バックエンド・ドキュメントを1プロジェクトに
  • 同じフォルダーを別プロジェクトにも参加可能
  • デザイン案件用・DBA案件用など文脈で切り替え
  • プロジェクトごとに独立した会話履歴を保持
Antigravityに合わせてプロジェクトを再構築する必要なし。既存の構成をそのまま使える！

⑬サブエージェントとGoalコマンド: 自動化の極み

Antigravity 2.0のサブエージェントはさらにサブエージェントを起動できるネスト構造に対応しています。また「goal」コマンドで、目標達成まで自動で反復するワークフローが構築できます。

  • サブエージェントがさらにサブエージェントを起動可能
  • /goal コマンドで目標達成まで自動反復
  • 例: 「テストカバレッジ100%まで自動で修正し続けて」
  • DevOps・QAエンジニアなどの「役割」をサブエージェントに付与
  • 「GrillMe」でプランに反論させて品質を向上
/* goalコマンドの例 */
/goal テストカバレッジ100%を達成せよ
→ テスト実行 → 失敗 → 修正 → 再実行...
→ 100%に到達するまで自動で繰り返す

⑭スキルの実践: ドキュメントを400個のスキルに変換

Rodyはウェブサイトのドキュメントを解析してMarkdownに変換するCLIツールを自作し、一度に400個ものスキルをインストールしました。特定バージョンのドキュメントにも対応できるのが強みです。

  • Webサイトをクロール → Markdown → スキル化
  • 400個のスキルを一括インストールした経験あり
  • 特定バージョンのドキュメントにピンポイントで対応
  • ネットワークレイテンシを回避して高速応答
  • Firebase、Flutter、デザインシステム用スキルも多数
トレーニングデータに入っていない特定バージョンのドキュメントでも、スキル化すれば正確に回答！

⑮使い分けガイド: CLI・Agent Manager・IDE・SDK

Rodyは4つのコンポーネントを用途に応じて使い分けています。Raspberry PiにはCLI、知識作業にはAgent Manager、コードレビューにはIDE、独自ワークフローにはSDKという具合です。

  • CLI → Raspberry Pi・SSH接続・サーバー環境
  • Agent Manager → API連携・Obsidian・Webクリップ
  • IDE → コードレビュー・変更・理解を深める作業
  • SDK → 特定ワークフローの自動オーケストレーション
SDKはAgent Manager自体にコードを書かせることも可能。非常に柔軟性が高い！

⑯ドキュメント更新スキル: コンテキストファイルを自動維持

Rodyが多用する実践的なスキルの1つが「Update Docs」スキルです。プロジェクトの各フォルダーにコンテキストファイルを作成し、常に最新の状態に自動更新します。

  • コンテキストファイルのないフォルダーを自動検出
  • ファイル更新より古いコンテキストを自動検知
  • スケジュールタスクで定期実行も可能
  • 常に最新のドキュメントを維持する仕組み
/* スキルのロジック */
1. コンテキストがないフォルダを検出
2. 変更より古いコンテキストを検出
3. 該当フォルダだけドキュメント更新
→ 大規模プロジェクトも常に文書が最新！

⑰ラピッドファイア予測: バイブコーディングの未来

Rodyが賛否を表明した7つの予測です。バイブコーディングの危険性から、今後のソフトウェアエンジニアの役割まで、多岐にわたる議論が展開されました。

  • 「従来のローカルIDEは終わる？」→ 「複雑だ」（プロジェクトによる）
  • 「非技術者がバイブコーディングだけで起業？」→ 「賛成」（2〜5年目が課題）
  • 「エージェントが本番直送に？」→ 「反対」（動的アプリの方向へ）
  • 「バイブコーディングで壊滅的障害？」→ 「賛成」（サイバー保安が更重要に）
  • 「最大のボトルネックはコードベースの健康状態」→ 「強く同意」
次のホットな仕事は「バイブコーディングアプリの障害解決コンサル」になる！
