既存スキル運用の4つの致命的弱点

④ 弱点①: Create-Use Mismatch

  • 作る時と使う時がずれてる問題
  • 例: CSVを処理するスキルを作る時は「1個の綺麗なファイル」を想定
  • 実際は文字化けや列名がバラバラの現実のデータに直面
  • 設計者の前提とエージェントが触る現実が大きく乖離
現場のデータって絶対に綺麗じゃないんですよね…これあるあるです。

⑤ 弱点②: スキル単位の記憶がない

  • スキルごとに教訓を残す場所が今までなかった
  • あるスキルで失敗した経験を、そのスキル自身が覚えておく仕組みがない
  • エージェント全体のメモリはあっても、スキル固有のメモリが存在しない
  • 同じ失敗を何度も繰り返してしまう構造的な欠陥
エージェント全体の記憶はあっても、スキル別の教訓場所がなかったのが問題！

⑥ 弱点③: 静的で未検証なスキル

  • 1回作ったら誰もチェックしない状態
  • ソフトウェア開発ならユニットテストは当然なのに…
  • AIエージェントのスキルはテストも動作確認も魔法なしが普通
  • 壊れていても分からないまま本番投入 → 大事故の元
NPMにも欲しい仕組みかも…テストなしで本番投入なんて恐ろしい！

⑦ 弱点④: コンテキスト履歴の破綻

  • 長時間タスクでエージェントの会話履歴が無限に伸びて破綻
  • コンテキストウィンドウからこぼれ落ちる問題
  • 古い出力を雑に捨てると肝心な情報まで消えてミスを連発
  • 頭の中の整理術がないと、長時間のタスクは事実上不可能
古い履歴を雑に捨てると大事な情報も消える…ジレンマなんです。
5段階ライフサイクル

⑧ 5段階ライフサイクルの全体像

1 Creation 生成 - 必要な瞬間にその場で作る
2 Memory 記憶 - 三層構造で学びを蓄積
3 Management 管理 - マージ・更新・プルーニング
4 Evaluation 評価 - ユニットテストで品質担保
5 Refinement 改善 - 失敗から自動でパッチ適用
生まれて → 覚えて → 整理されて → テストされて → 改善される。
植物の成長サイクルみたいですね！

⑨ Stage 1&2: 生成と記憶

  • オンデマンド合成: 事前に揃えるのではなく、必要な時にその場で生成
  • 三層メモリ構造がMUSEの肝
  • 短期記憶: 今のタスクの作業メモリ
  • 長期記憶: セッションをまたぐ全体の学び
  • スキル別記憶: 各スキル専用のメモ帳（新概念！）
人間のワーキングメモリと長期記憶に近いけど、
「スキル別」があるのが新しい！

⑩ Stage 3: スキル管理（Management）

  • マージ: 似たスキルを統合して重複を排除
  • 更新: 内容を最新の状態に保つ
  • プルーニング: 使わなくなったスキルは忘れる（刈り取り）
  • スキル数が増えると邪魔になるので、不要なものは剪定する仕組み
盆栽の剪定みたい！不要な枝を切って美しく保つんです。

⑪ Stage 4&5: 評価と改善

  • Evaluation: 新スキルはユニットテストに合格しないとスキルバンクに登録されない
  • 倉庫には信頼できるスキルしか並ばない設計
  • Refinement: Refinerエージェントが失敗の原因を見て自動パッチ適用
  • Create-Evaluate-Refine ループが品質を担保
エージェント自身がCIを回してるようなもの！
直して再テストして合格したら晴れてスキルバンク入り。
最大の発明: スキルレベルメモリー

⑫ 最大の発明: スキルレベルメモリー

  • 各スキルに専用のメモ帳 .memory.md を持たせる
  • SKL.mdの隣に memory.md を並べるだけのシンプル構造
  • 主に3つを記録: 失敗モード / 入力フォーマットの癖 / パフォーマンス注意点
  • タスクをまたいで蓄積され、別の案件の学びが次の案件で生きる

  パースに失敗する

  除外が必要

## 入力の癖

スキルが自分の取り扱い説明書を成長させていくんです。

⑬ 適的コンテキスト圧縮（DAG）

  • DAG（有向非巡回グラフ）で会話履歴を構造化管理
  • 会話の核ターンをノードで表し、線でつなぐ（マーメイド図のイメージ）
  • レベル1圧縮: ノードが多くなりすぎたら要約で置き換え
  • レベル2圧縮: 連続する複数ノードをまとめる
  • 元の履歴はフルで保存なのでいつでも巻き戻し可能
雑に古いのを捨てるんじゃなくて、段階的に丁寧に圧縮！
忘れるのではなく賢くまとめる発想です。

⑭ ReActループで自律動作

  • 全体のエージェントはReActループで動作
  • 考えて → 動いて → 観察するの繰り返し
  • 合間にスキルを取りに行ったり、作ったり、テストしたり
  • スキルの作成・評価・改善もループの中で自動で完結
Think → Act → Observe のサイクルの中で
スキルも自然に育っていく仕組みです。
