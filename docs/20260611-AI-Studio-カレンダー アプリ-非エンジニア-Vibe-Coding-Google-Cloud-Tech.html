  • プレゼンターは開発経験ゼロのビジネスパーソン
  • 毎日コンピューターで仕事をしている
  • 仕事の大部分が旅行の手配と管理
  • 人生で一度もコードを書いたことがない

プログラマーじゃない自分には

②最大のペインポイント: 旅行日程のカレンダー化

  • 複数都市・複数国にまたがる旅程のカレンダー登録
  • 国際日付変更線をまたぐと30分以上かかる
  • ホテル情報、フライト情報、確認番号を手入力
  • 最も嫌いな仕事タスクの筆頭
ニューヨーク→香港の赤の目便
乗り継ぎありの超複雑旅程を
毎回手作業で変換...地獄。

③Vibe Codingへの第一歩: AI Studioにアクセス

  • アクセス先: ai.studio.google.com
  • 「AIに話しかければプログラムを書いてくれる」と聞いた
  • 実際にそれが正解だった
  • プログラミング知識は一切不要
バイブコーディングの本質:
「AIに喋りかけるとアプリができる」
これだけ覚えればOK！

④最初のプロンプト: 自然言語でアプリをリクエスト

  • 「旅程のコピペテキストをカレンダー招待に変換するアプリを作って」
  • ホテル情報・フライト情報・確認番号を含める
  • Buildボタンをクリックするだけ
  • コーヒーを淹れる時間で最初のバージョンが完成
奇跡かエラーか？
正直、どちらでも驚かなかった...

⑤初版の失敗: 日付変更線を理解できない

  • 最初の invite.ics は大きな問題を抱えていた
  • 赤の目便（深夜便）の概念を理解できない
  • 国際日付変更線が全く処理されていない
  • ニューヨーク→香港を1時間と判定
  • 乗り継ぎ（Layover）の概念も混乱
1日が経過するはずのフライトが
たった1時間になってた...
がっかり度MAX。

⑥改善アプローチ: 「マネージャーに相談」するように改善指示

  • 出力に不満→そのままAIにフィードバック
  • 「日付変更線を理解して」「フライトの所要時間を計算して」
  • 「ニューヨークから香港が1時間じゃないことはわかるはず」
  • 現実世界のマネージャーに
    不満を言うのと同じ感覚で機能した
バイブコーディングの醍醐味:
フィードバックループが直感的！
コードを書かずに改善できる。

⑦モデル切り替え: Gemini 3.1 Pro に自動アップグレード

  • AI Studio がGemini 3.1 Proへ自動切り替え
  • Google Cloud 課金アカウントで設定可能
  • 高度な推論能力を獲得
  • 日付変更線・タイムゾーン認識を搭載
  • ユニバーサルタイムゾーン変換対応
Gemini 3.1 Pro の強み:
推論→時差認識→現地時刻表示
