「Chat is Dead」— OpenAIが宣言

ChatGPTはただのチャットボットから「スーパーアプリ」へ大変貌。IPO前夜の最大の賭けとは？
2026年6月11日

① 「チャットは終わった」— FT報道が衝撃を走らせる

  • Financial Timesが報じたOpenAI内部の爆弾発言。「Chat is dead」と宣言する上級エンジニアの声
  • 2022年のChatGPT登場から始まったAIブームの象徴が、「会話」を終わりにするという歴史的転換点
  • 変更は数週間以内にウェブサイトとモバイルアプリで順次ロールアウト開始の予定
  • OpenAIのエンジニア曰く：「チャットは単なる入り口にすぎない。未来は自律的なアクションだ」
「Chat is Dead」＝
チャットの死。AIのインターフェースが
対話から行動へ根本的に変わる。

② スーパーアプリの中身 — 3つの製品が統合

  • ChatGPT — 従来のチャットボット機能（残りますが、入り口の役割に）
  • Codex — AIコーディングプラットフォーム。500万の週間アクティブユーザーを抱える
  • Atlasブラウザ — エージェント型ブラウジング機能。ChatGPTのブラウジング体験を統合
  • サードパーティ連携も統合。CanvaやBooking.comなどのパートナーサービスを直接操作可能に
ツールを切り替えなくても
コーディングも検索もデザインも
1つの画面で完結する未来。

③ AIエージェントが自律的に動く — プロンプト不要の未来

  • 旅行の予約、カレンダー管理、リサーチなど複数ステップのワークフローを自律的に実行
  • ユーザーが細かい指示を出さなくても、AIが何をすべきかを自動理解して行動する設計
  • OpenAIのCROであるデニーズ・ドレッサー：「従業員が物事を成し遂げる統一されたAIスーパーアプリ」
  • ソティヤックス（元Codex責任者）：「画面を超越して、人生のすべてを支えるパーソナルエージェントに」
「何をしてほしい？」ではなく
「何が必要か」をAIが察知する。
インターフェースの進化の完成形。

④ Codexが主役に — コーディングから企業OSへ

  • Codexは立ち上げから6倍に成長。週間500万ユーザーがアクティブに利用中
  • Sites — スプレッドシートやドキュメントから即座にホスト済みのWebアプリを生成。フロントエンド開発不要
  • 役割別プラグイン — Snowflake、Figma、Salesforceなど62のビジネスアプリと110の自動スキルを接続
  • Annotations — ドキュメント全体を再生成せず特定のセルやブロックだけを精密に編集
Codexのユーザーの20%は
非開発者。金融アナリストや
マーケターがエンジニアより
3倍速く導入している！

⑥ IPOへのカウントダウン — 賭けの背景

  • OpenAIは2026年末に上場を計画。Goldman SachsとMorgan Stanleyがアドバイザー
  • 現在の評価額は$8520億（3月の資金調達時）。上場時には$1兆を超える可能性
  • 莫大なコンピュートコストでまだ赤字。投資家に「持続可能な収益」を証明する必要がある
  • 無料ユーザーを有料の企業ユーザーに転換するのがIPO成功の鍵
IPOに向けて「未来の
あり方」を投資家に見せる。
スーパーアプリはその答え。

⑦ Soraを閉鎖 — 「横道に逸れるな」の号令

  • Sora — OpenAIのAI動画生成プラットフォーム。3月に全面閉鎖を決定
  • コンシューマー向けアプリ、開発者API、ChatGPT内の動画機能のすべてを終了
  • アプリ責任者のフィジ・シモが「リソースを薄く広げすぎてスピードと品質が低下した」と社内で警告
  • 浮いたコンピュートリソースをCodex開発とスーパーアプリに全振りする判断
Soraは「サイドクエスト」
と切り捨てられた。
企業に集中しろという経営陣の意思表示。

⑧ OpenAI vs Anthropic — IPO前の激突

  • 評価額：OpenAI $8520億 vs Anthropic $9650億。Anthropicが逆転
  • 年間収益：OpenAI 約$240億 vs Anthropic $470億（年率）。2倍近い差
  • Anthropicは6月1日に機密S-1をSECに提出。OpenAIも5月末にIPO書類を提出済み
  • Anthropicは「Claude Code」で開発者市場を席巻。Claudeが自社コードの90%以上を記述
1年前はOpenAIが実験的挑戦、
Anthropicが収益重視。
今は両者が同じ方向へ疾走中。

⑨ 企業市場での実力比較 — Claudeが先行

  • 職場でAIが不可欠または生産性を大きく向上：Claude 81% vs ChatGPT 71%
  • Anthropicは最初から企業ファースト戦略。Claude CodeとCoworkでワークプレイスを開拓
  • OpenAIは消費者成長を優先してきたが、今は急速に企業向けへシフト
  • スーパーアアプリ戦略は、この企業市場での遅れを取り戻すための起死回生策
Codexの週間500万ユーザーを
スーパーアプリに誘導して
企業シェアを奪い返す作戦。

⑩ 新しいUIの姿 — ユーザー体験はどう変わる

  • ウェブサイトとモバイルアプリのデザインを全面改修。従来のチャット画面がメインではなくなる
  • 新しいプロンプトとナビゲーション要素がユーザーをCodex・画像生成・パートナーサービスへ誘導
  • 最終目標は手動プロンプトの完全撤廃。モデルがユーザーの意図を自動で理解
  • ChatGPTを開くだけで「何が必要か」をAIが察知してタスクを自動的に開始
あなたがOpenAIのアプリを開く理由は
「質問するため」から「任せるため」に変わる。

⑪ 経営陣の刷新 — Codex責任者が全体を統括

  • ティボー・ソティヤックスがOpenAIのコア製品・プラットフォーム全体を統括するリーダーに就任
  • ChatGPTチームとCodexチームが統合されて一本化。組織の重複を排除
  • 元CPOのケビン・ワイルなど複数の上級幹部が退任。組織の大規模リストラクチャリング
  • コンピュートリソースの無駄をなくし、Codexとスーパーアプリに集中する体制へ移行
Codexの責任者がChatGPT全体を
統括する＝コーディングが主役の
サイン。チャットの時代は終わった。

⑫ パートナーシップ戦略 — サードパーティ統合

  • Canva — デザイン作業をChatGPT内で直接。プロンプトからビジュアル制作まで
  • Booking.com — 旅行の検索・予約をAIが自律的に実行。ユーザーは結果を確認するだけ
  • 62のビジネスアプリ（Snowflake、Figma、Salesforceなど）と110の自動スキルを標準接続
  • すべてのSaaSアプリがバックエンドサービスになり、ChatGPTがフロントエンドになる構想
OpenAIの狙いはオーケストレーション層。
個別のツールより「繋ぐもの」の
方が価値があるという判断。

⑬ 非開発者の波 — 「バイブコーディング」の台頭

  • Codexユーザーの20%が非開発者。金融アナリスト、マーケター、オペレーション担当者
  • 非開発者の導入スピードはエンジニアの3倍。自然言語でアプリが作れる「バイブコーディング」が普及
  • スプレッドシートやドキュメントを渡すだけで、即座にWebアプリを生成するSites機能がキラーアプリ
  • アドレス可能な市場が開発者から全ビジネスパーソンへ拡大。市場規模が劇的に変化
「コードを書けない」から
「AIに書いてもらう」へ。
プログラミングの民主化が本格化。

⑭ リスクと課題 — 成功の条件

  • Forresterの分析：欧米の消費者は専門アプリを好む傾向。スーパーアプリが広がるかは未知数
  • ウェブ閲覧、メール閲読、ファイル管理、購入代行をAIに任せるプライバシーの懸念
  • 10億の無料ユーザーの何%がついてくるか。無料AIツールに流れるリスクもある
  • 「Chat is Dead」の宣言が時期尚早になる可能性。プロンプト不要の未来はまだ数年先
やりすぎて本来の良さを

⑮ 開発者への影響 — CursorやReplitに軋み

  • APIレイヤーは残る。GPT-4o、Codex API、Assistants APIは引き続き利用可能
  • ただしOpenAIの注目と価格決定力は統合スーパーアプリに向かう。APIへの投資優先度が下がる恐れ
  • Cursor、Windsurf、ReplitなどのスタンドアロンAIコーディングツールが苦戦の構図
  • オープンプラットフォームと囲い込み型スーパーアプリの間の緊張が次のフェーズを決める
コミュニティフォーラムでも
価格上昇とAPI軽視を
