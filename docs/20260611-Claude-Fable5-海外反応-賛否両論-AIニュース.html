Claude Fable 5 海外の反応まとめ

賛否両論！オーバースペック論！コストや安全装置批判論も！
📅 2026年6月11日

①AnthropicがFable 5を発表: ついにMythosクラスのAIが一般公開に

  • 2026年6月9日にClaude Fable 5がリリースされた
  • Mythos 5の安全性を高めた一般向けバージョンという位置づけ
  • 内部レッドチームと外部バグバウンティでテスト済み
  • ユニバーサルな脱獄（ジェイルブレイク）は発見されていないと主張
Mythosは「神話」、Fableは「寓話」という意味。
同じモデルを安全性で2つに分けた構造は今までにないアプローチ！

②「まさに化け物」: 初日の最強の評価

  • Simon Willison氏が「something of a beast（まさに化け物）」と評価
  • 投げたタスクを全てこなす力強さに驚きの声
  • 自身のオープンソースプロジェクトを正確にリスト化するテストに合格
  • Opus 4.8よりも多くのプロジェクトを識別して「モデルの規模感の proxies」として評価
「投げつけたものを全部ハッピーサイクルでこなしてる」
こんな評価、新モデルの発表ではめったに聞けないよ…！

③「Next Level」: WordPress開発者も衝撃

  • AutomatticのJamie Marsland氏がスクリーンショット1枚からWordPressテーマを1発生成
  • ネイティブWordPressパターンに完全対応のブロックテーマを作成
  • 「完全に編集可能」な状態で出力されたと証言
  • Search Engine Journalにも取り上げられ、話題に
スクショ1枚＋URLだけでテーマ完成とは…
フロントエンド開発者にとっては本当にNext Levelだね！

④開発者が実際に作っているもの: その具体例

  • Datasette Agentプロジェクトの機能開発でLLMライブラリのバグ4つを自動発見・修正
  • API設計・テスト・コード・ドキュメントすべてを高品質で出力
  • MicroPythonをWASMにコンパイル — 数ヶ月悩んでいた課題を数分で解決
  • DBマイグレーション開発者はメモリ割り当てを46倍削減と報告
「数日分の作業」が1午後に圧縮。
これが実際に現場で起きていることなんだ。

⑤Hacker Newsの声: CRDT研究・ファジングまで自動生成

  • CRDT研究者が正しい不変条件を導出し、検証ファザーをほぼ手放しで記述
  • 競合モデルでは見逃していたバグを発見する例が多数報告
  • 「これまではなかった感覚で仕事を奪われそう」という衝撃の声も
  • 難しいタスクこそがFable 5の真価を発揮する場面だと実証
HNのコメントからは「反応が良いのは難しいタスクだけ」という
はっきりとしたパターンが見えてくるね

⑥SWE-Bench Pro 80.3%: コーディング性能の圧倒的リード

  • SWE-Bench Proで80.3%を達成 — 前代未到のスコア
  • 主要モデルとの比較:
Fable 5: 80.3%
Opus 4.8: 69.2%
GPT-5.5: 58.6%
Gemini 3.1 Pro: 54.2%
2位のGPT-5.5に対して21.7ポイントの差。
コーディング界の王者は明確にFable 5だ！

⑦よりターゲットな差分・無駄のない変更: Clanker Cloudの検証

  • 「よりターゲットな差分」「不要な変更が少ない」と具体的に評価
  • 人間の介入なしでも保守性の高いコードを出力
  • 「通常の会話ではOpus 4.8と劇的な差はないが、難しいフロントエンドとエージェントコーディングでは明らかに上」
  • 「コードは書けるがプロダクションの文脈はまだ必要」との注意点も
つまり「使う場所を間違えなければ最強」
この一文がFable 5の本質を言い当ててる！

⑧コストショック: 1日で$110超えの事例も

  • Simon Willison氏がテスト1日で$110.42を消費 — 月額$100枠を24時間で超過
  • HNユーザーはコードレビュー1回でサブエージェントに$92を浪費と報告
  • 最高推論設定では5時間枠が8分で消費されるケースも
  • 価格: 入力$10/Mトークン 出力$50/Mトークン
月額$100のプランを1日で突破…
一般開発者にはかなり厳しい現実だね

⑨コストの正しい読み方: トークン単価ではなくタスク単価で比較

  • タスク単位で見ると1発正解で失敗のコストをゼロにできるので逆に安い場合も
  • ルーティンワークに使うとSonnetの10倍のコストになる
  • コンセンサス: 難しい高ステークスな仕事には正当
  • 大量の定型タスクにはOpus 4.8やSonnetで十分
「トークン単価」ではなく「完成したタスク単価」で
判断するのがFable 5時代の正しいコスト感覚！

⑩安全装置のフォールバック問題: 正規の仕事がブロックされる事態

  • サイバー・生物・化学系のリクエストを弱いOpus 4.8に自動フォールバック
  • Anthropicは「セッションの5%未満で発動」と説明
  • 実際には医療画像・ラボ自動化・ヘルスデータ分析・音楽ファームウェアまでブロック
  • 新しいセッションを開始すれば同じリクエストが通る — 分類器の過敏さが露呈
医療画像分析が「バイオセキュリティ」として弾かれる…
開発者のイライラは完全に理解できる！

⑪Nathan Lambertの鋭い批判: 「Safety Fables」問題

  • 「通知なしで自動的に劣化するAIはカテゴリ的にミスアライン」と痛烈に批判
  • サイバー・バイオは透明なフィルターだがAI研究・蒸留への制限は通知されない
  • 「安全性」より競争的地位の維持が目的だと主張
  • 「世界最高のAIモデルを自分の専門分野で信頼できない」と結論
「見えない劣化」こそが一番怖い。
フルパワーのモデルが何を黙って制限されてるかわからない状態…

⑫サイバー能力の真実: MythosクラスとFableのギャップ

  • Fable 5はMythos 5の同一モデルの安全制限版
  • ExploitBenchでMythosクラスが78.0%に対しOpus 4.8は40.0%
  • 偽の大学課題ルーブリック作成でセキュリティ回避の詳細な手順を出力してしまう事例
  • Anthropicは事後に謝罪し、AI開発制限の透明性を改善と表明
BeInCryptoは「世界最強のハッキングAI」と表現。
防御者にとっては強力な味方だが…1層の安全装置だけという不安も。

⑬3つの陣営: 24時間後の海外反応を分類

  • 楽観派（多数）: 長時間・エージェント・難しいタスクで本物の進歩を体感
  • 不満派（一部）: 過敏な安全分類器にブロック、または高額請求にショック
  • 懐疑派: 厳密な対照実験なしでは評価できないと主張
  • 「バイブス（勢い）で語るレビューは証拠にならない」vs「実務家の感覚も正当な判断材料」
どの陣営も「ワークロード次第」では一致。
Fable 5の立ち位置はすでに明確になりつつある。

⑭幻覚と「無能」モード: 予期せぬ弱点の報告

  • 一部ユーザーが「完全に無能」「絶え間ない幻覚」と報告
  • 間違いを指摘すると「上から目線で防御的」な態度を取るとの声
  • モデルは改善中だが「盲目的に信頼してはいけない」という注意が複数のレビューに共通
  • 「最高のパフォーマンス」と「最悪の失敗」の両極端が存在する不思議
タスクによって「化け物」にも「ポンコツ」にもなる。
この両極端さこそがFable 5の最大の特徴かも…！

⑮企業導入の速さ: 発表当日から主要プラットフォームで利用可能

  • Amazon Bedrockで即座に利用可能に
  • Microsoft Foundryにも同日対応
  • GitHub Copilotで一般利用可能に
  • 法務AI企業Harveyも即日サポートを発表
発表当日にAWS・Azure・GitHub全対応。
プラットフォーム側は事前にテスト済みってことだね！

⑯企業のコスト感覚: エンジニア1時間＞数ドルのトークン

  • エンジニア1時間のコストがトークン費用を圧倒するためコスト問題は enterprises では minor
  • 「数日の作業」を1午後にするモデルなら安いと判断
  • 個人開発者向けには6月22日まで無料、以降はクレジット購入制に移行
  • 「AI格差」を懸念する声も — 有料化で一部の開発者が取り残される懸念
企業は「安いか高いか」ではなく
「エンジニア時間を節約できるか」で判断する。この違いが大きい。

⑰Fable 5を使う際の実践チェックリスト

  • ① 一番難しいタスクに振る — 軽いチャットでは「高いOpus 4.8」に見えるだけ
  • ② トークン単価ではなくタスク完了単価で測る
  • ③ フォールバック通知を監視 — 自分のドメインがブロックされる頻度を記録
  • ④ 難しい10%にFable、残り90%は安いモデルというルーティングが最適解
「Fable 5を全面移行するモデルではなく、
意識的に手に取るモデル」というのが
海外開発者の最大の共通認識！

⑱24時間後の総括: 条件付き熱狂こそが最高の賛辞

  • 「ワークロード次第」が全レビューの共通結論
  • 熱狂は条件付きだからこそ本物 — 無批判な称賛より価値がある
  • 難しいタスクの10%にFable、残り90%はOpus 4.8/Sonnetが最適
  • 「全面的に切り替えるモデルではない、意識的に届くモデル」
条件付き熱狂 = 信頼できる評価。
全面的に最高とは言えないが、
特定の場面では「今までにない何か」を感じさせる。
それがClaude Fable 5の24時間後の結論だ。
