Claude新モデル「Fable 5」登場！

Dynamic WorkflowsなどClaude Codeの最新機能を徹底解説
2026年6月13日

① Claude Fable 5 がついに登場

  • 2026年6月9日 AnthropicがClaude Fable 5を一般リリース
  • Anthropicがこれまでに公開した中で最も高性能なモデルとして位置づけ
  • ソフトウェアエンジニアリングや知識作業で大幅な性能向上を実現
  • 視覚機能を搭載し、スクリーンショットから目標を確認できる
Mythos-classの仲間！ただし安全フィルター付き。ポケモンの画面からクリアまで実演したという噂も。

② Fable 5 の「Mythos-class」是什么か

  • Fable 5はAnthropicの「Mythos-class」に分類される新世代モデル
  • 姉妹モデルにClaude Mythos 5が存在。サイバーセキュリティ保護が解除されている
  • Mythos 5はProject Glasswingのパートナー限定でアクセスが制限
  • Fable 5には安全分類器が組み込まれ、特定の要求はOpus 4.8にルーティングされる
サイバー・バイオ・化学・蒸留系のリクエストは、能力の低いOpus 4.8に振り分けられます。

③ Fable 5 の料金と利用可能プラットフォーム

  • 入力$10 / 1Mトークン・出力$50 / 1Mトークン
  • 対応プラットフォーム: Claude API、AWS、Amazon Bedrock、Vertex AI、Microsoft Foundry
  • 6月22日まではPro・Max・Team・Enterpriseで追加費用なしで利用可能
  • それ以降は使用クレジットベースに移行する見込み
まだお試し期間があるから、今のうちに使い倒すのが正解かも！

④ Fable 5 のコーディング性能とは

  • 大規模な移行や複雑な実装など野心的なプロジェクト向けに設計
  • 自身でテストを作成・実行し、高い忠実度でデザインを実装
  • 複数日にわたる自律セッションにも対応可能
  • 長時間にわたる自律的なタスクでもゴールを見失わない構造
ただ動くだけじゃない。自分でテストして確認して修正する自律ループがすごい！

⑤ Dynamic Workflows の基本概念

  • 2026年5月下旬 Claude Opus 4.8と同時に研究プレビューとして導入
  • Claudeが作業を計画し、数百の並列サブエージェントを実行
  • JavaScriptのオーケストレーションスクリプトを自動生成
  • 逐次的ではなく、並列で多数のエージェントを調整して実行
従来の「1つのセッションで順番にやる」方式から、「たくさんのエージェントで同時にやる」方式へ進化！

⑥ Dynamic Workflows の仕組み

  • ClaudeがJavaScriptファイルとしてオーケストレーションを自動生成
  • フェーズごとに複数のエージェントが独立したタスクを実行
  • 統合エージェントが結果をまとめて最終出力を生成
  • 検証エージェントが生成結果をレビュー・フィードバックするアドバーサリアル検証も搭載
「ハーネス」と呼ばれるラッパープログラムをその場で動的に作る仕組みです。

⑦ Dynamic Workflows の起動方法

  • Claude Code v2.1.154以降が必要
  • プロンプトに「Ultra Code」と入力すると起動可能
  • Proプランでは /config からオプトインで利用できる
  • Bedrock・Vertex AI・Microsoft Foundryでも利用可能
「Ultra Code」と書くだけで自動的に並列エージェントが立ち上がる。簡単！

⑧ エージェントの3つの失敗モードを解決

  • エージェントの怠惰: コンテキストが長くなると作業を途中でやめてしまう
  • 自己ひいきの偏見: 自分が出した結果を無意識に優先してしまう
  • ゴールからの逸脱: 本来の目的からだんだんずれてしまう
  • Dynamic Workflowsでは独立したコンテキストで分離して実行するためこれらを回避
1つのセッションが長くなるほど問題が起きやすい。だから複数に分けるのが正解！

⑨ 向いているタスク・向いていないタスク

  • 向いている: 大規模リポジトリの監査、フレームワーク移行、大量のメール返信
  • 数百ファイルの一括リファクタリング、多角的な市場調査
  • 向いていない: 簡単な修正や小規模なタスク（リソースが無駄になる）
  • 規模感を見極めてから使うことがコスト管理の鍵
小さな作業なら普通のClaude Codeで十分。大規模になったらUltra Code！

⑩ コスト管理の重要性

  • Dynamic Workflowsはトークン使用量が増加し、コストも高くなる傾向
  • 小規模なパイロット実行で効果を確認してから本番へ
  • モデルをOpus・Sonnet・Haikuと使い分けてコストを最適化
  • トークンの上限を指定して予算オーバーを防ぐ戦略も有効
力が強い分コストも強い。賢く使い分けるのがプロのやり方！

⑪ Claude Opus 4.8 との連携

  • 2026年5月29日 にClaude Opus 4.8がリリース
  • コーディング・エージェント能力・推論・知識作業で改善
  • ユーザーが「努力レベル」を制御できる新機能を搭載
  • Opus 4.8がDynamic Workflowsの研究プレビューと同時に登場
Opus 4.8の努力レベル調整は、コストと品質のバランスをとる画期的な機能！

⑫ 2026年3月のClaude Code大アップデート

  • インタラクティブな視覚化: チャートや図、モバイルアプリを生成可能に
  • オートモード: Proプランで自律的なタスク実行が可能に
  • Dispatchやメッセージアプリからリモート操作にも対応
  • カスタム自動メモリディレクトリとメモリリークの修正
Opus 4.6のデフォルト出力が64k、上限128kに引き上げられたのもこの時期！

⑬ フックとワークフロー深度の拡張

  • フック機能: コミット前やツール実行後などに自動実行するシェルコマンドが利用可能
  • 開発ワークフローに自動チェックを組み込める
  • コミット前にリントやテストを自動実行する設定も簡単に追加
  • Gitのpre-commitフックと同じ感覚でAIのツール実行をフック可能
コミット前に自動で脆弱性チェックが走る、みたいなのが設定できるの便利すぎる！

⑭ 新しいスラッシュコマンド群

  • /goal: 自律的なタスク実行のための新コマンド
  • /usage: プラン制限の内訳を表示するコマンド
  • /context: コンテキスト提案の改善でより適切な情報を取得
  • エージェントビュー: 実行中・ブロック中・完了のタスクを一覧表示
/goalで目的を設定すると、Claudeが自律的に計画を立てて実行してくれる。

⑮ セキュリティと安全機能の強化

  • security-guidanceプラグイン: Claudeの変更内容を脆弱性について自動レビュー
  • Fable 5の安全分類器が危険なリクエストをOpus 4.8にルーティング
  • Mythos 5のアクセスは厳選されたパートナーにのみ制限
  • Dynamic Workflows内でのアドバーサリアル検証で事実誤認を自動検出
AIが自分のコードを自分で攻撃して検証する。まさにアドバーサリアル！

⑯ Claude Code GitHubアクションのロードマップ

  • CI結果の表示機能の向上
  • クロスリポジトリ対応: 複数リポを横断した操作が可能に
  • ワークフローファイルの変更とdispatchイベントへの対応
  • より良いコードレビュー動作の実装が予定されている
CI/CDパイプラインにClaudeが直接組み込まれる未来が近づいている！

⑰ Skills・サブエージェント・Agent Teamsとの違い

  • Skills: 再利用可能な定型プロシージャ（マニュアル的な手順）
  • サブエージェント: 1つのセッション内でタスクを分割して実行
  • Agent Teams: 役割を持ったエージェントを協調させる仕組み
  • Dynamic Workflows: JavaScriptでオーケストレーションし、最大1,000のエージェントを並列実行
Skillsは「手順書」、サブエージェントは「手伝い」、Dynamic Workflowsは「工場ライン」！
