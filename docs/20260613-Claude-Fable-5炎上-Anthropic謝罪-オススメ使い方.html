①Claude Fable 5とは？

  • 2026年6月9日にAnthropicがリリースしたMythosクラス初の一般向けAIモデル
  • 高性能すぎて公開を見送っていたMythosの能力を安全機構付きで一般提供
  • コーディングベンチマークSWE-bench Pro 80.3%を記録
  • コンテキストウィンドウ100万トークン、最大出力12.8万トークン
Opus 4.8を大幅に超える性能！
でも安全機構が複雑に…

②Mythosクラスの誕生経緯

  • Anthropicのモデル階層：Haiku → Sonnet → Opus → Mythos（新設）
  • 2026年4月のProject GlasswingでMythos Previewを限定公開
  • 米国政府と連携、サイバー防衛組織にのみ提供
  • 高度なセキュリティ・生物化学知識の悪用リスクを考慮
  • 約150の組織・15カ国以上に展開済み
10,000件以上の深刻な脆弱性を発見済み！

③Fable 5とMythos 5の違い

  • 同じ基盤モデル（パラメータ共通）でセーフガードの有無のみが違い
  • Fable 5: 一般向け、安全機構あり
  • Mythos 5: 制限付き、承認済みパートナーのみ
  • 高リスク判定時はOpus 4.8にフォールバック
  • モデル蒸留（ディスティレーション）もブロック対象
1つのモデルを2つの製品に分ける
Anthropic初の試み！

④Fable 5の圧倒的ベンチマーク

  • SWE-bench Pro: 80.3%（Opus 4.8は69.2%、GPT-5.5は58.6%）
  • ViBench（バイブコーディング）で#1を獲得
  • ほぼ全ベンチマークでOpus 4.8を+10%以上で上回る
  • 1,000時間以上のレッドチームテストでユニバーサル Jailbreak 0件
  • アライメント評価はOpus 4.8と同レベル
GPT-5.5の58.6%を大きく引き離す！

⑤料金体系と料金設定

  • API料金：入力$10 / 出力$50（百万トークンあたり）
  • Opus 4.8（入力$5/出力$25）の約2倍
  • プロンプトキャッシュ割引90%で実質入力$1に
  • 6月22日まではPro/Max/Team/Enterpriseで追加費用なし
  • 6月22日以降は使用量クレジットが必要
AnthropicはFable 5をプレミアムではなく
デフォルトにしたい意図あり

⑥セーフガードアーキテクチャ

  • 分類器AIがリクエストを判定し、危険ならOpus 4.8にルーティング
  • セッションの95%以上はガードレールが発動しない
  • 対象3領域：サイバー・生物学・化学
  • 拒否ではなくフォールバック方式で回答を継続
  • データ保持は30日間必須（安全監視目的、学習には不使用）
能力を落とさず安全性を担保する
新しいアプローチ！

⑦「見えないガードレール」発覚

  • 公開されたセーフガードとは別に密かに性能低下を仕込んでいた
  • フロンティアLLM開発目的の利用で応答を改ざんする隠し制限
  • AIモデル開発・コードデバッグ・アーキテクチャ最適化で発動
  • ユーザーへの通知なしでモデルの性能を意図的に低下
  • 高額なAPIトークンを意図的に低下した応答に消費させている状態に
正規品を買っているのに
こっそり性能を落とされていた！

⑧コミュニティの激しい反発

  • AI研究者・機械学習コミュニティから大規模な批判が巻き起こる
  • Anthropicの透明性の欠如を厳しく非難
  • 反競争的行為として研究者の仕事を秘密裏に妨害
  • 「倫理的企業」のイメージを損なう行為と指摘
  • RedditやXで炎上が拡大、業界全体に波紋
「エシカルAI」を標榜する企業が
秘密裏に性能を落とすとは…

⑨Anthropicの謝罪と対応

  • 炎上からわずか48時間以内に過ちを認めて謝罪
  • 「誤ったトレードオフをしてしまい、バランスを適切に取れなかったことをお詫びする」
  • 方針を転換しセーフガードの可視化を決定
  • 今後はフォールバック時に明確な理由を通知
  • 拒否やルーティング変更を透明に開示する仕組みへ
48時間での謝罪は速かったけど…
最初から透明にすべきだった

⑩なぜ見えないガードレールを作ったのか

  • Anthropicの主張：誤検知の削減と回避困難にするため
  • ガードレールの存在が分かると悪意ある利用者が回避できる
  • 多くの批評家は競争制限のビジネス動機と見なす
  • Fable 5はMythosの「ナーフ版」であり性能調整版として一般公開
  • 安全性と透明性のバランスを取る判断が失敗
安全のための隠蔽か、
ビジネスのための隠蔽か…

  • 可視化されたセーフガードに移行完了
  • 対象リクエストはClaude Opus 4.8に明確にフォールバック
  • APIユーザーには拒否理由やルーティング変更を明示
  • コミュニティの声に応じた透明性の改善
  • 今後は安全策の変更を事前に公開する方針
