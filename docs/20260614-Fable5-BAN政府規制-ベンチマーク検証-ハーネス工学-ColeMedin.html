⚡ パート1: 何が起きたのか

②Mythos 5とFable 5の関係

  • Mythos 5は選ばれた企業のみが使える超強力モデル
  • Fable 5はMythos 5にガードレールを追加した一般向け版
  • マルウェア開発・生物兵器関連をブロック
  • ジェイルブレイクでMythos 5の完全能力が解放される恐れ
Fable 5は「日本全国民解放版」ではなく「ガードレール付き安全版」という位置づけ

③Anthropicの反論: 納得できない

  • 「見つかった脆弱性は他の公開モデルでも見つけられる程度」
  • Fable 5特有の新しい悪意ある能力はないと主張
  • 本当かどうかは50/50とCole氏は評価
  • 顔を立てるための発言の可能性も否定できない
もしAnthropicの主張が真実なら、規制は行き過ぎと言わざるを得ない

④なぜ「全員停止」になったのか

  • 外国人のアクセスを米国内のみに限定できなかった
  • VPNやサーバー経由で回避する手法が多数存在
  • 米国市民を仲介して外国人が使う手口も阻止困難
  • 結果として全ユーザーを停止するしか選択肢がなかった
Anthropic側にも「分離技術がない」という弱みが露呈

⑤ジェイルブレイクの通報者は誰?

⑥業界全体への前例になる危険

  • ガードレール付きでも「ダメ」なら何も出せない

  • Fable: 入力$10/M・出力$50/M

  • 1時間常時稼働で$300〜$600という爆発的な費用

⑧バウンドタスク: Opusで十分

  • GitHub issue対応など「明確なゴール」がある作業
  • FableとOpusの差はわずか1ポイント
  • シンプルなタスクならKimmyでも良好
  • 差は「有意だが劇的ではない」レベル

⑨オープンエンドタスク: Fableが輝く

  • 「ゼロからアプリを構築」のような探索的タスク
  • 70点満点ルーブリックで6ポイント差でFable勝利
  • 計画・反復・論理構築で特に優位
  • 「解決経路が未知」の問題に最も強い
コード品質は同等だが、問題解決力でFableが勝つ

⑩ゲーム開発テストの結果

  • Fable版: グラフィック・バランスともに成功
  • Opus版: デモが自分のゲームに負けて敗北
  • 高次メタロジック(全体バランス等)で差が出た
スペック「勝てるゲームを作って」→ Opusは達成できず

⑪最適ワークフロー: 計画だけFable

  • 最強モデルが必要なのは計画ステップだけ

🔧 パート3: ハーネス工学という解決策

⑬モデルより「ハーネス」が重要

⑭6つのAIレイヤー要素

  • ルール: CLAUDE.md等のシステムプロンプト

  • サブエージェント: コンテキスト管理の要
  • LSP: コード検索精度を向上
  • フックス: 決定論的アクション自動実行
  • スキル: ワークフロー手順の定義
6要素を磨くだけで、モデル交換にも強くなる
