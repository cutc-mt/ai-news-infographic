① なぜAIは忘れるのか

  • 先週話したことをまた1から説明しなければならない
  • 昨日教えた好みを今日は忘れている
  • 何度も同じ質問をしてくる
  • これはAIのアーキテクチャの根本的な問題
コンテキストウィンドウを広げても一時的な記憶に過ぎない⚡

② VLDB2026採択の重要性

  • データベース分野の最高峰会議に採択
  • VLDB = Very Large Data Bases（超大型データ処理）
  • 世界最先端の査読を通過した実証システムの証
  • 2026年5月28日にarXiv投稿
単なるアイデアではなく実際に動くシステムとして認められた🏆

③ 1日10億トークンの現実

  • 最大テナントが1日に10億トークン生成
  • Geminiの200万トークン窓の500倍
  • コンテキストは有限で高コストなリソース
  • 全てを詰め込むアプローチは現実離れ
バイトダンス本番環境の極限データから生まれた🔥

④ 既存手法の3つの限界

  • 生ログ保存: ノイズで検索制度が低下
  • RAG: 動的な状態変化を捉えられない
  • ナレッジグラフ: ビルド時コストが高すぎる
  • 特定ユースケースへの過剰特化問題
従来手法は会話記憶には根本的に不向き❗

⑤ ミリックスの25秒問題

  • ミリックスのP95レイテンシが25.76秒
  • LongMemEval評価が24時間以内に終わらない
  • 精度が高くても実運用には使えない
  • リアルタイムアプリには実用的でない
ユーザーが10秒以上待つAIは実用にならない🐢

⑥ バイトダンスの3つの産業観察

  • 生ストリームの低シグナル・ノイズ比が高い
  • 状態の動的な性質: ユーザーは常に進化
  • アーキテクチャの断片化: ユースケース別に異なる
  • これらが設計思想の出発点
現場の豊富な経験から導き出された本質的観察👀

⑦ MemoryBaseという新概念

  • 記憶をデータベースとして設計する新パラダイム
  • イベントとエンティティで記憶を管理
  • Memory + DBMS = MemoryBase
  • RDBがテーブルで管理するように統一
DBMSの地見をAI記憶に応用した画期的アプローチ💡

⑧ 原則1: 選択的情報抽出

  • 最細粒度のセグメンテーションを採用
  • 低密度ストリームを価値ある信号に変換
  • トークンコストを最小化しながら精度向上
  • RAG比で83.18%のストレージ削減
全て保存せず、価値ある情報だけを取り出す✂️

⑨ 原則2: 固有の状態性と進化

  • 記憶を生きたライフサイクルとして扱う
  • 時間的重み付けで直近コンテキストを優先
  • 永続的エンティティの継続的な更新
  • 静的蓄積ではなく進化し続ける
学生から社会人になる変化も追跡できる🌱

⑩ 原則3: 汎用的な設計パラダイム

  • JSON風のスキーマで設定可能
  • 単一インフラで多様なメモリタイプを表現
  • ビジネスルールからロジックを分離
  • 最小限の再設定で異なるタスクに対応
1つのアーキテクチャで多様なユースケースに対応⚙️

⑪ Event-Entityモデル: 2層構造

  • Event: タイムスタンプ付きエピソード記録
  • Entity: 永続的で進化する状態表現
  • Event Store = 汎用ログ
  • Entity = マテリアライズドビュー
entity = SELECT content
FROM event
WHERE type = '...'
SQL的クエリでイベントとエンティティの関係を定義💻

⑫ オペレーターライブラリ

  • 統計系: SUM/AVG/MAX/COUNT
  • LLM系: LLM_Mergeで意味的統合
  • Time_Compress: 時系列圧縮
  • 状態更新を管理する関数群
シナリオ別に適切な演算を選べる🧰

⑬ シングルパス抽出の革新

  • 従来はNタイプにN回のLLM呼び出し
  • VikingMemは1回のパスで全タイプ抽出
  • コスト$0.35→$0.07に5分の1削減
  • 処理時間も11.2秒→7.42秒に短縮
精度は88.86%→88.77%でほぼ同等⚡

⑭ EUA: エンティティ更新アルゴリズム

  • Search-Replace形式のパッチを出力
  • 編集距離ベースの近似検索で適用
  • コード編集技術からインスパイア
  • 上位5件候補だけ取得でLLM呼び出し削減
コーディング技術が記憶更新に応用された✨

⑮ IMSM: インテリジェント・セグメンテーション

  • Phase1: 意味サリエンスフィルタ
  • Phase2: イベント中心パーティショニング
  • 挨拶などの会話的フィラーを除去
  • 除去すると5.32ポイント低下の最重要要素
検索時追加コストほぼ0で最大精度貢献💎

⑯ マルチパス検索とリランキング

  • プライマリ: 意味+語彙ハイブリッド
  • 時間減衰スコア + ビジネススコア
  • 補助: キーワードグラフ検索
  • ColBERT風リランキングで精度強化
score = w_s × semantic
       + w_t × time
       + w_p × popularity
多角的な検索で精度を確保🎯

⑰ 実験結果: 圧倒的性能

  • LoCoMo: 88.83%〜90.12%で首位
  • P95レイテンシ: 0.39秒で最低水準
  • ストレージ: 83.18%削減
  • 精度がベースライン比最大38%向上
少ない情報量で精度が上がる逆説を実証⭐
