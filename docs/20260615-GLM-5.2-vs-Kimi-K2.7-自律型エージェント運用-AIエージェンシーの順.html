② 自立型エージェントが主流に: API課金では太刀打ちできない

  • 今年の初めにOpenClawが登場して、自立型エージェントの普及が加速
  • 最近ではHermes Agentも登場し、汎用エージェントの話題がさらに盛り上がり
  • 自立型エージェントの特徴はトークン消費が半端ないこと。1回の指示でゴールまで自律的に動き続ける
  • 重量課金だと20ドル、30ドル、50ドルがあっという間に飛んでいく
使い方の芯が変わってきた。ワンショットのQ&Aから、ゴールに向かって自律的に動くエージェントへ。もうサブスク一択の時代。

③ GLM-5.2がいきなり登場: 海外でかなりの高評価

  • ZAI（智譜AI）がGLM-5.2をリリース。出たばかりの最新モデル
  • 海外の評価はかなり高く、優れたモデルとして注目を集めている
  • オーパスモデルに対抗するために設計されたコード特化モデルの位置づけ
  • 厳密な比較はまだ難しいが、特徴と使い道の整理は可能
まだ出たばかりなのでコーディングテストなどの厳密な比較は難しい段階。あくまで情報収集に基づくざっくりとした比較になります。

  • GLM-5.2もK2.7も出たばかりなので、どちらもまだ比較の情報が限定的
  • 能力的には近い水準ではないかという予想。テスト待ちの状態
どちらもまだリリースされたばかりなので、今後のテスト結果が楽しみですね！

⑥ インプット対応: K2.7はマルチモーダル入力に対応

  • GLM-5.2 はテキストのみの入力対応。画像や動画は読み込めない
  • K2.7 Code はテキスト＋画像＋動画に対応。マルチモーダル入力が可能
  • ただし画像や動画の「生成」ができるわけではなく、読み込んで解析できるという意味
  • K2.7ならデザインの画像を渡して「これをベースに作って」ができる。バイブコーディングにも有利
利用目的によって選び分けましょう。テキスト中心ならGLM、画像も使いたいならK2.7が便利です。

⑦ 推論モード: GLMは選択可能、K2.7は推論必須

  • GLM-5.2 は推論モードを選択可能。Max推論モードも用意されている
  • K2.7 Code は推論モードが必須。選べない仕様になっている
  • 推論モードを使わない場面では、GLMの方が柔軟なコスト管理が可能
  • 推論モード必須のモデルは、精度は高いがトークン消費が増える傾向がある
タスクの難易度に応じて推論モードのON/OFFを選べるのは、GLMの大きなメリットです。

⑧ 提供方法の違い: ZAIはコーディングプラン、KimiはAPI

  • GLM-5.2 はZAIコーディングプラン加入者向けに先行公開中
  • コーディングプランに入っていれば、すぐにGLM-5.2が使える状態
  • K2.7 Code はOpenRouterなどでAPI利用が可能。従量課金の料金が明確
  • K2.7はインプット・アウトプットの単価が明確。ワンショットの開発用途にはAPIが有利
ZAIの従量課金料金は不明確な点があるのに対し、Kimiの方がコスト計算がしやすいという違いも。

⑨ ZAI公式のおすすめ運用: 普段は4.7、難しい時は5.2

  • 普段使いはGLM-4.7が最適と公式ドキュメントにも記載されている
  • GLM-4.7はバランスが取れていて、スピードもあり、安定している万能モデル
  • 難易度の高いタスクにはGLM-5ターボやGLM-5.2に切り替えるのが正解
  • ただし万能ではあるが「有能ではない」ことも。時々おバカなことをするので工夫が必要
高い水準を求めるときだけ最新モデルに切り替える。普段は安定した4.7を使うのが賢い使い方です。

⑩ OpenClawやHermes Agent: 常時利用は2.5で十分

  • OpenClawやHermes Agentを常時動かす時にはK2.5で十分なレベル
  • 難易度の高いタスクに入ってきたら、その時だけGLM-5.2や5ターボに切り替える
  • GLM-4.7はメモリーや記憶媒体の工夫で賢く教育すれば、うまく動いてくれる
  • モデルの切り替え運用がコストパフォーマンスを最大化する秘訣
