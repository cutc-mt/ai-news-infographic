① Anthropicが重量課金を開始: 自立型エージェント使いにとって痛いニュース

  • 6月15日 ついにこの日がやってきた。Claude ProやエージェントSDKでの重量課金が本格開始
  • プログラム経由でClaude Codeを動かすと、従量課金になる仕組みに変更
  • コミュニティでは「これからどうすんだ？」とざわざわ騒然の状態
  • 自立型エージェントを動かしている人にとっては請求があっという間に膨れ上がる
Anthropicモデルでエージェント的なワークフローを構築している人は、まさに痛いニュース！対応策が急務です。

② 自立型エージェントが主流に: API課金では太刀打ちできない

  • 今年の初めにOpenClawが登場し、自立型エージェントの普及が加速
  • 最近ではHermes Agentも登場。汎用エージェントの話題がさらに盛り上がり
  • 自立型エージェントの特徴はトークン消費が半端ないこと。1回の指示でゴールまで自律的に動き続ける
  • 重量課金だと20ドル、30ドル、50ドルがあっという間に飛んでいく
使い方の芯が変わってきた。ワンショットのQ&Aから、ゴールに向かって自律的に動くエージェントへ。もうサブスク一択の時代です。

③ GLM-5.2がいきなり登場: 海外でかなりの高評価

  • ZAI（智譜AI）がGLM-5.2をリリース。出たばかりの最新モデル
  • 海外の評価はかなり高く、優れたモデルとして注目を集めている
  • オーパスモデルに対抗するために設計されたコード特化モデルの位置づけ
  • 厳密な比較はまだ難しいが、特徴と使い道の整理は可能
まだ出たばかりなのでコーディングテスト等の厳密な比較は難しい段階。情報収集に基づくざっくりとした比較になります。

④ Kimi K2.7 Codeもリリース: 中国系モデルの激しい競争

  • Kimi KシリーズのK2.7 Codeもリリースされたばかり
  • Kimiは月面のAI企業が開発するモデルシリーズで、コーディングに強みを持つ
  • GLM-5.2もK2.7も出たばかりで、どちらもまだ比較情報が限定的
  • 能力的には近い水準ではないかという予想。テスト結果が待たれる状態
どちらもリリースされたばかり。今後のテスト結果が楽しみですね！

  • チャットのラリーが長引いても、文脈を維持したまま作業を続けられる

⑥ インプット対応: K2.7はマルチモーダル入力に対応

  • GLM-5.2 はテキストのみの入力対応。画像や動画は読み込めない
  • K2.7 Code はテキスト＋画像＋動画に対応。マルチモーダル入力が可能
  • 画像や動画の「生成」ができるわけではなく、読み込んで解析できるという意味
  • K2.7ならデザイン画像を渡して「これをベースに作って」ができる
利用目的で選び分けましょう。テキスト中心ならGLM、画像も使いたいならK2.7が便利です。

⑦ 推論モード: GLMは選択可能、K2.7は推論必須

  • GLM-5.2 は推論モードを選択可能。Max推論モードも用意されている
  • K2.7 Code は推論モードが必須。選べない仕様
  • 推論なしで十分な場面では、GLMの方が柔軟なコスト管理が可能
  • 推論必須のモデルは精度は高いが、トークン消費が増える傾向がある
タスクの難易度に応じて推論モードのON/OFFを選べるのはGLMの大きなメリットです。

⑧ 提供方法の違い: ZAIはプラン制、KimiはAPI明確

  • GLM-5.2 はZAIコーディングプラン加入者向けに先行公開中
  • コーディングプランに入っていれば、すぐにGLM-5.2が使える
  • K2.7 Code はOpenRouterなどでAPI利用が可能。従量課金の料金が明確
  • K2.7はインプット・アウトプットの単価が明確。ワンショット開発にはAPIが有利
ZAIの従量課金料金は不明確な点が多いのに対し、Kimiの方がコスト計算がしやすいという違いもあります。

⑨ ZAI公式の運用術: 普段は4.7、難しい時は5.2に切り替え

  • 普段使いはGLM-4.7が最適と公式ドキュメントにも明記されている
  • GLM-4.7はバランスが良く、スピードもあって安定している万能モデル
  • 難易度の高いタスクにはGLM-5ターボやGLM-5.2に切り替えるのが正解
  • 万能ではあるが「有能ではない」ことも。工夫で賢く動かす必要がある
高い水準を求めるときだけ最新モデルに切り替える。普段は安定した4.7を使うのが賢い運用です。

⑩ OpenClawやHermes Agent: 常時利用ならK2.5で十分

  • OpenClawやHermes Agentを常時動かす時にはK2.5で十分なレベル
  • 難易度の高いタスクに入ったら、その時だけGLM-5.2や5ターボに切り替える
  • GLM-4.7はメモリーや記憶媒体の工夫で教育すれば、うまく動いてくれる
  • モデルの切り替え運用がコストパフォーマンスを最大化する秘訣
常時起動のエージェントは低コストモデルで回して、難しい時だけ高スペックに切り替える。これが賢い運用術です。

⑪ ZAIコーディングプラン: 値上がりが続いてお勧めできない状況に

  • 当初はGLM-4.5が出た時のキャンペーンで3ドルからスタート
  • キャンペーン終了後に月額6ドルに、さらにライトプランが今や16.2ドルにまで値上がり
  • ZAI一択で他のモデルが選べないのに月額16.2ドルは割高感が否めない
  • 配信者自身も「6ドルのままで行けたらよかった」と後悔している
早く入った人は安く使えているけど、今から入るには金額的に厳しい状況になっています。

⑫ OpenCode 5が激推し: 初月5ドル、翌月から10ドルで中国系LLM多数

  • OpenCode 5 は初月5ドル、翌月から月額10ドル。圧倒的にコスパが良い
  • GLMだけでなく中国系のLLMが多数選べるという最大のメリット
  • タスクに応じてGLM-5.2、K2.7 Code、DeepSeek V4 Proなど柔軟に切り替え可能
  • ライトユーザー向けの軽量モデルならほぼ無限に使えるレベルのボリューム
モデルを選び放題で月額10ドル。これが今のベストチョイスになりつつあります。

⑭ GLM-5.2のトークン消費倍率: ハイグレードモデルは2〜3倍消費

  • GLM-5.2と5ターボはオーパスモデル対抗のハイグレードコードモデル
  • ピーク時間は3倍、オフピークは2倍でトークン消費される
  • GLM-4.7など下のクラスのモデルよりも消費が増えることに注意が必要
  • オフピークで使ってもベースモデルの2倍かかる点は覚えておきたい
最新モデルは強力だけど、その分トークンも食う。プランのボリュームを確認してから使いましょう。

⑮ DeepSeek V4も選択肢: ライトタスクに最適な切り替え戦略

  • DeepSeek V4 Pro はGLM-5.1の約2倍のボリュームが使える
  • DeepSeek V4 Flash はさらに大量のリクエストが可能。ライトユーザーならほぼ無限
  • 公式で「1ドルや5ドルのプランでも十分使える」と謳われるモデル
  • 難しいタスクはGLM-5.2、簡単なタスクはDeepSeek V4 Flashと使い分けるのが賢い
OpenCode 5に入ればGLMとK2.7とDeepSeekが全部選べる。これが最強の運用です。

⑯ 併用運用の戦略: Anthropic＋中国系モデルのハイブリッド

  • Anthropicのサブスクも合わせて入っている人は、あえて早く選択する必要はない
  • AnthropicはSonnetやOpus中心で回し、中国系モデルでトークン消費を節約する使い方が可能
  • 「ソネットで十分じゃん」と思った時に、DeepSeek V4 Flashなどで軽く済ませる運用
  • マルチモデル併用が今の時代のベストプラクティスになりつつある
一つのベンダーに依存しない、マルチモデルの運用がコストを抑える最大の武器になります。

⑰ Z Code: ZAIのデスクトップエージェントアプリも登場

  • Z Code はZAIが開発したデスクトップ用のコーディングエージェントアプリ
  • イメージとしてはAnthropicのClaude Code DesktopやOpenAIのCodexアプリに近い
  • スキル登録も可能。デスクトップのアシスタントとしても使える
  • ただしスマホからの連携はまだ未対応。外出先からの遠隔操作は不可
最新モデルをすぐ使えるメリットもあるので、ZAIプランの残りの利点として覚えておきましょう。

⑱ まとめ: 今の時代のベスト戦略はマルチモデル運用

  • 今から入るならOpenCode 5が最もお勧め。月額10ドルで中国系LLMを選び放題
  • ZAIコーディングプランの既存加入者は維持した方がお得。解約しないのが正解
  • タスクに応じてモデルを切り替えるマルチモデル運用がコストを最小化する鍵
  • Anthropicの重量課金対策として、中国系モデルへの乗り換えが現実的な選択肢に
1つのモデルに依存しない、柔軟な運用がこれからのAI活用の基本戦略です！
