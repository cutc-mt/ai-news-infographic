⑥ ADKエージェントの基本構造

  • エージェントは自律的で目標指向。ツールを呼び出し、記憶を参照し、ゲーム環境と対話する
  • ゲームのテキスト出力をパースして理解し、適切なコマンドを生成して入力する
  • 静的なパイプラインや動的ルーティングで、複数エージェントの協調動作を制御可能
  • プロジェクト構造は `agent/agent.py` にエージェント定義とツールをまとめるシンプル設計

⑦ Geminiがエージェントの「脳」に

  • Gemini 2.5 Flashがゲームの描写を読み込み、文脈を理解して次の行動を決定する
  • 固定スクリプトではない。動的で適応的なゲームプレイが可能。パズルの解き方も柔軟
  • オブジェクトを識別し、利用可能なアクションを推測し、戦略的な行動計画を立てる
  • Flashモデルを採用することで高速な推論を実現。リアルタイムのゲームプレイに最適

⑧ OpenAPIツールでゲームと対話

  • ADKのツールシステムを使うと、エージェントは外部APIと対話できる
  • ゲームのOpenAPI仕様からツールを自動生成。手作業でAPIラッパーを書く必要がない
  • コマンドをゲームに送信し、ゲームの状態情報を受け取る双方向通信を実現
  • Pythonコード実行ツールも統合。ハッシュ計算などコードベースのパズルにも対応可能

⑬ セットアップ手順 — ゼロから始めるADK

  • Step 1 Python 3.13+ と uv をインストール。パッケージマネージャーで依存関係を管理
  • Step 2 GitHubリポジトリをクローンして uv sync で依存をインストール
  • Step 3 Google AI StudioでAPIキーを取得し、環境変数 ADK_API_KEY に設定
  • Step 4 uv run adk web でエージェントを起動。Web UIで監視しながら実行

cd adventure-game-ai-agent
