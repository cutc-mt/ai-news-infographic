② SaaSの現状 — 月額課金の罠

  • 手書き風動画を作るSaaSは多数存在：VideoScribe Vyond Doodly Powtoon
  • まともに使おうとすると月額50ドル前後〜それ以上のサブスク費用がかかる
  • 尺・書き出し回数・ウォーターマーク・素材数・AIクレジット・商用利用・再販権など自由が細かく制限される
  • 月額課金の画面の向こう側にある機能を、自分の手元に取り戻すことはできないのか？
月額課金し続けて
自由度まで制限される。

③ パラダイムシフト — 使う側から分解する側へ

  • 「SaaSをやめる」のではなく、SaaSが持っていた制作機能そのものを自分のAIエージェント環境に持ってくる
  • OpenAIのCodex AppやClaude Codeの「スキル」として再構築する発想
  • SaaSを使う側から、SaaSを分解して自分の道具にする側へ移行する
  • これは時代の入口を示す試み。単なる節約術ではない
重要なのは「やめる」ではなく
「取り込む」こと。

④ Claude Code — AIコーディングエージェント

  • Anthropic社のClaude Codeはターミナル上で動くAIコーディングアシスタント
  • 自然言語の指示でコードを書いたり実行したりできる強力なエージェントツール
  • 「スキル」という仕組みで独自の処理パイプラインを登録・再利用できる
  • 今回はこのスキル機能を使って手書き風アニメ動画制作ワークフローを構築した
Claude Codeの
スキル機能が
今回の主役！

⑤ Codex App — OpenAIの自律型AIエージェント

  • OpenAI社が提供するCodex Appは、自律的にコードを生成・実行するAIエージェント
  • Claude Codeと同様に独自のスキルやワークフローを登録して拡張できる
  • 外部APIとの連携も可能で、Aivis Cloud APIなどの音声合成も組み合わせられる
  • Codex App / Claude Codeのどちらでも動作するスキルとして開発した
2つのAIエージェントで
共通して動くスキルを

⑥ 手書き風アニメーションとは？

  • ペンを持った手の画像が線をなぞって描いていくスタイルの動画表現
  • 通称ホワイトボードアニメーション。解説動画やプレゼンで大人気の手法
  • SVG素材をベースにパスをトレースして、手書き風の描画アニメーションを生成する
  • 最終的に動画として書き出し、ナレーション付きにすることも可能
解説動画の定番スタイル！
視覚的に分かりやすい
から需要が高い。

⑦ SVG素材 — アニメーションの土台

  • SVG（Scalable Vector Graphics）は拡大・縮小しても画質が落ちないベクター画像形式
  • SVGのパスデータを解析して、手書き風に線が引かれるアニメーションに変換する
  • stroke-dasharrayとstroke-dashoffsetのCSSアニメーションで描画効果を実現
  • Excalidrawや手描き風ツールで作ったSVG素材もそのまま流用可能
SVGのパスをなぞる。
シンプルだけど効果的！
CSSだけで描けるのが強み。

⑧ 手の画像追従 — リアルな描画感

  • ペンを持った手の画像をSVGパスの進行に合わせて追従させる
  • パスの現在の描画位置を計算して、手の座標をリアルタイムに更新
  • 線の進行方向に合わせて手の向きも回転させることで自然に見える
  • これによって本物の手が描いているようなリアルな映像体験を再現
手の画像を動かすだけで
グッと本物っぽくなる！
細かい工夫が大事。

⑨ 動画としての書き出し

  • SVGアニメーションを動画ファイルとして書き出すステップが必要
  • PuppeteerやPlaywrightでブラウザ上のアニメーションを録画する手法が主流
  • CSSアニメーションのフレームを1フレームずつキャプチャしてffmpegで結合も可能
  • 長尺の動画でもAIが自動で処理するので、手動の手間はほぼゼロ
SVG→動画の変換は
Puppeteer + ffmpegの
コンボが定番！

⑩ 音声合成でナレーション付き動画へ

  • Aivis Cloud APIなどの音声合成（TTS）APIを組み合わせることでナレーションを追加
  • テキストから自然な日本語音声を自動生成。話者の声質も選択可能
  • 映像と音声のffmpegでの合成により、ナレーション付き解説動画が完成する
  • BGMも追加すれば、プロ品質の解説動画を1発で生成できる
ナレーションまで
AIで自動生成。
動画制作が完全自動化！
