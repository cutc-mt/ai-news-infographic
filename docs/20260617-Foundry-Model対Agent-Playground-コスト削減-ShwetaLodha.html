Model Playground vs Agent Playground

Microsoft Foundryでコストを節約する選び方のコツ
2026年6月17日

① 動画の概要

  • Microsoft Foundryにある2つのプレイグラウンドの違いを解説
  • 見た目は似ているが目的が全く異なる2つのツール
  • 間違った選び方をすると余計なコストと手間がかかる
  • 正しい使い分けで開発の効率を最大化する方法を紹介
両方ともAIとチャットできるけど、役割がまるで違うんだ！

② 一言でまとめると

  • Model Playground = 1つのモデルを実験する場所
  • Agent Playground = システム全体を構築する場所
  • Model: 「どのモデルとプロンプトを使うべき？」
  • Agent: 「私のエージェントは実際に動く？」
この2つの質問を分ければ、どちらを使うか迷わない！

③ チャンネル紹介

  • Shweta Lodhaが解説
  • Azure OpenAIやChatGPTなどのAI系コンテンツを中心に活動
  • MediumブログやGumroadで学習リソースも公開中
  • Instagram / X / Facebookで最新情報を発信
視聴者からのリクエストでこの比較動画を作ってくれた！

④ Microsoft Foundryとは

  • 旧称はAzure AI Foundry
  • MicrosoftのクラウドAIプラットフォームの一部
  • モデルの選択からエージェントの構築・デプロイまで一貫した環境
  • Playgroundが2種類あり用途が明確に分かれている
Azure AI Foundryから名前が変わったんだね！

⑤ サイドバイサイド比較

  • Model Playground最大の特徴: 最大3つのモデルを同時に比較
  • 同じプロンプトを複数モデルに一発で送信可能
  • プロジェクト初期でモデル未定の時に超便利
  • 3つの別々の実験を1つの画面で完了できる
これがModel Playgroundにしかない強力な機能！

⑥ パラメータの完全制御

  • Temperature: クリエイティブさの調整
  • Top P: トークン選択の範囲
  • Frequency Penalty: 繰り返しの抑制
  • Presence Penalty: 新しい話題の促進
  • すべてリアルタイムで調整できる
エージェント内でいじるよりここでチューニングするのが正解！

⑦ 組み込みツール群

  • Web Search: ウェブ検索機能
  • File Search: ファイル検索機能
  • Code Interpreter: コード実行機能
  • 3つとも追加設定なしですぐ使える
  • クイックな実験に最適な構成
セットアップ不要でサクサク使える！

⑧ SDKコードのエクスポート

  • 良い設定が見つかったらSDKコードを直接エクスポート
  • クリーンなスタート地点が手に入る
  • Agent Playgroundへの移行の足がかりとして最適
// エクスポートしたSDKコードの例
const response = await client.chat({
  model: "gpt-4o",
  messages: [...],
  temperature: 0.7
});
実験結果をそのままコードに！素晴らしいワークフロー

⑨ カスタムツールの統合

  • Agent Playgroundの核心: カスタムツールの追加
  • 外部APIの呼び出しが可能
  • 独自の関数やサービスをエージェントに接続
  • これがリアルなエージェントワークフローの基盤
組み込みツールじゃ足りない？ 自分専用ツールを繋げられる！

⑩ ナレッジソースの接続

  • ファイル、URL、SharePointなどに対応
  • 実際のデータをエージェントに接続
  • エージェントがデータを検索して推論できる
  • RAG（検索拡張生成）のセットアップをここで確認
データなしのエージェントじゃ意味ないもんね！

⑪ メモリ（状態の保持）

  • Model Playground: すべての会話が毎回リセット
  • Agent Playground: 状態をターン間で維持できる
  • 5〜6ターンのマルチターン会話をテスト可能
  • エージェントの記憶設定を詳細に構成できる
会話の文脈が途切れないのがポイント！

⑫ トレーシング（実行追跡）

  • Agent Playground専用の強力機能
  • ツール呼び出し、検索結果、推論ステップをすべて記録
  • 本番デプロイ前に実行の全容を確認できる
  • これをやらないと何時間ものデバッグに…
本番に行く前に必ずトレースを確認！後悔しないために！

赤い×がついてる機能はAgent専用だよ！

⑭ Modelだけにある機能

  • マルチモデル比較はModel Playground専用
  • Agent Playgroundでは2つのモデルを並べて比較できない
  • Agent Playgroundはデプロイ済みモデルのバージョン比較のみ
  • 「どのモデルを使うか」という疑問があれば→Modelへ
モデル選びの段階なら絶対Model Playground！

⑮ Agentだけにある機能

  • ナレッジソースの接続機能
  • メモリ（状態の保持）機能
  • トレーシング（実行の全追跡）機能
  • これらのどれかが必要ならAgent一択
ナレッジ・メモリ・トレースが必要ならAgent Playground一択！

⑯ コスト削減のコツ

  • 初期段階ではModel Playgroundを最大限に活用
  • Agent Playgroundはより高コストな処理が走る
  • モデルとプロンプトの決定が済むまでModelで済ませる
  • 設定が固まってからAgent Playgroundに移行する
無駄遣いしないための鉄則だね！

⑰ 推奨ワークフロー

  • Step 1: Model Playgroundでモデルを比較・選択
  • Step 2: 組み込みツールでプロンプトを調整
  • Step 3: SDKコードをエクスポート
  • Step 4: Agent Playgroundでカスタムツール・ナレッジ・メモリを追加
  • Step 5: トレースを確認して本番デプロイ
順番通りに進めれば確実！

⑱ まとめ: プロンプトからプロダクトへ

  • Model Playground: モデル選択 → プロンプト調整 → コード出力。軽量・高速・比較に最適
  • Agent Playground: カスタムツール → ナレッジ → メモリ → トレース確認 → 本番へ
  • 2つのツールで1つのプロダクションAIエージェントが完成する
  • 順番を守ればコストも手間も最小限に抑えられる
プロンプトがプロダクトになる道のりを段階的に進めよう！
