② n8nとは何か？

  • 公式の説明：「コードの柔軟さとノーコードの速さを合わせたワークフロー自動化」プラットフォーム
  • サービスからデータを取ってきて整え、別のサービスに登録する一連の流れを画面の上でブロックをつなぐように組み立てられる
  • マウス操作のノーコードで素早く作れる。複雑な処理にはプログラムを書くことも可能
  • 対応する外部サービスは400以上とされている。幅広い連携に対応
速さと柔軟さの両取り。
これがn8nの強み！
シンプルな作業はノーコードで、
複雑な処理はコードで。

③ 今日の3つのゴール

  • n8nがどんなツールで何ができるのかを自分の言葉で説明できるようになること
  • ノードを使って自動化するというn8nの根幹にある考え方を身近な例でイメージできること
  • このコース全体の地図を手に入れて、これから何をどんな順番で学ぶのか見通しが持てること
難しい知識は不要！
図を中心にゆっくり進めるので
安心してついてきてください。

④ ノードとは？ — 仕事の箱

  • ノード（Node）とは、1つの仕事を表す「箱」のこと。聞き慣れない言葉でも難しく考える必要はない
  • 例：何かを受け取る箱、APIでデータをやり取りする箱、条件で分ける箱、値を整える箱など
  • ノードを左から右へ線で繋いでいく。前のノードの出力結果が次のノードへ渡っていく仕組み
  • ちょうどレゴブロックを組み合わせる感覚に近い。プログラムを1行も書かなくても組み立てられる
ノード＝仕事の箱。
箱を並べて線で結ぶだけ。
これが基本の形！

⑤ トリガー — ワークフローの起点

  • ワークフローには必ず起点が必要。この起点になるノードをトリガーと呼ぶ
  • マニュアル：手動で実行する。作りかけのワークフローをテストする時に便利
  • スケジュール：決まった時刻に自動で動く。毎朝9時、毎週月曜など
  • Webhook：外部からの通知を受けて動き出す。LINEにメッセージが届いたら動く等
まず何をきっかけに動かすかを決める。
これが自動化の
出発点になる！

⑥ トリガーの使い分け

  • 「毎朝9時に動かしたい」→ スケジュールトリガーを選ぶ
  • 「LINEにメッセージが届いたら動かしたい」→ Webhookトリガーを選ぶ
  • 「今すぐテストしたい」→ マニュアルトリガーを選ぶ
  • 起点が決まらないとワークフローは動かない。トリガー選びが最初の設計判断になる
用途に合わせて
3つのトリガーを使い分ける。
最初に決めるべきこと！

⑦ n8nでできること：通知の自動化

  • 問い合わせフォームに新しい入力が来たら、その内容を自動でチャットに通知する
  • 見落としが減り、対応が早くなる。重要な連絡を即座に把握できる
  • Slack、Discord、LINEなど様々なチャットツールに通知を送れる
  • 24時間365日、人間が見ていなくても確実に通知を届けてくれる
地味だけど手間がかかる
定型作業こそ
自動化の第一歩！

⑧ n8nでできること：データの転記

  • フォームに集まった回答を自動でスプレッドシートに1行ずつ追加していく
  • 手動でのコピー&ペーストを繰り返す必要がなくなる。ミスも減る
  • Google Sheets、Airtable、Notionなど多様な先にデータを蓄積できる
  • データ入力の抜け漏れや転記ミスを防ぐことができる
コピペの繰り返し…
それn8nに任せよう！
ミスもゼロになる。

⑨ n8nでできること：定期処理

  • 毎朝決まった時刻に集計を取り、その結果を関係者へ自動で送る
  • 人が出社する前でも仕事が進んでいる状態を作れる
  • 週次レポート、月次集計など定期的なレポート作成も自動化できる
  • 関係者全員に同じタイミングで同じ情報を届けられる
朝出社したら
レポートが届いてる。

⑩ n8nの1番の得意分野

  • 人が毎回手でやろうとすると地味に手間がかかる定型作業こそn8nの最適なターゲット
  • 繰り返しの作業をノードの連鎖に置き換えて自動で回し続ける
  • 通知、データ転記、定期処理。どれも繰り返し作業の自動化という共通点
  • 一度作れば何度でも自動で実行してくれる。これが業務自動化の価値
定型作業の繰り返し →
ノードの連鎖に置き換えろ！
それがn8nの真骨頂。

⑪ 最終ゴール：LINE→AI→カレンダー

  • LINEで「明日の朝9時 歯医者」と普段通りの自然な文章を送る
  • n8nが受け取り、AIが文章の意味を読み取って予定の情報（日時・タイトル）に変換する
  • その情報をもとにGoogleカレンダーへ自動で予定を登録してくれる
  • カレンダーを開いて入力する手間なし。話しかけるだけで予定が入る仕組みが完成する
ただ話しかけるだけで
予定が入るんだ！

⑬ n8n Cloud — 手軽なクラウド版

  • 運営会社が動かす環境を貸してくれる有料サービス。登録すればすぐに使い始められる
  • 料金はワークフローの実行回数に応じた仕組みになっている
  • サーバーの管理や運用は運営会社が担当。利用者は自動化の構築に集中できる
  • 手軽さを重視するならクラウド版が便利。ただし実行回数による課金あり
サーバーの心配なしで
すぐ始められる。
ただし回数制限あり。

⑭ セルフホスト — 自分で立てる

  • コミュニティ版と呼ばれる無料版が公開されている。自分のサーバーに自分の手で立てて使う
  • サーバーの費用や運用手間は自分で負担するが、実行回数による課金はない
  • データを自分の手元で管理でき、自由度が高いのが魅力
  • このコースではより深く理解できるセルフホストを軸に進める方針
無料で回数制限なし。
データも自分の手元に。
深く学ぶならコレ！

⑮ Cloud vs セルフホスト — 選び方

  • Cloud：手軽・すぐ始められる・運用不要 / 実行回数による課金あり
  • セルフホスト：無料・回数無制限・データ自管理 / サーバー費用と運用手間あり
  • 料金やプランの詳しい中身は変わりやすい。次回じっくり扱う
  • 「自社で使う分には無料だけれど、それを再販するには条件がある」というフェアコードという独特なライセンスも紹介
どっちがいいかは
用途次第！
このコースはセルフホスト軸。

⑰ ノード間を流れるデータ — JSON

  • ノードとノードの間を流れるデータの形式はJSON（ジェイソン）という形式
  • 基本操作編でJSONの読み方をしっかり学ぶ。自動化の理解に欠かせない知識
  • 前のノードの出力結果がJSONとして渡され、次のノードがそれを受け取って処理を続ける
  • JSONの構造が読めれば、どこでデータがどう変換されているかが分かるようになる
{
  "message": "明日の朝9時 歯医者",
  "timestamp": "2026-06-17T09:00:00"
}
