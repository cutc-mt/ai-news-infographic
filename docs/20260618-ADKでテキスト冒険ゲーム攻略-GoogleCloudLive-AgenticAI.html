② Garden of Forgotten Prompts

  • Abdalと同僚Vitzが共同開発したテキストアドベンチャーゲーム
  • Zork風のクラシックなテキストベースゲーム
  • 教育的目的: エージェント開発を楽しく学べる仕組み
  • リーダーボード機能で競争要素を追加
  • Vitzがオランダ出身なのでゲーム内の地名がユニーク
名前の由来は「忘れられたプロンプトの庭」—AIのプロンプトをテーマにした世界観！

③ ADK（Agent Developer Kit）とは

  • Googleのエージェント開発キット — AIエージェント構築のためのSDK
  • 対応言語: Python Java Go TypeScript C# Kotlin
  • モデル非依存: OpenAIやClaudeなども使用可能
  • Geminiとは最適化されて動作
  • ツール統合・Web UI・デバッグ機能を内蔵
Kotlin版はまだベータ版。Pythonが最も安定して使える！

④ ゲームの4つのレベル構成

  • Level 0: 手動でプレイ — チュートリアル的役割
  • Level 1: エージェントで攻略開始
  • Level 2: より難易度が上がる
  • Level 3: 最も難しいレベル
  • 手動プレイはレベルが上がるほど困難に設計
  • Level 0を手動で体験後、Level 1からエージェントに切り替えが推奨
レベル0を手動でプレイすることで、ゲームの仕組みを理解できる！
手動プレイ体験

⑤ テキストアドベンチャーの操作コマンド

  • look — 周囲を見渡す
  • move — 北・南・東・西に移動
  • examine — オブジェクトを調べる
  • take — アイテムを拾う
  • use — アイテムを使う
  • drop — アイテムを捨てる
  • inventory — 所持品を確認
まるでダンジョンズ＆ドラゴンズのテキスト版！

⑥ Level 0: 腐敗したピクセルからの冒険

  • 開始地点で腐敗したピクセルを発見
  • examine → 物理的なデータキューブと判明
  • take cube → インベントリに追加
  • 北に移動するとファイアウォールが道を塞ぐ
  • debug toggleを使うとファイアウォールが消滅
  • サーバーノードで復号トークンとメモリアロケータを入手
「このファイアウォールはあまり安全じゃないね」— ホスト陣のツッコミも面白い！

⑦ Level 0: メモリリークを解決！

  • セキュリティゲートウェイにトークンを使って通過
  • 次の部屋でメモリリークが出現！
  • drop allocator → アロケータをプールに入れると解決
  • 南に進むとライセンスを取得
  • Level 0クリア！177ポイント獲得
「リアルのメモリリーク解消もこれくらい楽ならいいのにね」— 開発者なら笑えるジョーク満載！

⑧ リーダーボードの仕組み

  • 月次リーダーボード: 毎月リセット
  • 特別イベント: 今回は24時間限定で配信専用
  • 参加者にはランダムな表示名が割り当て
  • 自分の順位は青色でハイライト表示
  • 配信中からすでに「True Mind」がエージェントで猛追中！
非Googleツール（Claude Code等）でプレイすると5%ペナルティ！賢い設計！
ADKでエージェントを構築

⑨ Geminiへのアクセス方法

  • 方法1: Vertex AI — Google Cloudアカウント経由
  • 方法2: AI Studio — 無料APIキーを生成して使用
  • Vertex AIの場合: プロジェクトIDとリージョンが必要
  • リージョンはus-central1やglobalを指定可能
  • AI Studioはaistudio.google.comで無料登録
今回はVertex AIを使用。Cloud Shellなら認証済みで簡単！

⑩ Cloud Shellで開発環境を構築

  • Google Cloud Shell — ブラウザ内のVS Code環境
  • スペック: 16GBメモリ 2コア 5GBストレージ
  • 完全無料の仮想マシン
  • 注意: /home以外のファイルはリセットされる
  • 直接アクセス: shell.cloud.google.com
  • ローカル開発の場合はgcloud CLIのインストールが必要
Cloud Shellならツールが全部入ってるからセットアップゼロ！

⑪ UVでプロジェクト初期化

  • UV — Rustで書かれた超高速Pythonパッケージマネージャー
  • 仮想環境を自動管理、pip互換
uv init google-live
cd google-live
uv add google-adk
  • pyproject.tomlに依存関係が自動追加
  • 既存プロジェクトなら uv add -r requirements.txt で移行可能
UVは爆速！pipの完全書き直しだから体感で速さが違う！

⑫ ADKエージェントの初期化

  • コマンドでウィザードを起動:
uv run adk create garden_of_forgotten_prompts
  • モデル選択: Gemini 2.5 Flash（安定動作）
  • 3.5は指示に忠実でないことがあるので注意
  • フォルダ名はアンダースコア必須（Pythonの命名規則）
  • agent.pyがエージェントのエントリーポイント
ハイフンを使うと「有効なPython識別子でなければならない」とエラー！ライブ配信でハプニング！
OpenAPIツールとエージェント実行

⑬ OpenAPI Tool SetでゲームAPIを統合

  • ゲームのバックエンドがOpenAPI仕様で公開
  • ADKにOpenAPI Tool Setが組み込み
  • API仕様を読み込むと、エージェントが自動的にAPIの使い方を理解
  • 認証情報は.envファイルにゲームAPIキーを設定
  • Bearer認証もライブラリ内蔵で簡単対応
OpenAPI仕様を渡すだけでエージェントがAPIを理解！これがADKの強み！

⑭ ADK Web UIでエージェントをデバッグ

  • ADKにはWebデバッグUIが組み込まれている
uv run adk web \
  --reload-agents \
  --allow-origins '*'
  • --reload-agents: コード変更時に自動リロード
  • --allow-origins: CORS設定（開発用）
  • ポート8000でWeb UIが起動
  • 複数のエージェントを一度に管理可能
--allow-origins '*'は本番環境では使わないこと！

⑮ エージェントがゲームを自動プレイ！

  • プロンプト: 「solve level one」
  • エージェントが自動でlook → take → use → moveを繰り返す
  • 手動では数分かかるゲームを数十秒で攻略
  • ADK Web UIでツール呼び出しをリアルタイム確認可能
  • エージェントはスパークライターや信号弾を活用して暗闇を突破
エージェントが自分で考察して道具を使う姿は圧巻！非決定的システムの面白さ！

⑯ スコアシステムと最適化のコツ

  • ベーススコア: クリア時の基本ポイント
  • タイムボーナス: より速くクリアすると高得点
  • トレジャーハンターボーナス: 好奇心を報酬！探索すれば高得点
  • ADKボーナス: ADK使用で+5%マルチプライヤ
  • マイクロリワード: look・move・take・use・examineの実行で加点
  • 注意: 同じレベルを再プレイするとスコアがリセット
プロンプトに「もっと好奇心を持って探索して」を追加するとスコアが跳ね上がる！
Agent Platform & Q&A

⑰ Google Agent Platformとは

  • Agent Studio: プロンプトだけでノーコードでエージェント構築
  • マーケットプレイス: 既存のエージェントをデプロイ可能
  • Agent Registry: A2Aプロトコルでエージェントを登録・共有
  • Agent Identity: エージェント間の認証・セキュリティ
  • MCP Serverのデプロイにも対応
  • デプロイ先: Cloud Run または Agent Platform
Vertex AIの名称が「Enterprise Agent Platform」に進化中！
