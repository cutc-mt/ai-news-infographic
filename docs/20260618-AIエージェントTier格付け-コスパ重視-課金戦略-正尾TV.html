【2026年6月版】AIエージェント Tier 格付け！

コスパ重視でどれに課金すべきかを徹底解説
2026年6月17日配信 | 正尾TV

① Claude Code: ゴッドランクのエース

GOD RANK
  • Opus 4.8をオーケストレーターとして採用。マルチエージェントの流れが圧倒的
  • 汎用エージェントやハーネスエンジニアリングで他の追随を許さない魅力
  • Opus 4.8はステルスで品質低下の噂もあるが、それでもゴッドから落とすことはない
  • エージェントSDKの制限は延期になり、当面は使い続けられる環境
ハーネスエンジニアリングを
極めるとClaude Codeの
真価がわかる！

② Codex: 地に足ついたゴッドランク

GOD RANK
  • GPT 5.5を搭載。堅実さと賢さで無難に推奨できる
  • $20プランから始められる。初心者に最初に触らせるならコデックスがおすすめ
  • Computer Useにも対応。デスクトップアプリとしても優秀
  • 確実に取るところ取りに来ているという印象。開発効率の基盤となる
どっちから始める？
迷ったらまずコデックス！
ハーネス極めたらClaudeへ

③ ゴッドランク比較: 思想の違い

  • Claude Code: マルチエージェント主体、オーケストレーションが得意
  • Codex: 単体モデルとしての賢さ、Computer Use、堅実さが武器
  • コスパ重視ならClaudeをオーケストレーター、GPTをワーカーにするフュージョン構成もあり
  • プログラミング未経験者にはコデックスが入りやすいが、最終的には好みの問題
ぶっちゃけどっちでもいいけど
ハーネスの深さで選ぶのが正解！

④ Cursor: Sランクの実力派

S RANK
  • Composer 2.5（Qwen 2.5ベース）が登場。Qwen 2.6より使い勝手が良い
  • コンポーザーのレスポンス速度が速い。オートモードで十分使わせてくれる
  • ハーネス自体が非常に優秀。いち早く力を入れていた歴史がある
  • GitHub Copilotは制限が大きくなり評価を下げた。Cursorが座を確保
Qwen 2.7が出ても
Composer 2.5の方が
使いやすい不思議！

⑤ OpenCode 5: S+のコスパ最強

S+ RANK
  • $5/月からスタート。$10で約$60分使えるコスパの最強サブスク
  • Qwen 2.6がTPS 200+で高速動作。DeepSeek FlashやM2.5 Proも対応
  • フェイブルモデルの流行により、速さがより重視される時代に最適
  • CursorのComposer 2.5を使わないなら、OpenCode 5の方がコスパが良い
これめっちゃおすすめ！
まず抑えておいて損なし

⑥ OpenCode 5の賢い使い分け

  • マクロハーネス: 人間の認知負荷を軽減する汎用エージェント用途に最適
  • Opus 4.8は日本語が不安定、Codexは硬い → Qwenをフロントに置く戦略
  • コンテキスト整理や出力フォーマットなど、人間寄りのタスクにOpenCode 5が適任
  • 特定用途に絞れる点がCursorより高評価。$5〜$10の幅広い選択肢
メインはゴッドランク課金＋
サブ用途にOpenCode 5
これがベスト構成！

⑦ Grok Build: Aランクの期待株

A RANK
  • 速度感が抜群。Grok系モデルでハーネスもしっかりしている
  • XのサブスクでHermesも使える。xAI系のサービスがお買い得に
  • CursorのComposer 2.5と比べると少し劣るが、今後モデル進化で期待大
  • X未契約ならわざわざGrok Buildを選ぶ必要はない
Xサブスク済みなら
実質無料で使えるから
評価は高め！

⑧ Antigravity: Bランクの不確定要素

B RANK
  • 使用制限が厳しい。Xサブスク経由だが使用量に特徴がない
  • まだ不安定な部分があり、実戦投入には少し早い印象
  • IDE版やデスクトップアプリが嬉しいが、トレンドはターミナル経由・ループエンジニアリングへ移行
  • 画面を見ないループ回しの流れに沿ってBランクに評価
IDE版いいけど...
トレンド的には
もうループエンジニアリング！

⑨ トレンド: ループエンジニアリング

  • コーディングエージェントはターミナル経由が主流に。画面を見ない開発手法が定着
  • ループエンジニアリング: エージェントが自律的にコードを書いてテストするサイクル
  • クロードデスクトップやCodexのアプリから何でも作れる時代に
  • ワークフロー系（n8n、Dify等）はDランクまで低下。エージェントが代行するようになった
画面を見ない開発...
最初は怖いけど
慣れると戻れない！

⑩ Kimi: A+のオープン系最強

A+ RANK
  • オープン系の中でClaudeソネットクラスの実力に到達。モデル自体がかなり良くなっている
  • Qwen 2.7 Codeで6倍高速のハイスピードモード提供。開発効率に直結
  • マルチエージェント対応もいち早く強化。スライド等のハーネスも充実
  • Kimi Work（デスクトップアプリ）やKimi Code（コーディングエージェント）も登場
Kimiはオープン系のClaude
的な立ち位置に成長中！
速度もすごいよ

⑪ MiniMax: ボイス＆ミュージック特化

B+ RANK
  • $20プランでかなり使わせてくれる。サブスクとしてお買い得
  • Voice GenerationとMusic Generationが他社と一線を画す
  • GrokやMiniMaxは画像・動画生成で、ボイスとミュージックの分野で差別化
  • M3モデルで大規模トークン数に対応。日本語もある程度改善
画像APIも引っ張ってこれる
サブスクリプションとして
ユニークな選択肢！

⑫ GLM: 値上げでCランクに転落

C RANK
  • GLM 5.2登場。Claude Codeの裏側でも使われる実力派
  • しかし価格が2倍に値上げ。一時$8だった年間プランが大幅騰貴
  • 並列制限も厳しく、人が増えると重くなる・エラーも増える傾向
  • OpenCode 5にGLM 5.2が来るのを待つのが今の最適解
コスパバグってた時は
激安だったのに...
今はちょっと高いな

⑬ Coder: Alibaba系のAランク候補

A RANK
  • 様々なサービスに多数展開。Claude Code風CLIも登場し話題に
  • Qwen 3.7 MAがサブスク内で使い放題。これだけで十分な価値
  • デスクトップアプリやCLIも全部揃ってくる流れ。オープン系の有力株
  • Kimi・Coder・MiniMaxの中国系勢が次世代の選択肢として急成長
フロンティア系モデルが
使えなくなったら
ここに来る！

⑭ Devin: Cランクの重量課金

C RANK
  • 重量課金システムに移行。個人で使い続けるにはコストが高すぎる
  • 企業向けにはC+〜B程度の評価。スマートな部分はある
  • レビュー機能は良かったが、他の選択肢が増えたことで相対的に魅力低下
  • 正直めっちゃ応援しているが、現状はCの上くらいが精一杯
応援したいけど
コスパ的に厳しい...
企業案件ならアリかも

⑮ 除外・低評価サービス群

  • n8n / Dify: ワークフロー系はDランク。エージェントが自動化を代行
  • Manas / Skywork: 入門者向けだがD〜Eランク。頑張って上のランクへ行こう
  • Google Workspace系: 人による評価が分かれるため除外判定
  • GitHub Copilot: 制限が大きくなり、かつての地位から低下。今後再評価の余地あり
もうエージェントが
何でも作っちゃうから
ワークフロー系は不要に

⑯ 今後の懸念: フェイブルモデルの影響

  • フェイブルモデル（軽量高速モデル）の流行でサブスクの使われ方が変化
  • フロンティア系モデルへのアクセスが今後制限される可能性がある
  • OpenCode 5のサブスクやKimi・Coderが使えなくなったらMiniMax・Coderが次の候補
  • 複数のサブスクを使い分けるリスク分散が重要になる
1つに依存せず
複数サブスクで
リスク分散するのが安心！

  • メイン: Claude Code または Codex（$20〜）。ゴッドランクは必須
  • サブ: OpenCode 5（$5〜$10）。マクロハーネスや汎用エージェント用途に
  • オプション: XサブスクでGrok Build + Hermesをカバー
  • 保険: Kimi / Coder / MiniMax。フロンティア系が使えなくなった時のバックアップ
月$25〜$35程度で

  • ゴッドランク（Claude Code / Codex）は絶対外せない。最初に課金するべき
  • S+のOpenCode 5は隠れた最強サブスク。初月$5で試せる
  • オープン系（Kimi・Coder・MiniMax）は次世代の切り札として要注目
  • デスクトップアプリやCLIを全て揃える流れが加速。汎用エージェント系のサブスクは不要に
