白クラウドチャンネルによるAIエージェント活用シリーズ第2弾。今回はAIエージェントがローカルPCのWindowsアプリを自動操作するスキルを開発する。前回「CLIエニシング」を試して「すげえじゃん」と感じた続き。

CLIエニシング＝人間用アプリをAIが使えるようにするコンセプト！

②VOICEVOXとは: 無料の音声合成ソフト

VOICEVOXはズンダもんや四国めたんなどのキャラクターで音声合成ができる無料アプリ。Windows/macOS/Linux対応。HTTPサーバーを使ったAPI呼び出しも可能だが、今回はあえてエディターアプリをAIに操作させるアプローチをとる。

  • ズンダもん・四国めたんなど多彩なキャラ
  • GUIエディター形式のWindowsアプリ
  • HTTPサーバーAPIもあるが今回はエディター経由

③CLIエニシングのコンセプト: GUIアプリをAIに操作させる

人間が使うためのGUIアプリを、CLI（コマンドライン）化してAIエージェントが操作できるようにするというのが根本的なアプローチ。VOICEVOXエディターを自動化し、自然言語で指示を出すだけで音声合成ができるようになる。

「ズンダもんの口調で自己紹介して」→AIが自動で音声生成！

④3つのスキル構成: 連携でプレゼン動画を完成

今回は以下の3つのスキルを組み合わせて、ホワイトボードアニメーション付きのプレゼン動画を自動生成する。

  • スキル① 手書き風ホワイトボードアニメーション
  • スキル② VOICEVOX音声合成CLI
  • スキル③ キャラクター口パクアニメーション

⑤スキル①: 手書き風ホワイトボードアニメーション

このスキルは以前Codexで開発済みのものをClaude Codeに移植しただけ。手書き風のホワイトボードアニメーションをHTMLで生成し、動画化する機能。既に完成済みで安定動作している。

別動画でリンクあり→Codexで作成済み→Claude Codeに移植済み

⑥スキル②: VOICEVOX CLIの開発手順

VOICEVOXのGitHubリポジトリURLと全体構成MDをAIエージェントに提示して開発を依頼。モデルはOpus 4.8を使用し、ソフトウェア解析とCLI開発を担当させる。エディターアプリをCLI経由で操作できるようにする。

  • モデル: Claude Opus 4.8を使用
  • GitHubリポジトリ情報を添付
  • 全体構成MDも参照資料として提供

⑦Claude Codeスキル開発の流れ

CLI開発の依頼を出してからスキル完成まで予想外に早かった。Opus 4.8を使ってVOICEVOX CLIを作成し、その後Claude Code Desktopで使えるスキルとして統合まで一気に実行。

CLI完成→スキル化まで一気に完了！思ったより早い！

⑧動作確認: ズンダもんの声が生成された！

Opus 4.8でスキル開発後、そのまま動作確認を実施。「ズンダもんの口調で自己紹介して」と自然言語で指示すると、AIが自動でVOICEVOXを操作してセリフを生成。枝豆の枝豆、単なるテストだけど3部構成の自己紹介文まで作ってくれた。

「ズンダ餅ちってね、枝豆をすりつぶして作るんだよ。とっても美味しいんだよ」

⑨スキル③: 口パクアニメーションスキルの開発

新規セッションでキャラクターの口パク・目パチパチアニメーションスキルを開発。しかし予想外の事態が発生。ズンダもんをダウンロードするのではなく、著作権を考慮してオリジナルキャラクターを自作してしまった！

「なんやこれ？なんでオリジナルキャラ作ったんだろう？普通にダウンロードしてくれりゃよかったのに」

⑩偽ズンダもんの誕生: 思わぬオリジナルキャラ

「待って、ここまで作ったってことは色々表情が動くのか？」と途中で気づく。著作権配慮でオリジナルキャラが作られたが、目パチパチと口パクパクの両方に対応。髪の毛は枝豆をイメージしたデザイン。結局そのまま続行することに。

  • 目パチパチアニメーション対応
  • 口パクパクのリップシンク対応
  • 枝豆風の髪型デザイン

⑪重要な気づき: 参考画像でオリジナルキャラを作れる

この偶然から参考キャラクター画像やデザイン指示を提示すれば、オリジナルキャラを作成してくれることが分かった。著作権を気にしつつ、好みのデザインでキャラクターを生成できる可能性が示された。

参考画像＋「このイメージで」→オリジナルキャラが自動生成される！

⑫最終テスト: 3スキル連携で動画生成

3つのスキルを組み合わせて短いプレゼン動画を生成。「まずナレーション音声を作り、長さを測ってからホワイトボードの尺を合わせる」という適切な手順をAI自身が立案。VOICEVOXはボイスボックスと読ませるため発音テキストをカタカナにする工夫も。

  • 手順1: ナレーション音声をVOICEVOXで生成
  • 手順2: 音声の長さを測定（18.8秒）
  • 手順3: それに合わせてホワイトボード動画を生成

⑬AIの自己検証まで自動で実行

動画生成後、AIが自ら「ホワイトボードに右下ワイプの口が揃っているか確認」を実行。Opus 4.8は失敗しないモデルらしく、手順の計画から実行、最終確認まで一貫して高い精度でこなした。18.7秒の動画が完成。

「確認もちゃんとしてたっぽい」→自律的に品質チェックまで！

⑭完成品の質: プレゼン動画が見事に完成

完成した18.7秒の動画を見て「マジか。めっちゃいいじゃん。完成度高いね」と驚き。偽ズンダもんがナレーションに合わせて目パチパチ・口パクパクし、ホワイトボード動画の右下に配置されたワイプ映像がシンクロしている。

「こんなになるんだ。すっげえやべえな。マジで」→本人も驚きのクオリティ！

⑯スキル品質の検証: モデルランクを下げてテスト

開発したスキルの完成度を測るために、あえてモデルランクを下げてSonnetでテストする計画。Opusでできること Sonnetでどこまで再現できるかが、スキルそのもののポテンシャルを示す。18秒の動画はOpusで余裕だが、長尺動画での検証が必要。
