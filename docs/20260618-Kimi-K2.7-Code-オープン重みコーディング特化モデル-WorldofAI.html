モデル概要とアーキテクチャ

② 巨大なアーキテクチャ

  • 約1兆パラメータのMixture of Experts構成
  • 384エキスパートからトークンごとに8つを選択
  • 推論時の活性化パラメータは32Bに収まる
  • 61層構造（1つのDense層を含む）
  • MoonViTビジョンエンコーダーで4億パラメータ追加
1兆パラメータ全体のうち、推論で使うのはわずか32Bだから効率は悪くない！

③ コンテキストウィンドウ

  • コンテキスト長は262,144トークン（約262K）
  • 前世代K2.6の256Kからわずか6K増
  • 2026年の1兆パラメータモデルとしてはやや不満足
  • それでも長いセッションの履歴保持には十分実用的
  • ツール定義やコードベース全体を含むエージェントセッションに対応
1兆パラメータなのに262Kしかないのは正直物足りない…K3.0で大幅拡大に期待！

④ 指示追従性の向上

  • 指示へのコンプライアンスが大幅に改善
  • 「考えすぎ（Overthinking）」傾向が平均30%削減
  • 長文脈のコーディングワークフローで性能向上
  • より少ない推論ステップで正確な出力を生成
  • 複雑なマルチステップタスクでの精度が向上
「考えすぎ」が減る＝コストダウン＆高速化。実用的にはかなり嬉しい改善だね！
