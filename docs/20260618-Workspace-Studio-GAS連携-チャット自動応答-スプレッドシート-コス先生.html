  • 今後のアップデートで変わる可能性はある
  • 実務では共有スプシを使いたい場面が多い

⑤解決策: GASで中継パターン

  • WS Studioは非共有スプシ（マイドライブ）に書き込む
  • その変更をGASの変更トリガーで検知
  • GASが共有スプシにデータを転記する
  • 2つのスプシはレイアウトを全く同じにしておく
  • 結果的にWS Studioが共有スプシに書き込んだのと同じ状態に
WS Studio単体では無理でも
GASの中継で疑似的に実現！
「住み分け」が鍵だね

⑥中継パターンの流れ

  • WS Studio → 非共有スプシに行追加で書き込み
  • GAS が変更トリガーで起動
  • GASが共有スプシにデータを転記
WS Studio → [非共有スプシ] → GAS → [共有スプシ]

⑦課題②: 同一チャットに返信不可

  • 「チャットメッセージを受信」はスペース指定可
  • しかし「チャットで通知」はスペース指定不可
  • 受信したスペースではなく、WS Studioと同名のアプリスペースに返される
  • 同じスペースに返すことが現時点ではできない
  • 今後「チャットスペースに送信」ステップが追加される可能性あり
