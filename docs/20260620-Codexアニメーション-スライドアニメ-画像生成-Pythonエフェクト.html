Codexでアニメーション作成！

スライドアニメから漫画動画化まで — Codex単体でどこまでできるか

① Codex単体でアニメを作る挑戦

  • ワンダー佐藤さんがCodex単体でどこまでアニメーションが作れるかを実験的に検証
  • Rev 2.0は使わないという条件で、Codexの capability の限界に挑戦
  • 画像生成（GPT Image）× Python のエフェクト処理を組み合わせた独自のパイプラインを構築
  • スライドアニメ（紙芝居風）と漫画のアニメ化、2つの実演を通じて可能性を示した
動画生成AIは6〜10秒程度が限界。Codexは何分でも一気に生成できるのが最大の強みです！

② 現実的なアプローチの選択

  • 滑らかなアニメには大量の画像生成が必要で、時間もコストもネックになる
  • そこで紙芝居風のスライドアニメ × Python でエフェクト追加という現実的な道を選択
  • LumaやGeminiなどの動画生成AIは6〜10秒程度が限界
  • 一方 Codex は数十秒〜数分の動画を一発で生成できる強みを持つ
フルアニメではないけれど、
この組み合わせが一番現実的で
品質も高いんです。

③ 制作パイプラインの全体像

  • Step 1 ChatGPT でセリフとコンテ（絵コンテ）を作成
  • Step 2 GPT Image でシーンごとの画像を生成する
  • Step 3 Codex にコンテと画像を投入して動画を生成
  • Step 4 Python でキラキラ光などのエフェクトを追加する
  • Step 5 VoiceMochi でナレーション音声を合成して合成
全部ChatGPT × Codex × Pythonで完結。
外部ツールほぼ不要の手軽さ。

④ GPT Imageが画像を担当

  • アニメの各シーンの画像をGPT Imageで一枚ずつ生成していく
  • 海の風景アニメのシーン例：空の風景 → 移動の shot → 坂道 → 海 → クローズアップ → 足元の水 → 海を見る姿
  • 各シーンは静止画だが、Ken Burns 効果でズームイン・アウトで動きを演出する
  • 画像そのものは動かさず、カメラワーク的な動きだけでアニメーション感を生み出す
画像は動かさなくても、
Ken Burns効果だけで
十分に映画的な印象を与えられます。

⑤ Ken Burns効果で映画的に

  • Ken Burns効果とは、静止画に対してゆっくりとズームイン・ズームアウトをかける映像技法
  • 風景写真を少しずつズームしていくことで、まるでカメラが動いているような演出ができる
  • パン（左右への移動）と組み合わせれば、より立体的で動きのある動画になる
  • Codex が Python でズーム動作の timing を計算して自動的に適用してくれる
ドキュメント番組でよく使われる
テクニックをAIが自動制御してくれる
のはすごい進化です。

⑥ Pythonでキラキラ光エフェクト

  • Python の画像処理でキラキラと光るパーティクルエフェクトを動画に重ねる
  • 水面のきらめきや逆光の演出に効果的。静止画が一気にファンタジー感を帯びる
  • エフェクトの強さや timing はコードで細かく調整できるのが強み
  • Codex が指示通りにエフェクトのパラメータを設定して、一発で40秒の動画を生成
コーディングでエフェクトを追加できる —
これがCodexの独自の強みです。
他の動画生成AIにはない能力。

⑦ VoiceMochiでナレーション合成

  • VoiceMochiを使って恋するような声のナレーション音声を合成する
  • 海の風景に合うような、少し切なくて優しいトーンの声を選択
  • シーンの切り替わり timing に合わせて音声を配置していく
  • 結果として40秒の動画を1発で生成することに成功した
音声合成もAIで完結。
声のトーンや感情まで
コントロール可能な時代に。

  • 各シーンをKen Burns効果で繋ぎ、Pythonのキラキラ光エフェクトを重ねて完成
  • 全体で40秒の動画。画像生成からエフェクト追加まで一発で出力
  • 紙芝居風とはいえ、映画的な雰囲気がしっかり出ていて驚きのレベル
40秒の映像素材を

⑨ 漫画「レオハルト」をアニメ化

  • NotebookLMで書いたオリジナル漫画「レオハルト」をアニメ化に挑戦
  • 物語は未来の仮想空間を舞台に、AIに時間を巻き戻されるというSFテーマ
  • キャラクターの3面図（正面・横・背面）を設計図としてCodexに渡す
  • 設計図を参考にしてCodexが各シーンの画像を生成し、アニメとして組み立てる
3面図を渡すとキャラの
一貫性が保てる —
キャラ設計の重要性を実証。

⑩ キャラクター3面図の役割

  • 3面図とはキャラクターの正面・横顔・背面を描いた設計図のこと
  • これをCodexに渡すことで、シーンごとに同じキャラを一貫した姿で生成できる
  • 設計図がないと、シーンごとにキャラの顔や服装が変わってしまう問題が起きる
  • 3面図はアニメ制作の基礎情報として、人間の制作現場でも欠かせない
人間のアニメ制作の知見を
AIに応用する —
まさにハイブリッド制作。

⑪ 画像は綺麗、動きはまだ課題

  • 画像そのものの品質は非常に綺麗に生成できている
  • しかし動きの部分はまだ微妙。画面が意図せず揺れたり、不自然なトランジションが入る
  • ディルブ（dissolve）と呼ばれるフェードの合成が過剰に入ってしまう問題
  • 画面の揺れや不適切なトランジションの制御が難しいという課題が浮き彫りに
Rev 2.0を使わない分、
品質に限界があるのも事実。
でも今後の改良で改善の余地あり。

⑫ リップシンク（口パク）の試行

  • キャラクターのリップシンク（口の動きと音声の同期）も試した
  • しかし「レオハルト」は心の独白がメインのシーンなので、口パクは不要という判断に
  • 独白シーンなら口が動かなくても自然。逆に静けさの演出として機能する
  • 対話シーンにはリップシンクが必要になるので、今後の課題として残されている
作品のテイストに合わせて

大事なポイントです。

⑬ Codexの3大強み

  • 強み① コーディングでエフェクトを自由に追加。キラキラ光、ズーム動作などPythonで柔軟に制御
  • 強み② 数十秒〜数分の長尺を一発生成。短いクリップのつなぎ合わせが不要
  • 強み③ 修正指示を出すだけでバージョン2、3を継続的に改良できる

⑮ 現状の課題と限界

  • 課題① 滑らかなアニメーションには大量の画像生成が必要。時間とコストがかかる
  • 課題② 画面揺れやディルブ（過剰なフェード）の制御が難しい
  • 課題③ Rev 2.0を使わない分、他の動画AIと比べて品質に限界がある
  • 手動での微調整や複数回の修正指示が必要な場面も少なくない
完璧ではないけれど、
コーディングで改善できる
余白があるのがCodexの良さ。

⑭ 継続改良ができるワークフロー

  • v1 一発目で動画を出力。まず全体の構成を確認
  • v2 「画面の揺れを抑えて」「ズームをゆっくりに」など修正指示を出す
  • v3 さらにエフェクトの微調整や timing の修正を行う
  • このように会話ベースで段階的に品質を向上させていくのがCodexの強み
