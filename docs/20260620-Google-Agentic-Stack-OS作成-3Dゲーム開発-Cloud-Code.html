① コーディングは長時間ワークロードの典型

  • 本物のソフトウェアは単一プロンプトでは完成しない。依存する問題の連鎖があり、各ステップで予期せぬ失敗が起きる
  • チャットボット形式のコーディングは、最初の深刻なバグで失敗しがち。難問にぶつかると諦めてしまう
  • 解決してほしいのは自己修正して先に進み続ける能力。ハルシネーションで場当たり的に直すのではなく、ゴールまで走り続けること
  • まさにこの「長時間エージェント」の実証こそが、今回のデモの主題
コーディング＝長時間実行の
代表的なワークロード。
この事実こそが出発点です。

② Cloud CodeのGoal Primitiveが鍵

  • 使用したのはCloud Code（Googleのコーディングエージェント）。他のエージェントも好きだが、今回はこれを選択
  • Cloud CodeのGoal Primitiveが決定的な違い。ゴールを一度設定すれば、ターンを重ねて自律的に作業を続けられる
  • ゴールが満たされるまで止まらない。途中で失敗しても自己修正して継続する設計
  • この「長時間実行エージェント」こそが、複雑なタスクをこなす次世代のコーディングパラダイム
Goal Primitiveでゴールを設定。

③ プロジェクト「Adios」— ブラウザ上のOS

  • Cloud Codeに仕様書を1つだけ渡してスタート。エゴ的な理由から「Adios」と名付けたとジョーク
  • ウィンドウマネージャーを搭載。複数アプリを並べて操作できる
  • IndexedDBをファイルシステムとして活用。ブラウザ内で永続的なデータ保存が可能
  • ターミナル、ファイルエクスプローラー、テキストエディターなど、OSに必要な要素を一式そろえる構想
「もっと良い名前があるはず」

  • 新しいドキュメントの作成も保存もできる。パス表示や文字カウントなども実装されている

⑩ 156MBのBlenderファイル — 出発点の壁

  • ウェブ上から見つけた3Dアセットを出発点にしたが、156MBという巨大なファイルだった
  • ジオメトリ、テクスチャ、マテリアルが全て含まれている。ブラウザで表示するには到底許容できないサイズ
  • ブラウザでのロードは5〜10MBまでが理想。ユーザーを長時間待たせるわけにはいかない
  • この巨大ファイルをどう最適化するか。それがエージェントが解くべき最初の壁
156MB → 5〜10MBに。
30分の1以下に圧縮する
という超難題からスタート。

⑪ エージェントが解くべき3つの技術課題

  • ① エクスポートパイプライン — BlenderのDraco量子化を異なるメッシュとトランスフォームで壊さずに処理する必要があった
  • ② 最適化パイプライン — glTFファイルの圧縮、テクスチャのリサイズ、画像変換を、見た目を損なわずに実行
  • ③ ライティング — Blenderのエリアライトは極端な強度でエクスポートされる。ブラウザでもビデオストアの雰囲気を再現する必要があった
  • これらはどれもスキップできない連鎖する問題。1つ解いても次で失敗すれば全体が崩れる
エクスポート→最適化→ライティング。
全て繋がった課題で
1つも飛ばせない。

⑫ エクスポートパイプライン — Draco量子化の罠

  • BlenderからglTF形式へエクスポートする際、Draco量子化がジオメトリを破損させる問題に直面
  • 異なるメッシュ、異なるトランスフォームが混在するシーンでは、標準のエクスポートだけでは対応できない
  • エージェントはPythonスクリプトを自作してこの問題を解決。Blender APIを活用したカスタムエクスポート処理を書いた
  • Blenderの専門家ではないプレゼンターですら、エージェントの力でこの壁を越えられた
Draco量子化でジオメトリが
壊れる → Pythonで修正。
エージェントが自ら解決策を編み出す。

⑬ 最適化パイプライン — 見た目を守りながら圧縮

  • 156MBの巨大ファイルをブラウザで遅延ロードできるサイズに分割・圧縮するパイプラインを構築
  • テクスチャのリサイズ、画像の圧縮、フォーマット変換を一括で処理。ただし品質を維持することが絶対条件
  • glTF形式のアセットを管理。個別のファイルとして必要なものだけ読み込む仕組みを実装
  • 圧縮率と画質のバランスを自動で調整。エージェントが「これなら許容範囲」と判断しながらパラメーターをチューニング
サイズを減らしつつ
見た目を綺麗に保つ。
難しい二律背反をエージェントが解決。

⑭ ライティング — ビデオストアの雰囲気を忠実に

  • Blenderのエリアライトは極端な強度でエクスポートされる。そのままではブラウザで見た目が破綻する
  • ビデオストア特有の暖かい照明の雰囲気を忠実に再現したい。蛍光灯とカウンターの光の加減が重要
  • マテリアルカラーの補正、ミックスシェーダーの調整。エージェントがBlenderの元のシーンに近い見た目を目指して微調整
  • 単なる技術的な変換ではなく、感情的な雰囲気の再現まで要求されている点がこのタスクの難しさ
光の強さ、色、影。
ノスタルジーの雰囲気は
ライティングで決まります。
