⑬Claude CodeのデザインシステムをClaude Designへ

  • Claude Codeで作ったウェブサイトのデザインシステムをアップロード
  • デザインシステム化されたコンテンツのみが同期対象
  • 未対応のサイトはClaude CodeがReactコンポーネント化を提案
  • アップロード後に同じデザインのコンテンツを量産可能
1つのデザインシステムから
複数のページを一気に量産できる！

⑭連携コマンド: /design-loginと/design-sync

  • /design-loginでClaude Designと認証
  • /design-syncでデザインシステムを同期
  • 認証フロー: URL → ブラウザで承認 → 認証コードをコピー → Claude Codeに貼り付け
  • 認証が完了すれば、同期コマンドが使える
/design-login   → Claude Designと連携認証
/design-sync    → デザインシステムをアップロード
スラッシュコマンドを打つだけ！
コマンド1つで連携が完了する

⑮認証フローの手順: 3ステップで連携完了

  • Step 1: プロジェクトフォルダーで /design-login を実行
  • Step 2: 表示された長いURLをブラウザで開いて承認
  • Step 3: 認証コードをコピーしてClaude Codeに貼り付け
  • 認証完了後は /design-sync で同期スタート
3ステップだけだから、
初めてでも迷わないはず！

⑯Claude Design→Claude Code: デザインをコードに同期

  • Claude Designのシェアボタン → Sendタブ → Claude Codeを選択
  • 表示されたコードをコピーしてClaude Codeに貼り付け
  • プロジェクトフォルダー内にデザインデータが出力される
  • デザインを引き継いで実際の開発・実装に進められる
デザイン完成 → コード生成 → 実装
この流れが一気通貫でできる！

⑰ローカルとクラウドの同期: 双方向でデータを管理

  • ローカルデータ: プロジェクトフォルダー内のデザインデータ
  • クラウドデータ: Claude Design側のデータ
  • ローカルを変更 → 「クラウドにプッシュして」と指示
  • クラウドを変更 → 「クラウドの内容をローカルに反映して」と指示
双方向同期だから、
どっちで作業してもOK！

⑱注意点: 実装直後のためまだ不安定な面も

  • 連携作業中にエラーが出ることがある
  • エラーが出たら /design-login で再度ログインが必要
  • 実装されたばかりの機能なので不安定な面がある
  • 今後のアップデートでの安定化が期待される
新機能だから仕方ない！
