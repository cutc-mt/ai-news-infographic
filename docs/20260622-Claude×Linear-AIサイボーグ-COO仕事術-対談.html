ClaudeをLinearに住まわせる

AIサイボーグに近づくCOOの仕事術 — タスク管理の未来を探る対談
2026年6月22日

① 長津こ介さん: クリエイティブ業界のCOO

  • 株式会社「部屋」のCOO。2002年から24年間、クリエイティブ業界で活動
  • 会社の主な事業はクリエイターのエージェントとWeb制作。映像系からスタートした経歴
  • 業界で一番最初にAI化に取り組んだ自称。AIネイティブ組織づくりを推進中
  • 「エージェンティックAIの模索会」を主催。尖ったAI実践者たちを集めたコミュニティ
1981年生まれの45歳。
95年頃にインターネットと出会い
テキストサイトやFlashをやった世代。

② 45歳世代がキラキラしている — 互联网再来

  • 昔の友達からLINEが来て、久々に会うことが増えた。みんなAIが面白いと語り合っている
  • 1995年頃に高校生でインターネットに触れた世代。当時のワクワク感と同じ感覚
  • みんなClaude Codeで朝から晩まで喋っている状態。AIの実験に没頭する友達が急増
  • 当時インターネットが出てきた時と同じ感覚。テキストサイト、Flash、映像系をやった世代が再び熱狂
互联网ブームの再来！
30年ぶりの革命に
45歳世代が一番ワクワクしてる

③ AIで生産性が100倍に — 組織のシップが増大

  • AI企業は超高速リリースを続けている。Anthropicなら週34本の新機能
  • 個人のアウトプットが100倍になったと感じている。組織でまとめれば1000倍に
  • 会社全体として1万倍になっているイメージ。個人の能力向上がシップを増やしている
  • AIネイティブ組織の名題は「個人のアウトプットをどこまで最大化できるか」
このスピード感は
AI企業だけでなく一般企業も
求められるようになる！

④ タスク管理は人間には無理ゲー

  • タスク管理は人間の認知能力の限界を超えてしまった。遂行されず腐っていく
  • 「このタスクって何の意味があったんだっけ？」と忘れてしまう。目的を見失う
  • 自分だけのルールでタスクを遂行。「つかやる」を一生やらない問題
  • 上司との報告を丸投げ。リーダーは判断を先送り。理解できないから「やめましょう」になる
人間に依存しない
タスク管理が求められる。
AIの判断で解決しよう！

⑤ Linear × Claude Code: 対話がそのまま課題に

  • 社員全員がClaude Codeを使い、タスク管理にLinearを使用
  • Claude Codeで会話するだけで、内容がそのままLinearの課題（イシュー）として自動生成される
  • 生成された課題は高品質。テンプレートに従い、実行可能なタスクとして登録される
  • 1つの目的に対してサブタスクが自動で切り出され、親課題にサブイシューが立ち上がる
ゆるくチャットするだけで
課題がどんどん立ち上がる。
人間の負担が激減！

⑥ LinearはAPIファースト — 外部から管理できるツール

  • Linearの最大の特徴はAPIファースト。外から管理することが前提の設計
  • デフォルトでSlackから課題を起票可能。MCPも用意されている
  • ClaudeからAPI経由でLinearを操作できる。Claudeが自分で課題を作れる
  • OpenAIやAnthropicの開発組織でも使われている。自分たちのために作って公開した感覚
APIが前提だから
AIに住まわせるのに
最適なツールだった！

⑦ 10チームの並走: 人間中心 / AI軍 / ハイブリッド

  • 人間中心チーム：Web制作、エージェント部、マーケティング、セールス、人事など部門ごとのチーム
  • AI軍チーム：「AIと僕しかいないチーム」。AIを使いこなす人間 + COO + AI
  • 新規ソリューション開発：AIから読みやすいWebサイト制作、動画コンテンツ、3DキャラクターのIP化
  • 全部で10チーム前後が並走。人間だけ、AI中心、ハイブリッドの3タイプが混在
AIがいるチームも
当たり前の存在に。
人間とAIの協働が日常！

⑧ Claude MDがルールブック — 高品質な課題の秘密

  • Claude MDに課題の作り方のルールを書く。これがルールブックとして機能
  • 必須項目：背景・コンテキスト（ハルシネーション防止のため担当者に確認）、ゴール、制約
  • 達成条件（AC）を明記。「この課題はどうやったら達成されるか」を明確に
  • 「誰がいつまでにやるのか」を絶対に含める。これが品質を担保する
最近はClaudeがゴールと制約を
ほぼ100%当ててくる。
背景だけは確認！

⑨ 深夜2時: ClaudeがLinear全体をスキャン

  • 深夜2時にClaudeがScheduled Task（ルーティン）でLinearをスキャン
  • 全チームの課題の状態を一括でチェック。前日からの変化を比較
  • 14日間放置のバックログや期限超過の課題を自動検知してアラート
  • Slackにレポートを投稿。朝起きた時には整頓された状態で待っている
寝てる間にAIが仕事場を
スキャン＆整頓。
朝起きたらレポートが完成！

⑩ 朝会のアジェンダもLinearの課題で自動生成

  • 朝7時、クロードが朝会用アジェンダの課題を自動で起票・完成させる
  • 進捗状況、昨日終わったタスク、今日やるべきことを人別に整理
  • チームごとに朝会を実施。チームリーダーがClaude作成のレポートをベースに発表
  • 以前は人間が記憶ベースでまとめていた。報告漏れや忘れが起きていたのを完全に防止
報告漏れがゼロに！
人間のしょうもなさを
AIがカバーしている

⑪ 「おはよう」の一言で脳の整理 — 睡眠時記憶の再現

  • 毎朝Claude Codeに「おはよう」と言う。それだけでコンテキスト最新化が始まる
  • Git Pull/Rebase → メモリ最新化 → 夜間レポート確認 → 今日の優先タスクを提案
  • 人間の脳が睡眠中に情報を整理するように、Claudeが「おはよう」で整頓済み状態を提供
  • 1日何セッションもやる。同時セッションも。1日の中で何度も「寝て起きて」を繰り返す
「おはよう」1回で
整頓された状態から
1日をスタートできる！

⑫ 3種の夜間レポート: Dashboard / Health / Retro

  • Daily Dashboard：全チーム横断で課題の状態をスナップショット。前日比較で変化を可視化
  • Daily Health Report：14日放置・絶対必須未着手・期限超過を検知してアラート
  • Retro Week：1週間分の振り返りレポートを自動発行。COOにとって最重要
  • レトロ週の結果をベースに、COOとClaudeの定例会を実施。リスク・方向性を夜中に酒飲みながら議論
COOとAIだけの定例会。
夜中に酒飲みながら
リスクを議論する新習慣！

⑬ 「着手可能」ラベルで承認作業を劇的に削減

  • 課題に「着手可能」ラベルをつけると、Claudeが勝手に実行してくれる
  • 自動化が進む一方で承認作業が増加。1日中承認している状態に陥る
  • 着手可能ラベルで承認が1つ減る。これが今一番力を入れている課題
  • 30分以上かかるものはAIがサブタスクに自動細分化。1セッションで処理できるサイズに
自動化しすぎると
承認地獄になる。
「着手可能」で解決！

⑭ Linear = Claudeの長期記憶 — セッションを超えて残る

  • Claude Codeのセッション内にはTODOなどの独自ドキュメントがあるが、セッションが終わると消える
  • Linearに書かれたものは消えない。セッション間の長期記憶として機能
  • 優先度が完璧なタイミングで出てくるよう設計。必要な情報がいつでも即座に呼び出せる
  • LinearとGitHubを間に挟むことで、モデルの乗り換えが容易。Claude→GPTでも情報は残る
Claudeの記憶は消えても
Linearは残る。
モデル依存しない設計！

⑮ Distill: セッション終了時の3ステップ振り返り

  • ステップ1：セッション中に「やろう」と言ったのにやらなかったことを遡ってチェック
  • ステップ2：「今日こんなことやったね」「この失敗があったね」と振り返り・反省させる
  • ステップ3：メモリーのキュレーション。フィードバック反映・プロジェクト方向性の更新・ドキュメント整理
  • やらなかったことがあればLinearに記録。即座にできることはその場で実行させる
やり残しをゼロに。
失敗も記録して
次に活かす仕組み！

⑯ スキル化: 同じパターン3回で自動スキル生成

  • 同じ作業を3回ほどやったら「スキル化した方がいいよね」とAIがパターン検出
  • 検出されたパターンをスキルとして定義。再利用可能な定型作業にする
  • 現状は各人のハーネス内で別々に実行。全社横断でのパターン共有はまだできていない
  • 同じリポジトリを使っているので記録は残る。Aさんの失敗はBさんにも見える状態にはなっている
3回やったらスキル化。
全社で横断的に
共有できれば最強！

⑰ Spark検出: 課題の重複・接続・リスクをAIが発見

  • 重複検出：別々のチームで同じような課題を立ててしまうことをAIが自動で発見
  • 接続検出：「こことここは繋がっているのでは？」という関連性をAIが提案
  • リスク検出：Retro Weekレポートから危険なプロジェクトや方向性の問題を特定
  • これらを踏まえてCOOとClaudeの1対1定例会で戦略的議論。夜中にリラックスした状態で
人間では見えない繋がりを
AIが見つけてくれる。
COOの第3の目！

⑱ AIネイティブ組織のマネジメントはまだ誰も解いていない

  • AI企業のマネジメント方法はビジネス書にもなっていない。誰も答えを持っていない
  • だから自分たちで作るしかない。Claudeと相談しながら組織設計を進める
  • コンテキストエンジニアリング、ハーネス管理が鍵。最初のずれを防ぐことが最重要
  • 「人間はポイントの部分だけ伝えて、AIが作れる状態にする」。それがAIネイティブ組織のゴール
答えはまだ誰も持ってない。
Claudeと一緒に
組織を作っていく！
