クラウドの教訓が示す生成AI活用の正解

AWSで固める基盤設計 ／ Claudeを安全に使い倒す ／ 内製化が変える企業の競争力
📅 2026年6月22日

①クラウドとAIは不可分な関係

  • AIのモデルを作るには大量の計算資源が不可欠
  • 突発的に計算をどんどん増やせるのは大規模クラウドだけ
  • 利用者側はいつでもどこでもAIを使いたい — これもクラウドだからこそ実現
AIはクラウドがなければそもそも成立しない！
AWSの安田氏「ものすごく神性が高い」

②サーバーワークス大石社長の軌跡

  • 新卒で総合商社に入社し、インターネットビジネスに携わる
  • 2000年に会社を設立 — 最初は大学向け合格通知サービス
  • 年2回の10時〜10:15分だけ200台サーバーが必要 → RIが極めて悪い問題
  • その解決策としてAWSに出会い、2008年にAWS事業へピボット
「ITの世界が丸っきり変わる」と確信して
大きな賭けに出た！

③AWS安田氏のキャリア

  • 元々はシステムエンジニアとして10年間活躍
  • スタートアップを数社経験後、Amazon内のスタートアップ部門に参加
  • 2011年の東京リージョンローンチの翌年・2012年にAWSへ
  • 現在は日本の顧客の声を新サービス開発にフィードバックする役割
「お客様の声を元に
新しいサービスを作る」← これがAWSの強み

④POCから本格導入への転換

  • 昨年までは実験的に使う企業がほとんどだった
  • 2025年後半〜2026年に入ると「本格的に使える」と判断する企業が急増
  • 個人の業務効率化から組織全体の業務改革へシフトが始まっている
「文章の要約」レベルから
組織全体を変えるフェーズへ！

⑤Amazon Connectでコールセンター革命

  • AWSにはAmazon Connectというクラウド型コールセンターシステムがある
  • 高価なPBX不要で、PCとヘッドセットだけでコールセンターを構築可能
  • 問い合わせが急増してもクラウドなら一気に拡張できる
  • コロナ禍のリモート対応でも威力を発揮した実績
クラウドならではのスケーラビリティが
ビジネス継続性を支える

⑥AIが変えるアフターコールワーク

  • 通話後のメモ作成・次への引き継ぎを生成AIが大幅に削減
  • 人間は厳しい意見をオブラートに包む傾向があるが、AIはドライに分析
  • お客様の生の声をより正確に拾えるようになった
  • 感情の変化（悪化・改善）もリアルタイムに分析可能
人間は「オブラートに包む」が
AIは感情フラットに分析してくれる

⑦生産性向上の「タイムラグ」問題

  • 8時間かかる業務が1時間になる — GenAIで簡単に実現
  • しかし空いた7時間を戦略的な業務に振り向けないと会社のスループットは変わらない
  • この「時間の再配置」ができるかどうかが企業の真の課題
  • 今年すぐに業績にヒットするわけではない — 遅延効果がある
効率化だけでは不十分！
中長期の戦略と実行プランこそが肝

⑧内製化の仮説 — 何を自社で作るべきか

  • サーバーワークスは約200のSaaS契約がある中で検討を開始
  • 内製化が適さないもの：ドメイン固有知識が必要なもの
  • 内製化が適さないもの：ネットワーク効果があるもの（LINE、Instagramなど）
  • それ以外のSaaSは自分たちで作り直すチャレンジを始めている
LINEのコピーはClaude Codeで
簡単に作れるけど、価値は人が集まってこそ

⑨Claude Codeが拓いた非エンジニア開発

  • 知識がなくても自分たちでAIエージェントを作れるフェーズに突入
  • しかし逆に専門知識が必要なものはAIでも作れないことが分かってきた
  • 以前は「手段もなくコストも合わなかった」が今なら実装可能に
営業現場なら自社の方が
自社の営業に詳しいに決まってる！

⑩AIDLC — AI時代のシステム開発手法

  • AWSが提唱するAI Driven Life Cycleという新手法
  • 開発者とビジネス側の人間が一堂に会してその場でAIを使って作り切る
  • 仕様策定から実装まで一気に進められ、劇的な生産性向上を実現
「その場で全部最後までやってしまう」
バイブコーディングの企業版とも言える

⑪BedrockでClaudeを安全に活用

  • 現在圧倒的に人気なのはAnthropicのClaude
  • AWSのBedrockを通じてセキュアにClaudeを使える環境を提供
  • 企業のデータを外部に漏らさずにプライベート環境で利用可能
  • 4月にOpenAIと提携発表 — CodexもBedrockでサポート
「ものすごくいいですね」
皆さんにどんどんお使いいただいてる — 安田氏

⑫コスト最適化 — Claudeは万能ではない

  • Claudeは素晴らしい製品だが、そのまま使い続けるとコストがかさむ
  • 簡単なタスクはBedrock上の無料のオープンモデルに任せる
  • セキュリティ的に重要な処理や複雑な本番タスクはClaudeに任せる
  • 適材適所でモデルを使い分けることがコスト最適化の鍵
Bedrockがあるからこそ
選択肢が増えてコスト最適化ができる！

⑬FDE — AI開発の新たな職能

  • FDE（Frontier Development Engineer） — 世界で求められている新ロール
  • サーバーワークスでは社内AI推進チームから現場（バックオフィス・営業）へエンジニアを派遣
  • 現場の仕組みをBedrockやClaudeを使って高度に自動化する知見を蓄積中
  • 蓄積した知見を今度はお客様にも展開していく方針
「社内で試して → お客様に展開」
自分ごととして取り組む姿勢

⑭財務部が自作するAIエージェント

  • 財務部のメンバーが自分たちでAIツールを作り始めた
  • 決済の回覧・督促確認など日々の苦労をエージェントで解消
  • 「誰が書き忘れたか」をチェックし、ポジティブな報告者を評価するエージェントも登場
  • そのノウハウはお客様にも紹介するレベルに
「性格の悪いエージェント」も
実は超有効な業務改善ツールだった！😂

⑮MCPがSaaSを救う新戦略

  • データベースSaaSを解約しようとした — 「エージェントがネット検索すれば不要では？」
  • SaaS会社からMCPサーバーを公開して自社営業管理システムと繋げていいと提案
  • 結果としてサービスを継続 — AI時代のSaaSの新しい価値提案
MCPでエージェントと直結できる
これがSaaSの存続戦略になる！

⑯AIエージェント vs 従来の生成AI

  • 従来の生成AI：人間が指示 → AIが応答するだけ
  • AIエージェント：人間が目標を与える → AIが計画を立ててアクション
  • できることの幅が劇的に広がる — 自律的に行動する
「応答」から「行動」へ
AIが事実的に物事を考えるフェーズ

⑰経営会議でマルチエージェントが議論

  • 社内の会計基準を修正する際、経営・現場・経理の3つのエージェントで意見を集約
  • 各エージェントがプロコンを整理して経営陣に提案
  • 人間の議論では声が大きい人が勝ちがちだが、AIは感情フラットに検討
  • 結果は「よくできている」レベルで経営判断の質が向上
3つのエージェントが冷静に議論！
経営の質を高める新アプローチ

⑱エージェント時代の権限管理とポリシー

  • 新入社員のエージェントと社長のエージェントが同じ権限では困る
  • トレーサビリティが必須 — 1年前の権限を追跡できないと事故調査で支障
  • サーバーワークスのポリシー：「社長エージェントは絶対に作らない」
  • 理由：社長の考えが何十年も固化して時代変化に対応できなくなるから
  • 戦略を持ち、中長期に実行し続けられる会社こそ勝者になる
権限基盤 + 明確なポリシー
このセットがAIエージェント時代の必須条件！
