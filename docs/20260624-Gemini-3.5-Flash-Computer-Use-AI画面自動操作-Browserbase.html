Gemini 3.5 Flash × Computer Use

AIが画面を見てクリック・入力・操作を自動実行 — エージェントの新時代
2026年6月24日

① Gemini 3.5 FlashにComputer Useがネイティブ搭載

  • 2026年6月24日 GoogleがGemini 3.5 Flashに画面操作機能を標準ツールとして統合したことを発表
  • これまではGemini 2.5 Computer Useという専用モデルとしてのみ提供されていた機能が、主力のFlashモデルに統合された
  • Flash系モデルは軽量で高速という特徴があるが、画面操作でも上位モデルに迫る性能を実現している
  • Webブラウザ、モバイルアプリ、デスクトップソフトまでクロスプラットフォームで対応
専用モデルから主力モデルへの統合。
「テスト環境」から「本番環境」へ
一気に身近になった技術です。

② Computer Useとは何か — 仕組みを解説

  • AIがスクリーンショットを通じて画面を「見て」認識する仕組み。画像としてUIを理解する
  • 「この座標をクリックする」「ここに文字を入力する」など具体的な操作を自動生成する
  • 操作の提案 → 実行 → 画面の再撮影 → 再分析というループ処理でタスクを完了させる
  • AIが直接ブラウザを操作するのではなく、開発者のコードがAIの提案を実行する関数呼び出しの仕組み
文章や画像を生成する従来の使い方とは違う。
AIが実際に画面を操作して
一連の作業を完了させるのが特徴。

③ なぜFlashモデルに統合したのか — 速度とコスト

  • Flash系モデルは高速かつ低コストが特徴。大量の画面操作が必要なエージェント用途に最適
  • 画面操作は何度もループするため、1回の応答速度が全体の効率に直結する
  • Proモデルに比べてコストを抑えつつ、エージェント用途では十分な性能を発揮する
  • すでに検索やマップ連携などの組み込みツールに対応済み。画面操作がさらなるツールとして追加された形
エージェントは何百回も
画面を見て操作を繰り返す。
スピードが正義の世界です。

④ 処理の流れ — 目標から操作までのステップ

  • Step 1 AIに「目標」と画面のスクリーンショットを渡す
  • Step 2 AIが画面を分析して「この座標をクリック」「ここに入力」などの操作を提案する
  • Step 3 開発者のプログラムが提案された操作を実行する
  • Step 4 実行後の新しい画面を再びAIに送り返す
  • Step 5 タスクが完了するまで②〜④を繰り返す
AIは操作の意図も出力するので
「なぜそのボタンを押したのか」が
開発者に把握しやすい設計です。

⑧ 想定される用途 — エージェントが解決する課題

  • 複数の手順を伴う業務の自動化。定型作業をAIに任せて効率化
  • 企業向けアプリをまたぐ情報収集。複数システム間のデータ転記を自動化
  • ウェブアプリの動作確認。テストシナリオをAIが実行して画面遷移を検証
  • アクセシビリティテスト。画面の読み上げや操作可能かをAIが自動チェック
  • 継続的なソフトウェアテスト。リリースごとの回帰テストを自動で実行

⑩ Interactions APIの土台 — 歴史的な背景

  • 2025年12月にInteractions APIが提供開始。画面操作の土台となるAPIはすでにGA（一般提供）
  • 今回の3.5 Flash統合は、既存のInteractions APIの上に画面操作を乗せたもの
  • API土台が先に整い、半年余り遅れてモデル側が対応した形
  • 土台が安定しているため、エンタープライズ環境での採用も安心
APIの設計が先でモデルが後。

⑪ セキュリティリスク — 画面操作の落とし穴

  • 間接プロンプトインジェクションのリスク。悪意のあるページ上の文章にAIがだまされる危険
  • 画面上の信頼できない情報や指示にAIが従ってしまう可能性
  • 目標やページ内容を誤解して誤った操作をしてしまうリスクも指摘されている
  • Computer Useは現時点でプレビュー機能。エラーや脆弱性が生じやすい段階
AIが画面を見て操作する =
Web上の罠にも触れる。
セキュリティ設計は必須です。

⑫ Googleの安全対策 — 多層防御のアプローチ

  • 標的型の敵対的トレーニングを実施。Computer Use向けに特化した安全性強化
  • ユーザー確認の仕組み — 機密性の高い操作や取り消せない操作の前に確認を求める
  • 自動停止の仕組み — 間接的なプロンプトインジェクションを検知した場合にタスクを自動停止
  • サンドボックスによる隔離実行、人間による確認、厳格なアクセス制御の組み合わせを推奨
重要なタスクでは密に監督し、
重大な判断や機密データの
取り扱いには使わないよう推奨。

  • 現在の性能はClaude Opus 4.8の83.4がトップ。3.5 Flashの78.4を上回る

⑰ コード例 — Computer Useの呼び出し方

  • Gemini API経由でInteractions APIを使って画面操作を実行
  • スクリーンショットを画像として渡し、AIが座標とアクションを返す
// Gemini 3.5 Flash Computer Use
const response = await client.interactions.create({
  model: "gemini-3.5-flash",
  contents: [{
    role: "user",
    parts: [
      { text: "ログインボタンをクリックして" },
      { inlineData: {
          mimeType: "image/png",
          data: screenshotBase64
      }}
    ]
  }],
  tools: ["computer_use"]
});
// AIがクリック座標や入力内容を返す
screenshotsを渡すだけ。
あとはAIが操作を判断する。
シンプルなAPI設計です。
