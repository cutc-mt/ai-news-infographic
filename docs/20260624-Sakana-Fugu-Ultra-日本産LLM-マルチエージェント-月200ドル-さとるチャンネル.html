① 2026年6月22日、Sakana AIが「Fugu」を一般提供開始

  • 6月22日 東京拠点のSakana AIが、複数の最先端AIモデルを1つのAPIで統合する新プロダクト「Sakana Fugu」を発表
  • 日本発のフロンティア級AIが世界市場に参入。Sakana AIはEvolutionary AIの研究で世界的に知られる企業
  • 「Fugu」と「Fugu Ultra」の2モデルを提供。Ultraは精度重視の上位版で、通常版は速度とコストバランスを重視
  • OpenAI互換APIで提供され、既存のGPT用コードのbase_urlとモデル名を書き換えるだけで導入が可能
日本のAI企業がこんなに 大きな発表をするのは異例！ フグ＝毒も持つが美味い＝ リスクとリターンの命名？

② Fuguの仕組み — AIにAIを指揮させる

  • Fugu自体がLLMでありながら、GPT-5.5やClaude Opus 4.8、Gemini 3.1 Proといった複数のモデルを「エージェントプール」として管理
  • ユーザーのリクエストの難易度に応じて、最適なAIモデルを自動選択。必要に応じて複数の専門エージェントをチームとして編成
  • 固定ルールではなくAI自身がモデルの組み合わせや協調パターンを動的に判断する仕組み
  • 再帰的推論を搭載。自身の出力を読み返して不十分と判断すると、修正ワークフローを自ら立ち上げる
「どのAIを使うか」を ユーザーが悩む必要がない。 指揮官型AIが 全部やってくれる！

④ ベンチマークでFable 5を超える実績

  • Terminal-Bench 2.1（AIエージェントのプログラミング性能）でFable 5を上回るスコアを獲得
  • Charxiv Reasoning（複雑な図表の読み取り性能）でClaude Mythos Previewのスコアを上回ったと公表
  • GPQA-D（科学的推論ベンチマーク）で95.1%を達成。Gemini 3.1やGPT-5.4を凌駕
  • ルービックキューブ解法テストで、4つのモデル中最短の19ステップで解決（GPT-5.5、Claude Opus 4.8、Gemini 3.1 Proと比較）
Fable 5とMythos Previewを エージェントプールに含めずに この成績はすごい。

⑤ 料金プラン — 月20ドルから使える

  • Standard 月額20ドル（約3,200円） — 基本的なルーティン業務に対応する入門プラン
  • Pro 月額100ドル（約16,000円） — Standardの10倍の利用枠。中規模データ分析や並行処理に
  • Max 月額200ドル（約32,000円） — Standardの20倍。大規模分析や高度なマルチエージェント連携に
  • すべてのプランでFuguとFugu Ultraの両方が使える。2026年7月末までの登録で2ヶ月目が無料になるキャンペーンも実施中
Fugu UltraのAPI従量課金は 入力$5/出力$30（100万トークン） 272K超えると入力$10/出力$45

⑥ 月200ドルプランに課金して使った感想

  • ベータプログラムでは約500名のテスターが参加。コードレビューで他モデルが見逃すバグを20件以上発見した報告がある
  • 簡単なタスクには軽量モデルが自動で割り当てられるため、コスト効率が良いと好評
  • コーディングやチャットボット構築で高い性能と低レイテンシのバランスを実感できる
  • データやプライバシー要件に合わせて、特定のエージェントをプールから除外する設定も可能
GPT・Claude・Geminiを 個別契約すると費用がかさむが、 1つのAPIでまとめるのは合理的。

⑦ Anthropic規制をにらんだ戦略的設計

  • 6月10日にAnthropicがFable 5とMythos 5の提供を開始したものの、わずか3日後に米政府の命令で提供を一時停止
  • Sakana AIは「規制や輸出管理の条件は一夜にして変わる」と指摘。「重要インフラを1社のAPIに頼ることは現実的な弱点」と警鐘
  • Fuguの背後のモデル群は柔軟に入れ替え可能。特定ベンダーが利用を制限しても動的に迂回できる設計
  • 今後はSakana AIが開発する独自モデルやオープンモデルもエージェントプールに追加していく方針
輸出規制の地政学的リスクを 逆手にとったビジネスモデル。 ベンダーロックイン回避が 強みになる時代に。

⑧ OpenAI互換API — 移行コストほぼゼロ

  • OpenAI互換エンドポイントを採用。既存のGPTコードのbase_urlを書き換えるだけで動作する
  • PythonでもJavaScriptでも、OpenAIクライアントのbase_urlにSakana APIのURLを指定し、modelに「fugu-ultra」と書けば完了
  • SDKやプロンプトもそのまま流用できるため、専門のIT人材がいなくても短時間で導入可能
  • 企業向けの従量課金プランも用意。法人であればAPIキーはconsole.sakana.aiで生成
3ステップで移行完了！ ①エンドポイント更新 ②APIキー差し替え ③動作確認

⑨ 複数AIを組み合わせて使うのが最新トレンド

  • 2026年後半のAI業界トレンドは単一モデルの巨大化から複数モデルのオーケストレーションへの転換
  • 開発者の70%以上が2〜4個のAIを平行で使っているという調査結果もある。単一モデルへの依存を避ける動きが加速
  • バックエンドはGPT、設計レビューはClaude、文章はOpusという使い分けが定着しつつある
  • Fuguはその「手動での使い分け」をシステム化する。ユーザーが悩まずに最適な頭脳を自動で選んでくれる
単一モデルの壁に 業界全体が気づき始めた。 集合知の時代の到来。

⑩ 再帰的推論 — 自分で答えを検証する

  • Fuguは再帰的推論（Inference-time Scaling）を搭載。自身の出力を読み返して修正するサイクルを自動で回す
  • 最初の答えでは不十分と判断すると、追加のエージェントを起動して精度を引き上げる
  • 再帰の深さは推論時に調整可能で、再学習なしで精度を引き上げられるのが特徴
  • 人的ミスの削減と品質向上を同時に実現。コードレビューで他モデルが見逃すバグを多数発見した実績が裏付け
AIが自分の答えを 自分で検証する時代。 メタ認知のような能力が エージェントに備わった。

⑪ コード生成でも最強水準 — LCBv6で93.2%

  • コード生成ベンチマーク「LCBv6」で93.2%を達成。他の主要モデルを上回る結果
  • ベータテスターのコードレビューで、他モデルが見逃したバグを20件以上発見した実績
  • エージェントのプログラミング性能を測るTerminal-Bench 2.1でもFable 5を上回る成績
  • 高い性能と低レイテンシのバランスに優れ、日常業務のコーディングで即座に効果を実感
コードレビュー20件以上の バグ発見はすごい。 開発現場での実力が 数字で証明された。

⑮ 法人向け従量課金プランの詳細

  • Fuguは利用した基盤モデルの標準レートで課金。エージェントが1つならそのモデルの料金、複数なら最上位モデルのレート
  • Fugu Ultraは入力$5/出力$30（100万トークンあたり）。コンテキスト272Kを超えると入力$10/出力$45に
  • キャッシュ入力は$0.50（272K超えると$1.00）。繰り返し使うプロンプトの追加コストを最小化
  • APIキーはconsole.sakana.aiで生成。企業の既存システムに組み込みやすい設計
Claude Max月額$200と比べると、 Fugu UltraはAPI単価で見ると 競争力あり。

⑯ 導入手順 — 3ステップで完了

  • Step 1 既存パッケージのエンドポイントとモデル名をFugu仕様に更新
  • Step 2 現在のAPIキーをSakana Fugu APIキーに差し替え
  • Step 3 簡単なリクエストを送信して応答を確認し、必要に応じてパラメータを調整
  • Python / JavaScriptどちらでもOpenAIクライアントのbase_urlにSakana APIを指定するだけ。導入時の摩擦は最小限
# Pythonでの例
from openai import OpenAI
client = OpenAI(
    base_url="https://api.sakana.ai/v1",
    api_key="your-sakana-api-key"
)
response = client.chat.completions.create(
    model="fugu-ultra",
    messages=[{"role": "user",
               "content": "コードレビューして"}]
)
