② エージェント投資の3つの柱

  • ①アシスト機能 BigQuery Studio上でSQLの作成・トラブルシューティング・最適化を支援するアシスタント
  • ②データエージェント ペルソナ・タスク特化型。データエンジニアリング、データサイエンス、会話型アナリティクス
  • ③開発者ツール SDK、MCP APIなど、開発者が自分自身のエージェントを構築するためのプリミティブ
  • 3つの柱が連携して、すべてのデータユーザーをカバーする包括的なエコシステムを形成
単なるチャットボットではなく、
データのライフサイクル全体を
自律的にサポートする存在へ。

③ BigQueryアシスタントの進化

  • BigQuery Studioの右側チャットパネルから利用可能。SQLのバイブコーディングができるようになった
  • エラーのあるSQLを自動で検出・修正。ユーザーIDとアイテムIDの間違いなどを見つけて直す
  • クエリの最適化も自動で実行。日付のパーティションプルーニングや集約の改善を提案
  • BigQueryでGeminiが処理したデータ量が1年で30倍に急増。実用段階に完全に移行
「Fix it」ボタン一つで

  • エージェントが提案した仕様をレビュー・修正できる。エンジニアが常にコントロールを持てる設計
  • 既存パイプラインのトラブルシューティングと自動修正にも対応

⑥ 会話型アナリティクスの新機能

  • BigQuery上でGA。ビジネスユーザーが自然言語でデータにアクセス
  • 過去のデータの参照だけでなく、売上の予測や異常検知など高度な分析にも対応
  • マルチモーダル対応。テキストや画像などの複雑なデータも自然言語でクエリ可能
  • Lookerではダッシュボード上で直接チャット可能。リッチな可視化を伴う回答
「過去12ヶ月の売上は？」だけでなく
「今後6ヶ月の売上を予測して」

⑭ データ活用への3年半の旅路

  • 第1段階 データの中央集約。全組織のシステムをGCPに移行
  • 第2段階 BigQueryを唯一の真実の情報源に。ビジネス層を構築してドメインごとに整理
  • 第3段階 セルフサービスのダッシュボード。しかし、発見性とスケールに限界
  • 第4段階 会話型アナリティクスの導入。データチーム約800人の時間を解放
「良いデータ基盤がないと
エージェントはゴミを出力する」

⑮ ガバナンスとトレーニングの徹底

  • 会話型アナリティクスの利用者全員に必須トレーニングを実施。未受講者はアクセス権を剥奪
  • 「エージェントが同じ質問で毎回違う回答をするのは致命的な問題」— 基盤レイヤーの信頼性が鍵
  • 名前「Parity」のカスタムエージェントを社内構築。Gemini Enterprise上で運用
  • プライバシー、ガバナンス、コンプライアンスの壁にぶつかるリスクを強調。文脈の正確さが必須
