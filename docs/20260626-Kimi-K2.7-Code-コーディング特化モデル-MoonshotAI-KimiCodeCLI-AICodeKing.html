Kimi K2.7 Code 徹底解説

オープンソースのコーディング特化エージェントモデルが登場

📅 2026年6月26日 | 🎬 AICodeKing

①Kimi K2.7 Code がついにリリース

  • Moonshot AIが開発したコーディング特化のエージェントモデルが登場
  • 単なるチャットモデルに「Code」を付けたものではなく、本格的なソフトウェア開発タスク向けに設計
  • オープンソースで公開されており、誰でも利用可能
  • 長時間のエージェント的なコーディング作業に最適化
単機能のコード生成ではなく、
レポジトリ全体を理解して開発を進めるのが特徴です！

②MoEアーキテクチャと巨大パラメータ

  • Mixture-of-Experts方式を採用
  • 総パラメータ数は1兆（1 Trillion）
  • トークンごとの活性化パラメータは320億（32B）に抑えられ高速動作
  • テキスト・画像・動画のマルチモーダル入力に対応
1兆パラメータのうち320億だけを
使うから効率が良い仕組み！

③256Kコンテキストウィンドウ

  • 256Kの長いコンテキストウィンドウをサポート
  • 大規模なレポジトリ全体を一気に読み込んで理解可能
  • 複数ファイルにまたがる機能実装にも対応
  • 長時間のセッションでコンテキストを維持したまま作業継続
長いコードベースでも文脈を忘れないのが強みです。

④思考モードが常にオン

  • Thinking-only モデルで、推論モードがデフォルトで有効
  • 前世代K2.6比で思考トークンを約30%削減
  • 推論トークンが減る＝応答速度UP＋コストDOWN
  • 思考をOFFにするとK2.6に自動フォールバック
思考を減らしつつ精度を上げる…これすごい！

⑤K2.6からの大幅なベンチマーク改善

  • Kimi Code Bench v2: 62点（K2.6は50.9点）
  • SWE-benchでも大幅なスコア向上を実現
  • ProgramBenchやMLSBenchでも有意な改善を記録
  • ※Moonshot社の公表データなので、第三者検証が待たれる
5%のマージンでK2.6を上回る実績。
ただし実レポジトリでの検証が重要！

⑥Kimi Code CLIの登場

  • ターミナルで動くAIコーディングエージェント「Kimi Code CLI」
  • レポジトリの探索・ファイル編集・コマンド実行・テスト実行に対応
  • Kimiのサイトからコマンドをコピーして簡単インストール
  • 初回起動時にOAuthログインでKimiアカウントと連携
ターミナルから直接「kimi」と打つだけ。シンプル！

⑦Planモードで慎重に設計

  • Shift+TabでPlanモードに切り替え可能
  • Kimiがコードベースを調査してアプローチを提案
  • 主要なエントリーポイントとモジュールの依存関係をマッピング
  • プランを確認してから実際の変更を実行する安全なフロー
まず計画してから実行。
大規模リファクタに最適な戦略！

⑧Kimi Codeの豊富な機能一覧

  • ファイルの読み取りと書き込み
  • ローカルファイル検索とフィルタリング
  • テストの自動追加と実行
  • 差分の確認と説明
  • Webページの取得
  • シェルコマンドの実行
CLI内でほぼすべての作業が完結するCompleteness！

⑨安全な実行メカニズム

  • 読み取り専用操作はデフォルトで許可される安全設計
  • ファイルの変更やシェルコマンドはユーザー確認が必要
  • Forkで実験用コピーを作成して安全にテスト可能
  • 何かが壊れても自動リカバリーして作業を継続
危険な操作は確認を求めるから安心。
実験はForkで分けて進められます。
