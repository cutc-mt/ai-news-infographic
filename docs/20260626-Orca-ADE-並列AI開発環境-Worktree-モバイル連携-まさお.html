① Orcaとは？ — AIエージェントの司令塔IDE

  • Orcaは複数のAIコーディングエージェントをひとつの作業台に集めて並列に動かせる開発環境
  • ADE（Agent Development Environment）という新しいカテゴリに属するツール
  • GitHubで7.1kスターを獲得。最新リリースはv1.4.97
  • Ghostty、VS Code、Codexデスクトップアプリを別々に開く手間をなくす道具
Claude CodeやCodexを横並びで動かせる。
画面の散らかりを劇的に減らします！

② 25種類以上のエージェントに対応

  • Claude Code / Codex / OpenCode / Grok / Gemini など主要AIを一括管理
  • Cursor CLI、GitHub Copilot、Pi、Hermes Agent、Devin、Kimi、Cline、Goose、Continue などにも対応
  • エージェント同士を独立した隔離環境で同時に実行できるのが最大の強み
  • ロックインを避けたい開発者にとって、マルチエージェント運用の土台として最適

④ Orcaは誰に向いている？

  • 向いている人: Claude CodeとCodexを両方使い、作業ごとにエージェントを分けたい開発者
  • diff を読み、commit やPR を自分で管理する開発者にも最適
  • まだ薄い: 1つのターミナルで1つのエージェントだけを使っている場合
  • 注意: Computer Use や Orca CLI を使うには設定から明示的に導入が必要

⑤ 主役はParallel Worktrees — 並列開発の要

  • git worktree を活用して、1つのリポジトリから隔離された作業コピーを複数生成
  • 共有ブランチを直接汚さず、作業ごとに分かれた場所でAIが変更を進める仕組み
  • 複数のworktreeを並列で走らせ、完了したものから diff を確認していく
  • diff はAIがどのファイルをどう直したかを人間が確認する入口になる
Claude Codeに修正を任せつつ
Codexに別案を作らせる —
エージェントの分担作業が実現！

⑥ Worktree開発の4ステップ

  • Step 1: GitHub issue、ブランチ、PR から作業対象を探して選択。エージェントへの文脈が明確に
  • Step 2: その作業に使うエージェントを割り当て。Claude Codeに実装修理、Codexに別案を作らせる
  • Step 3: Create Worktree を押して隔離環境を作成。共有ブランチを汚さない安全な空間が完成
  • Step 4: 並列実行後、完了したものから diff を読んで人間がレビュー

⑦ エージェントの並列実行とステータス管理

  • 左メニューで複数プロジェクトと複数エージェントの稼働状況を一望できる
  • 完了するとアイコンが変わるので、まだ動いているのか確認待ちなのかを見失いにくい
  • タブは右方向や上方向に増やせる。実装、ブラウザ、エミュレーターを分けて配置可能
  • Agent Session History から新しいタブでレジューム。途中の文脈を失ったときの再説明を減らせる

⑨ Workspace Board — 散らかりを防ぐカンバン

  • 右下にカンバン形式のボードがあり、Todo / In Review / Done でタスクを視覚的に整理
  • ダブルクリックで該当 worktree に飛べるので、確認対象が画面内で閉じる設計
  • AIが複数走っていても、人間側のレビュー待ち、再指示待ち、完了済みが混ざる状況を防ぐ
  • 並列実行時の人間の注意力を守る機能。OrcaのUXで最も評価が高い部分

⑩ Computer Useでブラウザ操作も自動化

  • 設定から Computer Use と Orca CLI を入れるとブラウザやアプリ操作まで任せられる
  • 別PCでWindsurfを開き、「マークダウンを作って保存して」と指示したところ実際に操作を任せられた
  • ただし最初から全員が触る機能ではなく、通常のコーディング用途なら他の機能から慣れるのが推奨
  • おすすめの習熟順序：Workspace Board → 保存コマンド → セッション履歴 → Computer Use

⑪ Orcaの設計思想 — AI丸投げではない

  • diff を読み、commit を気にし、worktree を綺麗に保ちたい開発者のための足場
  • AIに丸投げする仕組みではなく、人間がコントロールするための環境
  • 複数のAIを走らせたあと、人間がどこを見て、どこをレビューし、どこへ戻るかまで画面に配置する
  • Claude CodeやCodexの個別性能を比べる前に、作業全体の見通しを整える道具としての価値
「派手な自動化に尽きない道具。
人間の作業全体の見通しを
整えるためのツール」

⑫ インストールと始め方

  • macOS なら Homebrew から一発インストール。公式サイトからもダウンロード可能
  • 起動後はClaudeかCodexのどちらか1つをデフォルトに設定して、既存リポジトリを開くところから
  • GitHub issue を1つ選んで Create Worktree で隔離環境を作ると、Orcaの並列開発の顔が分かる
  • 初日から数分で並列開発を体験できる設計。導入コストはかなり低い

⑬ モバイル連携 — スマホから開発セッションを操作

  • デスクトップ版のモバイル連携設定画面に表示されるQRコードをスキャンするだけ
  • ペアリングは数秒で完了。立ち上げっぱなしのMac miniを外からリモコンのように操作できる
  • デスクトップ側に操作中であることがリアルタイムに表示される
  • 完了通知やフォローアップの送信にも使える構成。ターミナルの復元も可能
