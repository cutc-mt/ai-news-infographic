①大発表の概要: Computer Useがネイティブ統合

2026年6月24日、GoogleはGemini 3.5 FlashにComputer Use機能をネイティブ統合したと発表。AIが画面を認識してクリックや入力などの操作を自動で行えるようになった。

  • 発表日: 2026年6月24日（米国時間）
  • 発表元: Google DeepMind
  • 発表者: プロダクトマネージャー Mateo Quiros氏
  • パブリックプレビュー 段階で提供
専用モデルから汎用モデルへ統合！これは大きな進化だよ👀

②Computer Useとは: 画面を見て操作するAI

Computer Useは、AIがスクリーンショットを受け取って画面の状態を把握し、次に実行すべきUI操作を出力する機能。

  • スクリーンショットで画面を認識
  • クリック位置を座標で指定
  • テキスト入力を自動生成
  • スクロール操作にも対応
  • 操作の意図もテキストで出力
AIが「どこをクリックすればいいか」を自分で考えて判断するの！🤖

⑤技術仕組み: Agentic Loopの処理の流れ

Computer UseはAIがブラウザを直接操作するわけではない。モデルは「脳」として機能し、クライアント側のPlaywrightが実際の操作を担当する。

  • クライアント側でスクリーンショットを取得
  • スクリーンショットと目標をAPIに送信
  • モデルが画面を「見て」操作を生成
  • クライアントがPlaywrightで操作を実行
  • 新しいスクリーンショットを取得してループ
  • タスク完了まで繰り返す
スクリーンショット→判断→操作→再撮影。このループが肝だよ🔄

⑥対応環境: ブラウザ・モバイル・デスクトップ

Computer Useは3つの操作環境に対応。Webブラウジングからスマホアプリ、PCのデスクトップアプリまで幅広くカバーする。

  • Browser Webアプリ操作・フォーム入力
  • Mobile スマホアプリの操作
  • Desktop デスクトップアプリの操作
  • 環境間の連携も可能
PCもスマホもブラウザも全部いける！すごい汎用性📱💻

⑨実装方法: Pythonで始めるComputer Use

開発者はgoogle-genai SDK v2.7.0以上とPlaywrightを使って簡単に実装できる。

pip install google-genai playwright
playwright install chromium
  • google-genai SDK（公式Python SDK）
  • Playwright（ブラウザ操作ライブラリ）
  • Google AI StudioでAPIキー取得
たった2行のコマンドで準備完了！開発者フレンドリー💻

⑩コード例: Interactions APIの基本形

Interactions APIを使えば最小限のコードでComputer Useを有効化できる。

from google import genai
client = genai.Client()
interaction = client.interactions.create(
  model="gemini-3.5-flash",
  input="Googleで検索してください。",
  tools=[{"type": "computer_use",
          "environment": "browser"}])
print(interaction)
toolsにcomputer_useを指定するだけ！API設計が美しい✨

⑫セキュリティリスク: AIが画面で騙される危険

画面操作をAIに任せるうえでプロンプトインジェクションや不可逆な操作のリスクが懸念されている。

  • プロンプトインジェクション攻撃のリスク
  • ファイル削除などの不可逆操作
  • 支払いやフォーム送信の誤操作
  • 意図しない機密情報の漏洩
  • ページ内容の誤解による操作ミス
AIがウェブページに騙されるかも！人間の確認は必須ね⚠️

⑬安全機能: Googleが用意した3つの対策

Googleは企業向けに複数の安全機能をオプションで提供。多層防御のアプローチを推奨している。

  • User Confirmation センシティブ操作前に承認要求
  • Injection Detection インジェクション検知時にタスク停止
  • Adversarial Training 敵対的トレーニングで耐性強化
  • Dockerサンドボックスでの隔離実行
  • 操作ログの記録・監査
安全対策もしっかり用意されてる！でも自己責任で使おうね🛡️

⑭競合比較: Claude vs OpenAI CUA vs Gemini

Computer Use領域の三大競合を比較。それぞれ異なる強みを持っている。

  • Gemini: ブラウザ自動化が強み、ネイティブ統合
  • Claude: VM・コンテナで隔離実行、安全性重視
  • OpenAI CUA: デスクトップネイティブ、垂直統合型
  • コスト: Geminiが最も安い
  • 統合: Geminiはメインモデルに統合済み
用途に合わせて選ぼう！Web中心ならGemini一択かも⚖️

⑱安全に使うための推奨事項

プレビュー段階であるため、本番環境での重要処理には利用しないことが推奨されている。以下のポイントを守って安全に運用しよう。

  • 本番環境での重要処理には利用しない
  • センシティブ操作はUser Confirmationを有効化
  • Dockerコンテナなどサンドボックスで実行
  • 操作ログを取得して監査可能に
  • 人間による確認を介在させる
  • 厳格なアクセス制御を設定
AIに仕事を任せるけど、責任は人間にある！安全第一でね❤️
