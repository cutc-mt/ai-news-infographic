マルチエージェントの継続的評価パイプラインを構築する

Gemini Enterprise Agent Platform × Cloud Run で自動テストを実現！
📅 2026年7月1日 | 発表者: Vlad Kolesnikov, Leonid Yankulin

①この配信のテーマ

  • 「AIエージェントが本当に動いているか」を客観的に検証する
  • 手動のバイブチェックからデータ駆動の評価へ移行
  • Gemini Enterprise Agent Platformを活用
  • プロダクション環境での品質保証がテーマ
「なんとなく動いてる」じゃなくて
数字で証明しよう！

②「見えない回帰」の問題

  • コード変更でエージェントの品質がこっそり低下する
  • 回帰は目に見えにくいから危険
  • プロトタイプでは手動確認で済むが…
  • プロダクションでは自動化された評価が必須
見えないバグが
プロダクションで発覚するの恐い！

③マルチエージェントシステムとは

  • 複数のAIエージェントが協調して動く仕組み
  • タスクが分散され、それぞれが専門的な役割を担う
  • オーケストレーターが全体を束ねる
  • 構成が複雑なほどテストが困難になる
1人のAIよりチームの方が
強いけど、管理が大変！

④データ駆動の評価とは

  • 定量的なメトリクスでエージェントを評価する
  • ルーブリック（評価基準）を明確に定義
  • ツール使用の品質も測定対象
  • コード変更のたびに自動で再評価が走る
「前より悪くなってない？」を
毎回自動で確認できる！

⑤適応型ルーブリック

  • 評価基準を動的に調整できる仕組み
  • Geminiが出力の品質スコアを自動採点
  • 複数の次元で多角的に評価
  • 基準のカスタマイズも柔軟に対応
評価基準もAIが
賢く調整してくれる！

⑥ツール使用品質メトリクス

  • エージェントが正しくツールを呼び出せたかを測る
  • 引数の妥当性・結果の正確性をチェック
  • 不要なツール呼び出しを検出する
  • ツールの使用頻度や順序も分析
「正しいツールを
正しく使ってる？」を数値化！

⑦Gemini Enterprise Agent Platform

  • Google Cloudのエージェント開発プラットフォーム
  • Pipelines機能で評価ワークフローを構築
  • 複数のエージェントをオーケストレーション
  • エンタープライズグレードのセキュリティ
Geminiの力で
エージェント管理が超便利に！

⑧Cloud Run Functions

  • 評価ロジックをサーバーレス関数として実装
  • 負荷に応じて自動スケーリング
  • Pipeline とシームレスに連携
  • 使った分だけコスト最適化
サーバー管理なしで
評価処理をサクサク動かせる！

⑨シャドウデプロイメント

  • 本番環境に影響なしで新しい版をテスト
  • Cloud Runのtagged revision機能を活用
  • プライベートなリビジョンに安全にデプロイ
  • トラフィックを分割して段階的に移行
ユーザーに気づかれずに
こっそりテストできる！

⑩CI/CD自動化との統合

  • コード変更のたびに自動で評価パイプラインが実行
  • PRごとに回帰テストが走る仕組み
  • 品質が基準を下回ればデプロイを自動阻止
  • 品質低下の通知も自動で届く
コード変えるたびに
AIが品質チェックしてくれる！

⑪評価パイプラインの全体像

  • テストスイートをトリガーとして起動
  • エージェントに入力データを送信
  • ルーブリックで出力を採点
  • 結果をレポートとして出力
入力→実行→採点→レポート
のサイクルを自動化！

⑫Codelabでハンズオン

  • 公式のCodelabが用意されている
  • goo.gle/prai-rs-2 でアクセス可能
  • 実際に手を動かしてパイプラインを体験
  • ステップバイステップのガイド付き
Codelabをやれば
すぐに試せるよ！

⑬評価の3つの柱

  • 適応型ルーブリックで出力品質を評価
  • ツール使用メトリクスで操作の正確性を検証
  • 回帰テストで品質低下を防止
  • 3つを組み合わせて網羅的に評価
品質・正確性・安定性
3方向からガード！

⑭テストスイートの設計

  • 多様な入力パターンを網羅的に用意
  • エッジケースや異常系も含める
  • 正常系・異常系・境界値をカバー
  • テストケースはバージョン管理する
良いテストは
良いエージェントの基盤！

⑮スコアリングとレポート

  • 各テストのスコアを集計して可視化
  • 前回実行との差分比較が可能
  • 結果を自動レポートとして出力
  • しきい値を下回るとアラート通知
数字で見えるから
改善の方向性が明確！

⑯分散エージェントの課題

  • 複数エージェント間の相互作用が予測困難
  • 実行時間やコストの管理が必要
  • 通信の遅延やエラーへの対応
  • 各エージェントの役割の明確化が重要
エージェントが増えるほど
複雑さが爆発する！

⑰回帰防止のベストプラクティス

  • 品質のベースラインを定義して守る
  • A/Bテストで新旧比較を徹底
  • 変更前にスナップショットを取得
  • 品質が下回ったら自動でデプロイ阻止
「前より悪くならない」
これが一番大事！

⑱まとめ：評価の自動化で
信頼性を確保する

  • データ駆動の評価がエージェント信頼性の鍵
  • Gemini + Cloud Run でパイプライン構築
  • シャドウデプロイで安全な移行を実現
  • CI/CDに組み込んで継続的に品質を守る
AIエージェントの品質を
自動で守る時代が来た！
