MCP vs API

「AIのためのUSB-C」— 知らないと損するプロトコルのすべて
2026年7月2日

① MCP（Model Context Protocol）とは

  • Anthropicが提唱する、AIモデルと外部ツール/データソースを接続する標準プロトコル
  • 「AIのためのUSB-C」— 1つのインターフェースで全てのAIツールに接続できる画期的アプローチ
  • 従来の課題：各サービスごとに異なる認証・フォーマット・エンドポイント。統合のたびに独自実装が必要だった
  • MCPの解決策 統一規格を提供し、一度の接続設定で複数AIツールに対応
USB-Cが全デバイスを統合したように、
MCPが全AIツールの接続規格に。
開発者にとってこれは革命です！

② MCPの仕組み — 3つの役割

  • Host（ホスト）: Claude Code、Codex等のAIアプリケーション。ユーザーが直接操作する側
  • Client（クライアント）: ホスト内でMCPサーバーと通信する中継コンポーネント
  • Server（サーバー）: GitHub、Slack、X、DB等の各サービスとのブリッジ
  • 通信プロトコル: JSON-RPC 2.0で標準化されたメッセージ交換
User → Host → Client → Server → Service
このシンプルな階層が全てを統合！

③ 3つの運用モード

  • Local（stdio） ローカルプロセス間通信。最も一般的なモード。Claude Code等で使用
  • Remote（SSE） HTTP Server-Sent Eventsでリモート接続。クラウド上のMCP Serverにアクセス
  • Streamable HTTP 双方向ストリーミング。最新モードでリアルタイム性を追求
  • 用途に応じて最適なトランスポートを選択可能。ローカル開発ならstdio、本番運用ならRemoteが一般的
2026年6月にXのリモート版MCPが追加。
Streamable HTTPの登場で
リモート接続がさらに強力に！

④ MCPでできること — 4つの能力

  • Tools（ツール）: AIが外部APIを呼び出す。GitHub PR作成、Slack送信、ファイル操作等
  • Resources（リソース）: 外部データの読み取り。DBクエリ、ファイルアクセス、APIレスポンス
  • Prompts（プロンプト）: 再利用可能なプロンプトテンプレートを提供。チームで共有可能
  • Sampling（サンプリング）: モデルに追加コンテキストを提供し、精度を向上
4つの能力を組み合わせると
AIエージェントが自律的に外部サービスを
操作できるようになります。

⑦ MCPアーキテクチャ — 通信の流れ

  • Step 1 ユーザーがClaude Code等で自然言語の指示を入力
  • Step 2 Host（Claude Code）がClientを介してMCP ServerにJSON-RPCで通信
  • Step 3 MCP Serverが対象サービス（GitHub / X / DB等）のAPIを呼び出し
  • Step 4 APIの結果をMCP Server→Client→Host経由でユーザーに返す
User → Host → Client → MCP Server → External API
  ← ← ← ← ← ← Result flows back ← ← ← ← ← ←
ユーザーは自然言語で指示するだけ。
MCPが裏側で全ての通信を処理。
複雑さをユーザーから隠蔽！

⑫ MCP Serverの作り方

  • TypeScript / Python — 公式SDKでサーバー構築
  • @modelcontextprotocol/sdk が公式npmパッケージ
import { Server } from '@modelcontextprotocol/sdk';

const server = new Server(
  { name: 'my-tool', version: '1.0' },
  { transport: 'stdio' }
);

server.tool('search', { query: z.string() },
  async ({ query }) => ({
    content: [{ type: 'text',
      text: `Result: ${query}` }]
  })
);
  • MCP Inspectorでテスト・デバッグ可能。GUIツールで対話的に確認
  • stdioまたはSSEトランスポートで配信。数行のコードでサーバー起動
TypeScriptかPythonを知っていれば
30分でMCP Serverが作れる。
公式SDKがしっかり整備されています。

⑯ ツール切替えがゼロコストに

  • 従来のAPI統合ではツールを変えるたびにSDKの学習・認証設定・コード書き換えが必要
