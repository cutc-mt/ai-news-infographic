①Fable5が遂に復帰: 3週間の停止から全サービス再開

Claude Fable 5は2026年6月9日に公開されたAnthropicの最上位AIモデル。しかし公開からわずか72時間後の6月12日、米商務省の輸出管理命令を受けて全世界で即時停止。3週間にわたる利用不可期間を経て、6月30日に規制が解除され、7月1日に利用が再開された。

  • 6/9 公開 → 6/12 全世界停止 → 6/30 規制解除 → 7/1 利用再開
  • 3週間の停止期間中、多くの企業で自動化パイプラインが影響を受けた
  • 「先月本番移行した矢先だったので頭が真っ白になった」という声も
💡 3週間の停止は「モデルはいつ止まるか分からない」という可用性リスクの教訓になったね

②Fable5の圧倒的性能: コーディング最強クラス

Fable 5はSWE-Bench Proで80.3%を記録。独立系評価機関vals.aiによるSWE-Bench Verifiedでは95.0%と、現時点でコーディング能力は最強クラス。1M（100万）トークンのコンテキストウィンドウと最大128Kトークンの出力に対応する。

  • SWE-Bench Pro 80.3%（Anthropic公式scaffold値）
  • SWE-Bench Verified 95.0%（vals.ai独立評価）
  • 大規模なコードベースを丸ごと把握したまま作業を続けられる
🎯 単純な質問答えよりも複雑な作業を分解して長く進めるのが得意なモデルだよ

③Fable5の料金体系: 高性能だがコストに注意

API価格は入力$10・出力$50（いずれも100万トークンあたり）。日常的な軽い質問に使うには重い価格帯。ただしプロンプトキャッシュで入力コストを最大90%割引できる仕組みがある。

  • 入力: $10 / 100万トークン
  • 出力: $50 / 100万トークン
  • 7月7日までは週間上限の最大50%まで追加費用なしで使える
  • 8日以降は使用クレジット（従量課金）経由のみに切り替わる
💸 キャッシュ設計なしで全社展開すると想定外の請求額に驚くかも！

④Sonnet5も同日リリース: コスパ最強の新モデル

2026年6月30日に発表されたClaude Sonnet 5は、Free/Proプランのデフォルトモデルに設定。Anthropicは「Sonnet史上もっともエージェント的」と位置付けている。料金はFable 5の約5分の1で、日常業務の常用モデルとして最適。

  • SWE-Bench Pro 63.2%（Sonnet 4.6の58.1%から向上）
  • Terminal-Bench 2.1 80.4%（前世代67.0%から大幅改善）
  • 料金: 入力$2・出力$10（8月31日までの導入価格）
  • 9月以降は入力$3・出力$15に移行
⭐ Sonnet 5は実用性とコストのバランス型。Fable 5は「重い作業用」という違いを押さえよう

⑤Fable5でできること: 長い資料の読み込みと整理

Fable 5の最大の強みは100万トークンの長文脈処理。複数のPDF、長い議事録、仕様書、契約書、研究資料などをまとめて読ませて、重要な論点や矛盾点を整理させることができる。

  • 複数の長文書を一度に読み込んで全体を踏まえた判断ができる
  • 従来のAIのように途中の情報を見落としにくい
  • 「重要な論点」「矛盾点」「次に確認すべきこと」の抽出に強い
  • note記事の構成案、専門用語のやさしい言い換えも得意
📚 「資料を3つ渡して要点を整理して」みたいな依頼が一番得意な使い方だよ

⑥曖昧な依頼からでも自律的に作業を進める

Fable 5は指示がざっくりでも自分で作業を分解して進める能力が高い。「簡単なWebアプリを作って」「この資料をもとに営業資料を作って」といった依頼に対し、状況を把握してファイルや制約を確認し、作業手順を組み立てて進める。

  • 環境を探索してファイルや制約を把握してから実装に入る動きが強い
  • CodeRabbitのレビューでも「曖昧なプロンプトに強い」と評価
  • ただし作業が長くなりやすく、コストや時間が膨らみやすい指摘も
🤔 「一問一答のAI」ではなく作業を任せるAIエージェント寄りと理解しよう

⑦コーディングやWeb制作で真価を発揮

HTML/CSS/JavaScriptでのページ作成、既存コードの修正、仕様に沿った機能追加、バグの原因調査などに強い。複数ファイルをまたぐ実装や仕様を読みながら構造を考える作業で特に力を発揮する。

  • 大規模なリファクタリングにはFable 5が最適
  • 問題を広く見つける力があるが、指摘が多すぎる場合も
  • 本番コードレビューの標準モデルに即置き換えは推奨されない
  • 自律的なコーディング作業に選択的に使うのが正解
🔧 「開発者を不要にするAI」ではなく開発作業の初速を上げるAIと見るのが現実的だね

⑧動画・デザイン制作の補助にも活用できる

Claudeのデザイン制作機能と組み合わせることで、Web制作や簡単なアニメーション制作にも活用可能。記事URLとざっくりしたイメージを渡すだけで、15〜25分ほどで解説動画のたたき台が完成した検証例がある。

  • 企画・構成・簡易アニメーションの下書き作成に適している
  • 完全自動化ではなく「たたき台」を作るAIとして見るのが現実的
  • 音声がつかない点や出力機能の制約には注意
  • トークン消費が非常に大きいことにも留意
🎬 動画制作の企画から簡易アニメーションまでをかなり短時間で作れるのは革命的

⑨Fable5の弱点と注意点: コスト・時間・安全制限

Fable 5は強力だが過大評価は危険。コストが高いことに加え、作業が長くなりやすい傾向がある。深く考えてくれる反面、必要以上に探索したり出力が膨らんだりすることがある。

  • 安全分類器により特定の依頼は拒否される（stop_reason: "refusal"）
  • 学習時点以降のニュースや価格情報は知らない可能性がある
  • 事実確認、法的判断、医療判断をAIに丸投げするのは危険
  • AIは「優秀な作業者」にはなれても「責任者」にはなれない
⚠️ 人間が目的・条件・判断基準を明確にして使うことが必須！

⑩Fable5 vs Sonnet5: 性能とコストの徹底比較

2つのモデルは「性能の高さ」ではなく費用対効果で使い分けるのが正解。複雑なコーディングや長時間の自律エージェント作業はFable 5、日常業務の常時運用はSonnet 5という棲み分けが現実的。

  • Fable 5 SWE-Bench 80.3% / 入力$10・出力$50
  • Sonnet 5 SWE-Bench 63.2% / 入力$2・出力$10（導入価格）
  • コスト比は約5:1。Fable 5はSonnet 5の約5倍のコスト
  • 定型業務では品質差を体感できないケースも多い
📊 「2週間試したけど品質はほぼ変わらず請求額だけ上がった」という声あり

⑪他のAIとの比較: ChatGPT・Geminiとの違い

ChatGPT（GPT-5.5系）は幅広くこなす万能型、GeminiはGoogle製品との連携やマルチモーダル処理に強い統合型。Fable 5は長い資料や複雑な判断に強い重量級モデルという明確な違いがある。

  • ChatGPT 文章・相談・分析・資料作成まで万能
  • Gemini Google検索や画像・動画連携に強い
  • Sonnet 5 日常的な作業を自律的に進める実用型
  • Fable 5 長時間の複雑な作業やコーディングに強い重量級
🔍 モデルごとの得意・不得意を理解して使い分けることが大事！

初心者がFable 5を使うなら、まずは用途を絞るのがおすすめ。「すごいAIだから何でも聞く」ではなくここぞという重い作業で使うのが正解。

  • 📄 長い資料を読ませて要点を整理する

  • 💭 自分の企画を壁打ちしてもらう
  • 💻 コードの意味を説明してもらう
✅ 「今日の投稿文を3つ考えて」くらいならSonnet 5で十分だよ

⑭用途別の判断フロー: どっちを選ぶべきか

以下の順番で自問するとFable 5かSonnet 5かの判断がしやすくなる。業務ごとに使い分ける設計が現実的な着地点。

  • ❶ 複雑なコーディング・大規模リファクタリング？ → YESならFable 5
  • ❷ 毎日・毎時間走らせる常時稼働のエージェント？ → YESならSonnet 5
  • ❸ 1週間止まったら事業に致命的な影響？ → フォールバック設計を組む
  • ❹ 予算は月いくらまで確保できる？ → Fable 5は約5倍のコスト差
🎯 定型業務はSonnet 5、一発勝負の難プロジェクトはFable 5って感じ

⑮可用性リスク対策: 単一モデル依存は危険

Fable 5の3週間停止の歴史を踏まえると、常時稼働の業務をFable 5一本に依存させるのは危険。Sonnet 5が新たに選択肢に加わった今こそ、フォールバック設計を見直す価値がある。

  • 自動化パイプラインをFable 5に全面依存した企業ほど影響が大きかった
  • 1つのモデルが使えなくても事業が止まらない設計が必要
  • Sonnet 5 + Opus 4.8 + Fable 5の組み合わせで分散リスク
  • 代替モデルへの切り替え手順を事前に文書化しておく
🛡️ 「Fable 5一本で全部回す」は最大の失敗パターン！

⑯コスト試算: 同じ作業量でここまで差が出る

月間で入力30M・出力8Mトークンを消費するエージェント運用を想定した場合、Fable 5は月$700、Sonnet 5（導入価格）は月$140。約5倍のコスト差が生じる。

  • Fable 5: $10×30 + $50×8 = 月$700
  • Sonnet 5（〜8/31）: $2×30 + $10×8 = 月$140
  • Sonnet 5（9/1〜）: $3×30 + $15×8 = 月$210
  • ただしキャッシュ活用でFable 5の入力コストは最大90%割引に
💰 キャッシュ設計なしで全社展開すると月$700は現実の数字になるかも

⑰よくある失敗パターン: 4つの落とし穴

モデル選定でよくある失敗を4つに整理。これを避ければFable 5とSonnet 5の使い分けはスムーズに進む。

  • ❶ ベンチマークの数字だけで最上位を選ぶ → 品質差が体感できないケースも
  • ❷ 7月7日の無料枠終了を把握せず放置 → 想定外の課金リスク
  • ❸ Sonnet 5をデフォルトのまま思考停止で使う → 難タスクで性能不足に
  • ❹ 単一モデルに全業務を依存 → 停止時に事業が止まるリスク
🚨 どれも実務で実際に起きた失敗だからしっかり覚えておこう！
