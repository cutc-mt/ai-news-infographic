① Cavemanテクニックの概要: AIを原始的な言葉で喋らせる

  • Caveman（原始人）プロンプトはRedditユーザーが発見した手法。AIの返答から無駄な言葉を削ぎ落とすシステムプロンプト
  • 通常のClaudeが180トークン使う検索タスクが、Caveman Claudeではわずか45トークンに圧縮される
  • JuliusBrusseeさんがGitHubでオープンソースのスキルとして公開。Claude Codeに1コマンドで導入可能
  • GitHub、Hacker News、開発者界隈で大バズり。ミームから本格的な運用ツールへと進化した
AIに「お手伝いさせてください」なんて 言わせるお金を節約するのが Cavemanの核のアイデアです。

② 出力トークントラップ: コストは見えない場所に隠れる

  • すべてのClaudeの返答には2つのトークン枠がある。思考トークン（内部推論）と出力トークン（表示されるもの）
  • Caveman圧縮は出力トークンにしか作用しない。思考と推論はそのまま維持される
  • 2026年3月の論文で、短い回答を強制すると特定のベンチマークで精度が26ポイント向上したことが確認された
  • 簡潔さはClaudeを劣化させない。集中力を高める効果があるという研究結果
長々とした前置きを削って 本質だけ答える。 AIの頭の良さは変わらない。

③ 変換の実例: 69トークン→19トークンの圧縮

  • 通常のClaude: 「The reason your React component is re-rendering is likely because you're creating a new object reference on each render cycle」
  • Caveman Claude: 「New object ref each render. Inline object prop = new ref = re-render. Wrap in useMemo.」
  • 技術的な中身は完全に同一。トークン数は69→19で72%削減
  • 冠詞（a, an, the）、フィラー（just, really, simply）、敬語（sure, certainly）をすべて削除するルール

⑧ 実際の削減率: タスクで22〜87%

⑩ Cavemanが効かない場面: エージェントワークフローの壁

  • エージェントの自律ワークフローでは節約効果が薄い。エージェントの通信ターンが全体のコストを支配するから
  • Claude Codeで8回のツール呼び出し（ファイル読み・検索・実行）を行うと、入力トークンだけで15,000消費
  • 最終回答で100出力トークンを節約しても、セッション全体の節約率は1%未満
  • 最適な用途：単発回答（コードレビュー・関数の説明・エラー解説）。苦手：深度のあるワークフロー

⑪ Auto-Clarity機能: 危険な操作では自動解除

  • CavemanスキルにはAuto-Clarityという安全機能が内蔵されている
  • ファイルの削除や認証ロジックの変更など危険な操作を検知すると、自動的に圧縮をオフにする
  • その時は完全な冗長な警告が表示される。安全が最優先の設計
  • タスクの30%が危険操作の場合、平均圧縮率は75%から50〜55%に低下

全力で警告してくれる。

⑫ Caveman + キャッシュの最強コンボ

  • Claudeのプロンプトキャッシュと組み合わせるのが正解。キャッシュ読み取りは通常の10%のコスト
  • キャッシュは5分間有効。1回のキャッシュ読み取りで元が取れる設計
  • 入力をキャッシュして出力をCavemanで圧縮する。両方向からコストを削るのが最大の戦略
  • さらにBatch API（非同期で50%割引）を追加すれば、理論上は97〜98%の総削減が可能

⑬ 学術的圧縮手法との違い: 入力 vs 出力

  • MicrosoftのLLMLingua-2: 小さな言語モデルでプロンプトの不要トークンを削除。入力を圧縮
  • Caveman: システムプロンプトで出力を圧縮。まったく別のアプローチ
  • LLMLingua-2は2〜5倍の圧縮率。RAGパイプラインで50,000トークンの文書を文脈に詰める時に有効
  • 両者は補完関係。LLMLinguaで入力を縮め、Cavemanで出力を縮めるのが最強の組み合わせ

⑰ 自分で測る: 5ステップのテスト方法

  • 自分のよく使う5つのタスクをリストアップする（コードレビュー・関数説明・デバッグなど）
  • 通常モードで各タスクを実行。返答をコピーしてトークン数をカウント

  • 各タスクの実際の削減率を計算。平均60%以上なら本番投入の価値あり
  • 月間API呼び出し数を掛けて月間の節約額を算出。セットアップ時間のコストと比較
