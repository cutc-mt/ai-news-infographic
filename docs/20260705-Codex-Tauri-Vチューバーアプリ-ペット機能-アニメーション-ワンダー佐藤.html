Codex × Tauri でVチューバーアプリ開発

ペット機能にアニメーションを組み合わせた作り方を徹底解説！

📅 2026年7月5日

①プロジェクトの発端: ペット × Vチューバー

  • ワンダー佐藤がCodexのペット機能を活用してVチューバーアプリを開発
  • デスクトップの右下で口がパクパク動くキャラクターが完成
  • 元々ZoomのデフォルトアバターでVチューバーを始めていた経歴を持つ
アイデアの起点はCodexのペット機能！
そこからVチューバーアプリへの道が開かれた

②Codexのペット機能とは

  • Codex上にオリジナルキャラクターを常駐させる機能
  • 画像をアップロードするだけでペットを作成可能
  • カメラで自分の口を認識させて、ペットの口も動かせる
  • サブエージェントを使って自動生成することもできる
プロンプトに「このキャラでペットを作って」と言うだけ！
21分ほどでペットが完成した実績あり

③ブラウザ版の制約: 最前面とリップシンク

  • ブラウザでは常に最前面に表示できない
  • ブラウザでは音声認識（マイク）へのアクセスが制限される
  • パソコンの全機能にアクセスするにはネイティブアプリの権限が必要
  • HTMLで作りやすいが、この2つの制約が壁になる
だからElectronやTauriのような
ネイティブアプリ化フレームワークが必要に！

④Tauriを選んだ理由: 軽さが正義

  • Electronは重い → Tauriなら軽量ネイティブアプリが作れる
  • Tauriでネイティブアプリ化すれば最前面表示が可能に
  • Rustベースでパフォーマンスが良い
  • ファイルサイズも小さく、配布に適している
プロンプトに「Tauriで作って」
と言うだけ！意外と簡単にネイティブ化できる

⑤DALL-E 2.0で精密アニメーション素材

  • DALL-E 2.0はレイヤー構造を持つ → 部分的な変更が可能
  • 口だけの変化ならDALL-E 2.0が最適
  • GPT Image 2.0だと体まで動いてしまう問題あり
  • コードベースのレイヤー設計 → ピンポイント編集ができる
画像をアップロード → 口の部分を
閉じる指示 → 口だけ動く素材が完成！

⑥代替手法: 画像の部分切り取り

  • DALL-E 2.0がなくても口の部分だけを切り取る方法で対応可能
  • 口が開いた状態・半開き・閉じた状態の3パターンを用意
  • Codexに画像を投げて部分的に切り取ってもらう
  • 切り取った口の画像を差し替えるだけでリップシンクが実現
レブ2.0を使わなくてもやれる！
Codexの画像処理能力だけで十分対応可能

⑦命を吹き込む3段階アニメーション

  • 第1段階: 口が開いた・閉じたの2パターン
  • 第2段階: 半開きを追加して3パターンに
  • 第3段階: 目のパチパチ（開き・半開き・閉じ）も追加
  • 3パターン以上用意すると滑らかな動きになる
2段階だけだとカクカクしてしまう
3段階以上が滑らかさの秘訣！

⑧リップシンクの実装: 音声で口を動かす

  • 音声入力でリップシンクさせるのが一番簡単
  • 映像（カメラ）でリップシンクも技術的には可能だが少し難易度が高い
  • プロンプトに「声と口を合わせてリップシンクするように」と指示
  • マイクの感度調整で外のノイズ対応も可能
声だけでリップシンク！
映像認識より音声の方が確実で簡単

⑨ノイズ除去: 誤作動を防ぐノイズゲート

  • 外の音がうるさいと口が勝手にパクパク動いてしまう
  • ノイズゲート（敷位）の強さで閾値を調整可能
  • 静かな部屋で使うならこの機能は不要
  • オン/オフで自由に切り替えられる設計
外でガガガッと音がしたら
口がポンポン動いちゃうから要注意！

⑩アイドル動作: 生き生き感を演出

  • 何もしていない時の自動アニメーション（アイドル状態）
  • 瞬き・手を少し動かす・尻尾を振る などを設定
  • 複数の画像を放り込んで、それぞれの動作に割り当て
  • 冷たさが消えて生き生きした温かみがグっと増す
静止画のままだと冷たい…
アイドル動作で命が吹き込まれる！

⑪ドラッグ＆スケール: 自由な配置

  • キャラクターをドラッグして自由な位置へ移動
  • サイズも環境に合わせて自由に変更可能
  • 何かに隠れてしまった時の調整にも便利
  • 自然言語で指示してもらえばちゃんと実装してくれる
星をドラッグで大きくしたり
小さくしたり自在に操作できる！

⑫次のステップ: モーション連動

  • 体が傾いたらキャラクターも傾く姿勢認識連動
  • 手を上げたらキャラの手も上がる動作
  • カメラでの姿勢認識技術を活用
  • もう少し進化させれば実装可能な次元
今はまだ先の未来…
でも挑戦すればできるかも！

⑬開発手順の全体像: 4ステップ

  • ステップ1: Codexでペットキャラクターを作る
  • ステップ2: アニメーション素材（口・目など）を生成
  • ステップ3: HTMLでリップシンク機能を実装
  • ステップ4: Tauriでネイティブアプリ化して完成
ペット作って → アニメーション作って →
Tauri化して → アプリ化する流れ！

⑭アプリの構造: コントロールパネル付き

  • 最前面にキャラクターの透過表示ウィンドウ
  • コントロールパネル: 音声・ノイズ除去・カメラ切替
  • 音声モードとカメラモードの切り替えが可能
  • 手や体も動かせる拡張性がある設計
音声モードが一番簡単！
カメラモードは少し上級者向け

⑮使用ツール: Core Agent × Codex

  • CoreMD: AIエージェントの基本設計とルーター定義
  • ThinkingMD: AIの思考・実行方法を決定
  • RuleMD: ガードレールとしての安全ルール
  • CodingMD: コーディングルール（自動的に読み込まれる）
4つのMDファイルでAIエージェントを
構造化して管理しているワケ

⑯サブエージェントで自動生成

  • サブエージェントを使うか、単独で作るかの選択肢がある
  • サブエージェント使用時は約21分でペットが完成
  • 裏でサブエージェントが動き、自動的にプランニング＆実行
  • 色々なファイルを読み込ませると精度が上がる
放っておいても21分で完成！
AIに任せる時代が来た
