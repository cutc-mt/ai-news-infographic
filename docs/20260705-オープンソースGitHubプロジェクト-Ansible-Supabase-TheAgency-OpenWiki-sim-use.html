④Ansible: IT自動化の世界的デファクト

  • Red Hatが主導するOSSの自動化プラットフォーム。GitHubスター数は68K+を超える。
  • プレイブック（YAML）で人間に読める設定を記述し、SSH経由でリモート操作。
  • エージェント不要でセキュア。対象サーバーに追加インストールが不要。
  • ネットワーク設定・クラウド管理・アプリデプロイまで幅広く対応。
「プレインイングリッシュに近い」がキラーフレーズ。
複雑な自動化をシンプルに書けるのが魅力✨

⑤Ansibleのコア機能: どう動くのか

  • Playbook — YAML形式で手順を定義。宣言的に記述できる。
  • Modules — 2,000+のモジュールで各種操作に対応。拡張も容易。
  • Inventory — 管理対象のホストをグループ化。動的インベントリも可能。
  • Idempotent — 何度実行しても同じ結果。安全な繰り返し実行。
---
- name: Nginxをインストール
  hosts: webservers
  become: yes
  tasks:
    - name: Install Nginx
      apt:
        name: nginx
        state: present
    - name: Start Nginx
      service:
        name: nginx
        state: started

⑥Ansibleのエコシステム: AWXとコレクション

  • AWX — Web UI・REST API・タスクエンジン付きの管理コンソール。
  • AWXはDockerで手軽にデプロイ可能。Ansible Automation Platformの上流。
  • Ansible Collections — 機能ごとのパッケージ。コミュニティが多数公開中。
  • AWS・Azure・GCPなどクラウド各社のコレクションが充実。
AWXを使えばチーム协作でのジョブ管理が
Web画面から簡単にできるようになります！

⑦Ansible: まとめと評価

  • 長所: 学習コストが低い・エージェント不要・コミュニティが巨大。
  • 注意点: 大規模環境では速度がネックになる場合も。
  • Red Hatの支援でエンタープライズ利用も安心。
  • 活発な開発が続いており、develブランチで最新機能を試せる。
10年以上の実績があり、信頼性は抜群。
インフラ自動化の第一選択としておすすめ！

⑧Supabase: Firebaseのオープンソース代替

  • PostgreSQLをベースにしたBaaS（Backend as a Service）。
  • 認証・ストレージ・リアルタイム・Edge Functionsをオールインワンで提供。
  • Firebaseに依存したくない開発者に人気No.1の代替。
  • GitHubスター85K+の大人気プロジェクト。コミュニティも活発。
「Firebaseから脱却したい」ならSupabase一択。
PostgreSQLの知識がそのまま活かせます！

⑨Supabaseの主要機能: すべて揃っている

  • Database — フルPostgreSQL。RLSで行レベルのセキュリティ。
  • Auth — Email・SSO・Magic Link・電話認証など多様な方法。
  • Realtime — WebSocketsでリアルタイム同期。チャットや通知に。
  • Storage — S3互換のファイルストレージ。CDN配信にも対応。
  • Edge Functions — Denoベースのサーバーレス関数。

⑩Supabase: 料金とセルフホスト

  • Freeプラン — 500MBデータベース・1GBファイル・5GB帯域。プロトタイプに十分。
  • セルフホスト可能 — Docker Composeで自前サーバーに完全無料で構築。
  • データのエクスポートも自由。ベンダーロックインの心配なし。
# Dockerでセルフホスト
git clone --depth 1 https://github.com/supabase/supabase
cd supabase/docker
cp .env.example .env
docker compose up -d
セルフホストならデータも自分の手元に。
セキュリティ重視のプロジェクトには最適です！

⑪The Agency: Claude CodeのマルチエージェントFW

  • Claude Code向けのマルチエージェント開発フレームワーク。
  • captain・tech-lead・reviewer・devexなど専門ロールのエージェントが協力。
  • 1つのcurlコマンドでセットアップ完了。既存リポジトリにも導入可能。
  • Convention over Configurationで設定不要ですぐに始められる。
Claude Codeを使っているなら必見！
1人で複数エージェントを指揮できます🚀

⑫The Agency: ISCPとセッション管理

  • ISCP — エージェント間の協調プロトコル。SQLite DBでディスパッチ管理。
  • Session Lifecycle — pause/pickupで作業を保存・再開可能。
  • Workstreams — 共有PVR・A&D・Plan・KNOWLEDGEで作業を整理。
  • Quality Gates — pre-commitフック・並列レビューで品質を担保。
エージェント同士が対話して協調する仕組み。
人間はPrincipalとして監督するイメージ！

⑬The Agency: 導入方法とエコシステム

  • Gitリポジトリにcdして1行でインストール。
cd your-project
curl -sL https://raw.githubusercontent.com/
  the-agency-ai/the-agency/main/
  agency/tools/agency-bootstrap.sh \
  | bash -s -- --principal your-name \
  --project your-project-name
  • Hookify — 危険なシェルパターンを検知・ブロックするルールエンジン。
  • MITライセンスでフレームワーク本体は無料。アプリ系はReference Source License。

⑭OpenWiki: AIエージェント向けドキュメント自動生成

  • LangChainが開発したCLIツール。コードベースのドキュメントを自動生成。
  • 特にエージェントがコードを理解するためのドキュメントに最適化。
  • GitHub Actionで毎日自動更新。PRとしてドキュメントの変更を提案。
  • OpenRouter・Fireworks・OpenAI・Anthropicなど複数プロバイダに対応。
npm install -g openwiki
openwiki --init
openwiki "リポジトリのドキュメントを生成して"

⑯sim-use: AIエージェントの「目と手」をモバイルに

  • lycorp-jpが開発。AIエージェントにiOSシミュレーターとAndroidを操作させるCLI。
  • Observe — 画面をトークン効率16倍のアウトラインに変換してLLMに渡す。
  • Act — エイリアス（@N）で要素をタップ。座標不要で高速操作。
  • Observe → Act → Verifyのループで自律的なモバイル開発が可能に。
「エージェントが作ったものをエージェントが検証」
開発ループが完全に閉じる！

⑰sim-use: アーキテクチャと対応コマンド

  • SwiftパッケージでmacOS 14+対応。Metaのidbを内部で活用。
  • コマンド一覧：ui・tap・swipe・type・paste・screenshot・record-video
  • デーモンがバックグラウンドで初期化を約300msに高速化。
  • sim-use initでClaude CodeなどのAIクライアントにスキルを自動インストール。
# インストール
brew tap lycorp-jp/tap
brew install lycorp-jp/tap/sim-use

# 基本的な使い方
sim-use ui            # 画面を読み取る
sim-use tap @9         # 要素をタップ
sim-use swipe 100,300 300,100  # スワイプ
