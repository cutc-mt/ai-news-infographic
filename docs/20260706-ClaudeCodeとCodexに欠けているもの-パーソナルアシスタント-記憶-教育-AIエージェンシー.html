Claude CodeとCodexに欠けているもの

OpenClaw / Hermes との決定的な違いを徹底比較
2026年7月6日

① Claude CodeとCodexに欠けているものとは？

  • 今回のテーマはClaude CodeとCodexに欠けているもの。批判ではなく、ある見方をしたときの違いに迫る
  • 全部同じ「AIエージェント」として扱うのが問題ではないのかという問いかけからスタート
  • 「汎用AIエージェント」「自立型エージェント」など全部を同じように捉えていることに異論
  • チャンネル「AIエージェンシー」が提唱する2つの区分がこの動画の核
同じ「AIエージェント」でも
設計思想が全く違う —
そこを切り分けようというお話。

② 2つの区分 — パーソナルアシスタント vs コーディングアシスタント

  • パーソナルアシスタント：OpenClaw（オープンクロー）、Hermesエージェント
  • コーディングアシスタント：Claude Code、Codex、OpenCode、Clineなど
  • この2つの区分は「AIエージェント」で括るには大きな違いがある
  • 区別しないまま使うと、錯覚が生まれてツールの真価を引き出せない
「AIエージェント」という
言葉で全てを一緒くたにするのが
そもそもの間違いの起点。

③ コーディングアシスタントの正体 — あくまで「道具」

  • Claude CodeもCodexもどこまで行ってもツール・道具。設計思想がそう
  • 捉え方としてはWord・Excel・PowerPointと同じ。人間が使いやすいように作られた道具
  • UI周りも人間中心に設計されている。人間が使い方を学ぶスタンス
  • 自然言語でチャットができるからパートナーの錯覚を起こしやすい
チャットで話せるからって
「有能な秘書」じゃない —
超便利なハンマーみたいなもの。

④ パーソナルアシスタント — 「パートナー」としての関係性

  • OpenClawやHermesは自分とパートナーの関係性。道具とは違う立ち位置
  • 同僚や部下、パートナーとして扱える存在になる
  • 自分の仕事や会社の方針、ルールを伝えればその文脈で動いてくれる
  • 「自分対パートナー」か「自分対道具」か — この関係性の違いが全てを変える
秘書のように振る舞うか、
工具箱の中の道具か —
これが決定的な違い。

⑤ 最大の差 — 「記憶」と「自己改善ループ」

  • OpenClawが大ブレイクした要因の1つは記憶周りの強さ
  • 自己改善ループが回る設計。「覚えておいて」と言えば覚えてくれる
  • 同じ失敗を繰り返さない。記憶メモリー周りで圧倒的な強さを発揮
  • Claude CodeやCodexにはこの記憶層が欠けている。設計思想の根本的な違い
記憶があるから「人間」に
近づく。記憶がないと毎回白紙の
やり取りになってしまう。

⑥ 教育できるかどうか — 最大の違い

  • 「自分対パートナー」であれば教育することができる。相手を成長させられる
  • 指導すればするほどより良い仕事をしてくれるようになる設計が可能
  • 「自分対道具」の場合は教育できない。人間側が使い方を覚えるだけ
  • パーソナルアシスタントは成長のループが組める。道具には成長の概念がない
部下に指導できるか、
ハンマーの使い方を
覚えるだけか —
天と地ほどの差。

⑦ OpenClawの衝撃的デビュー — 今年の幕開け

  • 今年の初め、OpenClawが衝撃的なデビューを果たした
  • 強い権限を持った自律的に行動できるエージェントとして話題に
  • その人気に対抗馬としてHermesエージェントが登場した
  • 海外国内を含めてAIエージェントの話題が一気に加速した年
OpenClawの登場が
「AIアシスタント」の概念を
大きく変えた記念すべき年。

⑧ Claude CodeとCodexの台頭 — 開発者から一般層へ

  • AnthropicやOpenAIから出ている基盤モデルのCLIツール
  • Claude Codeは当初エンジニア中心に使われていたツールだった
  • 人気が加速してビジネスパーソンや非エンジニアも使い始めるように
  • 今ではClaude Code DesktopやCodex AppのようなGUIアプリとしても使える
開発者だけのツールから
誰でも使えるアプリに進化。
でも中身はまだ「道具」。

⑨ AIの歴史 — チャットから自律型へ

  • 第1世代：ChatGPTの登場。1回質問して1回答えが返ってくるピンポン方式
  • 第2世代：Computer Use、Browser Useの登場。PCやブラウザを操作できるようになった
  • 第3世代：CLIの登場。真っ黒い画面で手がついて自由に動けるようになった
  • 第4世代：Claude Codeの登場。ゴールを渡せば自立的に計画を立てて実行
ピンポン → 操作 → CLI → 自律型。
進化の過程を辿ると
「道具」と「パートナー」の違いが
くっきり見えてくる。

⑩ チャットの錯覚 — 会話できるからパートナーではない

  • Claude CodeもCodexも普通にチャットで会話できる。だから錯覚が生まれる
  • 会話できるからといって秘書のように振る舞うわけではない
  • 会社の方針やルールを伝えても、その文脈で自律的に動く設計ではない
  • 初めから錯覚していると、永遠に人間が使い方を学ぶ側のまま
チャットができる ≠ パートナー。
この見かけの類似性が
最大の罠。

⑪ 欠けているポイント — 方針決定と教育の仕組み

  • Claude CodeとCodexに欠けているのは「方針を決める」「教育する」「成長させる」仕組み
  • ツールは人間が使い方を学ぶもの。AIを教育する発想が設計に組み込まれていない
  • これからの仕事は自然言語で指示を出して全てやってくれる方向へ進む
  • だからこそ教育とコミュニケーションの仕組みが不可欠になる
道具は使うもの。
パートナーは育てるもの。
この視点の転換が重要。

⑫ G-Brain × Claude MD — 記憶強化の実践

  • チャンネル主が1週間格闘した結果、G-Brainというマニュアルを導入
  • Claude CodeやCodexでも記憶のメモリ部分を強化すればパーソナルアシスタント化が可能
  • 手を入れれば有能な秘書として使えるようになる
  • この記憶強化こそが「道具」を「パートナー」に変える鍵
G-Brainで記憶を強化したら、
Claude Codeが秘書として
動き始めたそうです！

⑬ パラダイムシフト — 自分が使うのではなくAIが使う

  • これからの仕事：自分が道具を使いこなすのではなく、AIが道具を使いこなす
  • AIエージェントが道具を使いこなすことができる。人間の役割が変わる
  • 重要になるのはコミュニケーションと教育。そこを回すことが鍵
  • 全てのワークフローがコミュニケーションで成り立つようになる
「人間が道具を使う」→
「AIが道具を使う」
この転換がこれからの仕事の形。

⑭ 記憶の重要性 — メモリ設計が全てを決める

  • OpenClawもHermesもデフォルトで記憶が強い。これが差別化ポイント
  • サードパーティ製のメモリーレイヤー強化ツールも登場している
  • Claude CodeやCodexにもサードパーティの記憶強化を導入することは可能
  • 公式も記憶周りの強化に取り組んでいるが、設計思想の根底は異なる
メモリ設計はとっても重要。
記憶があるかないかで
「道具」と「パートナー」の
境界線が決まる。

⑮ 正しい切り分け — 3つの関係性

  • 自分のパートナーとして扱う：OpenClaw、Hermes — 教育できて成長する
  • 自分の部下として扱う：指示を渡して自律的に動いてもらう存在
  • 自分の道具として扱う：Claude Code、Codex — 人間が使い方を学ぶ
  • どっちなのかを切り分けた上で、何が足りないかを見極めるのが正解
パートナー・部下・道具 —
この3つを明確に分けることで
足りないものが見えてくる。

⑯ サードパーティの解決策 — 記憶を後付けする

  • 公式は記憶の強化に取り組んでいるが、設計思想の違いは依然としてある
  • サードパーティからメモリーレイヤーを強化するツールが多数登場
  • こうしたツールを導入していくことで不足部分を補える
  • G-Brainのようなマニュアル入力で記憶を後付けする手法も有効
欠けているなら足せばいい。
サードパーティ製ツールで
「道具」を「パートナー」に近づける。

⑰ これからの仕事のあり方 — コミュニケーションが主軸に

  • 対象が自分対パートナーになれば、全てはコミュニケーションで成り立つ
  • 重要なスキルはコミュニケーション能力と教育能力。技術力はAIに委ねる
  • ワークフローがコミュニケーション中心に変わる。ツールの使い方を覚える必要が減る
  • 「人間がAIを使う」から「人間がAIと共に働く」への転換
ツールのマニュアルを読むのではなく、
AIとの会話で仕事が回る —
これがこれからの日常。

⑱ まとめ — 道具かパートナーか、その選択が未来を変える

  • Claude CodeやCodexを批判しているわけではない。設計思想の違いを指摘しているだけ
  • 最大の欠落は記憶と教育の仕組み。これが「道具」と「パートナー」を分ける
  • サードパーティの記憶強化を導入すれば、パーソナルアシスタント化は可能
  • これからの仕事は「AIが道具を使いこなす」世界。コミュニケーションと教育が主軸になる
道具として使うか、
パートナーとして育てるか —
その選択があなたの
仕事の未来を決める。
