🧠 Claude Fable5はいつ使うべき？
4つのモデルの使い分け基準を徹底解説！

AI整体師
①

Claude Fable 5とは

  • Anthropicが一般提供する最も高性能なモデル
  • 従来のOpus・Sonnet・Haikuの上に位置するMythos-class
  • 2026年6月9日にClaude Mythos 5と同時にリリース
  • 6月12日〜30日は米政府の輸出管理指令で一時停止
  • 7月1日から分類器を更新してグローバル提供を再開
Opusのさらに上の最上位モデルがついに登場！👑
②

Mythos 5との違い

  • Fable 5とMythos 5は同じ基盤モデル
  • 違うのはセーフガードの強度と提供範囲
  • Fable 5は一般利用者向けに提供
  • Mythos 5はProject Glasswing経由の招待制限定提供
  • 料金とAPI挙動は完全に同じ
同じエンジンに違う安全装置の組み合わせ！🔧
③

安全性の仕組み

  • 分類器（Classifier）がすべてのリクエストを監視
  • 対象はサイバーセキュリティ・生物学・化学・蒸留の3領域
  • 検出時は自動的にOpus 4.8へリダイレクト
  • ユーザーには「Opusが応答しました」と通知される
  • 発生率はセッションの5%未満
分類器に引っかかっても諦めないで！💬
④

4モデルのスペック比較

  • Fable 5: 入力$10 / 出力$50・100万トークンあたり
  • Opus 4.8: 入力$5 / 出力$25・Fableのちょうど半額
  • Sonnet 4.6: 入力$3 / 出力$15・速度と知性のバランス型
  • Haiku 4.5: 入力$1 / 出力$5・最速・最安
  • 全モデルが100万トークンのコンテキスト（Haiku除く）
Fable 5はOpusの2倍の値段！用途を選ぼう💰
⑤

Fable 5が真に優れる領域

  • 数週間かかるエンジニアリング作業を数日に短縮
  • Stripeは5000万行のRubyコードの移行を1日で完了
  • Hebbia金融ベンチマークで全モデル中最高スコア
  • スクリーンショットからWebアプリのソースコードを再構築
  • Pokémon FireRedをビジョン入力のみで完全クリア
短いタスクじゃなくて長期作業で本領発揮！🚀
⑥

コーディング性能の圧倒的差

  • SWE-Bench Pro: Fable 5が80.3%（Opusは69.2%）
  • FrontierCode Diamond: Fable 5が29.3%（GPT-5.5は5.7%）
  • Terminal Bench: 88.0%でトップ
  • FrontierCodeではOpusの約2倍、GPT-5.5の約5倍
  • Fable 5のmedium設定がOpusのmaxを上回ることも
難しいほど差が広がるのが特徴！🏆
⑦

努力レベルの使い分け

  • low / medium: 速くて安価、日常的なタスク向け
  • high: ほとんどの実務で推奨されるデフォルト
  • xhigh: 最も難しく能力が求められるタスク向け
  • Claude.aiには努力レベルの切り替えスイッチなし
  • プロンプトの言い回しで同じ効果を得られる
「徹底的にやって」で高努力レベルが使える！🎯
⑧

Fable 5を使うべき場面

  • 大規模なリファクタリング・数時間〜数日の自律実行
  • 密度の高いドキュメント分析・複数ソースの調査
  • 特定のトーンで書かれた完全な長文ドラフト
  • 解釈が必要なスプレッドシートやデータ分析
  • 実際の選択肢を比較する意思決定分析
従来モデルでは届かなかった領域に投入！🎯
⑨

Fable 5をスキップすべき場面

  • 速い回答・1文の出力・クリエイティブなやり取り
  • 予算が厳しく大量のルーチンタスクを処理する場合
  • レイテンシが重要なリアルタイムツールやパイプライン
  • サイバー・生物学・化学が仕事の中心の場合
  • 短いメールの要約や簡単な質問への回答
荷物1つ運ぶのに貨物エレベーターは使わない！📦
⑩

4モデルの最適な使い分け

  • Fable 5: 最難関の推論・長時間自律エージェント
  • Opus 4.8: 日常のコーディング・設計・万能型
  • Sonnet 4.6: 速度と知能のバランス・量をこなす
  • Haiku 4.5: 分類・要約・大量処理・低レイテンシ
  • Sonnetで試して力不足ならFableに引き上げる
下から上へ段階的に使い分けるのが正解！⬆️
⑪

Claude Codeでの使い方

  • Claude Code v2.1.170以降が必要
  • 「claude update」で最新版に更新
  • セッション中に/model fableで切替
  • claude --model fableで起動時から指定
  • settings.jsonのmodelキーでデフォルトにピン留め
/model fable で一発切替！💻
⑫

アクセス方法とチャネル

  • Claude.ai: モデルドロップダウンからFable 5を選択
  • Claude Code: デスクトップアプリ・CLIの両方で対応
  • Claude Cowork: デスクトップアプリ内で利用可能
  • API: モデル文字列claude-fable-5を使用
  • Pro・Max・Team・Enterpriseの有料プランが必要
Bedrock・Vertex AI・Foundryは再開待ち！⏳
⑬

料金と利用条件

  • 7月1日〜7日: 週間使用枠の50%まで追加料金なし
  • 7月8日以降: Usage Credits消費で従量課金
  • $50で10%OFF・$250で20%OFF・$1000で30%OFF
  • Batch APIで50%割引が適用される
  • キャッシュ読取は90%割引でコスト大幅削減
7月7日まで無料枠で試せるチャンス！💵
⑭

思考は常時オン

  • 適応的思考（Adaptive Thinking）が常時有効
  • thinkingパラメータのdisabled指定は400エラー
  • 思考の深さはeffort（low〜xhigh）で制御
  • 生の思考過程は返されず要約のみ取得
  • temperature・top_p・top_kは廃止（400エラー）
常に深く考えてから回答するようになった！🧠
⑮

よくある5つの間違い

  • 小さなタスクに使うこと → Sonnet/Haikuで十分
  • 即時の返信を期待すること → 数分かかることも
  • 分類器の拒否を最終回答とみなすこと
  • 停止条件を伝えないこと → 余計な作業が増える
  • 内部推論を説明させること → 分類器が作動
これらを避ければFable 5を最大限活用できる！⚠️
⑯

プロンプトのコツ

  • 一番難しい問題から任せるのが正解
  • 指示は簡潔に → 列挙より一文で伝える
  • 進捗報告に根拠を要求する
  • やってほしくないことの境界を明示する
  • 並列サブエージェントとメモリを活用
細かすぎる指示は逆効果！ゴールを明確に！💡
⑰

分類器に引っかかったら

  • 背景情報を追加 → なぜその質問をしているか説明
  • 無害な目的を具体的に言い換える
  • 分類器は「意図のシグナル」を見ている
  • 推論確認はClaude Codeの思考出力を使用
  • 「思考プロセスを表示」は蒸留分類器を誤作動させる
文脈を追加すればだいたい通る！🔄
⑱

使い分けの判断基準まとめ

  • まずSonnet 4.6で試す → 速度とコストが最適
  • 力不足ならOpus 4.8に引き上げる
  • それでも届かないならFable 5を使う
  • 大量処理はHaiku 4.5で高速・低コスト
  • 「難度に応じて上げる」が一番賢い使い方
簡単な順から順番に試すのがコスパ最強！🧭
