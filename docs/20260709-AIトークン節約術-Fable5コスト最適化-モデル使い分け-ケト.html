💰 AIトークン節約術
高額AIモデルを賢く使い倒す方法

📝 チャンネル: ケト
①

なぜ今、トークン節約が重要？

  • Fable 5やGPT 5.6 Ultra Solなど最新モデルはコストが高い
  • モデルの性能が上がるほど使用量も増大する傾向にある
  • AI界隈で料金の節約方法が最も旬の話題
  • 節約の仕方を知らないと大量のお金を払うことになる
頭の良いAIほど賢く使おう！
②

まずは使用量を把握しよう

  • Claude: 設定画面の左下アイコン → 使用量を確認
  • Codex: Codex画面から使用量を確認可能
  • セッション終了時にMac通知で「〇〇円使いました」と表示するツールを作成
  • コマンド入力欄にリアルタイム費用を表示するカスタムスキル
「何%使ったか」より「何円使ったか」が分かる方が意識できる！
③

同じチャットで会話を続けない

  • 同じチャットルームで会話を続けると毎回全ての履歴を読み込む
  • 2回目のやり取りで1回目の内容を、3回目で1〜2回目の全てを再読み込み
  • やり取りが増えるほどトークンの無駄遣いが指数関数的に増加
  • /clearコマンドで文脈をリセットして新しい会話を開始
前後の文脈が必要ならそのまま続けてOK！バランスが大事
④

1回の指示でやりたいことを全部並べる

  • 例: 資料作りで構成案→スライド→画像→原稿を1回で指示
  • 分けると4回のやり取りになり、毎回コンテキストを読み直す
  • 1回でまとめるには裏側のスキル（ワークフロー）が鍵
  • うまくいかない場合は2回に分けるなど適切に調整
スキルをしっかり作り込んでおくと1発でまとまる確率UP！
⑤

思考時間（エフォート）を低めに設定

  • 最近のAIは裏側で推論を何往復も行ってから回答する
  • Claudeの設定で低い/高い/最高/マックスの4段階調整可能
  • エフォートを低めにするほどトークンを大幅節約
  • 文章の推敲なら最低レベルで十分、アプリ開発なら高め
エフォートを下げると回答品質も下がる。バランス重要！
⑥

モデル別のエフォート設定戦略

  • Fable 5: 公式によると「高い」で十分（他モデルのマックス以上）
  • Opus 4.8: マックス推奨（複雑なタスクを任せるなら）
  • Sonnet: マックスでも全然OK（安いので使い切っていい）
  • モデルによって最適なエフォートは違うことを意識
高いモデル×高エフォート＝最強だけど最も高い！
⑦

コネクターの設定をオフにする

  • GmailやGoogleドライブなど接続アプリがオンだと毎回読み込みに行く
  • Fable 5は賢すぎて推測して勝手に検索し始める
  • 接続先ごとにトークンがかかっているのと同じ状態
  • 必要ない時はボタン1クリックで全てオフにする
全部オンにしてた自分、気づかなかった…
⑧

出力を完潔にまとめさせる

  • トークン料金は入力＋出力＋思考の全てにかかる
  • 「300文字以内にまとめて」など数字で出力を制限
  • 画像生成のプロンプトも極端に短くさせる
  • 20項目で箇条書きなどボリューム指定で無駄を削る
出力を短く指定するだけで、その分のトークンが浮く！
⑨

ファイルは必要な部分だけ渡す

  • ExcelやPDFをまるっと添付すると全て読み込まれる
  • 本当に必要なデータ部分だけを抽出して渡す
  • 不要なシートや余計な列を事前に削除してから添付
  • 一見めんどくさいがトークン節約の文脈では非常に大事
余計なデータを読み込む＝お金をドブに捨てるのと同じ！
⑩

Claudeプロジェクト機能でキャッシュ活用

  • プロジェクト内にアップしたファイルはキャッシュ状態になる
  • 1度読み込んだデータは次回は高速に再読み込み
  • Webサイトと同じ仕組み:1回目は遅い、2回目以降は早い
  • 繰り返し使うPDFやPowerPointはプロジェクトに入れておく
キャッシュが効くだけで大幅にトークンを節約！
⑪

メモリ機能のメリットとデメリット

  • メモリONで過去のチャットを覚えてくれる
  • 同じプロジェクトの説明を0からしなくて済む
  • 逆に邪魔な記憶が邪魔をするリスクもある
  • 同じ作業の繰り返しならON、多様な仕事ならOFFがおすすめ
自分の使い方に合わせて判断！銀の弾丸じゃない
⑫

サブエージェントでモデルを使い分け

  • プランニング: Fable 5でハイ・エフォートで最難関タスク
  • 実行: Opus 4.8でマックス・エフォートで作業
  • 確認: Sonnet 5で十分軽量モデルでミスチェック
  • /agentsコマンドでサブエージェント設定を細かく調整可能
Fable 5に確認作業はさせない！最難関だけ任せる
⑬

/compactコマンドでコンテキスト圧縮

  • /compactコマンドで溜まったやり取りを圧縮
  • 内容をしっかり把握した上でコンテキストを大幅削減
  • Claude CodeもCodexも共通で利用可能な公式機能
  • 長いセッションほど圧縮の効果が絶大
公式機能だから使わない手はない！
⑭

外部ツールRTKでトークン自動圧縮

  • RTK: トークンを自動で圧縮する外部ツール
  • 公式計測: 11.8万トークン → 2.4万トークンに削減
  • 30分のセッションで約80%の節約効果を実測
  • 内容を把握した上で圧縮して次のタスクへ移行
80%削減はすごい！導入するだけで劇的に変わる
⑮

画像読み取り（ビジョン）を控える

  • Fable 5の画像読み取りは1回で約1.2万トークン消費
  • 裏側で拡大・ズーム・再読み込みを何度も繰り返す
  • テキスト情報だけで済む場合は画像読み取りをオフにする
  • Webサイトの内容取得はテキストベースで指示する
通常の4000tokenに対して3倍もかかるから注意！
⑯

チャットとエージェントを使い分ける

  • 相談・検討: 通常のAIチャットで行う
  • 実行・作業: Claude CodeやCodexなどのエージェントツールで一気に
  • エージェントはまとまった作業だけに使うのが正解
  • チャットで相談してから1回だけエージェントに送る
Claude Codeに慣れるとずっと使いがちだけど、分けるのが節約のコツ！
⑰

トークン節約の全体像をまとめる

  • 把握: 使用量を金額ベースで可視化する
  • 入力削減: /clear、ファイル部分抽出、コネクターOFF
  • 圧縮: /compact、RTK導入、出力の短縮指示
  • 最適化: エフォート調整、モデル使い分け、サブエージェント
どれか一つでもやるだけで効果はある！できることから始めよう
⑱

トークン節約＝AI活用の可持续性

  • トークン節約はAIを長く使い続けるための必須スキル
  • モデルはますます高額化・高性能化していく
  • チャンネル登録・概要欄のLINE公式アカウントでツール配布中
  • 賢く使って、長く活用しよう
節約＝我慢ではなく、賢い使い方の発見！
