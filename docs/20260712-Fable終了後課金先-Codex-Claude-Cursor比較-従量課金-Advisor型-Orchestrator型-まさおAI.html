💻 Fable終了後どこに課金？
Codex・Claude・Cursor比較まとめ

📝 チャンネル: まさおAIじっくり解説ch
①

Fable 5のサブスクが
いよいよ終了

  • 2026年7月13日（日本時間）をもって週次利用枠から外れる
  • 無料枠は当初7/7までだったが7/12まで延長された
  • 以降はUsage Credits（従量課金）で利用する方式に
  • 週50%まで無料だった時代から、予算を消費して使う時代へ
  • Anthropic公式からの突然の発表で開発界隈がざわつく
無料でFable 5が使える最後の日！延長されたけど残りわずか⏰
②

Fable 5のAPI単価：
業界最高クラスの高さ

  • 入力$10 / 出力$50（100万トークンあたり）
  • Opus 4.8の2倍の価格設定
  • 他のClaudeモデルと比べると圧倒的に高コスト
  • 賢い使い方をしないと予算がすぐ飛んでいく
Opus 4.8の2倍の値段！賢く使わないとお財布がヤバい💸
③

Codex（OpenAI）：
総合的におすすめ

  • 月額$20（Plus）/ $100（Pro）/ $200（Ultra）
  • GPT-5.6 Solを使えるようになり性能が大幅に向上
  • 画像生成やコンピューターユース、アプリまで総合的な体験
  • オープンソースターミナルエージェントでGitHubスター74,468
  • CI/CDパイプラインやGitHub Actionsとの統合も容易
一番バランスが良い！初心者からプロまでまずはCodexから🤖
④

Codexの強み：
非同期マルチタスク処理

  • 非同期マルチタスク処理が最大の強み
  • 複数のタスクを同時に並行して処理できる
  • CI/CDパイプラインとの連携がスムーズ
  • GitHub Actionsを活用した自動化ワークフローが構築可能
  • チーム開発でも導入ハードルが低い
複数タスクを同時に処理！仕事でマジで助かる⚙️
⑤

Claude（Anthropic）：
コーディング能力が最強

  • SWE-Bench Proで80.3%を記録（業界最高）
  • 2位とは11ポイント差という圧倒的なリード
  • 1Mトークンのコンテキストウィンドウを搭載
  • マルチファイルリファクタリングに最適
  • MCP（Model Context Protocol）の連携が最も成熟
コーディングはClaude一択！11ポイント差は他が追いつけない👑
⑥

Claude Code：
開発現場での実戦投入

  • Anthropicの公式CLIツールとして提供
  • 巨大なコードベースの理解と修正に強み
  • MCPサーバーとの連携で外部ツール統合が容易
  • 複数ファイルをまたぐ大規模リファクタリングに最適
  • エンタープライズ向けのセキュリティも充実
Claude Codeはターミナルから直接使えるから快適💻
⑦

Cursor：
IDE統合で即日導入

  • VS Codeフォークの統合IDEで違和感なく移行
  • 月額¥3,000のProプランで手軽に始められる
  • チーム全体への展開が3つの中で最も容易
  • Cloud Agentsを搭載（3.5アップデート）
  • 単一ファイル補完に強く、既存IDE環境を活かせる
VS Codeと同じだからチームで一斉に導入しやすい！🖥️
⑧

ベンチマーク比較：
SWE-Benchスコア

  • 🥇 Fable 5：SWE-Bench Pro 80.3%（ダントツ1位）
  • 🥈 Cursor：SWE-Bench Pro 約74%
  • 🥉 Codex CLI：SWE-Bench Verified 約58.6%
  • Fable 5のFrontierCodeスコアは29.3%（Opus比2倍以上）
  • ※ベンチマークと実務体感は異なるケースあり
Fable 5が圧倒的！でもベンチマークだけじゃ判断できない📊
⑨

料金体系の比較：
月額 vs 従量課金

  • Cursor Pro：¥3,000/月（定額で安心）
  • Codex：$20〜$200/月（3つの定額プラン）
  • Fable 5：従量課金のみ（入力$10 / 出力$50 per MTok）
  • Claude Sonnet 5：入力$3 / 出力$15 per MTok
  • Claude Opus 4.8：入力$5 / 出力$25 per MTok
定額派ならCursorかCodex、従量派ならClaudeで使い分け💰
⑩

相談役（Advisor）型：
賢くコストを抑える

  • 実行役のSonnet 5が手を動かし、迷った時だけFable 5に助言
  • Fable 5単体の約92%の品質を約63%の費用で達成
  • トークン生成の大半は安い実行役の単価で課金
  • 「専門家に相談しながら自分で手を動かす」イメージ
92%の品質を63%のコストで出せるならAdvisor型は最強💡
⑪

指揮役（Orchestrator）型：
最大のコスト削減

  • Fable 5は計画と采配に徹し、実作業はSonnet 5のワーカーに分担
  • Fable 5単体の約96%の性能を約46%の費用で達成
  • トークンを多く消費する実作業は安いワーカー単価で課金
  • 「社長が指示を出して社員が作業する」イメージ
46%のコストで96%の性能！これぞOrchestrator型の真価♟️
⑫

モデル選択と
Effort Levelの考え方

  • モデル＝「誰に頼むか」（知識と能力）
  • Effort Level＝「どれだけ働いてもらうか」（仕事量）
  • Fable 5＝「ほぼ誰も見たことのない問題を扱ってきた専門家」
  • Opus＝「エキスパート」
  • Sonnet＝「とても優秀なジェネラリスト」
モデルは人を選ぶのと同じ！簡単な作業ならSonnetで十分🤔
⑬

導入の失敗パターン：
よくある4つの罠

  • 全員に同じツールを強制導入する → 現場の反発
  • セキュリティポリシーなしで導入する → 情報漏洩リスク
  • ベンチマーク数字だけで判断してPoC期間を設けない
  • コスト計算をせずに全員API課金にする → 請求書ショック
どれもよくある失敗！特にAPI課金は請求書に泣く😭
⑭

使い分けの基本フロー：
4ステップで最適化

  • まずデフォルト（Sonnet / Opus）で試す
  • 渡し方を疑う（資料・前提条件は十分か？）
  • 効いているならEffort Levelを上げる
  • それでもダメならFable 5に投入する
いきなりFable 5を使わず段階的に上げていくのがコスパ最強🗺️
⑮

スタートアップ：
Cursor Proで全員展開

  • 5名以下の小規模チームに最適
  • 月額¥3,000で全員がAIコーディングを体験
  • VS Codeベースなのでエンジニアの学習コストがゼロ
  • 成長してきたらClaude Codeとの併用へ移行可能
小さいチームならCursor一択！安いし早いし🚀
⑯

中堅企業：
Cursor + Claude Code併用

  • 10〜30名の中規模チーム向けの構成
  • Cursorは日常開発に、Claude Codeは複雑タスクに
  • 役割に応じた棲み分けでコストと性能のバランスを取る
  • ジュニアはCursorで補完、シニアはClaudeで設計
使い方で棲み分け！複雑な作業だけClaudeに回すのがコツ🏢
⑰

大企業：
Claude + カスタムセキュリティ

  • セキュリティ要件が高い企業向けの構成
  • Claude Code + Claude.ai + カスタムセキュリティで統合
  • 自社サーバーにクローズド環境を構築可能
  • コンプライアンスやデータガバナンスの要件を満たせる
大企業はセキュリティ第一！Claudeの独自環境構築が定番🏛️
⑱

結論：実質3択の時代
賢く使い分けよう

  • Codex：総合力重視で迷ったらまずこれ
  • Claude：コーディング最強でOrchestrator型でコスパ良
  • Cursor：IDE統合でチーム展開が最も手軽
  • Advisor型かOrchestrator型でFable 5を賢く活用するのが鍵
  • 1つのツールですべてをやる時代は終わり
  • 自分のチーム規模と用途に合わせて組み合わせよう
どれか一つじゃなくて組み合わせ！これが2026年の正解⚖️✨
