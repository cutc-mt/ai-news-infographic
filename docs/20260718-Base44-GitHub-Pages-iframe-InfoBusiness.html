Base44 × GitHub Pages: iframe連携の完全解説

バックエンドとフロントエンドを分離するモダンなWebアプリ構成

📅 2026年7月18日 | 解説: Generative AI and InfoBusiness Institute

①連携の基本構成

  • Base44をバックエンドとして利用
  • GitHub Pagesをフロントエンドとして活用
  • 2つのサーバーをiframeでシームレスに連携
  • ノーコードとカスタムコードの良いとこ取り
バックエンドはノーコード、フロントエンドは自由自在！

②役割分担の考え方

  • Base44: データ管理・認証・APIを担当
  • GitHub Pages: ユーザーインターフェースを担当
  • 役割を明確に分けることで保守性が向上
  • フロントエンドの自由度が格段に広がる
責任の分割で管理がしやすくなる！

③iframeの基本構成

  • Base44のReactアプリ内にiframe要素を配置
  • iframeのsrcにGitHub PagesのURLを指定
  • 外部Webアプリを親ウィンドウ内に埋め込む
  • 親子関係で安全な通信を実現する仕組み
別サイトを自分のアプリ内にそのまま表示できる！

④postMessageの基本

  • postMessage APIで親子間の安全な通信を実現
  • 異なるオリジン間でもデータを送受信可能
  • same-origin policyの制約を安全に回避
  • イベント駆動で非同期通信を実行
セキュリティを守りながら通信する標準的な方法

⑤題材: Grave Reaper

  • レトロ横スクロールアクションゲーム
  • GitHub Pagesでゲーム本体をホスト
  • Base44側でランキングDBを管理
  • ゲームオーバー時にスコアを自動連携
実際のゲームでiframe連携をデモ！

⑥子→親: スコア送信

  • ゲームオーバー時に子(iframe)から親へスコアを送信
  • window.parent.postMessage()でデータ送信
  • イベント名とスコア値をペアで送信
  • 親側が受け取ってBase44のDBに保存
ゲームが終わったら自動でスコアが記録される！

⑦画面遷移イベントの送信

  • 「ランキングを見る」ボタン押下で画面遷移イベントを送信
  • 子から親へナビゲーション指示を送信
  • 親側のReactでイベントを受信して画面を切り替え
  • iframe内のボタンで親の画面を操作
iframeの中のボタンで外の画面を動かせる！

⑧親側のイベントハンドリング

  • 親側はmessageイベントをリッスン
  • イベント種別で分岐処理を実行
  • スコア保存と画面遷移の2つのハンドラ
  • Reactのステート管理と連動させる設計
イベントの種類を判定して適切に処理する仕組み

⑨安全なガード処理

  • window.parent !== windowでiframe内かを判定
  • 直アクセス時はpostMessageを送らない設計
  • GitHub Pagesで単体動作が壊れないように保護
  • 連携機能はiframe内の時だけ有効化
フロントエンド単体でも動くから安全！

⑩ランキングDB連携

  • Base44のデータベース機能でランキングを管理
  • ハンドルネームの登録とスコアの紐付け
  • ランキングの表示と更新をリアルタイムに
  • ゲーム内通貨の管理も今後の拡張で対応
ノーコードでバックエンドが完成するのが強み！

⑪postMessageの実装パターン

  • 送信側: window.parent.postMessage({ type: 'score', data: score }, '*')
  • 受信側: window.addEventListener('message', handler)
  • typeフィールドでイベントの種類を識別
  • dataフィールドでペイロードを受け渡し
シンプルだけど強力な通信パターン
