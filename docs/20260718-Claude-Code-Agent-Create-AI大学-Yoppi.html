②Claude Codeとは

  • Anthropicが提供するAIエージェントツール
  • CLIベースでターミナルから直接操作可能
  • プログラミングや開発支援に特化した設計
  • 自然言語での指示に基づいてコードを生成・編集
CLIで動くから開発者のワークフローに馴染む！

③エージェントの基本概念

  • AIに特定のタスクを自律的に実行させる仕組み
  • 指示 → 実行 → 結果確認 → 次のアクションのループ
  • ツール（ファイル操作・シェル実行）を組み合わせて活用
  • エージェントが自ら計画を立てて実行する流れ
AIが「自分で考えて動く」のがエージェントの醍醐味！

④エージェントの準備

  • Claude Code CLIがインストール済みであることが前提
  • Claudeアカウントとの連携が完了している
  • 作業ディレクトリをプロジェクトに設定
  • VS Code内でClaude Codeを起動済み
第1〜3回で環境構築が済んでいる状態からスタート

⑤CLAUDE.mdの設定

  • プロジェクトのCLAUDE.mdにエージェントの指示を記述
  • プロジェクトの構造やルールをAIに教えるファイル
  • エージェントに何をしてほしいかを明確に定義
  • 具体的な指示を書くほど精度の高い出力が得られる
CLAUDE.mdがエージェントの「設計書」になる！

⑥シンプルなエージェントの作成

  • まずは最小限の構成でエージェントを作成
  • 一つの明確なタスクに集中させる
  • 実行手順をステップバイステップで定義
  • 小さな成功体験を積んでから拡張していく
小さく始めるのがエージェント作成のコツ！

⑦エージェントワークフロー

  • 入力: ユーザーがタスクを指示
  • 計画: AIが実行計画を策定
  • 実行: ツールを使ってタスクを実行
  • 検証: 結果を確認して必要なら修正
計画→実行→検証のサイクルで自律的に動く！

⑧CLIでの実行方法

  • ターミナルでclaudeコマンドを実行
  • 自然言語でエージェントにタスクを指示
  • 実行ログをリアルタイムで確認できる
  • 途中で介入したり修正を指示することも可能
ターミナルで会話するようにエージェントを操作！

⑨使用可能なツール

  • ファイル操作: 読み書き・作成・編集
  • シェル実行: コマンドの実行と結果取得
  • コード検索: プロジェクト内の検索・解析
  • ブラウザ操作: Web画面の表示と操作
これらのツールを組み合わせて自律的に作業する！

⑩MCPとの連携

  • MCP (Model Context Protocol)で外部ツールと接続
  • データベースやAPIへのアクセスを拡張
  • 外部サービスとの統合をシームレスに実現
  • エージェントの能力を大幅に拡張できる仕組み
MCPでエージェントの可能性が無限に広がる！

⑪Skillsの活用

  • Skillsでエージェントに専門的な能力を追加
  • 特定のタスク向けの手順書を読み込ませる
  • 再利用可能なスキルパッケージとして管理
  • コミュニティでスキルを共有できるエコシステム
スキルを追加すればどんどん賢くなる！

⑫Hooksの仕組み

  • Hooksで特定のアクションの前後に処理を差し込む
  • 安全チェックやバリデーションを自動化
  • 定型処理をフックとして登録しておく
  • エージェントの動作にガードレールを設定
自動で安全チェックを挟めるから安心！

⑬Subagentsの活用

  • Subagentsで複数のエージェントを協調動作
  • タスクを分割してそれぞれのエージェントに割り当て
  • エージェント同士が情報をやり取りしながら進行
  • 並行処理で効率的なタスク実行を実現
チームで作業するようにエージェントを協調させる！

⑭Pluginsの拡張

  • Pluginsで外部機能をシームレスに追加
  • プラグイン市場から必要な機能を導入
  • 独自のプラグインを開発して公開も可能
  • エージェントの能力の境界を大きく広げる
プラグインで機能をどんどん拡張できる！

⑮エージェントチーム機能

  • 複数のClaude Codeエージェントが連携して作業
  • タスクごとに専門のエージェントを割り当て
  • エージェント間の通信と調整が自動化される
  • 大規模プロジェクトの完全自律型AIの実現
AIチームでプロジェクトを自律的に進める未来！

⑯注意点とベストプラクティス

  • エージェントに過度な権限を与えない
  • 実行結果を必ず確認・レビューする
  • 破壊的な操作には確認フックを設定
  • APIコストに注意してタスクの規模を調整
自律性とセキュリティのバランスが大事！

⑰学習のステップアップ

  • 第1回→第4回で基礎から実践まで段階的に学習
  • まずはシンプルなエージェントで成功体験を
  • MCP・Skills・Hooksを段階的に組み込む
  • 最終的にエージェントチームの構築へ
4回のシリーズで着実にステップアップ！

  • エージェントワークフローで業務の自動化が進む
  • MCPやプラグインで無限の拡張性
  • プログラミング経験がなくても誰でも始められる
