Devin SWE-1.7 実機検証

GPT-5.6 Lunaとの比較で見えるコスパエージェントの実力
2026年7月18日

① Devin SWE-1.7が登場: 性能とコスト効率を両立

  • Cognition社のDevinからSWE-1.7がリリース。最高性能ではなく性能とコストのバランスに特化したモデル
  • フロンティアコードベンチマークでアピール。GPT-5.5やClaude Opus 4.8に近づいているが超えられないレベル
  • 前モデルから大幅に進化。ターミナルベンチやSWE-benchで好成績を記録
  • 売りは「圧倒的覇者」ではなく、長時間の開発エージェントとしてのコスパ最強を目指す
圧倒的な覇者ではないけれど、性能・品質・速度・コストのバランスをとった実用的なモデルです。

② 無料で使える隠れコスパ枠: Devinのフリーモード

  • Devinは現在無料でSWE-1.7を使える状態。262Kコンテキストウィンドウがフリー
  • Windows Copilotにも統合済み。VS Code拡張機能から直接利用可能
  • Devin CIも存在。Claude CodeのようにCLIから操作できる
  • OpenCode 5等は枠制限があるが、Devinは今完全無料で使い放題のチャンス
Cursorとも比較されるけれど、無料で使えるDevinは隠れたコスパ枠として優秀です。

④ ゲーム開発テスト: マーブルランから3D格闘まで

  • マーブルラン: SWEはボールが意味不明な挙動。Lunaの方が動作が安定
  • アクアリウム: どちらも魚の描写が雑。Devinは時間がかかる割にイマイチ
  • 3D格闘ゲーム: SWEが54分かけて作成。他モデルと比べクオリティは高い
  • ピンボール: SWEは実装できるが38分要する。Lunaは動かない不具合が発生
長くかければ品質は上がるけれど
時間あたりのコスパで差が出ます。

⑤ SVG・視覚的品質: Grok 4.5がダントツ

  • Grok 4.5がSVG品質で圧倒。ライオンや電気ネズミの描写が精密
  • Lunaはレイヤー構造をしっかり作成。12分で高品質なFigma風UIを生成
  • SWE-1.7は22分で同等レベル。約2倍の時間が必要
  • 人間のアニメーション: Grokが4-5分で高品質。SWEとGPTは10分以上で微妙
Grok 4.5のスピード×品質は凄い。
開発者も驚くレベルです。

⑥ Devinのエコシステム: IDE統合と多様な選択肢

  • Windows Copilotに統合。VS Code拡張機能からDevinを直接利用可能
  • スマホアプリ版もあり、どんな環境からでもDevinが使える体制が整備
  • モデル選択肢: SWE-1.7、Lightning、Cascade、Local、Cloudと多彩なラインナップ
  • Devin CLIとDevin CIも提供。Claude CodeのようなCLI開発が可能
あらゆる場所にDevinが統合される流れ。選択肢が増えるのは嬉しい！

⑦ GPT-5.6 Lunaの可能性: 最安モデルの新スタイル

  • Lunaは最安モデルながら美しさや情報整理で高い評価を獲得
  • Figma風UI制作でレイヤー構造を意識。12分で高品質な結果を生成
  • マーブルラン等で「何もないのにうまく動く」不可思議な挙動も確認
  • 資格的な美しさが評価トレンドに。ベンチでは測れない新しい価値観
Fable等が重視する「美しさ」という
新しい評価基準。
SWEはそこがやや残念。

⑧ オセロAI対決: 全モデルが負ける結果に

  • Sonnet 5とオセロ対決をさせた結果。GPT-5.6もSWE-1.7もGrokも全負け
  • 論理的思考のテストとしてオセロは有効だが、どのモデルも戦略性に難あり
  • 物理シミュレーションはSWEが無難にこなすが、グロクの方が短時間で高品質
  • 信号で止まる車のアニメーション: GPTとDevinは微妙、Grokは4分で高品質
オセロには全モデル負ける...。
AIの論理的思考はまだ発展途上。

⑨ プレゼン・スライド生成: 全モデルが対応

  • プレゼンテーションエンジンの生成テスト。どのモデルもしっかり実装
  • GrokもGPT-5.6もLunaもプレゼンモードを実装。プレス機能も動作
  • 品質面で差はあまりないが、Lunaの視覚的完成度がやや上回る
  • Web Audio APIのデモも全モデルクリア。最低限の実装は確実にこなす
基本的なWeb開発タスクは
どのモデルでもクリアできる時代に。

⑩ 物理シミュレーション: 微妙な差が気になる

  • 風のシミュレーション: SWE-1.7は実装するが、動きがやや不自然
  • 波動シミュレーションは無難にこなす。課題へのアプローチは正確
  • 小さな実装テストではどのモデルも普通に解決。基本力は確か
  • 物理エンジンの精度でGrok 4.5がリード。時間も短いので実務で有利
できることは当たり前、
問題は品質と速度です。

⑪ 折り紙アニメーション: Grokが圧倒的短時間

  • 折り紙アニメーション: Grok 4.5が5分で高品質に完成
  • SWE-1.7は大幅に時間がかかる。GPT-5.6も10分以上が必要
  • 最終的なクオリティはGrokが最高評価。動きの滑らかさが段違い
  • 関節のある人形の歩行アニメーションでもGrokのレベルが高い
Grok 4.5のアニメーション能力は
本当に別格です。

⑫ バグと不具合: 全モデル共通の課題

  • ノーエディター実装でエンター押すと即座に展開される不具合が発生
  • WASD移動の3DゲームでSWEが全く動かない。バグが多数発生
  • 検索ボタンが装飾されていないなど、細かい部分のズレが目立つ
  • TPSが落ちる場面も。特に複雑なタスクで速度低下が顕著
AIエージェントはまだ自己デバッグが
甘い場面が多いです。

⑬ Fableとの比較: 美しさが足りないSWE

  • Fable等は「資格的な美しさ」や「情報の整理」を重視する新しい評価基準
  • SWE-1.7はこの基準で見ると少し残念。ベンチスコアは良いが体験が伴わない
  • MetaのFableも含め、ベンチマークだけでは測れない部分が非常に評価されている
  • 開発者には受けるが、デザイナーやクリエイターには不十分な印象
ベンチマーク以外の体験価値も
今後のAI評価で重要になるでしょう。

⑭ SWE-1.7 Lightning: 上位版も存在

  • SWE-1.7 Lightningという上位モデルも存在。速度と品質が更に向上
  • ただしLightningはコストが高い。フリー版とは別料金設定
  • Lightning版でのFigma風UI制作は12分で高品質。Lunaと同等以上の仕上がり
  • レイヤー構造もしっかり実装。Grokは4分で同等レベルだがLightningも悪くない
有料版のLightningは確かに良いが
Grokの無料と比べると
コスパで悩むところ。
