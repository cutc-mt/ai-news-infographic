Hallmark: AI生成サイトを「スロップ」から解放する

Claude Code / Cursor / Codex対応のアンチAIスロップ・デザインスキル

📅 2026年7月18日

①「AIスロップ」問題とは

  • AIが生成するウェブサイトは、構造がどれも同じになりがち
  • ヒーロー → 3つの機能カード → CTA → フッター、という定型パターン
  • 見た目が「生成された感」ただ丸出しで、オリジナリティに欠ける
  • カラーテーマも似たり寄ったりで、差別化が難しい現状
Hallmarkはこの「AIスロップ」を排除して、意図的でユニークなデザインを実現するよ！

②Hallmarkとは何か

  • オープンソースのアンチAIスロップ・デザインスキル
  • Claude Code・Cursor・Codexに対応
  • nurlope/hallmarkとしてGitHubで公開されている
  • AI生成のフロントエンドを意図的で個性的にする
AIコーディングアシスタントのスキルとして動作する仕組み

③21種類のマクロ構造

  • ページ全体の骨組みパターンを21種類用意
  • 毎回異なる構造を選択して、同じレイアウトを繰り返さない
  • 「ヒーロー → 機能カード → CTA」の定型化を回避
  • プロジェクトに最適な構造を自動的にマッチング
構造が変わるだけで、一気にオリジナル感が出るね！

④22種類のデザインテーマ

  • Editorial系：雑誌風の編集レイアウト
  • Modern-Minimal系：洗練されたミニマルデザイン
  • Atmospheric系：雰囲気重視のビジュアル
  • Playful系：遊び心のあるカジュアルデザイン
  • テーマ×構造の組み合わせで数百パターンが可能
4ジャンルに分かれていて用途に合わせて選べる！

⑤スロップテスト機能

  • 57〜65のスロップ検出ゲートを内蔵
  • 生成前にAIデザインの典型ミスをチェック
  • 数ダースの一般的なAIデザインミスを事前に検出
  • 不合格なら自動的に再生成を試みる
出力前に品質チェックするから安心！

⑥フェイク情報の防止

  • フェイクのメトリクスを自動的に排除
  • フェイクの顧客ロゴを表示しない
  • フェイクの testimonialsを生成しない
  • 実データが提供された場合のみ、リアルな情報を表示
信頼性を損なうフェイク情報は一切出さない主義

⑦Hallmark Study機能

  • スクリーンショットやURLからデザインDNAを抽出
  • 既存サイトの配色・レイアウト・構造を分析
  • 抽出したDNAをベースに新しいデザインを生成
  • リファレンスを入力するだけでスタイルを再現
好きなサイトのデザインを学習できるのがすごい！

⑧プロジェクトメモリ

  • 複数ページ間で構造とテーマを記憶
  • 同じパターンの繰り返しを自動的に回避
  • プロジェクト全体で一貫性を保ちつつ変化をつける
  • ページ間の多様性を確保しつつ統一感も維持
同じサイト内のページでもバラバラにならない工夫

⑨対応ツールと環境

  • Claude Code: CLIから直接利用可能
  • Cursor: AIコードエディタ内で統合
  • OpenAI Codex: CLIエージェントにも対応
  • スキルファイルとしてプロジェクトに配置して使用
メジャーなAIコーディングツール3つに対応！

⑩アンチパターンカタログ

  • AIデザインの悪習パターンを網羅的に収録
  • 使い慣れたセクション分割や段落の配置をチェック
  • ジェネリックなタイポグラフィの問題も検出
  • 配色の典型パターンもスロップ判定の対象
「AIっぽく見える」要素を体系的に特定する仕組み

⑪構造の多様性が鍵

  • マクロ構造 × テーマの組み合わせで数百のバリエーション
  • ランダムではなく、プロジェクトに最適な構造を選択
  • ビーフ（ブリーフ）を分析して最適な骨組みを決定
  • プロジェクトごとにユニークな指紋のようなデザインを生成
結果：2つのプロジェクトが同じ見た目になることはない！

⑫デザインDNAの抽出と再現

  • StudyコマンドでURLや画像からスタイルを解析
  • 配色・フォント・間隔・配置の「DNA」を抽出
  • 抽出したDNAを種にして新しいデザインを成長させる
  • 同じDNAでも構造を変えれば全く違う印象に
好きなサイトの「良さ」だけを抽出して活かせる！

⑬監査（Audit）機能

  • 既存のデザインをアンチパターンカタログと照合
  • 問題のある要素をピンポイントで指摘
  • 改善提案とともに再デザインを実行
  • 別の構造パターンでの再構築も可能
既存サイトの「AIっぽさ」を診断して改善する機能

⑭トークン消費の注意点

  • トークン消費が多めになる傾向がある
  • 57〜65のゲートチェックでコンテキストが増加
  • 品質とコストのバランスを考慮する必要がある
  • 結果の品質向上を考えると投資に見合う価値あり
高品質なデザインにはそれなりのコストがかかるね

⑮最適なユースケース

  • ランディングページの生成に最適
  • マーケティングサイトの構築で真価を発揮
  • ポートフォリオサイトの作成にも適している
  • 小規模なSPAやダッシュボードには過剰な場合も
LPやマーケティングサイトでこそ最大の効果を発揮！

⑯「生成されない感」をどう出すか

  • 構造の多様性で「テンプレート感」を消し去る
  • テーマの選択で意図的な雰囲気を演出
  • スロップテストで「AIっぽさ」を品質管理
  • 結果的に「手作り感のある質の高いサイト」に仕上がる
見る人が「これAIで作った？」と疑うレベルの品質！

⑰総合評価

  • AIコーディングエージェント向けの最も思考的なデザインスキルの一つ
  • オープンソースで無料利用可能
  • コミュニティの評価も高い
  • 継続的なアップデートで進化中
AIフロントエンド生成の常識を変える画期的スキル！

⑱注意点と今後の展望

  • デザインの方向性がやや強めに固定される場合がある
  • 独自のデザインルールがあるためカスタマイズに工夫が必要
  • forkして自分好みに改造することも可能
  • エージェント系AIツールの進化とともにさらに重要に
AIコーディングの世界では欠かせない存在になっていく予感！
