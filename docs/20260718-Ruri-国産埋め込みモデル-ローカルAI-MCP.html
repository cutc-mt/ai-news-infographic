① Ruriとは? — 名古屋大学が開発した日本語特化モデル

  • 名古屋大学が開発した日本語に特化した汎用テキスト埋め込みモデル
  • 日本語の文章をベクトル（数値の並び）に変換して意味の近さを計算
  • キーワード一致だけでなく、質問の意味に近い順に検索結果を並べ替える
  • RAG（Retrieval-Augmented Generation）システムでの活用が特に期待される
「埋め込みモデル」って難しそうだけど、
要するに「言葉の意味を数字にして
似ているものを探す」仕組みです。

② RAGシステムでのRuriの役割

  • RAGは「検索→関連文書の取得→AIが回答を生成」の3ステップで動く
  • Ruriが検索の精度を劇的に向上させる。質問に最も関連する情報を上位に持ってくる
  • ローカルLLMは良い情報が上位に来ていると、より精度の高い文章を生成できる
  • 検索APIからの結果をベクトル化して、意味的な類似度で並べ替えるのがRuriの仕事
Ruriは「AIの検索部品」。

④ Ruriの導入方法

  • HuggingFaceからRuri-v3モデルをダウンロードして利用可能
  • PythonのSentenceTransformersライブラリで簡単に実行できる
  • infinity_embコマンドでAPIサーバーとして起動。OpenAI互換のembedding APIに対応
  • ローカル環境で完全に動作するため、APIキー不要・コストゼロ
# infinity_embでAPIサーバー起動
infinity_emb --model-name ruri-v3 \
  --port 7997 --engine torch
数行のコマンドで起動！
OpenAI互換APIだから既存のツールに
そのまま繋げられる。

⑤ 使う際の注意点 — プレフィックスが重要!

  • 「クエリ: 」というプレフィックスを検索クエリにつけるのを忘れると精度が大幅に低下
  • プレフィックスなしでの検索精度は大幅に悪化するため必須
  • 文書側にはプレフィックスは不要。検索側のクエリだけに付ける
  • これはRuriモデルの学習方式によるもので、「クエリ」と「文書」の役割を区別するため
「クエリ: 」を忘れると
精度がガタ落ち！
これ一番よくあるミスです。

⑥ MCPでRuriをClaude Codeに接続

  • MCP（Model Context Protocol）を通じてClaude CodeからRuriを利用可能
  • infinity_embをAPIサーバーとして起動し、MCPサーバーとしてClaude Codeに登録
  • Claude Codeの設定ファイルにMCPサーバーのエンドポイントを記述
  • これでClaude Codeがローカルの高精度な日本語検索を利用できる
// Claude Code MCP設定例
[mcp_servers.ruri]
command = "infinity_emb"
args = ["--model-name", "ruri-v3",
        "--port", "7997"]
Claude Code + Ruri = 最強のローカルRAG
日本語の精度がOpenAI超えになる
組み合わせです。

⑨ ローカルAIのメリット

⑩ 必要なハードウェア

  • GPU または大容量メモリが必要。Ruriは比較的軽量なので8GB以上のVRAMで動作
