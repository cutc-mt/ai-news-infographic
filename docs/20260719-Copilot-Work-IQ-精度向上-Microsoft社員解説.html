Copilotの精度を劇的に上げる「Work IQ」とは？

Microsoft社員が解説 — 仕事の文脈を理解してCopilotを進化させるインテリジェンス基盤の全貌
📅 2026年7月19日

①Work IQとは: Copilotの精度を根底から変える仕組み

  • Work IQはMicrosoft 365 CopilotとAIエージェントが組織の業務文脈を理解するためのインテリジェンス基盤
  • メール、ファイル、会議、チャットなど組織に散らばる仕事のデータを意味的につなぎ合わせるレイヤー
  • 個人の役割・好み・仕事の進め方まで含めてAIに渡すため、よりパーソナライズされた応答が可能に
  • 単にデータを検索するだけでなく、「今の仕事の文脈に合った実行」を支える設計が最大の特徴
Copilotの裏側の頭脳とも
言える存在。表面のチャット体験を
裏で決めているのがWork IQ

②3層構造の全体像: データ・コンテキスト・スキル＆ツール

  • データ層: Microsoft 365・Dynamics 365・外部システムへの権限を尊重した安全なアクセスを提供
  • コンテキスト層: 業務文脈と関係性を継続的に学習・索引化してAIに渡す
  • スキル＆ツール層: スキル定義に基づくタスク実行と外部システムとの統合を担う
  • 下の層の出力を前提に上の層が動く階層的な設計。データ→文脈→実行の順で処理される
元々はデータ・メモリ・推論
と呼ばれていたが、2026年3月に
現在の3層名に整理し直された

③データ層の深掘り: M365から外部システムまで統合

  • Microsoft 365テナントデータ: SharePoint・OneDrive・Outlook・Teamsの内容にアクセス
  • ビジネスデータ: Dynamics 365やPower Appsの構造化データもDataverse経由で統合可能
  • コネクタ: プリビルトやカスタムコネクタで非Microsoftシステムも取り込める
  • 全データソースにおいて既存のアクセス権限とポリシーを尊重した安全なアクセスが特徴
単なるAPIラッパーじゃない。
権限を守りながら幅広い業務データに
統一的にアクセスできるのがポイント

④コンテキスト層: 「文書を見つける」から「文脈を理解する」へ

  • Copilotメモリ: ユーザーの明示的指示と暗黙的な活動パターンの両方を保持
  • セマンティックインデックス: キーワード検索の上に意味で引ける2層インデックスを構築
  • Dataverse intelligence: 売上や在庫といった構造化データも推論対象に組み込む
  • ファイルの内容だけでなく、人とチームの関係性・業務パターンも学習していく
「先週の営業MTGの懸念を
まとめて」と言ったら、関連案件の
Dataverseの進捗まで踏まえて答える

⑤スキル＆ツール層: AIの知識を「実際の行動」に変換

  • Business skills: 組織のプロセス・ポリシー・ドメイン知識を自然言語のスキルとして定義
  • Work IQ CLI: MCPサーバーとして社内データを公開し、開発中AIアシスタントが文脈を自動取得
  • MCPサーバー・APIプラグイン: メール送信・会議設定・文書操作などの具体的な呼び出し先
  • 複数エージェント間で同じスキルを共有することで、組織横断で一貫した動作を担保
Copilot Studioで作った

⑥Work IQの変遷: Ignite 2025からAPIs一般提供まで

  • 2025年11月: Microsoft Ignite 2025でWork IQが初公開。「データ・メモリ・推論」の3層として紹介
  • 2026年3月: E7 Frontier Suite発表。Work IQが最初のFrontier Suiteとして束ねられる
  • 2026年4月: Work IQ APIパブリックプレビュー開始（A2A・Local MCP対応）
  • 2026年6月: Copilotリデザイン公開。同月16日にWork IQ APIsが一般提供（GA）へ
発表から半年で一気に本格運用
の基盤が揃った。

⑦Work IQ APIsの4コンポーネント: Chat・Context・Tools・Workspaces

  • Chat: Copilotの「会話」をプログラムから呼び出し、応答と引用を取得
  • Context: エージェント向けに準備済みの文脈と元データを返す合成応答でない形式
  • Tools: M365エンティティへの「動詞」を開放し、エージェントが実際にアクションを実行
  • Workspaces: エージェントの実行状態・メモリ・進捗をテナント境界内で保持する「作業机」
3層構造がWork IQの中身なら
4コンポーネントは外から呼ぶ入り口。
別レイヤーで併存する関係

⑧MCP連携でエージェントを拡張: Work IQ Mail・Calendar・Teams

  • Work IQ Mail: エージェントがメールの内容を理解して文脈に応じた応答が可能に
  • Work IQ Calendar: スケジュールの文脈を読み取り、会議の設定や調整を自動で実行
  • Work IQ Teams: チャットや会議のやり取りから業務の流れを把握して提案
  • Copilot Studioで「Add Tool」からMCPを選択するだけで、エージェントにWork IQの知能を追加可能
カスタムMCPサーバーも

⑨セキュリティとガバナンス: 管理者の統制が担保される

  • 管理者コントロール: M365管理センターでWork IQ MCPサーバーの許可・ブロックを一元管理
  • スコープ付きアクセス: エージェントに必要な権限だけを最小限に付与する設計
  • 監査機能: Microsoft Defender経由でツール呼び出しのトレースログを検査可能
  • ポリシー強制: レート制限・ペイロードチェック・セキュリティスキャンを実行時に適用
権限が常に設定に優先。
管理者がブロックすれば

⑩料金体系: Copilot Creditsの従量課金モデル

  • Work IQ APIsには別途SKUやサブスクリプションは不要。API利用分がCopilot Creditsで課金される
  • Tools: APIコール単位の固定単価（例: 0.1 Credits/call）
  • Chat・Context: 応答内容に応じたシナリオ別のper call課金で変動
  • 管理画面でSpending limitを設定でき、PoC段階の上限管理から本番のチャージバック管理まで対応
従量課金だから利用量＝コスト。

⑪プラン別のWork IQ利用範囲: 無料版には含まれない

  • Copilot Chat（無料）: Web情報中心のチャット。Work IQの組織文脈グラウンディングは含まれない
  • Copilot Business（$18/月）: Work IQ搭載。組織文脈に基づく応答品質を享受可能
  • Copilot Enterprise（$30/月）: 大企業向け。Work IQ-powered chatが利用できる
  • E7 Frontier Suite（$99/月）: E5＋Copilot＋Agent 365＋Work IQの統合バンドルで最強構成
無料版で「Copilotは使えない」と
判断するのは早合点。Work IQの
組織文脈が経由するかどうかが鍵

⑫Agent 365との関係: 頭脳と管制塔の役割分担

  • Work IQ = 頭脳: AIが判断材料をどう持つかという知識側のレイヤー
  • Agent 365 = 管制塔: エージェントの観察・ガバナンス・管理・セキュリティを担う統制基盤
  • 両者はE7 Frontier Suiteで組み合わさるが、役割は別レイヤー
  • Agent 365はOSSやパートナー製の外部エージェントも管理対象に含む柔軟な設計
頭脳と管制塔を混同しない
こと。Work IQは知識で、Agent 365は
管理。両方揃って初めて本格運用

⑬Copilot Cowork: 複数ステップの仕事をAIに委任する新体験

  • Copilot Cowork: 要求をタスク計画に分解し、バックグラウンドで長時間の仕事を進める新機能
  • 途中経過の確認・修正・一時停止ができる。AIに丸投げではなく監督しながら委任する設計
  • Anthropicの技術を活用しつつ、Work IQで仕事の全体文脈を持たせることで精度を担保
  • 2026年3月時点ではFrontier program経由の研究プレビュー。全社展開はまだ先
Coworkの価値は推論モデルより

⑭Work IQ搭載の精度向上: Copilot Chatとの違いが決定的

  • 同じプロンプトでもCopilot Chat（無料）と有料プランで応答が明らかに違う
  • 無料版はWeb情報とファイル参照中心だが、有料版は組織データの意味的文脈を経由
  • 「特定の文書を要約した」ではなく「あなたの仕事の文脈を踏まえて整理した」という質の変化
  • オーケストレーションのオーバーヘッドを排除し、応答速度と関連性を継続的に向上させる
「Copilotは使えない」って
言ってる人は無料版で試してる
可能性大。有料プランで再検証を！

⑮中小企業での活用: 営業・管理部門・経営層への効果

  • 営業: 過去の提案書・商談メモ・顧客メールを横断して、文脈に合う提案の叩き台を高速に作成
  • 管理部門: 規程・会議・問い合わせ回答の整合性を保ち、古いテンプレートだけで回答するリスクを低減
  • 経営層: 部門ごとに散らばった報告と会議内容を横断して要約・論点整理が可能に
  • 人数が少なく情報共有が属人化しやすい組織ほど、文脈統合の価値が大きくなる
全社に広げる必要はない。
1部署で小さく回すのが現実的。

⑯導入前に整えるべきこと: データ品質がWork IQの上限を決める

  • データの正確性と一貫性: 古い・重複したデータが混在すると、Copilotの回答も的を外れる原因に
  • アクセシビリティの向上: 適切に構造化されたデータセットで、AIが効率的に情報を解析できる
  • メタデータの充実: 同じ文書の複数バージョンが存在すると、Copilotが間違った版を提示するリスク
  • 関連性の低い古いデータにはスマートな保持・処分ポリシーを導入することが推奨される
Work IQは散らかった情報を
自動で正しくしてくれるわけじゃない。
まず情報の整備が先！

⑰プロンプトエンジニアリング: 効果的な使い方でさらに精度アップ

  • 明確で具体的な指示: 曖昧なリクエストは精度や関連性の低い出力につながる
  • カスタム指示の設定: Copilotがプロンプトにどう応答するかをチャット設定で事前に定義
  • 継続的な利用: 使えば使うほどCopilotのサポートがパーソナライズされて効果的になる
  • 最もROIを上げる方法は実際のワークフローについてチームをトレーニングすること

⑱導入判断のまとめ: 段階的なアプローチが成功の鍵
