Claude FA5 / GPT-5.6 Sol / Opus 実践比較

論文執筆の現場から見た3つの最強モデル — ワンダー佐藤が徹底解説
2026年7月23日

① Claude FA5: 総合トップの圧倒的性能

  • 3つのモデルの中で最も強力で一番使ってみていいという評価
  • 宇宙物理学の論文執筆で標準理論を超えるスコアを出すレベルの論理力を発揮
  • Arena AIのスコアでも圧倒的トップ。AIエージェントカテゴリでも首位
  • ただしトークン消費が激しい。Opusでもまあまあ激しいのに、FA5はさらに上を行く
FA5は「1発で大きいものを狙いたい」方向け。
速度重視なら別モデルが吉です。

② FA5の最大の壁: トークン消費問題

  • Opusでもトークン消費がまあまあ激しいのに、FA5はさらに激しい。プロプランではすぐにクレジットが尽きる
  • 時間を置いてからチャットに戻ると、また一から文章を読み込むという現象が発生する
  • 連続して作業できない。毎回読み込み直しになるので長文の処理がきつい
  • まもなくプロプランではクレジット制に移行。Maxプランならサブスクで使えるがコストが心配
トークン問題は深刻。
プロジェクト機能と工夫で
消費を抑える戦略が必須。

③ GPT-5.6 Sol: 慎重で批判的な頼れる相棒

  • FA5に比べて非常に遅い。「ド速で全然進んでいかない」というレベルの遅さ
  • ただしめちゃくちゃ慎重に、批判的にやってくれるので安心感がすごい
  • 独自の強み：難しい内容を画像にして説明してくれる。Claudeではできない機能
  • OpenAI公式は「SolならMythos級まで行く。トークン消費も少ない」と説明するが、体感は全然違う
不安が強い方、心配症の方には
Solがおすすめ。
安心感は抜群です。

④ Solの独自機能: 視覚化で理解を深める

  • ダークマターとダークエネルギーの相互作用を図解してくれる。これができるのはSolだけ
  • 複雑な概念を視覚的に表現できるのは、理解度が全く違う
  • OpenAIは「Solが何回かに渡ってやれば、Mythos級まで行くはず」と主張
  • ただし体感速度は遅い。「トークン消費が少ない」いう公式説明は疑わしいとワンダー佐藤が指摘
論文の概念を画像で説明してくれるのは
Solだけの特権機能。
これができるのは大きい！

⑤ Arena AIスコア: 意外な盲点

  • FA5は当然テキスト生成総合トップだが、実は2位との差はそれほど大きくない
  • 意外とClaude 4.6が良い。非常にエッセンシャルで、読むのが楽で疲れない
  • Claude 4.7も優秀。4.6より少し温度が高めで、言うことをよく聞いてくれる。4.8より4.7の方がいい場面が多い
  • Gemini Flashも意外といい。評判が良くないと思っていたが、実はしっかりした性能
スコアを見るとFA5と4.6の差は
そこまでない。
コスパで考えると4.6も魅力的。

⑥ エージェント力 vs インタラクティブ力のトレードオフ

  • AIエージェントでは4.8が強い。自律的に長く動かす用途なら4.8一択
  • インタラクティブでは4.8が12位に急降下。対話用途には向かない
  • エージェンティックな力とインタラクティブな力はトレードオフの関係にある
  • 長く走らせるほどアテンションを温存が必要。短いタスクならアテンションを集中できる。Solも同じ構造
エージェント用途なら4.8、
対話用途なら4.6か4.7。
使い方でモデルを選ぶのが正解。

⑦ Opusの意外な強み: 説明が分かりやすい

  • FA5の出力は難しすぎて何が書いてあるか分からないことがある。それがOpusだとだいぶ分かる
  • Opusはめちゃくちゃ良い。FA5が作った理論をOpusで見ると、説明が格段に分かりやすい
  • Opus 4.6のファクトチェックはFA5より全然強い。温度設定が低くてエッセンシャルだから安定感が抜群
  • 4.7は言うことをよく聞く上に文章の柔らかみもある。ブログ生成などに適している
FA5は「強すぎて理解不能」な面も。
Opusは人間のためのAI。
やっぱり使えるモデルです。

⑧ FA5 vs Sol: 使い分けの決定版

  • FA5派：1発ででかいものを狙いたい人。早くやりたい人。地道な作業が苦手な人
  • Sol派：失敗したくない人、失敗が怖い人。コツコツ地道にできる人。不安感が強い人
  • Sol独自の画像生成機能が必要な時はSol一択
  • Solは進捗率を見ながら3回でやる、5回でやると注文をつけると効果的
FA5とSolは性格が正反対。
自分の性格に合った方を
選ぶのが一番大事です。

⑨ 注意: モデル切り替えの罠

  • タスクを変えてモデルを切り替えると、「モデル切り替えますか？」と1回キャッシュを読み直すことに
  • 長文を読み込ませた状態で切り替えると、めちゃくちゃトークンを消費してしまう
  • 解決策：4.6、4.7、4.8のどれかに決めて、途中で切り替えない
  • ワンダー佐藤の選択：4.7を普段使いにしている。想像的なことをやるなら4.7、安定したことなら4.6
途中でモデル変えると
トークンがドパンと消える！
最初に決めて貫きましょう。

⑩ Claude Project: 長文対策の決め手

  • アプリ版で論文を書くと、前のソースの上に増築し続けてFA5の限界に達する
  • Project機能を使えば、読み込ませてもキャッシュ扱いなので消費を抑えられる
  • PDFではなくMDファイルを中心に入れるのがコツ。まだ13%程度の消費で余裕あり
  • アプリで生成した場合はMDファイルで出力してProjectに入れるワークフローが最適解
Project + MDファイルの組み合わせで
FA5でも長文がいける。
これ知らないと損します。

⑪ FA5の力を引き出す設定テクニック

  • Effortを最低に設定すると、読み込み時の消費を抑えられる可能性がある
  • 「深く考えて」のようなChain of Thoughtプロンプトで思考の深さを調整
  • 深さと生成の長さをプロンプトで決めることで、FA5の能力をコントロール可能
  • ワンダー佐藤はEffortとプロンプトの組み合わせを試しており、体感的に効いていると報告
Effort設定とCoTプロンプトで
FA5のエネルギー配分を制御。
地道に試行錯誤が大事。

⑫ Claude Code: コーディングだけじゃない

  • Claude Codeはコーディング専用だと思われがちだが、文章生成も全然できる
  • Projectの内容を要約させてから読み込ませることで、トークン消費を大幅に抑えられる
  • 万が一全部読み込んでもエッセンシャルな文章なのでダメージが少ない
  • コンパクト機能が使える。長文になったらうまく活用してトークン消費を下げる
Claude Codeで論文書いた人いる？
要約して読み込ませるのが
長文対策のポイントです。

⑬ Opus 4.7とMythosの繋がり

  • ワンダー佐藤が調べた結果、先行するMythosプレビューの技術が4.7で使われている可能性が高い
  • システムカード内には明記されていないが、MythosイコールFA5と考えていい可能性がある
  • Mythosの要素 → 4.7に継承 → さらに4.8にも継承されている可能性が高い
  • 4.6は非常にエッセンシャルで安定しているが、面白みはない。4.7がバランスの良い選択
4.7の裏にはMythosの影が。
「FA5的な要素」が
下位モデルにも継承されているかも。

⑭ 「憲法MD」で下位モデルを強化する方法

  • プロジェクトに「憲法」ファイルを入れると出力が非常に安定する。ルールMDでも何でもOK
  • FA5との会話ログを参考に、作業の進め方や出力の特徴を抽出して憲法に書き込む
  • 例文をどんどん入れることで、FA5に近い出力が4.7でもできるようになる
  • FA5がどういう風に作業を進めるかを見て、そのタスクの実行方法や考え方を抽象的に抽出するのがコツ
FA5のやり方を「憲法」に書いて
4.7に読み込ませる。
下位モデルを上位に引き上げる技！

⑮ 4.7でFA5レベルの成果を出せた事例

  • 4.7で論文を1つ作成し、それをFA5に評価させた
  • FA5の反応：「すでにそれを計算していて、私と同じことを考えていました。遜色ないロジック」
  • FA5に近いロジックの組み立て方が4.7でもできていると確認
  • 方向性は「FA5の特徴を観察して抽象化し、憲法MDとプロンプトで再現する」アプローチが正解
4.7でもFA5レベルが目指せる！
憲法 + 例文 + 会話ログ
この組み合わせが鍵。

⑯ 通常チャットの意外な使い勝手

  • 通常チャットは窓を2つ・3つ・4つと開けることができる。CurokやClaude Codeにはない柔軟性
  • モデル切り替えがささっとできる。切り替えても読み込みが発生しない（キャッシュ問題なし）
  • Opus 4.6でファクトチェックや検索をするのに最適。FA5より全然強い
  • 4.7でブログ生成、4.6でファクトチェック、と窓を開いて使い分けるのがスマート
通常チャットは「一番原子的」だけど
窓をたくさん開けて切り替える
使い方が実はすごく便利。

  • 量子力学の非局在性を「花は無数あるが根はただ1つ」というメタファーで解決
  • ダークマターとダークエネルギーを同じロトム場の氷と液体として統一的に説明
  • エネルギー交換係数の計算でπが自然に出現。パラメータ調整なしで標準理論を上回るBICスコアを達成
人間の直感 × AIの論理力

⑱ 最終まとめ: あなたに合った使い方を見つけよう

  • GPT-5.6 Solは慎重・批判的。画像生成が独自の強み。心配症に安心感

  • もうすぐOpus 5が出る。それまでのつなぎとして4.7がおすすめ。
