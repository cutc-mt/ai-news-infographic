① Claude Code単体の能力

Claude Codeを単体で使っている人は本来のポテンシャルの1割しか引き出せていない。単体でもプログラムのコード作成、ローカルファイル操作、ターミナル（黒画面）の自動操作ができるが、残り9割は連携次第で開花する。

② Claude Codeを「司令塔」にする

Claude Codeがレストランのシェフ兼オーナーに例えられる。各種連携手法が「材料の仕入れルート」に当たる。データベース接続、クラウドサービス操作、SNS投稿、メール送信 — すべてClaude Codeが司令塔として使いこなす。

③ 5つの連携パターン

Claude Codeが外部データとやり取りするための5つの方法。それぞれ使い勝手が異なり、状況に応じて使い分ける必要がある。

  • 標準コネクター — いつもの仕入れ業者（一番簡単）
  • MCP — ECサイト（少し設定すれば使える）
  • API — プロ向け業務用サイト（FAX注文）
  • CLI — マニア専門裏サイト（電話で依頼）
  • ブラウザ操作 — 自分で市場に直行（直接買出し）

④ レストランの例えで5パターンを覚える

難解な用語も例え話で一気に理解できる。コネクター＝いつもの業者（LINEで済む）、MCP＝楽天/Amazon（少し設定）、API＝FAXで発注（プロ向け）、CLI＝毎回電話（面倒だが強力）、ブラウザ＝自分で市場へ（最後の手段）。

⑤ 標準コネクター：ワンクリックで接続

最も簡単な連携方法。Claude Codeのカスタマイズ → コネクター → プラスで一覧が表示される。Google Drive、Slack、Notion、Figma、営業ツールなど多数対応。ワンクリック+アカウント認証で即座に使える。

  • カスタマイズ → コネクター → ＋
  • 使いたいツールを選択
  • アカウントで連携・認証
  • 即座にClaude Codeから操作可能！

⑥ コネクターの権限設定

コネクターごとに3つの権限レベルを設定できる。Notionの例では「読み取りは承認不要」「書き込みは毎回確認」「ページ作成はOK」「コメント・データベースはNG」など細かく制御可能。英語表記があればAIに聞けば日本語で解説してくれる。

✅ 承認不要 → ⚠️ 承認必要 → 🚫 禁止

⑦ MCP（Model Context Protocol）とは

AIエージェントが各種サービスと接続するための統一規格。USBの概念に似ている — 以前はN対Nの接続パターンが必要だったが、MCP対応なら1つのインターフェースで全AIと繋がる。2024年に登場後、1万〜2万のMCPサーバーが存在し、1億ダウンロードを突破。

⑧ カスタムMCPの追加方法

標準コネクターにないサービスもMCP対応なら追加可能。ElevenLabs（音声AI）、Airtable、Notion、Figmaなど。設定手順は、コネクター→カスタム追加→設定情報をAIに聞いてコピペ。ChatGPTの「考える」モードが活躍する。

  • カスタムコネクターを追加
  • 設定画面のスクショを撮影
  • AIに「MCP追加方法を教えて」と聞く
  • 指示通りに入力 → 認証完了

    ⑩ API連携：プロ向けの強力な方法

    APIはFAXで発注するようなイメージ。ネット注文（コネクター/MCP）で対応できないサービスでも、APIがあれば直接連携できる。設定は面倒だが、コネクターやMCPがないサービスへの唯一の接続手段になることも。

    コネクターもMCPもない場合でも、APIの知識があれば連携ができる。覚えておくと幅が格段に広がる。

    ⑪ API連携の実践手順

    APIキーを取得し、Claude Codeに環境変数として設定する。.envファイルやclaude mcp addコマンドで設定可能。APIドキュメントをClaudeに読み込ませれば、あとはAIが自動的に適切なAPIリクエストを生成してくれる。

    • APIキーを取得（各サービスの開発者画面）
    • 環境変数に設定（.env または claude mcp add）
    • APIドキュメントをClaudeに共有
    • Claudeが自動的にリクエスト生成

    ⑫ CLIでPCを直接操作

    CLIは「マニア専門の裏サイト」のようなイメージ。ターミナルコマンドを通じて、Claude CodeがPC上のあらゆるツールを直接操作できる。面倒だが他の方法では不可能な操作も可能で、まとめて一括処理にも強い。

    ⑯ Claude Code vs Claude Desktopの使い分け

    CLIベースのClaude Codeとデスクトップアプリ版は得意分野が異なる。Claude Codeはプログラミング・ファイル操作・自動化に強く、Claude Desktopはチャットベースの対話・軽いタスクに適している。用途に応じて使い分けることが重要。

    ⑰ 連携の注意点とセキュリティ

    外部サービスとの連携には権限の適切な設定が不可欠。不要な書き込み権限は与えず、読み取り専用を基本とする。APIキーの管理にも注意が必要で、.envファイルで安全に管理し、Gitにはコミットしないよう気をつける。

    全部一度に覚える必要はない。今使っているのがどの連携方法か意識することが重要。
